# <a name="sound-cones"></a>音效圓錐

描述導向音效音量的模型。

沒有方向的音效在所有方向的指定距離都有相同的波幅。 方向的音效會以方向 loudest。 描述導向音效音量的模型稱為「音效錐形」。 音效的錐形是由內部 (或內部) 錐形和 (外部) 錐形所組成。 外的錐形角度必須一律等於或大於內錐角度。

在內部錐形內的任何角度，音效的音量會設定為內部錐量。 這會考慮緩衝區的基本磁碟區、接聽程式的距離、接聽程式的方向（如果接聽程式有自己的錐形）等等。

在外部錐形以外的任何角度，標準磁片區是由應用程式所設定的因素所衰減。 外部錐形音量層級是以線性振幅 scaler 表示： 1.0 f 代表未套用至原始信號的衰減，0.5 f 代表6個資料庫的衰減，而 0.0 f 表示無回應。 也允許放大 (磁片區 > 1.0 f) ，且不會壓制。 有效的磁片區範圍實際上是 0.0 f 到 2.0 f。

內部和外部圓錐之間的範圍是從內部和外部的磁片區轉換到外部磁片區。 當角度增加時，音量會接近錐形的外部音量。

圓錐可能會影響磁片區以外的參數。 低通過篩選和送出行程的傳送層級可能也會受到影響，讓技術更有其更大的影響。 例如，在接聽程式上有一個錐形，可以指定接聽程式後方的所有音效都有一點 muffled，而且有稍微更高的待命至直接比率內容。 這些會提供更多的提示，指出音效位於使用者後方。 這可增強您的真實性。

下圖顯示音效圓錐的概念。

適當地設計音效圓錐可以為您的應用程式增加大幅效果。 例如，您可以在房間中央放置音效來源，將其方向設定為走廊中的開放門。 然後設定在錐形內的角度，使其延伸至門口的寬度、讓外的錐形更寬一點，最後將外部錐形的音量設定為聽不清楚。 在走廊時移動的接聽程式，只會在接近門口的時候開始聆聽音效。 當接聽程式傳入開啟門之前，會 loudest 音效。
