# <a name="about-system-event-notification-service"></a>關於系統事件通知服務

專為行動使用者所設計的應用程式需要一組獨特的連接功能和通知，在過去的個別應用程式中，必須在內部執行這些功能。 系統事件通知服務 (SENS) 現在會在作業系統中提供這些功能，為應用程式建立統一的連線和通知介面。 SENS、同步處理管理員和用戶端快取會合並，以提供基礎結構，以完全支援行動計算。

本節中的主題提供 SENS 的總覽：

-   [適用于 SENS 的行動計算設定](mobile-computing-configurations-for-sens.md)

    列出數個可受益于使用已啟用 SENS 之應用程式的行動電腦設定。

-   [SENS 的應用程式案例](application-scenarios-for-sens.md)

    列出數種可利用 SENS 功能獲益的應用程式類型。

-   [通知](notifications.md)

    列出 SENS 監視和分派的系統事件。

-   [SENS 架構](sens-architecture.md)

    SENS 的內部架構描述。
