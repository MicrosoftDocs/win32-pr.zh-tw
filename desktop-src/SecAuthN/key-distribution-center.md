# <a name="key-distribution-center"></a><span data-ttu-id="cbdfa-104">金鑰發佈中心</span><span class="sxs-lookup"><span data-stu-id="cbdfa-104">Key Distribution Center</span></span>

<span data-ttu-id="cbdfa-105">金鑰發佈中心 (KDC) 會實作為網域服務。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-105">The Key Distribution Center (KDC) is implemented as a domain service.</span></span> <span data-ttu-id="cbdfa-106">它會使用 Active Directory 作為其帳戶資料庫和通用類別目錄，以將參考導向其他網域中的 Kdc。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-106">It uses the Active Directory as its account database and the Global Catalog for directing referrals to KDCs in other domains.</span></span>

<span data-ttu-id="cbdfa-107">如同 [*Kerberos 通訊協定*](../secgloss/k-gly.md)的其他執行方式，KDC 是提供兩項服務的單一進程：</span><span class="sxs-lookup"><span data-stu-id="cbdfa-107">As in other implementations of the [*Kerberos protocol*](../secgloss/k-gly.md), the KDC is a single process that provides two services:</span></span>

-   <span data-ttu-id="cbdfa-108">驗證服務 (為) </span><span class="sxs-lookup"><span data-stu-id="cbdfa-108">Authentication Service (AS)</span></span>

    <span data-ttu-id="cbdfa-109">這項服務會發出票證授與票證 (Tgt) ，以連接到其本身網域或任何受信任網域中的票證授與服務。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-109">This service issues ticket-granting tickets (TGTs) for connection to the ticket-granting service in its own domain or in any trusted domain.</span></span> <span data-ttu-id="cbdfa-110">用戶端必須先從用戶端帳戶網域中的驗證服務要求 TGT，用戶端才能要求票證給另一部電腦。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-110">Before a client can ask for a ticket to another computer, it must request a TGT from the authentication service in the client's account domain.</span></span> <span data-ttu-id="cbdfa-111">驗證服務會傳回目的電腦網域中票證授與服務的 TGT。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-111">The authentication service returns a TGT for the ticket-granting service in the target computer's domain.</span></span> <span data-ttu-id="cbdfa-112">TGT 可以重複使用到到期為止，但第一次存取任何網域的票證授與服務時，一律需要前往用戶端帳戶網域中的驗證服務。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-112">The TGT can be reused until it expires, but the first access to any domain's ticket-granting service always requires a trip to the authentication service in the client's account domain.</span></span>

-   <span data-ttu-id="cbdfa-113">Ticket-Granting 服務 (TGS) </span><span class="sxs-lookup"><span data-stu-id="cbdfa-113">Ticket-Granting Service (TGS)</span></span>

    <span data-ttu-id="cbdfa-114">這項服務會發出票證，以連接到其本身網域中的電腦。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-114">This service issues tickets for connection to computers in its own domain.</span></span> <span data-ttu-id="cbdfa-115">當用戶端想要存取電腦時，會在目的電腦的網域中，與票證授與服務聯繫，出示 TGT，然後要求該電腦的票證。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-115">When clients want access to a computer, they contact the ticket-granting service in the target computer's domain, present a TGT, and ask for a ticket to the computer.</span></span> <span data-ttu-id="cbdfa-116">票證可以重複使用到到期為止，但第一次存取任何電腦一律需要前往目的電腦帳戶網域中的票證授與服務。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-116">The ticket can be reused until it expires, but the first access to any computer always requires a trip to the ticket-granting service in the target computer's account domain.</span></span>

<span data-ttu-id="cbdfa-117">網域的 KDC 位於網域控制站上，如同網域的 Active Directory。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-117">The KDC for a domain is located on a domain controller, as is the Active Directory for the domain.</span></span> <span data-ttu-id="cbdfa-118">網域控制站的 [*本地安全性授權*](../secgloss/l-gly.md) 單位會自動啟動這兩個服務 (lsa) 並在 lsa 的進程中執行。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-118">Both services are started automatically by the domain controller's [*Local Security Authority*](../secgloss/l-gly.md) (LSA) and run as part of the LSA's process.</span></span> <span data-ttu-id="cbdfa-119">這兩個服務都無法停止。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-119">Neither service can be stopped.</span></span> <span data-ttu-id="cbdfa-120">如果網路用戶端無法使用 KDC，則 Active Directory 也無法使用，而且網域控制站不再控制網域。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-120">If the KDC is unavailable to network clients, then the Active Directory is also unavailable—and the domain controller is no longer controlling the domain.</span></span> <span data-ttu-id="cbdfa-121">系統可讓每個網域擁有數個網域控制站（所有對等），以確保這些網域服務和其他網域服務的可用性。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-121">The system ensures availability of these and other domain services by allowing each domain to have several domain controllers, all peers.</span></span> <span data-ttu-id="cbdfa-122">任何網域控制站都可以接受向網域的 KDC 定址的驗證要求和票證授與要求。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-122">Any domain controller can accept authentication requests and ticket-granting requests addressed to the domain's KDC.</span></span>

<span data-ttu-id="cbdfa-123">在任何網域中，KDC 所使用的 [*安全性主體*](../secgloss/s-gly.md) 名稱是 "krbtgt"，如 [RFC 4120](https://www.ietf.org/rfc/rfc4120.txt)所指定。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-123">The [*security principal*](../secgloss/s-gly.md) name used by the KDC in any domain is "krbtgt", as specified by [RFC 4120](https://www.ietf.org/rfc/rfc4120.txt).</span></span> <span data-ttu-id="cbdfa-124">建立新網域時，會自動建立此安全性主體的帳戶。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-124">An account for this security principal is created automatically when a new domain is created.</span></span> <span data-ttu-id="cbdfa-125">無法刪除帳戶，也無法變更名稱。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-125">The account cannot be deleted, nor can the name be changed.</span></span> <span data-ttu-id="cbdfa-126">系統會在建立網域期間，自動將隨機密碼值指派給帳戶。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-126">A random password value is assigned to the account automatically by the system during the creation of the domain.</span></span> <span data-ttu-id="cbdfa-127">KDC 帳戶的密碼用來衍生密碼編譯金鑰，以加密和解密其所發出的 Tgt。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-127">The password for the KDC's account is used to derive a cryptographic key for encrypting and decrypting the TGTs that it issues.</span></span> <span data-ttu-id="cbdfa-128">網域信任帳戶的密碼用來衍生範圍內的金鑰，以加密參考票證。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-128">The password for a domain trust account is used to derive an inter-realm key for encrypting referral tickets.</span></span>

<span data-ttu-id="cbdfa-129">網域內的所有 KDC 實例都使用網域帳戶做為安全性主體 "krbtgt"。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-129">All instances of the KDC within a domain use the domain account for the security principal "krbtgt".</span></span> <span data-ttu-id="cbdfa-130">用戶端會將服務的主體名稱 "krbtgt" 和網域的名稱包含在內，以將訊息定址到網域的 KDC。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-130">Clients address messages to a domain's KDC by including both the service's principal name, "krbtgt", and the name of the domain.</span></span> <span data-ttu-id="cbdfa-131">這兩項資訊也會用於票證，以識別發行授權單位。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-131">Both items of information are also used in tickets to identify the issuing authority.</span></span> <span data-ttu-id="cbdfa-132">如需名稱表單和定址慣例的詳細資訊，請參閱 [RFC 4120](https://www.ietf.org/rfc/rfc4120.txt)。</span><span class="sxs-lookup"><span data-stu-id="cbdfa-132">For information about name forms and addressing conventions, see [RFC 4120](https://www.ietf.org/rfc/rfc4120.txt).</span></span>
