# <a name="quality-of-protection-and-ciphers"></a>保護品質和加密

保護品質可決定傳送和接收訊息時所使用的安全性層級。 Microsoft Digest 會根據用戶端和伺服器的需求來選取層級。

使用 Digest 作為 SASL 機制時，應用程式可以指定使用加密將訊息保持機密。 伺服器會指定它針對加密訊息所支援的密碼，而用戶端會從這份清單中選取一個密碼。

下列主題會詳細討論這些安全性機制：

-   [保護品質](quality-of-protection.md)
-   [加密](ciphers.md)
