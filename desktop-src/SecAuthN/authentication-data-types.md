# <a name="authentication-data-types"></a>驗證資料類型

## <a name="lsa-data-types"></a>LSA 資料類型

 (LSA) 的「 [*本地安全機構*](/windows/desktop/SecGloss/l-gly) 」會使用下列資料類型。



| 資料類型                                            | 描述                                                                                                              |
|------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| [**PLSA \_ 用戶端 \_ 要求**](plsa-client-request.md) | LSA 內部使用的不透明資料類型，可維護與個別要求相關的用戶端資訊。<br/> |



 

## <a name="sspi-data-types-and-values"></a>SSPI 資料類型和值

[SSPI](sspi.md) 會使用下列資料和列舉類型。



| 資料類型或值                         | Description                                                                      |
|--------------------------------------------|----------------------------------------------------------------------------------|
| [SSPI 控制碼](sspi-handles.md)           | SSPI 會處理這些控制碼的類型和指標。<br/>                      |
| [SSPI 狀態碼](sspi-status-codes.md) | 在 SSPI 應用程式中傳回的狀態碼。<br/>                           |
| [時間 戳](timestamp.md)                 | 定義的類型，用以保存安全性權杖的時間有效性資訊。<br/> |
