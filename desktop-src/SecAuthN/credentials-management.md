# <a name="credentials-management"></a>認證管理

針對 Windows 撰寫的開發人員可以使用認證管理 API 包括認證管理消費者介面 (UI) 功能來取得和管理認證資訊，例如使用者名稱和密碼。 這些函式會要求使用 Windows 帳戶資訊，而不是登入時所建立的認證。 當登入認證沒有應用程式所需的許可權時，通常會發生這類要求。

認證管理 UI 函式提供具有 Windows 使用者介面外觀的介面。 這些函數包括可自訂的選項，可將使用者的資訊新增至使用者的認證存放區。

下列主題提供有關認證管理 API 的詳細資訊：

-   [認證類型](kinds-of-credentials.md)
-   [使用者名稱格式](user-name-formats.md)

如需認證管理 API 中功能的相關資訊，請參閱 [驗證參考](authentication-reference.md)。
