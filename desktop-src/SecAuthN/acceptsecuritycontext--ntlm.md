## <a name="remarks"></a><span data-ttu-id="56933-205">備註</span><span class="sxs-lookup"><span data-stu-id="56933-205">Remarks</span></span>

<span data-ttu-id="56933-206">**AcceptSecurityCoNtext (ntlm)** 函數是 [**InitializeSecurityCoNtext (NTLM)**](initializesecuritycontext--ntlm.md)函數的伺服器對應項。</span><span class="sxs-lookup"><span data-stu-id="56933-206">The **AcceptSecurityContext (NTLM)** function is the server counterpart to the [**InitializeSecurityContext (NTLM)**](initializesecuritycontext--ntlm.md) function.</span></span>

<span data-ttu-id="56933-207">當伺服器收到來自用戶端的要求時，伺服器會使用 *fCoNtextReq* 參數來指定它需要的會話。</span><span class="sxs-lookup"><span data-stu-id="56933-207">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="56933-208">如此一來，伺服器可以指定用戶端必須能夠使用機密或 [*完整性*](../secgloss/i-gly.md)檢查的會話，而且它可以拒絕不符合該要求的用戶端。</span><span class="sxs-lookup"><span data-stu-id="56933-208">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="56933-209">或者，伺服器可能不需要任何專案，而且用戶端可以提供或需要的任何專案，都會在 *pfCoNtextAttr* 參數中傳回。</span><span class="sxs-lookup"><span data-stu-id="56933-209">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>

<span data-ttu-id="56933-210">針對支援多階段驗證的封裝，例如相互驗證，呼叫順序如下所示：</span><span class="sxs-lookup"><span data-stu-id="56933-210">For a package that supports multiple-leg authentication, such as mutual authentication, the calling sequence is as follows:</span></span>

1.  <span data-ttu-id="56933-211">用戶端會將權杖傳送至伺服器。</span><span class="sxs-lookup"><span data-stu-id="56933-211">The client transmits a token to the server.</span></span>
2.  <span data-ttu-id="56933-212">伺服器會在第一次呼叫 **AcceptSecurityCoNtext (NTLM)** ，它會產生回復權杖，然後傳送至用戶端。</span><span class="sxs-lookup"><span data-stu-id="56933-212">The server calls **AcceptSecurityContext (NTLM)** the first time, which generates a reply token that is then sent to the client.</span></span>
3.  <span data-ttu-id="56933-213">用戶端會接收權杖，並將它傳遞給 [**InitializeSecurityCoNtext (NTLM)**](initializesecuritycontext--ntlm.md)。</span><span class="sxs-lookup"><span data-stu-id="56933-213">The client receives the token and passes it to [**InitializeSecurityContext (NTLM)**](initializesecuritycontext--ntlm.md).</span></span> <span data-ttu-id="56933-214">如果 **InitializeSecurityCoNtext (NTLM)** 會傳回 SEC \_ E \_ OK，相互驗證已完成，而且可以開始安全的會話。</span><span class="sxs-lookup"><span data-stu-id="56933-214">If **InitializeSecurityContext (NTLM)** returns SEC\_E\_OK, mutual authentication has been completed and a secure session can begin.</span></span> <span data-ttu-id="56933-215">如果 **InitializeSecurityCoNtext (NTLM)** 傳回錯誤碼，相互驗證協商就會結束。</span><span class="sxs-lookup"><span data-stu-id="56933-215">If **InitializeSecurityContext (NTLM)** returns an error code, the mutual authentication negotiation ends.</span></span> <span data-ttu-id="56933-216">否則， **InitializeSecurityCoNtext (NTLM)** 所傳回的安全性權杖會傳送至用戶端，並重複步驟2和3。</span><span class="sxs-lookup"><span data-stu-id="56933-216">Otherwise, the security token returned by **InitializeSecurityContext (NTLM)** is sent to the client, and steps 2 and 3 are repeated.</span></span>
