# <a name="vml-introduction"></a>VML 簡介

本主題說明 VML，這是 Windows Internet Explorer 9 淘汰的功能。 依賴 VML 的網頁和應用程式應該遷移至 SVG 或其他廣泛支援的標準。

> [!Note]  
> 從2011年12月起，本主題已封存。 因此，它不會再主動維護。 如需詳細資訊，請參閱封存的 [內容](/previous-versions/windows/internet-explorer/ie-developer/)。 如需有關 Windows Internet Explorer 目前版本的資訊、建議和指引，請參閱[Internet Explorer 開發人員中心](https://msdn.microsoft.com/ie/)。

 

本檔提供 Microsoft Office 2000 隨附的 Vector Markup Language (VML) 之執行的完整詳細參考。 其他實施可能會有所不同。 如需 VML 的詳細資訊，請參閱 [vml 規格](https://www.w3.org/TR/NOTE-datetime.html) 。

VML 定義為一組 XML 元素，以及每個專案的對應屬性。 由於 **Shape** 元素是 VML 的基本組建區塊，請按一下 [這裡](shape-element--vml.md) 開始參考。

請注意，此參考包含數個範例和示範。 您必須安裝 Microsoft Internet Explorer 5.0 或更高版本，才能查看即時的 VML。

除了使用專案和屬性作為擴充 HTML 的 XML 標記的相關資訊之外，此參考還包含語法及範例，說明如何使用腳本和 Internet Explorer 5 或更新版本的檔物件模型功能。 使用具有 VML 的腳本可讓您即時建立元素，並即時操作屬性。

這項參考中的範例會使用 VBScript 和 JAVAscript。 如果您使用的指令碼語言與範例所示的不同，則您必須符合所有元素和屬性名稱的大小寫。 參考提供正確的腳本語法，適用于區分大小寫的語言，例如 JavaScript。 如果不確定是否有正確的字元大小寫，請使用物件瀏覽器 (例如 OLEView) 流覽 VGX.DLL 來判斷元素或屬性的確切大小寫。 請注意，當使用 VML 作為 XML 標記時，區分大小寫取決於基礎檔的檔案類型。 如果您使用 strict 模式！DOCTYPE、元素和屬性會區分大小寫。
