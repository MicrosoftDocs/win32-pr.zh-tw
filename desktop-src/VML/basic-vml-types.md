## <a name="introduction"></a>簡介

這份提案使用下表所列的少數基本類型。



| 類型                  | 元素           | 基本標記法 | 描述                                                                                          |
|-----------------------|-------------------|----------------------------|------------------------------------------------------------------------------------------------------|
| [boolean](#boolean)   |                   | 1位                      | 布林值： true 或 false。                                                                      |
| [分數](#fraction) |                   | 數位 2 6                 | 數值，以 2 6 () 65536 進行調整，並儲存為帶正負號的整數。                               |
| [座標](#ordinate) |                   | 30位整數加正負號   | 座標的一部分 (例如，在 ) 的路徑中，是由 coord 所定義的值。                                |
| [length](#length) (長度)     |                   | [動 車組](#length)             | 實體長度，例如線條的寬度或字型的大小。                                |
| [措施](#measure)   |                   | EMU 或數位 2 6          | 實體長度，包括數個裝置圖元或部分其他數量的部分。 |
| [角度](#angle)       |                   | 度數 2 6                | 角度;正面是順時針。                                                                     |
| [color](#color)       | [c](#color)       | 複雜                    | 允許衍生色彩的元素。                                                        |
| [字體](#font)         | [字體](#font)     | 複雜                    | 字型的描述。                                                                             |
| [點陣圖](#bitmap)     | [點陣圖](#bitmap) | href                       | 外部圖片檔案的參考。                                                             |
| [向量](#vector)     | [v](#vector)      | 複雜                    | 向量路徑的描述                                                                       |



 

「基本標記法」是最高精確度的標記法，此表示提案需要符合規範的實施才能維持;如果執行不能將資料表示為所需的精確度，資料將會遺失。 色彩、字型和向量類型會對應至本身具有結構的專案，而不是基本類型;不過，將這些專案視為此提案內的方式很方便。

每個非複雜的基本類型都有相同名稱的相關聯元素。 這些專案名稱會保留，而且可能無法用於延伸模組中的任何其他用途，即使在 onview = "skip" 擴充專案中使用也是如此。 基於這個原因，您可能會遇到未知 XML 的執行，以提供未知 XML 的有效內部儲存，只要這些值是以 "type" 元素括住。


```HTML
<new:tag>1.578</new:tag>
<new:tag><v:fraction>1.578</v:fraction></new:tag>
```



在上述的第一個範例中，字串 "1.578" 必須儲存為字元序列 (而不知道它是否為字串或數位) ;在第二個範例中，分數元素表示內容為數字，因此可以轉換成高精確度分數標記法。

複雜類型 (包括點陣圖) 有相關聯的專案名稱，可用來分隔值。 這會藉由確保更複雜的剖析工作與唯一的專案標記建立關聯，以簡化剖析。

## <a name="fraction"></a>分數


```HTML
fraction
<!entity % fraction "#pcdata" -- or nmtoken in an attribute -- >
```



所有數值 (也就是，在此提案中) 的量值量值，可以藉由 2 6 (65536) 進行調整來儲存為整數。 您可以使用此格式來指定數位，尾碼為 f 或不含尾碼的十進位數。 因此，下列假設元素代表相同的值。


```HTML
<fillwidth>0.25</fillwidth>
<fillwidth>16384f</fillwidth>
```



具有 f 尾碼的數量必須是整數;不允許使用小數數位。 結果整數必須可表示為32位的補數帶正負號的數位;因此，表示的有效範圍是 32768 (事實上，小於32768且大於或等於-32768。 ) 

需要符合規範的實值，才能保留以 f 值表示的值。 以十進位數表示的值可能會轉換成 f 值，並以該方式儲存。 允許應用程式在任何適當的單位中記錄內部產生的值;不過，從現有檔中讀取的值必須維持為完整原始有效位數，或必須轉換為 f 值。

如果執行無法這麼做，就必須警告使用者資料可能會遺失。  (在第一次遇到外部產生的資料時，可以接受發出這類警告一次的情況。 ) 

當 decimal 值轉換為 f 格式時，實值可能會使用任何算術舍入模式;不過，整數必須完全轉換成 f 格式。 建議將實作為轉換成減號無限大，且轉換一律是精確的。

## <a name="ordinate"></a>座標


```HTML
ordinate
<!entity % ordinate "#pcdata" -- or nmtoken in an attribute -- >
```



Coord 所建立的座標系統單位是某些名義型別，稱為「縱座標」。 這是長度的量值，但只會與 coord 所建立的矩形相對應。 任何類型為縱座標的值都會以 coord 的 *w* 和 *h* 值，以及用來在輸出裝置上建立實際度量的結果比例進行縮放。

符合規範的執行必須能夠處理最多30個位的座標值，以及正負號 (例如31位帶正負號的整數，而不是32位帶正負號的整數) 。 不過，建議您嘗試針對具有大約16位精確度的路徑和類似元素產生座標。 這可將不符合規範的實作為下溢或溢位的機率降到最低。

座標值一律為整數。 小點可能不會出現在類型座標的值中。 不可以將任何單位規範附加至縱座標類型的值。

## <a name="length"></a>長度


```HTML
length
<!entity % length "#pcdata" -- or nmtoken in an attribute -- >
```



長度是真實世界的測量，有時也是裝置圖元的度量。 建議您避免使用裝置圖元 (px) 。

所有標準 [CSS1](https://www.w3.org/pub/WWW/TR/REC-CSS1) 單位限定詞的長度都是允許的。 此外，也可以使用限定詞的 emu。 此辨識符號指的是單位（EMU (英文計量單位) --這是在電腦圖形中廣泛使用之測量數量的常見分母。 EMU 為 <sup>英寸</sup> /914400，也就是每英寸有914400個 EMU。 下表列出少數常見單位中的 EMUs 數目。



| EMUs 數目 | 每英寸的數目 | 每個毫米的數位 | 描述             |
|----------------|-----------------|-----------------------|-------------------------|
| 360            |                 | 0.01                  | Win32 HIMETRIC          |
| 12700          | 72              |                       | 時候                 |
| 635            | 1440            |                       | Win32 TWIP              |
| 762            | 1200            |                       | 高解析度印表機 |



 

不允許 EMUs 的小數數位。 任何量測都必須以32位帶正負號的整數 EMUs 形式來表示，這會將量測的大小限制為2348英寸--大約59計量或65碼。 因為量測一律會參考名義上螢幕或頁面大小輸出裝置上的轉譯大小，它們一律會在此範圍內。

不過，請注意，此標記法不適合實際的度量，而且會記錄在 (例如記錄路徑的真實世界大小) 必須使用其他標記法。

需要符合規範的實值，才能保留精確 EMUs 數目的值。 以任何其他方式表示的值可能會轉換成 EMU 值，並以該方式儲存。 允許應用程式在任何適當的單位中記錄內部產生的值;不過，從現有檔中讀取的值必須維持為完整原始有效位數，或必須轉換成 EMU 值。

如果執行無法這麼做，就必須警告使用者資料可能會遺失。  (在第一次遇到外部產生的資料時，可以接受發出這類警告一次的情況。 ) 

實際上，在此提案中，實體長度是用於相對較少的度量。 通常最重要的資料是路徑資料，而這會以個別圖形為基礎，以 coord 的方式在定義的座標系統中編碼。

### <a name="alternative-representations"></a>替代標記法

HTML 的標準長度表示是由 [CSS1](https://www.w3.org/pub/WWW/TR/REC-CSS1#length-units) 所定義。 相對單位（圖元除外）在此提案中使用長度的內容中沒有意義，而且不得使用。 如果檔記錄了預期的 (目標) 圖元大小，這會將圖元的轉譯定義為 EMU;否則，應該使用 [CSS1](https://www.w3.org/pub/WWW/TR/REC-CSS1#length-units) 所定義的 90 DPI 預設值。

除了 emu 以外，任何值都可以指定為小數小數。 當 decimal 值轉換成 EMU 時，實值可能會使用任何算術舍入模式。  (製作應用程式保證特定結果的唯一方式，就是以 emu 來指定。 ) 

如果長度值未指定任何單位規範，則實作為必須假設為 emu。

## <a name="measure"></a>measure


```HTML
measure
<!entity % measure "#pcdata" -- or nmtoken in an attribute -- >
```



量值是可能是 [長度](#length) 或 [分數](#fraction)的數量。 這與 HTML 和 CSS 長度的測量非常類似，在許多情況下，可能是實體量測或某些其他數量的百分比。 如果未指定任何單位規範，則值必須假設為小數小數 (因此，此行為會從分數繼承，而不是從長度繼承。 ) 

不同于長度，圖元值具有內容定義的意義，因此轉換成 emu 通常是不正確的。 有三種可能的基本標記法必須維持 (，也就是其中一個標記法無法轉換成另一個，而不會遺失任何資訊) 。

1.  小數值應以 [小數](#fraction) 格式進行維護， ("f" 值) 。
2.  實體長度應以 EMU 維護。
3.  圖元值應該維持為整數的圖元數。

不允許小數的圖元數。

## <a name="angle"></a>角度


```HTML
angle
<!entity % angle "#pcdata" -- or nmtoken in an attribute -- >
```



角度的基本標記法是由 2 6 (65536) 所乘以的程度，並儲存為整數。 因為座標空間會反轉 (正 y 軸會) 關閉，所以順時針角度是正面的。 需要符合規範的實，才能保留這類值的完整精確度。

您可以使用任何角度的範圍，並允許 normalise 角度 (例如-180 到 + 180 或0至 360 ) 。 執行不需要一致;不過，角度的整數標記法不能超過32位帶正負號整數的範圍。

尾碼 fd 會用來識別此標記法的角度 (分數) 。 請注意，這會從 f 尾碼分辨成維度分數，雖然可以使用相同的算術來支援它。 角度值的預設值為簡單度，也就是無比例的值。 這也可能會使用後置字元 "" (角度符號) ;不過，使用這項功能取決於是否有適當的檔編碼--因此，後置字元度數也會定義為平均度數。 一組完整的可能尾碼，如下所示。



| 後置詞       | 轉換因數 | 註解                               |
|--------------|-------------------|----------------------------------------|
| Fd           | 1                 | 高精確度內部標記法 |
| 無、度、   | 65536             | Degrees                                |
| rad          | 65536pi/180       | Radians                                |



 

### <a name="alternative-representations"></a>替代標記法

縮放轉換的不連續是45的奇數倍數。 因此，將任何不精確的數量轉換成妥善定義非常重要。 基於這個理由，轉換的算術定義為舍入減去無限大。

因為這在某些實作為中可能很困難或無法保證，所以使用下列內容會定義為層級3功能：

1.  任何小數度值。
2.  任何弧度值

因此，值限定的 fd 和不合格的整數值或整數值的限定度，必須完全由符合層級0的實作為處理;其他值則不需要。 強烈建議任何實值都能處理從小數度值到已縮放 (fd) 度值的轉換。 這可以在沒有浮點支援的情況下完成。

更嚴格的轉換需求區分 fd 與 f。

## <a name="color"></a>color


```HTML
c
<!element c  %color;>
```



色彩是新式電腦圖形中不可或缺的一部分。 提案會使用已建立的方法來指定固定色彩。 不過，圖表中的色彩很少是簡單的靜態色彩;它們通常衍生自圖表中的其他元素。 大部分的資訊都是應用程式特定的，因此，這個提案提供兩種非常基本的機制來指定這類行為：

1.  色彩可能會衍生自相同形狀中的另一種色彩。
2.  定義少量的算數運算來衍生或修改色彩。

原型圖形機制會藉由允許從可繼承的色彩來定義原型來補充此項。


```HTML
color
<!entity % color "( %color-unit; | pure | %color.adjustment; )*">
```



色彩值是 [CSS1 色彩定義](https://www.w3.org/pub/WWW/TR/REC-CSS1#color-units) 的超集合。 擴充功能可讓您從圖形內的其他色彩，或從使用 CSS1 所定義的整體色彩配置來判斷 RGB 色彩值。

如果元素的值定義為色彩，則元素的 *內容* 會透過對應 RGB 色彩上的算數運算選擇性地修改的單一色彩標記，來定義色彩值。

### <a name="scheme-colors"></a>配置色彩

配置所參考的配置色彩是在檔層級定義，並使用名稱屬性為「主題色彩配置」的中繼標記。


```HTML
<meta name="Theme Color Scheme"
  content="rrggbb, rrggbb, rrggbb, rrggbb, rrggbb, rrggbb, rrggbb, rrggbb">
```



此標記最多可定義八種配置色彩。 未定義的色彩應該預設為黑色。 配置色彩可讓您只需變更主題色彩配置內容，就能變更整個檔所用的色彩配置。 為了確保從不同撰寫應用程式匯入的圖形會一致地使用配置色彩，會定義下列解釋。 「使用方式」是目的的簡短描述，而「描述」資料行會提供其他詳細資料。



| 索引 | 名稱              | 使用量                         | 描述                                                                                                              |
|-------|-------------------|-------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| 0     | 配置。背景 | 背景                    | 用於圖形背景的色彩 (，通常是整個頁面) 。                                    |
| 1     | 配置。文字       | 文字和線條                | 附加至圖形和標準線條色彩的文字色彩。                                                      |
| 2     | 配置：陰影     | 陰影                       | 標準陰影色彩：通常用於圖形陰影的色彩。                                                     |
| 3     | 配置. 標題      | 標題文字                    | 用於標題或標題文字的色彩。                                                                              |
| 4     | 配置：填滿       | 充滿                         | 標準填滿色彩：通常用來填滿圖形的色彩。                                                          |
| 5     | 配置：輔色     | 口音                        | 一般「反白顯示」色彩，用來強調圖形的重要元素。                                             |
| 6     | 配置：超連結  | 輔色和超連結          | 反白顯示用於超連結的色彩。 可用於其他用途，其中的色彩代表其他資訊的連結。 |
| 7     | 配置。遵循   | 輔色和後續的超連結 | 醒目提示已跟隨超連結的色彩;也適用于「回溯」連結。                                         |



 

配置色彩可依名稱或索引來參考，因此配置. 填滿和配置 (4) 的色彩相同。

如果未指定色彩，配置色彩不會參與預設配置。 無論配置色彩4的色彩為何，未指定的填滿色彩都必須被視為白色。 "輔色" 色彩應該與背景 (0) 和填滿 (4) 色彩相同，且文字和標題文字色彩應該具有相同的屬性。 其中一個標準技巧是讓強調色彩和標準文字 uncolored (通常是黑色或白色) 。

### <a name="color-adjustments"></a>色彩調整

基底色彩可透過 RGB 值上的算數運算來調整。 這些作業會使用色彩值內的其他元素來定義。 這類調整只有在套用至衍生自其他元素的色彩時才有用。 對固定色彩值指定這類調整是有效的;不過，您可以將此實值縮減為對應的 RGB 值，並儲存該值，而不是使用原始值。

本節所述的所有色彩調整功能都是層級1的功能。


```HTML
color.adjustment
<!entity % color.adjustment  -- change to make to a color --
  "( %color.op; )?, ( %color.adj; )*"
>

color.op
<!entity % color.op          -- arithmetic operation --
  "( darken | lighten | add | subtract | reverseSubtract |
  blackWhite )"
>
<!element   darken           %color.parameter;>
<!element   lighten          %color.parameter;>
<!element   add              %color.parameter;>
<!element   subtract         %color.parameter;>
<!element   reversesubtract  %color.parameter;>
<!element   blackwhite       %color.parameter;>

color.parameter
<!entity % color.parameter  "#pcdata" >

color.adj
<!entity % color.adj          -- additional adjustment to color --
  "invert | invert128 | gray"
>
<!element   invert       empty>
<!element   INVERT128    empty>
<!element   gray         empty>
```



前六個作業的參數是範圍0到255的單一整數數值。 調整會在 3x8bit RGB 值上執行，如下所示：

1.  如果 <gray/> 指定了，則 RGB 值會由 yyy 取代，其中 y 是從709下的 sRGB 值計算而來的亮度 (y ' ) 值。 這項計算如下：
    ```HTML
    y = 0 2125xr + 0 7154xg + 0 0721xb
    ```

    

2.  如果指定了色彩，則會根據下表分別調整每個元件。 c 是元件值，而 p 是 color. 參數值。

    | 作業       | Formula                            |
    |-----------------|------------------------------------|
    | 變 暗          | c： = cxp/255                       |
    | 減輕         | c： = 255- (255-c) xp/255           |
    | add             | c： = c + p                         |
    | 減        | c： = c-p                         |
    | reversesubtract | c： = p-c                         |
    | blackwhite      | if c < p thenc： = 0elsec： = 255 |

    

     

    在每個案例中，如果計算的元件值 c 超過255，則會使用255，如果小於0，則會使用0。

3.  如果 <INVERT128/> 指定了，則會根據元件是否小於128，將值128減去或加入至每個元件。
    ```HTML
    if c < 128
        then
       c := c + 128
    else
       c := c - 128
    ```

    

4.  如果 <invert/> 指定了，則會將每個元件取代為255減去元件的值。
    ```HTML
    c := 255-c
    ```

## <a name="bitmap"></a>點陣圖


```HTML
bitmap
<!element   bitmap   empty>
<!attlist   bitmap
   XML-link    cdata               #fixed "simple"
   href        cdata               #REQUIRED
   title       cdata               #implied
   behavior    cdata               #implied
   show        (embed|replace|new) #fixed "embed"
   inline      (true|false)        #fixed "true"
   actuate     (auto|user)         #fixed "auto"
>
```



點陣圖元素可讓您參考非線上圖片資源 (通常是要包含在圖形中的點陣圖) 。

點陣圖是層級1的功能。

行為屬性可用來指出點陣圖應該如何由編輯應用程式處理。 值可以是下列其中一個或兩個標記。



| Token    | 描述                                                                                                                                                                                                                                                                             |
|----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 分離 | 將點陣圖標記為個別的實體，不應視為檔的整數部分。 點陣圖不應與檔一起保存。 如果複製檔，則不應複製點陣圖;如果移動檔，則不應移動點陣圖。 |
| 原始 | Title 屬性會將點陣圖的原始位置識別為 URL;否則，就不會指定 title 的意義。                                                                                                                                                          |



 

這些值都是預期行為的提示。 個別的選項是指 href 所參考資料的行為。 如果同時指定了個別和原始的，應用程式應該會忽略 href URI，並從原始資料重新產生點陣圖。 如果只指定原始的，應用程式應該使用 href URI 來尋找點陣圖，但是可讓使用者選擇重新產生它。

將 href URI 和 title 屬性設為相同的 (詞法) 值是有效的，這適用于參考的點陣圖未「儲存于」檔時。 它的目的是要 (但不是) 必要的，因為 href 用於檔本身的點陣圖複本，如果刪除參考圖形，則可能會遭到刪除，而且該標題會用來表示共用副本。 因此，如果兩者都包含相同的值，就不會有檔特定的複本。

如果應用程式不符合 XML 資料的實際儲存模型，則可能會忽略該提示。

### <a name="picture-file-formats"></a>圖片檔案格式

在此提案的內容中，外部資料一定是用來產生點陣圖的點陣圖或檔案。 在轉譯層級0，不需要支援外部點陣圖格式，只會以純色填滿路徑。 若要轉譯一組完整的轉譯層級1填滿，必須支援點陣圖。 轉譯層級1包含 (僅) 下列格式：

1.  JFIF，也就是在具有 JFIF 標頭的檔案中內嵌的 ISO/IEC 10918 格式資料， (可以在 SOI maker) 之後被視為特定 APP0 標記，並包括 (只) IJG v6 程式碼所支援的 JPEG 格式範圍。
2.  .Png （如 PNG 版本1.0 規格所定義）。

轉譯層級2也包含下列支援：

-   GIF，如1987中 CompuServ 所發佈的 GIF 規格所定義 (通常稱為 "GIF87a" ) 。 此層級也必須支援 GIF89a，受限於資料不能包含任何需要解讀的延伸區塊，以顯示圖形控制項 extensionswithouta 在使用者輸入或延遲時間的需求以外的點陣圖。 這可包含批註，而不是純文字延伸。 應用程式可能會將應用程式插入 (0x21、0xFF) 擴充功能，但使用此提案的術語時，這些必須只包含編輯而非轉譯的資料。

圖形中使用的任何其他資料格式都會強制圖形至少為編輯層級3，而且可能會轉譯層級 3 (如果需要資料才能轉譯圖形) 。 建議應用程式發佈其所支援的格式。 例如，Microsoft Office 原本就支援下列其他格式，因此可能會以下列格式寫入編輯資料：

1.  WMF--Windows 的中繼檔 (Win 3.1 格式) 
2.  EMF--Windows "增強" 中繼檔 (Win32 格式) 
3.  PICT--Mac OS QuickDraw PICT 檔案 (所有版本，但不含 QuickTime 記錄或其他延伸模組) 
4.  BMP--Windows 點陣圖檔案格式、"os/2" (BITMAPCORE) 、BITMAPINFO、BITMAPV4 和 BITMAPV5 格式
