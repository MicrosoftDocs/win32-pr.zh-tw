## <a name="web-restrictions"></a>Web 限制

-   使用免費動態評等服務的現成 Web 內容篩選器。 可個別設定每個受控制的使用者。
-   會在啟用時篩選所有 HTTP 流量，如果遭到封鎖，則會傳回自訂格式的頁面。 可接受所有主要瀏覽器的功能。 藉由監視頁面的所有 HTTP Get 和 Post 要求，可讓個別網頁元件遭到封鎖。
-   您可以使用允許或封鎖 URL 清單、簡化分級分類預設值的使用者介面，或 12 + 類別的 ActiveX 控制項，以及封鎖檔案下載的原則，來進行高度可設定。

> [!Note]  
> 實際的檔案下載封鎖需要應用程式執行限制，並遵循所提供的設定。

 

-   實作為 Windows 篩選平台 (WFP) 驅動程式與在使用者會話中執行的家庭安全監視程式進行通訊。 執行已從多層式服務提供者變更 (LSP) 篩選至 Windows 篩選平台 (WFP) 驅動程式與使用者會話中執行的家庭安全監視程式進行通訊。

    **Windows 7 和 Windows Vista：** 實作為多層式服務提供者 (LSP) 篩選與低許可權服務進行通訊，以進行整體的管理和監視。 當停用時，篩選會保留在 LSP 鏈中，但會通過所有流量。 只有列出的作業系統才支援此執行。

-   應用程式可能會提供使用者介面來顯示部分頁面封鎖，並接受原則來封鎖檔案下載。 他們也可以叫用 API，讓使用者可以要求許可權來查看已封鎖的頁面。 此系統管理覆寫會叫用小型應用程式，以顯示已嘗試的 Url，並要求認證以允許它們。
-   當應用程式需要註冊來略過篩選，或一律允許特定的 Url 時，會提供 API 給您的特殊情況。
-   由於一次只能執行一個 Web 內容篩選，因此協力廠商可能會註冊成為作用中篩選，並停用內建篩選準則。 此擴充性結合了在 UI 中顯示協力廠商篩選的功能，可讓您進行簡單的取代。 篩選準則必須在卸載時移除其註冊。

## <a name="time-limits"></a>時間限制

-   內建的時間限制已增強，可讓您控制電腦可用的總時間量 (時間額度) 除了能夠控制電腦可用的時間之外，還可以在舊版 Windows 中使用 (curfew) 。
-   適用于 curfew 的 UI 可讓您在一周的每一天進行獨立控制，並提供半小時的資料細微性
-   時間額度的 UI 可讓您在一周中的每一天進行工作日和週末的控制，或使用15分鐘的資料細微性進行獨立控制
-   在封鎖的時間週期內，快速使用者切換 (FUS) 機制不再用來強制鎖定或封鎖受控制使用者的登入。 切換至不同的桌面會用於這些用途。 請注意，FUS 會在鎖定時保留使用者的狀態。

## <a name="games"></a>遊戲

-   與遊戲瀏覽器搭配運作。
-   可讓系統管理員控制哪些遊戲可透過豐富的 UI 來播放，以選取遊戲分級系統和層級 (加描述項（如果有的話）) ，以及/或明確允許或封鎖標題。
-   遊戲分級的中繼資料是以下列兩種方式之一來取得：
    -   支援的標題會將自己的遊戲定義檔案部署 (GDFs) 。
    -   現成的資料庫涵蓋大約2000個舊版的標題。
-   有三種機制可用於強制執行：
    -   拒絕控制使用者對遊戲資料夾的寫入權限 Acl。
    -   使用應用程式相容性填充碼技術處理舊版標題的程式。
    -   由支援的標題進行的自我檢查 API 使用，以封鎖執行。
-   建議您知道上一節所述的時間限制事件。
-   GDFs 和遊戲瀏覽器的檔主要是透過 DirectX SDK 版本來提供。

## <a name="allow-and-block-specific-programs"></a>允許和封鎖特定程式

-   也稱為 (GAR) 的一般應用程式限制。
-   預設為關閉狀態。 如果開啟，它只允許受控制的使用者執行系統管理員核准的應用程式，並具有合理的例外狀況。
-   UI 提供具有對應路徑的程式名稱清單，每個名稱都有一個 [允許] 核取方塊。 也會提供 [流覽] 按鈕。
-   使用軟體限制原則來實行 (SRP) ，也稱為更安全：
    -   防止 (USB 金鑰、磁片等) 的所有媒體執行。
    -   使用路徑規則來指定允許執行的程式。
    -   NTFS ACL 寫入權限會從受控制的使用者可執行檔任何作業撤銷。
    -   如果已封鎖，之後又覆寫為 [允許]，則必須手動 relaunched 應用程式。
-   例外狀況包括：
    -   要運作的基本 Windows 子集所需的所有二進位檔。
    -   使用 API 註冊的所有可執行檔都可供指定的使用者使用。
    -   遊戲限制下所指定為允許的遊戲。
-   請注意，當 GAR 為 on 時，會針對使用者的設計封鎖 RunAs 命令。
