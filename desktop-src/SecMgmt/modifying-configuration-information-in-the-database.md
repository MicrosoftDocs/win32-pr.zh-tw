# <a name="modifying-configuration-information-in-the-database"></a>修改資料庫中的設定資訊

附件嵌入式管理單元延伸模組必須讓使用者透過其分析節點變更設定資訊。 這可讓使用者進行變更，然後在一段時間後重新套用設定。 要這麼做的邏輯應該與修改設定檔中的資訊相同。 所做的變更將會在安全性資料庫中儲存的設定生效。
