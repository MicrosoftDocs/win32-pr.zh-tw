# <a name="private-data-object"></a>私用資料物件

在每個系統上都可以使用有限數目的私用資料物件，目的是要以受保護、加密的方式儲存資訊。

私用資料物件主要是用來支援伺服器帳戶密碼的儲存。 這適用于在特定帳戶中執行的伺服器。 伺服器帳戶的密碼是私用資料，應該受到保護，但需要用來登入伺服器。

私用資料物件可能是一般用途，也可能是以下三種特製化類型的其中一種：本機、全域和電腦。

本機私用資料物件只能從儲存物件的電腦讀取本機。 嘗試從遠端讀取它們會導致狀態 \_ 拒絕存取 \_ 錯誤。 本機私用資料物件的索引鍵名稱開頭為 "L $" 前置詞。 除了您所建立的本機私用物件之外，作業系統也會定義下列本機私用物件： $machine. acc、SAC、SAI 和 SANSC。

全域私用資料物件是全域的，如果它們是在網域控制站上建立的，就會自動複寫到該網域中的所有網域控制站。 換句話說，該網域中的每個網域控制站都可以存取全域私用資料物件所包含的值。 相反地，在不是網域控制站的系統上建立的全域私用資料物件，以及 nonglobal 的私用資料物件則不會進行複寫。 全域私用資料物件的索引鍵名稱是以 "G $" 開頭。

只有作業系統才能存取電腦私用資料物件。 這些物件的索引鍵名稱以 "M $" 開頭。

**注意**  您可以設定，但無法取出電腦私用資料物件。

除了這些前置詞之外，下列值也會指出本機或電腦物件。 為了回溯相容性支援這些值，當您建立新的本機或電腦物件時，不應使用這些值。 本機私用資料物件的索引鍵名稱也可以開頭為 "RasDialParms" 或 "RasCredentials"。 電腦物件的索引鍵名稱可能也會以 "NL $" 或 " \_ sc \_ " 開頭。

非上述任何特製化類型中的私用資料物件使用不是以前置詞開頭的索引鍵名稱。 這些物件不會進行複寫，而且可以在本機或遠端讀取應用程式。
