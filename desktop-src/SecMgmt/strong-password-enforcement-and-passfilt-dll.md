# <a name="strong-password-enforcement-and-passfiltdll"></a>強式密碼強制執行和 Passfilt.dll

您可以使用系統管理工具來啟用強式密碼強制。 如果已啟用系統管理原則，則在建立或變更密碼時，密碼必須符合下列最低需求：

-   密碼可能不會包含使用者的 samAccountName (帳戶名稱) 值或整個 displayName (完整名稱值) 。 這兩種檢查不區分大小寫。
-   SamAccountName 會完整地簽入，以判斷它是否為密碼的一部分。 如果 samAccountName 長度少於三個字元，則會略過此檢查。
-   DisplayName 會針對分隔符號進行剖析：逗號、句號、連字號或連字號、底線、空格、字型大小及定位字元。 如果找到任何這些分隔符號，則會分割 displayName，並確認 (token) 的所有剖析區段都不會包含在密碼中。 系統會忽略小於三個字元的權杖，而且不會檢查標記的子字串。 例如，名稱 "Erin M. Hagens" 會分割成三個標記： "Erin"、"M" 和 "Hagens"。 因為第二個權杖的長度只是一個字元，所以會被忽略。 因此，此使用者的密碼不能包含 "erin" 或 "hagens" 做為密碼中任何位置的子字串。
-   密碼必須包含下列五個類別中的三個字元。



| 字元類別                                                                                                                                                      | 範例                               |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------|
| 歐洲語言的大寫字母 (A 至 Z，包含變音符號、希臘文和斯拉夫文字元) <br/>                                                     | A、B、C、Z<br/>                |
| 歐洲語言的小寫字母 (a 至 z、銳角、變音符號、希臘文及斯拉夫文字元) <br/>                                            | a、b、c、z<br/>                |
| 以 10 為基底的數字 (0 到 9)。<br/>                                                                                                                                   | 0、1、2、9<br/>                |
|  (特殊字元的非英數位元) <br/>                                                                                                               | $,!,%，^， () {} \[ \] ;： <>？<br/> |
| 分類為字母字元但不是大寫或小寫的任何 Unicode 字元。 這包括亞洲語言的 Unicode 字元。<br/> |                                        |



 

**若要啟用強式密碼強制**

1.  從管理主控台找出 [ **本機安全性原則**]。
2.  選取 [ **帳戶原則**]，然後選取 [ **密碼原則**]。
3.  [啟用 **密碼必須符合複雜性需求** ] 設定。

> [!Note]  
> 指定的字元只能滿足一個類別目錄。 [GetStringTypeW](/windows/win32/api/stringapiset/nf-stringapiset-getstringtypew)函式可用來測試密碼中的每個字元是否為大寫、小寫或英數位元。
