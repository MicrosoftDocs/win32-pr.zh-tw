# <a name="security-management"></a>安全性管理

## <a name="purpose"></a>目的

安全性管理技術可以用來管理 [*本地安全機構*](/windows/desktop/SecGloss/l-gly) (LSA) 原則和 [*密碼篩選*](/windows/desktop/SecGloss/p-gly) 原則、查詢來自外部來源的程式能力，以及擴充安全性設定工具功能的服務附件。

Microsoft 安全性管理技術包括 LSA 原則 API、密碼篩選 API、更安全的 API，以及服務安全性附件 API。 這些技術可讓軟體發展人員建立應用程式來管理系統和應用程式。

## <a name="developer-audience"></a>開發人員對象

LSA 原則 API、密碼篩選 API、更安全的 API 和服務安全性附件 API 都是供應用程式開發人員使用，讓系統管理員能夠管理和保護其系統。 開發人員應該熟悉 C 和 c + + 程式設計語言，以及使用以 Windows 為基礎的程式設計。
