# <a name="error-codes-in-com"></a><span data-ttu-id="359c9-103">COM 中的錯誤碼</span><span class="sxs-lookup"><span data-stu-id="359c9-103">Error Codes in COM</span></span>

<span data-ttu-id="359c9-104">為了表示成功或失敗，COM 方法和函式會傳回 **HRESULT** 類型的值。</span><span class="sxs-lookup"><span data-stu-id="359c9-104">To indicate success or failure, COM methods and functions return a value of type **HRESULT**.</span></span> <span data-ttu-id="359c9-105">**HRESULT** 是32位的整數。</span><span class="sxs-lookup"><span data-stu-id="359c9-105">An **HRESULT** is a 32-bit integer.</span></span> <span data-ttu-id="359c9-106">**HRESULT** 的高序位位表示成功或失敗。</span><span class="sxs-lookup"><span data-stu-id="359c9-106">The high-order bit of the **HRESULT** signals success or failure.</span></span> <span data-ttu-id="359c9-107">零 (0) 表示成功，1表示失敗。</span><span class="sxs-lookup"><span data-stu-id="359c9-107">Zero (0) indicates success and 1 indicates failure.</span></span>

<span data-ttu-id="359c9-108">這會產生下列數值範圍：</span><span class="sxs-lookup"><span data-stu-id="359c9-108">This produces the following numeric ranges:</span></span>

-   <span data-ttu-id="359c9-109">成功碼：0x0 –0x7FFFFFFF。</span><span class="sxs-lookup"><span data-stu-id="359c9-109">Success codes: 0x0–0x7FFFFFFF.</span></span>
-   <span data-ttu-id="359c9-110">錯誤碼：0x80000000 –0xFFFFFFFF。</span><span class="sxs-lookup"><span data-stu-id="359c9-110">Error codes: 0x80000000–0xFFFFFFFF.</span></span>

<span data-ttu-id="359c9-111">少數的 COM 方法不會傳回 **HRESULT** 值。</span><span class="sxs-lookup"><span data-stu-id="359c9-111">A small number of COM methods do not return an **HRESULT** value.</span></span> <span data-ttu-id="359c9-112">例如， [**AddRef**](/windows/desktop/api/unknwn/nf-unknwn-iunknown-addref) 和 [**Release**](/windows/desktop/api/unknwn/nf-unknwn-iunknown-release) 方法會傳回不帶正負號的 long 值。</span><span class="sxs-lookup"><span data-stu-id="359c9-112">For example, the [**AddRef**](/windows/desktop/api/unknwn/nf-unknwn-iunknown-addref) and [**Release**](/windows/desktop/api/unknwn/nf-unknwn-iunknown-release) methods return unsigned long values.</span></span> <span data-ttu-id="359c9-113">但是傳回錯誤碼的每個 COM 方法都會傳回 **HRESULT** 值。</span><span class="sxs-lookup"><span data-stu-id="359c9-113">But every COM method that returns an error code does so by returning an **HRESULT** value.</span></span>

<span data-ttu-id="359c9-114">若要檢查 COM 方法是否成功，請檢查傳回之 **HRESULT** 的高序位位。</span><span class="sxs-lookup"><span data-stu-id="359c9-114">To check whether a COM method succeeds, examine the high-order bit of the returned **HRESULT**.</span></span> <span data-ttu-id="359c9-115">Windows SDK 標頭提供兩種可簡化此作業的宏： [**成功**](/windows/desktop/api/winerror/nf-winerror-succeeded) 的宏和 [**失敗**](/windows/desktop/api/winerror/nf-winerror-failed) 的宏。</span><span class="sxs-lookup"><span data-stu-id="359c9-115">The Windows SDK headers provide two macros that make this easier: the [**SUCCEEDED**](/windows/desktop/api/winerror/nf-winerror-succeeded) macro and the [**FAILED**](/windows/desktop/api/winerror/nf-winerror-failed) macro.</span></span> <span data-ttu-id="359c9-116">如果 **HRESULT** 是成功碼，**成功** 的宏會傳回 **TRUE** ，如果是錯誤碼，則會傳回 **FALSE** 。</span><span class="sxs-lookup"><span data-stu-id="359c9-116">The **SUCCEEDED** macro returns **TRUE** if an **HRESULT** is a success code and **FALSE** if it is an error code.</span></span> <span data-ttu-id="359c9-117">下列範例會檢查 [**CoInitializeEx**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializeex) 是否成功。</span><span class="sxs-lookup"><span data-stu-id="359c9-117">The following example checks whether [**CoInitializeEx**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializeex) succeeds.</span></span>


```C++
HRESULT hr = CoInitializeEx(NULL, COINIT_APARTMENTTHREADED | 
    COINIT_DISABLE_OLE1DDE);

if (SUCCEEDED(hr))
{
    // The function succeeded.
}
else
{
    // Handle the error.
}
```



<span data-ttu-id="359c9-118">有時候，測試反向條件會比較方便。</span><span class="sxs-lookup"><span data-stu-id="359c9-118">Sometimes it is more convenient to test the inverse condition.</span></span> <span data-ttu-id="359c9-119">[**失敗**](/windows/desktop/api/winerror/nf-winerror-failed)的宏會與 [**成功**](/windows/desktop/api/winerror/nf-winerror-succeeded)的相反。</span><span class="sxs-lookup"><span data-stu-id="359c9-119">The [**FAILED**](/windows/desktop/api/winerror/nf-winerror-failed) macro does the opposite of [**SUCCEEDED**](/windows/desktop/api/winerror/nf-winerror-succeeded).</span></span> <span data-ttu-id="359c9-120">它會針對錯誤碼傳回 **TRUE** ，並針對成功代碼傳回 **FALSE** 。</span><span class="sxs-lookup"><span data-stu-id="359c9-120">It returns **TRUE** for an error code and **FALSE** for a success code.</span></span>


```C++
HRESULT hr = CoInitializeEx(NULL, COINIT_APARTMENTTHREADED | 
    COINIT_DISABLE_OLE1DDE);

if (FAILED(hr))
{
    // Handle the error.
}
else
{
    // The function succeeded.
}
```
