# <a name="thread-pool-api"></a>執行緒集區 API

 (API) 的執行緒集區應用程式開發介面會使用以物件為基礎的設計。 下列每一個物件都是以使用者模式資料結構來表示：

-   集區物件是一組可以用來執行工作的背景工作執行緒。 每個進程都可以視需要建立具有不同特性的多個隔離集區。 每個進程也有一個預設集區。
-   清除群組與一組回呼產生的物件相關聯。 函數存在，可等候並釋出每個清除群組成員的所有物件。 如此一來，應用程式就不會追蹤它所建立的所有物件。
-   工作物件會指派給集區，並選擇性地指派給清除群組。 它可以張貼，導致集區中的背景工作執行緒執行其回呼。 工作物件可以有多個未完成的貼文;每個都會產生回呼。 因為缺少資源，所以 post 作業無法失敗。
-   計時器物件控制回呼的排程。 每次計時器過期時，其回呼都會張貼到其背景工作集區。 因為缺少資源，所以設定計時器無法失敗。
-   等候物件會導致等候者執行緒等候可等候控制碼。 在完成等候或超時期限到期後，等候者執行緒會將等待物件回呼張貼至等候的背景工作角色集區。 設定等候失敗，因為資源不足。
-   I/o 物件會將檔案控制代碼與執行緒集區的 i/o 完成通訊埠產生關聯。 當非同步 i/o 作業完成時，背景工作執行緒會挑選作業的狀態，並呼叫 i/o 物件的回呼。
