<span data-ttu-id="5cf73-104"> (API) 的執行緒集區應用程式開發介面會使用以物件為基礎的設計。</span><span class="sxs-lookup"><span data-stu-id="5cf73-104">The thread pool application programming interface (API) uses an object-based design.</span></span> <span data-ttu-id="5cf73-105">下列每一個物件都是以使用者模式資料結構來表示：</span><span class="sxs-lookup"><span data-stu-id="5cf73-105">Each of the following objects is represented by a user-mode data structure:</span></span>

-   <span data-ttu-id="5cf73-106">集區物件是一組可以用來執行工作的背景工作執行緒。</span><span class="sxs-lookup"><span data-stu-id="5cf73-106">A pool object is a set of worker threads that can be used to perform work.</span></span> <span data-ttu-id="5cf73-107">每個進程都可以視需要建立具有不同特性的多個隔離集區。</span><span class="sxs-lookup"><span data-stu-id="5cf73-107">Each process can create multiple isolated pools with different characteristics as necessary.</span></span> <span data-ttu-id="5cf73-108">每個進程也有一個預設集區。</span><span class="sxs-lookup"><span data-stu-id="5cf73-108">There is also a default pool for each process.</span></span>
-   <span data-ttu-id="5cf73-109">清除群組與一組回呼產生的物件相關聯。</span><span class="sxs-lookup"><span data-stu-id="5cf73-109">A clean-up group is associated with a set of callback-generating objects.</span></span> <span data-ttu-id="5cf73-110">函數存在，可等候並釋出每個清除群組成員的所有物件。</span><span class="sxs-lookup"><span data-stu-id="5cf73-110">Functions exist to wait on and release all objects that are members of each clean-up group.</span></span> <span data-ttu-id="5cf73-111">如此一來，應用程式就不會追蹤它所建立的所有物件。</span><span class="sxs-lookup"><span data-stu-id="5cf73-111">This frees the application from keeping track of all the objects it has created.</span></span>

-   <span data-ttu-id="5cf73-116">計時器物件控制回呼的排程。</span><span class="sxs-lookup"><span data-stu-id="5cf73-116">A timer object controls the scheduling of callbacks.</span></span> <span data-ttu-id="5cf73-117">每次計時器過期時，其回呼都會張貼到其背景工作集區。</span><span class="sxs-lookup"><span data-stu-id="5cf73-117">Each time a timer expires, its callback is posted to its worker pool.</span></span> <span data-ttu-id="5cf73-118">因為缺少資源，所以設定計時器無法失敗。</span><span class="sxs-lookup"><span data-stu-id="5cf73-118">Setting a timer cannot fail due to lack of resources.</span></span>
-   <span data-ttu-id="5cf73-119">等候物件會導致等候者執行緒等候可等候控制碼。</span><span class="sxs-lookup"><span data-stu-id="5cf73-119">A wait object causes a waiter thread to wait on a waitable handle.</span></span> <span data-ttu-id="5cf73-120">在完成等候或超時期限到期後，等候者執行緒會將等待物件回呼張貼至等候的背景工作角色集區。</span><span class="sxs-lookup"><span data-stu-id="5cf73-120">After the wait is satisfied or the time-out period expires, the waiter thread posts the wait objects' callback to the wait's worker pool.</span></span> <span data-ttu-id="5cf73-121">設定等候失敗，因為資源不足。</span><span class="sxs-lookup"><span data-stu-id="5cf73-121">Setting a wait cannot fail due to lack of resources.</span></span>
-   <span data-ttu-id="5cf73-122">I/o 物件會將檔案控制代碼與執行緒集區的 i/o 完成通訊埠產生關聯。</span><span class="sxs-lookup"><span data-stu-id="5cf73-122">An I/O object associates a file handle with the I/O completion port for the thread pool.</span></span> <span data-ttu-id="5cf73-123">當非同步 i/o 作業完成時，背景工作執行緒會挑選作業的狀態，並呼叫 i/o 物件的回呼。</span><span class="sxs-lookup"><span data-stu-id="5cf73-123">When an asynchronous I/O operation completes, a worker thread picks up the status of the operation and calls the I/O object's callback.</span></span>
