# <a name="parts-and-states"></a>Parts and States

本主題說明在啟用視覺化樣式時，您用來變更控制面板的部分和狀態。 Windows 會繪製控制項和視窗的特定部分，而且這些元件會分成父元件和子元件。 大部分的部分都有狀態，例如停用、使用中或已選取。 當您建立控制項時，您會決定要將視覺效果樣式套用至控制項的哪些部分。 您可以將控制項分成一個主要部分，以及零或多個名為的子元件。 部分範例包括文字區域、繪製的圖形、影像和線條。 針對每個部分，您可以在單一點陣圖檔案中定義一個或多個背景影像。 不同的背景通常會與控制項的不同狀態相關聯，但不一定如此。 例如，如果控制項已停用，則可能會使用陰影背景。 在執行時間，您可以使用 *iStateId* 值作為正確影像的1索引，來選取適當的影像。

> [!Note]  
> 大部分的主題 API 可讓您在參數清單中使用 (0，0) 的部分和狀態識別碼，最常見的方式是在使用 [**DrawThemeBackground**](/windows/desktop/api/Uxtheme/nf-uxtheme-drawthemebackground)繪製背景時。 這表示要使用較高層級元素的資料。 第0部分可解讀為「此專案的所有部分都相同」的「資料」，並將狀態0視為「此元件的所有狀態都相同」的「資料。」 只有在未定義任何狀態時，才使用 *stateId*= 0。
