## <a name="up-down-control-styles"></a><span data-ttu-id="d4450-114">Up-Down 控制項樣式</span><span class="sxs-lookup"><span data-stu-id="d4450-114">Up-Down Control Styles</span></span>

<span data-ttu-id="d4450-116">具有 ud [**\_ ALIGNLEFT**](up-down-control-styles.md) 或 ud [**\_ ALIGNRIGHT**](up-down-control-styles.md) 樣式的上下按鈕會對齊其好友視窗的左邊緣或右邊緣。</span><span class="sxs-lookup"><span data-stu-id="d4450-116">An up-down control with the [**UDS\_ALIGNLEFT**](up-down-control-styles.md) or [**UDS\_ALIGNRIGHT**](up-down-control-styles.md) style aligns with the left or right edge of its buddy window.</span></span> <span data-ttu-id="d4450-117">好友視窗的寬度會減少以容納上下按鈕控制項的寬度。</span><span class="sxs-lookup"><span data-stu-id="d4450-117">The width of the buddy window is decreased to accommodate the width of the up-down control.</span></span>

<span data-ttu-id="d4450-118">具有 ud [**\_ SETBUDDYINT**](up-down-control-styles.md) 樣式的上下按鈕控制項，會在目前的位置變更時，設定其好友視窗的標題。</span><span class="sxs-lookup"><span data-stu-id="d4450-118">An up-down control with the [**UDS\_SETBUDDYINT**](up-down-control-styles.md) style sets the caption of its buddy window whenever the current position changes.</span></span> <span data-ttu-id="d4450-119">除非指定了 ud [**\_ NOTHOUSANDS**](up-down-control-styles.md) 樣式，否則控制項會在十進位字串的每三位數之間插入千位分隔符號。</span><span class="sxs-lookup"><span data-stu-id="d4450-119">The control inserts a thousands separator between every three digits of a decimal string unless the [**UDS\_NOTHOUSANDS**](up-down-control-styles.md) style is specified.</span></span> <span data-ttu-id="d4450-120">如果 [合作者] 視窗是清單方塊，則會將其目前的選取範圍（而不是其標題）設定為由上而下的控制項。</span><span class="sxs-lookup"><span data-stu-id="d4450-120">If the buddy window is a list box, an up-down control sets its current selection instead of its caption.</span></span>

<span data-ttu-id="d4450-121">您可以指定 ud [**\_ ARROWKEYS**](up-down-control-styles.md) 樣式，以提供最新控制項的鍵盤介面。</span><span class="sxs-lookup"><span data-stu-id="d4450-121">You can specify the [**UDS\_ARROWKEYS**](up-down-control-styles.md) style to provide a keyboard interface for an up-down control.</span></span> <span data-ttu-id="d4450-122">如果指定此樣式，控制項會處理向上和向下鍵。</span><span class="sxs-lookup"><span data-stu-id="d4450-122">If this style is specified, the control processes the up and down arrow keys.</span></span> <span data-ttu-id="d4450-123">控制項也會將好友視窗子類別化，讓它可以在好友視窗具有焦點時處理這些按鍵。</span><span class="sxs-lookup"><span data-stu-id="d4450-123">The control also subclasses the buddy window so that it can process these keys when the buddy window has the focus.</span></span>

<span data-ttu-id="d4450-124">如果您使用上下按鈕控制項來進行水準滾動，則可以指定 ud [**\_ HORZ**](up-down-control-styles.md) 樣式。</span><span class="sxs-lookup"><span data-stu-id="d4450-124">If you use an up-down control for horizontal scrolling, you can specify the [**UDS\_HORZ**](up-down-control-styles.md) style.</span></span> <span data-ttu-id="d4450-125">此樣式會讓下一個控制項的箭號向左和向右指向，而不是向上和向下。</span><span class="sxs-lookup"><span data-stu-id="d4450-125">This style causes the up-down control's arrows to point left and right instead of up and down.</span></span>

<span data-ttu-id="d4450-126">根據預設，如果使用者嘗試遞增或遞減超過最大值或最小值，則目前的位置不會變更。</span><span class="sxs-lookup"><span data-stu-id="d4450-126">By default, the current position does not change if the user attempts to increment it or decrement it beyond the maximum or minimum value.</span></span> <span data-ttu-id="d4450-127">您可以使用 ud [**\_ 換行**](up-down-control-styles.md) 樣式來變更此行為，讓位置「換行」至相反的極端。</span><span class="sxs-lookup"><span data-stu-id="d4450-127">You can change this behavior by using the [**UDS\_WRAP**](up-down-control-styles.md) style, so the position "wraps" to the opposite extreme.</span></span> <span data-ttu-id="d4450-128">例如，以超過上限的遞增會將位置包裝回下限值。</span><span class="sxs-lookup"><span data-stu-id="d4450-128">For example, incrementing past the upper limit wraps the position back to the lower limit.</span></span>

## <a name="position-and-acceleration"></a><span data-ttu-id="d4450-129">定位與加速</span><span class="sxs-lookup"><span data-stu-id="d4450-129">Position and Acceleration</span></span>

<span data-ttu-id="d4450-130">建立上下按鈕控制項之後，您可以藉由傳送訊息來變更控制項的目前位置、最小位置和最大位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-130">After an up-down control is created, you can change the control's current position, minimum position, and maximum position by sending messages.</span></span> <span data-ttu-id="d4450-131">您也可以變更用來顯示好友視窗中目前位置的基本基底，以及按一下向上或向下箭號時，目前位置變更的速率。</span><span class="sxs-lookup"><span data-stu-id="d4450-131">You can also change the radix base used to display the current position in the buddy window and the rate at which the current position changes when the up or down arrow is clicked.</span></span>

<span data-ttu-id="d4450-132">若要取出上下按鈕控制項的目前位置，請使用 [**UDM \_ GETPOS**](udm-getpos.md) 訊息。</span><span class="sxs-lookup"><span data-stu-id="d4450-132">To retrieve the current position of an up-down control, use the [**UDM\_GETPOS**](udm-getpos.md) message.</span></span> <span data-ttu-id="d4450-133">針對具有合作者視窗的上下按鈕控制項，目前的位置是在好友視窗標題中的數位。</span><span class="sxs-lookup"><span data-stu-id="d4450-133">For an up-down control with a buddy window, the current position is the number in the buddy window's caption.</span></span> <span data-ttu-id="d4450-134">由於標題可能已變更 (例如，使用者可能已編輯編輯控制項的文字) ，由上而下的控制項則會取得目前的標題，並據以更新其目前的位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-134">Because the caption may have changed (for example, the user may have edited the text of an edit control), the up-down control retrieves the current caption and updates its current position accordingly.</span></span>

<span data-ttu-id="d4450-137">[**UDM \_ SETPOS**](udm-setpos.md)訊息會設定合作者視窗的目前位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-137">The [**UDM\_SETPOS**](udm-setpos.md) message sets the current position of a buddy window.</span></span> <span data-ttu-id="d4450-138">請注意，和捲軸不同的是，當按下向上鍵和向下箭號時，上下按鈕會自動變更其目前的位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-138">Note that unlike a scroll bar, an up-down control automatically changes its current position when the up and down arrows are clicked.</span></span> <span data-ttu-id="d4450-139">因此，應用程式在處理 [**WM \_ VSCROLL**](wm-vscroll.md) 或 [**WM \_ HSCROLL**](wm-hscroll.md) 訊息時，不需要設定目前的位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-139">An application, therefore, does not need to set the current position when processing the [**WM\_VSCROLL**](wm-vscroll.md) or [**WM\_HSCROLL**](wm-hscroll.md) message.</span></span>

<span data-ttu-id="d4450-140">您可以使用 [**UDM \_ SETRANGE**](udm-setrange.md) 訊息來變更上下按鈕控制項的最小和最大位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-140">You can change the minimum and maximum positions of an up-down control by using the [**UDM\_SETRANGE**](udm-setrange.md) message.</span></span> <span data-ttu-id="d4450-141">最大位置可能小於最小值，而在此情況下，按一下向上箭號按鈕會減少目前的位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-141">The maximum position may be less than the minimum, and in that case clicking the up arrow button decreases the current position.</span></span> <span data-ttu-id="d4450-142">若要以另一種方式進行，向上指的是移到最大的位置。</span><span class="sxs-lookup"><span data-stu-id="d4450-142">To put it another way, up means moving toward the maximum position.</span></span> <span data-ttu-id="d4450-143">若要取得上下按鈕控制項的最小和最大位置，請使用 [**UDM \_ GETRANGE**](udm-getrange.md) 訊息。</span><span class="sxs-lookup"><span data-stu-id="d4450-143">To retrieve the minimum and maximum positions for an up-down control, use the [**UDM\_GETRANGE**](udm-getrange.md) message.</span></span>
