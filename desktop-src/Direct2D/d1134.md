# <a name="d1134-buffer-precision-disallowed-by-caps"></a>D1134： Caps 不允許的緩衝區有效位數

裝置沒有支援指定緩衝區精確度的功能。






 

## <a name="possible-causes"></a>可能的原因

Direct2D 可讓應用程式針對某些作業指定所需的緩衝區精確度。 例如，使用 [**ID2D1DeviceCoNtext：： SetRenderingControls**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1devicecontext-setrenderingcontrols(constd2d1_rendering_controls))來指定用於映射管線的精確度。 不過，硬體 Direct2D 裝置可能不支援所有緩衝區精確度，視其功能等級而定。 例如，功能層級9的裝置可能不支援每個通道8個位元組以外的緩衝區精確度選項。

## <a name="possible-fixes"></a>可能的修正

針對在硬體上執行的應用程式，請確定硬體的功能層級支援所有緩衝區精確度規範。 請考慮使用 [**ID3D11Device：： CheckFormatSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) 和 [**ID3D11Device：： CheckFeatureSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkfeaturesupport) ，以取得硬體支援哪些格式的詳細資訊，並判斷將允許哪個表面精確度。 針對所有功能層級9裝置的一致作業，請設計可避免高精確度緩衝區的回溯機制。
