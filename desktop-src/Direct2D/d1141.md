# <a name="d1141-print-requires-srgb-color-space"></a>D1141： Print 需要 sRGB 色彩空間

Direct2D Print：不支援自訂色彩空間。 改為使用 sRGB。






 

## <a name="possible-causes"></a>可能的原因

建立 Direct2D 列印控制項時，應用程式會使用 \_ \_ \_ 具有色彩空間屬性的 D2D1 列印控制項屬性。 Direct2D 支援某些作業的 [**D2D1 \_ 色彩 \_ 空間 \_ 自訂**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_color_space) ，例如使用 ICC 資料建立色彩內容。 不過，在建立列印控制項時，Direct2D 只支援內建的色彩空間，例如 sRGB 或 scRGB。

## <a name="possible-fixes"></a>可能的修正

若要處理訊息，請在建立列印控制項時指定非自訂的色彩空間，例如 sRGB 或 scRGB。 訊息不會指出錯誤，但表示 Direct2D 會忽略自訂規範，並使用 sRGB 色彩空間來建立列印控制項。
