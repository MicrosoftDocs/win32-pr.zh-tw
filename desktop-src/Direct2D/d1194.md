# <a name="d1194-flush-outstanding-dc"></a>D1194：排清未完成的 DC

在 GetDC/ReleaseDC 配對之間呼叫 Flush 或 EndDraw 是不正確。






 

## <a name="possible-causes"></a>可能的原因

使用 GDI interoperated rendertarget 時，應用程式可以使用 IGdiInteropRenderTarget：： GetDC 或 IGdiInteropRenderTarget：： ReleaseDC 來控制目標 DC 的擁有權。 不過，當應用程式具有 DC 的控制權時，呼叫 ID2D1RenderTarget：： Flush 或 ID2D1RenderTarget：： EndDraw 是不正確。 Flush 和 EndDraw 可能會導致在目標資源上進行轉譯，而 Direct2D 不應該嘗試同時修改資源與應用程式。

## <a name="possible-fixes"></a>可能的修正

若要解決此訊息，請確定應用程式在具有 IGdiInteropRenderTarget DC 的擁有權時，不會呼叫 ID2D1RenderTarget：： Flush 或 ID2D1RenderTarget：： EndDraw。 可能的話，請組織應用程式以將其 Direct2D 轉譯行為與其 GDI 行為分開，讓 DC 擁有權的區別更清楚。
