# <a name="d1149-print-detected-an-invalid-unicode-character"></a>D1149： Print 偵測到不正確 Unicode 字元

Direct2D Print：在 DWRITE \_ \_ 字元執行描述中偵測到不正確 Unicode 字元 \_ 。 它已取代為空格 (0x20) 。






 

## <a name="possible-causes"></a>可能的原因

Direct2D 列印會將 [Unicode 檔](https://unicode.org/faq/utf_bom.html#utf16-7)中所述的無效字元以及可見範圍外的特定 ASCII 字元視為不可列印。 使用這些字元並不會造成錯誤。 相反地，Direct2D Print 會以空格取代這些字元。

## <a name="possible-fixes"></a>可能的修正

若要避免此通知，請在繪製文字以進行 printability 時，不要使用格式不正確的 Unicode 字元或不可列印的 ASCII。
