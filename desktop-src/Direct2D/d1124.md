# <a name="d1124-dxgi-format-unknown-with-null-target-bitmap"></a>D1124：使用 Null 目標點陣圖時未知的 DXGI 格式

\_ \_ 當目標設為 Null 或命令清單時，不允許使用 DXGI 格式來做為此 API 的參數。






 

## <a name="possible-causes"></a>可能的原因

若為 ID2D1RenderTarget：： CreateCompatibleRenderTarget 等作業，來源轉譯目標必須已指派目標點陣圖。 若沒有目標點陣圖，Direct2D 將無法判斷新轉譯目標資源將擁有的介面格式。

## <a name="possible-fixes"></a>可能的修正

若要解決此情況，請確定對應的 ID2D1DeviceCoNtext 具有使用 ID2D1DeviceCoNtext：： SetTarget 指派的目標點陣圖。 或者，如果應用程式只需要能夠轉譯至兩個不同的目標表面，請考慮建立 ID2D1Bitmap，並將它設定為裝置內容的目標，而不是使用 ID2D1RenderTarget：： CreateCompatibleRenderTarget。
