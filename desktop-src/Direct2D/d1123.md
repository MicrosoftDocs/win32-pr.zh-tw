# <a name="d1123-dxgi-format-unknown-not-allowed"></a>D1123：不允許未知的 DXGI 格式

\_ \_ 無法使用 DXGI 格式 UNKNOWN 作為此 API 的參數。






 

## <a name="possible-causes"></a>可能的原因

針對接受介面格式作為參數的 Direct2D 功能，應該完整指定介面格式。 在某些情況下，它可能會保持未知;例如，在呼叫 ID2D1RenderTarget：： CreateHwndRenderTarget 或 ID2D1RenderTarget：： CreateDxgiSurfaceRenderTarget 時，Direct2D 可能會從交換鏈格式或支援 Direct3D 介面格式推斷出介面格式，例如，可能的情況。

## <a name="possible-fixes"></a>可能的修正

在無法推斷表面格式的情況下，Direct2D 將不會建立資源，而且會輸出此訊息。 若要解決此情況，請修改應用程式，為造成此訊息的作業指定未知的表面格式。 要使用的合理預設介面格式（如果有疑問）是 DXGI \_ 格式 \_ B8G8R8A8 \_ UNORM。 如果應用程式必須建立 Direct2D 資源以與 Direct3D 或 WIC 資源交互操作，請選擇符合 interoperated 資源的格式。 WIC 資源的語義與 DXGI 資源稍有不同;如需其他指引，請參閱 [支援的像素格式和 Alpha 模式](supported-pixel-formats-and-alpha-modes.md) 。
