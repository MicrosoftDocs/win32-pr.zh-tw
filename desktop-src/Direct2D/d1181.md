# <a name="d1181-enddraw-incorrect-state"></a>D1181： EndDraw 不正確的狀態

已呼叫 EndDraw，但 rendertarget 不是處於有效的狀態。 這可能是因為呼叫 EndDraw 而沒有相符的 BeginDraw。






 

## <a name="possible-causes"></a>可能的原因

應用程式會對 EndDraw 進行多次呼叫，而不需要在相同的目標上呼叫 BeginDraw。 使用 Direct2D 時，應用程式應該使用先前的 BeginDraw 呼叫來比對 EndDraw 的每個呼叫。 或者，如果目標處於錯誤狀態且未呈現，就可能會發生此訊息。

## <a name="possible-fixes"></a>可能的修正

確定 BeginDraw 的每個呼叫都符合後續的 EndDraw。 使用 Flush 或 EndDraw 提交命令至目標時，請先確認傳回的結果為 \_ [確定]，然後再嘗試使用目標。
