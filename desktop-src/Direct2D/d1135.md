# <a name="d1135-bitmap-option-must-be-subset"></a>D1135： Bitmap 選項必須是子集

點陣圖選項 \[ *值* \] 必須是與 DXGI 介面相關聯之旗標的子集。

</dd> </dl>

## <a name="possible-causes"></a>可能的原因

在將 Direct3D 材質與 Direct2D 交互操作為 ID2D1Bitmaps 時，[D2D1 點陣圖選項] 欄位中指定的選項 \_ \_ 應符合紋理的 Direct3D 屬性。 尤其是：

-   [**D2D1 \_ 點陣圖 \_ 選項 \_ CPU \_ 讀取**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options)選項需要 Direct3D CPU \_ 存取權 \_ 讀取選項。
-   [**D2D1 \_ 點陣圖 \_ 選項 \_ 目標**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options)選項需要 [系結轉譯目標] \_ \_ 選項。
-   [**D2D1 \_ 點陣圖 \_ 選項 \_ NONE**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options)選項與 \_ \_ 僅限 CPU 存取寫入的 Direct3D 使用方式選項不相容 \_ 。

## <a name="possible-fixes"></a>可能的修正

若要解決這個問題，請確定 [**ID2D1Bitmap**](/windows/win32/api/d2d1/nn-d2d1-id2d1bitmap) 的建立使用 [**D2D1 \_ 點陣圖 \_ 選項**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options) 規範，其與基礎 Direct3D 材質的屬性一致。 為了使用 **D2D1 \_ 點陣圖 \_ 選項**，這可能需要使用 [**ID2D1DeviceCoNtext：： CreateBitmap**](id2d1devicecontext-createbitmap-overload.md) 而不是 [**ID2D1RenderTarget：： CreateBitmap**](id2d1rendertarget-createbitmap.md)，因為它是 [**D2D1 \_ 點陣圖 \_ PROPERTIES1**](/windows/desktop/api/D2D1_1/ns-d2d1_1-d2d1_bitmap_properties1)的成員。
