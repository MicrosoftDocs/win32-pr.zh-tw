# <a name="d1180-begindraw-incorrect-state"></a>D1180： BeginDraw 不正確的狀態

BeginDraw 被呼叫多次，沒有對應的 EndDraw 呼叫。






 

## <a name="possible-causes"></a>可能的原因

應用程式會對 BeginDraw 進行多次呼叫，而不需要在相同的目標上呼叫 EndDraw。 使用 Direct2D 時，應用程式應該會比對 BeginDraw 的每個呼叫和後續的 EndDraw。 或者，如果目標處於錯誤狀態，且 BeginDraw 呼叫不允許呈現目標，則可能會發生此訊息。

## <a name="possible-fixes"></a>可能的修正

確定 BeginDraw 的每個呼叫都符合後續的 EndDraw。 使用 Flush 或 EndDraw 提交命令至目標時，請先確認傳回的結果為 \_ [確定]，然後再嘗試使用目標。
