# <a name="d1136-text-rendering-mode-conflict"></a>D1136：文字轉譯模式衝突

DWRITE 轉譯 \_ \_ 模式和在這個轉譯 \_ \_ 目標上設定的 D2D1 文字消除鋸齒 \_ 模式不相容。






 

## <a name="possible-causes"></a>可能的原因

針對以文字為基礎的作業，例如 ID2D1RenderTarget：:D rawGlyphRun 和 ID2D1DeviceCoNtext：： GetGlyphRunWorldBounds，Direct2D 會考慮裝置內容的目前文字消除鋸齒選項，以及 DirectWrite 消除鋸齒選項。 Direct2D 和 DirectWrite 之間的消除鋸齒語義會稍有不同。 當指定的兩個消除鋸齒選項不一致時，就會出現此訊息。

## <a name="possible-fixes"></a>可能的修正

為了確保 Direct2D 文字消除鋸齒模式會同意 DirectWrite 轉譯模式，請確定符合下列需求：•針對 D2D1 \_ 文字 \_ 消除鋸齒 \_ 模式 \_ CLEARTYPE，DirectWrite 轉譯模式可能不是大綱或別名。•若為 D2D1 \_ 文字 \_ 消除鋸齒 \_ 模式 \_ ，則 DirectWrite 轉譯模式可能不會有別名。•若要讓 D2D1 \_ 文字 \_ 消除鋸齒 \_ 模式為 \_ 別名，DirectWrite 轉譯模式可能不是任何 CLEARTYPE 選項。 \[\]要變更目前 Direct2D 文字消除鋸齒模式的分行符號，請使用 ID2D1RenderTarget：： SetTextAntialiasMode。
