# <a name="r-volume-shadow-copy-service"></a>R (磁碟區陰影複製服務) 

[](vssgloss-a.md) [B](vssgloss-b.md) [C](vssgloss-c.md) [D](vssgloss-d.md) [E](vssgloss-e.md) [F](vssgloss-f.md) [G](vssgloss-g.md) [H](vssgloss-h.md) [I](vssgloss-i.md) J K [L](vssgloss-l.md) M [N](vssgloss-n.md) [O](vssgloss-o.md) [P](vssgloss-p.md) Q R [S](vssgloss-s.md) [T](vssgloss-t.md) [](vssgloss-v.md) [W](vssgloss-w.md) X Y Z

<dl> <dt>

<span id="base.vssgloss_requester"></span><span id="BASE.VSSGLOSS_REQUESTER"></span>**請求**
</dt> <dd>

與 VSS 互動以建立和管理一或多個磁片區陰影複製和陰影複製集的任何程式，這是最少的。

通常，要求者會管理陰影複製以支援一些其他功能，例如備份和還原作業和磁片鏡像。 另請參閱 [*陰影*](vssgloss-s.md)複製、 [*陰影複製集*](vssgloss-s.md)。

</dd> <dt>

<span id="base.vssgloss_restore_method"></span><span id="BASE.VSSGLOSS_RESTORE_METHOD"></span>**restore 方法**
</dt> <dd>

Restore process 方法的規格。 它會控制還原時是否覆寫檔案等問題。 如果還原方法設定與還原目標的設定衝突，則會優先使用還原目標。 另請參閱 *還原目標*。

</dd> <dt>

<span id="base.vssgloss_restore_target"></span><span id="BASE.VSSGLOSS_RESTORE_TARGET"></span>**還原目標**
</dt> <dd>

在 VSS 下，還原目標是指要求者如何還原檔案的規格，例如，如果它應該覆寫現有的檔案。

</dd> </dl>
