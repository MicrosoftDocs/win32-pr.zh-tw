<dl> <dt>

<span id="_security_national_institute_of_standards_and_technology_gly"></span><span id="_SECURITY_NATIONAL_INSTITUTE_OF_STANDARDS_AND_TECHNOLOGY_GLY"></span>**美國國家標準與技術局**
</dt> <dd>

 (NIST) 美國 Commerce 部門的部門，發佈政府和私用部門電腦系統的官方標準。 這些標準會以聯邦資訊處理標準的形式發行 (FIPS) 發行集。 在1987中，NIST 被導向定義標準，以確保政府電腦系統中敏感性但非機密資訊的安全性。

</dd> <dt>

<span id="_security_negotiate_gly"></span><span id="_SECURITY_NEGOTIATE_GLY"></span>**洽談**
</dt> <dd>

[*安全性支援提供者*](s-gly.md) (SSP) ，可作為 [*安全性支援提供者介面*](s-gly.md) (SSPI) 和其他 ssp 之間的應用層。 當應用程式呼叫 SSPI 以登入網路時，它可以指定 SSP 來處理要求。 如果應用程式指定 Negotiate，Negotiate 會分析要求，並根據客戶設定的安全性原則挑選最適合的 SSP 來處理要求。

</dd> <dt>

<span id="_security_nist_gly"></span><span id="_SECURITY_NIST_GLY"></span>**Nist**
</dt> <dd>

請參閱國家標準 *和技術局*。

</dd> <dt>

<span id="_security_nonce_gly"></span><span id="_SECURITY_NONCE_GLY"></span>**Nonce**
</dt> <dd>

隨機產生的值，可用來使「重新執行」攻擊失效。

</dd> <dt>

<span id="_security_nonrepudiation_gly"></span><span id="_SECURITY_NONREPUDIATION_GLY"></span>**拒絕**
</dt> <dd>

此功能可指出執行特定動作的使用者，因此毫無疑問地將讓使用者無法拒絕承擔責任。 例如，系統可能會在刪除檔案時，記錄使用者的識別碼。

</dd> </dl>
