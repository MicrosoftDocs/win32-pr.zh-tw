# <a name="t-security-glossary"></a>T (安全性詞彙) 

[](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [H](h-gly.md) [I](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](u-gly.md) [](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_t_0_protocol_gly"></span><span id="_SECURITY_T_0_PROTOCOL_GLY"></span>**T = 0 通訊協定**
</dt> <dd>

非同步字元導向半雙工傳輸通訊協定。

</dd> <dt>

<span id="_security_t_1_protocol_gly"></span><span id="_SECURITY_T_1_PROTOCOL_GLY"></span>**T = 1 通訊協定**
</dt> <dd>

非同步、區塊導向的半雙工傳輸通訊協定。

</dd> <dt>

<span id="_security_terminal_gly"></span><span id="_SECURITY_TERMINAL_GLY"></span>**終端**
</dt> <dd>

監視器、鍵盤、滑鼠和共置週邊設備（例如智慧卡讀卡機）的組合。 多個進程可能與單一終端機相關聯，但只有一個進程會在任何指定時間控制終端機。

</dd> <dt>

<span id="_security_three_phase_key_exchange_protocol_gly"></span><span id="_SECURITY_THREE_PHASE_KEY_EXCHANGE_PROTOCOL_GLY"></span>**三階段金鑰-exchange 通訊協定**
</dt> <dd>

一種通訊協定，用來在非安全網路上的兩個使用者之間產生經過驗證和加密的連接。 使用者交換一組訊息以協調一組加密金鑰。 寄件者會使用一個金鑰來加密傳送給接收者的訊息，而接收者會使用另一個金鑰來加密傳送給傳送者的訊息。 此通訊協定可確保這兩個使用者都處於使用中狀態，並將訊息直接傳送給彼此。

</dd> <dt>

<span id="_security_thumbprint_gly"></span><span id="_SECURITY_THUMBPRINT_GLY"></span>**指紋**
</dt> <dd>

憑證的 [*sha-1*](s-gly.md) 雜湊。

</dd> <dt>

<span id="_security_tls_gly"></span><span id="_SECURITY_TLS_GLY"></span>**TLS**
</dt> <dd>

請參閱 *傳輸層安全性通訊協定*。

</dd> <dt>

<span id="_security_token_gly"></span><span id="_SECURITY_TOKEN_GLY"></span>**令 牌**
</dt> <dd>

請參閱 [*存取權杖*](a-gly.md)。

</dd> <dt>

<span id="_security_transaction_gly"></span><span id="_SECURITY_TRANSACTION_GLY"></span>**交易**
</dt> <dd>

這項作業可讓您以單一作業的方式執行多個互動。 在所有互動都順利完成之前，不會實際採取中繼步驟。 如果任何互動失敗，則會將所有步驟都傳回至其原始設定。

</dd> <dt>

<span id="_security_transport_layer_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_GLY"></span>**傳輸層**
</dt> <dd>

同時負責服務品質與正確資訊傳遞的網路層。 此網路層所負責執行的工作當中，包含錯誤偵測與修正。

</dd> <dt>

<span id="_security_transport_layer_security_protocol_gly"></span><span id="_SECURITY_TRANSPORT_LAYER_SECURITY_PROTOCOL_GLY"></span>**傳輸層安全性通訊協定**
</dt> <dd>

 (TLS) 通訊協定，可在兩個透過網路進行通訊的應用程式之間提供通訊隱私權和安全性。

</dd> <dt>

<span id="_security_triple_des_gly"></span><span id="_SECURITY_TRIPLE_DES_GLY"></span>**三重 DES**
</dt> <dd>

DES 區塊加密演算法的變化，它會使用一個金鑰來加密純文字，並使用第二個金鑰來加密產生的加密文字，最後使用第三個金鑰來加密第二個加密的結果。 三重 DES 是對稱演算法，使用相同的演算法和金鑰來進行加密和解密。

</dd> <dt>

<span id="_security_trustee_gly"></span><span id="_SECURITY_TRUSTEE_GLY"></span>**受託 人**
</dt> <dd>

套用存取控制專案 (ACE) 的使用者帳戶、群組帳戶或登入會話。 存取控制清單中的每個 ACE (ACL) 適用于一個信任者。

另請參閱 [*存取控制專案*](a-gly.md) 和 [*存取控制清單*](a-gly.md)。

</dd> <dt>

<span id="_security_trust_list_gly"></span><span id="_SECURITY_TRUST_LIST_GLY"></span>**信任清單**
</dt> <dd>

請參閱 [*憑證信任清單*](c-gly.md)。

</dd> <dt>

<span id="_security_trust_provider_gly"></span><span id="_SECURITY_TRUST_PROVIDER_GLY"></span>**信任提供者**
</dt> <dd>

決定是否信任特定檔案的軟體。 此決策是依據檔案相關的憑證而定。

</dd> </dl>
