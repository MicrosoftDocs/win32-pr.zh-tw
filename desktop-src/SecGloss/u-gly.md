# <a name="u-security-glossary"></a>U (安全性詞彙) 

[](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F [G](g-gly.md) [H](h-gly.md) [I](i-gly.md) J [K](k-gly.md) [L](l-gly.md) [M](m-gly.md) [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [](v-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_ucs_gly"></span><span id="_SECURITY_UCS_GLY"></span>**UCS**
</dt> <dd>

請參閱 *通用字元集*。

</dd> <dt>

<span id="_security_unicode_gly"></span><span id="_SECURITY_UNICODE_GLY"></span>**Unicode**
</dt> <dd>

全球字元編碼標準，可透過定義16位字元字串，而不是標準的8位字元字串，讓每個字串中包含更多資訊。 Unicode 允許通用資料交換，並改善多語系文字處理。 Unicode 字串也稱為 *寬字元串*。

</dd> <dt>

<span id="_security_universal_character_set_gly"></span><span id="_SECURITY_UNIVERSAL_CHARACTER_SET_GLY"></span>**通用字元集**
</dt> <dd>

 (UCS) 16 位字元集。

</dd> <dt>

<span id="_security_universal_well_known_security_identifier_gly"></span><span id="_SECURITY_UNIVERSAL_WELL_KNOWN_SECURITY_IDENTIFIER_GLY"></span>**通用知名安全識別碼**
</dt> <dd>

 (SID) 的 [*安全識別碼*](s-gly.md) ，這些系統會根據已驗證的使用者和任意存取控制來遵守安全性模型。

</dd> <dt>

<span id="_security_upn_gly"></span><span id="_SECURITY_UPN_GLY"></span>**Upn**
</dt> <dd>

請參閱 *使用者主體名稱*。

</dd> <dt>

<span id="_security_user_gly"></span><span id="_SECURITY_USER_GLY"></span>**使用者**
</dt> <dd>

系統的使用者是由登入程式所識別，它會建立應用程式可以執行的程式，並存取其他與安全性相關的物件。 登入程式的範例是使用智慧卡，可能會搭配密碼或個人識別碼 (PIN) 。

</dd> <dt>

<span id="_security_user_interface_smart_card_gly"></span><span id="_SECURITY_USER_INTERFACE_SMART_CARD_GLY"></span>**(智慧卡) 的使用者介面**
</dt> <dd>

通用對話方塊，可讓使用者連線至智慧卡並在應用程式中使用。 使用對話方塊時，使用者可以指定特定的卡片，或搜尋要開啟的智慧卡。

</dd> <dt>

<span id="_security_user_principal_name_gly"></span><span id="_SECURITY_USER_PRINCIPAL_NAME_GLY"></span>**使用者主體名稱**
</dt> <dd>

 (UPN) 使用者帳戶名稱 (有時稱為使用者登入名稱) ，以及識別使用者帳戶所在網域的功能變數名稱。 這是登入 Windows 網域時的標準用法。 格式為： someone@example.com () 的電子郵件地址。

</dd> </dl>
