# <a name="m-security-glossary"></a>M (安全性詞彙) 

[](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) [](u-gly.md) [](v-gly.md) F [G](g-gly.md) [H](h-gly.md) [I](i-gly.md) J [K](k-gly.md) [L](l-gly.md) M [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [W](w-gly.md) [X](x-gly.md) Y Z

<dl> <dt>

<span id="_security_mac_gly"></span><span id="_SECURITY_MAC_GLY"></span>**Mac**
</dt> <dd>

請參閱 *訊息驗證碼*。

</dd> <dt>

<span id="_security_mac_key_gly"></span><span id="_SECURITY_MAC_KEY_GLY"></span>**MAC 金鑰**
</dt> <dd>

搭配 [*Schannel*](s-gly.md) 通訊協定使用的訊息驗證碼金鑰。

</dd> <dt>

<span id="_security_master_key_gly"></span><span id="_SECURITY_MASTER_KEY_GLY"></span>**主要金鑰**
</dt> <dd>

用戶端和伺服器用來產生所有工作階段金鑰的金鑰。 主要金鑰是用來產生用戶端讀取金鑰、用戶端寫入金鑰、伺服器讀取金鑰，以及伺服器寫入金鑰。 主要金鑰可以匯出為簡單的金鑰 Blob。

</dd> <dt>

<span id="_security_md2_gly"></span><span id="_SECURITY_MD2_GLY"></span>**MD2**
</dt> <dd>

MD2 雜湊演算法的 CryptoAPI 演算法名稱。 其他雜湊演算法包括 *MD4*、 *MD5* 和 [*SHA*](s-gly.md)。

另請參閱 *MD2 演算法*。

</dd> <dt>

<span id="_security_md2_algorithm_gly"></span><span id="_SECURITY_MD2_ALGORITHM_GLY"></span>**MD2 演算法**
</dt> <dd>

 (MD2) 雜湊演算法，它會建立128位的雜湊值。 MD2 已經過優化，可搭配8位的電腦使用。 CryptoAPI 會依類型 (CALG \_ MD2) 、名稱 (MAC) ，以及類別 (ALG \_ 類別 \_ 雜湊) 來參考此演算法。 MD2 是由 RSA Data Security，Inc. 所開發。

另請參閱 *MD4 演算法*、 *MD5 演算法*。

</dd> <dt>

<span id="_security_md4_gly"></span><span id="_SECURITY_MD4_GLY"></span>**MD4**
</dt> <dd>

MD4 雜湊演算法的 CryptoAPI 演算法名稱。 其他雜湊演算法包括 *MD2*、 *MD5* 和 [*SHA*](s-gly.md)。

請參閱 *MD4 演算法*。

</dd> <dt>

<span id="_security_md4_algorithm_gly"></span><span id="_SECURITY_MD4_ALGORITHM_GLY"></span>**MD4 演算法**
</dt> <dd>

 (MD4) 一個雜湊演算法，它會建立128位的雜湊值。 MD4 已針對32位電腦優化。 現在會將它視為已中斷，因為可以快速且輕鬆地找到衝突。 MD4 是由 RSA Data Security，Inc. 所開發。

另請參閱 *MD2 演算法*、 *MD5 演算法*。

</dd> <dt>

<span id="_security_md5_gly"></span><span id="_SECURITY_MD5_GLY"></span>**MD5**
</dt> <dd>

MD5 雜湊演算法的 CryptoAPI 演算法名稱。 其他雜湊演算法包括 *MD2*、 *MD4* 及 [*SHA*](s-gly.md)。

另請參閱 *MD5 演算法*。

</dd> <dt>

<span id="_security_md5_algorithm_gly"></span><span id="_SECURITY_MD5_ALGORITHM_GLY"></span>**MD5 演算法**
</dt> <dd>

 (MD5) 建立128位雜湊值的雜湊演算法。 MD5 已針對32位電腦進行優化。 CryptoAPI 會依據演算法的演算法識別碼來參考此演算法 (CALG \_ md5) 、名稱 (MD5) 以及類別 (ALG \_ 類別 \_ 雜湊) 。 MD5 是由 RSA Data Security，Inc. 所開發，由 >PROV \_ rsa \_ FULL、>prov \_ RSA \_ SIG、>prov \_ DSS、>prov \_ DSS \_ DH 和 >prov \_ MS \_ EXCHANGE 提供者類型所指定。

另請參閱 *MD2 演算法*、 *MD4 演算法*。

</dd> <dt>

<span id="_security_message_gly"></span><span id="_SECURITY_MESSAGE_GLY"></span>**消息**
</dt> <dd>

已編碼，以傳送至或接收自個人或實體的任何資料。 訊息可能會針對隱私權進行加密、以數位方式簽署以進行驗證，或兩者都有。

</dd> <dt>

<span id="_security_message_digest_gly"></span><span id="_SECURITY_MESSAGE_DIGEST_GLY"></span>**訊息摘要**
</dt> <dd>

請參閱 [*hash*](h-gly.md)。

</dd> <dt>

<span id="_security_message_authentication_code_gly"></span><span id="_SECURITY_MESSAGE_AUTHENTICATION_CODE_GLY"></span>**訊息驗證碼**
</dt> <dd>

 (MAC) 使用對稱工作階段金鑰的金鑰雜湊演算法，以協助確保資料區塊的完整性是從傳送到收到的時間為止。 使用這種類型的演算法時，接收應用程式也必須擁有工作階段金鑰才能重新計算雜湊值，以便驗證基底資料未變更。 CryptoAPI 會依據其類型 (CALG \_ mac) 、名稱 (mac) ，以及類別 (ALG \_ 類別 \_ 雜湊) 來參照此演算法。

</dd> <dt>

<span id="_security_message_encoding_type_gly"></span><span id="_SECURITY_MESSAGE_ENCODING_TYPE_GLY"></span>**訊息編碼類型**
</dt> <dd>

定義訊息的編碼方式。 訊息編碼類型會儲存在編碼類型結構的高序位字中。 目前定義的編碼類型為： CRYPT \_ ASN \_ 編碼、X509 \_ ASN \_ 編碼和 PKCS \_ 7 \_ ASN \_ 編碼。

</dd> <dt>

<span id="_security_message_management_functions_gly"></span><span id="_SECURITY_MESSAGE_MANAGEMENT_FUNCTIONS_GLY"></span>**訊息管理功能**
</dt> <dd>

提供兩種訊息管理層級的函式：低層級訊息函式和簡化的訊息函數。 低層級的訊息函式比簡化的訊息函式提供更大的彈性;不過，它們需要更多函式呼叫。

</dd> <dt>

<span id="_security_message_signing_functions_gly"></span><span id="_SECURITY_MESSAGE_SIGNING_FUNCTIONS_GLY"></span>**訊息簽署函式**
</dt> <dd>

用來簽署訊息和資料的函式。

另請參閱 [*簡化的訊息函數*](s-gly.md)。

</dd> <dt>

<span id="_security_mpr_gly"></span><span id="_SECURITY_MPR_GLY"></span>**Mpr**
</dt> <dd>

請參閱 *多個提供者路由器*。

</dd> <dt>

<span id="_security_multiple_provider_router_gly"></span><span id="_SECURITY_MULTIPLE_PROVIDER_ROUTER_GLY"></span>**多個提供者路由器**
</dt> <dd>

 (MPR) 系統元件來處理系統和所有網路提供者之間的通訊。 MPR 會呼叫每個網路提供者所公開的網路提供者函式。

</dd> </dl>
