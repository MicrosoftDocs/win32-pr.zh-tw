# <a name="microsoft-enhanced-cryptographic-provider"></a>Microsoft 增強型密碼編譯提供者

Microsoft 增強型密碼編譯提供者（稱為增強型提供者）支援與 Microsoft 基底密碼編譯提供者相同的功能，稱為「基底提供者」。 增強型提供者透過更長的金鑰和其他演算法來支援更強的安全性。 它可以與所有 CryptoAPI 版本搭配使用。

為了維持與舊版提供者版本的回溯相容性，提供者名稱（如 Wincrypt .h 標頭檔中所定義）會保留1.0 版指定。 不過，此提供者的2.0 版目前為出貨。 若要判斷使用中的提供者版本，請呼叫 [**CryptGetProvParam**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptgetprovparam) ，並將 *dwParam* 引數設定為 **PP \_ 版本**。 如果傳回0x0200，則會使用2.0 版。

|                |                        |
|----------------|------------------------|
| 提供者類型： | **>PROV \_ RSA \_ FULL**    |
| 提供者名稱： | **MS \_ 增強的 \_ >PROV** |



 

下表強調基底提供者、強式提供者和增強型提供者之間的差異。 所顯示的金鑰長度為預設金鑰長度。



| 演算法                                                                                | 基底提供者金鑰長度 | 強式提供者金鑰長度 | 增強的提供者金鑰長度                |
|------------------------------------------------------------------------------------------|--------------------------|----------------------------|---------------------------------------------|
| RSA 公開金鑰簽章演算法                                                       | 512位                 | 1024位                 | 1024位                                  |
| RSA 公開金鑰交換演算法                                                        | 512位                 | 1024位                 | 1024位                                  |
| RC2 區塊加密演算法                                                           | 40位                  | 128 位元                   | 可以設定128位 Salt 長度。<br/> |
| RC4 串流加密演算法                                                          | 40位                  | 128 位元                   | 可以設定128位 Salt 長度。<br/> |
| DES                                                                                      | 56位                  | 56位                    | 56位                                     |
| [*三重 DES*](../secgloss/t-gly.md) (2 金鑰)  | 不支援            | 112位                   | 112位                                    |
| 三重 DES (3 金鑰)                                                                        | 不支援            | 168位                   | 168位                                    |



 

強式提供者和增強型提供者與基底提供者回溯相容，不同之處在于提供者只能產生預設 [*金鑰長度*](../secgloss/k-gly.md)的 RC2 或 RC4 金鑰。 基底提供者的預設長度是40位。 增強型提供者的預設長度是128位。 因此，增強型提供者無法建立具有基底提供者相容金鑰長度的金鑰。 不過，增強型提供者可以匯入最多128位的 RC2 和 RC4 金鑰。 因此，增強型提供者可以匯入和使用使用基底提供者所產生的40位金鑰。
