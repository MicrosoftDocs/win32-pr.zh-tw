# <a name="certificate-services-architecture"></a>憑證服務架構

憑證服務是一種開發平臺，可為企業或安全的網際網路應用程式建立憑證授權單位單位。 已設定且可運作的憑證授權單位單位，可讓網站以最少量的系統管理額外負荷和最大的安全性來發出、追蹤、管理及撤銷憑證。

憑證服務包含伺服器引擎、伺服器資料庫，以及一組一起運作以做為憑證授權單位單位的模組和工具。 外部應用程式、模組和系統管理工具會使用元件物件模型 (COM) 介面與伺服器引擎互動。 下圖顯示伺服器引擎使用的介面：

![憑證服務架構](images/certapi.png)

操作認證系統通常會有四個主要子系統。



| Subsystem             | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
|-----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 用戶端                | 用戶端是使用者用來產生 [*憑證要求*](../secgloss/c-gly.md)、傳送要求和接收完成憑證的軟體。 用戶端的範例是 Microsoft Internet Explorer 第5版。 用戶端通常會與中繼應用程式所維護的自訂介面互動。                                                                                                                                                                                                                                                                                              |
| 仲介          | 媒介是由安裝程式) 中的中繼應用程式和憑證服務用戶端介面 (*憑證服務網頁用戶端* 所組成的子系統。 中繼應用程式會直接與用戶端互動、接收憑證要求，並傳回完成的憑證。 它會透過憑證服務用戶端介面（包含 [**ICertConfig**](/windows/desktop/api/Certcli/nn-certcli-icertconfig) 和 [**ICertRequest**](/windows/desktop/api/Certcli/nn-certcli-icertrequest) COM 介面）與伺服器引擎進行通訊。 Microsoft Internet Information Services 的中繼應用程式範例。 您可以透過使用中的伺服器頁面，完全實作為中繼應用程式。 |
| 伺服器                | 伺服器是組建憑證的系統。 除了伺服器引擎之外，還包含兩個可設定的元件;原則模組和結束模組。 原則模組會透過 [**ICertPolicy**](/windows/desktop/api/Certpol/nn-certpol-icertpolicy) 和 [**ICertServerPolicy**](/windows/desktop/api/Certif/nn-certif-icertserverpolicy) 介面與伺服器引擎互動。 結束模組 (可以有一個以上的) 透過 [**ICertExit**](/windows/desktop/api/Certexit/nn-certexit-icertexit) 和 [**ICertServerExit**](/windows/desktop/api/Certif/nn-certif-icertserverexit) 介面與伺服器引擎互動。                                                                                                                                                                                       |
| 系統管理用戶端 | 系統管理用戶端是監視和管理憑證和要求的系統。 系統管理用戶端會使用 [**ICertAdmin**](/windows/desktop/api/Certadm/nn-certadm-icertadmin) 介面與伺服器引擎進行通訊。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |



 

如需憑證服務架構的詳細資訊，請參閱 [密碼編譯介面](cryptography-interfaces.md)、 [建立憑證](building-a-certificate.md)和下列主題。



| 區段                                      | Content                                                                                                                                                                                                                                                                    |
|----------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [原則模組](policy-modules.md)         | 可自訂的程式，可在 [*憑證要求*](../secgloss/c-gly.md)評估期間使用;這些程式會強制執行憑證服務發出或拒絕要求的規則。 |
| [結束模組](exit-modules.md)             | 可自訂的程式，會在作業發生時收到來自伺服器引擎的通知，例如當憑證發出時。                                                                                                                                       |
| [延伸模組處理常式](extension-handlers.md) | 提供常式以便編碼更複雜的延伸模組和資料類型的 COM 物件。                                                                                                                                                                                 |
