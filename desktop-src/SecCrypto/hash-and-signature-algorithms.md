# <a name="hash-and-signature-algorithms"></a>雜湊和簽章演算法

下列演算法會計算 [*雜湊*](../secgloss/h-gly.md) 和 [*數位簽章*](../secgloss/d-gly.md)。 Microsoft Base、強式和增強型密碼編譯提供者都支援這些演算法。 這些演算法的內部詳細資料已超出本檔的範圍。 如需其他來源的清單，請參閱 [有關密碼編譯的其他檔](additional-documentation-on-cryptography.md)。



| 演算法                                     | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|  (CBC) MAC 的加密區塊鏈<br/>     | Microsoft 提供者所執行的其中一種演算法 (CALG \_ MAC) 是 (MAC) 的 [*區塊加密*](../secgloss/b-gly.md)[*訊息驗證碼*](../secgloss/m-gly.md)。 這個方法會使用區塊密碼來加密基本資料，然後使用最後一個加密的區塊做為 [*雜湊*](../secgloss/h-gly.md) 值。 用來建立 MAC 的加密演算法是建立工作階段金鑰時所指定的演算法。 <br/>                                                                            |
| HMAC<br/>                                | \_由 Microsoft 提供者所執行的 (CALG HMAC) 的演算法。 此演算法也會使用 [*對稱金鑰*](../secgloss/s-gly.md) 來建立 [*雜湊*](../secgloss/h-gly.md)，但比簡單的 [*加密區塊連結*](../secgloss/c-gly.md) (CBC) MAC 演算法更為複雜。 它可以搭配任何反復的密碼編譯雜湊演算法（例如 MD5 或 SHA-1）使用。 如需詳細資訊，請參閱 [建立 HMAC](creating-an-hmac.md)。<br/>                                                                                       |
| MD2、MD4 和 MD5<br/>                   | 這些雜湊演算法全都由 RSA Data Security，Inc. 所開發。這些演算法是以順序開發。 這三個都產生128位的雜湊值。 這三個都是已知的弱點，因此應該只在基於相容性而需要時使用。 針對新的程式碼，我們建議採用 SHA-1 系列的雜湊。<br/> 這些演算法是已知的，而且可以在 [*密碼*](../secgloss/c-gly.md)編譯的任何參考中詳細檢查。<br/>                                                                                                                                                           |
| 訊息驗證碼 (MAC) <br/>   | MAC 演算法類似于 [*雜湊*](../secgloss/h-gly.md) 演算法，但使用 [*對稱*](../secgloss/s-gly.md) 式 (會話) 機碼來計算。 需要原始 [*工作階段金鑰*](../secgloss/s-gly.md) 才能重新計算雜湊值。 重新計算的雜湊值是用來驗證基底資料未變更。 這些演算法有時也稱為索引雜湊演算法。 若要查看哪些 Microsoft 提供者支援 MAC，請參閱 [Microsoft 加密服務提供者](microsoft-cryptographic-service-providers.md)。<br/>    |
|  (SHA-1) 的安全雜湊演算法<br/>       | 此雜湊演算法是由 [*美國國家標準和技術協會*](../secgloss/n-gly.md) 所開發 (NIST) 和全國安全機構 (NSA) 。 此演算法的開發目的是要與 DSA (數位簽章演算法) 或 DSS (數位簽章標準) 一起使用。 此演算法會產生160位的 [*雜湊*](../secgloss/h-gly.md) 值。 已知 SHA-1 具有弱點，因此應該只在基於相容性而需要時使用。 針對新的程式碼，我們建議採用 SHA-1 系列的雜湊。<br/> |
| 安全雜湊演算法-2 (SHA-1) <br/>   | 此雜湊演算法是根據美國國家標準與 [*技術局*](../secgloss/n-gly.md) （ (NIST) 和國家安全機構 (NSA) ）作為 sha-1 的後續版本。 它有四個變異（224 SHA-1、sha-1、SHA-256、384 SHA-1 和 SHA-512），這些變數是根據其輸出中的位數來命名。 在這些情況下，會在 Microsoft AES 密碼編譯提供者中執行 256 SHA-1、SHA-384 和 SHA-512。<br/>                                                                                                                                |
| SSL3 用戶端授權演算法<br/> | 此演算法用於 SSL3 用戶端驗證。 在 SSL3 通訊協定中，MD5 雜湊和 SHA 雜湊的串連會以 RSA [*私密金鑰*](../secgloss/p-gly.md)進行簽署。 CryptoAPI 2.0 和 Microsoft Base 和增強型密碼編譯提供者，都支援使用 [*雜湊*](../secgloss/h-gly.md) 類型 CALG \_ SSL3 \_ SHAMD5。 如需詳細資訊，請參閱 [建立 CALG \_ SSL3 \_ SHAMD5 Hash](creating-a-calg-ssl3-shamd5-hash.md)。<br/>                                                                                                                                               |
