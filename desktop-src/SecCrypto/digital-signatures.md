# <a name="digital-signatures"></a>數位簽章

當收件者必須識別並驗證訊息寄件者時，可以使用 [*數位簽章*](../secgloss/d-gly.md)，以 [*純文字*](../secgloss/p-gly.md)形式散發訊息。 簽署訊息並不會改變訊息;它只會產生數位簽章字串，您可以將它與訊息配套或分開傳輸。 數位簽章是以寄件者的 [*私密金鑰*](../secgloss/p-gly.md)加密的簡短資料片段。 使用傳送者的 [*公開金鑰*](../secgloss/p-gly.md) 解密簽章資料，證明資料是由寄件者或有權存取寄件者私密金鑰的人員所加密。

使用 [*公開金鑰*](../secgloss/p-gly.md) 簽章演算法來產生數位簽章。 [*私密金鑰*](../secgloss/p-gly.md)會產生簽章，而且必須使用對應的公開金鑰來驗證簽章。 下圖顯示此程式。

![產生數位簽章](images/capi02.png)

從訊息建立數位簽章牽涉到兩個步驟。 第一個步驟包含建立 [*雜湊*](../secgloss/h-gly.md) 值 (也稱為訊息 [*摘要*](../secgloss/m-gly.md)) 訊息。 然後使用簽署者的私密金鑰簽署此雜湊值。 以下是建立數位簽章所需步驟的說明。

![從訊息建立數位簽章](images/capi06.png)

若要驗證簽章，需要訊息和簽章。 首先，必須以建立簽章的相同方式，從訊息建立雜湊值。 然後，會使用簽署者的公開金鑰來對簽章驗證此雜湊值。 如果雜湊值和簽章相符，您就可以確信訊息的確是簽署者最初簽署的訊息，而且尚未遭到篡改。 下圖說明驗證數位簽章的相關程式。

![驗證數位簽章](images/capi07.png)

雜湊值是由少量的二進位資料所組成，通常大約是160位。 這是使用 [*雜湊演算法*](../secgloss/h-gly.md)產生的。 本節稍後會列出一些演算法。

所有雜湊值都會共用下列屬性，不論使用的演算法為何：

-   雜湊值的長度取決於所使用的演算法類型，而且其長度與訊息的大小不盡相同。 最常見的雜湊值長度是128或160位。
-   每一對不相同的訊息都會轉譯成完全不同的雜湊值，即使這兩個訊息只有一個位不同也是一樣。 使用今日的技術時，找不到可轉譯為相同雜湊值，而不會中斷雜湊演算法的一對訊息並不可行。
-   每次使用相同的演算法來雜湊特定訊息時，就會產生相同的雜湊值。
-   所有的雜湊演算法都是單向的。 有了雜湊值，就無法復原原始訊息。 事實上，只有在指定雜湊值的情況下，才能判斷原始訊息的任何屬性。
