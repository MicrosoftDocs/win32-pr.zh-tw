<span data-ttu-id="61dd6-117">若要驗證簽章，需要訊息和簽章。</span><span class="sxs-lookup"><span data-stu-id="61dd6-117">To verify a signature, both the message and the signature are required.</span></span> <span data-ttu-id="61dd6-118">首先，必須以建立簽章的相同方式，從訊息建立雜湊值。</span><span class="sxs-lookup"><span data-stu-id="61dd6-118">First, a hash value must be created from the message in the same way the signature was created.</span></span> <span data-ttu-id="61dd6-119">然後，會使用簽署者的公開金鑰來對簽章驗證此雜湊值。</span><span class="sxs-lookup"><span data-stu-id="61dd6-119">This hash value is then verified against the signature by using the public key of the signer.</span></span> <span data-ttu-id="61dd6-120">如果雜湊值和簽章相符，您就可以確信訊息的確是簽署者最初簽署的訊息，而且尚未遭到篡改。</span><span class="sxs-lookup"><span data-stu-id="61dd6-120">If the hash value and the signature match, you can be confident that the message is indeed the one the signer originally signed and that it has not been tampered with.</span></span> <span data-ttu-id="61dd6-121">下圖說明驗證數位簽章的相關程式。</span><span class="sxs-lookup"><span data-stu-id="61dd6-121">The following diagram illustrates the process involved in verifying a digital signature.</span></span>

<span data-ttu-id="61dd6-123">雜湊值是由少量的二進位資料所組成，通常大約是160位。</span><span class="sxs-lookup"><span data-stu-id="61dd6-123">A hash value consists of a small amount of binary data, typically around 160 bits.</span></span> <span data-ttu-id="61dd6-124">這是使用 [*雜湊演算法*](../secgloss/h-gly.md)產生的。</span><span class="sxs-lookup"><span data-stu-id="61dd6-124">This is produced by using a [*hashing algorithm*](../secgloss/h-gly.md).</span></span> <span data-ttu-id="61dd6-125">本節稍後會列出一些演算法。</span><span class="sxs-lookup"><span data-stu-id="61dd6-125">A number of these algorithms are listed later in this section.</span></span>

<span data-ttu-id="61dd6-126">所有雜湊值都會共用下列屬性，不論使用的演算法為何：</span><span class="sxs-lookup"><span data-stu-id="61dd6-126">All hash values share the following properties, regardless of the algorithm used:</span></span>

-   <span data-ttu-id="61dd6-127">雜湊值的長度取決於所使用的演算法類型，而且其長度與訊息的大小不盡相同。</span><span class="sxs-lookup"><span data-stu-id="61dd6-127">The length of the hash value is determined by the type of algorithm used, and its length does not vary with the size of the message.</span></span> <span data-ttu-id="61dd6-128">最常見的雜湊值長度是128或160位。</span><span class="sxs-lookup"><span data-stu-id="61dd6-128">The most common hash value lengths are either 128 or 160 bits.</span></span>
-   <span data-ttu-id="61dd6-129">每一對不相同的訊息都會轉譯成完全不同的雜湊值，即使這兩個訊息只有一個位不同也是一樣。</span><span class="sxs-lookup"><span data-stu-id="61dd6-129">Every pair of non-identical messages translates into a completely different hash value, even if the two messages differ only by a single bit.</span></span> <span data-ttu-id="61dd6-130">使用今日的技術時，找不到可轉譯為相同雜湊值，而不會中斷雜湊演算法的一對訊息並不可行。</span><span class="sxs-lookup"><span data-stu-id="61dd6-130">Using today's technology, it is not feasible to discover a pair of messages that translate to the same hash value without breaking the hashing algorithm.</span></span>
-   <span data-ttu-id="61dd6-131">每次使用相同的演算法來雜湊特定訊息時，就會產生相同的雜湊值。</span><span class="sxs-lookup"><span data-stu-id="61dd6-131">Each time a particular message is hashed using the same algorithm, the same hash value is produced.</span></span>
-   <span data-ttu-id="61dd6-132">所有的雜湊演算法都是單向的。</span><span class="sxs-lookup"><span data-stu-id="61dd6-132">All hashing algorithms are one-way.</span></span> <span data-ttu-id="61dd6-133">有了雜湊值，就無法復原原始訊息。</span><span class="sxs-lookup"><span data-stu-id="61dd6-133">Given a hash value, it is not possible to recover the original message.</span></span> <span data-ttu-id="61dd6-134">事實上，只有在指定雜湊值的情況下，才能判斷原始訊息的任何屬性。</span><span class="sxs-lookup"><span data-stu-id="61dd6-134">In fact, none of the properties of the original message can be determined given the hash value alone.</span></span>
