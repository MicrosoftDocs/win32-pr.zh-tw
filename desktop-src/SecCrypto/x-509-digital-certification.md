# <a name="x509-digital-certification"></a>X.509 數位認證 (英文)

[*數位憑證*](../secgloss/d-gly.md)的主要工作就是提供主體 [*公開金鑰*](../secgloss/p-gly.md)的存取權。 憑證也會確認憑證的公開金鑰屬於憑證的主體。 例如， (CA) 的 [*憑證授權單位*](../secgloss/c-gly.md) 單位可以數位簽署特殊訊息 (包含某些使用者名稱的憑證資訊) ，例如「Alice」和她的公開金鑰。 這樣做的方式必須讓任何人都能確認憑證是由 CA 發出和簽署的。 如果 CA 是受信任的，且可以驗證 Alice 的憑證是由該 CA 所發行，則 Alice 的憑證的任何接收者都可以信任來自該憑證的 Alice 公開金鑰。

數位簽章的一般執行牽涉到簽署憑證的程式。

程式如下所示：

1.  Alice 傳送已簽署的 [*憑證要求*](../secgloss/c-gly.md) ，其中包含她的名稱、她的公開金鑰，以及可能的一些額外資訊給 CA。
2.  CA 會從 Alice 的要求建立訊息（ *m*）。 CA 會使用其私密金鑰來簽署訊息，並建立個別的簽章訊息（ *sig*）。 CA 會將訊息、 *m* 和簽章（ *sig*）傳回到 Alice。 同時， *m* 和 *Sig* 會形成 Alice 的憑證。
3.  Alice 將憑證的這兩個部分傳送給 Bob，以授與他公開金鑰的存取權。
4.  Bob 使用 CA 的公開金鑰來驗證簽章（ *sig* ）。 如果簽章證明有效，他接受憑證中的公開金鑰作為 Alice 的公開金鑰。

如同任何 [*數位簽章*](../secgloss/d-gly.md)，具有 CA 公開金鑰存取權的任何接收者都可以判斷特定 ca 是否已簽署憑證。 此流程不需要任何秘密資訊的存取權。 剛剛呈現的案例假設 Bob 可以存取 CA 的公開金鑰。 如果 Bob 的 CA 憑證包含該公開金鑰，就可以存取該金鑰。

[*X.509*](../secgloss/x-gly.md) 數位憑證不僅包含使用者的名稱和公開金鑰，也包含使用者的其他相關資訊。 這些憑證不是以數位階層的方式執行的石頭。 他們可讓 CA 為憑證的接收者提供一種信任，而不只是憑證主體的公開金鑰，也是憑證主體的其他相關資訊。 其他資訊也可以包含其他專案、電子郵件地址、簽署指定值檔的授權，或授權成為 CA 並簽署其他憑證。

X.509 憑證及許多其他憑證都有有效的持續時間。 憑證可能會過期，而且不再有效。 CA 可能會因為許多原因而撤銷憑證。 為了處理撤銷，CA 會維護並散發稱為 [*憑證撤銷清單*](../secgloss/c-gly.md) (CRL) 的已撤銷憑證清單。 網路使用者會存取 CRL 來判斷憑證的有效性。
