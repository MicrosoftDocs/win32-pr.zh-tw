# <a name="message-countersignatures"></a>訊息副署

有時簽署的訊息需要 [*副署*](../secgloss/c-gly.md)。 例如，使用者 A 可能會將已簽署的資料訊息傳送給使用者 B，要求 B 以檔中包含的條款來確認協定。 使用者 B 會解碼訊息、讀取條款，並在協定中來副署訊息。 然後，副署訊息會傳送回使用者 A。使用者 A 現在知道，而且可以證明該使用者 B 是否同意這些條款。
