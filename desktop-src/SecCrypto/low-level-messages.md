<span data-ttu-id="4b608-105">[*低層級訊息函式*](../secgloss/l-gly.md)會針對已接收的傳輸和解碼資料來編碼資料。</span><span class="sxs-lookup"><span data-stu-id="4b608-105">The [*low-level message functions*](../secgloss/l-gly.md) encode data for transmission and decode data that has been received.</span></span> <span data-ttu-id="4b608-106">低層級訊息函式也會解密並驗證已接收訊息的簽章。</span><span class="sxs-lookup"><span data-stu-id="4b608-106">Low-level message functions also decrypt and verify the signatures of received messages.</span></span>

<span data-ttu-id="4b608-107">使用低層級的訊息開啟函式開啟訊息時，它會保持開啟且可用 (維護其 [*狀態*](../secgloss/s-gly.md)) 直到關閉為止。</span><span class="sxs-lookup"><span data-stu-id="4b608-107">When a message is opened using a low-level message open function, it remains open and available (maintains its [*state*](../secgloss/s-gly.md)) until it is closed.</span></span> <span data-ttu-id="4b608-108">這可讓您使用多個呼叫 [**CryptMsgUpdate**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptmsgupdate) 函式來分次訊息。</span><span class="sxs-lookup"><span data-stu-id="4b608-108">This allows a message to be constructed piecemeal using multiple calls to the [**CryptMsgUpdate**](/windows/desktop/api/Wincrypt/nf-wincrypt-cryptmsgupdate) function.</span></span>

<span data-ttu-id="4b608-109">使用低層級的訊息函式需要比使用簡化的訊息函式更多的函式呼叫 (請參閱) 的 [簡化訊息](simplified-messages.md) 。</span><span class="sxs-lookup"><span data-stu-id="4b608-109">Using low-level message functions requires more function calls than using simplified message functions (see [Simplified Messages](simplified-messages.md)).</span></span> <span data-ttu-id="4b608-110">如果使用簡化的訊息函式，則會在 API 的函式內執行更多工作。</span><span class="sxs-lookup"><span data-stu-id="4b608-110">If the simplified message functions are used, more of the work is done inside the functions of the API.</span></span>

<span data-ttu-id="4b608-111">使用低層級訊息函式牽涉到額外的工作，以呼叫其他憑證或密碼編譯功能。</span><span class="sxs-lookup"><span data-stu-id="4b608-111">Using low-level message functions involves the additional work of making calls to other certificate or cryptographic functions.</span></span> <span data-ttu-id="4b608-112">例如，可能需要呼叫憑證函式的資料，才能初始化這些低層級訊息函式所使用的結構。</span><span class="sxs-lookup"><span data-stu-id="4b608-112">For example, data from calls to certificate functions may be needed to initialize structures used by these low-level message functions.</span></span> <span data-ttu-id="4b608-113">簡化的訊息函數會在內部初始化許多這些結構。</span><span class="sxs-lookup"><span data-stu-id="4b608-113">Simplified message functions initialize many of these structures internally.</span></span>
