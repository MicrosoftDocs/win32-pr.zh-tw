# <a name="digital-messages"></a>數位訊息

訊息將會定義為已編碼的任何資料。 訊息是編碼來傳輸給另一個人或實體的資料，或是從另一個人或實體收到的編碼資料。 在密碼編譯環境中，訊息通常是經過簽署、加密或兩者。 詞彙訊息通常會包含憑證。

在 CryptoAPI 2.0 版中，訊息處理函式分成 [*低層級的訊息函數*](../secgloss/l-gly.md) 和 [*簡化的訊息函數*](../secgloss/s-gly.md)。
