# <a name="i-wmi"></a>我 (WMI) 

[B](gloss-a.md) [C](gloss-c.md) [D](gloss-d.md) [E](gloss-e.md) [F](gloss-f.md) G [H](gloss-h.md) I J [K](gloss-k.md) [L](gloss-l.md) [M](gloss-m.md) [N](gloss-n.md) [O](gloss-o.md) [P](gloss-p.md) [Q](gloss-q.md) [R](gloss-r.md) [S](gloss-s.md) [T](gloss-t.md) [W](gloss-w.md) X Y Z

<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**類比**
</dt> <dd>

執行緒在一個進程中的功能，可採用另一個進程的安全性身分識別和許可權。

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**執行個體提供者**
</dt> <dd>

一種 COM 伺服器，它會提供類別的實例並執行 [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) 介面的方法，以支援資料抓取、修改、刪除、列舉或查詢處理。

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**內建事件**
</dt> <dd>

WMI 提供的事件，這是標準 WMI 資料模型變更的通知。 例如，建立新的 [**Win32 \_ 進程**](/windows/desktop/CIMWin32Prov/win32-process)實例是 [**\_ \_ InstanceCreationEvent**](--instancecreationevent.md)類型的內部事件。 另請參閱外建 [*事件*](gloss-e.md)。

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**儀錶**
</dt> <dd>

一種機制，用來報告系統硬體和軟體的狀態。 WMI 會根據 [*通用訊息模型 (CIM)*](gloss-c.md)，以類別建立系統模型來實行檢測。 .NET Framework 中的 WMI 會藉由新增屬性來建立已檢測的類別。

</dd> </dl>
