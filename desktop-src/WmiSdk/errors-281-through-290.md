# <a name="errors-281-through-290"></a>錯誤281到290

描述 WMI SNMP 提供者錯誤281到290。

[嚴重錯誤282](#fatal-error-282)

[嚴重錯誤283](#fatal-error-283)

## <a name="fatal-error-282"></a>嚴重錯誤282

<dl> <dt>

<span id="_282__Fatal_____fileName___line____Syntax_error._Last_token_read_is__token__"></span><span id="_282__fatal_____filename___line____syntax_error._last_token_read_is__token__"></span><span id="_282__FATAL_____FILENAME___LINE____SYNTAX_ERROR._LAST_TOKEN_READ_IS__TOKEN__"></span>**<282、嚴重>： " <fileName> ： <行 \#>：語法錯誤。上次讀取的權杖是 <token> "**
</dt> <dd>

當編譯器知道發生語法錯誤時，就會發生此錯誤，但無法對語法錯誤提供任何更明確的資訊。

</dd> </dl>

## <a name="fatal-error-283"></a>嚴重錯誤283

<dl> <dt>

<span id="_283__Fatal_____fileName___line_____Unrecognized_character__character__in_input_"></span><span id="_283__fatal_____filename___line_____unrecognized_character__character__in_input_"></span><span id="_283__FATAL_____FILENAME___LINE_____UNRECOGNIZED_CHARACTER__CHARACTER__IN_INPUT_"></span>**<283，嚴重>： " <fileName> ： <行 \# \#>：輸入中有無法辨識 <character> 的字元**
</dt> <dd>

輸入中有無法辨識的字元。 通常會忽略無法辨識的字元，而此錯誤訊息會報告第一個無法辨識的字元。

</dd> </dl>
