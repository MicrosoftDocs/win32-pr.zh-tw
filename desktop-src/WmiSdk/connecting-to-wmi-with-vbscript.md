<span data-ttu-id="cb80a-111">下列程式說明如何使用 **wbemscripting.swbemlocator** 連接到 WMI。</span><span class="sxs-lookup"><span data-stu-id="cb80a-111">The following procedure describes how to connect to WMI using **SWbemLocator**.</span></span>

<span data-ttu-id="cb80a-112">**使用 Wbemscripting.swbemlocator 連接到 WMI**</span><span class="sxs-lookup"><span data-stu-id="cb80a-112">**To connect to WMI using SWbemLocator**</span></span>

    ```VB

    ```

    

2.  <span data-ttu-id="cb80a-114">使用 [**ConnectServer**](swbemlocator-connectserver.md) 方法的呼叫來登入命名空間。</span><span class="sxs-lookup"><span data-stu-id="cb80a-114">Log on to the namespace using a call to the [**ConnectServer**](swbemlocator-connectserver.md) method.</span></span>

    ```VB

    Set objService = objLocator.ConnectServer(".", "root\cimv2")
    ```

    

    <span data-ttu-id="cb80a-115">如果您未在 [**ConnectServer**](swbemlocator-connectserver.md)的呼叫中指定電腦，則 WMI 會連接到本機電腦。</span><span class="sxs-lookup"><span data-stu-id="cb80a-115">If you do not specify a computer in the call to [**ConnectServer**](swbemlocator-connectserver.md), then WMI connects to the local computer.</span></span> <span data-ttu-id="cb80a-116">如果您沒有指定命名空間，則 WMI 會連接到登錄機碼中指定的命名空間。</span><span class="sxs-lookup"><span data-stu-id="cb80a-116">If you do not specify a namespace, then WMI connects to the namespace specified in the registry key.</span></span>

    <span data-ttu-id="cb80a-117">**HKEY \_本機 \_ 電腦** \\ **軟體** \\ **Microsoft** \\ **WBEM** \\ **腳本** \\ **預設命名空間**</span><span class="sxs-lookup"><span data-stu-id="cb80a-117">**HKEY\_LOCAL\_MACHINE**\\**SOFTWARE**\\**Microsoft**\\**WBEM**\\**Scripting**\\**Default Namespace**</span></span>

    <span data-ttu-id="cb80a-118">預設命名空間是 \\ 根 \\ cimv2。</span><span class="sxs-lookup"><span data-stu-id="cb80a-118">The default namespace is \\root\\cimv2.</span></span> <span data-ttu-id="cb80a-119">如需命名空間的詳細資訊，請參閱 [在 WMI 中建立](creating-hierarchies-within-wmi.md)階層。</span><span class="sxs-lookup"><span data-stu-id="cb80a-119">For more information about namespaces, see [Creating Hierarchies Within WMI](creating-hierarchies-within-wmi.md).</span></span>

3.  <span data-ttu-id="cb80a-120">使用 [**SWbemServices \_**](swbemservices-security-.md)方法的呼叫來設定模擬等級。</span><span class="sxs-lookup"><span data-stu-id="cb80a-120">Set the impersonation level with a call to the [**SWbemServices.Security\_**](swbemservices-security-.md) method.</span></span>

    ```VB
    objService.Security_.ImpersonationLevel = 3 
    ```

    

    <span data-ttu-id="cb80a-121">如需詳細資訊，請參閱 [使用 VBScript 設定預設進程安全性層級](setting-the-default-process-security-level-using-vbscript.md)。</span><span class="sxs-lookup"><span data-stu-id="cb80a-121">For more information, see [Setting the Default Process Security Level Using VBScript](setting-the-default-process-security-level-using-vbscript.md).</span></span>

4.  <span data-ttu-id="cb80a-122">實行腳本的用途。</span><span class="sxs-lookup"><span data-stu-id="cb80a-122">Implement the purpose of your script.</span></span>

    <span data-ttu-id="cb80a-123">WMI 公開各種腳本物件，這些物件是用來存取和操作網路上的資料。</span><span class="sxs-lookup"><span data-stu-id="cb80a-123">WMI exposes a variety of scripting objects that use to access and manipulate data across your network.</span></span> <span data-ttu-id="cb80a-124">如需詳細資訊，請參閱操作 WMI 的 [類別和實例資訊](manipulating-class-and-instance-information.md) 和 [腳本 API](scripting-api-for-wmi.md)。</span><span class="sxs-lookup"><span data-stu-id="cb80a-124">For more information, see [Manipulating Class and Instance Information](manipulating-class-and-instance-information.md) and [Scripting API for WMI](scripting-api-for-wmi.md).</span></span>

    ```VB
    Set objLocator = CreateObject("WbemScripting.SWbemLocator")
    Set objService = objLocator.ConnectServer(".", "root\cimv2")
    objService.Security_.ImpersonationLevel = 3
    Set Jobs = objService.ExecQuery("SELECT * FROM Win32_ScheduledJob")
    i=0
    For each Job in Jobs
        i = i+1   
        WScript.Echo Job.JobId & "  " & Job.Command & VBNewLine
    Next
    If i = 0 Then
        WScript.Echo "No Jobs Scheduled with the AT command were found"
    End If
    ```

    

<span data-ttu-id="cb80a-125">下列程式描述如何連接至 WMI，並使用標記來取得物件。</span><span class="sxs-lookup"><span data-stu-id="cb80a-125">The following procedure describes how to connect to WMI and retrieve an object using a moniker.</span></span>

<span data-ttu-id="cb80a-126">**若要連接至 WMI 並使用標記抓取物件**</span><span class="sxs-lookup"><span data-stu-id="cb80a-126">**To connect to WMI and retrieve an object using a moniker**</span></span>

1.  <span data-ttu-id="cb80a-127">使用輸入參數中的標記來呼叫 [**GetObject**](https://msdn.microsoft.com/library/ebdktb00(v=VS.71).aspx) 。</span><span class="sxs-lookup"><span data-stu-id="cb80a-127">Call [**GetObject**](https://msdn.microsoft.com/library/ebdktb00(v=VS.71).aspx) with a moniker in the input parameter.</span></span>

    ```VB
    'Or the more complex version
    ```

    <span data-ttu-id="cb80a-128">Moiniker 包含數個您可以用來連線到 WMI 的元素：</span><span class="sxs-lookup"><span data-stu-id="cb80a-128">The moiniker contains a number of elements that you can use to connect to WMI:</span></span>

    -   <span data-ttu-id="cb80a-129">"Winmgmts：" 會告知 WSH 使用 [腳本 API 物件](scripting-api-objects.md)。</span><span class="sxs-lookup"><span data-stu-id="cb80a-129">The "winmgmts:" tells WSH to use [Scripting API objects](scripting-api-objects.md).</span></span> <span data-ttu-id="cb80a-130">在此特定範例中，WSH 會知道它應該會傳回描述系統上第一個 Win32 Register-scheduledjob 的 SWbemObject \_ 。</span><span class="sxs-lookup"><span data-stu-id="cb80a-130">In this particular example, the WSH will know that it should return an SWbemObject that describes the first Win32\_scheduledJob on the system.</span></span> <span data-ttu-id="cb80a-131">其他可能傳回的物件將會是 SWbemCollection 或 [**SWbemServices**](swbemservices.md) 物件，取決於所述的名字。</span><span class="sxs-lookup"><span data-stu-id="cb80a-131">Other possible objects to return would be an SWbemCollection or an [**SWbemServices**](swbemservices.md) object, depending on what the moniker described.</span></span>

    -   <span data-ttu-id="cb80a-135">您可以選擇性地定義 WMI 物件的路徑。</span><span class="sxs-lookup"><span data-stu-id="cb80a-135">You can optionally define the path to the WMI object.</span></span> <span data-ttu-id="cb80a-136">這包括本機或遠端電腦、命名空間，以及類別的名稱。</span><span class="sxs-lookup"><span data-stu-id="cb80a-136">This includes either the local or remote computer, the namespace, as well as the name of the class.</span></span> <span data-ttu-id="cb80a-137">如需在 WMI 腳本中使用 VBScript [**GetObject**](https://msdn.microsoft.com/library/ebdktb00(v=VS.71).aspx) 的詳細資訊，請參閱 [建立實例](creating-an-instance.md) 和抓取 [wmi 實例](retrieving-an-instance.md)。</span><span class="sxs-lookup"><span data-stu-id="cb80a-137">For more information about using the VBScript [**GetObject**](https://msdn.microsoft.com/library/ebdktb00(v=VS.71).aspx) in WMI scripts, see [Creating an Instance](creating-an-instance.md) and [Retrieving a WMI Instance](retrieving-an-instance.md).</span></span>

2.  <span data-ttu-id="cb80a-138">除了抓取單一專案或集合，您也可以選擇取出 [**SWbemServices**](swbemservices.md) 物件 (如先前範例) 所述。</span><span class="sxs-lookup"><span data-stu-id="cb80a-138">Instead of retrieving a single item or collection, you can also choose to retrieve the [**SWbemServices**](swbemservices.md) object (as described in the previous example).</span></span> <span data-ttu-id="cb80a-139">之後，您就可以在傳回的物件上呼叫其他查詢。</span><span class="sxs-lookup"><span data-stu-id="cb80a-139">Afterwards, you can then call additional queries on the returned object.</span></span>

    ```VB
    Set objWMIService = GetObject("winmgmts:{impersonationLevel=impersonate}!\\" & strComputer & "\root\cimv2")
    Set colScheduledJobs = objWMIService.ExecQuery("Select * from Win32_ScheduledJob")
    For Each objJob in colScheduledJobs
        Wscript.Echo "Job ID: " & objJob.JobId & "Command: " & objJob.Command & VBNewLine
    Next
    ```

    

    <span data-ttu-id="cb80a-140">在上述範例中，模擬或 impersonationLevel = 3 是預設的進程安全性層級。</span><span class="sxs-lookup"><span data-stu-id="cb80a-140">In the previous example, impersonate, or impersonationLevel=3, is the default process security level.</span></span> <span data-ttu-id="cb80a-141">在下列範例中，除非您需要將進程安全性變更為 **委派**，否則不需要指定此進程的安全性層級。</span><span class="sxs-lookup"><span data-stu-id="cb80a-141">In the following example, it is not necessary to specify this process security level unless you need to change the process security to **delegate**.</span></span> <span data-ttu-id="cb80a-142">如需詳細資訊，請參閱 [使用 VBScript 設定預設進程安全性層級](setting-the-default-process-security-level-using-vbscript.md)。</span><span class="sxs-lookup"><span data-stu-id="cb80a-142">For more information, see [Setting the Default Process Security Level Using VBScript](setting-the-default-process-security-level-using-vbscript.md).</span></span>
