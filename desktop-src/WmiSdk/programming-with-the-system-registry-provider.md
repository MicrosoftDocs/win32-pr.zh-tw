# <a name="programming-with-the-system-registry-provider"></a>使用系統登錄提供者進行程式設計

下列主題描述應用程式如何使用系統登錄提供者來提供您所建立之類別的資料。



| 主題                                                                                                                      | 描述                                                                                                                                                                                                                    |
|----------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [定義 System Registry 提供者的類別](defining-classes-for-the-system-registry-provider.md)                 | 若要存取系統登錄，您必須建立將登錄資訊儲存在 Windows Management Instrumentation (WMI) 存放庫中的 Windows 類別。                                                                     |
| [使用系統登錄提供者作為屬性提供者](using-the-system-registry-provider-as-a-property-provider.md) | 系統登錄提供者會要求您使用數個限定詞來查看登錄屬性。                                                                                                                             |
| [描述登錄的資源](describing-a-resource-for-the-registry.md)                                       | 您可以存取登錄中描述系統資源的資料。 若要修改資訊，系統登錄提供者需要特定的語法。                                                                      |
| [存取未命名的登錄值](accessing-an-unnamed-registry-value.md)                                             | 若要存取大部分的登錄值，請建立一個類別來接收 WMI 中的值，或者您可以檢查特定的登錄屬性。 不過，系統登錄提供者需要特殊的語法來查看未命名的登錄值。 |
