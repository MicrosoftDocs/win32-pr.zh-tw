# <a name="lun-object"></a><span data-ttu-id="2259a-103">LUN 物件</span><span class="sxs-lookup"><span data-stu-id="2259a-103">LUN Object</span></span>

<span data-ttu-id="2259a-105">LUN (邏輯單元編號) 物件會將由硬體提供者所建立並由子系統所呈現之可定址儲存空間的邏輯單元建立模型。</span><span class="sxs-lookup"><span data-stu-id="2259a-105">A LUN (logical unit number) object models a logical unit of addressable storage space that is created by a hardware provider and surfaced by a subsystem.</span></span> <span data-ttu-id="2259a-106">每個 LUN 都包含至少一個 LUN plex，而後者又是由一或多個磁片磁碟機的範圍所組成。</span><span class="sxs-lookup"><span data-stu-id="2259a-106">Each LUN comprises at least one LUN plex, which is in turn composed of extents from one or more drives.</span></span>

## <a name="lun-types"></a><span data-ttu-id="2259a-107">LUN 類型</span><span class="sxs-lookup"><span data-stu-id="2259a-107">LUN Types</span></span>

<span data-ttu-id="2259a-108">VDS 支援五種 LUN 類型：簡單、跨距、等量、鏡像，以及具有同位的等量。</span><span class="sxs-lookup"><span data-stu-id="2259a-108">VDS supports five LUN types: simple, spanned, striped, mirrored, and striped with parity.</span></span> <span data-ttu-id="2259a-109">簡單、跨區和等量 Lun 都不能容錯;鏡像和同位 Lun 具有容錯能力。</span><span class="sxs-lookup"><span data-stu-id="2259a-109">Simple, spanned, and striped LUNs are non-fault tolerant; mirrored and parity LUNs are fault tolerant.</span></span> <span data-ttu-id="2259a-110">本節的其餘部分將說明每個 VDS LUN 類型。</span><span class="sxs-lookup"><span data-stu-id="2259a-110">The remainder of this section describes each of the VDS LUN types.</span></span>

-   <span data-ttu-id="2259a-111">簡單的 LUN 是一種非容錯 LUN，由單一磁片磁碟機的單一連續磁片磁碟機範圍組成。</span><span class="sxs-lookup"><span data-stu-id="2259a-111">A simple LUN is a non-fault tolerant LUN that is made up of a single contiguous drive extent from a single drive.</span></span> <span data-ttu-id="2259a-112">連續的範圍可以由單一區塊範圍或連結在一起的多個區塊範圍所組成。</span><span class="sxs-lookup"><span data-stu-id="2259a-112">The contiguous extent can comprise of a single range of blocks or multiple ranges of blocks linked together.</span></span>
-   <span data-ttu-id="2259a-113">跨距 LUN 是非容錯 LUN，由多個磁片磁碟機的多個不連續範圍所組成。</span><span class="sxs-lookup"><span data-stu-id="2259a-113">A spanned LUN is a non-fault tolerant LUN that is made up of multiple discontiguous extents from multiple drives.</span></span> <span data-ttu-id="2259a-114">資料會以線性方式寫入第一個磁片磁碟機上的每個範圍，直到所有第一個磁片區的範圍填滿，然後再填滿第二個磁片磁碟機上的每個範圍等等。</span><span class="sxs-lookup"><span data-stu-id="2259a-114">Data is written linearly to each of the extents on the first drive until all of the first drive extents are filled, and then to each of the extents on the second drive, and so on.</span></span> <span data-ttu-id="2259a-115">跨距 Lun 可有效地使用子系統中的磁片磁碟機空間，這些子系統包含各種不同大小的磁片磁碟機。</span><span class="sxs-lookup"><span data-stu-id="2259a-115">Spanned LUNs provide for efficient use of drive space in subsystems that comprise of drives of various sizes.</span></span>
-   <span data-ttu-id="2259a-116">等量 LUN 是非容錯 LUN，由多個磁片磁碟機的多個交錯、連續的範圍所組成。</span><span class="sxs-lookup"><span data-stu-id="2259a-116">A striped LUN is a non-fault tolerant LUN made up of multiple, interleaved, contiguous extents from multiple drives.</span></span> <span data-ttu-id="2259a-117">等量 Lun 使用 RAID-0 設定，因此資料會在參與磁片磁碟機的範圍中「等量」迴圈。</span><span class="sxs-lookup"><span data-stu-id="2259a-117">Striped LUNs use a RAID-0 configuration, such that data is "striped" cyclically across the extents on the contributing drives.</span></span> <span data-ttu-id="2259a-118">等量 Lun 最適合與相同大小、型號和製造商的磁片磁碟機搭配使用。</span><span class="sxs-lookup"><span data-stu-id="2259a-118">Striped LUNs work best with drives of the same size, model, and manufacturer.</span></span>
-   <span data-ttu-id="2259a-119">鏡像 Lun 是容錯 Lun，藉由將資料複製到多個 LUN 的 plex 來提供嚴重損壞修復。</span><span class="sxs-lookup"><span data-stu-id="2259a-119">Mirrored LUNs are fault tolerant LUNs that provide for disaster recovery by duplicating the data to multiple LUN plexes.</span></span> <span data-ttu-id="2259a-120">鏡像 LUN 中的每個 plex 都包含一份儲存在原始 plex 上的資料複本。</span><span class="sxs-lookup"><span data-stu-id="2259a-120">Each plex in a mirrored LUN contains a copy of the data that is stored on the original plex.</span></span> <span data-ttu-id="2259a-121">每個 plex 都位於不同的磁片磁碟機上。</span><span class="sxs-lookup"><span data-stu-id="2259a-121">Each of the plexes resides on a separate drive.</span></span> <span data-ttu-id="2259a-122">寫入至鏡像 LUN 的所有資料都會同時寫入至它的每個 plex。</span><span class="sxs-lookup"><span data-stu-id="2259a-122">All data that is written to a mirrored LUN is written simultaneously to each of its plexes.</span></span> <span data-ttu-id="2259a-123">如果其中一部參與的磁片磁碟機失敗，則該磁片磁碟機上的 plex 會變成無法使用，但系統會使用不受影響的 plex 或 plex 繼續運作。</span><span class="sxs-lookup"><span data-stu-id="2259a-123">Should one of the contributing drives fail, the plex on that drive becomes unavailable, but the system continues to operate using the unaffected plex or plexes.</span></span> <span data-ttu-id="2259a-124">鏡像的 LUN 可以有任意數目的 plex。</span><span class="sxs-lookup"><span data-stu-id="2259a-124">A mirrored LUN can have any number of plexes.</span></span>
-   <span data-ttu-id="2259a-125">具有同位 Lun 的等量是容錯 Lun，可透過在三個或更多磁片磁碟機上間歇性分割的同位資料來提供嚴重損壞修復。</span><span class="sxs-lookup"><span data-stu-id="2259a-125">Striped with parity LUNs are fault-tolerant LUNs that provide for disaster recovery by striping parity data intermittently across three or more drives.</span></span> <span data-ttu-id="2259a-126">如果其中一部參與的磁片磁碟機失敗，則可以從剩餘的資料和同位重新建立遺失的資料。</span><span class="sxs-lookup"><span data-stu-id="2259a-126">Should one of the contributing drives fail, the lost data can be recreated from the remaining data and parity.</span></span>

<span data-ttu-id="2259a-128">VDS 支援四個可讓應用程式建立 Lun 的模型：明確導向、部分導向、automagic 和供應商特定。</span><span class="sxs-lookup"><span data-stu-id="2259a-128">VDS supports four models by which applications can create LUNs: explicitly directed, partially directed, automagic, and vendor-specific.</span></span> <span data-ttu-id="2259a-129">所有硬體提供者都必須支援明確且部分導向的 LUN 建立，且強烈建議您支援 automagic LUN 建立。</span><span class="sxs-lookup"><span data-stu-id="2259a-129">All hardware providers must support explicitly and partially directed LUN creation, and are strongly encouraged to support automagic LUN creation.</span></span> <span data-ttu-id="2259a-130"> (廠商特定的 LUN 建立超出本指南的範圍。 ) </span><span class="sxs-lookup"><span data-stu-id="2259a-130">(Vendor-specific LUN creation is outside the scope of this guide.)</span></span>
