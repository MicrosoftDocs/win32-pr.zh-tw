<span data-ttu-id="f6ea0-107">當兩個 pin 連線時，他們必須同意連接的媒體類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-107">When two pins connect, they must agree on a media type for the connection.</span></span> <span data-ttu-id="f6ea0-108">媒體類型描述資料的格式。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-108">The media type describes the format of the data.</span></span> <span data-ttu-id="f6ea0-109">如果沒有媒體類型，篩選器可能會提供一種資料，只是讓另一個篩選器將它視為其他的篩選準則。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-109">Without the media type, a filter might deliver one kind of data, only to have another filter treat it as something else.</span></span>

<span data-ttu-id="f6ea0-110">協商媒體類型的基本機制是 [**IPin：： ReceiveConnection**](/windows/desktop/api/Strmif/nf-strmif-ipin-receiveconnection) 方法。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-110">The basic mechanism for negotiating media types is the [**IPin::ReceiveConnection**](/windows/desktop/api/Strmif/nf-strmif-ipin-receiveconnection) method.</span></span> <span data-ttu-id="f6ea0-111">輸出圖釘會以建議的媒體類型，在輸入 pin 上呼叫這個方法。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-111">The output pin calls this method on the input pin with a proposed media type.</span></span> <span data-ttu-id="f6ea0-112">輸入 pin 接受連接或拒絕連接。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-112">The input pin accepts the connection or rejects it.</span></span> <span data-ttu-id="f6ea0-113">如果拒絕連線，輸出 pin 可以嘗試其他媒體類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-113">If it rejects the connection, the output pin can try another media type.</span></span> <span data-ttu-id="f6ea0-114">如果找不到適合的類型，連接就會失敗。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-114">If no suitable types are found, the connection fails.</span></span> <span data-ttu-id="f6ea0-115">輸入 pin 可以選擇性地透過 [**IPin：： EnumMediaTypes**](/windows/desktop/api/Strmif/nf-strmif-ipin-enummediatypes) 方法，通告其慣用的類型清單。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-115">Optionally, the input pin can advertise a list of types that it prefers, through the [**IPin::EnumMediaTypes**](/windows/desktop/api/Strmif/nf-strmif-ipin-enummediatypes) method.</span></span> <span data-ttu-id="f6ea0-116">當輸出 pin 提議媒體類型時，就可以使用這份清單，雖然不需要這麼做。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-116">The output pin can use this list when it proposes media types, although it does not have to.</span></span>

<span data-ttu-id="f6ea0-117">**CTransformFilter** 類別會為此程式實作為一般架構，如下所示：</span><span class="sxs-lookup"><span data-stu-id="f6ea0-117">The **CTransformFilter** class implements a general framework for this process, as follows:</span></span>

-   <span data-ttu-id="f6ea0-118">輸入 pin 沒有慣用的媒體類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-118">The input pin has no preferred media types.</span></span> <span data-ttu-id="f6ea0-119">它完全依賴上游篩選器來提議媒體類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-119">It relies entirely on the upstream filter to propose the media type.</span></span> <span data-ttu-id="f6ea0-120">對於影片資料而言很有意義，因為媒體類型包含影像大小和畫面播放速率。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-120">For video data, this makes sense, because the media type includes the image size and the frame rate.</span></span> <span data-ttu-id="f6ea0-121">一般而言，該資訊必須由上游來源篩選器或剖析器篩選器提供。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-121">Typically, that information must be supplied by an upstream source filter or parser filter.</span></span> <span data-ttu-id="f6ea0-122">在音訊資料的情況下，可能會有一組較小的格式，因此輸入 pin 可能會提供一些慣用的類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-122">In the case of audio data, the set of possible formats is smaller, so it may be practical for the input pin to offer some preferred types.</span></span> <span data-ttu-id="f6ea0-123">在此情況下，請覆寫輸入釘選上的 [**CBasePin：： GetMediaType**](cbasepin-getmediatype.md) 。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-123">In that case, override [**CBasePin::GetMediaType**](cbasepin-getmediatype.md) on the input pin.</span></span>
-   <span data-ttu-id="f6ea0-124">當上游篩選提議媒體類型時，輸入 pin 會呼叫 [**CTransformFilter：： CheckInputType**](ctransformfilter-checkinputtype.md) 方法，以接受或拒絕該類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-124">When the upstream filter proposes a media type, the input pin calls the [**CTransformFilter::CheckInputType**](ctransformfilter-checkinputtype.md) method, which accepts or rejects the type.</span></span>
-   <span data-ttu-id="f6ea0-125">除非先連接輸入 pin，否則輸出 pin 將不會連接。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-125">The output pin will not connect unless the input pin is connected first.</span></span> <span data-ttu-id="f6ea0-126">這是轉換篩選準則的一般行為。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-126">This behavior is typical for transform filters.</span></span> <span data-ttu-id="f6ea0-127">在大部分的情況下，篩選準則必須先判斷輸入類型，才能設定輸出類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-127">In most cases, the filter must determine the input type before it can set the output type.</span></span>
-   <span data-ttu-id="f6ea0-128">當輸出連接連接時，它會有一份向下游篩選器提出的媒體類型清單。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-128">When the output pin does connect, it has a list of media types that it proposes to the downstream filter.</span></span> <span data-ttu-id="f6ea0-129">它會呼叫 [**CTransformFilter：： GetMediaType**](ctransformfilter-getmediatype.md) 方法來產生這份清單。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-129">It calls the [**CTransformFilter::GetMediaType**](ctransformfilter-getmediatype.md) method to generate this list.</span></span> <span data-ttu-id="f6ea0-130">輸出釘選也會嘗試下游篩選準則所建議的任何媒體類型。</span><span class="sxs-lookup"><span data-stu-id="f6ea0-130">The output pin will also try any media types that the downstream filter proposes.</span></span>
