# <a name="cbaserenderer-class"></a><span data-ttu-id="432a1-103">CBaseRenderer 類別</span><span class="sxs-lookup"><span data-stu-id="432a1-103">CBaseRenderer class</span></span>

![cbaserenderer 類別階層](images/rbase02.png)

<span data-ttu-id="432a1-105">`CBaseRenderer`類別是用來執行轉譯器篩選器的基類。</span><span class="sxs-lookup"><span data-stu-id="432a1-105">The `CBaseRenderer` class is a base class for implementing renderer filters.</span></span> <span data-ttu-id="432a1-106">它支援 [**CRendererInputPin**](crendererinputpin.md) 類別所執行的一個輸入 pin。</span><span class="sxs-lookup"><span data-stu-id="432a1-106">It supports one input pin, implemented by the [**CRendererInputPin**](crendererinputpin.md) class.</span></span> <span data-ttu-id="432a1-107">若要使用這個類別，請宣告繼承的衍生類別 `CBaseRenderer` 。</span><span class="sxs-lookup"><span data-stu-id="432a1-107">To use this class, declare a derived class that inherits `CBaseRenderer`.</span></span> <span data-ttu-id="432a1-108">衍生類別至少必須執行下列方法，這些方法會在基類中宣告為純虛擬：</span><span class="sxs-lookup"><span data-stu-id="432a1-108">At a minimum, the derived class must implement the following methods, which are declared as pure virtual in the base class:</span></span>

-   <span data-ttu-id="432a1-109">[**CBaseRenderer：： CheckMediaType**](cbaserenderer-checkmediatype.md)：接受或拒絕建議的媒體類型。</span><span class="sxs-lookup"><span data-stu-id="432a1-109">[**CBaseRenderer::CheckMediaType**](cbaserenderer-checkmediatype.md): Accepts or rejects proposed media types.</span></span> <span data-ttu-id="432a1-110">篩選會在 pin 連線程式期間呼叫這個方法。</span><span class="sxs-lookup"><span data-stu-id="432a1-110">The filter calls this method during the pin connection process.</span></span>
-   <span data-ttu-id="432a1-111">[**CBaseRenderer：:D orendersample**](cbaserenderer-dorendersample.md)：呈現範例。</span><span class="sxs-lookup"><span data-stu-id="432a1-111">[**CBaseRenderer::DoRenderSample**](cbaserenderer-dorendersample.md): Renders a sample.</span></span> <span data-ttu-id="432a1-112">篩選準則會針對它在執行時所收到的每個範例，呼叫這個方法。</span><span class="sxs-lookup"><span data-stu-id="432a1-112">The filter calls this method for every sample that it receives while running.</span></span>

<span data-ttu-id="432a1-113">基底類別會處理狀態變更和同步處理問題。</span><span class="sxs-lookup"><span data-stu-id="432a1-113">The base class handles state changes and synchronization issues.</span></span> <span data-ttu-id="432a1-114">它也會排程轉譯的範例，不過它不會執行任何品質控制量值。</span><span class="sxs-lookup"><span data-stu-id="432a1-114">It also schedules samples for rendering, although it does not implement any quality-control measures.</span></span> <span data-ttu-id="432a1-115">基類也會宣告數個 "handler" 方法。</span><span class="sxs-lookup"><span data-stu-id="432a1-115">The base class also declares several "handler" methods.</span></span> <span data-ttu-id="432a1-116">這些是篩選準則在串流處理常式中的特定點呼叫的方法。</span><span class="sxs-lookup"><span data-stu-id="432a1-116">These are methods that the filter calls at specific points in the streaming process.</span></span> <span data-ttu-id="432a1-117">它們不會在基類中執行任何動作，但衍生類別可以覆寫它們。</span><span class="sxs-lookup"><span data-stu-id="432a1-117">They do nothing in the base class, but the derived class can override them.</span></span> <span data-ttu-id="432a1-118">在接下來的表格中，它們會列在 [公用方法：處理常式] 標題下方。</span><span class="sxs-lookup"><span data-stu-id="432a1-118">In the table that follows, they are listed under the heading Public Methods: Handlers.</span></span>

<span data-ttu-id="432a1-119">[**CBaseRenderer：： OnReceiveFirstSample**](cbaserenderer-onreceivefirstsample.md)處理常式值得特別提及。</span><span class="sxs-lookup"><span data-stu-id="432a1-119">The [**CBaseRenderer::OnReceiveFirstSample**](cbaserenderer-onreceivefirstsample.md) handler deserves special mention.</span></span> <span data-ttu-id="432a1-120">篩選準則會在篩選暫停時，呼叫此方法以接收範例。</span><span class="sxs-lookup"><span data-stu-id="432a1-120">The filter calls this method if it receives a sample while the filter is paused.</span></span> <span data-ttu-id="432a1-121">如果圖形從停止切換為已暫停，或圖形在暫停時 seeked，可能會發生這種情況。</span><span class="sxs-lookup"><span data-stu-id="432a1-121">That can occur if the graph switches from stopped to paused, or if the graph is seeked while paused.</span></span> <span data-ttu-id="432a1-122">影片轉譯器通常會使用範例來顯示靜止框架。</span><span class="sxs-lookup"><span data-stu-id="432a1-122">Video renderers typically use the sample to display a still frame.</span></span> <span data-ttu-id="432a1-123">當篩選從暫停切換為執行時，它會將相同的範例傳送至 [**CBaseRenderer：:D orendersample**](cbaserenderer-dorendersample.md) 方法，作為串流中的第一個範例。</span><span class="sxs-lookup"><span data-stu-id="432a1-123">When the filter switches from paused to running, it sends the same sample to the [**CBaseRenderer::DoRenderSample**](cbaserenderer-dorendersample.md) method, as the first sample in the stream.</span></span>

<span data-ttu-id="432a1-124">`CBaseRenderer`類別會透過 [**CRendererPosPassThru**](crendererpospassthru.md)物件公開 **IMediaSeeking** 和 **IMediaPosition** 介面。</span><span class="sxs-lookup"><span data-stu-id="432a1-124">The `CBaseRenderer` class exposes the **IMediaSeeking** and **IMediaPosition** interfaces through the [**CRendererPosPassThru**](crendererpospassthru.md) object.</span></span> <span data-ttu-id="432a1-125">它會將所有搜尋要求傳遞至下一個篩選上游。</span><span class="sxs-lookup"><span data-stu-id="432a1-125">It passes all seek requests to the next filter upstream.</span></span>

## <a name="scheduling"></a><span data-ttu-id="432a1-126">排程</span><span class="sxs-lookup"><span data-stu-id="432a1-126">Scheduling</span></span>

<span data-ttu-id="432a1-127">當上游篩選呼叫輸入 pin 的 [**IMemInputPin：： receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive) 方法來傳遞範例時，pin 會將此呼叫傳遞給篩選器的 [**CBaseRenderer：： Receive**](cbaserenderer-receive.md) 方法。</span><span class="sxs-lookup"><span data-stu-id="432a1-127">When the upstream filter calls the input pin's [**IMemInputPin::Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive) method to deliver a sample, the pin passes this call to the filter's [**CBaseRenderer::Receive**](cbaserenderer-receive.md) method.</span></span> <span data-ttu-id="432a1-128">篩選器會卸載範例、立即加以轉譯，或排程它以進行轉譯。</span><span class="sxs-lookup"><span data-stu-id="432a1-128">The filter either drops the sample, renders it immediately, or schedules it for rendering.</span></span>

<span data-ttu-id="432a1-129">如果範例沒有時間戳記，或沒有可用的參考時鐘，則篩選會立即轉譯範例。</span><span class="sxs-lookup"><span data-stu-id="432a1-129">If the sample has no time stamps, or if no reference clock is available, the filter renders the sample immediately.</span></span> <span data-ttu-id="432a1-130">否則，篩選準則會呼叫 [**CBaseRenderer：： ShouldDrawSampleNow**](cbaserenderer-shoulddrawsamplenow.md) 方法來判斷要執行的動作。</span><span class="sxs-lookup"><span data-stu-id="432a1-130">Otherwise, the filter calls the [**CBaseRenderer::ShouldDrawSampleNow**](cbaserenderer-shoulddrawsamplenow.md) method to determine what to do.</span></span> <span data-ttu-id="432a1-131">根據預設，範例會根據其時間戳記進行排程。</span><span class="sxs-lookup"><span data-stu-id="432a1-131">By default, the sample is scheduled based on its time stamps.</span></span> <span data-ttu-id="432a1-132">衍生類別可以覆寫 **ShouldDrawSampleNow** 以支援品質控制。</span><span class="sxs-lookup"><span data-stu-id="432a1-132">The derived class can override **ShouldDrawSampleNow** to support quality control.</span></span>

<span data-ttu-id="432a1-133">若要排程範例，篩選準則會呼叫 [**IReferenceClock：： AdviseTime**](/windows/desktop/api/Strmif/nf-strmif-ireferenceclock-advisetime) 方法，以建立建議要求。</span><span class="sxs-lookup"><span data-stu-id="432a1-133">To schedule a sample, the filter calls the [**IReferenceClock::AdviseTime**](/windows/desktop/api/Strmif/nf-strmif-ireferenceclock-advisetime) method, which creates an advise request.</span></span> <span data-ttu-id="432a1-134">接收方法接著 [**會**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive) 封鎖直到排定的時間為止，或直到篩選變更狀態為止。</span><span class="sxs-lookup"><span data-stu-id="432a1-134">The [**Receive**](/windows/desktop/api/Strmif/nf-strmif-imeminputpin-receive) method then blocks until the scheduled time, or until the filter changes state.</span></span> <span data-ttu-id="432a1-135">封鎖可防止上游篩選器傳遞更多範例，直到轉譯目前的樣本為止。</span><span class="sxs-lookup"><span data-stu-id="432a1-135">Blocking prevents the upstream filter from delivering more samples until the current sample is rendered.</span></span>

<span data-ttu-id="432a1-136">當上游篩選準則呼叫 [**IPin：： EndOfStream**](/windows/desktop/api/Strmif/nf-strmif-ipin-endofstream) 方法以表示資料流程的結尾時，篩選會將 [**EC \_ 完成**](ec-complete.md) 事件傳送至篩選圖形管理員。</span><span class="sxs-lookup"><span data-stu-id="432a1-136">When the upstream filter calls the [**IPin::EndOfStream**](/windows/desktop/api/Strmif/nf-strmif-ipin-endofstream) method to signal the end of the stream, the filter sends an [**EC\_COMPLETE**](ec-complete.md) event to the filter graph manager.</span></span> <span data-ttu-id="432a1-137">篩選準則會在傳送事件之前等候目前的範例停止時間。</span><span class="sxs-lookup"><span data-stu-id="432a1-137">The filter waits for the current sample's stop time before sending the event.</span></span>
