# <a name="overview-of-mpeg-2-systems"></a>MPEG 2 系統總覽

本節提供 MPEG-2 系統層的一般非技術總覽。 MPEG-2 系統是定義在 MPEG-2 中如何多工音訊和影片串流的標準。

**基本資料流程**

MPEG-2 多工處理是以一或多個位元組串流（稱為基礎串流 (ES) ）開始，其中包含影片、音訊或其他資料。 例如，某段影片包含壓縮的影片畫面，加上序列標頭、圖片群組 (GOP) 標頭，以及將資料流程解碼所需的任何其他方法。 系統層不會定義 ES 位元組資料流程的內容。

基礎資料流程會細分成封包， (PE) 形成 *packetized 的基本資料流程* 。 PE 封包的長度不定。 封 *包的內容* 稱為「承載」。 每個 PE 封包也都包含一個標頭。 多工器會將1位元組資料流程識別碼指派給每個 PE;個別的 PE 封包是由封包標頭中的資料流程識別碼所識別。 針對音訊串流，資料流程識別碼的格式為 110 *xxxxx*。 若為影片，資料流程識別碼的格式為 1110 *yyyy*。

MPEG-2 標準定義了兩種傳遞 packetized 基本資料流程的方式： *程式串流* 和 *傳輸資料流程*。

**程式資料流程**

程式資料流程是針對相對無錯誤的環境而設計，例如本機檔案儲存體。 在程式資料流程中，會將 PE 封包多工化並組織成稱為「 *套件*」的單位。 程式資料流程中的所有 PE 資料流程都會同步處理至相同的時鐘。

**傳輸資料流程**

 (TS) 的傳輸串流是針對不可靠或容易出錯的環境所設計，例如網路廣播。 此外，它們也可以包含多個同步處理至不同時鐘的程式。 傳輸資料流程新增了第二層的 packetizing，也就是將 PE 資料流程封裝到傳輸資料流程封包內，每封包的大小都固定為188位元組。 TS 封包也可以包含程式資訊串流，如下一節所述。

每個 TS 封包都有一個4位元組的標頭，再加上一個選擇性的調整欄位，其中包含額外的標頭資訊。 多工器會將程式識別碼 (PID) 指派給每個 PE 資料流程或程式資訊串流。 Pid 可用來識別 TS 封包，類似于串流識別碼識別 PE 封包的方式。  (如果傳輸資料流程包含多個程式，則 *資料流程* 識別碼可能不是唯一的，但 PID 指派在傳輸資料流程中是唯一的。 ) 

**程式特定資訊**

因為傳輸資料流程可以包含多個程式，所以必須有方法將各種 PE 封包與它們所屬的程式產生關聯。 這是使用可識別程式資料流程的資料表來完成的。 這項資料統稱為程式特定資訊 (PSI) 。 PSI 資料會隨附于 TS 封包，就像 PE 資料一樣。 有各種類型的 PSI 資料，包括：

-   程式關聯資料表 (PAT) 。 PAT 一律會指派給 PID 0x000。 PAT 中的每個專案都是用來識別該程式之 PMT 封包的 PID (請參閱下一個專案) 。
-   程式對應表 (PMT) 。 每個 PMT 都會定義一個程式。 PMT 包含資料流程清單;每個資料表專案都會提供該資料流程的 PID，以及可識別資料流程類型的程式碼。 ISO/IEC 13818-1 定義一些標準串流類型;下表顯示縮寫清單。

    | 資料流程 \_ 類型 | Description  |
    |--------------|--------------|
    | 0x01         | MPEG-2 影片 |
    | 0x02         | MPEG-2 影片 |
    | 0x03         | MPEG-2 音訊 |
    | 0x04         | MPEG-2 音訊 |
    | 0x80-0xFF  | 使用者私用 |

    

     

    以 MPEG-2 為基礎的其他標準（例如 ATSC）可能會在「使用者私用」範圍中定義其他串流類型。 例如，ATSC 將0x81 定義為杜比 AC-3 音訊。

-   條件式存取資料表 (貓) 
-   網路識別表 (NIT)
