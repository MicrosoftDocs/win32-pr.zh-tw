<span data-ttu-id="9b90a-106">**CBaseAllocator** 類別是一種抽象基類，可實作為配置器。</span><span class="sxs-lookup"><span data-stu-id="9b90a-106">The **CBaseAllocator** class is an abstract base class that implements an allocator.</span></span> <span data-ttu-id="9b90a-107">配置器會公開 [**IMemAllocator**](/windows/desktop/api/Strmif/nn-strmif-imemallocator) 介面。</span><span class="sxs-lookup"><span data-stu-id="9b90a-107">Allocators expose the [**IMemAllocator**](/windows/desktop/api/Strmif/nn-strmif-imemallocator) interface.</span></span>

<span data-ttu-id="9b90a-108">配置 *器是可* 配置記憶體緩衝區的物件。</span><span class="sxs-lookup"><span data-stu-id="9b90a-108">An *allocator* is an object that allocates memory buffers.</span></span> <span data-ttu-id="9b90a-109">配置器會維護可用緩衝區的清單。</span><span class="sxs-lookup"><span data-stu-id="9b90a-109">The allocator maintains a list of available buffers.</span></span> <span data-ttu-id="9b90a-110">當用戶端 (通常是篩選) 要求緩衝區時，配置器會從清單中抓取一個。</span><span class="sxs-lookup"><span data-stu-id="9b90a-110">When a client (generally a filter) requests a buffer, the allocator retrieves one from the list.</span></span> <span data-ttu-id="9b90a-111">用戶端會將資料填滿緩衝區，而且可能會將緩衝區傳遞至另一個物件。</span><span class="sxs-lookup"><span data-stu-id="9b90a-111">The client fills the buffer with data, and might pass the buffer to another object.</span></span> <span data-ttu-id="9b90a-112">最後會釋出緩衝區，而配置器會將它傳回給可用的緩衝區清單。</span><span class="sxs-lookup"><span data-stu-id="9b90a-112">Eventually the buffer is released and the allocator returns it to the list of available buffers.</span></span>

<span data-ttu-id="9b90a-113">每個緩衝區都是由稱為 *媒體範例* 的物件所封裝。</span><span class="sxs-lookup"><span data-stu-id="9b90a-113">Each buffer is encapsulated by an object called a *media sample*.</span></span> <span data-ttu-id="9b90a-114">媒體範例是一種將指標封裝至元件物件模型中之記憶體區塊的方式， (COM) 架構。</span><span class="sxs-lookup"><span data-stu-id="9b90a-114">Media samples are a way to package pointers to memory blocks within the Component Object Model (COM) framework.</span></span> <span data-ttu-id="9b90a-115">媒體範例會公開 [**IMediaSample**](/windows/desktop/api/Strmif/nn-strmif-imediasample) 介面，並且使用 [**CMediaSample**](cmediasample.md) 類別來執行。</span><span class="sxs-lookup"><span data-stu-id="9b90a-115">Media samples expose the [**IMediaSample**](/windows/desktop/api/Strmif/nn-strmif-imediasample) interface, and are implemented using the [**CMediaSample**](cmediasample.md) class.</span></span> <span data-ttu-id="9b90a-116">媒體範例包含相關緩衝區的指標，可透過呼叫 [**IMediaSample：： GetPointer**](/windows/desktop/api/Strmif/nf-strmif-imediasample-getpointer) 方法來存取。</span><span class="sxs-lookup"><span data-stu-id="9b90a-116">A media sample contains a pointer to the associated buffer, which can be accessed by calling the [**IMediaSample::GetPointer**](/windows/desktop/api/Strmif/nf-strmif-imediasample-getpointer) method.</span></span> <span data-ttu-id="9b90a-117">如需詳細資訊，請參閱 [範例和配置器](samples-and-allocators.md)。</span><span class="sxs-lookup"><span data-stu-id="9b90a-117">For more information, see [Samples and Allocators](samples-and-allocators.md).</span></span>
