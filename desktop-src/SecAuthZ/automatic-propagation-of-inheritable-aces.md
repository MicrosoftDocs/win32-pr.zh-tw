<span data-ttu-id="26ce8-104">[**SetNamedSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setnamedsecurityinfoa)和 [**SetSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setsecurityinfo)函式支援將可繼承的 [*存取控制專案*](/windows/desktop/SecGloss/a-gly)自動傳播 (ace) 。</span><span class="sxs-lookup"><span data-stu-id="26ce8-104">The [**SetNamedSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setnamedsecurityinfoa) and [**SetSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setsecurityinfo) functions support automatic propagation of inheritable [*access control entries*](/windows/desktop/SecGloss/a-gly) (ACEs).</span></span> <span data-ttu-id="26ce8-105">例如，如果您使用這些函式將可繼承的 ACE 新增至 NTFS 中的目錄，系統會適當地將 ACE 套用至 [*存取控制清單*](/windows/desktop/SecGloss/a-gly) ， (acl) 任何現有的子目錄或檔案。</span><span class="sxs-lookup"><span data-stu-id="26ce8-105">For example, if you use these functions to add an inheritable ACE to a directory in an NTFS, the system applies the ACE as appropriate to the [*access control lists*](/windows/desktop/SecGloss/a-gly) (ACLs) of any existing subdirectories or files.</span></span>

<span data-ttu-id="26ce8-106">直接套用的 Ace 優先于繼承的 Ace。</span><span class="sxs-lookup"><span data-stu-id="26ce8-106">Directly applied ACEs have precedence over inherited ACEs.</span></span> <span data-ttu-id="26ce8-107">系統會在任意的存取控制清單中將直接套用的 Ace 放在 (DACL) 的 [*任意存取控制清單*](/windows/desktop/SecGloss/d-gly) 中，以實作為此優先順序。</span><span class="sxs-lookup"><span data-stu-id="26ce8-107">The system implements this precedence by placing directly applied ACEs ahead of inherited ACEs in a [*discretionary access control list*](/windows/desktop/SecGloss/d-gly) (DACL).</span></span> <span data-ttu-id="26ce8-108">當您呼叫 [**SetNamedSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setnamedsecurityinfoa) 和 [**SetSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setsecurityinfo) 函式來設定物件的安全性資訊時，系統會將目前的繼承模型強加在階層中所有物件的 acl 上（目標物件底下）。</span><span class="sxs-lookup"><span data-stu-id="26ce8-108">When you call the [**SetNamedSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setnamedsecurityinfoa) and [**SetSecurityInfo**](/windows/desktop/api/Aclapi/nf-aclapi-setsecurityinfo) functions to set the security information of an object, the system imposes the current inheritance model on the ACLs of all objects in the hierarchy below the target object.</span></span> <span data-ttu-id="26ce8-109">如果物件已轉換為目前的繼承模型，則 \_ \_ \_ \_ \_ \_ 會在物件 [*安全描述項*](/windows/desktop/SecGloss/s-gly) 的控制項欄位中設定「se DACL 自動繼承」和「se SACL 自動繼承位」。</span><span class="sxs-lookup"><span data-stu-id="26ce8-109">For objects that have been converted to the current inheritance model, the SE\_DACL\_AUTO\_INHERITED and SE\_SACL\_AUTO\_INHERITED bits are set in the control field of the [*security descriptor*](/windows/desktop/SecGloss/s-gly) of the object.</span></span>

<span data-ttu-id="26ce8-110">當您建立新的安全描述項以反映目前的繼承模型時，請小心不要變更安全描述項的語法。</span><span class="sxs-lookup"><span data-stu-id="26ce8-110">When you build a new security descriptor that reflects the current inheritance model, care is taken not to change the semantics of the security descriptor.</span></span> <span data-ttu-id="26ce8-111">如此一來，「允許」和「拒絕」 Ace 永遠不會彼此移動。</span><span class="sxs-lookup"><span data-stu-id="26ce8-111">As such, allow and deny ACEs are never moved in relation to one another.</span></span> <span data-ttu-id="26ce8-112">如果需要這類移動 (例如，將所有 noninherited Ace 放在 ACL) 的前方，ACL 會標示為受保護，以防止語義變更。</span><span class="sxs-lookup"><span data-stu-id="26ce8-112">If such movement is needed (for instance to place all noninherited ACEs at the front of an ACL), the ACL is marked as protected to prevent the semantic change.</span></span>

<span data-ttu-id="26ce8-113">將繼承的 Ace 傳播到子物件時，系統會使用下列規則：</span><span class="sxs-lookup"><span data-stu-id="26ce8-113">The system uses the following rules when propagating inherited ACEs to child objects:</span></span>

-   <span data-ttu-id="26ce8-114">如果沒有 DACL 的子物件繼承 ACE，則結果會是具有 DACL 的子物件，其中只包含繼承的 ACE。</span><span class="sxs-lookup"><span data-stu-id="26ce8-114">If a child object with no DACL inherits an ACE, the result is a child object with a DACL that contains only the inherited ACE.</span></span>
-   <span data-ttu-id="26ce8-115">如果具有空白 DACL 的子物件繼承 ACE，則結果會是具有 DACL 的子物件，其中只包含繼承的 ACE。</span><span class="sxs-lookup"><span data-stu-id="26ce8-115">If a child object with an empty DACL inherits an ACE, the result is a child object with a DACL that contains only the inherited ACE.</span></span>
-   <span data-ttu-id="26ce8-116">如果您從父物件移除可繼承的 ACE，自動繼承會移除子物件所繼承之任何 ACE 的複本。</span><span class="sxs-lookup"><span data-stu-id="26ce8-116">If you remove an inheritable ACE from a parent object, automatic inheritance removes any copies of the ACE that were inherited by child objects.</span></span>
-   <span data-ttu-id="26ce8-117">如果自動繼承會導致從子物件的 DACL 移除所有 Ace，則子物件具有空白的 DACL，而不是 DACL。</span><span class="sxs-lookup"><span data-stu-id="26ce8-117">If automatic inheritance results in the removal of all ACEs from the DACL of a child object, the child object has an empty DACL rather than no DACL.</span></span>

<span data-ttu-id="26ce8-118">這些規則可能會導致無法預期的結果，將沒有 DACL 的物件轉換為具有空白 DACL 的物件。</span><span class="sxs-lookup"><span data-stu-id="26ce8-118">These rules can have the unexpected result of converting an object with no DACL to an object with an empty DACL.</span></span> <span data-ttu-id="26ce8-119">沒有 DACL 的物件允許完整存取，但是具有空白 DACL 的物件不允許存取。</span><span class="sxs-lookup"><span data-stu-id="26ce8-119">An object with no DACL allows full access, but an object with an empty DACL allows no access.</span></span> <span data-ttu-id="26ce8-120">例如，假設您將可繼承的 ACE 新增至物件樹狀結構的根物件，就可以使用這些規則來建立空白的 DACL。</span><span class="sxs-lookup"><span data-stu-id="26ce8-120">As an example of how these rules can create an empty DACL, suppose you add an inheritable ACE to the root object of a tree of objects.</span></span> <span data-ttu-id="26ce8-121">自動繼承會將可繼承的 ACE 傳播至樹狀結構中的所有物件。</span><span class="sxs-lookup"><span data-stu-id="26ce8-121">Automatic inheritance propagates the inheritable ACE to all the objects in the tree.</span></span> <span data-ttu-id="26ce8-122">以沒有 DACL 啟動的子物件現在具有繼承 ACE 的 DACL。</span><span class="sxs-lookup"><span data-stu-id="26ce8-122">Child objects that started with no DACL now have a DACL with the inherited ACE.</span></span> <span data-ttu-id="26ce8-123">如果您從根物件移除可繼承的 ACE，系統會自動將變更傳播到子物件。</span><span class="sxs-lookup"><span data-stu-id="26ce8-123">If you remove the inheritable ACE from the root object, the system automatically propagates the change to the child objects.</span></span> <span data-ttu-id="26ce8-124">啟動但沒有 DACL 的子物件 (允許完整存取) 現在有空白的 DACL (不允許存取) 。</span><span class="sxs-lookup"><span data-stu-id="26ce8-124">Child objects that started with no DACL (allowing full access) now have an empty DACL (allowing no access).</span></span>
