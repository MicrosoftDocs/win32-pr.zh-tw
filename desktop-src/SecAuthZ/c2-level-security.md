# <a name="c2-level-security"></a>C2 層級安全性

下列清單包含 C2 層級安全性的一些最重要需求，如美國國防部所定義：

-   您必須授與或拒絕個別使用者或命名使用者群組的存取權，才能控制資源的存取權。
-   記憶體必須受到保護，如此一來，在 [*處理*](/windows/desktop/SecGloss/p-gly) 程式釋出之後，才能讀取其內容。 同樣地，安全檔案系統（例如 NTFS）必須防止讀取已刪除的檔案。
-   使用者必須在登入時以獨特的方式（例如密碼）識別自己。 所有可審核的動作都必須識別執行動作的使用者。
-   系統管理員必須能夠審核安全性相關事件。 不過，對安全性相關事件的存取權必須限制為授權的系統管理員。
-   系統必須受到保護，以防止外部干擾或篡改，例如修改執行中的系統或儲存在磁片上的系統檔案。
