# <a name="authorization-constants-authorization"></a>授權常數 (授權) 

授權常數會根據使用方式進行分類，如下所示。

## <a name="in-this-section"></a>本節內容



| 主題                                                                         | 描述                                                                                                                                                                                                                                                          |
|-------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**帳戶許可權常數**](account-rights-constants.md)<br/>       | 帳戶許可權會決定使用者帳戶可以執行的登入類型。 系統管理員會將帳戶許可權指派給使用者和群組帳戶。 每個使用者的帳戶許可權包含授與使用者的許可權，以及使用者所屬的群組。<br/> |
| [**應用程式容器 SID 常數**](app-container-sid-constants.md)<br/> | 指示應用程式封裝授權單位。<br/>                                                                                                                                                                                                                |
| [**審核常數**](auditing-constants.md)<br/>                   | 代表稽核原則事件的類別和子類別。<br/>                                                                                                                                                                                            |
| [**功能 SID 常數**](capability-sid-constants.md)<br/>       | 使用 [**AllocateAndInitializeSid**](/windows/win32/api/securitybaseapi/nf-securitybaseapi-allocateandinitializesid) 函式來定義應用程式的知名功能。<br/>                                                                                                                       |
| [**許可權常數**](privilege-constants.md)<br/>                 | 許可權會決定使用者帳戶可以執行的系統作業類型。 系統管理員將許可權指派給使用者和群組帳戶。 每個使用者的許可權包含授與使用者的許可權，以及使用者所屬的群組。<br/> |
