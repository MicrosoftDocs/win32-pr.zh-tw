# <a name="dns-standards-documents"></a>DNS 標準檔

網域名稱系統 (DNS) 是開放的通訊協定。 因此，整個產業的共同作業，確保在各種系統上執行的工作不會限制互通性。 標準本文會監督這類建議，也就是 (IETF) 的網際網路工程任務推動力。 以下是 (RFC) 的的 IETF 要求。 如需上述任何檔的詳細資訊，請造訪 [www.ietf.org](https://www.ietf.org/)。

## <a name="dns-rfcs"></a>DNS Rfc

-   [RFC 1034](https://www.ietf.org/rfc/rfc1034.txt)：功能變數名稱-概念和設備
-   [RFC 1035](https://www.ietf.org/rfc/rfc1035.txt)：功能變數名稱-實作為和規格
-   [RFC 1122](https://www.ietf.org/rfc/rfc1122.txt)：網際網路主機的需求-通訊層
-   [RFC 1123](https://www.ietf.org/rfc/rfc1123.txt)：網際網路主機需求-應用程式和支援
-   [RFC 1876](https://www.ietf.org/rfc/rfc1876.txt)：在網域名稱系統中表示位置資訊的方法
-   [RFC 1995](https://www.ietf.org/rfc/rfc1995.txt)： DNS 中的增量區域轉送
-   [RFC 1996](https://www.ietf.org/rfc/rfc1996.txt)： (DNS 通知的區域變更的提示通知機制) 
-   [RFC 2136](https://www.ietf.org/rfc/rfc2136.txt)：網域名稱系統中的動態更新 (DNS 更新) 
-   [RFC 2181](https://www.ietf.org/rfc/rfc2181.txt)：說明 DNS 規格
-   [RFC 2308](https://www.ietf.org/rfc/rfc2308.txt)： dns 查詢的反向快取 (dns NCACHE) 
-   [RFC 2535](https://www.ietf.org/rfc/rfc2535.txt)：網域名稱系統安全性延伸模組
-   [RFC 2782](https://www.ietf.org/rfc/rfc2782.txt)：用於指定服務位置的 dns RR (dns SRV) 
-   [RFC 2845](https://www.ietf.org/rfc/rfc2845.txt)： DNS (TSIG) 的秘密金鑰交易驗證
-   [RFC 2915](https://www.ietf.org/rfc/rfc2915.txt)：命名授權指標 (NAPTR) DNS 資源記錄
-   [RFC 2930](https://www.ietf.org/rfc/rfc2930.txt)： DNS (TKEY RR) 的秘密金鑰建立
-   [RFC 2931](https://www.ietf.org/rfc/rfc2931.txt)： ( SIG (0) s 的 DNS 要求和交易簽章 ) 
-   [RFC 3110](https://www.ietf.org/rfc/rfc3110.txt)：網域名稱系統中的 RSA/sha-1 SIGS 和 rsa 金鑰 (DNS) 
-   [RFC 3445](https://www.ietf.org/rfc/rfc3445.txt)：限制金鑰資源記錄的範圍 (RR) 
-   [RFC 3596](https://www.ietf.org/rfc/rfc3596.txt)：支援 IP 第6版的 DNS 擴充功能
-   [RFC 3645](https://www.ietf.org/rfc/rfc3645.txt)： DNS 的密碼金鑰交易驗證的一般安全性服務演算法 (GSS-TSIG)
