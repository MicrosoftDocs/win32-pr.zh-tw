# <a name="kernel-objects"></a><span data-ttu-id="8d986-103">核心物件</span><span class="sxs-lookup"><span data-stu-id="8d986-103">Kernel Objects</span></span>

<span data-ttu-id="8d986-104">核心物件控制碼是進程特定的。</span><span class="sxs-lookup"><span data-stu-id="8d986-104">Kernel object handles are process specific.</span></span> <span data-ttu-id="8d986-105">也就是說，處理常式必須建立物件或開啟現有的物件，才能取得核心物件控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-105">That is, a process must either create the object or open an existing object to obtain a kernel object handle.</span></span> <span data-ttu-id="8d986-106">核心控制碼的每個處理常式限制為 2 ^ 24。</span><span class="sxs-lookup"><span data-stu-id="8d986-106">The per-process limit on kernel handles is 2^24.</span></span> <span data-ttu-id="8d986-107">不過，處理常式會儲存在分頁集區中，因此您可以建立的實際控制碼數目是以可用的記憶體為基礎。</span><span class="sxs-lookup"><span data-stu-id="8d986-107">However, handles are stored in the paged pool, so the actual number of handles you can create is based on available memory.</span></span> <span data-ttu-id="8d986-108">您可以在32位 Windows 上建立的控制碼數目明顯低於 2 ^ 24。</span><span class="sxs-lookup"><span data-stu-id="8d986-108">The number of handles that you can create on 32-bit Windows is significantly lower than 2^24.</span></span>

<span data-ttu-id="8d986-109">任何程式都可以為現有的核心物件建立新的控制碼， (即使該進程知道物件的名稱，而且具有該物件的安全性存取權，也是由另一個進程) 所建立。</span><span class="sxs-lookup"><span data-stu-id="8d986-109">Any process can create a new handle to an existing kernel object (even one created by another process), provided that the process knows the name of the object and has security access to the object.</span></span> <span data-ttu-id="8d986-110">核心物件控制碼包括存取權限，指出可授與或拒絕處理常式的動作。</span><span class="sxs-lookup"><span data-stu-id="8d986-110">Kernel object handles include access rights that indicate the actions that can be granted or denied to a process.</span></span> <span data-ttu-id="8d986-111">應用程式會在建立物件時指定存取權限，或取得現有的物件控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-111">An application specifies access rights when it creates an object or obtains an existing object handle.</span></span> <span data-ttu-id="8d986-112">每種類型的核心物件都支援其本身的存取權限集。</span><span class="sxs-lookup"><span data-stu-id="8d986-112">Each type of kernel object supports its own set of access rights.</span></span> <span data-ttu-id="8d986-113">例如，事件控制碼可以設定或等候存取 (或兩個) ，檔案控制代碼可以有 (或) 等的讀取或寫入存取權。</span><span class="sxs-lookup"><span data-stu-id="8d986-113">For example, event handles can have set or wait access (or both), file handles can have read or write access (or both), and so on.</span></span> <span data-ttu-id="8d986-114">如需詳細資訊，請參閱 [安全物件](/windows/desktop/SecAuthZ/securable-objects)。</span><span class="sxs-lookup"><span data-stu-id="8d986-114">For more information, see [Securable Objects](/windows/desktop/SecAuthZ/securable-objects).</span></span>

<span data-ttu-id="8d986-115">在下圖中，應用程式會建立事件物件。</span><span class="sxs-lookup"><span data-stu-id="8d986-115">In the following illustration, an application creates an event object.</span></span> <span data-ttu-id="8d986-116">[**CreateEvent**](/windows/desktop/api/synchapi/nf-synchapi-createeventa)函數會建立事件物件，並傳回物件控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-116">The [**CreateEvent**](/windows/desktop/api/synchapi/nf-synchapi-createeventa) function creates the event object and returns an object handle.</span></span>

![建立事件物件的應用程式](images/cshob-03.png)

<span data-ttu-id="8d986-118">建立事件物件之後，應用程式可以使用事件控制碼來設定或等候事件。</span><span class="sxs-lookup"><span data-stu-id="8d986-118">After the event object has been created, the application can use the event handle to set or wait on the event.</span></span> <span data-ttu-id="8d986-119">控制碼會維持有效，直到應用程式關閉控制碼或終止為止。</span><span class="sxs-lookup"><span data-stu-id="8d986-119">The handle remains valid until the application closes the handle or terminates.</span></span>

<span data-ttu-id="8d986-120">大部分的核心物件都支援單一物件的多個控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-120">Most kernel objects support multiple handles to a single object.</span></span> <span data-ttu-id="8d986-121">例如，上圖中的應用程式可以使用 [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa) 函式來取得額外的事件物件控制碼，如下圖所示。</span><span class="sxs-lookup"><span data-stu-id="8d986-121">For example, the application in the preceding illustration could obtain additional event object handles by using the [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa) function, as shown in the following illustration.</span></span>

![建立具有多個控制碼之事件物件的應用程式](images/cshob-04.png)

<span data-ttu-id="8d986-123">這個方法可讓應用程式擁有具有不同存取權限的控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-123">This method enables an application to have handles with different access rights.</span></span> <span data-ttu-id="8d986-124">例如，控制碼1可能已設定並等候事件的存取，而處理2可能只會有等候存取權。</span><span class="sxs-lookup"><span data-stu-id="8d986-124">For example, Handle 1 might have set and wait access to the event, and Handle 2 might have only wait access.</span></span>

<span data-ttu-id="8d986-125">如果另一個進程知道事件名稱，而且具有物件的安全性存取權，則可以使用 [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa)來建立自己的事件物件控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-125">If another process knows the event name and has security access to the object, it can create its own event object handle by using [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa).</span></span> <span data-ttu-id="8d986-126">建立應用程式也可以使用 [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) 函式，將其中一個控制碼複製到相同的進程或另一個進程中。</span><span class="sxs-lookup"><span data-stu-id="8d986-126">The creating application could also duplicate one of its handles into the same process or into another process by using the [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) function.</span></span>

<span data-ttu-id="8d986-127">只要至少有一個物件控制碼存在，物件就會保留在記憶體中。</span><span class="sxs-lookup"><span data-stu-id="8d986-127">An object remains in memory as long as at least one object handle exists.</span></span> <span data-ttu-id="8d986-128">在下圖中，應用程式會使用 [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) 函式來關閉其事件物件控制碼。</span><span class="sxs-lookup"><span data-stu-id="8d986-128">In the following illustration, the applications use the [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) function to close their event object handles.</span></span> <span data-ttu-id="8d986-129">如果沒有任何事件控制碼，系統就會從記憶體中移除物件，如下圖所示。</span><span class="sxs-lookup"><span data-stu-id="8d986-129">When there are no event handles, the system removes the object from memory, as shown in the following illustration.</span></span>

![應用程式關閉事件物件控制碼，以從記憶體中移除物件](images/cshob-08.png)

<span data-ttu-id="8d986-131">系統管理檔案物件的方式與其他核心物件稍有不同。</span><span class="sxs-lookup"><span data-stu-id="8d986-131">The system manages file objects somewhat differently from other kernel objects.</span></span> <span data-ttu-id="8d986-132">檔案物件包含檔案指標，也就是要在檔案中讀取或寫入下一個位元組的指標。</span><span class="sxs-lookup"><span data-stu-id="8d986-132">File objects contain the file pointer — the pointer to the next byte to be read or written in a file.</span></span> <span data-ttu-id="8d986-133">只要應用程式建立新的檔案控制代碼，系統就會建立新的檔案物件。</span><span class="sxs-lookup"><span data-stu-id="8d986-133">Whenever an application creates a new file handle, the system creates a new file object.</span></span> <span data-ttu-id="8d986-134">因此，有一個以上的檔案物件可以參考磁片上的單一檔案，如下圖所示。</span><span class="sxs-lookup"><span data-stu-id="8d986-134">Therefore, more than one file object can refer to a single file on disk, as shown in the next illustration.</span></span>

![多個檔案物件參考磁片上的檔案](images/cshob-09.png)

<span data-ttu-id="8d986-136">只有透過重複或繼承才能有一個以上的檔案控制代碼參考相同的檔案物件，如下圖所示。</span><span class="sxs-lookup"><span data-stu-id="8d986-136">Only through duplication or inheritance can more than one file handle refer to the same file object, as shown in the following illustration.</span></span>

![兩個檔案控制代碼參考相同的檔案物件](images/cshob-10.png)
