# <a name="list-view-control-msaa-ui-element-reference"></a>List-View 控制項 (MSAA UI 元素參考) 

> [!Note]  
> 本主題說明 MSAA UI 專案參考的 **清單視圖控制項** 物件。 此處未說明如何在各種 UI 架構中建立 **清單視圖控制項** 物件。 請參閱您所使用之 UI 架構的 API 參考檔。

 

清單視圖控制項會顯示專案的集合，每個專案都包含一個圖示和一個標籤，並提供數種方式來顯示及排列專案。 例如，每個專案的其他資訊會顯示在圖示和標籤右邊的資料行中。 WindowsExplorer 會使用清單視圖控制項來顯示檔案。

清單視圖控制項的視窗類別名稱是 WC \_ LISTVIEW，在 Commctrl 中定義為 "SysListView"。

## <a name="notes"></a>備註

-   Microsoft Active Accessibility 不會在使用回呼函式將文字加入至專案的應用程式中公開清單視圖專案文字。
-   已裁剪的清單視圖專案沒有 [**狀態 \_ 系統 \_ 隱藏**](object-state-constants.md) 式設定。 若要判斷是否要裁剪清單專案，用戶端可以將 [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) 所傳回的座標與視窗的裁剪區域進行比較。
-   Microsoft Active Accessibility 不支援使用 [**accNavigate**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) 方法流覽至位於 [詳細資料] 視圖之清單視圖控制項內的標題控制項。 不過，用戶端會使用 [**AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) 函式來取得標題控制項。
-   若為非矩形物件（例如，在詳細資料檢視中的清單視圖專案），則 [**accLocation**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) 所抓取之專案周框矩形的座標會因為使用 [**accHitTest**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) 進行測試而失敗，因為 **accHitTest** 會以圖元為單位來決定物件的界限。
