# <a name="ui-automation-control-patterns-overview"></a><span data-ttu-id="9ae2f-114">UI 自動化控制項模式概觀</span><span class="sxs-lookup"><span data-stu-id="9ae2f-114">UI Automation Control Patterns Overview</span></span>

<span data-ttu-id="9ae2f-115">*控制項模式* 是介面執行，可將控制項功能的特定層面公開給 Microsoft 消費者介面自動化用戶端應用程式。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-115">A *control pattern* is an interface implementation that exposes a particular aspect of a control's functionality to Microsoft UI Automation client applications.</span></span> <span data-ttu-id="9ae2f-116">用戶端會使用透過控制項模式所公開的屬性和方法，以抓取控制項特定功能的相關資訊，或操作控制項行為的特定層面。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-116">Clients use the properties and methods exposed through a control pattern to retrieve information about a particular capability of the control, or to manipulate a particular aspect of the control's behavior.</span></span> <span data-ttu-id="9ae2f-117">例如，呈現表格式介面的控制項會使用 [方格](uiauto-implementinggrid.md) 控制項模式來公開資料表中的資料列和資料行數目，以及讓用戶端從資料表中取出專案。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-117">For example, a control that presents a tabular interface uses the [Grid](uiauto-implementinggrid.md) control pattern to expose the number of rows and columns in the table, and to enable a client to retrieve items from the table.</span></span>

<span data-ttu-id="9ae2f-118">消費者介面自動化使用控制項模式來表示常見的控制項行為。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-118">UI Automation uses control patterns to represent common control behaviors.</span></span> <span data-ttu-id="9ae2f-119">例如，您可以針對可以叫用的控制項（例如按鈕），以及具有捲軸的控制項（例如清單方塊、清單視圖或下拉式方塊）的[滾動](uiauto-implementingscroll.md)控制項模式，使用[Invoke](uiauto-implementinginvoke.md)控制項模式。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-119">For example, you use the [Invoke](uiauto-implementinginvoke.md) control pattern for controls that can be invoked, such as buttons, and the [Scroll](uiauto-implementingscroll.md) control pattern for controls that have scroll bars, such as list boxes, list views, or combo boxes.</span></span> <span data-ttu-id="9ae2f-120">由於每個控制項模式都代表不同的功能，因此可以結合控制項模式來描述特定控制項所支援的完整功能集。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-120">Because each control pattern represents a separate functionality, control patterns can be combined to describe the full set of functionality supported by a particular control.</span></span>

> [!Note]  
> <span data-ttu-id="9ae2f-121">您可以使用子控制項來建立匯總控制項，以提供父項所公開之功能的使用者介面，而父系應該執行通常與其子控制項相關聯的所有控制項模式。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-121">An aggregate control is built with child controls that provide the user interface for functionality that is exposed by the parent, and the parent should implement all control patterns that are typically associated with its child controls.</span></span> <span data-ttu-id="9ae2f-122">然後相同的控制項模式便不需要由子控制項所實作。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-122">In turn, those same control patterns are not required to be implemented by the child controls.</span></span>

## <a name="ui-automation-control-pattern-components"></a><span data-ttu-id="9ae2f-129">使用者介面自動化控制項模式元件</span><span class="sxs-lookup"><span data-stu-id="9ae2f-129">UI Automation Control Pattern Components</span></span>

<span data-ttu-id="9ae2f-130">控制項模式支援方法、屬性、事件和關聯性，這是定義控制項中可用之離散功能所需的關聯性。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-130">Control patterns support methods, properties, events, and relationships that are required to define a discrete piece of functionality available in a control.</span></span>

-   <span data-ttu-id="9ae2f-133">消費者介面自動化專案與其父系、子系和同級之間的關聯性描述消費者介面自動化樹狀目錄中的元素結構。</span><span class="sxs-lookup"><span data-stu-id="9ae2f-133">The relationship between a UI Automation element and its parent, children, and siblings describes the element structure in the UI Automation tree.</span></span>
