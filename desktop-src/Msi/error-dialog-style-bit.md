# <a name="error-dialog-style-bit"></a>錯誤對話方塊樣式位

如果設定此位，對話方塊會是錯誤對話方塊。

可以有一個以上的對話方塊具有這個樣式。 **ErrorDialog** 屬性會決定要使用哪一個對話方塊做為錯誤對話方塊。 選取的對話只能是已設定此樣式位的對話方塊之一。 錯誤對話方塊必須有一個名為 ErrorText 的靜態文字控制項。 此控制項會接收錯誤訊息的文字。 對話方塊也應該具有對應至可能傳回值的七個推播按鈕。 錯誤訊息會決定實際顯示的按鈕。 顯示的按鈕會重新排列，因此會平均地分散在對話方塊上。 此重新排列會變更按鈕的 X 座標，但不會變更其他三個座標。 因此，建議您不要在對話方塊的相同水準區域中撰寫其他控制項做為按鈕。 如果錯誤訊息未指定按鈕，則會顯示 [ **確定]** 按鈕。 [錯誤] 對話方塊會忽略 [ **預設** ] 按鈕、[第一個使用中控制項] 和 [ **取消** ] 按鈕的值。 錯誤訊息中定義的 **預設** 按鈕將會指派給這三個值。 推送這些按鈕的效果必須在 [ControlEvent 資料表](controlevent-table.md) 中定義，就像所有其他按鈕一樣。 對話方塊的標題會以類似于其他對話的方式來撰寫。 如果錯誤訊息指定了按鈕清單之後的標頭文字，就會覆寫它。

## <a name="value"></a>值



| Decimal | 十六進位 | 常數                       |
|---------|-------------|--------------------------------|
| 65536   | 0x00010000  | **msidbDialogAttributesError** |
