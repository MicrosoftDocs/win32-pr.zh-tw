# <a name="component-management"></a>元件管理

Windows Installer 藉由提高客戶在安裝和運行時間管理和維護應用程式元件的能力，來降低您應用程式的擁有權總成本 (TCO) 。 安裝資料庫會追蹤哪些應用程式需要特定元件、哪些檔案包含每個元件、每個檔案安裝在系統上的位置，以及元件來源所在的位置。 這可讓開發人員撰寫提供下列優點的封裝：

-   提高應用程式的復原能力。 您可以使用安裝程式來偵測和重新安裝損毀的元件，而不需要重新執行安裝程式。 安裝程式會在執行時間檢查元件的路徑。 這可將應用程式從靜態檔案路徑的相依性釋出，這通常會在電腦之間不同，並可指向遺失的元件。 如需詳細資訊，請參閱 [復原](resiliency.md)。
-   隨選安裝。 此功能集不會在安裝期間安裝，但在資料庫中指定為及時安裝，以供未來應用程式要求使用。 使用者不需要重新執行安裝程式。 如需詳細資訊，請參閱 [隨選安裝](installation-on-demand.md)。
-   在使用者介面中通告功能、應用程式或整個產品的快捷方式。 使用者可以使用快捷方式隨選安裝。 使用者也可以視需要移除功能、應用程式或整個產品。 如需詳細資訊，請參閱 [公告](advertisement.md)。
-   安裝自訂。 系統管理員可以將轉換套用至資料庫，以量身訂做特定使用者群組的安裝。 如需詳細資訊，請參閱[自訂](customization.md)。
-   更輕鬆地部署應用程式更新。 使用安裝程式來更新您的產品。 如需詳細資訊，請參閱 [修補和升級](patching-and-upgrades.md)。
-   功能快捷方式顯示。 安裝程式會顯示在本機執行之功能的快捷方式，以及遠端執行功能的快捷方式。 由於安裝資料庫會指定每個功能的執行內容，因此您可以視需要向使用者呈現可見的相等進入點。
-   保留使用計量的功能。 開發人員可以提供安裝套件，以保留所有用戶端應用程式的功能使用計數，並移除未使用的元件。
-   納入安裝。 開發人員可以撰寫安裝套件，並在其應用程式程式碼中使用 [安裝程式功能](installer-functions.md) ，將安裝程式的元件管理功能併入應用程式中。 下圖說明要求安裝功能的應用程式。
