# <a name="ice43"></a>ICE43

ICE43 會驗證未將功能視為其目標 (非公告快速鍵的快捷方式) 是在具有 HKCU 登錄專案作為其金鑰路徑的元件中。

## <a name="result"></a>結果

如果非公告的快捷方式位於沒有 HKCU 登錄專案的元件中，ICE43 會張貼錯誤訊息做為它的金鑰路徑。

## <a name="example"></a>範例

ICE43 會針對所顯示的範例報告下列錯誤。



| ICE43 錯誤                                                                                                                             | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|-----------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 元件 Component1 具有非公告的快捷方式。 它必須使用 HKCU 下的登錄機碼作為其 KeyPath，而不是檔案。                    | Component1 的 attributes 資料行是0，這表示元件會使用檔案作為其 KeyPath。 如此一來，就只會針對電腦上的第一個使用者安裝此元件中的非公告快捷方式。 稍後安裝元件的使用者看不到快捷方式，因為該元件在電腦上會顯示為已存在的安裝程式。 若要修正這個錯誤，請設定屬性的 RegistryKeyPath 位以將元件切換至登錄專案，然後將 KeyPath 值變更為登錄資料表中的有效專案。<br/> |
| 元件 Component2 具有非公告的快捷方式。 它必須使用 HKCU 下的登錄機碼作為其 KeyPath。 KeyPath 目前為 null。 | [屬性] 資料行設定為使用登錄，但 KeyPath 為 null。 KeyPath 必須參考登錄表中的專案。 若要修正這個錯誤，請將 KeyPath 值變更為登錄表中的有效專案。<br/>                                                                                                                                                                                                                                                                                                                               |
| 元件 Component3 具有非公告的快捷方式。 其 KeyPath 登錄機碼必須落在 HKCU 下。                                       | [屬性] 資料行設定為使用登錄，但參考的登錄專案不在 HKCU 下。 若要修正這個錯誤，請切換到不同的登錄專案作為此元件的 KeyPath，或將登錄專案的根值變更為-1 或1。<br/>                                                                                                                                                                                                                                                                             |
| 元件 Component4 的 KeyPath 登錄專案不存在。                                                                     | 元件的 KeyPath 資料行中所參考的登錄專案不在登錄資料表中。 若要修正這個錯誤，請建立一個專案。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 登錄專案 Reg5 設定為元件 Component5 的 KeyPath，但該登錄專案不屬於 Component5。          | 在 HKCU 樹狀目錄下的元件的 KeyPath 資料行中，有一個登錄專案參考，但是登錄專案的 Component 資料 \_ 行並未回頭參考到列為 KeyPath 的相同元件。 這表示，只有在安裝其他元件時，才會建立用來做為元件 KeyPath 的登錄專案。 若要修正這個錯誤，請將 KeyPath 值變更為參考屬於元件的登錄專案，或是將登錄專案變更為屬於 KeyPath 的元件。<br/>   |
