# <a name="volumecostlist-control"></a>VolumeCostList 控制項

VolumeCostList 控制項會顯示與不同磁片區上的選取相關聯的成本相關資訊。 此控制項會顯示目前安裝所涉及的所有磁片區，以及屬性位中指定之類型的所有磁片區。 如果所需的磁碟空間超過某些磁片磁碟機上的可用數量，則會在表格中反白顯示該磁片區。 按一下資料行標頭會根據所選的資料行來排序磁片區。

下列字串會用於控制項中的資料行標題，而且必須在 [UIText 資料表](uitext-table.md)中指定：

-   **VolumeCostAvailable**
-   **VolumeCostDifference**
-   **VolumeCostRequired**
-   **VolumeCostSize**
-   **VolumeCostVolume**

下列四個當地語系化字串用於格式化檔案的大小：

-   **Bytes**
-   **KB**
-   **MB**
-   **GB**

作者可以在任何字型規格之後附加資料行寬度，藉以設定 VolumeCostList 控制項中的資料行寬度。 資料行寬度是以大括弧括住的一連串正整數來輸入。 空的大括弧或 {0} 隱藏資料行。 負數或無法轉換成正整數的字串是不正確資料行寬度。 當數列中出現不正確資料行寬度時，會隱藏其餘的資料行。 最多可以指定五個數據行的寬度。

VolumeCostList 控制項絕不會顯示 [控制項資料表](control-table.md) 之文字欄位的內容。 相反地，這個欄位會指定控制項顯示的文字樣式，並包含螢幕審核公用程式所使用之控制項的描述。 若要設定文字字串的字型與字型樣式，請在顯示的字元字串前面加上 { \\ style} 或 {&style}。 其中 style 是在 [資料行] 資料表的 [樣式 [表單](textstyle-table.md)] 資料行中所列的識別碼。 如果這兩個都不存在，但 [**DefaultUIFont**](defaultuifont.md) 屬性定義為有效的文字樣式，則會使用該字型。 下列資訊是由螢幕審核公用程式讀取，作為控制項的描述。 請參閱 [協助工具](accessibility.md)。

請注意，當使用者在 [PathEdit 控制項](pathedit-control.md)、 [流覽對話方塊](browse-dialog.md)、 [DirectoryList 控制項](directorylist-control.md)或 [DirectoryCombo 控制項](directorycombo-control.md) 中輸入不同的路徑時，安裝程式不會更新 VolumeCostControl 的內容，因為沒有與 VolumeCostList 控制項相關聯的屬性。
