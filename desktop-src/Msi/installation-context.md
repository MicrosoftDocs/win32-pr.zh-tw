# <a name="installation-context"></a>安裝內容

Windows安裝程式可以將電腦上的封裝安裝到兩個安裝內容：每部電腦和每個使用者。 需要個別電腦安裝封裝，才能讓電腦的所有使用者存取和使用應用程式。 因為每部電腦安裝會對影響所有使用者的系統進行變更，所以具有有限許可權的標準使用者可能會無法在不先取得許可權的情況下，將套件安裝到個別電腦的內容中。

您可以針對每個使用者或每部電腦安裝撰寫套件，並使用 [**ALLUSERS**](allusers.md) 和 [**MSIINSTALLPERUSER**](msiinstallperuser.md) 屬性，來指定安裝內容。 根據這些屬性，Windows Installer 會自動將資料夾屬性和註冊的值重新導向至每個使用者或每個電腦內容的位置。

> [!Note]  
> [**MSIINSTALLPERUSER**](msiinstallperuser.md)屬性（從 Windows Installer 5.0 和 Windows Server 2008 R2 和 Windows 7 開始提供）可加速開發能夠安裝在每部電腦或每一使用者內容中的單一套件。 如需開發可讓使用者在安裝時選擇安裝內容之雙重用途封裝的相關資訊，請參閱 [單一套件撰寫](single-package-authoring.md)。 Windows如果 [**ALLUSERS**](allusers.md)的值不是2，則安裝程式會忽略 **MSIINSTALLPERUSER** 屬性。 Windows當安裝程式在每一部電腦的內容中安裝時，安裝程式一律會將 **allusers** 的值重設為1，然後將 **allusers** 的值重設為空字串 ( "" ) 在每個使用者的內容中安裝時。

|-----------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------|
| 應用程式會出現在電腦上的 [新增/移除程式] 底下，主控台給電腦的所有使用者。 <br/> | 應用程式只會顯示在已安裝應用程式之使用者的主控台上的 [新增/移除程式] 下。 <br/> |
| 快速鍵會安裝到 [所有使用者] 設定檔。<br/>                                              | 快速鍵只會安裝到該使用者的設定檔。<br/>                                                                 |
| 圖示和轉換會儲存在% WINDOWS% \\ Installer \\ {ProductCode} 中。<br/>                        | 圖示和轉換會儲存在% USERPROFILE% 的 \\ 應用程式資料中 \\ Microsoft \\ INSTALLER \\ {ProductCode GUID}<br/>         |
