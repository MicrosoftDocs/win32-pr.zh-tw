# <a name="session-control"></a>會話控制項

會話或呼叫代表兩個或更多位址之間的連接。 此連接是動態的，且必須在必要時建立、操作和釋放相關聯的程式設計物件。 在最簡單的情況下，這表示會進行通話並中斷連線。 針對先進的應用程式，會話控制可能牽涉到透過 IP 網路管理多媒體會議到個別參與者的層級。

會話控制項牽涉到兩個基本區域：

-   [會話作業](session-operations-ovr.md) 是初始化、維護和終止通訊會話的控制項。
-   [會話資訊](session-information-ovr.md) 描述可針對通訊會話取得的資料。

本節未涵蓋兩種特製化的會話控制項類型。 如需自動通話配送中心的相關資訊，請參閱 (TAPI 2.x 的 [電話中心控制](./call-center-control.md)) 或 (TAPI 3.X 的 [電話中心控制項](about-call-center-controls.md)) 。 如需 IP 會議的相關資訊，請參閱 [關於會合 Ip 電話語音會議](about-rendezvous-ip-telephony-conferencing.md)。
