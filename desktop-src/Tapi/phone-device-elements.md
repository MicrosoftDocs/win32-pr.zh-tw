# <a name="phone-device-elements"></a>電話裝置元素

電話裝置是一種支援 phone 裝置類別的裝置，其中包含部分或全部的下列元素：

-   **Hookswitch/感應器**：這是音訊輸入和輸出的方法。 手機裝置可以有數個可啟動和停用的感應器 (在應用程式或手動使用者控制下 offhook 或放置 onhook) 。

    電話語音辨識許多電話組通用的三種 hookswitch 裝置類型：

     **話筒**：傳統的嘴和 ear 組合必須以手動方式從底座中拔出，並保留在使用者的 ear 上。  
    **話筒**：讓使用者可以自由地進行通話。 話筒可能是在電話裝置內部或外部。 話筒的喇叭部分允許多個接聽程式。  
    **耳機**：讓使用者可以自由地進行通話。  
    

    Hookswitch 必須是 offhook，以允許將音訊資料傳送給對應的感應器，以及/或接收。

-   音量控制/增益控制/靜音：每個 hookswitch 裝置都是喇叭和麥克風元件的配對。 此 API 提供喇叭元件的音量控制和靜音，以及取得麥克風元件的控制權或靜音。
-   [響鈴](ring.md)：警示使用者的一種方式，通常是透過鐘。 電話裝置可能會以各種模式或模式環形。
-   [顯示](display.md)：以視覺化方式向使用者呈現訊息的機制。 手機顯示的特性是其資料列和資料行的數目。
-   [電話按鈕](phone-buttons.md)：按鈕的陣列。 當使用者按下電話組上的按鈕時，API 會報告對應的按鈕是否已按下。 按鈕燈識別碼可識別按鈕和燈配對。 當然，有可能會有按鈕的燈配對，而不是按鈕或無燈。 按鈕燈識別碼是整數值，範圍從0到電話裝置上可用的按鈕-燈泡最大數目（減一）。 每個按鈕都屬於按鈕類別。 類別包括呼叫外觀按鈕、功能按鈕、鍵盤按鈕和本機按鈕。
-   [燈](lamps.md)：燈的陣列 (例如 led) 可從 API 個別控制。 您可以使用不同的模式來發亮燈，方法是改變開和關的頻率。 按鈕燈泡識別碼可識別燈泡。
-   [資料區域](data-areas.md)：電話裝置中的記憶體區域，可在其中下載指示程式碼或資料，以及從中或從中上傳。 下載的資訊會影響 (的行為，也就是) 電話裝置的程式。

TAPI 可讓應用程式監視和控制電話裝置的元素。 應用程式最有用的元素是 hookswitch 裝置。 電話組可以像音訊 i/o 裝置一樣 (電腦) 具有音量控制、取得控制權和靜音、使用響鈴 (來警示使用者) 、將電話進行程式設計 (的資料區域，甚至是顯示，不過電腦的顯示器功能更強大。 應用程式寫入器不建議直接控制或使用電話燈或電話按鈕，因為燈光和按鈕功能在電話組間可能會有很大的差異，而且應用程式可以快速地針對特定的電話組量身打造。

所有電話裝置都不支援任何保證核心的服務集，因為線路裝置 (基本的電話語音服務) 。 因此，在應用程式可以使用手機裝置之前，應用程式必須先判斷電話裝置的確切功能。 電話語音功能會因設定 (用戶端和用戶端/伺服器) 、電話硬體和服務提供者軟體而有所不同。 應用程式應該不會假設有哪些電話語音功能可供使用。 應用程式會藉由呼叫 [**phoneGetDevCaps**](/windows/desktop/api/Tapi/nf-tapi-phonegetdevcaps) 函式來判斷電話裝置的裝置功能。 手機的裝置功能指出系統中的每個電話裝置有哪些元素，以及它們的功能。 雖然非常適合真實的電話組，但這種抽象概念也可以為其他裝置提供有意義的實 (或) 的子集。 以個別的耳機作為範例，直接連線並可從電腦控制，並以手機裝置的形式運作。 Hookswitch 變更可透過偵測語音能源 (offhook) 或 (onhook) 的無回應時間來觸發;可以透過將可聽見的信號產生到耳機來模擬響鈴;您可以透過文字轉換語音來模擬顯示器。

手機裝置不需要在硬體中實現，而是可以使用滑鼠或鍵盤驅動的圖形命令介面以及電腦的喇叭或音效系統，在軟體中模擬。 這類「軟電話」可以是使用 TAPI 的應用程式。 它也可以是服務提供者，其可透過 API 列為其他應用程式可使用的電話裝置，因此會被指派電話裝置識別碼。

根據環境和設定，電話組可以是應用程式與交換器之間的共用裝置。 某些次要布建是在 API 中進行，而在此 API 中，切換可以暫時暫停 API 的手機裝置控制。
