<span data-ttu-id="8f097-104">資源管理是將資源從系統記憶體儲存體升級到可存取裝置的儲存體，並從裝置可存取的儲存體中捨棄的程式。</span><span class="sxs-lookup"><span data-stu-id="8f097-104">Resource management is the process where resources are promoted from system-memory storage to device-accessible storage and discarded from device-accessible storage.</span></span> <span data-ttu-id="8f097-105">Direct3D 執行時間有自己的管理演算法，以最少最近使用的優先順序技術為基礎。</span><span class="sxs-lookup"><span data-stu-id="8f097-105">The Direct3D run time has its own management algorithm based on a least-recently-used priority technique.</span></span> <span data-ttu-id="8f097-106">當 Direct3D 偵測到的資源超過可在裝置可存取記憶體中並存的資源時，它會切換為最新使用的優先順序技術， [**IDirect3DDevice9：： BeginScene**](/windows/desktop/api) 和 [**IDirect3DDevice9：： EndScene**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-endscene) 呼叫之間會使用這些資源。</span><span class="sxs-lookup"><span data-stu-id="8f097-106">Direct3D switches to a most-recently-used priority technique when it detects that more resources than can coexist in device-accessible memory are used in a single frame - between [**IDirect3DDevice9::BeginScene**](/windows/desktop/api) and [**IDirect3DDevice9::EndScene**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-endscene) calls.</span></span>

<span data-ttu-id="8f097-107">在建立時使用 [**D3DPOOL \_ 預設**](./d3dpool.md) 旗標，指定將資源放在記憶體集區中，最適合指定資源所要求的一組使用方式。</span><span class="sxs-lookup"><span data-stu-id="8f097-107">Use the [**D3DPOOL\_DEFAULT**](./d3dpool.md) flag at creation time to specify that a resource are placed in the memory pool most appropriate for the set of usages requested for the given resource.</span></span> <span data-ttu-id="8f097-108">這通常是視訊記憶體，包括本機視訊記憶體和高速圖形連接埠 (AGP) 記憶體。</span><span class="sxs-lookup"><span data-stu-id="8f097-108">This is usually video memory, including both local video memory and accelerated graphics port (AGP) memory.</span></span> <span data-ttu-id="8f097-109">預設集區中的資源不會透過裝置的遺失和操作狀態之間的轉換來保存。</span><span class="sxs-lookup"><span data-stu-id="8f097-109">Resources in the default pool do not persist through transitions between the lost and operational states of the device.</span></span> <span data-ttu-id="8f097-110">這些資源必須在呼叫 reset 之前釋出，然後必須重新建立。</span><span class="sxs-lookup"><span data-stu-id="8f097-110">These resources must be released before calling reset and must then be re-created.</span></span>

<span data-ttu-id="8f097-111">在建立時使用 [**D3DPOOL \_ managed**](./d3dpool.md) 旗標來指定受管理的資源。</span><span class="sxs-lookup"><span data-stu-id="8f097-111">Use the [**D3DPOOL\_MANAGED**](./d3dpool.md) flag at creation time to specify a managed resource.</span></span> <span data-ttu-id="8f097-112">受控資源會透過裝置的遺失和操作狀態之間的轉換來保存。</span><span class="sxs-lookup"><span data-stu-id="8f097-112">Managed resources persist through transitions between the lost and operational states of the device.</span></span> <span data-ttu-id="8f097-113">這些資源都存在於影片和系統記憶體中。</span><span class="sxs-lookup"><span data-stu-id="8f097-113">These resources exist both in video and system memory.</span></span> <span data-ttu-id="8f097-114">當轉譯時需要 managed 資源，它會複製到視頻記憶體中。</span><span class="sxs-lookup"><span data-stu-id="8f097-114">A managed resource will be copied into video memory when it is needed during rendering.</span></span> <span data-ttu-id="8f097-115">您可以使用 [**IDirect3DDevice9：： Reset**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-reset)的呼叫來還原裝置，如此一來，這類資源仍可繼續正常運作，而不需要重載資料。</span><span class="sxs-lookup"><span data-stu-id="8f097-115">The device can be restored with a call to [**IDirect3DDevice9::Reset**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-reset), and such resources continue to function normally without being reloaded with data.</span></span> <span data-ttu-id="8f097-116">但是，如果必須終結並重新建立裝置，則必須重新建立所有資源。</span><span class="sxs-lookup"><span data-stu-id="8f097-116">However, if the device must be destroyed and re-created, all resources must be re-created.</span></span>

<span data-ttu-id="8f097-117">對於變更頻率較高的資源，不建議使用資源管理。</span><span class="sxs-lookup"><span data-stu-id="8f097-117">Resource management is not recommended for resources which change with high frequency.</span></span> <span data-ttu-id="8f097-118">例如，用來跨越場景圖形的頂點和索引緩衝區，每個畫面格只轉譯使用者可以看到的幾何變更每個畫面格。</span><span class="sxs-lookup"><span data-stu-id="8f097-118">For example, vertex and index buffers which are used to traverse a scene graph every frame rendering only geometry visible to the user changes every frame.</span></span> <span data-ttu-id="8f097-119">因為受管理的資源是由系統記憶體所支援，所以必須在系統記憶體中更新常數的變更，這可能會導致效能嚴重降低。</span><span class="sxs-lookup"><span data-stu-id="8f097-119">Since managed resources are backed by system memory, the constant changes must be updated in system memory, which can cause a severe degradation in performance.</span></span> <span data-ttu-id="8f097-120">在此特定案例中，應該使用 [**D3DPOOL \_ DEFAULT**](./d3dpool.md) 以及 [**D3DUSAGE \_ DYNAMIC**](d3dusage.md) 。</span><span class="sxs-lookup"><span data-stu-id="8f097-120">For this particular scenario, [**D3DPOOL\_DEFAULT**](./d3dpool.md) along with [**D3DUSAGE\_DYNAMIC**](d3dusage.md) should be used.</span></span>

<span data-ttu-id="8f097-121">不建議 (資源管理的另一個範例，且執行時間不會明確允許) 管理使用 [**D3DUSAGE \_ RENDERTARGET**](d3dusage.md)所建立的材質。</span><span class="sxs-lookup"><span data-stu-id="8f097-121">Another example for which resource management is not recommended (and explicitly not allowed by the runtime) is management of a texture created with [**D3DUSAGE\_RENDERTARGET**](d3dusage.md).</span></span> <span data-ttu-id="8f097-122">如果轉譯目標所使用的記憶體是由執行時間所管理，則在轉譯期間可能需要持續將內容複寫到系統記憶體，造成效能大幅下降。</span><span class="sxs-lookup"><span data-stu-id="8f097-122">If the memory used by render targets were managed by the runtime, it's contents would have to constantly be copied to system memory during rendering, causing large performance penalties.</span></span> <span data-ttu-id="8f097-123">基於這個理由，您必須在預設集區中建立呈現目標。</span><span class="sxs-lookup"><span data-stu-id="8f097-123">For this reason, render targets must be created in the default pool.</span></span> <span data-ttu-id="8f097-124">如果需要包含在轉譯目標中之資料的 CPU 存取，則必須將資料複製到使用 [**IDirect3DDevice9：： UpdateTexture**](/windows/desktop/api)或 [**IDirect3DDevice9：： UpdateSurface**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-updatesurface)在 [**D3DPOOL \_ SYSTEMMEM**](./d3dpool.md)中建立的資源。</span><span class="sxs-lookup"><span data-stu-id="8f097-124">If CPU access of the data contained in a render target is needed, the data must be copied to a resource created in [**D3DPOOL\_SYSTEMMEM**](./d3dpool.md) using [**IDirect3DDevice9::UpdateTexture**](/windows/desktop/api), or [**IDirect3DDevice9::UpdateSurface**](/windows/win32/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-updatesurface)</span></span>
