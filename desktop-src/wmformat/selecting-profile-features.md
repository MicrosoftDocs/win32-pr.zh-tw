# <a name="selecting-profile-features"></a>選取設定檔功能

下表列出設定檔功能，並說明何時或不想要使用它們。



| 功能                            | 使用時機                                                                                                                                                                                                                                                                                                                | 不使用時                                                                                                                                                                                                                                                                    |
|------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 依位元速率 (MBR) 相互排除 | 檔案將會串流處理至物件，其連線速度會因使用者而異 (例如，透過網際網路) 傳遞的檔案。                                                                                                                                                                              | 檔案不會透過網路進行資料流程處理。檔案會透過具有一致可靠可用頻寬的網路進行串流處理 (例如，透過公司區域網路提供的檔案) 。<br/>                                                              |
| 依語言的相互排除       | 檔案包含以不同語言複製相同資料的資料流程。  (例如，以數種語言來配音的電影會有以「互斥」資料流程的形式編碼的聲段。 )                                                                                                                          | 檔案只包含單一語言的資料流程。如果檔案只包含必須針對個別語言變更的資料流程，則為每種語言建立新檔案可能會比較簡單 (例如，針對影片串流中有大量文字的定型電影) 。<br/> |
| 依簡報的相互排除   | 您想要以一個以上的外觀比例提供影片。 例如，檔案可能包含針對電視格式化的相同影片，以及適用于原始電影全螢幕格式的影片。                                                                                                                                     | 每個影片串流只有一個版本。                                                                                                                                                                                                                                    |
| 頻寬共用                  | 檔案將會進行串流處理，並有兩個或多個資料流程（合併時）使用較少的可用頻寬，而這兩個數據流的位元速率值會加在一起。 因為資料流程的位元速率基本上是資料流程所需的最大頻寬，所以某些資料流程可能會有一些較低的資料傳輸時間。 | 檔案不會透過網路進行資料流程處理。所有串流都具有一致的位元速率。<br/>                                                                                                                                                                                     |
| 資料流程優先順序              | 檔案會經過資料流程處理，有些資料流程比其他串流更重要。                                                                                                                                                                                                                                                 | 檔案不會透過網路進行資料流程處理。所有串流都有相等的重要性。<br/>                                                                                                                                                                                       |
| 資料單位延伸模組               | 資料流程中的範例必須與範例保持相關的重要資訊。                                                                                                                                                                                                                      | 此檔案的目的是要透過受限制的頻寬進行傳遞。 在此情況下，資料單位延伸可能會使用太多額外負荷。                                                                                                                                                    |



 

在建立設定檔時，請務必考慮您想要搭配檔案使用的編解碼器功能。 如需詳細資訊，請參閱 [編解碼器功能](codec-features.md)。
