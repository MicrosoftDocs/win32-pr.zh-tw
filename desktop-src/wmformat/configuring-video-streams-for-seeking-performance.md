# <a name="configuring-video-streams-for-seeking-performance"></a><span data-ttu-id="0a7b5-108">設定影片串流以搜尋效能</span><span class="sxs-lookup"><span data-stu-id="0a7b5-108">Configuring Video Streams for Seeking Performance</span></span>

<span data-ttu-id="0a7b5-109">有些播放應用程式會對個別資料流程執行許多搜尋。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-109">Some playback applications perform a lot of seeking on individual streams.</span></span> <span data-ttu-id="0a7b5-110">搜尋是一個區域，根據資料流程的設定而定，效能可能會有很大的差異。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-110">Seeking is an area where performance can vary greatly depending upon the settings of the stream.</span></span> <span data-ttu-id="0a7b5-111">如果您知道您的內容需要針對快速搜尋進行優化，您可以量身打造您的串流設定以改善效能。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-111">If you know your content needs to be optimized for quick seeking, you can tailor your stream configuration to improve performance.</span></span>

<span data-ttu-id="0a7b5-112">影響影片中搜尋作業速度的最大因素就是主要畫面格的間距。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-112">The biggest factor affecting the speed of seeking operations in video is the spacing of the key frames.</span></span> <span data-ttu-id="0a7b5-113">因為主要畫面格之間的每個畫面格都必須根據其前面的畫面格來重建，所以廣泛的主要畫面格結果會比搜尋時間更長。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-113">Because every frame between key frames needs to be reconstructed based on the frames that come before it, widely spaced key frames result longer seek times.</span></span> <span data-ttu-id="0a7b5-114">例如，如果影片資料流程的每秒30個畫面格的最大索引鍵-畫面格間距為10秒，主要畫面格之間可能會有300的框架。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-114">For example, if a video stream with 30 frames per second has a maximum key-frame spacing of 10 seconds, there are potentially 300 frames between key frames.</span></span> <span data-ttu-id="0a7b5-115">如果您搜尋到最後一個 [*delta 框架*](wmformat-glossary.md)，則必須重建299的框架，才能解壓縮框架。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-115">If you seek to the last [*delta frame*](wmformat-glossary.md), 299 frames have to be reconstructed for the frame to be decompressed.</span></span> <span data-ttu-id="0a7b5-116">如果每個畫面格重建花費了 .01 秒，則搜尋需要大約3秒的時間。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-116">If each frame reconstruction took .01 second, the seek would take almost 3 seconds.</span></span> <span data-ttu-id="0a7b5-117">如果您想要提高搜尋效率，請降低主要畫面格間距的協助。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-117">If you want to increase the efficiency of seeking, lowering the key-frame spacing can help.</span></span> <span data-ttu-id="0a7b5-118">但是，如果您設定的主要畫面格太緊密，可能會失去品質。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-118">However, if you set the key frames too close together, you can lose quality.</span></span>

<span data-ttu-id="0a7b5-119">您可以藉由呼叫 [**IWMVideoMediaProps：： SetMaxKeyFrameSpacing**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmvideomediaprops-setmaxkeyframespacing)來設定最大的主要畫面格間距。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-119">You can set the maximum key-frame spacing by calling [**IWMVideoMediaProps::SetMaxKeyFrameSpacing**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmvideomediaprops-setmaxkeyframespacing).</span></span> <span data-ttu-id="0a7b5-120">下表列出建議值（根據資料流程的位元速率）。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-120">The recommended values, based on the bit rate of the stream, are listed in the following table.</span></span> <span data-ttu-id="0a7b5-121">這些值可提供良好的平衡來尋找效能和品質。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-121">These values provide a good balance of seeking performance and quality.</span></span> <span data-ttu-id="0a7b5-122">SDK 不會在主要畫面格之間的時間強制執行任何限制。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-122">The SDK does not enforce any limit on the time between key frames.</span></span> <span data-ttu-id="0a7b5-123">一般情況下，超過30秒的時間可能會在內容透過網路進行串流處理，以及在本機播放時，對搜尋時間造成負面影響。</span><span class="sxs-lookup"><span data-stu-id="0a7b5-123">In general, times longer than 30 seconds can adversely affect seek times both when the content is streamed over a network, and when it is played back locally.</span></span>
