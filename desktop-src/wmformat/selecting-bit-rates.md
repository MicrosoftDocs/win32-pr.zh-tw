# <a name="selecting-bit-rates"></a>選取位元速率

針對將透過網路進行串流處理的檔案，您必須仔細考慮應該使用的位速度。 在大多數情況下，您可以將檔案中所有資料流程的位元速率加在一起，以取得串流處理檔案所需的可用頻寬的一般概念。 不過，每個串流也都需要特定的負荷量。 下表摘要說明此額外負荷。



| 位速率範圍 (Kbps)  | 額外負荷 (Kbps 所需的頻寬)  |
|-----------------------|---------------------------------------------------|
| 10–16               | 3                                                 |
| 17–30               | 4                                                 |
| 31-45               | 5                                                 |
| 46–70               | 6                                                 |
| 71–225              | 7                                                 |
| >225               | 9                                                 |



 

資料流程所需的一般負擔不會將資料單位延伸列入考慮。 每個資料單位延伸都會新增至附加的範例大小。 視資料單位延伸模組的類型而定，這可能會大幅增加資料流程的位元速率。

您也必須考慮到可透過網路連線使用的理論最大頻寬並不是實際的目標位元速率。 由於網路流量及許多其他因素，任何指定連線的平均可用頻寬都是連接的頻寬容量很短。
