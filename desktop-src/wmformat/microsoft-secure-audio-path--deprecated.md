# <a name="microsoft-secure-audio-path-deprecated"></a>Microsoft 安全音訊路徑 (已淘汰) 

本頁面記載了在未來版本的 Windows 中，將會受到不同技術解決方案支援的功能。

內容建立者和散發者可以在 DRM 授權中指定，只允許在具有 Microsoft 安全音訊路徑的系統上播放音訊檔案 (SAP) 元件。 安全音訊路徑可讓不受信任的應用程式或音訊驅動程式存取未加密的音訊位，以提供更高程度的音訊內容保護。 Microsoft Windows® Me 和 Windows XP 支援「安全音訊路徑」。 它可保護作業系統核心中的數位音樂。 此外，數位 Millennium 著作權法案也可讓您規避軟體 a 犯罪中的 antipiracy 措施。

當 DRM 授權需要 SAP 時，Microsoft DRM 元件會自動啟用並自動執行安全音訊路徑。 針對在 Windows® XP Service Pack 1 上執行的應用程式，您可以使用安全音訊路徑 SDK，在 Microsoft DRM 解決方案的內容之外，對任何音訊內容啟用 SAP 加密。 如需有關「安全音訊路徑 SDK」的詳細資訊，請參閱 [Microsoft](/documentation/)網站。
