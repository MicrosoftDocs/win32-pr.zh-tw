# <a name="icons-design-basics"></a> (設計基本概念的圖示) 

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

圖示是物件的圖形標記法，但不只是因為美觀的原因，而是程式視覺化身分識別的一部分，也有 utilitarian 的原因，也就是用來傳達使用者幾乎立即觀察到之意義的縮寫。 WindowsVista 引進了一種新的圖示樣式，為 Windows 帶來更高層級的詳細資料和複雜性。

**注意：** 與 [標準圖示](vis-std-icons.md) 相關的指導方針會在個別的文章中顯示。

## <a name="design-concepts"></a>設計概念

Aero 是 Windows Vista 的使用者體驗的名稱，代表美學設計中所包含的值，以及使用者介面 (UI) 背後的願景。 Aero 代表：真實、energetic、反射和開放式。 Aero 的目標是要建立既專業又美觀的設計。 Aero 美觀會建立高品質且簡潔的體驗，以協助使用者生產力，甚至推動情緒的回應。

WindowsVista 圖示不同于 Windows XP 樣式的圖示，方法如下：

-   樣式比說明更實際，但不是真正的照片。 圖示是符號影像，其外觀應優於照片影像！
-   圖示的大小上限為256x256 圖元，使其適用于每英寸) 顯示的高 DPI (點。 這些高解析度圖示可以在具有大型圖示的清單視圖中提供高視覺品質。
-   在實際的情況下，固定的檔圖示會以內容縮圖取代，讓您更容易識別及尋找檔。
-   工具列圖示的詳細資料和觀點都較小，因此可針對較小的大小和視覺效果 distinctiveness 進行優化。

設計完善的圖示：

-   改善程式的視覺效果通訊。
-   強烈影響使用者對您程式的視覺設計的整體印象，並感謝其符合度和完成度。
-   藉由讓程式、物件和動作更容易識別、學習和尋找，進而改善可用性。

## <a name="guidelines"></a>指導方針

### <a name="perspective"></a>檢視方塊

-   **Windows Vista 中的圖示是三維，並以實體物件的觀點來顯示，或直接顯示為二維物件。** 針對檔案以及實際為一般平面的物件（例如檔或檔片段）使用一般圖示。

    ![3d 電腦和平面2d 紙的影像 ](images/vis-icons-image7.png)

    一般3D 和一般圖示。

-   **三維物件是以實體物件的觀點表示，從低鳥觀點來看，有兩個沒影點。**

    ![具有線條顯示觀點的筆記本影像](images/vis-icons-image8.png)

    此範例會顯示3D 圖示的一般透視圖和沒影點。

-   **在較小的大小中，相同的圖示可能會從觀點變更為「直接」。** 在16x16 圖元的大小和較小的情況下，會將圖示直接呈現 (front) 。 針對較大的圖示，請使用透視圖。

    -   **例外狀況：** 工具列圖示一律是正面的，甚至是較大的大小。

    ![大型3d 電腦和小型2d 電腦的影像 ](images/vis-icons-image9.png)

    此範例顯示如何以不同的方式處理相同的圖示，視大小而定。

### <a name="light-source"></a>光源

-   **在 [透視圖] 方格中，物件的光源是在上方、稍微的前方，以及稍微指向物件左邊。**
-   **光源會將陰影轉換成物件基底的後端和右邊。**
-   **所有光線都是平行的，並將物件沿著相同的角度 (，就像 sun) 一樣。** 其目標是讓所有圖示和焦點效果都有一致的光源外觀。 平行光線產生的陰影，全都具有相同的長度和密度，可在多個圖示之間提供進一步的 unity。

### <a name="shadows"></a>陰影

**一般**

-   **您可以使用陰影，從背景以視覺化方式增益出物件，並讓3D 物件顯示為接地，而不是滑雪浮動的空間。**
-   **針對陰影使用30-50% 的不透明度範圍。** 有時應該使用不同的陰影層級，視圖標的形狀或色彩而定。
-   **如有必要，請將陰影羽化或縮短，以防止圖示方塊大小裁剪。**
-   **請勿在24x24 或較小大小的圖示中使用陰影。**

    ![具有陰影的3d 圖示影像 ](images/vis-icons-image10.png)

    一般圖示遮蔽。

**平面圖標**

-   **一般圖示通常用於檔案圖示和一般的真實世界物件，例如檔或檔** 片段。
-   **平面圖標光源來自左上角，130度。**
-   **較小的圖示 (例如，16x16 和 32x32) 已簡化為可讀性。** 但是，如果它們在圖示內包含反映 (通常會簡化) ，它們可能會有緊密的陰影。 從30-50% 的陰影範圍不透明度。
-   **圖層效果可用於一般圖示，但應該與其他一般圖示比較。** 物件的陰影會根據最適合的外觀，以及 Windows Vista 中的其他圖示，在大小設定和最一致的情況下，會有不同的變化。 在某些情況下，甚至可能需要修改陰影。 當物件配置給其他人時，這會特別適用。
-   **您可以使用較微妙的色彩範圍來達成所需的結果。** 陰影說明物件位於空間中。 色彩會影響陰影的觀察權數，如果影像太繁重，可能會扭曲影像。

### <a name="color-and-saturation"></a>色彩和飽和度

-   **相較于 Windows XP，色彩通常較不飽和。**
-   **使用漸層來建立更逼真的影像。**
-   **雖然標準圖示沒有特定的色彩選擇區，但請記住，它們必須在許多內容和主題中妥善地搭配使用。** 偏好使用一組標準的色彩;請勿重新著色標準圖示，例如警告圖示，因為這會干擾使用者解讀意義的能力。 如需詳細指導方針，請參閱 [色彩](vis-color.md)。
-   **圖示檔也需要8位和4位的調色板版本，** 以支援遠端桌面的預設設定。 您可以透過批次處理來建立這些檔案，但您應該先檢查這些檔案，因為有些檔案需要修飾才能讓可讀性更好。

    ![[色彩選擇器] 對話方塊的螢幕擷取畫面 ](images/vis-icons-image15.png)

    沒有嚴格的色板限制。 只會避免 (右上方) 的全飽和。

-   位層級：包含32位 (Alpha 的 .ICO 設計) + 8 位 + 4 位 (遞色會自動將最重要的) 圖元。 只應包含256x256 圖元影像的32位複本，而且只應壓縮256x256 圖元影像，以保持檔案的大小。 有幾個圖示工具提供 Windows Vista 的壓縮。
-   位層級：工具列24位 + Alpha (1 位遮罩) 、8位和4位。
-   工具列或 AVI 檔案：使用洋紅色 (R255 G0 B255) 作為背景透明度色彩。

### <a name="size-requirements"></a>大小需求

**一般**

-   **特別注意高可見度圖示，** 例如主要應用程式圖示、可以出現在 Windows 檔案總管中的檔案圖示，以及顯示在 [開始] 功能表或桌面上的圖示。
    -   **應用程式圖示和主控台專案：** 完整集合包含16x16、32x32、48x48 和 256x256 (程式碼會在32和 256) 之間進行縮放。 需要 .ico 檔案格式。 若為傳統模式，完整的集合為16x16、24x24、32x32、48x48 和64x64。
    -   **清單專案圖示選項：** 使用即時縮圖或檔案類型的檔案圖示 (例如 .doc) ;完整集合。
    -   **工具列圖示：** 16x16、24x24、32x32。 請注意，即使在32x32 大小的情況下，工具列圖示一律是平坦的，而非3D。
    -   **對話方塊和嚮導圖示：** 32x32 和48x48。
    -   覆迭 **：** 核心 shell 程式碼 (例如，) 10x10 (適用于 16x16) 、16x16 (用於 32x32) 、24x24 (for 48x48) 、128x128 (for 256x256) 。 請注意，其中有些會稍微小一點，但會接近此大小，視圖形和光學餘額而定。
    -   **快速啟動區域：** 圖示會從 Alt + Tab 動態重迭中的48x48 向下延展，但若要使用更清晰的版本，請將40x40 新增至 .ico 檔案。
    -   **氣球圖示：** 32x32 和40x40。
    -   **其他大小：** 這些都可作為資源，以將其他檔案 (例如，批註、工具列帶、重迭、高 DPI 和特殊案例) ：128x128、96x96、64x64、40x40、24x24、22x22、14x14、10x10 和8x8。 您可以根據該區域中的程式碼，使用 .ico、.png、.bmp 或其他檔案格式。

**高 DPI**

-   WindowsVista 的目標是 96 DPI 和 120 DPI。

### <a name="level-of-detail"></a>詳細資料層級

-   其中許多圖示的16x16 大小仍廣泛使用，因此很重要。
-   此大小圖示中的詳細資料必須清楚地顯示圖示的按鍵點。
-   當圖示變得較小時，應該會犧牲透明度和一些較大大小的特殊細節，以簡化和取得重點。
-   屬性和色彩應該會放大，並用來強調主要表單。

### <a name="icon-development"></a>圖示開發

**設計和產生圖示**

-   **雇用經驗豐富的圖形設計工具。** 如需絕佳的圖形、影像和圖示，請與專家合作。 建議您在使用向量藝術或3D 程式的圖例中體驗。
-   **規劃一系列** 反復專案，從初步的概念草圖到實際的實際執行審核，以及工作產品中的圖示的完成。
-   **預先考慮圖示的建立可能很昂貴。** 收集所有現有的詳細資料和需求，例如：所需的完整圖示集;main 函式和每個函數的意義;您想要清楚設定集中的系列或群集;品牌需求;確切的檔案名;程式碼中使用的影像格式;和大小需求。 事先確保您可以使用設計工具來充分利用您的時間。
-   **請記住，設計工具可能不熟悉您的產品，因此請視需要提供功能資訊、螢幕擷取畫面和規格區段。**
-   **適當地規劃地緣政治和法律評論。**
-   **對應時間範圍，並定期進行通訊。**

**生產**

> [!TIP]
> 遵循下列步驟來建立包含多個影像大小和色彩深度的單一 .ico 檔案。

**步驟1：概念化**

-   盡可能使用已建立的概念，以確保圖示的意義和與其他用途的相關性一致。
-   請考慮圖示在 UI 內容中的顯示方式，以及它如何作為一組圖示的一部分來運作。
-   如果修改現有的圖示，請考慮是否可以降低複雜性。
-   請考慮您圖形的文化影響。 避免在圖示中使用字母、單字、手或臉部。 如有需要，請盡可能一般地描述人員或使用者的標記法。
-   如果將多個物件結合為圖示中的單一影像，請考慮影像如何調整為較小的大小。 在圖示中使用三個以上的物件 (兩個是慣用的) 。 若為16x16 大小，請考慮移除物件或簡化影像以改善辨識。
-   請勿在圖示中使用 Windows 旗標。

**步驟2：說明**

-   若要說明 Windows Aero 樣式圖示，請使用向量工具，例如 Macromedia 手繪或 Adobe Illustrator。 如本文稍早所述，使用調色板和樣式特性。
-   使用手繪或 Illustrator 來說明影像。 將向量影像複製並貼到 Adobe Photoshop 中。
-   在 Photoshop 中建立並使用範本圖層，以確保工作是在受管制大小的正方形區域內完成。
-   建立大小小於整體圖示大小需求的影像，以允許陰影 (需要一個) 之大小的空間。
-   將影像放在正方形底部，讓目錄中的所有圖示一致定位。 避免遮蔽遮蔽。
-   如果您要將另一個物件加入影像或數列中，請將主物件保留在固定的位置，並將平面較小的影像放在固定的位置，例如左下角或右上角（視案例而定）。

**步驟3：建立24位映射**

-   當您在 Photoshop 中貼上大小之後，請檢查影像的可讀性，尤其是在16x16 和較小的大小。 可能需要使用色彩百分比的圖元刺探。 可能也需要減少透明度。 通常會以較小的放大層面，也可以消除層面，以便專注于關鍵點。
-   8點陣圖標將會以低於32位的任何色彩模式顯示，且不會有8位 Alpha 色板，因此它們可能需要將邊緣或更多的部分清除，因為沒有消除鋸齒 (邊緣可能會有鋸齒狀，而影像可能難以讀取) 。
-   在 Photoshop 中，複製24位映射圖層，並將圖層重新命名為4位影像。 Windows 16 色調色板的索引4位影像。
-   只使用16色調色板中的色彩來清除影像。 從較深或較淺的物件色彩版本所建立的輪廓通常偏好為灰色或黑色。
-   如果使用點陣圖，請確定影像本身不使用背景色彩，因為該色彩會是透明色彩。 洋紅色 (R255 G0 B255) 通常用來做為背景透明度色彩。

**步驟4：建立8位和4位的映射**

-   現在24位映射已準備好進入32位的圖示，必須建立8位版本。
-   這是測試內容相關螢幕擷取畫面的絕佳時機。 您可以藉由在內容中查看其他圖示或一系列圖示來探索這項驚喜。 此步驟可以節省時間和金錢。 在檔案進入生產階段之前，最好先捕捉問題，然後再進行。
-   以需要的大小將陰影新增至影像。
-   將陰影和24位影像合併在一起。
-   為每個大小建立新的 Photoshop 檔案。 複製並貼上適當的影像。 將每個檔案儲存為 .psd 檔。
-   請勿將影像圖層與背景圖層合併。 使用時，在檔案名中包含大小和色彩深度很有説明，但檔案最後可能需要重新命名。

**步驟5：建立 .ico 檔案**

-   選擇最符合其需求的應用程式和演出者的技能。 請記住，要在出貨產品中使用的圖示必須在已購買或授權的工具中建立。 這表示無法使用試用版。
-   下列兩項產品都是由已為 Windows Vista 產生圖示的設計人員所使用，而且每個都提供匯出至 Adobe Photoshop CS 的能力。
    -   Gamani Gif Movie 齒輪：產生 .ico 檔案
    -   Axialis 圖示研討會：產生 .ico 檔案
-   Visual Studio 不支援 Windows 的 Vista 圖示 (不支援 Alpha 色板或超過256色) ，因此不建議使用。
-   圖示 ( .ico 格式) 檔案必須包含4和8位版本，以及24位 + Alpha。
-   無論您選擇使用哪一個圖示建立程式，請將檔案儲存為「Windows 圖示 ( .ico) 」。
-   某些 iconographic 資產實際上可能是點陣圖條紋，這也需要 Alpha 通道 (例如，針對工具列) ，或 .png 以透明度儲存的檔案。 並非全部都必須是 .ico 格式;檢查程式碼中支援的格式。

**步驟6：評估**

-   查看各種大小。
-   一起查看系列非常相似、光學餘額和差異。
-   查看內容以評估相對權數和可見度 (確定其中一個不) 。
-   請考慮目前可能不會用到，但在不久的未來可能會用到的案例。 此圖示是否曾經加上批註或有重迭？
-   查看程式碼。

-   也請參閱程式碼中的圖示。 動作和其他因素可能會影響圖示的成功;在某些情況下，可能需要進行進一步的反覆運算。
