# <a name="software-branding"></a>軟體商標

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

商標是產品的情緒定位，如同客戶的認知。 成功的商標需要 skillful 製作產品影像，而不只是透過產品標誌和色彩配置來達成。

商標是產品的情緒定位，如同客戶的認知。 產品品牌是透過各種因素組合來達成，包括產品名稱和標誌、色彩、文字、圖形和音效的使用、各種其他設計項目的樣式、行銷，以及最重要的是產品體驗本身的屬性。

成功的商標需要 skillful 製作產品影像，而不只是在每個表面上 plastering 產品標誌，並在每個商機中使用產品的色彩配置來達成。 相反地，增強使用者體驗的有意義且高品質的品牌，將會更成功。

**注意：** 與 [圖示](vis-icons.md) [、字型、](vis-fonts.md)[色彩](vis-color.md)、[動畫](vis-animations.md)、[音效](https://msdn.microsoft.com/library/windows/desktop/aa511278.aspx)和 [視窗框架](win-window-frames.md)相關的指導方針會顯示在不同的文章中。

## <a name="design-concepts"></a>設計概念

在競爭激烈的 marketplace 中，公司會為其產品提供協助，使其與競爭對手有所區別。 建議您在一般情況下，建議產品品牌錯誤或應避免，但請確定軟體品牌的執行頻率太差。 軟體品牌的目標是要將品牌與產品的樣式和品質及其體驗產生關聯。 通常，開發人員會藉由強調程式本身來嘗試達成此目的。 結果是要干擾使用者，而不是取悅。

### <a name="attributes-of-good-software-branding"></a>良好軟體品牌的屬性

完成時，軟體商標會有下列屬性：

-   建立清楚、獨特的樣式和特性。
-   建立情緒連接。
-   具有高品質。
-   策略性地放置和一致地執行。
-   與整體品牌策略相符。
-   長期持續 .。。因為它是第一次的時間。

相反地，軟體商標的不良嘗試會有下列屬性：

-   沒有明顯的主題或點。
-   在使用者的臉部中。
-   很討厭。
-   隨處都是。
-   擁有自訂外觀和風格，沒有任何使用者的權益。
-   會快速一定乏味。

### <a name="start-with-the-product-itself"></a>開始使用產品本身

成功的軟體商標從產品本身的設計開始。 設計完善的程式具有精心設計的功能，目的在於適當的目標物件。 獨特的功能和特別注意的詳細資料，可提供強大的商標聲明。 如需詳細資訊，請參閱 [如何設計絕佳的使用者體驗](how-to-design-desktop-ux.md)。

### <a name="carefully-choose-the-products-name"></a>請小心選擇產品的名稱

絕佳的產品名稱驅動了強大品牌。 其中有一個絕佳的軟體產品名稱，可讓您清楚地傳達產品的優點，並在擁擠的市場上提供差異。 雇用商標專家，協助您選擇正確的產品名稱。 長期來說，選擇的名稱對於您的品牌工作而言更加重要，因為標誌、色彩配置和控制主題等詳細資料。

### <a name="what-to-brand"></a>什麼是品牌

軟體品牌要素可分類如下：

**主要**

-   產品名稱
-   產品標誌
-   產品色彩配置
-   產品特定音效

![windows 標誌的螢幕擷取畫面 ](images/exper-branding-image1.png)![windows 主題的螢幕擷取畫面 ](images/exper-branding-image2.png)

Windows Vista 的一些主要品牌要素。

主要商標元素通常會有很多的注意，因此應該與擋板一起使用。 **將主要商標元素的使用限制為一些策略性體驗。** 不建議對大部分的程式使用特定產品的音效。

**次要**

-   元素圖形
-   圖示和圖形樣式
-   次要圖形元素
-   輔色
-   動畫
-   轉換
-   陰影
-   背景和透明度

![用於商標的戳記和行事曆的圖表 ](images/exper-branding-image3.png)

![純色、主要色彩的設計圖 ](images/exper-branding-image4.png)![藍色和綠色功能區的圖設計 ](images/exper-branding-image5.png)

Windows Vista 的一些次要商標元素。

次要商標元素通常較微妙，因此可以更頻繁地使用。 雖然其中有些次要商標元素可能不會個別影響，但在一起時，它們可以提供您的產品字元和樣式。 轉換可能會有比固定圖形更多的影響，而使用者會學習在經過一段時間後略過。 偏好主要層級的次要品牌層級。

**第三**

最後，還有另一個要注意的商標元素類別。

-   自訂視窗框架
-   自訂控制項

雖然它適用于特定類型的程式 (例如遊戲) 若要根據自訂控制項和視窗建立完全不同的沉浸式體驗，大部分的程式都應該使用標準種類。 讓您的程式看起來和 act 都不會成為強大的品牌身分識別。 相反地，您的目標應該是建立一個具有字元的程式，也就是一項產品，並在進行調整時。

### <a name="where-to-brand"></a>品牌

並非所有專案都需要品牌化。 有些策略性放置的商標元素可提供比工匠不協調商標元素更強大的印象。

將您的品牌精力集中在程式的特殊體驗上。 這些是具有最情緒影響的位置，例如：

-   [第一次體驗](exper-first-exper.md)，特別是在第一次使用程式時。
-   主視窗或首頁。
-   重要工作的開始和完成。
-   工作或程式區域之間的重要轉換。
-   長時間執行之工作的等候時間。
-   登入並登出。

### <a name="where-not-to-brand"></a>非品牌

雖然您可能會在程式中使用任何元素作為商標的機會，但請不要使用 Windows desktop (包括[工作區域](glossary.md)、[快速啟動](glossary.md)列或[通知區域](winenv-notification.md)) 的商標。

**桌面是 Windows 的使用者進入點。讓使用者保持控制。** 適當地使用這些進入點，絕對不會將其視為提升程式或其品牌的認知方式。 如需詳細資訊，請參閱 [Desktop](winenv-desktop.md)。

### <a name="use-branding-professionals"></a>使用品牌專業人員

商標是由經驗豐富的專業人員所做的專業技能。 更好的方式是將您的使用者公開到最短的品牌，而不是使用令人討厭且不正確大量品牌。 與您的品牌和行銷團隊合作，建立良好的端對端品牌體驗。

**如果您只進行五件事 .。。**

1.  從產品設計開始著手。 最強大的商標陳述是為了滿足客戶的需求。
2.  選擇易記、獨特且簡潔地傳達產品權益的良好產品名稱。
3.  您可在體驗方面考慮商標，並建立情緒的連線，而不是產品標誌和色彩配置。
4.  偏好次要品牌要素。 將主要商標元素的使用限制為一些策略性體驗。
5.  從商標專家取得協助。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **選擇易記、獨特且簡潔地傳達產品權益的良好產品名稱。** 這將是您品牌的基礎。
-   將 **您的品牌精力集中在程式的特殊體驗上，** 例如：
    -   [第一次體驗](exper-first-exper.md)，特別是在安裝期間，以及第一次使用程式的時候。
    -   主視窗或首頁。
    -   重要工作的開始和完成。
    -   工作或程式區域之間的重要轉換。
    -   登入並登出。
-   **偏好次要品牌要素。** 將主要商標元素的使用限制為一些策略性體驗。 例如，請考慮使用次要圖形、轉換和色彩，而不是標誌。 此外，請避免使用者花很多時間，因為它們可能會被視為雜亂的地方。

    **可以接受：**

    ![具有 windows 標誌的 [控制台] 的螢幕擷取畫面 ](images/exper-branding-image6.png)

    **較佳：**

    ![沒有標誌的控制台的螢幕擷取畫面](images/exper-branding-image7.png)

    在更好的範例中，會使用次要圖形元素，而不是 Windows 控制台專案的產品標誌。

-   **請勿使用會造成困擾或損害可用性或效能的商標。**
-   **請勿使用 Windows 桌面進行商標。** 如需詳細資訊，請參閱 [Desktop](winenv-desktop.md)。

### <a name="names-and-logos"></a>名稱和標誌

-   **限制在使用者介面中使用產品和公司標誌。** 請勿在每個 UI 介面上 plaster 公司或產品標誌。

    -   將產品和公司標誌限制為最多兩個不同的表面，例如主視窗或首頁，以及 [關於] 方塊。
    -   將產品和公司標誌限制在任何單一表面上最多兩次。
    -   將產品和公司名稱限制在任何表面上最多三次。

    **不正確：**

    ![圖形過度使用公司名稱的螢幕擷取畫面 ](images/exper-branding-image8.png)

    在此範例中，公司名稱過度過度。

    **不正確：**

    ![具有數個標誌的 windows 螢幕擷取畫面 ](images/exper-branding-image9.png)

    在此範例中，雖然可以接受個別標誌的使用，但整體效果相當龐大。

-   **使用小型產品和公司標誌。** 將標誌放在使用者的工作流程外，然後選擇適合其位置的大小。
-   **使用圖形標誌。** 圖形標誌比文字標誌更穩定，因為它們不會受到字型、文字大小、語言套件或主題變更的影響。
-   **請勿使用動畫標誌。**

### <a name="controls"></a>控制項

-   **請勿將自訂控制項用於商標。** 相反地，您可以在必要時使用自訂控制項，以建立特殊的沉浸式體驗或需要特殊功能。

    **不正確：**

    ![具有商標之登入按鈕的螢幕擷取畫面 ](images/exper-branding-image10.png)

    此範例顯示不正確地用於商標的自訂控制項。

### <a name="splash-screens"></a>啟動顯示畫面

-   **請勿將啟動顯示畫面用於商標。** 請避免使用啟動顯示畫面，因為它們可能會導致使用者將您的程式與效能不佳產生關聯。 只使用它們來提供意見反應，並減少載入時間異常的程式對時間的認知。
-   **請勿使用動畫啟動顯示畫面。** 使用者通常會假設動畫啟動顯示畫面是很長載入時間的原因。 太過頻繁，該假設是正確的。

### <a name="sound"></a>音效

-   **一般來說，不建議只針對商標使用音效。** 如果您使用音效作為商標：
    -   **只在程式啟動時播放音效，** 但只有在程式是由使用者啟動時。
    -   將 **音效同步處理至視覺效果事件，** 例如 UI 轉換，如程式視窗的顯示。

如需詳細資訊，請參閱 [音效](vis-sound.md)。
