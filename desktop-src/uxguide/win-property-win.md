# <a name="property-windows"></a>屬性視窗

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

[屬性視窗] 是下列使用者介面類別型的集體名稱 (Ui) ：

-   屬性工作表：用於 **在對話方塊中，用來查看和變更物件或物件集合的屬性**。
-   屬性偵測器： **在窗格中用來查看和變更物件或物件集合的屬性**。
-   [選項] 對話方塊：用來 **查看和變更應用程式的選項**。

物件的屬性是下列其中一項：

-   使用者可變更的設定 (例如檔案的名稱和唯讀屬性) 。
-   物件的屬性，使用者無法直接變更 (例如檔案的大小和建立日期) 。

不同于對話方塊 ([選項] 對話方塊以外) 和嚮導，屬性視窗通常支援數個工作，而不是單一工作。

屬性視窗通常會組織成可透過索引標籤存取的頁面。 屬性視窗通常會與索引標籤 (相關聯，反之亦然) ，但索引標籤 **不是屬性視窗的必要選項**。

![[檔案屬性] 屬性工作表的螢幕擷取畫面 ](images/win-property-win-image1.png)

一般的屬性工作表。

**注意：** 與 [版面](vis-layout.md) 配置和索引標籤 [相關的指導方針會顯示](ctrl-tabs.md) 在不同的文章中。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定使用時機，請考量下列問題：

-   **設定屬性是否需要使用者執行固定、非簡單的步驟順序？** 若是如此，請改用 [wizard](win-wizards.md) 或工作 [流程](glossary.md) 。
-   **內容僅限應用程式的選項嗎？** 如果是，請使用 [選項] 對話方塊。
-   **內容只是應用程式的屬性嗎？** 如果是，請使用 [ [關於](glossary.md)] 方塊。
-   **內容大多是物件的屬性 (其設定或屬性) ？** 如果沒有，請使用標準 [對話方塊對話方塊](win-dialog-box.md) 或索引標籤 [式對話方塊](glossary.md)。
-   使用者是否 **可能經常或超過一段長時間來查看或變更屬性？** 如果是，請使用屬性偵測器;否則，請使用屬性工作表。
-   使用者是否 **可能一次看到或變更數個不同物件的屬性？** 如果是，請使用屬性偵測器;否則，請使用屬性工作表。

**屬性工作表和屬性偵測器不是獨佔的。** 您可以在屬性偵測器中顯示最常存取的屬性，以及在屬性工作表中顯示完整的設定。

## <a name="design-concepts"></a>設計概念

**屬性視窗通常會成為奇數分類的低層級、以技術為基礎之設定的傾印。** 這些屬性通常會組織成索引標籤，但不是針對任何特定工作或使用者所設計。 如此一來，當使用者在屬性視窗中遇到工作時，通常不會知道該怎麼做。

若要確保您的屬性視窗有用且可用，請遵循下列步驟：

-   請確定屬性是必要的。
-   以使用者目標為依據，而不是以技術呈現屬性。
-   將屬性顯示在正確層級。
-   特定工作的設計頁面。
-   特定使用者的設計頁面， (非系統管理員) ，尤其是限制使用者。
-   有效率地組織屬性頁。

**如果您只做一件事 .。。**

以使用者目標為依據，而不是以技術呈現屬性。 假設您正在說明屬性，以及它對 friend 而言很有用的原因。 您要如何加以解釋？ 您使用何種語言？ 這就是您在屬性頁中使用的語言。

## <a name="usage-patterns"></a>使用模式

屬性視窗有數種使用模式。

-   屬性工作表。 單一物件的屬性會顯示在非強制回應對話方塊中。
-   多物件屬性工作表。 多個物件的屬性會顯示在非強制回應對話方塊中。
-   有效的設定屬性工作表。 單一物件的有效屬性會顯示在非強制回應對話方塊中。
-   選項對話方塊。 應用程式的屬性會顯示在強制回應對話方塊中。
-   屬性偵測器。 目前選取範圍的屬性 (單一物件或物件群組) 顯示在非強制回應視窗窗格或未移除視窗中。

屬性偵測器以外的所有屬性視窗模式都使用延遲認可，這表示只有當使用者按一下 [確定] 或 [套用] 時，變更才會生效。 屬性偵測器會使用立即認可 (當使用者變更) 時，就會變更屬性，因此不需要 [確定]、[取消] 和 [套用] 按鈕。

## <a name="guidelines"></a>指導方針

### <a name="property-sheets"></a>屬性工作表

-   **當使用者時顯示內容表：**
    -   選取物件的 [屬性] 命令。
    -   設定物件的輸入焦點，然後按 Alt + Enter。

**多物件屬性工作表**

-   **顯示所有選取物件的通用屬性。** 如果屬性值不同，則會使用混合狀態顯示與這些值相關聯的控制項。  (參閱使用混合狀態值的個別控制指導方針。 ) 
-   如果選取的物件是多個離散物件的集合 (例如檔案資料夾) ，則會 **顯示單一群組物件的屬性，而不是個別物件的多物件屬性工作表。**

### <a name="options-dialog-boxes"></a>選項對話方塊

-   **請勿將選項與自訂區隔開。** 也就是說，沒有選項命令和自訂命令。 使用者通常會因為這種分隔而混淆。 相反地，透過選項來存取自訂。

### <a name="property-pages"></a>「屬性頁」

-   **請針對頁面順序遵循下列指導方針：**
    -   將 [一般] 頁面或其對等專案設為第一頁。
    -   使 [Advanced] 頁面或其對等的最後一頁。
    -   針對其餘的頁面：
        -   將它們組織成相關頁面的群組。
        -   依使用量的可能性來排序群組。
        -   在每個群組內，依其關聯性或其使用可能性來排列頁面的順序。
        -   您不應該有這麼多頁面需要依字母順序顯示。
-   **藉由將每個頁面上的所有屬性關聯至單一、特定、以工作為基礎的目的，使頁面一致。**
-   **如果空間允許，請在頁面頂端說明 [屬性] 視窗的用途（如果您的目標使用者並不明顯的話）。** 如果頁面只是用來執行單一工作，請 **將文字視為有關如何執行該工作的清楚指示**。 使用完整句子，以句號結尾。

    ![[防火牆內容] 屬性工作表的螢幕擷取畫面 ](images/win-property-win-image2.png)

    在此範例中，會在 [一般] 頁面頂端說明 Microsoft Windows 防火牆的用途。

-   **使用一致的控制項名稱和位置，在頁面之間保持類似的內容。** 例如，如果有數個頁面有名稱方塊，請試著將它們放在頁面上的相同位置，並使用一致的標籤。 類似內容不應從頁面彈跳到頁面。
-   **將相同的屬性放在整個應用程式的相同頁面上。** 例如，不要在某個物件類型的 [一般] 索引標籤上，以及另一種類型的 [Advanced] 索引標籤上放置到期屬性。
-   **如果使用者可能會在最後一個顯示的頁面上開始，請將頁面索引標籤保存，並依預設選取它。** 將設定保留在每個屬性的視窗上，以每個使用者為基礎。 否則，預設會選取第一個頁面。
-   **請勿讓頁面上的設定相依于其他頁面上的設定。** 請改為將相依設定放在單一頁面上。 變更單一頁面上的設定時，不應該自動變更其他頁面上的設定。
    -   **例外狀況：** 如果相依設定在兩個不同的屬性視窗中，請使用靜態文字標籤來說明這兩個位置中的這項關聯性。
-   **不要滾動屬性頁。** 索引標籤和捲軸可用來增加視窗的有效區域，但其中一種機制應該就已足夠。 不使用捲軸，讓屬性頁面變得更大，並有效率地配置頁面。

**第一頁**

-   針對物件屬性， **將物件的名稱放在第一頁。**
-   如果您要將 (選擇性) [圖示](vis-icons.md) 與物件產生關聯，請在第一頁的 **左上角顯示適當的圖示** 。

**一般頁面**

-   **避免一般頁面。** 您不需要有 [一般] 頁面。 只有在下列情況時，才使用一般頁面：
    -   這些屬性適用于數個工作，對大部分的使用者而言都有意義。 請勿在 [一般] 頁面上放置特製化或 advanced 屬性，但您可以透過 [一般] 頁面上的 [命令] 按鈕來存取它們。
    -   屬性不符合更明確的類別。 如果有的話，請改為使用該頁面的名稱。

**Advanced 頁面**

-   **避免 [Advanced] 頁面。** 只有在下列情況時，才使用 Advanced 頁面：
    -   這些屬性會套用至不常見的工作，而且主要是對 advanced 使用者有意義的。
    -   屬性不符合更明確的類別。 如果有的話，請改為使用該頁面的名稱。
-   **請勿只以技術量值為基礎來呼叫 advanced 屬性。** 例如，印表機裝訂選項可能是先進的印表機功能，但對所有使用者而言都有意義，因此不應該在 [Advanced] 頁面上。

-   如果所擁有的視窗是用來執行輔助工作，請 **不要在按下命令按鈕時關閉 [擁有者] 屬性視窗。** 這麼做是 disorienting，並假設使用者顯示內容視窗的唯一原因是要執行該命令。

### <a name="tabs"></a>定位點

-   **使用簡潔的索引標籤標籤。** 使用可清楚描述頁面內容的一或兩個字組。 較長的標籤會導致螢幕空間的使用效率不佳，尤其是當標籤已當地語系化時。
-   **使用有意義的特定索引標籤標籤。** 避免可套用至任何索引標籤的一般索引標籤標籤，例如 [一般]、[Advanced] 或 [設定]。
-   **使用水準索引標籤的情況：**
    -   屬性視窗有七個或更少的索引標籤 (包括) 的任何協力廠商延伸模組。
    -   **所有索引標籤都能容納在一個資料列上，即使 UI 已當地語系化也一樣。**
    -   您可以在應用程式中的其他屬性視窗上使用水準索引標籤。
-   **使用垂直索引標籤的情況：**

    -   屬性視窗有八個以上的索引標籤 (包括) 的任何協力廠商延伸模組。
    -   **使用水準索引標籤需要一個以上的資料列。**
    -   您可以在應用程式中的其他屬性視窗上使用垂直索引標籤。

    ![具有垂直索引標籤之屬性視窗的螢幕擷取畫面 ](images/win-property-win-image7.png)

    在此範例中，垂直索引標籤可用來容納八個以上的索引標籤。

-   **針對屬性偵測器，若要節省空間，請考慮使用下拉式清單（而非** 索引標籤），特別是當使用者很少變更目前的索引標籤時。
-   如果索引標籤 **不會套用至目前的內容，而且使用者不會預期它，請移除該** 索引標籤。這麼做可簡化 UI，使用者也不會錯過。

    **不正確：**

    ![灰色的 [檔案位置] 索引標籤的螢幕擷取畫面 ](images/win-property-win-image8.png)

    在此範例中，當 Microsoft Word 2003 當做電子郵件編輯器使用時，會不正確地停用 [檔案位置] 索引標籤。 應該移除頁面，因為使用者不會預期會在此內容中查看或變更檔案位置。

-   **如果索引標籤不適用於目前的內容，使用者可能會預期：**

    -   **顯示索引標籤。**
    -   **停用頁面上的控制項。**
    -   **包含說明控制項停用原因的文字。**

    請勿停用此索引標籤，因為這樣做並不容易理解，且禁止探索。 此外，需要特定內容的使用者才能查看所有其他索引標籤。

    ![灰色視圖控制項的螢幕擷取畫面 ](images/win-property-win-image9.png)

    在此範例中，從 Word 2003 開始，任何 View 選項都不會套用在閱讀配置中。 不過，使用者可能會預期他們會根據索引標籤標籤來套用，因此會顯示頁面，但選項會停用。

-   **請勿指派變更索引標籤的效果。** 變更目前的索引標籤絕不會有副作用、套用設定或產生錯誤訊息。
-   **請勿使用垂直索引標籤來嵌套索引標籤或合併水準索引標籤。** 相反地，請減少索引標籤數目、僅使用垂直索引標籤，或使用另一個控制項（例如下拉式清單）。
-   **如果屬性視窗只有一個索引標籤，而且無法擴充，請不要使用索引標籤。** 請改用 [確定]、[取消] 和 [選擇性套用] 按鈕的一般對話方塊。 可由協力廠商擴充的可延伸屬性 windows () 一律需要使用索引標籤。
-   **請勿將圖示放在索引標籤上。** 圖示通常會加入不必要的視覺效果雜亂、取用螢幕空間，而且通常不會改善使用者理解。 只新增可協助理解的圖示，例如標準符號。

-   **請勿使用標籤圖形的產品標誌。** 索引標籤不適用於商標。
-   **不要滾動水準索引標籤。** 無法立即探索水準滾動。 不過，您可以滾動垂直索引標籤。

### <a name="command-buttons"></a>命令按鈕

-   **放置套用至屬性視窗底部所有屬性頁的命令按鈕。** 將按鈕靠右對齊，並使用此順序 (從左至右) ： [確定]、[取消] 和 [套用]。
-   **只將命令按鈕放在屬性頁上，直接套用至個別的屬性頁。**

### <a name="commit-buttons"></a>認可按鈕

**確定按鈕**

-   **若為 [擁有者] 屬性視窗，[確定] 按鈕表示套用自視窗開啟或上次套用) 之後 (的暫止變更，然後關閉視窗。**
-   **針對擁有的屬性視窗，[確定] 按鈕表示保留變更、關閉視窗，並在套用主控視窗的變更時套用變更。**
-   **請勿重新命名 [確定] 按鈕。** 不同于其他對話方塊，屬性視窗不會用來執行任何一項特定工作。 如果重新命名 [確定] (按鈕（例如) ）是合理的，則視窗不是屬性視窗。
-   **請勿指派存取金鑰。**

**取消按鈕**

-   **[取消] 按鈕表示捨棄自從開啟視窗或上次套用) 之後 (所做的所有暫止變更，然後關閉視窗。**
-   **如果無法放棄所有暫止的變更，請重新命名 [取消] 按鈕以關閉。** 按一下 [取消] 必須放棄所有暫止的變更。
-   **如果 [擁有的屬性] 視窗需要立即認可，請將擁有者視窗上的 [取消] 按鈕重新命名為關閉，以顯示已認可的變更。**
-   **請勿指派存取金鑰。**

**套用按鈕**

-   **在 [擁有者] 屬性工作表中，[套用] 按鈕表示套用自視窗開啟後 (所做的暫止變更，或上次套用) ，但讓視窗保持開啟狀態。** 這麼做可讓使用者在關閉屬性工作表之前評估變更。
-   **若為擁有的屬性工作表，請不要使用。** 使用所擁有屬性工作表上的 [套用] 按鈕，可讓 [擁有者] 屬性工作表上的 [認可] 按鈕的意義難以理解。
-   **只有當屬性工作表的設定 (至少一個) 具有使用者可透過有意義的方式評估的效果時，才提供 [套用] 按鈕。** 一般情況下，[套用] 按鈕會在設定進行可見變更時使用。 使用者應該能夠套用變更、評估變更，然後根據該評估進行進一步的變更。 如果沒有，請移除 [套用] 按鈕，而不要停用它。

    **不正確：**

    ![具有 [套用] 按鈕之系統屬性的螢幕擷取畫面 ](images/win-property-win-image12.png)

    在此範例中，沒有任何系統屬性具有視覺效果，因此 [套用] 按鈕沒有任何值，而且應該移除。

-   **放置使用者可能想要在擁有者頁面上套用的所有設定。** 不要在擁有的屬性工作表上使用 [套用] 按鈕，因為這樣做會造成混淆。
-   **只使用 [套用] 按鈕與 [屬性工作表]，而非 [選項] 對話方塊。**
-   **只有在有暫止的變更時，才啟用 [** 套用] 按鈕;否則，請將它停用。
-   **指派 "A" 作為存取金鑰。**

**關閉按鈕**

-   **如果無法放棄所有暫止的變更，請重新命名 [取消] 按鈕以關閉。** 按一下 [取消] 必須放棄所有暫止的變更。
-   **請勿確認使用者是否捨棄其變更。**
    -   **例外狀況：** 如果 [屬性] 視窗有需要大量設定的設定，而且使用者已進行變更，您可能會在使用者按一下標題列上的 [關閉] 按鈕時顯示 [確認](mess-confirm.md) 。 原因是某些使用者誤以為標題列上的 [關閉] 按鈕與 [確定] 按鈕的效果相同。
-   **除了確認訊息以外，請確定標題列上的 [關閉] 按鈕與 [取消] 或 [關閉] 具有相同的效果。**

### <a name="page-contents"></a>頁面內容

-   **請確定屬性是必要的。** 不要讓您的頁面具有不必要的屬性，就可以避免進行硬性的設計決策。
-   **以使用者目標為依據，而不是以技術呈現屬性。** 因為屬性會設定特定的技術，並不表示您必須根據該技術來呈現該屬性。
    -   如果您必須以技術來呈現設定 (可能是因為您的使用者辨識技術的名稱) ，請提供使用者如何從該設定獲益的簡短描述。
-   **將屬性顯示在正確層級。** 您不需要在屬性頁上顯示個別的低層級設定，因此可讓您的使用者有意義的層級上提供屬性。
-   **針對特定工作設計屬性頁。** 判斷使用者將執行的工作，並確定有明確的路徑可以執行這些工作。
-   藉由減少索引標籤數目、決定根據邏輯分組和連貫性的頁面，以及簡化頁面的簡報，來 **有效率地整理屬性頁**。

### <a name="help"></a>Help

-   提供使用者協助時， **請考慮使用下列選項** (依照其喜好設定) 順序列出：
    -   提供互動式控制項自理解標籤。 使用者比較有可能讀取互動式控制項上的標籤，而不是任何其他文字。
    -   使用靜態文字標籤提供內容中的說明。
    -   提供相關說明主題的特定 [連結](ctrl-links.md) 。
-   **找出每個頁面底部的 [說明] 連結。** 如果某個頁面有幾個不同的設定群組，其中有說明主題 (可能在群組方塊) 內，請找出群組底部的 [說明] 連結。
-   **請勿使用一般或模糊的說明主題連結或一般説明按鈕。** 使用者通常會忽略一般說明。

如需詳細資訊和範例， [請參閱說明](winenv-help.md)。

### <a name="standard-users-and-protected-administrators"></a>標準使用者和受保護的系統管理員

**許多設定都需要系統管理員許可權才能變更。** 如果進程需要系統管理員許可權，Windows 和更新版本需要 [標準使用者](glossary.md) 和 [受保護](glossary.md) 的系統管理員明確地提升其許可權。 這樣做有助於防止惡意程式碼以系統管理員許可權執行。

如需詳細資訊和範例，請參閱 [使用者帳戶控制](winenv-uac.md)。

### <a name="default-values"></a>預設值

-   **屬性視窗內的設定必須反映應用程式、物件或物件集合的目前狀態。** 否則會產生誤導，可能會導致不想要的結果。 例如，如果設定反映出建議，而不是目前的狀態，則使用者可能會按一下 [取消]，而不是進行變更，以為不需要任何變更。
-   **選擇最安全的 (，以防止資料遺失或系統存取) 和最安全的初始狀態。** 假設大部分的使用者不會變更設定。
-   **如果安全性和安全性不是因素，請選擇最有可能或方便的初始狀態。**

## <a name="text"></a>Text

### <a name="commands"></a>命令

-   若要顯示程式選項，請使用 [選項]。
-   若要顯示物件的屬性視窗，請使用「屬性」。
-   若要顯示常用程式自訂設定的摘要，請使用「[個人](glossary.md)化」。
-   請勿使用「設定」或「喜好設定」。
-   請勿將 [省略號](ctrl-command-buttons.md) 用於這些命令。

### <a name="property-sheet-titles"></a>屬性工作表標題

-   若為單一物件，請使用「 \[ 物件名稱 \] 屬性」。
    -   如果物件沒有名稱，請使用物件的類型名稱。  (例如使用者帳戶屬性。 ) 
-   針對多個物件，請使用「 \[ 第一個物件名稱 \] ，.。。屬性。」
    -   如果物件沒有名稱，請使用物件的類型名稱。 例如， (使用者帳戶屬性。 ) 
    -   如果物件有不同的類型，請使用「選取屬性」。
-   使用 [標題樣式的大小寫](glossary.md)。
-   請勿使用結束標點符號。
-   請勿使用連字號，例如「 \[ 物件名稱 \] -屬性」。

### <a name="property-inspector-titles"></a>屬性偵測器標題

-   使用「屬性」。
-   使用標題樣式的大小寫。
-   請勿使用結束標點符號。

### <a name="options-dialog-box-titles"></a>選項對話方塊標題

-   使用 [選項]。
-   使用標題樣式的大小寫。
-   請勿使用結束標點符號。

### <a name="property-page-tab-names"></a>屬性頁索引標籤名稱

-   **使用簡潔的索引標籤標籤。** 使用可清楚描述頁面內容的一或兩個字組。 使用較長的索引標籤名稱會導致螢幕空間的使用效率不佳，尤其是在已當地語系化索引標籤名稱的情況下。
-   **使用有意義的特定索引標籤標籤。** 避免可套用至任何索引標籤的一般索引標籤標籤，例如 [一般]、[Advanced] 或 [設定]。
-   將標籤寫入為一或兩個單字的片語，且不使用結束標點符號。
-   使用 [句子樣式的大小寫](glossary.md)。
-   請勿指派唯一的 [存取金鑰](glossary.md)。

### <a name="property-page-text"></a>屬性頁文字

-   避免大型文字區塊。
-   為文字提供足夠的空間，以在其當地語系化時將其擴充30%。
-   請勿使用文字片語作為屬性視窗上的命令。 由於使用者可能只想要查看設定，因此您不會想要提示他們變更設定。
-   使用句子樣式的大小寫和結束標點符號。

## <a name="documentation"></a>文件

參考屬性視窗時：

-   在程式設計和其他技術檔中，請將 [屬性工作表] 和 [選項] 對話方塊稱為屬性工作表。 在其他地方使用對話方塊，尤其是在使用者檔中。
-   使用確切的標題文字，包括其大小寫。
-   若要描述使用者互動，請使用 [開啟] 和 [關閉]。
-   可能的話，請使用粗體文字來格式化標題。 否則，請只在必要時才將標題放在引號中，以避免混淆。

參考屬性頁時：

-   在程式設計和其他技術檔中，請將屬性頁稱為屬性頁。 在其他地方，使用 tab 鍵，尤其是在使用者檔中。
-   使用確切的標題文字，包括其大小寫。
-   若要描述使用者互動，請使用按一下滑鼠右鍵，以參考按一下索引標籤。
-   可能的話，請使用粗體文字來格式化名稱。 否則，請只在必要時才將名稱放在引號中，以避免混淆。
