## <a name=""></a><a name="alink">A</a>

**關於 box**

提供一般程式資訊的對話方塊，例如版本識別碼、著作權、授權合約，以及存取技術支援的方式。

**高於折迭**

從報紙新聞學拍攝的螢幕佈局比喻。 報紙折迭的內容必須特別吸引激勵 sales。 同樣地，在螢幕版面配置中，必須在沒有滾動的情況下顯示最重要的內容。 使用者必須採取一些時間，才會在一開始就在折迭的上方看到內容。」

**存取金鑰**

與 ALT 鍵結合的英數位鍵，會啟用控制項。 存取金鑰的表示方式是將控制項標籤中的其中一個字元加上底線。 例如，按 Alt + O 會啟用標籤為 "Open" 且指派的存取金鑰為 "O" 的控制項。 存取金鑰不區分大小寫。 啟用控制項的效果取決於控制項的類型。

相較于最適合用於 advanced 使用者的快速鍵，存取金鑰是設計來改善協助工具。 因為它們會直接記錄在使用者介面 (UI) 本身，所以不能一律以一致的方式指派，也不能記下。

**主動監視**

正在執行使用中程式的監視。

**網址列**

導覽元素（通常出現在視窗頂端）會顯示，並允許使用者變更其目前位置。 另請參閱：階層連結 [欄](#b)。

**affordance**

物件的視覺屬性，指出如何使用或處理它。 例如，命令按鈕具有真實世界按鈕的視覺外觀，以建議推送或按一下。

**應用程式**

用來執行一組相關使用者工作的程式;通常相當複雜且精密。 另請參閱： [程式](#p)。

**應用程式金鑰**

具有內容功能表圖形的鍵盤按鍵。 此索引鍵是用來顯示所選取專案的內容功能表。

**應用程式功能表**

提供命令功能表的控制項，這些命令牽涉到與檔或工作區（例如檔案相關的命令）執行某些工作。

顯示目前選取範圍的內容功能表 (與按下 Shift + F10) 。

**應用程式主題**

使用相關的視覺效果技術（例如自訂控制項），為應用程式建立獨特的外觀或商標。

**外觀比例**

物件寬度與其高度之間關聯性的運算式。 例如，高定義電視使用16:9 的外觀比例。

**自動完成**

在文字方塊中使用的清單類型和可編輯的下拉式清單，其中可能的輸入可以推斷並自動填入先前輸入的。 使用者輸入最少的文字數量來填入自動完成清單。

**自動結束**

文字方塊，在使用者輸入最後一個字元時，輸入焦點就會自動移至下一個相關的文字方塊。

## <a name="b"></a>B

**氣球**

常見的 Windows 控制項，可通知使用者非重大問題或特殊狀況。

**階層連結列**

導覽元素（通常出現在視窗頂端）會顯示，並允許使用者變更其目前位置。 「階層連結」是指將目前的位置分成一連串的連結，使用者可以直接與使用者互動，並以箭號分隔。 改為使用網址列。 另請參閱： [位址](#alink)列。

## <a name="c"></a>C

**核取方塊**

一般的 Windows 控制項，可讓使用者在明顯不同的選項之間做出決定，例如切換選項開啟或關閉。

**雪 佛 龍**

小型的控制項或按鈕，表示所配置空間中的專案數超過可顯示的專案數。 使用者按一下箭號以查看其他專案。

**子視窗**

完全包含在另一個視窗中的視窗（例如控制項或窗格），稱為父視窗。 另請參閱： [父視窗](#p)、 [擁有的視窗](#o)。

**已清除**

在核取方塊中，表示未設定此選項。 另請參閱：已 [選取](#s)、 [混合的狀態](#m)。

**下拉式方塊**

結合下拉式清單或標準清單方塊之特性的通用 Windows 控制項，以及一個可編輯的文字方塊。 另請參閱： [清單方塊](#l)、 [下拉式清單](#d)。

**命令區域**

視窗中的 [認可] 按鈕所在的區域。 對話方塊和嚮導通常會有命令區域。 另請參閱：認可按鈕。

**命令按鈕**

一般的 Windows 控制項，可讓使用者立即起始動作。

**命令連結**

用來在一組互斥、相關選項之間做選擇的控制項。 在其正常狀態下，命令連結的外觀類似于超連結，但其行為更類似命令按鈕。

**認可按鈕**

用來認可工作的命令按鈕，繼續進行多步驟工作中的下一個步驟，或取消工作。 另請參閱：命令區。

**認可頁面**

一種嚮導頁面，使用者可在其中認可執行工作。 這麼做之後，您可以按一下 [上一步] 或 [取消] 按鈕來復原工作。

**完成頁面**

用來表示 wizard 結尾的 wizard 頁面。 有時會使用，而不是恭喜頁面。 另請參閱：恭喜頁面。

**恭喜頁面**

用來表示 wizard 結尾的 wizard 頁面。 不再建議使用這些頁面。 使用認可頁面，或（如有需要）後續或完成頁面，即可更有效率地結束使用。 另請參閱：認可頁面、完成頁面、 [後續頁面](#f)。

**同意 UI**

使用者帳戶控制所使用的對話方塊 (UAC) ，可讓受保護的系統管理員暫時提升其許可權。

**constraint (條件約束)**

在牽涉到使用者輸入的控制項（例如文字方塊）中，輸入條件約束是避免錯誤的重要方法。 例如，如果特定控制項的唯一有效輸入是數值，則控制項可以使用適當的值條件約束來強制執行這項需求。

**內容區域**

UI 表面的部分，例如對話方塊、控制台專案和嚮導，專門用來呈現選項、提供資訊和描述控制項。 從命令區域、工作窗格和導覽區域辨別。

**內容索引標籤**

此索引標籤包含只有當使用者選取特定物件類型時，才會相關的命令集合。 另請參閱： [功能區](#r)。

**控制台**

為使用者收集及顯示電腦系統層級功能的 Windows 程式，包括硬體和軟體安裝與設定。 在主控台中，使用者可以按一下個別專案來設定系統層級的功能，並執行相關工作。 另請參閱：控制台專案。

**控制台專案**

主控台提供的個別功能。 例如，[程式和輕鬆存取] 都是兩個 [控制台] 專案。

**認證 UI**

使用者帳戶控制所使用的對話方塊 (UAC) ，可讓標準使用者要求其許可權的暫時提升許可權。

**關鍵**

嚴重性的最高程度。 例如，在錯誤和警告訊息中，重大情況可能牽涉到資料遺失、隱私權遺失或系統完整性遺失。

**自訂圖示**

程式 (獨特的圖形化標記法，而不是 Windows 系統圖示) 。

**自訂視覺效果**

特別為程式開發的圖形、動畫、圖示和其他視覺元素。

## <a name="d"></a>D

**預設命令按鈕或連結**

當使用者按下 Enter 鍵時，所叫用的命令按鈕或連結。 預設的命令按鈕或連結是由開發人員所指派，但是在使用者索引標籤時，任何命令按鈕或連結都會變成預設值。

**預設監視**

具有 [開始] 功能表、工作列和通知區域的監視器。

**延遲的認可模型**

在使用者按一下 [認可] 按鈕明確認可之前，不會進行變更的控制台專案輪輻頁面所使用的認可模型。 因此，使用者可以放棄工作、使用 [上一頁] 按鈕、關閉或網址列離開。 另請參閱： [立即認可模型](#i)。

**桌面**

Windows 所提供的螢幕上的工作區，類似于實體桌面。 另請參閱： [工作區](#w)。

**破壞性命令**

具有廣泛效果且無法輕易復原或不會立即被察覺的動作。

**詳細資料窗格**

Windows 檔案總管視窗底部的窗格，會顯示有關所選取專案之任何) 的詳細資料 (;否則，它會顯示資料夾的詳細資料。 例如，Windows 影像中心會顯示圖片名稱、檔案類型、拍攝日期、標籤、評等、維度和檔案大小。 另請參閱： [預覽窗格](#p)。

**對話方塊**

允許使用者執行命令、詢問使用者問題，或是為使用者提供資訊或進度回饋的次要視窗。

**對話方塊啟動器**

在功能區中，位於某些群組底部的按鈕，其會開啟對話方塊，其中包含與群組相關的功能。 另請參閱： [功能區](#r)。

**對話方塊單位**

對話單位 (DLU) 是與裝置無關的量值，用於根據目前系統字型配置的版面配置。

**直接操作**

使用者與 UI 中物件之間的直接互動 (例如圖示、控制項和導覽元素) 。 滑鼠和觸控是直接操作的常用方法。

**固定視窗**

出現在其擁有者視窗邊緣之固定位置的視窗。 另請參閱： [浮動視窗](#f)。

**下拉箭號**

與下拉式清單、下拉式方塊、分割按鈕和功能表按鈕相關聯的箭號，表示使用者可以按一下箭號來查看相關聯的清單。

**下拉式清單**

通用的 Windows 控制項，可讓使用者從互斥值的清單中選取。 與清單方塊不同的是，此可用選項清單通常是隱藏的。

## <a name="e"></a>E

**有效解決方式**

以目前 DPI () 設定的 DPI 為單位正規化的監視器實體解析度。 在 96 DPI，有效解析度與實體解析相同，但在其他 DPI 中，有效解析度必須按比例調整。 一般來說，您可以使用下列方程式來計算有效的解決方式：

有效解析度 = 實體解析度 x (96/目前的 DPI 設定) 

另請參閱： [相對圖元](#r)、 [實體解析度](#p)。

**提高許可權的系統管理員**

在 [使用者帳戶控制] 中，提高許可權的系統管理員擁有系統管理員許可權。 如果沒有提高許可權，系統管理員會以最低許可權的狀態執行。 只有在必要時，才會使用同意 UI 對話方塊來提高系統管理員的許可權。 另請參閱： [受保護的系統管理員](#p)、 [標準使用者](#s)。

**增強型工具提示**

簡潔說明所指向命令的快顯視窗。 就像一般的工具提示一樣，增強的工具提示可能會提供命令的快速鍵。 但與一般工具提示不同的是，他們也可以提供補充資訊、圖形，以及可提供協助的指標。 它們也可以使用 rtf 和分隔字元。 另請參閱： [工具提示](#t)。

**error**

發生問題的狀態。 另請參閱： [警告](#w)。

**可擴充的標題**

逐漸洩漏的燕形箭號，可展開或折迭標題以顯示或隱藏專案群組。 另請參閱： [漸進式洩漏](#p)。

**擴充選取範圍**

在清單視圖和清單方塊中，您可以拖曳或使用 Shift + 按一下或 Ctrl + 按一下，分別選取連續或非相鄰值的群組，藉以擴充選取單一專案的多重選取模式。 另請參閱： [多重選取專案](#m)。

## <a name="f"></a>F

**電影**

螢幕上的手指或畫筆的快速、直接筆觸。 筆觸會辨識為手勢，並被解釋為導覽或編輯命令。

**浮動視窗**

可以出現在使用者想要的螢幕上任何位置的視窗。 另請參閱：停駐 [視窗](#d)。

**彈出**

暫時顯示詳細資訊的快顯視窗。 在 Windows 桌面上，按一下小工具以顯示 flyouts，然後按一下飛出視窗外的任何位置來關閉。 您可以在停駐和浮動狀態中使用 flyouts。

**後續頁面**

用來呈現使用者可能會進行後續操作的相關工作的 wizard 頁面。 有時會使用，而不是恭喜頁面。

**字體**

一組文字字元的屬性。

**全螢幕**

沒有框架的最大化視窗。

## <a name="g"></a>G

**工具**

裝載在使用者桌面上的簡單迷你應用程式。 另請參閱： [提要](#s)欄位。

**庫**

以圖形方式呈現的命令或選項清單。 以結果為基礎的資源庫說明命令或選項（而非命令本身）的作用。 可能標示或分組。 例如，您可以在縮圖圖庫中顯示格式選項。

**姿態**

在電腦上將手指或畫筆轉譯為命令，而不是以滑鼠移動、書寫或繪圖的方式快速移動。

**開始使用頁面**

選用的 wizard 頁面，其中概述成功執行 wizard 的必要條件，或說明 wizard 的用途。

**玻璃**

以 translucence 為特色的視窗框架選項，協助使用者專注于內容和功能，而不是周圍的介面。

**字形**

用來參考任何圖形或符號影像的泛型詞彙。 箭號、燕號和專案符號是 Windows 中常用的圖像。

**群組方塊**

顯示一組相關控制項之間關聯性的通用 Windows 控制項。

## <a name="h"></a>H

**手寫辨識**

將筆墨轉換成文字的軟體。

**說明**

比主要 UI 提供更詳細的本質的使用者協助。 通常會從功能表存取，或按一下 [說明] 連結或圖示，此內容可能會採用各種形式，包括逐步程式、概念文字或更以視覺方式呈現的教學課程。

**高對比模式**

針對前景和背景視覺效果元素提供極大對比的特殊顯示設定 (黑色) 的白色或白色。 對協助工具特別有用。

**中樞頁面**

在 [控制台] 專案中，中樞頁面會顯示高階選項，例如 (與以工作為基礎的中樞頁面) 的最常用工作，或 (與以物件為基礎的中樞頁面) 的可用物件。 使用者可以流覽至輪輻頁面來執行特定的工作。 另請參閱： [輪輻頁面](#s)。

**混合式中樞頁面**

在 [控制台] 專案中，混合式中樞頁面是中樞頁面，其中也有直接的一些屬性或命令。 當使用者最有可能使用 [控制台] 專案來存取這些屬性和命令時，強烈建議使用混合式中樞頁面。

## <a name="i"></a>I

**立即認可模型**

混合式中樞頁面所使用的認可模型，變更會在使用者進行變更時立即生效。 此模型中不會使用認可按鈕。 另請參閱： [延遲的認可模型](#d)。

**就地訊息**

出現在目前 UI 介面內容中的訊息，而不是另一個視窗。 不同于不同的視窗，就地訊息需要可用的螢幕空間或動態版面配置。

**間接對話方塊**

顯示在內容中的對話方塊，可能是因為工作的間接結果，或是系統或背景進程發生問題的結果。

**感應使用者介面**

使用明確的目的，將複雜的工作細分為簡單、簡單說明、清楚陳述的步驟的 UI。

**提示**

簡要描述所指向物件的小型快顯視窗，例如工具列控制項、圖示、圖形、連結、Windows 檔案總管物件、[開始] 功能表專案和工作列按鈕的描述。 提示是一種逐漸洩漏的形式，不需要在畫面上隨時具有描述性文字。

**油墨**

畫筆的原始輸出。 這個數位筆跡可以像書面一樣保持不變，也可以使用手寫辨識軟體將它轉換成文字。

**內嵌**

將連結或訊息直接放在其相關 UI 的內容中。 例如，內嵌連結會出現在其他文字內，而不是個別。

**輸入焦點**

使用者目前導向輸入的位置。 請注意，由於 UI 中的位置已反白顯示，因此不一定表示此位置具有輸入焦點。

**實例**

程式會話。 例如，Windows Internet Explorer 可讓使用者執行程式的多個實例，因為使用者一次可能會有數個獨立的會話正在執行。 您可以跨程式會話儲存設定。 另請參閱： [持續](#p)性。

## <a name="j"></a>J

## <a name="k"></a>K

**keytip**

在功能區中，用來顯示存取金鑰的機制。 存取金鑰會以小型提示的形式出現在每個命令或群組上，相對於通常用來顯示存取金鑰的加底線字母。 另請參閱： [存取金鑰](#alink)。

## <a name="l"></a>L

**橫向模式**

將物件排列成寬度超過高度的呈現選項。 另請參閱：直向 [模式](#p)。

**最低許可權使用者帳戶**

一般以較低許可權執行的使用者帳戶。 另請參閱： [使用者帳戶控制](#u)。

**清單方塊**

一般的 Windows 控制項，可讓使用者從清單中顯示的一組值中進行選取（與下拉式清單不同的是，這種方式不同于下拉式清單）。 支援單一或多個選取專案。

**清單視圖**

一種通用的 Windows 控制項，可讓使用者使用單一選取或多重選取專案來查看資料物件的集合，並與之互動。

**即時預覽**

一種預覽技術，可顯示命令在選取或暫留時的效果，而不會讓使用者認可動作。 例如，主題、字型和色彩等格式化選項，可讓使用者以最不費力的效果來顯示效果，以受益于即時預覽。

**定位**

針對不同的國家/地區、語言、文化特性或市場調整軟體的流程。

**記錄檔**

以檔案為基礎的存放庫，用來取得電腦系統上的各種活動相關資訊。 系統管理員通常會參考記錄檔;一般使用者通常不會這麼做。

## <a name="m"></a>M

**主要指示**

以醒目方式顯示的文字，簡要說明要在視窗或頁面中進行的動作。 指令應該是特定的語句、命令式方向或問題。 良好的主要指示傳達使用者的目標，而不是只著重在操作 UI。

**受控環境**

由 IT 部門或協力廠商提供者（而不是個別使用者）管理的網路電腦環境。 系統管理員可以將效能優化，並套用作業系統和應用程式更新，還有其他工作。

**manipulation**

一種觸控互動，其中的輸入會直接對應到所觸及的物件如何自然地回應真實世界中的動作。

**最大化**

以最大的大小來顯示視窗。 另請參閱：最小化、 [還原的視窗](#r)。

**功能表**

可供使用者在目前內容中使用的命令或選項清單。

**訊息方塊**

顯示用來通知使用者特定條件的次要視窗。

**迷你工具列**

停留時顯示的內容工具列。

**最小 化**

隱藏視窗。 另請參閱：最大化、 [還原的視窗](#r)。

**混合狀態**

針對套用至一組專案的核取方塊，混合狀態表示已選取某些專案，而其他專案則會被清除。

**模 態**

因為在模式中運作，所以有限制性或受限的互動。 強制回應通常會描述可限制使用者與擁有者視窗互動的次要視窗。 另請參閱：非模式。

**非強制回應**

不受限或不受限的互動。 非強制回應通常會描述不會限制使用者與擁有者視窗互動的次要視窗。 另請參閱：強制回應。

**多重選取**

使用者可以在清單或樹狀結構中選擇一個以上的物件。

## <a name="n"></a>N

**非重大系統事件**

不需要立即注意的系統事件類型，通常與系統狀態有關。 另請參閱： [重大](#c)。

**通知**

短暫顯示給使用者的非關鍵本質資訊;通知會從工作列的通知區域中的圖示，取得球的表單。

## <a name="o"></a>O

**加入宣告**

使用者可以明確地選取選擇性功能。 比退出宣告更不具侵入性，特別是針對隱私權和行銷的相關功能，因為沒有使用者的願望了推測:。 另請參閱：退出宣告、選項。

**選擇退出**

使用者可以藉由清除其選取專案來移除他們不想要的功能。 比加入宣告更具侵入性的使用者，特別是針對隱私權和行銷的相關功能，因為假設有使用者的願望。 另請參閱：加入宣告、選項。

**options**

可供使用者自訂程式的選項。 例如，[選項] 對話方塊可讓使用者查看和變更程式選項。 另請參閱： [屬性](#p)。

**非內容 UI**

顯示在快顯視窗中且未直接與使用者目前活動相關的任何 UI。 例如，使用者存取控制的通知和同意 UI 是不在內容上的 UI。

**擁有的視窗**

用來執行輔助工作的次要視窗。 它不是最上層視窗 (所以不會顯示在工作列) ;相反地，它是由其擁有者視窗「擁有」。 例如，大部分的對話方塊都是擁有的視窗。 另請參閱： [子視窗](#c)、擁有者視窗。

**擁有者控制項**

提示、氣球或飛出視窗的來源。 例如，具有輸入條件約束的文字方塊可能會顯示提示，讓使用者知道這些限制。 在此情況下，會將文字方塊視為擁有者控制項。

**擁有者視窗**

從中產生所擁有視窗的視窗。 以 Z 順序顯示在擁有的視窗底下。 另請參閱：擁有的視窗、 [父視窗](#p)、 [Z 順序](#z)。

## <a name="p"></a>P

**page**

工作型 UI 的基本導覽單位，例如，[嚮導]、[屬性工作表]、[控制台專案] 和 [網站]。 使用者可以從單一主控制項視窗內的頁面流覽至頁面來執行工作。 另請參閱：頁面流程、 [視窗](#w)。

**頁面流程**

使用者執行工作的頁面集合。 另請參閱：頁面[、工作、](#t) [wizard](#w)[主控台](#c)。

**頁面空間控制項**

可讓使用者查看階層式排列的物件集合，並與其互動。 頁面空間控制項就像是樹狀控制項，但視覺外觀稍微不同。 它們主要是 Windows 檔案總管使用。

**[調色板] 視窗**

顯示工具列或其他選項的無模式次要視窗，例如色彩、模式、字型或字型屬性。

**泛**

若要以兩個維度移動場景，例如地圖或相片，請直接拖曳。 這與以兩種方式進行滾動的方式不同：滾動內容通常會有一個主要維度，而且通常只會沿著該維度進行滾動;而以逆的方式滾動內容時，會顯示捲軸，讓使用者以相反方向拖曳滾動動作。

**窗 格**

視窗內的矩形區域，使用者可能可以移動、調整大小、隱藏或關閉。 窗格一律停駐在其父視窗的側邊。 它們可以與其他窗格相鄰，但永遠不會重迭。 移除窗格會將它轉換成子視窗。 另請參閱： [window](#w)。

**父視窗**

子視窗 (的容器，例如控制項或窗格) 。 另請參閱： [擁有者視窗](#o)。

**筆**

用於指標、手勢、簡單文字輸入和自由格式手寫的手寫筆。 畫筆有一個順暢、流暢的秘訣，可支援以筆跡精確地指向、書寫或繪製。 它們也可以有選擇性的畫筆按鈕 (用來執行以滑鼠右鍵按一下的) 和橡皮擦 (用來清除筆墨) 。

**堅持**

系統會自動保留物件的狀態或屬性的原則。

**個人化**

自訂對使用者的個人身分識別與程式很重要的核心體驗。 相反地，一般的選項和屬性對使用者使用程式的個人身分識別並不重要。

**角色**

虛構人員的詳細說明。 角色是由清楚瞭解、與真實人員相關的高度指定資料所構成。

**實體解析**

電腦監視器的硬體可顯示的水準和垂直圖元。

**彈出群組按鈕**

在功能區中，會合並群組內所有命令和選項的功能表按鈕。 用來在小型空間中顯示功能區。

**直向模式**

將物件排列成高於寬度的呈現選項。 另請參閱： [橫向模式](#l)。

**偏好**

請勿使用。 請改用 [選項](#o) 或屬性。

**預覽**

當使用者選取選項時，會看到哪些內容的表示。 預覽可以靜態方式顯示為選項的一部分，或在要求中使用 [預覽] 或 [套用] 按鈕。

**預覽窗格**

用來顯示所選物件之預覽和其他資料的視窗窗格。

**主要命令**

滿足視窗主要用途的集中動作。 例如，Print 是 [列印] 對話方塊的主要命令。 另請參閱： [次要命令](#s)。

**主要工具列**

設計為足以避免使用功能表列的命令集合。 另請參閱： [補充工具列](#s)。

**主視窗**

主視窗沒有擁有者視窗，而且會顯示在工作列上。 主要的程式視窗一律是主要視窗。 另請參閱： [次要視窗](#s)。

**程式**

電腦可以執行的一連串指令。 常見的程式類型包括生產力應用程式、取用者應用程式、遊戲、kiosk 及公用程式。

**進度列**

將特定作業的進度顯示為圖形化列的通用 Windows 控制項。

**漸進式洩漏**

這項技術可讓使用者視需要 (一般、資料、選項或命令) 顯示較少使用的資訊。 例如，如果有時候需要更多選項，使用者可以按一下箭號按鈕，在內容中公開這些選項。

**漸進式擴大**

當事件變得更重要時，用來通知使用者的 UI 會逐漸更內斂的順序。 例如，通知可用於使用者可以在一開始就安全地忽略的事件。 當情況變得很重要時，應使用更內斂的 UI，例如強制回應對話方塊。

**prompt**

放在文字方塊內或可編輯的下拉式清單中的標籤或簡短指令，作為其預設值。 與靜態文字不同的是，當使用者在控制項中輸入內容或取得輸入焦點時，就會消失提示。

**properties**

使用者可以變更之物件的設定，例如檔案的名稱和唯讀狀態，以及使用者無法直接變更之物件的屬性，例如檔案的大小和建立日期。 屬性通常會定義物件的狀態、值或外觀。

**受保護的系統管理員**

在 [使用者帳戶控制] 中，系統管理員以最低許可權狀態執行。 另請參閱：提高 [許可權的系統管理員](#e)、 [標準使用者](#s)。

## <a name="q"></a>Q

**快速存取工具列**

一個可供自訂的小型工具列，可顯示常用的命令。

**快速啟動 bar**

Windows 桌面上的直接存取點（位於 [開始] 按鈕旁邊），填入使用者選擇之程式的圖示。 已在 Windows 7 中移除。

## <a name="r"></a>R

**選項按鈕**

通用的 Windows 控制項，可讓使用者從一組互斥、相關的選項中進行選取。

**相對圖元**

與裝置無關的計量，與 96 DPI 的實體圖元相同， (每英寸) 的點，但在其他 DPI 中依比例調整。 另請參閱： [有效的解決](#e)方式。

**還原的視窗**

可見的部分畫面視窗，不會最大化或最小化。 另請參閱： [最大化](#m)、最小化。

**絲帶**

命令和選項的索引標籤式容器，位於視窗或工作區頂端，並具有固定的位置和高度。 功能區通常有應用程式功能表和快速存取工具列。 另請參閱： [功能表](#m)、 [工具列](#t)。

**具風險的動作**

可能具有負面後果且無法輕易復原的使用者動作品質。 有風險的動作包括可能會危害電腦安全性性的動作、影響電腦的存取，或導致非預期的資料遺失。

## <a name="s"></a>S

**掃描路徑**

當使用者進行掃描以找出視窗中的專案時，可能會執行路由。 特別重要的是，如果使用者並未參與文字的沉浸式閱讀。

**螢幕閱讀程式**

一種輔助技術，可讓視力受損的使用者藉由將視覺效果轉換成音訊來解讀和流覽使用者介面。 因此，文字、控制項、功能表、工具列、圖形和其他螢幕專案都是由螢幕讀取器的電腦化語音所讀出。

**捲軸**

控制項，可讓使用者以垂直或水準方式滾動視窗的內容。

**次要命令**

雖然有説明，但對視窗用途而言並不重要的周邊動作。 例如，[尋找印表機] 或 [安裝印表機] 是 [列印] 對話方塊的次要命令。 另請參閱： [主要命令](#p)。

**次要視窗**

具有擁有者視窗，因此不會顯示在工作列上的視窗。 另請參閱： [主視窗](#p)。

**安全桌面**

受保護的環境，與系統上執行的程式隔離，用來提高高度安全工作（例如登入、密碼變更和 UAC 提高許可權 UI）的安全性。 另請參閱： [使用者帳戶控制](#u)。

**安全性保護盾**

用於安全性商標的盾牌圖示。

**選擇**

由使用者選擇，以便執行作業;強調。

**句子樣式大小寫**

若為句子樣式的大小寫：

-   一律將新句子的第一個單字大寫。
-   除非字組是適當的名詞，或是冒號後面的文字是完整的句子，否則請不要將字的後面加上冒號。
-   請勿以 em 虛線來將字組大寫，除非它是正確的名詞，即使虛線後面的文字是完整的句子。
-   一律將新句子的第一個單字大寫，並遵循任何結尾標點符號。 請重寫以區分大小寫的小寫字開頭的句子。

**設定**

 (使用者或預設) 設定程式或物件時，所選擇的特定值。

**快速鍵**

使用者可以按的按鍵或按鍵組合，以快速存取它們經常執行的動作。 Ctrl + 字母組合和函式按鍵 (F1 到 F12) 通常是快速鍵的最佳選擇。 根據定義，快速鍵是鍵盤上對等的功能，可在介面中的其他地方充分支援。 因此，請避免使用快速鍵作為存取特定作業的唯一方式。

相對於存取金鑰（其設計目的是為了改善協助工具），快速鍵的設計主要是供 advanced 使用者使用。 因為它們不會直接記錄在 UI 本身的 (雖然它們可能會記錄在功能表和工具列工具提示) 中，但它們的目的是要記下，因此必須在應用程式內和不同的應用程式之間一致地指派。

**提要欄位**

使用者桌面上的區域，用來顯示 Windows Vista 中的小工具。 另請參閱： [小工具](#g)。

**單點錯誤**

與單一控制項相關的使用者輸入錯誤。 例如，輸入不正確的信用卡號碼是單點錯誤，而不正確的登入是雙點錯誤，因為使用者名稱或密碼可能是問題。

**滑 塊**

常見的 Windows 控制項，可顯示並設定連續範圍的可能值（例如亮度或音量）的值。

**特殊體驗**

在 [程式] 中，特殊體驗與程式的主要功能有關，這是程式的獨特功能，或對使用者建立情緒的連接。 例如，播放音訊或影片是媒體播放機的特殊體驗。

**微調方塊**

文字方塊及其關聯之微調控制項的組合。 使用者按一下微調方塊的向上或向下箭號，以增加或減少數值。 不同于用於相對數量的滑杆控制項，微調方塊只會用於確切的已知數值。

**微調控制項**

使用者按一下以變更值的控制項。 微調控制項會使用向上和向下箭號來增加或減少值。

**啟動顯示畫面**

顯示為程式的過渡畫面影像正在啟動中。

**分割按鈕**

Bipartite 命令按鈕，其中包含一個小按鈕，其中主要按鈕的最右邊部分具有向下指向三角形。 使用者按一下三角形，可在下拉式功能表中顯示命令的變化。 另請參閱： [命令按鈕](#c)。

**輪輻頁面**

在 [控制台] 專案中，輪輻頁面是使用者執行工作的地方。 有兩種類型的輪輻頁面：工作頁面和表單頁面：工作頁面會在工作中顯示工作，或使用以工作為基礎的特定主要指令進行的步驟;表單頁面會根據一般的主要指示，顯示相關屬性和工作的集合。 另請參閱： [中樞頁面](#h)。

**標準使用者**

在 [使用者帳戶控制] 中，標準使用者在電腦上擁有最低許可權，而且必須從電腦的系統管理員要求許可權，才能執行系統管理工作。 與受保護的系統管理員不同的是，標準使用者無法自我提升。 另請參閱：提高 [許可權的系統管理員](#e)、 [受保護的系統管理員](#p)。

**靜態文字**

不屬於互動式控制項的使用者介面文字。 包含標籤、主要指示、補充指示和補充說明。

**補充指示**

選用形式的使用者介面文字，可將資訊、詳細資料或內容新增至主要指令。 另請參閱： [main 指令](#m)。

**補充工具列**

設計來搭配功能表列使用的命令集合。 另請參閱： [主要工具列](#p)。

**系統色彩**

Windows 針對特定用途所定義的色彩，可使用 GetSysColor 應用程式設計介面 (API) 來存取。 例如，[色彩 \_ 視窗] 會定義視窗背景色彩和色彩 WINDOWTEXT，以 \_ 定義視窗文字色彩。 系統色彩不像主題色彩那麼豐富。 另請參閱： [主題色彩](#t)。

**系統功能表**

基本視窗命令的集合，例如 [移動]、[大小]、[最大化]、[最小化] 和 [關閉]，可從標題列上的程式圖示取得，或以滑鼠右鍵按一下工作列按鈕。

## <a name="t"></a>T

**索引標籤式對話方塊**

對話方塊，其中包含個別標記頁面的相關資訊 (索引標籤) 。 不同于通常也包含索引標籤的屬性工作表，不會使用索引標籤式對話方塊來顯示物件的屬性。 另請參閱： [屬性](#p)。

**任務**

使用者活動的單位，通常會以單一 UI 介面表示 (例如對話方塊) ，或 (例如 wizard) 的一連串頁面。

**工作對話方塊**

使用工作對話方塊 API 執行的對話方塊。 需要 Windows Vista 或更新版本。

**工作流程**

一系列的頁面，可協助使用者在 wizard、explorer 或瀏覽器中執行工作。

**工作連結**

用來起始工作的連結，相較于流覽至其他頁面或視窗的連結，選擇 [選項] 或 [顯示説明]。

**工作窗格**

類似于對話方塊的 UI 類型，不同之處在于它會在視窗窗格中顯示，而不是在另一個視窗中顯示。 如此一來，工作窗格就會有比對話方塊更直接、內容相關的感覺。 工作窗格可以包含功能表，以提供使用者一個與所選物件或程式模式相關的小型命令集。

**任務**

執行具有桌上型電腦狀態之程式的存取點。 使用者會與名為工作列按鈕的控制項互動，以顯示、隱藏及最小化程式視窗。

**文字方塊**

專為文字輸入設計的控制項;允許使用者查看、輸入或編輯文字或數位。

**主題色彩**

Windows 針對特定用途所定義的色彩，使用 GetThemeColor API 以及元件、狀態和色彩進行存取。 例如，windows 元件定義了 FillColor 和 TextColor。 主題色彩比系統色彩更豐富，但需要主題服務才能執行。 另請參閱： [系統色彩](#s)。

**標題樣式的大小寫**

針對標題樣式的大小寫：

-   全部大寫的名詞、動詞 (包括，以及其他形式的) 、副詞 (包括) 、形容詞 (（包括 this 和) ），以及代名詞 (包括其) 。
-   將第一個和最後一個字組大寫，不論它們的語音部分為何 (例如，要尋找) 的文字。
-   將屬於動詞 (片語的介係詞大寫，例如，備份您的磁片) 。
-   除非文章是標題中的第一個單字，否則請不要將文章 (、) 。
-   請勿將座標結合變成大寫，除非 (是標題中的第一個單字，否則為、、或) 。
-   除非介係詞是標題中的第一個單字，否則請不要將四個或更少字母的介係詞大寫。
-   請勿以不定片語 (例如，除非片語是標題中的第一個單字，否則如何格式化硬碟) 。
-   如果是名詞或適當的形容詞、「電子字」或單字有相等的權數 (（例如電子商務、交叉參考、預先 Microsoft 軟體、讀取/寫入存取、執行時間) ），請將複合字組中的第二個單字變成大寫。 如果第二個單字是語音的另一個部分（例如，介係詞或其他次要字 (例如增益集、操作說明、) ，請不要將它變成大寫。
-   將使用者介面和應用程式開發介面詞彙全部大寫，除非它們區分大小寫 (例如，fdisk 命令) 。 遵循傳統的關鍵字和程式設計語言中的其他特殊詞彙 (例如，printf 函式，使用偶數和 ALIGN 指示詞) 。
-   只會將每個資料行標題的第一個單字大寫。

**工具 欄**

以圖形方式呈現優化的命令以有效率地存取。

**提示**

一個小型的快顯視窗，它會標記所指向的未標記控制項，例如未標記的工具列控制項或命令按鈕。

**觸摸**

使用手指直接與電腦顯示互動。

## <a name="u"></a>U

**可用性研究**

一種研究技術，可協助您藉由測試 UI 設計以及從真實目標使用者收集意見反應，來改善您的使用者體驗。 可用性研究可從像是可用性實驗室等設定的正式技術，到使用者自己的辦公室等設定中的非正式技巧。 但這類研究的常數如下：從參與者捕捉資訊;評估該資訊以取得有意義的趨勢和模式;最後，執行的邏輯變更可解決研究中所識別的問題。

**使用者帳戶控制**

使用「使用者帳戶控制 (或 UAC （先前稱為「最低許可權使用者帳戶」或「LUA) 啟用）時，互動式系統管理員通常會以最少的使用者權限執行，但他們可以自行提高許可權，以執行系統管理工作，方法是授與同意 UI 的明確同意。 這類系統管理工作包括安裝軟體和驅動程式、變更全系統設定、查看或變更其他使用者帳戶，以及執行系統管理工具。

**使用者帳戶控制防護**

用來表示命令或選項需要提高使用者帳戶控制許可權的盾牌圖示。

**使用者輸入問題**

使用者輸入導致的錯誤。 使用者輸入問題通常不重要，因為必須先修正這些問題才能繼續。

**使用者案例**

一組特定情況下的使用者目標、問題或工作的描述。

## <a name="w"></a>W

**warning**

描述可能會在未來發生問題之狀況的訊息。 警告不是錯誤或問題。 在 Windows Vista （含）以後版本中，警告訊息通常會顯示在工作對話方塊中，包括清楚、簡潔的主要指示，而且通常會包含文字之視覺效果增強式的標準警告圖示。

**歡迎頁面**

Wizard 的第一頁，用來說明 wizard 的用途。 歡迎頁面不再建議使用。 使用者在沒有這類頁面的情況下有更有效率的體驗。

**視窗**

於電腦螢幕中，顯示程式與內容的長方形區塊。 您可以移動、調整大小、最小化或關閉視窗;它可能會重迭其他視窗。 停駐子視窗會將它轉換成窗格。 另請參閱： [窗格](#p)。

**Windows 標誌鍵**

具有 Windows 標誌的輔助按鍵。 此金鑰可用於許多 Windows 快捷方式，並保留供 Windows 使用。 例如，按 Windows 標誌鍵會顯示或隱藏 Windows [開始] 功能表。

**框線**

UI 原型，顯示視窗的功能和版面配置，但不是其完成的外觀。 框線只會使用線條區段、控制項和文字，而不使用色彩、複雜的圖形或主題。

**嚮導**

一系列的頁面，可引導使用者進行多步驟、不常執行的工作。 有效的嚮導可減少執行工作所需的知識，相較于替代的 Ui。

**工作區域**

使用者可以執行其工作的螢幕區域，以及儲存程式、檔和其快捷方式。 另請參閱： [desktop](#d)。
