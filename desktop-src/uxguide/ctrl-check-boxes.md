# <a name="check-boxes"></a>核取方塊

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用核取方塊時，使用者會在兩個明顯相反的選擇之間做出決定。 核取方塊標籤表示選取的狀態，而已清除狀態的意義必須與所選狀態的明確相反。 因此， **核取方塊只能用來開啟或關閉選項，或選取或取消選取專案。**

![選取四個核取方塊之一的螢幕擷取畫面 ](images/ctrl-check-boxes-image1.png)

一般的核取方塊群組。

> [!Note]  
> 與 [版面](vis-layout.md) 配置相關的指導方針會在個別的文章中顯示。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **這是用來切換選項或選取或取消選取專案的核取方塊？** 如果不是，請使用其他控制項。
-   **選取和清除的狀態是清楚且明確的相反嗎？** 如果沒有，請使用 [選項按鈕](ctrl-radio-buttons.md) 或 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) ，讓您可以個別標記狀態。
-   **在群組中使用時，群組是否會包含獨立的選項，讓使用者選擇零或更多？** 如果沒有，請考慮相依選項的控制項，例如選項按鈕和 [核取方塊樹狀檢視](ctrl-tree-views.md)。
-   **在群組中使用時，群組是否會組成相依選項，使用者必須從中選擇一或多個專案？** 如果是的話，請在未選取任何選項時，使用一組核取方塊並處理錯誤。
-   **群組中的選項數目是否為10或更少？** 由於使用的螢幕空間與選項數目成正比，因此請將核取方塊的數目維持在10個以下。 如果有10個以上的選項，請使用 [核取方塊清單](ctrl-list-boxes.md)。
-   **選項按鈕是否是較佳的選擇？** 如果核取方塊僅適用于開啟或關閉選項，選項按鈕可以用於完全不同的選項。 如果這兩種解決方案都可行：
    -   如果已清除核取方塊的意義不完全明顯，請使用選項按鈕。

        **不正確：**

        ![一個標示為 [橫向] 核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image2.png)

        在此範例中，來自橫向的相反選擇並不清楚，因此核取方塊不是理想的選擇。

        **正確：**

        ![兩個選項按鈕的螢幕擷取畫面 ](images/ctrl-check-boxes-image3.png)

        在此範例中，不會相反選項，因此選項按鈕是較佳的選擇。

    -   在 [wizard] 頁面上使用選項按鈕，讓替代方案更清楚，即使核取方塊是可接受的。
    -   如果您有足夠的螢幕空間，而且選項很重要，足以充分利用該螢幕空間，請使用選項按鈕。 否則，請使用核取方塊或下拉式清單。

        **不正確：**

        ![顯示和不顯示比例按鈕的螢幕擷取畫面 ](images/ctrl-check-boxes-image4.png)

        在此範例中，選項的重要性不足以使用選項按鈕。

        **正確：**

        ![[不顯示訊息] 核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image5.png)

        在此範例中，核取方塊是此周邊選項的螢幕空間有效率的使用方式。

-   如果視窗有其他核取方塊，請使用核取方塊。
-   **選項是否會顯示程式選項，而不是資料？** 選項的值不應以內容或其他資料為基礎。 針對資料，請使用核取方塊清單或 [多重選取清單](ctrl-list-boxes.md)。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **群組相關的核取方塊**。 如有必要，請使用多個群組，將相關選項和不相關的選項分開至10個或更少的群組。

    ![相關且不相關核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image10.png)

    相關、獨立選項的群組範例。

-   重新 **考慮使用群組方塊來組織核取方塊的群組，** 這通常會造成不必要的螢幕雜亂。
-   **以邏輯順序列出核取方塊**，例如將高度相關的選項群組在一起，或先放置最常見的選項，或遵循一些其他自然進展。 不建議依字母順序排序，因為它是語言相依，因此不可當地語系化。
-   **垂直對齊核取方塊，而不是水準對齊**。 水準對齊較難閱讀。

    **正確：**

    ![垂直對齊之核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image11.png)

    在此範例中，核取方塊已正確對齊。

    **不正確：**

    ![水準對齊之核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image12.png)

    在此範例中，水準對齊較難閱讀。

-   **請勿使用混合狀態來表示第三個狀態。** 混合狀態用來指出某個選項是針對部分（而非全部）子物件所設定。 使用者應該無法直接設定混合狀態，而是混合狀態是子物件的反映。 混合狀態不會當做個別專案的第三個狀態使用。 若要表示第三個狀態，請改用選項按鈕或下拉式清單。

    **不正確：**

    ![純色藍色主題服務核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image13.png)

    在此範例中，混合狀態應該指出未安裝主題服務。

    **正確：**

    ![包含三個選項之下拉式清單的螢幕擷取畫面 ](images/ctrl-check-boxes-image14.png)

    在此範例中，使用者可以從三個清楚的選項清單中選擇。

-   **按一下 [混合狀態] 核取方塊，應該會迴圈顯示所有選取的、全部清除，以及原始的混合狀態。** 針對容許，請務必要能夠還原原始混合狀態，因為這些設定對使用者而言可能很複雜或未知。 否則，若要放心還原混合狀態，唯一的方法就是取消工作並重新開始。
-   **請勿使用核取方塊作為進度指標**。 請改用 [進度指標](progress-bars.md) 控制項。

    **不正確：**

    ![顯示進度的四個核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image15.png)

    在此範例中，會使用不正確的核取方塊作為進度指標。

    **正確：**

    ![部分填滿進度列的螢幕擷取畫面 ](images/ctrl-check-boxes-image16.png)

    一般進度列的範例。

-   **使用正確的選取狀態顯示已停用的核取方塊。** 雖然使用者無法變更它們，但停用的核取方塊會傳達資訊，以便與結果一致。

    **不正確：**

    ![將兩個核取方塊的螢幕擷取畫面變成灰色 ](images/ctrl-check-boxes-image17.png)

    在此範例中，應該清除 [永遠閱讀此區段] 選項，因為停用選項時不會讀取區段。

-   **請勿使用核取方塊的選取專案來**：
    -   執行命令。
    -   顯示其他視窗，例如用來收集更多輸入的對話方塊。
    -   以動態方式顯示與所選取控制項相關的其他控制項 (畫面讀取器無法偵測到這類事件) 。

### <a name="dont-show-this-ltitemgt-again"></a>不要再顯示此 &lt; 專案 &gt;

-   **&lt;如果沒有更好的替代方案，請考慮使用 [不要再顯示這個專案 &gt; ] 選項來允許使用者隱藏週期性對話方塊。** 如果使用者真的需要此對話方塊，請嘗試事先判斷：如果有，請一律顯示對話方塊，如果沒有，則刪除對話方塊。

如需詳細的指導方針和範例，請參閱 [對話方塊](win-dialog-box.md)。

### <a name="subordinate-controls"></a>次級控制項

-   將從屬控制項放在 (縮排的右方或下方，並將核取方塊標籤) 核取方塊和其標籤。 以冒號結束核取方塊標籤。

    ![下方核取方塊標籤的螢幕擷取畫面 ](images/ctrl-check-boxes-image18.png)

    在此範例中，核取方塊和其從屬控制項共用核取方塊標籤及其存取金鑰。

-   **如果相依的可編輯文字方塊和下拉式清單共用核取方塊的標籤，請將它保持啟用狀態。** 當使用者在方塊中輸入或貼上任何文字時，請自動選取對應的選項。 這麼做可簡化互動。

    ![頁首和頁尾文字方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image19.png)

    在此範例中，輸入頁首或頁尾會自動選取此選項。

-   如果您使用選項按鈕或其他核取方塊來嵌套核取方塊，請 **停用這些次級控制項，直到選取高階選項為止**。 這樣做可避免與從屬控制項的意義混淆。
-   使用定位順序中的核取方塊，將次級控制項設為連續的核取方塊。
-   **如果選取選項表示選取從屬核取方塊，請明確地選取這些核取方塊以使關聯性清楚。**

    **不正確：**

    ![螢幕擷取畫面：選取的按鈕、已清除的核取方塊 ](images/ctrl-check-boxes-image20.png)

    在此範例中，不會選取附屬核取方塊。

    **正確：**

    ![螢幕擷取畫面：選取的按鈕，選取的核取方塊 ](images/ctrl-check-boxes-image21.png)

    在此範例中，會選取附屬核取方塊，使其與所選選項的關聯性為 clear。

-   **如果替代專案增加不必要的複雜性，請使用相依的核取方塊**。 雖然核取方塊應該是獨立的選項，有時其他的選項（例如選項按鈕）會增加不必要的複雜度。

    **正確：**

    ![令人困惑的按鈕和核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image22.png)

    在此範例中，使用選項按鈕是正確的，但會產生不必要的複雜性。

    **較佳：**

    ![僅限核取方塊的螢幕擷取畫面 ](images/ctrl-check-boxes-image23.png)

    在此範例中，使用核取方塊比較簡單，而且可讓使用者專注于選取所需的選項，而不是其複雜的關聯性。

    **重要事項：只有在極少數的情況下，才會在極罕見的情況下套用此指導** 方針。 在上述範例中，使用者不太可能會嘗試選擇上標和注標，如果有的話，就很容易瞭解它們是專屬的選項。

### <a name="default-values"></a>預設值

-   如果核取方塊適用于使用者選項，請 **設定最安全的 (，以防止資料或系統存取) 、最安全和私用狀態的遺失。** 如果安全性和安全性不是因素，請選取最有可能或方便的值。

## <a name="labels"></a>標籤

**核取方塊標籤**

-   標記每個核取方塊。
-   將唯一的 [存取金鑰](glossary.md) 指派給每個標籤。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
-   使用 [句子樣式的大小寫](glossary.md)。
-   以片語或命令式句子的形式寫入標籤，並不使用結束標點符號。
    -   **例外狀況：** 如果核取方塊標籤也會標示其後的次級控制項，請以冒號結束標籤。
-   寫入標籤，使其描述核取方塊的選取狀態。
-   針對一組核取方塊，請使用平行片語，並嘗試保留所有標籤的相同長度。
-   針對一組核取方塊，請將標籤文字放在選項之間的差異。 如果所有選項都有相同的簡介文字，請將該文字移至群組標籤。
-   使用正片語。 請勿將標籤加上標籤，以便選取核取方塊，表示不執行動作。

    -   **例外狀況：不要 &lt; &gt; 再顯示此專案** 核取方塊。

    **不正確：**

    ![負標籤 [關閉] 的螢幕擷取畫面](images/ctrl-check-boxes-image25.png)

    在此範例中，選項不會使用正片語。

-   只描述標籤的選項。 將標籤保持簡短，讓您可以輕鬆地在訊息和檔中參考它們。 如果選項需要進一步說明，請使用完整的句子和結束標點符號在 [靜態文字](./glossary.md) 控制項中提供說明。

    > [!Note]
    >
    > 將說明新增至群組中的一個核取方塊，並不表示您必須為群組中的所有核取方塊提供說明。 如果您可以的話，請在標籤中提供相關資訊，並且只在必要時使用說明。 請勿只重新聲明標籤以保持一致性。

     

    ![核取方塊、標籤和描述的螢幕擷取畫面 ](images/ctrl-check-boxes-image26.png)

    在此範例中，核取方塊標籤底下會有其他的解說文字。

-   如果強烈建議選項，請考慮將「 (建議的) 」新增至標籤。 請務必新增至控制項標籤，而不是增補附注。
-   如果您必須使用多行標籤，請將標籤的頂端與核取方塊對齊。
-   請勿使用從屬控制項、它所包含的值，或其單位標籤來建立句子或片語。 這類設計無法當地語系化，因為句子結構會隨著語言而有所不同。

    **不正確：**

    ![包含文字方塊之核取方塊標籤的螢幕擷取畫面 ](images/ctrl-check-boxes-image27.png)

    在此範例中，文字方塊不正確地放在核取方塊標籤內。

**核取方塊群組標籤**

-   使用群組標籤來說明群組的用途，而不是如何進行選取。 假設使用者知道如何使用核取方塊。 例如，不要說「選取下列任何選項」。
-   以冒號結束每個標籤。
-   請勿指派標籤的存取金鑰。 這並不是必要的，而是讓其他存取金鑰更難指派。
-   若要選取一或多個相依選項，請說明標籤上的需求。

    **正確：**

    ![兩個控制項標籤的螢幕擷取畫面：通訊協定 ](images/ctrl-check-boxes-image28.png)

    在此範例中，使用者可能會認為他們只能進行一個選取。

    **較佳：**

    ![標籤的螢幕擷取畫面：選取一或多個通訊協定 ](images/ctrl-check-boxes-image29.png)

    在此範例中，使用者可以選擇是否要進行一個以上的選擇。

## <a name="documentation"></a>文件

參考核取方塊時：

-   使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線或冒號。 包含 [單字] 核取方塊。
-   請將核取方塊稱為核取方塊，而不是 [選項]、[核取方塊] 或 [僅限 box]，因為對於當地語系化人員來說，box 並不明確。
-   若要描述使用者互動，請使用選取和清除。
