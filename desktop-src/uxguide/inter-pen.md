# <a name="pen"></a>手寫筆

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

所有 Microsoft Windows 應用程式都應啟用畫筆。 這樣做會比您想像的更容易。

手寫筆輸入是指 Windows 讓您使用畫筆直接與電腦互動的方式。 畫筆可以用於指標，也可用於筆勢、簡單文字輸入，並以數位筆跡來捕捉自由形式的想法。

用於輸入的畫筆具有順暢、流暢的提示，可支援以筆跡精確地指向、書寫或繪製。 畫筆也可以有選擇性的畫筆按鈕 (用來執行以滑鼠右鍵按一下的) 和橡皮擦 (用來清除筆墨) 。 大部分的畫筆都支援滑鼠停留。

![一般畫筆的圖 ](images/inter-pen-image1.png)

一般畫筆。

當畫筆用於手寫時，使用者的筆觸可以使用手寫辨識來轉換成文字。 您可以像撰寫筆劃一樣地保留筆劃，並在背景中執行辨識，以支援以文字形式進行搜尋和複製。 這類未轉換的筆劃稱為數位筆跡。

![onenote 頁面上的手寫螢幕擷取畫面 ](images/inter-pen-image2.png)

筆墨輸入的範例。

大部分 Windows 程式都已經很容易使用，因為可以使用畫筆而非滑鼠，因此畫筆可以順暢地進行大部分的重要工作和互動，而且程式會回應筆勢。 當程式協助手寫文字輸入時，就會變得更容易手寫。 當程式可以直接處理筆墨時，就會變成啟用筆墨，而不需要將筆刷筆觸轉譯成文字或對等的滑鼠移動。 這可讓使用者以自由流動、高品質的數位筆跡撰寫、繪製和新增批註。 收集筆墨與收集滑鼠事件不同，因為筆墨需要較高的解析度和較高的取樣率，而且也可以增加壓力和傾斜的微妙。 如需建立手寫易記和啟用筆墨之程式的相關資訊，請參閱[使用畫筆](/previous-versions/windows/desktop/ms695501(v=vs.85))[整合筆跡](/previous-versions/windows/desktop/ms700674(v=vs.85))和文字輸入。

在定位畫筆時，資料指標的需求較少，因為秘訣代表本身。 不過，若要以協助作為目標，Windows 提供一個小畫筆游標來指出目前的畫筆位置。 不同于它所取代的滑鼠指標，除非畫筆接近顯示器，否則不需要畫筆游標，因此它會在閒置幾秒鐘後消失，以讓資訊看不見。

大部分的畫筆易記程式都支援手勢。 手勢是在電腦上轉譯為命令，而不是滑鼠移動、書寫或繪製的螢幕上的畫筆快速移動。 要執行的最快速且最簡單的筆勢之一是筆勢。 筆勢是一種簡單的手勢，會產生導覽或編輯命令。 導覽筆觸包括向上拖曳、向下移動、向後移動和向前移動，而編輯筆觸包含複製、貼上、復原及刪除。

除了忙碌指標以外的所有指標都具有單一圖元熱點，可定義指標的確切畫面位置。 作用點決定受互動影響的物件。 物件會定義熱區域，也就是將作用點視為物件上方的區域。 通常，熱區域會與物件的框線一致，但可能會更大，讓互動更容易。

**因為畫筆可以更精確地指向手指，所以如果您的使用者介面適用于觸控，則它也適合用於畫筆。** 因此，本文主要著重于將畫筆支援新增至已針對觸控設計的程式。

**注意：** 與 [滑鼠](inter-mouse.md)、 [協助工具](inter-accessibility.md)和 [觸控](inter-touch.md) 相關的指導方針會顯示在不同的文章中。

## <a name="design-concepts"></a>設計概念

使用畫筆進行輸入具有下列特性：

-   **自然又直覺。** 每個人都知道如何使用畫筆來指向和點擊。 物件互動的設計目的是要與使用者以一致的方式與真實世界中的物件互動。
-   **表達。** 畫筆的筆觸很容易控制，因此撰寫、繪製、繪製、繪製和注釋的方式比使用滑鼠更容易。
-   **更個人。** 就像手寫筆記或簽章比輸入的還多，使用數位手寫筆記或簽章也會更個人化。
-   **較不具侵入性。** 使用畫筆是無訊息的，因此會比輸入或按下更少的內容，尤其是在社交情況（例如會議）中。
-   **可擕式。** 具有畫筆功能的電腦可以更精簡，因為大部分的工作都可以在不使用鍵盤、滑鼠或觸控板的情況下完成。 它可能更有彈性，因為它不需要工作介面。 它可讓您使用電腦的新地點和案例。
-   **直接且吸引人。** 使用畫筆可讓您覺得直接與畫面上的物件互動，而使用滑鼠或觸控板時，一律會要求您將手上的移動與個別的螢幕指標移動互動，而這些移動在比較上是間接的。

**所有 Windows 程式都應該有良好的畫筆體驗。** 使用者應該能夠使用畫筆有效率地執行您的程式最重要的工作。 某些工作（例如輸入或詳細圖元操作）不適合畫筆，但至少應該要有可能。

幸運的是，如果您的程式已經過妥善設計，而且易於接觸，則提供良好的畫筆支援很簡單。 基於這個目的，設計完善的程式：

-   **具有良好的滑鼠支援。** 互動式控制項具有明確、可見的 affordances，且指標意見反應有暫止狀態。 物件具有標準滑鼠互動的標準行為 (單一和按兩下、按一下滑鼠右鍵、拖放) 。 [ [指標] 圖形](inter-mouse.md) 會適當地變更，以指出直接操作的類型。
-   **具有良好的鍵盤支援。** 這項程式可提供標準的快速鍵指派，讓使用者更有效率，特別是可透過手勢產生的導覽和編輯命令。
-   **具有夠大的控制項可供觸控。** 控制項的大小下限為23x23 圖元 (13x13 對話單位為 \[ dlu \]) ，而最常使用的控制項至少是40x40 圖元 (23x22 dlu) 。 為了避免沒有回應的行為，目標端之間應該不會有微小的間距，如此一來，連續的目標就能觸及或至少有5個圖元 (三個 Dlu) 的空間。
-   **可以存取。** 使用 Microsoft Active Accessibility (MSAA) 提供輔助技術 UI 的程式設計存取。 程式會適當地回應主題和系統計量變更。
-   **適用于 120 DPI (每英寸) 的 DPI，** 也就是啟用畫筆的電腦建議預設顯示解析度。
-   **使用通用控制項。** 最常見的控制項是設計來支援良好的畫筆體驗。 必要時，程式會使用設計完善的自訂控制項，以支援簡單的目標和互動式操作。
-   **使用受限制的控制項。** 當設計為方便目標時，限制的控制項（例如清單和滑杆）可以比不受限制的控制項（例如文字方塊）更好，因為它們可減少文字輸入的需要。
-   **提供適當的預設值。** 程式依預設會選取最安全的 (，以防止遺失資料或系統存取) 和最安全的選項。 如果安全性和安全性不是因素，程式就會選取最有可能或方便的選項，以避免不必要的互動。
-   **提供文字自動完成。** 提供最可能或最近輸入值的清單，讓文字輸入更容易。

可惜的是，如果您的程式未妥善設計，則反向也是正確的，因為使用畫筆的使用者可能會特別察覺其缺點。

### <a name="model-for-pen-interaction"></a>畫筆互動模型

如果您不熟悉使用畫筆，最好的方式是透過進行學習。 取得啟用畫筆的電腦，將滑鼠和鍵盤放在旁邊，然後進行通常只使用畫筆來進行的工作。 請務必嘗試啟用筆墨的程式，例如 Windows 筆記本，以及未啟用筆墨的程式。 如果您有 Tablet PC，請試著把它放在不同的位置，例如在您的膝上、平面上的平面上，或在您的手中。 請嘗試在直向和橫向的方向使用它，並將畫筆放在您的左邊以及右邊，以供書寫。

當您使用畫筆進行實驗時，您將會發現：

-   **很難使用小型控制項。** 控制項的大小會大幅影響您有效互動的能力。 10x10 圖元的控制項相當適合畫筆，但較大型的控制項更適合使用。 例如， [微調控制項](ctrl-spin-controls.md) (15x11 圖元) 太小，無法輕易地使用畫筆。
-   **Handedness 是一個因素。** 您有時會看到您可能想要查看或與之互動的事項。 比方說，如果是右手使用者的操作功能表出現在點擊點的右邊，就很難使用，所以如果它們出現在左邊，就比較好。 Windows 可讓使用者在 Tablet PC 設定控制台專案中指出他們的 handedness。
-   **工作位置可協助您。** 雖然您可以將指標移動到具有3英寸滑鼠移動的14英寸螢幕上，但是使用畫筆會要求您將手移滿14英寸。 在不同的目標之間重複移動可能會很繁瑣，因此最好盡可能將工作互動保留在一段離手的範圍內。 快顯功能表很方便，因為它們不需要手上移動。
-   **文字輸入和選取很困難。** 使用畫筆時，冗長的文字輸入特別困難，因此自動完成和可接受的預設文字值可以真正簡化工作。 文字選取也可能很困難，因此當工作不需要精確的游標放置時，工作就會更容易。
-   **靠近顯示器邊緣的小型目標可能很難點一下。** 某些顯示 bezels protrude，有些觸控式螢幕技術在邊緣較不敏感，讓邊緣附近的控制項更難使用。 例如，當視窗最大化時，標題列上的 [最小化]、[最大化/還原] 和 [關閉] 按鈕可能較難使用。

**應用程式手勢**

應用程式也可以定義和處理其他筆勢。 Microsoft 手勢辨識器可辨識超過 [40 的手勢](../tablet/application-gestures-and-semantic-behavior.md)。 若要使用應用程式手勢，您的程式必須定義它可辨識的手勢，然後處理產生的事件。

**回應性與一致性**

**回應能力對於建立直接且吸引人的畫筆體驗而言是不可或缺的。** 若要直接操作，手勢必須立即生效，而物件的接觸點必須在筆勢的整個手勢中保持一致。 任何延遲、不穩定的回應、連絡人遺失或不正確的結果，都可終結直接操作和品質的認知。

**一致性對於建立自然且直覺化的畫筆體驗而言是不可或缺的。** 一旦使用者學習標準手勢之後，就會預期筆勢在所有適用的程式中都具有相同的效果。 為了避免混淆和挫折，絕對不要將非標準意義指派給標準手勢。 相反地，請使用自訂手勢進行程式的唯一互動。

**編輯筆跡和文字**

使用畫筆時，編輯筆跡和文字是最具挑戰性的互動。 使用受限制的控制項、適當的預設值和自動完成功能，可以消除或減少輸入文字的需要。 但是，如果您的程式牽涉到編輯文字或筆跡， **您可以在使用畫筆時，自動將輸入 UI 自動縮放至150%，讓使用者更具生產力。**

例如，電子郵件程式可能會以一般大小顯示 UI，但會將輸入 UI 縮放至150% 來撰寫訊息。

![大型字型中 outlook 郵件的螢幕擷取畫面 ](images/inter-pen-image7.png)

在此範例中，輸入 UI 會縮放為150%。

**如果您只進行四件事 .。。**

1.  1. 讓您的 Windows 程式有良好的畫筆體驗！ 使用者應該能夠使用畫筆有效率地執行您的程式最重要的工作， (至少那些不涉及大量輸入或詳細圖元操作) 的工作。
2.  2. 建議您在最相關的案例中，直接使用筆墨來新增撰寫、繪製和新增批註的支援。
3.  3. 若要建立直接且吸引人的體驗，讓手勢立即生效，請在使用者的手勢下順暢地保持接觸點，並將手勢的效果直接對應至使用者的動作。
4.  4. 若要建立自然且直覺的體驗，請支援適當的標準手勢，並為其指派標準意義。 使用自訂手勢進行程式的唯一互動。

## <a name="guidelines"></a>指導方針

### <a name="control-usage"></a>控制使用方式

-   **偏好使用通用控制項。** 最常見的控制項是設計來支援良好的畫筆體驗。
-   **偏好受限制的控制項。** 盡可能使用受限制的控制項（例如清單和滑杆），而不是不受限制的控制項（例如文字方塊），以減少文字輸入的需要。
-   **提供適當的預設值。** 依預設，請選取最安全的 (，以防止遺失資料或系統存取) 和最安全的選項。 如果安全性和安全性不是因素，請選取最有可能或方便的選項，以避免不必要的互動。
-   **提供文字自動完成。** 提供最可能或最近輸入值的清單，讓文字輸入更容易。
-   **針對使用多重選取的重要工作，如果通常使用標準的多重選取清單，請提供選項來改用核取方塊清單。**
-   **遵循系統計量。** 使用所有大小的系統計量都不會 hardwire 大小。 如有必要，使用者可以變更系統計量或 DPI 來滿足其需求。 不過，將此視為最後的手段，因為使用者通常不應該調整系統設定來讓 UI 可供使用。

![具有一般大小和大調整大小之功能表的螢幕擷取畫面 ](images/inter-pen-image8.png)

在此範例中，功能表高度的系統度量已變更。

### <a name="control-sizing-layout-and-spacing"></a>控制項大小調整、版面配置和間距

-   **針對通用控制項，請使用建議的控制項大小。** 這些大小夠大，足以提供良好的畫筆體驗，但微調控制項 (不能使用畫筆，而是多餘的) 。
-   **選擇將控制項放在最可能使用之位置的版面配置。** 盡可能在社區域內保持工作互動。 避免長距離的移動，特別是針對一般工作和拖曳。
-   **使用建議的間距。** 建議的間距適合使用畫筆。
-   **互動式控制項應該觸及或最好至少有5個圖元 (3 個 Dlu 之間的空間) 。** 這樣做可避免使用者在想要的目標外部時產生混淆。
-   請考慮在控制項群組（例如命令連結、核取方塊和選項按鈕，以及群組之間）**新增超過建議的垂直間距**。 這麼做可讓您更輕鬆地區分。

### <a name="interaction"></a>互動

-   **針對設計為接受手寫的程式，啟用預設筆跡。** 預設筆跡可讓使用者藉由直接開始書寫來輸入筆墨，而不需要點一下、提供命令或採取特殊動作。 這樣做可讓您使用畫筆的最自然體驗。 針對未設計為接受手寫的程式，請在文字方塊中處理畫筆輸入做為選取專案。
-   如果您的程式有需要編輯文字的工作，**可讓使用者縮放內容 UI** 。 使用畫筆時，請考慮自動縮放至150%。
-   **因為手勢是記下的，所以請在程式之間指派一致的意義。** 請勿對具有固定語義的手勢提供不同意義。 請改用適當的程式特定手勢。

### <a name="handedness"></a>Handedness

-   **如果視窗是內容相關的，請一律將它顯示在其啟始物件附近。** 將它放在外，使來源物件不會包含在視窗中。
    -   如果使用滑鼠顯示，可能的話，會將內容視窗位移向下和向右。

        ![內容視窗放置於物件右邊的圖 ](images/inter-pen-image9.png)

        顯示與其啟始物件接近的內容視窗。

    -   如果使用畫筆顯示，可能的話，請將內容視窗放在使用者手中，以免被取代。 針對右手許可權的使用者，向左顯示;否則會顯示在右側。

        ![內容視窗放置於物件左方的圖 ](images/inter-pen-image10.png)

        使用畫筆時，也會顯示內容視窗，使其不會涵蓋在使用者的手中。

-   **開發人員：** 您可以使用 [GetMessageExtraInfo](../tablet/system-events-and-mouse-messages.md) API 來區分滑鼠事件和畫筆事件。 您可以使用[SystemParametersInfo](/windows/win32/api/winuser/nf-winuser-systemparametersinfoa) API 搭配 SPI GETMENUDROPALIGNMENT 來判斷使用者的[handedness](/previous-versions/ms819495(v=msdn.10)) \_ 。

### <a name="forgiveness"></a>寬恕

-   **提供復原命令。** 在理想的情況下，您應該為所有命令提供復原，但您的程式可能會有一些無法復原其效果的命令。
-   **提供良好的暫止意見反應。** 明確指出畫筆何時超過可點擊的目標。 這類意見反應是防止意外操作的絕佳方式。
-   **在實際情況下，請在按下畫筆時提供良好的意見反應，但不要採取動作，直到移動或畫筆為止。** 這麼做可讓使用者在他們進行之前更正錯誤。
-   **只要可行，就能讓使用者輕鬆地更正錯誤。** 如果動作會在畫筆上生效，可讓使用者在畫筆仍關閉時滑動，以修正錯誤。

## <a name="documentation"></a>文件

參考畫筆輸入時：

-   以畫筆的形式參考畫筆手寫筆的輸入裝置。 第一次提及時，請使用 tablet 畫筆。
-   請參閱畫筆側邊的按鈕做為畫筆按鈕，而不是圓筒按鈕。
-   以一般方式參考鍵盤、滑鼠、軌跡、畫筆或手指作為輸入裝置。
-   在記錄使用畫筆的特定程式時，請使用點一下 (並按兩下) ，而不是按一下。 點一下以按下螢幕，然後在按住時間之前提起。 它可能會或可能不會用來產生滑鼠點擊。 針對不包含畫筆的互動，請繼續使用按一下。
