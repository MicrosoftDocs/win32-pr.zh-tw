# <a name="graphic-elements"></a>圖形元素

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

*圖形元素* 會以視覺化方式顯示關聯性、階層和強調。 它們包括背景、橫幅、玻璃、匯總工具、分隔符號、陰影和控點。

![具有陰影等的 windows explorer 螢幕擷取畫面。 ](images/vis-graphic-image1.png)

具有數種圖形元素類型的範例。

圖形元素通常不是互動式的。 不過，分隔符號對於可調整大小的內容和控制碼是互動式的，可顯示互動性的圖形。

**注意：** 與 [群組方塊](ctrl-group-boxes.md)、 [動畫](vis-animations.md)、 [圖示](vis-icons.md)和 [商標](exper-branding.md) 相關的指導方針會在個別的文章中顯示。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

圖形元素是表示關聯性的強大視覺方式，過度使用它們會新增視覺效果，並減少表面上的可用空間。 應謹慎使用。

Microsoft Windows 中的設計趨勢是消除不必要的圖形和線條，以簡化更清楚的外觀。

若要決定是否需要圖形元素，請考慮下列問題：

-   **設計的簡報和通訊是否會在沒有元素的情況下清楚且有效？** 如果是，請將它移除。
-   **您可以使用單獨的版面配置來有效地傳達關聯性嗎？** 若是如此，請改用 [layout](vis-layout.md) 。 您可以將相關的控制項彼此相鄰，並在不相關的控制項之間放置額外的間距。 您也可以使用縮排來顯示階層式關聯性。

![四個圖示版面配置的螢幕擷取畫面 ](images/vis-graphic-image2.png)

在此範例中，只會使用版面配置來顯示控制項關聯性。

-   **通訊是否有效且沒有文字？** 如果沒有，請使用 [群組方塊](ctrl-group-boxes.md)、標示為分隔符號或其他 [標籤](text-ui.md)。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **不要單獨透過圖形元素傳達基本資訊。** 這樣做會為視覺效果不便或受損的使用者提供協助工具問題。

### <a name="graphic-designs"></a>圖形設計

-   **當圖形強化單一簡單構想時，最有效。** 需要考慮解讀的複雜圖形無法正常運作。 Hieroglyphics 最適合用來洞穴繪圖。

    **不正確：**

    ![使用複雜圖形的警告螢幕擷取畫面 ](images/vis-graphic-image14.png)

    在此範例中，來自 Windows XP 不正確複雜圖形會嘗試說明複雜的信任決策。

-   **請勿使用箭號、燕號、按鈕框架，或與互動式控制項相關聯的其他 affordances。** 這麼做會邀請使用者與您的圖形互動。
-   **避免在您的設計中大量純紅色、黃色和綠色。** 為了避免混淆，請保留這些色彩以傳達狀態。 如果您必須針對狀態以外的內容使用這些色彩，請使用靜音，而不是單純的色彩。
-   **使用非文化特性設計。** 某個國家/地區、區域或文化特性中的某些意義可能在另一個國家/地區、區域或文化特性中的意義不同。
-   **避免使用人員、臉部、性別或內文部分，以及宗教、政治和國家（地區）符號。** 這類影像可能無法輕易轉譯或可能具有冒犯性。
-   **當您必須代表人員或使用者時，請以一般方式加以描述;** 避免實際的描述。

### <a name="backgrounds-and-banners"></a>背景和橫幅

-   **若要強調內容，請在淺色背景使用深色文字。** 淺灰色或黃色背景的黑色文字效果很好。

    ![黃色背景上藍色連結文字的螢幕擷取畫面 ](images/vis-graphic-image15.png)

    在此範例中，連結會取得使用者的注意，因為它是在黃色背景上。

-   **若要取消強調內容，請在深色背景上使用淺文字。** 暗灰色或藍色背景上的白色文字效果很好。

    ![綠色背景白色 [說明] 連結的螢幕擷取畫面 ](images/vis-graphic-image16.png)

    在此範例中，深色背景會反強調內容。

-   **如果使用漸層，請確定文字色彩在整個漸層中都有良好的對比。**
-   **一律使用16x16 圖元圖示來吸引橫幅。** 否則橫幅太容易忽略。 如需更多指導方針和範例，請參閱 [標準圖示](vis-std-icons.md)。
-   **請小心使用背景和橫幅。** 雖然背景或橫幅的意圖可能是要強調內容，但通常會有結果與所謂的「橫幅色盲」相反。

### <a name="glass"></a>玻璃

-   **在不使用文字的情況下，于小型區域中策略性地使用半透明的範圍。** 這樣做可以讓某個程式更簡單、更淺、更緊密的外觀，讓區域看起來像是框架的一部分。
-   **請勿在一般視窗背景更吸引人或更容易使用的情況下使用半透明。**

### <a name="separators"></a>分隔符號

-   **針對分隔符號使用垂直和水平線條。** 請確定分隔符號與要分隔的內容之間有足夠的空間。
-   針對可調整大小的內容 (分隔器) 之間的分隔符號，請在滑鼠停留時顯示調整大小指標

![顯示具有調整大小指標之分隔器的螢幕擷取畫面。](images/vis-graphic-image17.png)

![具有調整大小指標的分隔器螢幕擷取畫面 ](images/vis-graphic-image18.png)

在這些範例中，會在滑鼠停留時顯示調整大小指標。

### <a name="shadows"></a>陰影

-   **您只能使用，讓您的程式最重要的內容或物件以視覺化方式呈現在背景上。** 在其他情況下，將陰影視為視覺雜亂。

### <a name="high-dpi-support"></a>高 DPI 支援

-   **支援96和120點（每英寸 (DPI）) 影片模式。** 在啟動時偵測 DPI 模式，並處理 DPI 變更事件。 Windows 已針對96和 120 DPI 優化，並預設使用 96 DPI。
-   **偏好在調整圖形時，提供特別針對96和 120 DPI 呈現的個別點陣圖。** 至少為最重要的可見點陣圖提供96和 120 DPI 版本，以及置中或縮放其他的點陣圖。 這類應用程式會被視為「高 DPI 感知」，並提供比 Windows 自動調整的程式更好的整體視覺體驗。
    -   開發人員：您可以宣告程式的高 DPI 感知 (，並防止自動調整) 在程式的資訊清單中設定 DPI 感知旗標，或在程式初始化期間呼叫 SetProcessDPIAware () API。 您可以使用宏來簡化選取正確的圖形。 針對 Win32 點陣圖，您可以使用 SS \_ CENTERIMAGE 將 REALSIZECONTROL 置中或 ss \_ 以進行調整。
-   以96和 120 DPI 檢查您的程式：
    -   太小或太大的圖形。
    -   正在裁剪、重迭或未適當調整的圖形。
    -    ( "晃動" ) 的不佳延伸圖形。
    -   在圖形背景中裁剪或未調整的文字。

## <a name="text"></a>Text

-   **針對協助工具和當地語系化，請勿在圖形中使用任何文字。** 僅將 [標記](exper-branding.md) 和文字表示為抽象概念，以提供例外狀況。
