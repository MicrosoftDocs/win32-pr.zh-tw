# <a name="command-buttons"></a>命令按鈕

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用命令按鈕時，使用者會起始立即動作。

![[確定] 命令按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image1.png)

一般的命令按鈕。

當使用者按下 Enter 鍵時，就會叫用預設的命令按鈕。 它是由開發人員所指派，但在使用者索引標籤上，任何命令按鈕都會變成預設值。

> [!Note]  
> 與 [版面](vis-layout.md) 配置相關的指導方針會在個別的文章中顯示。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **命令按鈕是否用來起始立即動作？** 如果不是，請使用其他控制項。
-   **連結是否是較佳的選擇？** 如果有下列情況，請使用連結：
    -   動作是流覽至另一個頁面、視窗或說明主題。 **例外** 狀況： Wizard 導覽會使用 [上一頁] 和 [下一個] 命令按鈕。
    -   此命令內嵌在文字主體中。
    -   命令的本質是次要。 也就是說，它與視窗的主要用途沒有關聯。 在此情況下，可以使用輕量的命令按鈕或連結。
    -   此命令是功能表或相關連結群組的一部分。
    -   標籤很長，由五個以上的單字所組成，因此讓命令按鈕具有繁瑣的外觀。
-   **選項按鈕和一般命令按鈕的組合是否是較佳的選擇？** 使用選項按鈕時，通常會使用選項按鈕 ([確定]、[取消) 取代一組特定的命令按鈕（如果下列任一條件成立）：
    -   有五個或更多可能的動作。
    -   使用者必須先查看其他資訊，再進行決策。
    -   使用者需要與選擇互動 (可能會在進行決策之前查看其他資訊) 。
    -   使用者會將選擇視為選項，而不是不同的命令。

        **正確：** ![對話方塊、選項按鈕和文字的螢幕擷取畫面 ](images/ctrl-command-buttons-image2.png)

        在此範例中，選項按鈕會與 [確定] 和 [取消] 按鈕結合，以提供選項的其他相關資訊。

        **不正確：** ![具有命令按鈕之訊息的螢幕擷取畫面 ](images/ctrl-command-buttons-image3.png)

        在此範例中，單獨的命令按鈕讓使用者難以做出明智的決策。

## <a name="design-concepts"></a>設計概念

**使用省略號**

當命令按鈕用於立即動作時，可能需要更多的資訊來執行動作。 **指出需要額外資訊的命令 (包括確認) ，方法是在按鈕標籤的結尾加上省略號**。

![包含省略號之 [列印命令] 按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image4.png)

在此範例中，Print .。。命令會顯示 [列印] 對話方塊，以收集詳細資訊。

![列印命令按鈕、沒有省略號的螢幕擷取畫面 ](images/ctrl-command-buttons-image5.png)

相反地，在此範例中，Print 命令會將檔的單一複本列印到預設印表機，而不需要進一步的使用者互動。

**適當地使用省略號很重要，這表示使用者可以在執行動作之前做出進一步的選擇，或甚至完全取消動作**。 省略號提供的視覺提示可讓使用者探索您的軟體，而不需要擔心。

這並不表示每當動作只在執行動作需要額外的資訊時，才會 **顯示另一個視窗時，您應該使用省略號**。 因此， **隱含動詞的任何命令按鈕都不會採用省略號**，例如使用 [關於]、[Advanced]、[說明] (，或任何其他連結至說明主題) 、選項、屬性或設定的命令。

一般情況下，會在使用者介面中使用省略號來表示不完整性。 顯示其他視窗的命令不完整，它們必須顯示另一個視窗，而且不需要額外的資訊來執行其動作。 這種方法可在橢圓形沒有值的情況下，消除螢幕雜亂。

**注意：** 判斷命令按鈕是否需要省略號時，請不要使用將 [許可權提升](winenv-uac.md) 為因素的需求。 提高許可權不是執行命令 (所需的資訊，而是用於許可權) ，而提高許可權的需求則會以安全性保護盾表示。

**如果您只做一件事 ...** 使用簡潔、明確、簡單易懂的標籤，清楚描述命令按鈕所執行的動作，並在適當的情況下使用省略號。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **如果按一下命令按鈕的結果不是瞬間，則顯示忙碌指標。** 如果沒有意見反應，使用者可能會認為按一下未發生，然後再按一次。
-   如果有相同的命令按鈕出現在多個視窗中，請 **嘗試使用相同的標籤文字和存取金鑰，並在實際情況下，在每個視窗中的大約相同位置中找到它。**
-   **針對具有文字標籤的命令按鈕，請使用最小按鈕寬度和標準命令按鈕高度。** 如需詳細資訊，請參閱 [建議的調整大小和間距](#recommended-sizing-and-spacing)。
-   針對每個視窗， **讓命令按鈕的寬度相同**。 如果不可行，請將命令按鈕的不同寬度數目限制為兩個文字標籤。
-   當另一個控制項與命令按鈕（例如具有 [流覽] 按鈕的文字方塊）互通時，請 **將命令按鈕放在三個位置的其中一個位置來表示關聯性：**
    -   與另一個控制項在頂端對齊的右方。
    -   和其他控制項靠左對齊。
    -   在可交互操作 (的控制項（例如，在兩個互通性清單) 框之間的 [新增] 和 [移除] 按鈕）
-   如果有多個命令按鈕與相同的控制項相交互操作，請 **垂直將它們堆疊到右邊，並以另一個控制項靠上對齊，或是以水準方式將它們置於控制項下靠左對齊。**
-   當命令按鈕隸屬于其他控制項時，請 **使用上述位置並停用從屬命令按鈕，直到選取了上層控制項為止。**
-   **請勿使用文字標籤的窄、短或高度命令按鈕** ，因為它們通常會看起來不專業。 請嘗試使用預設的寬度和高度。

**正確：** ![預設大小 [確定] 按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image18.png)

在此範例中，按鈕大小是標準的，而且看起來是 professional。

**不正確：** ![簡短 [確定] 按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image19.png)

在此範例中，按鈕太小。

**不正確：** ![大型、正方形 [確定] 按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image20.png)

在此範例中，按鈕周圍具有太多空間。

-   **避免在命令按鈕上結合文字標籤和圖形。** 結合文字和圖形通常會增加不必要的視覺效果，而不會改善使用者的理解。 請考慮只在圖形輔助時才合併文字和圖形，例如，當它是命令的標準符號時，或可協助使用者將命令的結果視覺化。 否則，偏好文字，但使用文字或圖形。

在此範例中，箭號圖形可協助使用者視覺化命令的結果。

**正確：** ![具有圖示和文字的網址列螢幕擷取畫面 ](images/ctrl-command-buttons-image22.png)

在此範例中，標準符號會與文字結合以協助理解

**不正確：** ![具有 x 圖示和取消之按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image23.png)

在此範例中，[取消] 圖形不會將任何內容加入至文字。

-   **請勿使用命令按鈕設定狀態**。 改為使用選項按鈕或核取方塊。 命令按鈕只適用于起始動作。

### <a name="split-buttons"></a>分割按鈕

-   **讓最有可能的命令成為預設行為**。 如果有一個以上的可能命令，請選擇不需要額外資訊的命令。
-   **如果最可能的命令是最後一個使用者選取的專案，請將按鈕標籤變更為最後一個選取專案。**
-   **使用功能表中的粗體文字來顯示預設命令**。 這麼做可讓使用者更輕鬆地尋找預設命令，特別是當預設命令為動態時，或分割按鈕使用圖形而非文字標籤時。

### <a name="default-values"></a>預設值

-   在每個對話方塊上包含預設的命令按鈕。 **選取最安全的 (，以防止資料遺失或系統存取) ，而且最安全的命令是預設值**。 如果安全性和安全性不是因素，請選取最有可能或方便的命令。
-   除非有簡單的方法可復原命令，否則 **請勿將破壞性動作設為預設的命令按鈕**。

## <a name="labels"></a>標籤

-   為每個命令按鈕加上標籤。
-   如果按鈕只有圖形標籤，請將其 Name 屬性指派給適當的文字標籤。 這可讓輔助技術產品（例如螢幕讀者）為使用者提供圖形的其他相關資訊。

    ![[向上]、[下移] 和 [複製] 按鈕的螢幕擷取畫面 ](images/ctrl-command-buttons-image25.png)

    此範例顯示圖形按鈕;就內部而言，這些按鈕會標示為 [上一步]，然後再複製。

-   針對簡短的瀏覽按鈕 (標示為「...」 ) ，內部標籤應該是「流覽」。
-   指派唯一的 [存取金鑰](glossary.md)。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。

    **異常：**

    -   請勿將存取金鑰指派給 [確定] 和 [取消] 按鈕，因為 Enter 是預設按鈕的存取金鑰 (這通常是 [確定] 按鈕) ，而 Esc 則是 [取消] 的存取金鑰。 這麼做可讓您更輕鬆地指派其他存取金鑰。
    -   請勿將存取金鑰指派給簡短瀏覽按鈕 (標示為「...」 ) ，因為它們無法以獨特方式指派。

-   偏好泛型標籤上的特定標籤。 在理想的情況下，使用者不需要讀取任何其他人來瞭解標籤。 使用者比靜態文字更可能讀取命令按鈕標籤。
