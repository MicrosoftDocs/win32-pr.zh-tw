# <a name="color"></a>Color

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

Color 是大部分使用者介面的重要視覺元素。 除了純美學之外，color 具有相關聯的意義和 elicits 情緒回應。 為了避免混淆，必須一致地使用色彩。 若要取得所需的情緒回應，必須適當地使用色彩。

色彩通常是以色彩空間的角度來思考，其中 RGB (紅色、綠色、藍色) 、HSL (色調、飽和度、亮度) 和 HSV (色調、飽和度、值) 是最常使用的色彩空間。

![顯示色彩關聯性之 cube 的圖 ](images/vis-color-image1.png)

RGB 色彩空間可哥視化為 cube。

雖然顯示器技術會使用 RGB 值，因此開發人員通常會在 RGB 方面考慮色彩，但 RGB 色彩空間不會對應到人們觀察色彩的方式。 例如，如果您將紅色新增至深青，則結果不會被視為紅色但較淺的青色。

![深色和淺青方塊的圖例 ](images/vis-color-image2.png)

在此範例中，新增紅色到深青會讓它變得更淺，而不是紅色。 RGB 色彩空間不會與人們觀察色彩的方式相對應。

HSL/HSV 色彩空間是由三個元件所組成：色調、飽和度和亮度或值。 這些色彩空間通常是用來取代 RGB，因為它們比較符合人們觀察色彩的方式。

HSL 色彩空間會形成一個雙錐形，也就是在頂端的白色、最底部的黑色，以及中間的中性：

-   **色調：** 色輪中的基本色彩，範圍從0到360度，其中0和360度都是紅色。

    ![顯示色彩關聯性的圓形圖 ](images/vis-color-image3.png)

    色彩滾輪（紅色為0度）、黃色為60度、綠色為120度、青色為180度、藍色為240度，而紅色為300度。

-   **飽和度：** 純色的純 (與單調) 的色彩是，範圍從0到100，其中100是完全飽和，0是灰色。
-   **亮度：** 色彩的亮度（範圍從0到100），其中100的 (白色，不論色調和飽和度) ，0的亮度越深 (黑色) 。

    ![說明 hsl 色彩空間的圖表 ](images/vis-color-image4.png)

    HSL 色彩空間可哥視化為雙錐形。

HSV 色彩空間很類似，不同之處在于其空間形成單一錐形：

-   **色調：** 色輪中的基本色彩，範圍從0到360度，其中0和360度都是紅色。
-   **飽和度：** 純色的純 (與單調) 的色彩是，範圍從0到100，其中100是完全飽和，0是灰色。
-   **值：** 色彩的明暗程度（範圍從0到100），100越亮越好 (這在 HSL 空間中是半亮度) 而0越深 (黑色) 。

    ![說明 hsv 色彩空間的圖表 ](images/vis-color-image5.png)

    HSV 色彩空間可哥視化為單一錐形。

在 HSL 和 HSV 空間中，如果飽和度為0，則亮度會指定灰色的陰影。 在 Windows 中，HSL 和 HSV 空間通常會重新對應到0到240之間的小數位數，因此可以使用32位值來表示色彩。

**注意：**[與字型和](vis-fonts.md)[協助工具](inter-accessibility.md)相關的指導方針會顯示在不同的文章中。

## <a name="design-concepts"></a>設計概念

色彩的有效使用可讓您的程式使用者介面 (UI) 更有效率。 色彩可協助使用者一眼就瞭解某些意義。 色彩也可以讓您的產品更容易以賞心悅目，而且更精簡。

可惜的是，使用色彩無效很簡單，特別是當您未在視覺化設計中進行定型時。 使用色彩結果不佳的設計外觀不專業、日期、混淆或單純不美觀。 色彩不佳的用法可能會比完全不使用色彩更糟。

本節說明有效使用色彩的須知事項。

### <a name="how-color-is-used"></a>使用色彩的方式

色彩通常是在 UI 中用來進行通訊：

-   **意義。** 您可以透過色彩來摘要訊息的意義。 例如，色彩通常用來傳達紅色有問題或錯誤的狀態、黃色表示警告或警告，綠色表示良好。
-   **狀態。** 物件的狀態可透過色彩表示。 例如，Windows 使用 color 來表示選取範圍和暫留狀態。 網頁中的連結會使用藍色進行未造訪，並使用紫色進行流覽。
-   **分化。** 人們假設相同色彩的專案之間有關聯性，因此色彩編碼是區分物件的有效方式。 例如，在 [控制台] 專案中，工作窗格會使用綠色背景，以視覺方式將它們與主要內容分開。 此外，Microsoft Outlook 可讓使用者將不同的彩色旗標指派給訊息。
-   **重點。** 色彩可以用來繪製使用者的注意力。 例如，Windows 使用藍色的[主要指示](text-ui.md)來協助它們從其他文字中脫穎而出。

當然，圖形中通常只會使用色彩來進行純粹的美觀原因。 雖然美學很重要，但您應該根據它們所代表的意義來選擇 UI 元素的色彩，而不是它們的外觀。

### <a name="color-interpretation"></a>色彩解讀

**使用者的色彩解釋通常會相依于文化特性。** 例如，在美國，新娘的婚禮服裝主要與白色色彩相關聯，而黑色則與 funerals 相關聯。 不過，在日本，色彩符號正好相反：白色是在 funerals 的主要色彩，而黑色則被視為很幸運婚禮的色彩。

話雖如此， **紅色、黃色和綠色對於狀態的解讀是全域一致的。** 這是因為 [UNESCO 維也納慣例的道路正負號和信號](https://www.unece.org/trans/conventn/signalse.pdf)，會定義交通燈的全球慣例 (其中 red 表示 stop、綠表示繼續，而黃色表示繼續) 的警告。 您可以使用這些狀態色彩，而不需要考慮與文化特性相關的解釋。

除了狀態色彩，Windows 會根據慣例將意義指派給色彩，如本文的指導方針一節所示。 確定您的程式色彩使用方式與這些色彩慣例相容。

### <a name="use-color-to-reinforce-visually"></a>使用色彩以視覺化方式強化

色彩解讀和協助工具問題的最佳解決方法是使用色彩，以視覺方式強調其中一個主要通訊方法的意義：

-   **Text**。 簡潔的文字通常是直接在 UI 上或透過工具提示最有效的主要通訊。

![具有工具提示之小紅色圖示的螢幕擷取畫面 ](images/vis-color-image10.png)

在此範例中，會使用工具提示文字來傳達圖示的意義。

-   **設計。** 這些圖示很容易由設計來區分，尤其是其大綱圖形。

![灰色陰影 (灰階) 的圖示螢幕擷取畫面 ](images/vis-color-image11.png)

在此範例中，標準圖示可以根據其設計立即進行辨別。

-   **位置。** 也可以使用相對位置，但這種方法比替代方案更弱。 為了有效，此位置應該是標準且知名的，如同流量燈。

雖然 color 是許多設計中最明顯的屬性，但它必須永遠是多餘的。

### <a name="designing-with-color"></a>使用色彩設計

諷刺是，設計色彩的最佳方式是從使用 [框線](glossary.md) 或單色設計沒有色彩的方式開始，然後稍後再加入色彩。 這樣做有助於確保資訊不會單獨使用色彩進行通訊。 它也有助於確保您的列印功能在單色印表機上看起來很棒。

### <a name="use-theme-or-system-colors"></a>使用主題或系統色彩

雖然在使用色彩時有許多複雜的因素，但在 Windows 的 UI 中，選擇 [色彩] 通常會根據一些簡單的規則來選擇適當的[主題色彩](glossary.md)或[系統色彩](glossary.md)。 然後，使用者可以選擇並自訂這些色彩配置。

如此一來，您不僅可以容納所有使用者的色彩喜好設定，還可以消除選擇一個適用于所有 tastes、樣式和文化特性之完美色彩配置的負擔 (當然，這種情況下無法) 。

**如果您只做一件事 .。。**

選取適當的主題色彩或系統色彩來選擇色彩。 絕對不要使用色彩做為通訊的主要方法，而是以視覺化方式強調意義的次要方法。 使用框線或單色進行設計，以協助確保色彩為次要。

### <a name="use-theme-or-system-colors-correctly"></a>使用正確的主題或系統色彩

假設使用者根據自己的個人需求選擇主題或系統色彩，並適當地建造主題或系統色彩。 根據這項假設，如果您一律根據其預定用途和與其相關聯的背景來選擇 [主題] 或 [系統色彩]，則可保證在所有的影片模式中都能清楚地顯示色彩，並尊重使用者的願望，包括 [高對比模式](glossary.md)。 例如，視窗文字系統色彩保證可以針對視窗背景系統色彩進行可讀性。

具體而言，一律：

-   **根據其用途來選擇色彩。** 請勿根據其目前的外觀來選擇色彩，因為使用者或未來版本的 Windows 可能會變更外觀。
-   **比對前景色彩與其相關聯的背景色彩。** 前景色彩保證只能針對與其相關聯的背景色彩進行清楚的色彩。 請勿混搭前景色彩，並將其與其他背景色彩比對，或其他前景色彩更糟。
-   **不要混合色彩類型。** 也就是說，一律比對主題色彩與其相關聯的主題色彩、系統色彩與其相關聯的系統色彩，以及使用其他寫死色彩寫死色彩。 例如，不保證會針對寫死背景，提供主題文字色彩的可讀性。
-   **如果您必須 hardwire 色彩，請以特殊案例處理高對比模式。**

**如果您只做一件事 .。。**

一律根據其預定用途選擇主題或系統色彩，並將 foregrounds 與其相關聯的背景配對。

### <a name="using-other-colors"></a>使用其他色彩

雖然 Windows 主題會定義一組完整的主題元件，但您可能會發現您的程式需要在主題檔案中未定義的色彩。 雖然您可以 hardwire 這類色彩，但更好的方法是從主題或系統色彩衍生色彩。 策略性地使用這種方法，可提供您使用主題和系統色彩的所有優點，但有更大的彈性。

例如，假設您需要比主題視窗背景色彩更深色的視窗背景。 在 HSL 色彩空間中，具有較深的色彩表示亮度較低的色彩。 因此，您可以使用下列步驟來衍生較深的視窗背景色彩：

-   取得視窗背景主題色彩 RGB。
-   將 RGB 轉換成其 HSL 值。
-    (除以 20%) ，減少亮度值。
-   轉換回 RGB 值。

使用這種方法時，衍生的色彩一定會被視為原始色彩的較深陰影 (除非原始色彩的開頭非常深。 ) 

![顯示降低亮度之效果的圖例 ](images/vis-color-image12.png)

在此範例中，較深色的視窗背景色彩衍生自主題色彩。

### <a name="testing-colors"></a>測試色彩

若要判斷您的程式使用色彩是否可供存取，而不是用來做為主要的通訊方法，建議您使用 [Fujitsu ColorDoctor](https://www.fujitsu.com/global/about/businesspolicy/tech/design/) 或 [Vischeck](https://www.vischeck.com/) 公用程式來檢查：

-   使用灰色尺規篩選的整體色彩相依性。
-   使用 Protanopia、Deuteranopia 和 Tritanopia 篩選器的特定色彩混淆問題。

若要判斷您的程式使用色彩是否正確地進行程式設計，請以下列模式測試您的程式：

-   使用預設 Windows 主題來啟用主題。
-   使用非預設主題來啟用主題。
-   已停用的主題 ( 「個人化主控台專案) 主題設定中的「Windows 傳統樣式」。
-   高對比黑色背景) 上的黑色 (白色文字。
-   高對比白色背景) 的白色 (黑色文字。

所有的螢幕元素應該都能清晰清晰，並如預期般出現，即使在模式變更後立即顯示。

### <a name="using-color-with-data"></a>使用色彩搭配資料

-   當有説明時，請 **將色彩指派給資料，以協助使用者區分。** 請注意，使用者會假設具有類似色彩的資料具有類似的意義。
-   **依預設，指派易於區別的色彩。** 一般來說，如果在 HSL/HSV 色彩空間中彼此不同，色彩很容易區分，同時維持高對比與其背景：
    -   選擇色彩時，偏好使用三角理論 harmonies 或互補色調，而不是連續的色調。

        ![顯示如何選擇對比色彩的圖 ](images/vis-color-image14.png)

        在此範例中，如果第一個色彩指派為紅色，則下一個色彩應該是藍色、綠色或青色，但不能是洋紅、紫色、橙色或黃色。

    -   如果色調、飽和度或亮度有很大的差異，色彩會有高對比。

        ![顯示不同背景上的一種色彩的圖表 ](images/vis-color-image15.png)

        在此範例中，淺藍色基底色彩會與背景有很大的色調、飽和度或亮度有很大的差異。

    -   使用白色或非常淺的背景，可讓對比前景色彩更容易區別。

        ![說明良好且不良對比度的圖 ](images/vis-color-image16.png)

        在此範例中，白色和淺色背景色彩讓前景色彩更容易區分。

-   **允許使用者自訂這些色彩指派** ，因為色彩選擇是主觀和個人喜好設定。 如果有許多協調色彩，可讓使用者使用色彩配置將它們變更為群組。
-   **允許使用者標記這些色彩指派。** 這麼做可讓您更輕鬆地識別和尋找。
-   **不同于 UI 色彩，當系統色彩變更時，資料不應該變更。**
