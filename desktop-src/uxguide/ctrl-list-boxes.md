# <a name="list-boxes"></a>清單方塊

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用清單方塊時，使用者可以從一律可見清單中顯示的一組值中進行選取。 使用單一選取清單方塊時，使用者會從互斥值清單中選取一個專案。 使用多重選取清單方塊時，使用者會從值清單中選取零個以上的專案。

![單一選取清單方塊的螢幕擷取畫面 ](images/ctrl-list-boxes-image1.png)

典型的單一選取清單方塊。

> [!Note]  
> 與 [版面](vis-layout.md) 配置和 [清單視圖](ctrl-list-views.md) 相關的指導方針會顯示在不同的文章中。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **清單是否會顯示資料，而不是程式選項？** 無論是哪一種方式，清單方塊都是合適的選擇，無論專案數為何。 相較之下， [選項按鈕](ctrl-radio-buttons.md) 或 [核取方塊](ctrl-check-boxes.md) 僅適用于少數的程式選項。
-   **使用者是否需要變更視圖、群組、依資料行排序，或變更資料行的寬度和順序？** 若是如此，請改用 [清單視圖](ctrl-list-views.md) 。
-   **控制項必須是拖曳來源或放置目標？** 若是如此，請改用清單視圖。
-   **是否需要將清單專案複製到剪貼簿或從剪貼簿貼上？** 若是如此，請改用清單視圖。

**單一選取清單**

-   **控制項是否可用來從互斥值清單中選擇一個選項？** 如果不是，請使用其他控制項。 若要選擇多個選項，請改為使用標準的多重選取清單、核取方塊清單、清單產生器或新增/移除清單。
-   **大部分的使用者在大多數情況下，是否有建議使用的預設選項？** 看到選取的選項遠比看到替代方案更重要嗎？ 如果是的話，請考慮使用下拉式清單，如果您不想要讓使用者藉由隱藏替代方案來進行變更，請考慮使用 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) 。

![最高品質作為預設按鈕的螢幕擷取畫面 ](images/ctrl-list-boxes-image2.png)

在此範例中，最高的色彩品質是大部分使用者的最佳選擇，因此下拉式清單是把替代方案的理想選擇。

-   **清單需要持續互動嗎？** 若是如此，請使用單一選取清單來簡化互動。

![選項清單的螢幕擷取畫面，例如純文字 ](images/ctrl-list-boxes-image3.png)

在此範例中，使用者會不斷地變更 [顯示專案] 清單中選取的專案，以設定前景和背景色彩。 在此情況下，使用下拉式清單相當繁瑣。

-   **設定看起來像相對數量嗎？使用者是否能從立即的意見反應中獲益** 于設定變更的影響？ 如果是的話，請考慮改用 [滑杆](ctrl-sliders.md) 。
-   **清單專案之間是否有重要的階層式關聯性？** 如果是的話，請改用 [樹狀檢視](ctrl-tree-views.md) 控制項。
-   **螢幕空間是否在 premium？** 如果是的話，請改用下拉式清單，因為使用的螢幕空間是固定的，而且與清單專案的數目無關。

**標準多重選取清單和核取方塊清單**

-   **這項工作或通常使用的是多重選取專案嗎？** 如果是的話，請使用核取方塊清單讓多個選取明顯，尤其是當您的目標使用者不是 advanced。 許多使用者都不知道標準的多重選取清單支援多重選取。 如果核取方塊會對多個選取專案造成太多的注意，或造成太多螢幕雜亂，請使用標準的多重選取清單。
-   **多重選取專案的穩定性是否重要？** 如果是的話，請使用核取方塊清單、清單產生器或新增/移除清單，因為按一下一次只變更一個專案。 使用標準的多重選取清單時，很容易就能清除所有選取專案，即使是意外的。
-   **控制項是否可用來從值清單中選擇零個以上的專案？** 如果不是，請使用其他控制項。 若要選擇一個專案，請改為使用單一選取清單。

**預覽清單**

-   **使用影像選取的選項是否比單獨使用文字更容易？** 若是如此，請使用預覽清單。

**列出產生器和新增/移除清單**

-   **控制項是否可用來從值清單中選擇零個以上的專案？** 如果不是，請使用其他控制項。 若要選擇一個專案，請改為使用單一選取清單。
-   **選取專案的順序是否重要？** 如果是，則清單產生器和新增/移除清單模式支援順序，而其他多重選取模式則不支援。
-   **使用者是否要查看所有選取專案的摘要？** 如果是，清單產生器和新增/移除清單模式只會顯示選取的專案，而其他多重選取模式則不會顯示。
-   **可能的選擇是否受限制？** 如果是，請使用 [新增/移除] 清單，讓使用者可以選擇目前不在清單中的值。
-   **將值新增至清單時需要特殊的對話方塊來選擇物件嗎？** 如果是，請使用 [新增/移除] 清單，並在使用者按一下 [新增] 時顯示對話方塊。
-   **螢幕空間是否在 premium？** 若是如此，請改用 [新增/移除] 清單，因為它不一定會顯示一組選項，因此使用較少的螢幕空間。

針對清單方塊， **清單中的專案數目不是選擇控制項的因素** ，因為它們會從上千個專案一直到一個選取清單 (，而不是多重選取清單) 。 由於清單方塊可用於資料，因此可能事先不知道專案的數目。

**注意：** 有時候看起來像清單方塊的控制項是使用清單視圖來執行，反之亦然。 在這種情況下，請根據使用方式套用指導方針，而不是執行。

## <a name="guidelines"></a>指導方針

### <a name="presentation"></a>簡報

-   **以邏輯順序排序清單專案，例如將** 相關選項群組在一起，並先放置最常使用的專案，或使用字母順序。 依字母順序排序名稱、依數位順序排序數位和日期（以時間順序排列）。 具有12個以上專案的清單應依字母順序排序，讓專案更容易尋找。

**正確：** ![對齊 (左、置中、右) 的螢幕擷取畫面 ](images/ctrl-list-boxes-image10.png)

在此範例中，清單方塊專案依其空間關聯性排序。

**不正確：** ![雜亂無章清單的螢幕擷取畫面 ](images/ctrl-list-boxes-image11.png)

在此範例中，有很多清單專案應依字母順序排序。

**正確：** ![依字母順序排列清單的螢幕擷取畫面 ](images/ctrl-list-boxes-image12.png)

在此範例中，清單專案比較容易尋找，因為它們會依字母順序排序。 不過，「所有 Windows 產品」專案都是在清單的開頭，而不是它的排序次序。

-   無論其餘專案的排序次序為何，都 **可在清單的開頭放置代表 All 或 None 的選項**。
-   **以括弧括住中繼選項。**

![下拉式清單的螢幕擷取畫面，其中已選取 [無] ](images/ctrl-list-boxes-image13.png)

在此範例中，「 (無) 」是中繼選項，因為它不是有效的選項值，而是表示選項本身未使用。

-   **不要有空白清單專案改用中繼選項。** 使用者不知道如何解讀空白專案，而中繼選項的意義是明確的。

**不正確：** ![已選取空白的下拉式清單螢幕擷取畫面 ](images/ctrl-list-boxes-image14.png)

在此範例中，空白專案的意義不清楚。

**正確：** ![下拉式清單的螢幕擷取畫面，其中已選取 [無] ](images/ctrl-list-boxes-image13.png)

在此範例中，會改為使用「 (無) 」中繼選項。

### <a name="interaction"></a>互動

-   **請考慮提供按兩下行為。** 按兩下應該與選取專案並執行其預設命令的效果相同。
-   **讓按兩下行為重複。** 應該一律會有相同效果的命令按鈕或內容功能表命令。
-   **如果使用者無法對選取的專案進行任何動作，則不允許選取。**

**正確：** ![已完成的 wizard 變更清單螢幕擷取畫面 ](images/ctrl-list-boxes-image15.png)

此清單方塊會顯示變更的唯讀清單;不需要選擇。

-   **停用清單方塊時，也會停用任何相關聯的標籤和命令按鈕。**
-   **請勿在清單方塊中使用所選取專案的變更來：**
    -   執行命令。
    -   顯示其他視窗，例如用來收集更多輸入的對話方塊。
    -   以動態方式顯示與所選取控制項相關的其他控制項 (畫面讀取器無法偵測到這類事件) 。 **例外狀況：** 您可以動態變更用來描述所選取專案的靜態文字。

**可接受：** ![選取清單專案詳細資料的螢幕擷取畫面 ](images/ctrl-list-boxes-image16.png)

在此範例中，變更選取的專案會變更描述。

-   **避免水準滾動。** 多行清單依賴水準滾動，通常比垂直捲動更難使用。 當您有許多字母順序的專案，且有足夠的螢幕空間可供寬控制項使用時，可能會使用需要水準滾動的多行清單。

**可接受：** ![windows explorer 中的資料夾清單螢幕擷取畫面 ](images/ctrl-list-boxes-image17.png)

在此範例中，會使用多個需要水準滾動的資料行，因為有許多專案和大量的可用螢幕空間可供廣泛的控制項使用。

### <a name="multiple-selection-lists"></a>多重選取清單

-   **請考慮在清單下方顯示選取的專案數目，** 特別是當使用者可能選取數個專案時。 這項資訊不僅能提供有用的意見反應，也清楚指出清單方塊支援多重選取。

![已選取四個專案的清單方塊螢幕擷取畫面 ](images/ctrl-list-boxes-image5.png)

在此範例中，選取的專案數目會顯示在清單下方。

-   您可以提供可能更有意義的其他選取範圍計量，例如選取專案所需的資源。

![windows 功能的核取方塊清單螢幕擷取畫面 ](images/ctrl-list-boxes-image18.png)

在此範例中，安裝元件所需的磁碟空間比選取的專案數目更有意義。

-   如果有可能有許多清單專案，並選取或清除所有的專案，請加入 [全選] 和 [全部清除] 命令按鈕。
-   針對標準的多重選取清單，請勿使用多重選取模式，因為此選取模式已被取代。 針對相等的行為，請改用核取方塊清單。

### <a name="default-values"></a>預設值

-   **依預設，請選取最安全的 (，以防止遺失資料或系統存取) 和最安全的選項。** 如果安全性和安全性不是因素，請選取最有可能或方便的選項。

**例外狀況：** 如果控制項代表 [混合狀態](glossary.md)中的屬性，則不選取任何專案，這會在顯示多個沒有相同設定之物件的屬性時發生。

## <a name="recommended-sizing-and-spacing"></a>建議的大小和間距

![清單方塊調整大小和間距的螢幕擷取畫面 ](images/ctrl-list-boxes-image19.png)

清單方塊的建議大小和間距。

-   **選擇適合最長有效資料的清單方塊寬度。** 標準清單方塊無法水準滾動，因此使用者只能看到控制項中顯示的內容。
-   **包含額外的 30%** (高達200% 的文字，以提供任何文字 (較短的文字) ，而不是) 將當地語系化的數位。
-   **選擇顯示整數專案數目的清單方塊高度。** 避免垂直截斷專案。
-   **選擇可消除不必要垂直捲動的清單方塊高度。** 清單方塊應該會顯示3到20個專案，而不需要滾動。 如果這樣做會消除垂直捲動條，請考慮讓清單方塊稍微長一點。 可能有多個專案的清單應該會顯示至少五個專案，藉由一次顯示更多專案，並讓捲軸更容易放置，來加速滾動。
-   **如果使用者受益于使清單方塊變得更大，請讓清單方塊和其父視窗可調整大小。** 這麼做可讓使用者視需要調整清單方塊的大小。 不過，可調整大小的清單方塊應該不會顯示三個以上的專案。

## <a name="labels"></a>標籤

**控制項標籤**

-   所有清單方塊都需要標籤。 將標籤以單字或片語的形式寫入，而不是以句子撰寫;使用標籤結尾的冒號。

**例外狀況：** 如果標籤只是對話方塊 [主要指示](glossary.md)的重述，請省略標籤。 在此情況下，主要指令會採用冒號 (，除非它是問題) 和存取金鑰。

**可接受：** ![具有重複標籤清單的螢幕擷取畫面 ](images/ctrl-list-boxes-image20.png)

在此範例中，清單方塊標籤只會下方重述主要指令。

**更好：** ![沒有重複標籤清單的螢幕擷取畫面 ](images/ctrl-list-boxes-image21.png)

在此範例中，會移除多餘的標籤，所以主要指令會接受冒號和存取金鑰。

-   如果清單方塊是指向選項按鈕或核取方塊，且該控制項的標籤以冒號結尾，請勿在清單方塊控制項上放置額外的標籤。

![使用相同標籤的按鈕和清單螢幕擷取畫面 ](images/ctrl-list-boxes-image22.png)

在此範例中，清單方塊是一個選項按鈕，並共用其標籤。

-   指派唯一的 [存取金鑰](glossary.md)。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
-   使用 [句子樣式的大小寫](glossary.md)。
-   將標籤置於控制項的左邊或上方，然後將標籤與控制項的左邊緣對齊。
    -   如果標籤位於左邊，則垂直對齊標籤文字與控制項中的第一行文字。

**正確：** ![清單中具有靠左對齊標籤且靠左對齊標籤的清單螢幕擷取畫面 ](images/ctrl-list-boxes-image23.png)![](images/ctrl-list-boxes-image24.png)

在這些範例中，頂端的標籤會與清單方塊的左邊緣對齊，而左邊的標籤會與清單方塊中的文字對齊。

**不正確：** ![左邊對齊標籤的清單螢幕擷取畫面上方清單中的螢幕擷取畫面 ](images/ctrl-list-boxes-image25.png)![](images/ctrl-list-boxes-image26.png)

在這些不正確的範例中，頂端的標籤會與清單方塊中的文字對齊，而左邊的標籤會與清單方塊的頂端對齊。

-   若為多重選取清單方塊，請使用可清楚指出可能有多重選取的標籤。 核取方塊清單標籤可能較不明確。

**正確：** ![[選取一或多個標籤] 清單的螢幕擷取畫面 ](images/ctrl-list-boxes-image27.png)

在此範例中，標籤會清楚指出可能會有多個選取專案。

**不正確：** ![具有附加元件標籤之清單方塊的螢幕擷取畫面 ](images/ctrl-list-boxes-image28.png)

在此範例中，標籤不會提供關於多重選取的明顯資訊。

**最佳：** ![包含附加元件標籤的核取方塊清單螢幕擷取畫面 ](images/ctrl-list-boxes-image29.png)

在此範例中，核取方塊會清楚指出可能有多重選取，因此標籤不需要明確。

-   您可以在標籤後面的括弧中，指定單位 (秒、連接等) 。

**選項文字**

-   將唯一名稱指派給每個選項。
-   除非專案是適當的名詞，否則請使用 [句子樣式的大小寫](glossary.md)。
-   將標籤以單字或片語的形式寫入，而不是以句子形式寫入，而且不使用結尾標點符號。
-   使用平行片語，並嘗試保留所有選項的相同長度。

**教學和補充文字**

-   如果您需要新增有關清單方塊的解說文字，請將它加入標籤上方。 使用結尾標點符號的完整句子。
-   使用 [句子樣式的大小寫](glossary.md)。
-   有説明但不需要的其他資訊應該保持簡短。 將此文字放在標籤和冒號之間的括弧中，或是在控制項下方的括弧內。

![核取方塊清單和描述性文字的螢幕擷取畫面 ](images/ctrl-list-boxes-image30.png)

在此範例中，補充文字會放置在清單下方。

## <a name="documentation"></a>文件

參考清單方塊時：

-   使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線或冒號。 包含字組清單。 請勿以清單方塊或欄位形式來參考清單方塊。
-   針對清單專案，使用確切的專案文字，包括其大小寫。
-   在程式設計和其他技術檔中，請以清單方塊的形式來參考清單方塊。 在其他地方，使用 list。
-   若要描述使用者互動，請使用 select。
-   可能的話，請使用粗體文字來格式化標籤和清單專案。 否則，請只在必要時才將標籤和專案放在引號中，以避免混淆。

範例：在 [ **移至目標** ] 清單中選取 [ **書簽**]。
