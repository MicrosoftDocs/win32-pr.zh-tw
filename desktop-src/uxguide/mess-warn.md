# <a name="warning-messages"></a>警告訊息

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

警告訊息是強制回應對話方塊、就地訊息、通知或氣球，會警示使用者可能會在未來發生問題的狀況。

![一般警告訊息的螢幕擷取畫面](images/mess-warn-image1.png)

一般的強制回應警告訊息。

警告的基本特性是，它們牽涉到遺失下列一或多項的風險：

-   重要的資產，例如重要的財務或其他資料。
-   系統存取或完整性。
-   隱私權或對機密資訊的控制。
-   使用者的時間 (大量的時間，例如30秒或更多的) 。

相反地，確認是一個強制回應對話方塊，會詢問使用者是否要繼續執行動作。 某些類型的警告會顯示為確認，若是如此，則也適用確認方針。

**注意：** 與 [對話方塊](win-dialog-box.md)、 [確認](mess-confirm.md)、 [錯誤訊息](mess-error.md)[標準圖示](vis-std-icons.md)、 [通知](mess-notif.md)和 [版面](vis-layout.md) 配置相關的指導方針會顯示在不同的文章中。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定使用時機，請考量下列問題：

-   **使用者是否會收到警示，指出未來可能會造成問題的情況？** 如果沒有，訊息就不會是警告。
-   **UI 是否呈現已發生的錯誤或問題？** 若是如此，請改用錯誤訊息。
-   **使用者是否有可能執行動作，或將其行為變更為訊息的結果？** 如果沒有，則條件不會讓使用者中斷，因此最好隱藏警告。
-   **條件是由使用者起始之動作的直接結果嗎？** 如果沒有，請考慮使用 [非重大的事件通知](mess-notif.md)。
-   **條件是否為控制項中的特殊條件？** 如果是的話，請改用 [球形球形](ctrl-balloons.md) 。
-   **針對確認，使用者是否要執行具風險的動作？** 如果是的話，如果動作有重大的後果或無法輕易復原，則會有適當的警告。
-   **針對其他類型的警告，使用者是否需要立即或立即行動？** 如果使用者可以在沒有立即發生問題的情況下持續運作，不顯示警告。 延後警告，直到條件更快速且相關。

## <a name="design-concepts"></a>設計概念

### <a name="avoid-overwarning"></a>避免 overwarning

我們在 Microsoft Windows 程式中 overwarn。 一般的 Windows 程式在所有地方都有警告，有一點重要性的警告。 在某些程式中，幾乎每個問題都會顯示為警告。 Overwarning 讓程式感覺像是有害的活動，而且會使真正的重大問題。

**不正確：**

![不必要警告訊息的螢幕擷取畫面 ](images/mess-warn-image2.png)

Overwarning 讓您的程式感覺危險，並看起來像是由律師所設計。

單純的資料遺失或未來的問題可能就是呼叫警告的可能性不足。 此外，任何非預期的結果都應該是非預期或非預期的結果，而且不容易更正。 否則，您可以將任何使用者錯誤解釋為可能導致資料遺失或某個問題的潛在問題，並提供警告。

### <a name="characteristics-of-good-warnings"></a>良好警告的特性

良好的警告：

-   **牽涉到風險。** 良好的警告會警示使用者有重大問題。

**不正確：**

![[您要結束嗎？] 的螢幕擷取畫面 warning ](images/mess-warn-image3.png)

那又怎樣？ 這項確認會假設使用者通常會意外離開程式。

-   **具有立即相關性。** 使用者不但必須在意，他們現在必須在意。 使用者通常不會有興趣解決這些問題，因為他們現在可以進行工作。

**不正確：**

![電池的螢幕擷取畫面-低-3 小時的警告 ](images/mess-warn-image4.png)

在此情況下，最好是在三個小時內警告使用者。

-   **導致動作。** 有一些使用者必須做的事，或是要注意的是警告的結果。 也許它們必須立即採取動作，或在未來的某個時間進行。 可能會以不同的結果來執行工作。 忽略警告的結果應該是明確的。 沒有動作的警告只會讓使用者感覺擇善固執。

**不正確：**

![[即時 messenger 正在執行] 警告的螢幕擷取畫面 ](images/mess-warn-image5.png)

為什麼此通知會出現警告？ 使用者應該怎麼做 (在擔心) ？

-   **不明顯。** 不要顯示警告來陳述動作的明顯結果。 例如，假設使用者瞭解未完成工作的後果。

**不正確：**

![您要結束 wizard 的螢幕擷取畫面嗎？警告 ](images/mess-warn-image6.png)

取消不完整的 wizard 表示工作未完成 .。。誰知道？

-   **不常發生。** 常數警告很快就會變得無效和討厭。 使用者通常會更專注于消除警告而非解決問題。

**不正確：**

![[更新病毒碼標記] 警告的螢幕擷取畫面 ](images/mess-warn-image7.png)

使用者比較可能會專注于消除警告，而不是修正基礎問題。

沒有這些特性的訊息可能仍然是很好的訊息，而不是好的警告。

### <a name="determine-the-appropriate-message-type"></a>判斷適當的訊息類型

根據強調和措辭，某些問題可能會顯示為錯誤、警告或資訊。 例如，假設網頁無法根據目前的 Windows Internet Explorer 設定載入未簽署的 ActiveX 控制項：

-   **錯誤。** 「此頁面無法載入未簽署的 ActiveX 控制項。」  (片語做為現有的問題。 ) 
-   **警告。** 「此頁面可能無法如預期般運作，因為 Windows Internet Explorer 未設定為載入未簽署的 ActiveX 控制項。」 或「允許此頁面安裝未簽署的 ActiveX 控制項？ 從不受信任的來源執行此動作可能會危害您的電腦。」  (這兩個片語為可能會導致未來問題的狀況。 ) 
-   **資訊。** 「您已設定 Windows Internet Explorer 封鎖未簽署的 ActiveX 控制項。」  (片語作為事實的陳述。 ) 

**若要判斷適當的訊息類型，請將焦點放在使用者需要知道或採取行動之問題的最重要層面。** 一般來說，如果問題封鎖使用者繼續進行，您應該將它顯示為錯誤;如果使用者可以繼續，請將它顯示為警告。 根據焦點製作 [主要指示](text-ui.md) 或其他對應的文字，然後選擇符合文字的圖示 ([標準](vis-std-icons.md) 或) 。 主要的指令文字和圖示應該一律相符。

### <a name="be-specific"></a>特定

當下列資訊是特定且明確的時，警告會更吸引人：

-   警告的來源。
-   特定條件和潛在問題。
-   使用者應怎麼做。
-   如果使用者沒有執行任何動作，會發生什麼事。

**不正確：**

![明顯風險清楚警告的螢幕擷取畫面 ](images/mess-warn-image8.png)

在此範例中，可能發生的問題為何？ 除了無法透過網路使用投影機之外，使用者應做什麼？ 如果沒有更明確的資訊，使用者就無法繼續進行。

**正確：**

![問題和結果警告的螢幕擷取畫面 ](images/mess-warn-image9.png)

在此範例中，問題和後果很清楚。

有時候可能會有一個合法的潛在問題可告知使用者，但不知道解決方案和後果。 您可以提供最有可能的資訊或最常見的範例，而不是提供不明確的警告。

**正確：**

![網路錯誤警告和解決方案的螢幕擷取畫面 ](images/mess-warn-image10.png)

在此範例中，提供最有可能的解決方案來使警告成為特定的。

不過，在這種情況下，請使用表示其他可能性的用語。 否則，使用者可能會誤導。

**不正確：**

![網路纜線已拔掉警告的螢幕擷取畫面 ](images/mess-warn-image11.png)

**正確：**

![纜線的螢幕擷取畫面可能已被拔掉警告 ](images/mess-warn-image12.png)

在不正確的範例中，如果已清楚地插入纜線，使用者將會混淆。

**如果您只進行兩件事 .。。**

1. 不要 overwarn。 將警告限制為牽涉到風險且立即相關、可採取動作、不明顯且不頻繁的狀況。 否則，請移除或改寫訊息。

2. 提供特定且有用的資訊。

## <a name="guidelines"></a>指導方針

### <a name="presentation"></a>簡報

-   **選擇以資訊類型為基礎的展示 UI：**



| 使用者介面  | 最適合用於 |
|-------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| 強制回應對話方塊<br/> | 重大警告 (包括使用者必須立即回應的確認) 。<br/>                                                 |
| 就地<br/>           | 可能會妨礙問題的資訊，特別是當使用者進行選擇時。<br/>                                         |
| 橫幅<br/>            | 可能會妨礙問題的資訊，特別是在完成工作的相關資訊時。<br/>                                     |
| 通知<br/>      | 可放心忽略的重大事件或狀態（至少暫時）。<br/>                                              |
| 汽球<br/>           | 控制項處於影響輸入的狀態。 此狀態可能是非預期的，使用者可能不會察覺到輸入受到影響。<br/> |



 

-   **針對強制回應對話方塊：**
    -   **在適當時使用工作對話方塊，以達成一致的外觀和配置。** 工作對話方塊需要 Windows Vista 或更新版本，因此不適用於舊版 Windows。
    -   **每個條件只顯示一則警告訊息。** 例如，顯示單一警告以完整說明條件，而不是每個訊息一次描述一個詳細資料。 針對單一條件顯示一系列的警告對話方塊會造成混淆和討厭。
    -   **請勿針對每個條件顯示一次以上的警告。** 常數警告很快就會變得無效和討厭。 使用者通常會更專注于消除警告而非解決問題。 如果您必須針對單一條件重複發出警告，請使用 [漸進式擴大](mess-notif.md)。
-   **不要伴隨有音效效果或嗶聲的警告。** 這麼做是 jarring 且不必要的。
    -   **例外狀況：** 如果使用者必須立即回應，您可以使用音效效果。

### <a name="icons"></a>圖示

-   請勿在對話方塊的標題列中放置警告圖示。
-   **使用警告圖示。** 例外狀況：
    -   如果有圖示的功能出現警告，您可以使用功能圖示搭配警告重迭。

        **正確：**

        ![具有警告圖示重迭的鎖定圖示螢幕擷取畫面 ](images/mess-warn-image21.png)

        在此範例中，功能圖示會有警告重迭。

-   針對具有警告註腳的強制回應對話方塊，請將警告圖示放在註腳而非內容區域中。

    **正確：**

    ![對話方塊註腳中警告圖示的螢幕擷取畫面 ](images/mess-warn-image22.png)

    在此範例中，註腳具有警告圖示。

如需更多指導方針和範例，請參閱 [標準圖示](vis-std-icons.md)。

### <a name="dont-show-this-message-again"></a>不要再顯示此訊息

-   **如果警告對話方塊需要此選項，請重新考慮警告和其頻率。** 如果它具有良好警告的所有特性 (牽涉到風險，而且立即相關、可採取動作、不明顯且不頻繁的) ，使用者就不能將它隱藏起來。

如需詳細指導方針，請參閱 [對話方塊](win-dialog-box.md)。

### <a name="progressive-disclosure"></a>漸進式揭露

-   **如果您必須在警告訊息中包含 advanced 資訊，請使用累進洩漏按鈕加以顯示** (例如「顯示詳細資料」 ) 。 這麼做可簡化一般使用方式的警告。 請勿隱藏所需的資訊，因為使用者可能找不到它。
-   **除非真的有更多詳細資料，否則請不要使用 [顯示詳細資料]。** 請勿只以不同的格式重新聲明現有的資訊。

如需標籤指導方針，請參閱 [漸進式洩漏](ctrl-progressive-disclosure-controls.md)。

### <a name="default-values"></a>預設值

-   **選取最安全、最不具破壞性或最安全的回應作為預設值。**

## <a name="text"></a>Text

### <a name="general"></a>一般

-   **移除多餘的文字。** 在標題、主要指示、補充指示、內容區域、命令連結和認可按鈕中尋找。 一般而言，請在指示和互動式控制項中保留完整文字，然後從其他位置移除任何多餘的。
-   **請勿在文字中使用「警告」或「警告」字詞。** 當 [正確使用](vis-std-icons.md)時，警告圖示可充分地傳達使用者必須謹慎地進行通訊。

**不正確：**

![文字中不必要使用警告的螢幕擷取畫面 ](images/mess-warn-image23.png)

在此範例中，「警告」一詞是不必要的。

### <a name="titles"></a>標題

-   **使用標題來識別警告來源的命令或功能。** 例外狀況：
    -   如果有許多不同的命令顯示警告，請考慮改為使用程式名稱。
    -   如果該標題是多餘的，或與主要指示混淆，請改用程式名稱。

**不正確：**

![安全性警告對話方塊標題的螢幕擷取畫面 ](images/mess-warn-image24.png)

在此範例中，「安全性警告」不會識別警告來源的命令或功能。

-   **請勿使用標題來說明在主要指令用途的對話方塊中要執行** 的動作。
-   使用 [標題樣式的大小寫](glossary.md)，但不含結束標點符號。

### <a name="main-instructions"></a>主要指示

-   警告的主要指示是根據其設計模式：



| 模式                        | 主要指示                                               |
|--------------------------------------|----------------------------------------------------------------------|
| 感知<br/>                 | 描述條件或潛在問題。<br/>              |
| 即將發生問題<br/>          | 描述使用者現在需要做什麼。<br/>                   |
| 有風險的動作確認<br/> | 詢問問題，以判斷使用者是否想要繼續進行。<br/> |



 

-   ![電力偏低通知的螢幕擷取畫面 ](images/mess-warn-image25.png)
-   在此範例中，電力偏低通知是感知警告，因此主要指示會說明此狀況。
-   ![立即變更電池警告的螢幕擷取畫面 ](images/mess-warn-image1.png)
-   在此範例中，[低電池] 對話方塊是即將發生的問題，因此主要指示描述使用者現在需要做什麼。
-   **請務必只使用單一的完整句子。** 將主要指示帶到基本資訊。 如果您必須進一步說明，請使用補充指示。
-   **如果使用者必須立即採取行動，請使用「立即」和「立即」等字。** 如果沒有緊急，請不要使用這些字組。
-   **如果有相關的物件，請特別提供其完整名稱。**
-   使用 [句子樣式的大小寫](glossary.md)。

### <a name="supplemental-instructions"></a>補充指示

-   警告的補充指示是根據其設計模式：



| 模式            | 補充指示                                            |
|--------------------------------------|------------------------------------------------------------------------------------|
| 感知<br/>                 | 解釋隱含意義，以及其重要性。<br/>                        |
| 即將發生問題<br/>          | 說明條件以及其重要性。<br/>                          |
| 有風險的動作確認<br/> | 說明使用者可能不想繼續的任何非明顯原因。<br/> |



 

-   **請勿以稍微不同的用語重複主要指令。** 相反地，如果沒有其他要新增的指令，請省略補充指令。
-   使用完整句子、句子樣式的大小寫和結束標點符號。

### <a name="commit-buttons"></a>認可按鈕

-   在警告對話方塊中，認可按鈕是根據其設計模式：



| 模式               | 認可按鈕        |
|--------------------------------------|-----------------------------------------------------------------------------------------------------------------|
| 感知<br/>                 | 很接近了。 請勿使用 [確定]，因為它會建議可能的問題是正常的。<br/>                              |
| 即將發生問題<br/>          | 每個選項的命令按鈕或命令連結，如果動作發生在對話方塊外，則為 [確定]。<br/> |
| 有風險的動作確認<br/> | 是，否。<br/>                                                                                             |



 

-   **不正確：**
-   ![具有 [確定] 按鈕之警告對話方塊的螢幕擷取畫面 ](images/mess-warn-image26.png)
-   問題不正常，請改用 Close。

## <a name="documentation"></a>文件

參考警告時：

-   如果警告提出問題，請以其問題參考警告;否則，請使用 main 指令。 如果問題或主要指示很長或更詳細，請總結。
-   如有必要，您可以將警告對話方塊參考為訊息。
-   可能的話，請使用粗體格式化文字。 否則，請只在必要時才將文字放在引號中，以避免混淆。

範例：在 [ **您要顯示不安全的專案嗎？** ] 訊息中，按一下 [是]。
