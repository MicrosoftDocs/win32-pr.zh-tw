# <a name="search-boxes"></a>搜尋方塊

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用搜尋方塊，使用者可以藉由篩選或反白顯示相符專案，快速找出一組大型資料中的特定物件或文字。 沒有標準的搜尋控制項，但您應該努力讓程式的搜尋功能與 Windows 的功能保持一致。

搜尋的類型有兩種：

-   立即 **搜尋**，結果會立即顯示為使用者類型。 不需要按一下按鈕，因此放大鏡搜尋符號會顯示為圖形，而不是按鈕。

    ![顯示即時搜尋方塊的螢幕擷取畫面，其中包含指向搜尋方塊的 [提示] 標注，以及指向放大鏡圖形的 [搜尋符號] 標注。](images/ctrl-search-boxes-image1.png)

    使用立即搜尋的一般搜尋方塊。 每次擊鍵時都會自動執行搜尋。

-   **一般搜尋**，在使用者按一下 [搜尋] 按鈕時執行搜尋。 放大鏡搜尋符號會顯示在按鈕上。

    ![一般搜尋方塊的螢幕擷取畫面 ](images/ctrl-search-boxes-image2.png)

    使用一般搜尋的一般搜尋方塊。 使用者按一下按鈕來執行搜尋。

    您可以為您的使用者提供一種或兩種搜尋選項。

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **是否有特定物件難以尋找？** 發生的時機為：
    -   有許多物件。
    -   物件不是位於單一位置。 搜尋特別適用于在樹狀結構中尋找物件。
    -   搜尋資料很難找到 (例如，中繼資料) 。
-   **使用者是否需要尋找檔中的特定文字？**
-   **您的功能是否會在五秒內傳回相關的搜尋結果？** 如果沒有，您可以提供搜尋功能，但會使用替代的設計來提供可見的意見反應，以容納長時間執行的搜尋，例如搜尋對話方塊。

## <a name="design-concepts"></a>設計概念

在許多案例中，搜尋是很重要的第一步：使用者必須先尋找物件才能使用它們。 使用者會在越來越大的硬碟上儲存越來越多的物件，但是流覽物件無法妥善調整。 搜尋必須是使用者體驗的簡單、一致、可靠的部分。

Windows 中的搜尋方塊：

-   是所有 Explorer 視窗的一部分，因此很容易找到和辨識。
-   具有一致的外觀和行為。
-   既有效率又快速，提供立即搜尋模式的即時結果。

在下列位置中，Windows 會使用搜尋方塊：

-   總管
-   體驗 (Microsoft Windows Media Player、Windows 影像中心、Windows Internet Explorer) 
-   [開始] 功能表 (尋找程式和最近使用的檔案) 
-   主控台首頁] (尋找 [控制台] 專案和工作) 
-   協助 (尋找相關說明主題) 

### <a name="look-and-feel"></a>外觀與風格

在 Windows 中搜尋的風格，藉由支援立即搜尋來大幅增強。 有了立即的結果，讓 Windows 感覺更強大且更直接。

在 Windows 檔案總管和應用程式視窗中，如果是增補進入點，則搜尋位於右上角。 在此情況下，當使用者在視窗中找不到想要的內容時，就會尋找搜尋機制。 但是，如果搜尋是主要進入點，則會在視窗的頂端置中。

[搜尋] 按鈕會以視覺化方式連線到搜尋方塊。 若要將空間最小化，請在搜尋方塊中使用選擇性的 [提示](ctrl-text-boxes.md) ，而不是使用標籤。 提示可能是指令 (例如，輸入搜尋) 或指出搜尋的範圍 (例如，搜尋) 的圖片。

![立即搜尋方塊的螢幕擷取畫面 ](images/ctrl-search-boxes-image3.png)

如果沒有標籤和個別按鈕，Windows 中的即時搜尋就會有輕量的外觀。

執行成功搜尋會建立搜尋結果的虛擬頁面，並將其新增至後端堆疊和位址列。 使用者有數種方式可還原原始頁面，並清除搜尋方塊，包括按一下 [上一步]、按一下網址列中的原始頁面、按下 Esc 或清除 [搜尋] 方塊。

使用者也可以直接清除 [搜尋] 方塊，而不還原上一頁的結果。 在 [立即搜尋] 模式中，當使用者開始鍵入之後，會出現 [清除] 按鈕;"x" 會取代放大鏡搜尋符號。 停留時，"x" 會取得按鈕外觀，且可以按一下。

![具有 ' x ' 圖示之搜尋方塊的螢幕擷取畫面 ](images/ctrl-search-boxes-image4.png)

使用者可以按一下控制項右端的 [x] 來清除搜尋方塊。

在一般搜尋模式中，[清除] 按鈕是選擇性的。 例如，如果搜尋需要很長的時間才能完成，使用者可能會發現它很有用。 使用者可以按一下 "x" 來停止搜尋進行中。 如果搜尋已完成，使用者可以按一下 "x" 來清除搜尋方塊。

## <a name="guidelines"></a>指導方針

### <a name="location"></a>Location

-   若為應用程式視窗，請在右上角尋找 [搜尋]。
-   若為快顯視窗，請在最適合且方便的地方尋找搜尋。
-   **例外狀況：** 如果搜尋通常是使用者在視窗 (主要進入點) 的第一件事，請將其置中在視窗的頂端。

### <a name="look"></a>看

-   使用標準搜尋按鈕圖形。 有三個版本：
    -   **只有在滑鼠停留) 上 (沒有按鈕時，才會顯示放大鏡搜尋符號。** 用於立即搜尋。
    -   **放大鏡搜尋符號與按鈕。** 當您需要按一下按鈕來開始搜尋時，請使用。
    -   **具有按鈕和下拉箭號的放大鏡搜尋符號。** 當使用者可以變更範圍或有其他設定可供使用時，請新增下拉式箭號。
        -   若為立即搜尋，請只使用下拉式箭號，並在滑鼠停留時顯示按鈕。
        -   針對一般搜尋，請在按鈕上顯示下拉箭號。

![不同狀態之一般搜尋方塊的圖 ](images/ctrl-search-boxes-image6.png)

-   請勿使用標籤;請改用選用的提示。 如果使用者傾向于假設搜尋是一般檔案搜尋，請使用提示來提供範圍。 否則，請使用類型來搜尋或類似的簡潔片語。

    在這些範例中，簡短的文字提示可協助使用者使用搜尋。

### <a name="interaction"></a>互動

-   **在輸入焦點上，自動選取任何先前輸入的文字。** 這麼做可讓使用者藉由輸入來輸入新的搜尋，或使用方向鍵來定位插入號來修改先前的搜尋。

    ![具有反白顯示文字之搜尋方塊的螢幕擷取畫面 ](images/ctrl-search-boxes-image9.png)

    在此範例中，會在輸入焦點上選取先前輸入的文字。

-   **將 [搜尋] 方塊的鍵盤快速鍵指派為 Ctrl + E。**

### <a name="functionality"></a>功能

-   **盡可能支援立即搜尋。** 如果有一般搜尋需要額外等候時間的情況，請提供一般和立即搜尋。
-   一般搜尋必須在五秒內傳回相關的結果，而立即搜尋必須在2秒內傳回結果。 在此之後，只要程式有回應，而且使用者可以執行其他工作，搜尋就會繼續在較短的時間內填滿較不相關的結果。 您可能必須編制搜尋資料的索引，以確保此回應性。
-   如果您同時提供一般和立即搜尋模式，則即時搜尋結果必須是一般搜尋結果的子集。
-   所有搜尋都是首碼型 (沒有搜尋) 的子字串或尾碼。 使用尾端萬用字元是選擇性的，不會影響結果。 如果輸入多個單字，請使用或搜尋。
-   成功搜尋會將包含搜尋結果的虛擬頁面新增至後端堆疊和位址列。 單一虛擬頁面中有多個搜尋結果，所以按一下 [上一步] 一律會傳回原始頁面。
-   如果需要調整規模，請依相關性排名搜尋結果。
-   空白搜尋會傳回原始頁面。

## <a name="text"></a>Text

-   針對 [搜尋] 方塊中的提示文字，請將其設為指示 (例如，輸入以搜尋) 或指出搜尋的範圍 (例如，搜尋) 的圖片。
-   提示文字應簡短。 單一單字或簡短片語應已足夠。
-   使用句型大寫。
-   請勿使用結尾標點符號或省略號。

## <a name="documentation"></a>文件

-   請將此控制項稱為搜尋方塊。 將第一個單字的初始字母大寫：不要將方塊的初始字母大寫。
-   請以立即搜尋和一般搜尋的形式來參考這兩種搜尋類型。 將立即搜尋的初始字母大寫：請勿將一般搜尋的初始字母變成大寫。
