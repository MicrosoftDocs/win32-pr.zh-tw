# <a name="progressive-disclosure-controls"></a>漸進式洩漏控制項

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

透過漸進式洩漏控制項，使用者可以顯示或隱藏其他資訊，包括資料、選項或命令。 漸進式洩漏透過將焦點放在必要的情況下提高簡易性，但視需要洩漏其他詳細資料。

![漸進式洩漏控制項的螢幕擷取畫面](images/progressive-disclosure-controls-image1.png)

漸進式洩漏控制項的範例。

> [!Note]  
> 與 [版面](vis-layout.md)配置、 [功能表](cmd-menus.md)和 [工具列](cmd-toolbars.md) 相關的指導方針會顯示在不同的文章中。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **使用者是否需要在部分（而非全部）的情況下查看資訊，或是部分但不是所有的時間？** 如果是，使用漸進式洩漏來顯示資訊會簡化基準體驗，但可讓使用者輕鬆地存取訊號。

    ![顯示 [安全性中心] 狀態顯示的螢幕擷取畫面。](images/progressive-disclosure-controls-image2.png)

    在此範例中，資訊安全中心會隨時顯示重要的安全性狀態，但會使用漸進式洩漏來顯示隨選的詳細資料。

-   **如果預設會顯示資訊，使用者是否有可能選擇隱藏該資訊？** 是否有使用者需要更多空間的情況？ 使用者是否有充分的動機可以自訂使用者介面 (UI) ？ 如果沒有，請在不使用漸進式洩漏的情況下顯示資訊。

    **不正確：**

    ![預設顯示的資料選項螢幕擷取畫面 ](images/progressive-disclosure-controls-image3.png)

    在此範例中，使用者不會想要隱藏資訊。

-   **額外的資訊是 advanced、重大、複雜，還是與獨立子工作相關的？** 如果是的話，請考慮使用 [命令按鈕](ctrl-command-buttons.md) 或 [連結](ctrl-command-links.md) ，將資訊顯示在另一個視窗中，而不是使用漸進式洩漏控制項。 如果適用于 advanced users， (其他資訊為 advanced。 如果它讓其他資訊難以閱讀或配置，則會很複雜。 ) 

    ![您要執行這個檔案的螢幕擷取畫面嗎？ ](images/progressive-disclosure-controls-image4.png)

    在此範例中，軟體名稱和發行者的相關資訊主要對 advanced users 有意義，因此會使用不同視窗的連結。

-   **其他資訊是句子或句子片段的詳細資訊，可描述專案的用途或使用方式？** 如果是的話，請考慮使用 [工具提示或提示](ctrl-tooltips-and-infotips.md) 。
-   **這是與目前工作相關的其他資訊，但與目前顯示的資訊無關嗎？** 如果是的話，請考慮改用 [tab 鍵](ctrl-tabs.md) 。 不過，可折迭清單通常偏好用於索引標籤，因為它們較具彈性且可擴充。
-   **顯示或隱藏其他資訊基本上是資料篩選器？** 如果是的話，請考慮使用 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) 或 [核取方塊](ctrl-check-boxes.md) ，將篩選套用至整個清單。

## <a name="design-concepts"></a>設計概念

漸進式洩漏的目標是：

-   將焦點放在必要時 **簡化 UI** ，但視需要洩漏其他詳細資料。
-   藉由減少雜亂的認知，**簡化 UI 的外觀**。

您可以使用漸進式洩漏控制項來達成兩個目標，讓使用者按一下以查看更多詳細資料。 不過，您可以透過下列方式，達到簡化外觀的第二個目標，而不需要使用明確的漸進式洩漏控制項：

-   **僅顯示內容中的內容詳細資料。** 例如，您可以在與所選物件或模式相關的情況下，自動顯示內容相關的命令或工具列。
-   **減少次要 UI 的 affordances 權數。**[Affordances](glossary.md) 是視覺屬性，建議使用物件的方式。 趨勢是讓使用者可以就地進行互動，但讓所有的這類 UI 都繪製成 scream 「按我！」 導致太多視覺效果雜亂。 針對次要 UI，通常最好使用微妙的 affordances，並在滑鼠上方提供完整的效果。

    ![用來分級相片的星星圖示螢幕擷取畫面 ](images/progressive-disclosure-controls-image5.png)

    在此範例中，[分級] 欄位是互動式的，但在滑鼠停留之前不會出現。

-   **只有在必要條件完成後才顯示後續步驟。** 這種方法最適用于熟悉的工作，使用者可以放心採取第一步。

    ![[使用者名稱] 和 [密碼] 文字方塊的螢幕擷取畫面 ](images/progressive-disclosure-controls-image6.png)

    在此範例中，[使用者名稱] 和 [密碼] 頁面一開始只會顯示 [使用者名稱] 和 [選用密碼] 方塊。 確認和提示方塊會在使用者輸入密碼之後顯示。

雖然漸進式洩漏是簡化 Ui 的絕佳方法，但是有下列風險：

-   **缺乏可探索性。** 使用者可能會假設，如果看不到任何東西，就不會存在。 如果使用者沒有看到他們要尋找的內容，則可能不會停留或按一下。 有時候使用者可能不會按到更多選項。
-   **缺乏穩定性。** 漸進式洩漏應預期或至少是自然的。 如果控制項意外出現並消失，則產生的 UI 可能會覺得不穩定。

### <a name="progressive-disclosure-controls"></a>漸進式洩漏控制項

漸進式洩漏控制項通常會在沒有描述其行為的直接標籤的情況下顯示，因此使用者必須能夠根據控制項的視覺外觀，來執行下列動作：

-   辨識控制項可提供漸進式的洩漏。
-   判斷目前的狀態為展開或折迭。
-   判斷是否需要額外的資訊、選項或命令來執行工作。
-   判斷如何還原原始狀態（如有需要）。

雖然使用者可以根據試用和錯誤來判斷上述情況，但您應該嘗試進行這類實驗。

漸進式洩漏控制項有相當弱的 [affordances](glossary.md)，這表示其視覺效果屬性會建議其使用方式，雖然弱。 下表比較常見漸進式洩漏控制項的外觀：

## <a name="usage-patterns"></a>使用模式

漸進式洩漏控制項有數種使用模式。 其中一些內建于通用控制項中。

### <a name="chevrons"></a>角括弧

燕尾顯示或隱藏完整或部分隱藏內容中的其餘專案。 通常會顯示專案，但是也可以在快顯功能表中顯示這些專案。 當您準備好時，專案會一直展開，直到使用者折迭為止。

以下列方式使用燕：



|      使用狀況                                                                                                                                                          |    範例                                                                                                                                                                                                                                                                                                             |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **就地 UI**<br/> 相關聯的物件會收到輸入焦點，而且會使用空格鍵來啟用單一燕號。<br/>                       | ![[安全性中心] 狀態顯示的螢幕擷取畫面 ](images/progressive-disclosure-controls-image11.png)<br/> 在這些範例中，就地單一燕點位於其相關聯控制項的右邊。<br/>                                                                            |
| **具有外部標籤的命令按鈕**<br/> [命令] 按鈕會接收輸入焦點，而且會使用空格鍵來啟用單一的燕號線。<br/> | ![具有 [更多選項] 標籤的燕號螢幕擷取畫面 ](images/progressive-disclosure-controls-image12.png)<br/> 在此範例中，會將單一的燕形按鈕標示，並放在標籤的左邊。 使用這個模式時，在沒有標籤的情況下，按鈕會很難瞭解。<br/> |
| **具有內部標籤的命令按鈕**<br/> 命令按鈕會接收輸入焦點，並以空格鍵啟用。<br/>                    | ![[更多] 和 [less] 命令按鈕的螢幕擷取畫面 ](images/progressive-disclosure-controls-image13.png)<br/> 在這些範例中，一般的命令按鈕會有雙箭號，以建議其意義。<br/>                                                                          |



 

### <a name="arrows"></a>箭頭

箭號會顯示快顯命令功能表。 專案會一直展開，直到使用者選擇或按一下任何位置為止。

如果箭號按鈕是獨立的控制項，則會收到輸入焦點，並以空格鍵啟用。 如果箭號按鈕具有父控制項，父控制項就會收到輸入焦點，而且會使用 Alt + 向下鍵和 Alt + 向上鍵和 Alt + 向上鍵來啟用箭號，如同下拉式清單控制項一樣。

箭號會以下列方式使用：



|    使用狀況                                                                                   |    範例                                                                                                                                                                                                                                   |
|---------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **分隔按鈕**<br/> 箭號位於個別的按鈕控制項中。<br/> | ![控制項右邊箭號的螢幕擷取畫面 ](images/progressive-disclosure-controls-image14.png)<br/> 在這些範例中，位於右邊的不同箭號按鈕表示命令功能表。<br/>               |
| **命令按鈕**<br/> 箭號是命令按鈕的一部分。<br/>      | ![命令按鈕上標籤和箭號的螢幕擷取畫面 ](images/progressive-disclosure-controls-image15.png)<br/> 在這些範例中，功能表按鈕和分割按鈕的箭號位於文字的右邊。<br/> |



 

### <a name="plus-and-minus-controls"></a>加號和減號控制項

加號和減號控制項會展開或折迭，以在流覽階層時，就地顯示容器內容。 專案會一直擴充，直到使用者折迭為止。 雖然它們看起來像按鈕，但它們的行為是就地的。

相關聯的物件會收到輸入焦點。 加號會以向右箭號來啟動，並以向左箭號鍵來啟用。

加號和減號控制項的使用方式如下：



|       使用狀況                                                                                         |       範例                                                                                                                                                                                                                               |
|------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **可折迭的樹狀結構**<br/> 顯示容器內容的多層級階層。<br/> | ![顯示已選取 [行為] Windows 檔案總管資料夾樹狀結構的螢幕擷取畫面。](images/progressive-disclosure-controls-image16.png)<br/> 在此範例中，加號和減號控制項位於相關容器的左邊。<br/>       |
| **可折迭清單**<br/> 顯示容器內容的兩層階層。<br/>   | ![展開清單以顯示兩個層級的螢幕擷取畫面 ](images/progressive-disclosure-controls-image17.png)<br/> 在此範例中，加號和減號控制項位於相關清單標題的左邊。<br/> |



 

### <a name="rotating-triangles"></a>旋轉三角形

旋轉三角形會顯示或隱藏個別專案的其他資訊。 它們也可用來展開容器。 專案會一直擴充，直到使用者折迭為止。

相關聯的物件會收到輸入焦點。 折迭的 (右) 三角形會以向右箭號來啟動，並以向左方向鍵展開 (向下定位) 三角形。

旋轉三角形的使用方式如下：



|     使用狀況                                                                                                       |    範例                                                                                                                                                                                                                             |
|------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **可折迭的樹狀結構**<br/> 顯示容器內容的多層級階層。<br/>             | ![windows explorer 資料夾樹狀結構的螢幕擷取畫面 ](images/progressive-disclosure-controls-image18.png)<br/> 在此範例中，旋轉三角形位於相關容器的左邊。<br/>       |
| **可折迭清單**<br/> 用來顯示其他資訊的二級階層。<br/> | ![顯示其他資料的清單螢幕擷取畫面 ](images/progressive-disclosure-controls-image19.png)<br/> 在此範例中，旋轉三角形位於其相關清單專案的左邊。<br/> |



 

### <a name="preview-arrows"></a>預覽箭號

如同燕尾，會顯示或隱藏其他資訊。 專案會一直擴充，直到使用者折迭為止。 和燕號不同的是，圖像以圖形表示動作，通常是以箭號指出將會發生什麼事。

![指向對角線的箭號字元螢幕擷取畫面 ](images/progressive-disclosure-controls-image20.png)

在這些來自 Microsoft Windows Media Player 的範例中，圖像會有箭號來建議將會發生的動作。

預覽箭號最適合用於標準箭號未充分傳達控制項行為的情況，例如，當洩漏很複雜或有多種類型的洩漏時。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **根據其使用方式選取漸進式洩漏模式。** 如需每個使用模式的說明，請參閱上一個表格。
-   **請勿使用漸進式洩漏控制項的連結。** 使用 [使用模式] 區段中顯示的漸進式控制項。 不過，請使用連結來流覽至 [說明主題](winenv-help.md)。

    **正確：**

    ![具有「顯示混音器」標籤的燕號螢幕擷取畫面 ](images/progressive-disclosure-controls-image21.png)

    **不正確：**

    ![[顯示混音器] 連結文字的螢幕擷取畫面 ](images/progressive-disclosure-controls-image22.png)

    在不正確的範例中，會使用連結來就地顯示更多選項。 如果流覽至另一個頁面或對話方塊，或顯示說明主題，此使用方式會是正確的。

### <a name="interaction"></a>互動

-   **針對未直接標記的燕號和箭號，請使用工具提示來描述其功能。**

    ![[展開查詢產生器] 工具提示的螢幕擷取畫面 ](images/progressive-disclosure-controls-image23.png)

    在此範例中，工具提示指出未標記的燕號控制項效果。

-   **如果使用者展開或折迭專案，請讓狀態保持不變，以便在下一次顯示視窗時生效**，除非使用者可能偏好以預設狀態啟動。 將狀態保持在每個視窗、每位使用者的基礎上。
-   **請確定所有展開的內容都可以折迭，反之亦然，相反的作業很明顯。** 這樣做會促進探索並減少挫折。 使反向操作變得顯而易見的最佳方式，是將控制項保持在相同的固定位置。 如果您需要移動控制項，請將它保留在視覺上不同區域內的相同相對位置。

    **不正確：**

    ![[取代] 按鈕的螢幕擷取畫面（具有燕號） ](images/progressive-disclosure-controls-image24.png)

    ![[取代] 按鈕的螢幕擷取畫面（沒有箭號） ](images/progressive-disclosure-controls-image25.png)

    在此範例中，按一下 [取代] 按鈕時，會顯示 [ **取代成** ] 文字方塊。 完成這項操作之後，Replace 擴充器會變成 Replace 命令，所以沒有辦法還原原始狀態。

-   請 **僅使用適用于漸進式洩漏模式的存取金鑰**，如使用模式一節中所列。 請勿使用 Enter 來啟用漸進式洩漏。

### <a name="presentation"></a>簡報

-   **請勿針對漸進式洩漏以外的目的使用三角形形狀的箭頭。**

    **不正確：**

    ![具有右手三角形的標籤螢幕擷取畫面 ](images/progressive-disclosure-controls-image26.png)

    雖然此範例不是漸進式的洩漏模式，但是使用箭號表示命令會顯示在快顯視窗中。

    **正確：**

## <a name="labels"></a>標籤

-   針對具有外部標籤的命令按鈕上的燕形：
    -   指派唯一的 [存取金鑰](glossary.md)。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
    -   使用 [句子樣式的大小寫](glossary.md)。
    -   以片語的形式寫入標籤，並不使用結束標點符號。
    -   寫入標籤，讓它描述按一下按鈕的效果，並在狀態變更時變更標籤。
    -   如果介面一律顯示一些選項、命令或詳細資料，請使用下列標籤配對：
        -   **更多/較少的選項。** 用於選項或混合使用選項、命令和詳細資料。
        -   **更多/較少的命令。** 僅用於命令。
        -   **更多/較少的詳細資料。** 僅供資訊使用。
        -   **更多/較少 <object name> 。** 針對其他物件類型（例如資料夾）使用。
    -   否則：
        -   **顯示/隱藏選項。** 用於選項或混合使用選項、命令和詳細資料。
        -   **顯示/隱藏命令。** 僅用於命令。
        -   **顯示/隱藏詳細資料。** 僅供資訊使用。
        -   **顯示/隱藏 <object name> 。** 針對其他物件類型（例如資料夾）使用。
-   若為具有內部標籤之命令按鈕上的燕，請遵循標準 [命令按鈕](ctrl-command-buttons.md) 指導方針。

## <a name="documentation"></a>文件

參考漸進式洩漏控制項時：

-   如果控制項有固定標籤，請只依其標籤參考控制項;請勿嘗試描述控制項。 使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線。
-   如果控制項沒有標籤或未固定，請使用控制項的類型來參考控制項：箭號、箭號、三角形或加號/減號按鈕。 如有必要，也請描述控制項的位置。 如果控制項以動態方式出現（例如 [頁面空間](glossary.md) 控制項），請藉由描述如何顯示控制項來開始參考。

    **範例：** 若要顯示資料夾內的檔案，請將指標移到資料夾名稱的開頭，然後按一下資料夾旁的三角形。

-   請勿將控制項參考為按鈕，除非與其他不是按鈕的漸進式控制項相較之下。
-   若要描述使用者互動，請使用按一下。 如果需要清楚起見，請使用按一下 .。。以展開或折迭。
-   可能的話，請使用粗體文字來格式化標籤。 否則，請只在必要時才將標籤放在引號中，以避免混淆。

範例：

-    (箭號) 來判斷檔案大小，請按一下 [ **詳細資料**]。
-    (箭號) 若要查看所有選項，請按一下 [ **搜尋** ] 方塊旁邊的箭號。
-    (針對加號/減號) 若要查看您的圖片，請按一下 [ **圖片**]。
