# <a name="visuals"></a>視覺效果

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

視覺效果包括控制項以外的視覺元素。 這些指導方針可協助您在應用程式中進行版面配置、字型、色彩、圖示等的決策。

## <a name="in-this-section"></a>本節內容



| 主題                                                       | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|-------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [版面配置](vis-layout.md)<br/>                         | 版面配置是視窗或頁面中內容的大小調整、間距和位置。 有效的版面配置對於協助使用者找到所需的內容，以及讓外觀以視覺效果吸引人來說非常重要。 有效的版面配置可讓使用者立即瞭解的設計，以及讓使用者感覺 puzzled 和不知所措的不同。<br/>                                                                      |
| [字型](vis-fonts.md)<br/>                           | 使用者與文字的互動比 Microsoft Windows 中的任何其他元素更多。 Segoe UI (發音為「看不到」 ) 是 Windows 系統的字型。 標準字型大小已增加至9點。<br/>                                                                                                                                                                                                                                                       |
| [色彩](vis-color.md)<br/>                           | Color 是大部分使用者介面的重要視覺元素。 除了純美學之外，color 具有相關聯的意義和 elicits 情緒回應。 為了避免混淆，必須一致地使用色彩。 若要取得所需的情緒回應，必須適當地使用色彩。<br/>                                                                                                                                                    |
| [圖示](vis-icons.md)<br/>                           | 圖示是物件的圖形標記法，但不只是因為美觀的原因，而是程式視覺化身分識別的一部分，也有 utilitarian 的原因，也就是用來傳達使用者幾乎立即觀察到之意義的縮寫。 Windows Vista 引進了一種新的圖示樣式，為 Windows 帶來更高層級的詳細資料和複雜性。<br/>                                                                          |
| [標準圖示](vis-std-icons.md)<br/>              | 標準圖示是 Windows 中的錯誤、警告、資訊和問號圖示。<br/>                                                                                                                                                                                                                                                                                                                                                  |
| [動畫和轉換](vis-animations.md)<br/> | 使用動畫和轉換的策略可以讓您的程式更容易瞭解、更順暢、更自然、更高的品質，而且更吸引人。 但是，不使用動畫和轉換的方式可能會讓您的程式雜亂，甚至令人討厭。 <br/>                                                                                                                                                                           |
| [圖形元素](vis-graphic.md)<br/>              | *圖形元素* 會以視覺化方式顯示關聯性、階層和強調。 它們包括背景、橫幅、玻璃、匯總工具、分隔符號、陰影和控點。<br/>                                                                                                                                                                                                                                                                                      |
| [音效](vis-sound.md)<br/>                           | *音效* 是使用者體驗的音訊元素。 當適當地使用時，音效可能是有效的通訊形式，可與您的使用者建立非口頭甚至情緒的關聯性。 音效可以單獨使用，也可以作為視覺效果 UI 的補充。 例如，將音效效果新增至通知會增加它所注意到的可能性，特別是當使用者未在事件發生時查看畫面時。<br/> |
