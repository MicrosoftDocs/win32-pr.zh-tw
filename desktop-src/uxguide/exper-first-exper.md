# <a name="first-experience"></a>首次執行經驗

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

在理想的第一個經驗中，使用者會立即安裝您的程式並立即使用它，而不需要回答一連串問題或學習一大堆問題。

第一個體驗使用者介面可協助使用者從第一次暴露到新程式或功能，再轉換成日常使用。

針對 Windows 程式，當使用者執行安裝程式時，會發生初次首次的體驗。 安裝程式通常：

-   要求使用者接受 (EULA) 的使用者授權合約。
-   索取產品金鑰。
-   顯示必要的設定相關選項，包括安裝選用的軟體。
-   將軟體複製到使用者的硬碟。
-   提供適用于所有使用者的程式選項。

![[輸入產品金鑰] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image1.png)

一般 Windows 安裝體驗的一部分。

第一個經驗接著會繼續使用程式或功能。 第一次使用體驗可以：

-   提供僅適用于目前使用者的程式選項。
-   提供產品或功能教學課程。

![[歡迎中心] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image2.png)

第一個使用體驗。

**注意：**[程式選項](win-property-win.md)的相關指導方針會以個別文章的形式出現。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定，請考慮下列問題。

### <a name="setup-experience"></a>設定體驗

適用下列條件嗎？

-   使用程式需要正確的設定，並套用至所有使用者。
-   這些設定會自訂核心體驗，或自訂對使用者個人身分識別的重要體驗。
-   沒有安全的預設值，使用者可能會選擇不是預設值的設定，或預設設定需要使用者同意。
-   在安裝之後，使用者不太可能會變更設定。
-   變更設定需要提高許可權。

如果是，請考慮在安裝體驗期間呈現設定。

### <a name="first-use-experience"></a>首次使用體驗

適用下列條件嗎？

-   使用程式或功能時，需要正確的設定或工作，並將它們套用至個別使用者。
-   這些設定會自訂核心體驗，或自訂對使用者個人身分識別的重要體驗。
-   沒有安全的預設值，使用者可能會選擇不是預設值的設定，或預設設定需要使用者同意。
-   在程式的內容中，使用者可能會比安裝程式更好的選擇。
-   使用者不太可能使用選項來變更設定。

如果是的話，請考慮在第一次使用程式或功能時，呈現工作和設定。

## <a name="design-concepts"></a>設計概念

在理想的第一個經驗中，使用者安裝您的程式 (或甚至是在不需要安裝) 的情況下啟動，並立即以有效率的方式使用它，而不需回答一連串問題或學習一堆問題。

這很適合用於大部分的程式，因此您應該在可以的情況下，盡力取得這項理想的體驗。 不過，對於需要大量系統整合、有許多選用功能或有隱私權含意的程式來說，通常無法使用這個目標。 例如，如果您的程式具有可能會向未受信任的物件顯示個人資訊的功能，則可 [信賴](https://www.microsoft.com/mscorp/twc/default.mspx) 運算的原則會要求您先取得使用者同意，才能啟用這些功能。

### <a name="questions-arent-choices"></a>問題不是選擇

問題需要回應，使用者才能繼續。 第一次體驗期間的問題是使用者必須跳過的問題，使用者才能有效率地使用您的程式。 相反地，選擇是選擇性的。 使用者不需要對其進行回應，也可以選擇只在想要時查看。

因此，在安裝程式的主要流程中所顯示的設定是問題，而主要設定流程之外或 [程式選項] 對話方塊中的設定則是選擇。 不必要的問題讓程式的第一次體驗變得繁瑣且冗長，有效地免除使用者開始使用您的程式時的正面預期和驚喜。

### <a name="use-the-first-experience-when-you-must"></a>當您需要時，請使用第一次體驗

當您必須在第一次體驗期間向使用者呈現設定和工作，但通常有更好的替代方案：



| 首次體驗  | 替代方案       |
|---------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| 設定問題<br/>                  | 選取適當的預設值。<br/> 允許使用者從程式選項變更。<br/> 提供一般的 vs 自訂安裝路徑。 <br/> |
| 第一個使用問題<br/>              | 選取適當的預設值，並允許使用者從程式選項變更。<br/>                                                            |
| 第一次使用工作<br/>                  | 改為呈現內容。<br/>                                                                                                           |
| 首次使用功能公告<br/> | 讓最常見且最重要的工作更容易探索和內容相關。<br/>                                                                   |
| 第一個使用教學課程<br/>              | 使程式功能一目了然。<br/>                                                                                                 |
| 產品註冊<br/>             | 在 [說明] 功能表和 [關於] 方塊中提供命令。<br/>                                                                                             |



 

**如果您只做一件事 .。。**

儘量簡化您的第一個經驗。 立即讓您的程式正常運作。 選擇安全、安全、方便使用的預設值，並在安裝期間詢問問題，並只在您必須時才使用。

您只有一個機會可以成為良好的第一印象，而第一印象是持續的印象。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **限制使用程式或功能所需之工作和設定的第一種體驗，而且只有在沒有更好的替代方案時才包含這些專案。** 請參閱上表以瞭解替代方案。
    -   **例外狀況：** 將個人化或程式自訂設定新增至第一個體驗，如果他們的自訂是核心體驗的一部分，或對使用者的個人身分識別來說很重要。

![[輸入電腦名稱稱] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image3.png)

Windows 會在安裝期間要求使用者輸入電腦名稱稱和選擇背景，因為這些設定有助於形成與產品的情緒連接。

-   如果工作和設定適用于所有使用者，或變更設定需要提高許可權，**請使用其安裝體驗**。
-   如果工作和設定適用于個別使用者，**請使用第一次使用體驗**。

### <a name="presentation"></a>簡報

-   **偏好選用的工作和設定，以進行必要的工作和設定。** 避免強制使用者設定您的程式。

    ![[找到新硬體] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image4.png)

    [找到新硬體] 對話方塊可讓您選擇是否要安裝驅動程式軟體，而不是讓它成為必要的工作。

-   **在實際執行時，從主要工作流程中，採取選擇性的工作和設定。** 例如，許多安裝程式都會提供自訂安裝路徑，以從主要工作流程中移除不常變更的設定。

    ![完整和自訂選項按鈕的螢幕擷取畫面 ](images/exper-first-exper-image5.png)

    如果使用者不想要自訂安裝，可協助主要工作流程的安裝體驗。

-   **使用工作和設定來避免使用者負荷：**
    -   **從簡入繁。** 從簡單的個人化設定開始，到更複雜、技術工作和設定的進度。 例如，Windows 安裝程式會以個人資訊為開頭，並以網路設定為結尾。
    -   如果工作和設定只適用于不是主要程式的基本功能，**請使用工作和設定的內容優先經驗**。

        ![[音訊和影片設定] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image6.png)

        Windows Live Messenger 具有適用于音訊和影片的內容設定，因為次要功能會使用這些設定。

-   **不要一次呈現所有內容。** 合併以使用單一 UI，而不是使用多個 UI 介面，或在不同的時間（而不是一次）顯示工作。

    **不正確：**

    ![五個重迭對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image7.png)

    在此範例中，第一個使用經驗很龐大。

-   **以使用者的目標和工作（而不是技術方面）表達問題和選項。** 提供使用者瞭解並清楚區分的選項。 請務必提供足夠的資訊，讓使用者做出明智的決策。
-   **如果不清楚需要個人資訊，請解釋您的程式需要資訊的原因，以及其使用方式。**

    ![指出電子郵件地址使用的文字螢幕擷取畫面 ](images/exper-first-exper-image8.png)

    在此範例中，電子商務應用程式會說明如何使用個人資訊。

-   **只有在使用者無法有效率地執行其他工作時，才會顯示第一個全螢幕體驗。** 例如，Windows 安裝程式會以全螢幕顯示，以防止使用者在安裝 Windows 時執行其他工作。 大部分的第一個體驗都不應該是全螢幕。

### <a name="settings"></a>設定

-   **針對所有設定，請選取最安全的 (，以防止遺失資料或系統存取) 、最安全且私用的值。** 如果安全性和安全性不是因素，請選取最有可能或方便的值。 選擇良好的預設值是簡化第一個體驗的有效方式。
-   **要求使用者加入宣告：**

    -   設定的法律影響，例如終端使用者授權合約 (Eula) 。 這類設定不能有預設選項。
    -   執行自動系統組態變更的功能，例如 Windows 自動更新。
    -   顯示個人識別資訊 (PII) 或系統資訊的功能。
    -   除了將專案新增至 [開始] 功能表之外，使用者桌面的變更，例如將圖示新增至桌面或快速啟動列。
    -   選用軟體，例如產品增強功能、訂閱和協力廠商產品。

    ![[選擇您要的功能] 對話方塊的螢幕擷取畫面 ](images/exper-first-exper-image9.png)

    在此範例中，使用者加入宣告產品增強功能、訂用帳戶和協力廠商產品。

-   **如果強烈建議設定，請將「 (建議的) 」新增至標籤。** 若為選項按鈕和核取方塊，請務必將加入至控制項標籤，而不是附加的附注。
-   **如果設定僅供 advanced 使用者使用，請將「 (advanced) 」新增至標籤。** 若為選項按鈕和核取方塊，請務必將加入至控制項標籤，而不是附加的附注。

### <a name="tasks"></a>工作

-   **協助使用者有效率地傳遞等待時間。**
    -   如果等候時間通常介於一到兩分鐘之間，請考慮在使用者等待時提供有用的資訊，例如，在安裝過程中的新功能。
    -   如果等候時間通常超過兩分鐘，讓使用者可以輕鬆地執行其他工作。 顯示預估的等候時間，建議使用者在一段時間內執行其他動作，並大幅變更畫面來使工作完成變得很明顯。
-   **在第一次體驗期間重新考慮呈現教學課程。** 最有可能的使用者想要立即使用您的程式，並對稍後的教學課程有興趣。
-   **在第一次體驗中，請勿使用功能公告通知。** 您可以將功能設計成更容易在需要的內容中探索，或不使用任何特殊功能，讓使用者自行探索功能，而不是使用 [功能公告通知](mess-notif.md)。
-   **在初始 Windows 體驗期間，請不要使用任何通知。** 為了改善其第一個經驗，Windows 7 會隱藏在前幾個小時的使用量中顯示的所有通知。 設計您的程式，假設使用者不會看到任何這類通知。
