# <a name="sound"></a>音效

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

*音效* 是使用者體驗的音訊元素。 當適當地使用時，音效可能是有效的通訊形式，可與您的使用者建立非口頭甚至情緒的關聯性。 音效可以單獨使用，也可以作為視覺效果 UI 的補充。 例如，將音效效果新增至通知會增加它所注意到的可能性，特別是當使用者未在事件發生時查看畫面時。

![[音效] 對話方塊的螢幕擷取畫面 ](images/vis-sound-image1.png)

從音效控制台專案的 [音效] 索引標籤中，使用者可以對其系統音效進行變更。

本文涵蓋如何在程式中使用音效作為事件和使用者動作的回應，以及將程式的音效控制項與 Windows 整合。 它並未涵蓋音樂或語音的使用。

**注意：** 與 [通知](mess-notif.md) 和 [商標](exper-branding.md) 相關的指導方針會顯示在不同的文章中。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定是否應該使用音效，請考慮下列問題：

-   **使用音效是否有明顯的使用者權益？** 由於使用音效的缺點很容易超過優點，因此只有在有明顯的優勢時才能使用音效。
-   **使用音效是否合適？** 使用音效會對值得注意的事情有注意嗎？ 如果使用者不存在，是否會錯過音效？ 將焦點放在讓使用者知道的音效，很可能會變更其行為，或提供有用的意見反應。
-   **使用音效會造成干擾嗎？** 有沒有頻繁、大聲、jarring 的音效嗎？ 使用者是否有可能會因為使用音效而減少系統磁碟區或您的程式磁片區？
-   **您使用音效作為主要的通訊形式嗎？** 在許多情況下，例如對於有某種程度的聽力遺失的使用者，不應該使用音效作為主要的通訊方式。 音效更能作為其他通訊方式的補充，例如 (的文字或視覺效果) 。
-   **主要目標使用者是 IT 專業人員嗎？** 音效通常不適用於以 IT 專業人員為目標的工作，因為許多工作都是自動執行。 此外，音效也無法針對他們想像一下，一次執行數百項工作，並在完成或失敗時收到聲音。

## <a name="design-concepts"></a>設計概念

通常音效可達成下列任何或所有用途：

-   **通知。** 音效可以與特定事件產生關聯。 例如，「新郵件」音效會在郵件送達時通知使用者，而不會中斷其目前的工作。
-   **回饋。** 音效可以提供特定使用者動作的意見反應。 例如，當您放開音量控制項上的滑杆時，所播放的微妙音效會提供目前設定層級的相關意見反應。
-   **品牌。** 音效可以與特定內容產生關聯，以品牌您的產品、應用程式或服務。 在啟動作業系統時，Windows 會以這種方式使用音效。
-   **娛樂。** 音效通常用來增強娛樂產品，並讓任何產品更吸引人。 例如，大部分的遊戲、訓練應用程式和消費者產品都使用音效來榮幸使用者並增強他們的體驗。

某些音效可以一次滿足許多這些用途。 例如，Windows 啟動音效表示啟動程式已完成，而且桌面已可供使用。 它也提供強大的產品品牌形式，甚至還會短暫地參與使用者。

無法滿足這些目的的音效可能會被排除。

### <a name="inappropriate-use-of-sound"></a>不適當的音效使用

**儘管音效的優點，適當的音效使用也需要大量的程式，否則可能會造成程式討厭和雜亂。** 如果使用者經常感到苦惱、重複、jarring、中斷、設計不佳的音效，則會完全關閉其音效;部分原因是因為它的本質，音效需要注意，而且很難忽略。 如需尋找合理餘額的秘訣，請參閱 [音效設計指導方針](#sound-design)。

由於使用音效的缺點很容易超過優點，因此只有在有明顯的優勢時才能使用音效。 **如果不確定，請不要使用音效。**

### <a name="make-sound-supplemental"></a>製作音效補充

即使音效已適當使用，在許多情況下，聲音可能對所有使用者而言都不會有效：

-   有些使用者可能會在無法聽到音效的雜訊環境中工作。
-   有些使用者可能會在安靜的環境中工作，需要在低磁片區關閉或設定音效。
-   有些使用者可能會有聽力障礙或遺失。
-   電腦可能沒有喇叭。

基於這些理由， **用於通知和意見反應的音效永遠不是唯一的通訊方法，** 而是應該補充視覺或文字提示。

### <a name="desirable-characteristics-of-sound"></a>音效的理想特性

一般情況下，聲音應該是：

-   從中到高頻率 (600 赫茲 \[ Hz \] 到2千赫茲 \[ kHz \]) 。
-   短 (小於一秒的) 。
-   大量的軟或中度。
-   意義。
-   令人愉快，不是太驚人或 jarring。
-   非口頭。
-   非重複。

使用音效，越少越好。 **理想的音效效果是使用者不會注意到的音效，但如果沒有的話，就會錯過。**

**常見的誤解是關鍵事件的音效必須有很大的 jarring，才能讓使用者注意。** 這並不是真正的，因為音效其實是一種補充的通訊方式。 在發生嚴重錯誤訊息的情況下，其呈現 (可能在強制回應對話方塊中) 、其圖示 (錯誤圖示) ，以及其文字和色調全都結合以傳達錯誤的性質。 有效的錯誤音效可能會比一般的 Windows 音效稍微高一點，但不需要大幅增加。

### <a name="characteristics-of-windows-sounds"></a>Windows 音效的特性

除了此 minimalism 的一般呼叫之外，Windows 音效美觀也會使用淺、純色調和 glassy 和 airy 音效，並以虛淡入和淡出 (軟式「邊緣」 ) 來防止突然、jarring、percussive 效果。 它們是設計來感覺微妙、溫和和輔音。 Windows 音效使用 echo、回音和均衡來達成自然、環境的感覺。

Windows 的預設音效配置通常不會使用太特定或音效的實用或可辨識的聲音。 它可避免的音效範例包括 pianos 或 percussion 儀器、動物音效、環境雜訊、語音、語音、類似電影的音效效果或其他音效。 此外，Windows 音效也不會被視為音樂 (也就是 melodies) 的多筆記。 這使得 Windows 音效與其他類型的音效有不同的功能。

由於 Windows 音效的設計目的是要讓廣大的物件具有適當的特性，並且吸引廣大的物件，因此 **請考慮在適當時使用這些內建的 Windows 音效。**

### <a name="designing-your-own-sounds"></a>設計您自己的聲音

如果您必須建立自己的音效，請將其設計為具有先前所述的特性。 努力讓它們補充相關聯的工作或事件。

瞭解建立原始音效很難特別針對廣大觀眾的音效。 音效可以是 polarizing 設計項目。 針對熱愛音效的每個使用者，將會有很多人不喜歡它。

**將您的程式的音效設計為群組，就像是主題的相關變異。** 您的程式的聽覺體驗應與視覺體驗協調。 此外，音效的「語氣」應該與 [文字的色調](text-style-tone.md)協調。 想像一下文字如何使用愉快，自然色調可以在破壞時，以驚人的音效音效。

**如果您只進行四件事 .。。**

1.  使用音效搭配擋板，可確保有明顯的整體使用者權益。 如果不確定，請不要使用音效。
2.  適當時，請使用內建的 Windows 音效。
3.  如果您設計自己的音效，請確定它們具有適當的音效特性，也就像是主題的變化一樣。
4.  不要假設音效需要 jarring，才能讓使用者注意。

## <a name="guidelines"></a>指導方針

### <a name="usage"></a>使用方式

-   **使用音效搭配擋板** ，可確保有明顯的整體使用者權益。 將焦點放在讓使用者知道的音效，很可能會變更其行為，或提供有用的意見反應。 如果不確定，請不要使用音效。
-   **根據其使用方式，選取音效及其特性。** 如需每個使用模式的說明，請參閱上一節中的表格。
-   **針對通知和意見反應，請不要使用音效作為唯一的通訊方法。** 相反地，請使用音效作為補充方法，以強化視覺或文字提示。這樣做可確保使用者在聽不到音效時可以看到資訊。
-   **請勿經常播放或太粗糙音效。** 這是不必要的，因此會導致使用者體驗不佳。 播放音效的頻率愈多，就越內斂。 音效不需要太大或太粗糙，就能吸引注意力。
-   **請勿嗶聲。** 嗶聲不適合新式程式。 嗶聲不能有指派的特定意義，使用者也無法控制它們。
    -   **例外狀況：** 重要的系統功能可能會在使用者必須立即參與的情況下發出警示，例如電力偏低。

### <a name="playback"></a>播放

-   **請勿連續重複使用兩次音效。**
-   **針對連續的相關音效事件順序，只播放第一個事件的音效。** 請避免使用多個音效，因為它們可能會彼此衝突，否則會導致不愉快的使用者體驗。

### <a name="sound-selection"></a>音效選取範圍

-   **選擇 [愉快的聲音]。** 請勿使用不愉快的音效，例如，蜂鳴、損毀和中斷。
-   **使用短** (小於一秒的聲音) 。
-   **使用與一般 Windows 音效大致相同磁片區的音效。** 使用者不喜歡在啟動電腦或程式時關閉音量，尤其是在像是會議和簡報等公用環境中。 Microsoft Windows 音效檔案位於 Windows 資料夾內的 Media 資料夾中。
-   **請勿選擇需要當地語系化的聲音。** 您可以使用不使用語音的音效或具有與文化特性相關的意義或 connotations 來達到此目的。

### <a name="windows-system-sounds"></a>Windows 系統音效

-   **適當時，請使用內建的 Windows 系統音效。**
-   **選擇使用系統音效的相關意義，而不只是聲音本身的聲音。** 系統音效必須一致地使用。

### <a name="sound-design"></a>音效設計

建立您自己的音效時：

-   **使用預期的音效特性建立音效。**
-   **以最高頻率的中等範圍撰寫音效 (600 Hz 到 2 kHz) 。** 請勿使用低頻率，因為它們會更遠移動、難以找出，而且可能會很難。
-   **將一般音效的相對波幅設定為一般 Windows 音效的層級。** Windows 音效已針對家用和工作環境適當地進行了調配。 針對您的音效使用不同的層級，將會強制使用者進行音量調整。
    -   將重要的聲音設定為稍微增加一點。 這類音效包括動作完成、動作失敗，以及重要的系統事件。
    -   將經常發生的音效設定為稍微柔和。 其中包括 FYIs、商標音效和音效效果。
-   **選擇與 Windows 音效的意義一致的音效。** 若要建立 Windows 音效的自訂版本，請保留相同的音調和間隔，但變更協調流程或 timbre。 請勿將不同意義指派給具有類似簡報和間隔的音效作為 Windows 音效。
-   **為您的程式設計音效，讓它們與主題的相關變化。** 您的程式的聽覺體驗應與視覺體驗協調。
    -   **同時設計場景轉換和音訊轉換。** 例如，如果場景逐漸淡出，則任何音效也都應該逐漸淡出。 不要因為音效轉換而造成無縫的視覺轉換。
-   **聲音必須採用 .wav 檔案格式。** 建議使用16位、44.1 kHz 身歷聲的非壓縮脈衝程式碼調製 (PCM) 格式。 如果檔案大小很重要，請使用壓縮或 monaural (mono) 格式，但請注意，在您的應用程式上可能會在應用程式上反映錯誤的明顯品質損失。

### <a name="mixing"></a>混合

-   **在您的程式中沒有音量或靜音控制項。** 相反地，讓使用者使用 Windows 磁片區混音器來控制應用程式之間的相對磁片區設定。 如果您的程式具有音量控制項，則會有多個位置讓使用者調整其設定，這可能會導致混淆。

    ![windows 磁片區混音器的螢幕擷取畫面 ](images/vis-sound-image8.png)

    Windows 磁片區混音器可讓使用者控制主要音量設定，以及每個目前播放音訊的程式的磁片區。

<!-- -->

-   **例外狀況：** 如果您的程式是音訊或影片播放或建立的主要用途，在程式中具有音量控制項可能會很有用。 使用滑杆控制項來實現此目的，並在使用者變更磁片區時提供立即的意見反應。

### <a name="windows-integration"></a>Windows 整合

-   **在 Windows 音效登錄中註冊程式的聲音。** 這麼做可讓 Windows 磁片區混音器為您的程式新增滑杆。
-   **註冊程式的自訂音效事件。** 這麼做可讓 [Windows 音效] 控制台專案顯示這些專案。 為每個自訂音效事件建立下列機碼： HKEY \_ CURRENT \_ USER \| AppEvents \| Event 標籤事件 \| 名稱 = event Name。
-   **請勿 hardwire 程式音效事件的音效。** 相反地，請使用登錄專案指定要播放的聲音。 這麼做可讓使用者透過音效控制台專案來自訂音效事件。
    -   **例外狀況：** 您可以選擇 hardwire 用於商標的聲音。
-   **請勿讓使用者在您的程式選項中設定音效。** 相反地，請使用 [Windows 音效] 控制台專案進行此用途。
-   **根據預設，請考慮不要將音效指派給經常發生的事件。** 不需要使用者將其設定為不令人討厭的初始體驗。

### <a name="directsound-programming-issues"></a>DirectSound 程式設計問題

-   針對具有自己的磁片區控制的 DirectSound 程式， **預設會將程式磁片區設定為100%。** 這樣做會將音訊的動態範圍最大化。
-   **請勿在獨佔模式中執行您的程式，以鎖定其他音效事件。** 這樣做可能會導致其他程式無法正常運作。 例如，使用獨佔模式會防止電腦作為電話語音裝置使用。

## <a name="text"></a>Text

-   請勿使用片語音效介面卡。 請改用音效卡。
-   使用裝置來一般地參考喇叭、耳機和麥克風。
-   使用控制器來參照控制裝置的音訊硬體，例如音效卡和晶片組。
-   使用「片語音效」配置來描述一般程式事件的音效集合，例如登入或接收新的電子郵件。 使用片語桌面主題來描述電腦桌面的視覺元素和音效的集合。
-   使用字詞來廣泛參考語音、音樂和音效。 使用「音效」來深入瞭解本文中所述的程式和 Windows 音效。
