# <a name="notification-area"></a>通知區域

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

通知區域會提供通知和狀態。 設計完善的程式會適當地使用通知區域，而不會干擾或困擾。

通知區域是工作列的一部分，可提供通知和狀態的暫時來源。 它也可以用來顯示桌面上沒有任何狀態的系統和程式功能圖示。

通知區域中的專案稱為通知區域圖示，如果已清楚地建立通知區域的內容，則只是圖示。

![通知區域、時間和日期的螢幕擷取畫面 ](images/winenv-notification-image1.png)

通知區域。

若要讓使用者能夠在 Windows 7 中控制其桌面，預設不會顯示所有通知區域圖示。 相反地，除非使用者將圖示升級為通知區域，否則圖示會顯示在通知區域溢位中。

![顯示通知區域和溢位的螢幕擷取畫面。](images/winenv-notification-image2.png)

通知區域溢位。

**注意：** 與 [工作列](winenv-taskbar.md)、通知和批註 [提示](mess-notif.md)相關的 [](ctrl-balloons.md)指導方針會顯示在不同的文章中。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定使用時機，請考量下列問題：

-   **您的程式是否需要顯示通知？** 如果是，您必須使用通知區域圖示。
-   **圖示是否會暫時顯示，以顯示狀態的變更？** 如果有的話，可能會有適當的通知區域圖示，視下列因素而定：

    -   **狀態是否有用且相關？** 也就是說，使用者是否有可能監視圖示，並因為這項資訊的結果而變更其行為？ 如果不是，則不會顯示狀態，也不會將它放入記錄檔中。

        **不正確：**

        ![具有磁片磁碟機圖示之通知區域的螢幕擷取畫面 ](images/winenv-notification-image3.png)

        在此範例中，磁片磁碟機活動圖示並不適當，因為使用者不太可能會根據它來變更其行為。

    -   **狀態是否危急？是否需要立即採取動作？** 如果是的話，請以需要注意的方式來顯示資訊，而且無法輕易地忽略，例如 [對話方塊](win-dialog-box.md)。

    針對 Windows 7 設計的程式可以使用程式工作列按鈕上的重迭圖示來顯示狀態的變更，以及工作列按鈕的進度列，以顯示長時間執行之工作的進度。

-   **功能是否已經有「桌上型電腦目前狀態」？** 也就是說，執行時，功能是否會出現在桌面的視窗中 (可能會最小化) ？ 如果是的話，請在程式的 [狀態列](ctrl-status-bars.md)、其他狀態區域，或是直接在工作列按鈕上，顯示 Windows 7 的狀態。 如果功能沒有桌上型電腦存在，您可以使用圖示進行程式存取，並顯示狀態。
-   **圖示主要是用來快速啟動程式或存取其功能或設定？** 若是如此，請改用 [開始] 功能表來啟動程式。 通知區域不適用於快速程式或命令存取。

    ![[快速啟動] 工具列的螢幕擷取畫面 ](images/winenv-notification-image4.png)

    在此範例中，Windows Vista 會使用快速啟動來快速啟動 Windows 檔案總管和 Windows Internet Explorer。

    針對 Windows 7 的設計程式，使用者可以釘選工作列按鈕以進行快速程式存取。 程式可以使用捷徑清單或縮圖工具列，直接從程式的工具列按鈕存取常用的命令。 在 Windows 7 中，預設不會顯示快速啟動區域。

    ![具有圖示的工作列和跳躍清單螢幕擷取畫面 ](images/winenv-notification-image5.png)

    在此範例中，會使用捷徑清單來存取快速命令。

## <a name="design-concepts"></a>設計概念

### <a name="the-windows-desktop"></a>Windows 桌面

Windows 桌面具有下列程式存取點：

-   **工作區。** 使用者可以執行其工作的螢幕區域，以及儲存程式、檔和其快捷方式。 雖然在技術上，桌面包含工作列，但在大部分的情況下，它只會參考工作區域。
-   **[開始] 按鈕。** 所有程式和特殊 Windows 的存取點都將 (檔、圖片、音樂、遊戲、電腦主控台) 、使用「最近使用的」清單來快速存取最近使用的程式和檔。
-   **快速啟動。** 使用者選取之程式的直接存取點。 快速啟動已從 Windows 7 移除。
-   **任務。** 執行具有桌上型電腦狀態之程式的存取點。 雖然在技術上，工作列橫跨 [開始] 按鈕到通知區域的整個橫條，但是在大部分的內容中，工作列都指的是介於中間的區域，其中包含工作列按鈕。 此區域有時稱為 taskband。
-   **Deskbands.不建議使用。**
-   **通知區域。** 通知和狀態的短期來源，以及不存在於桌面上的系統和程式相關功能的存取點。

![識別桌面存取點的螢幕擷取畫面 ](images/winenv-notification-image6.png)

Windows 桌面存取點包含 [開始] 按鈕、工作列和通知區域。 請注意工作列按鈕的縮圖功能。

**桌面是受限的共用資源，是使用者進入 Windows 的進入點。** 讓使用者保持控制。 您應以預期的方式使用桌面區域，否則應該將任何其他用途視為濫用。 例如，絕對不要將桌面區域視為推廣程式或其 [品牌](exper-branding.md)的方式。

### <a name="using-the-notification-area-appropriately"></a>適當地使用通知區域

通知區域最初是作為通知和狀態的暫時來源。 它的效率與便利性鼓勵開發人員提供其他用途，例如啟動程式和執行命令。 可惜的是，這些新增專案讓通知區域變得太大和雜訊，並與其他桌面存取點混淆。

Windows XP 藉由使區域可折迭並隱藏未使用的圖示，來解決調整問題。 Windows Vista 藉由移除不必要、討厭的通知來解決干擾。 Windows 7 更進一步的原因是，將通知的原始目的放在通知來源。 **在 Windows 7 中，大部分的圖示都是隱藏的，但使用者可以手動升級至通知區域。為了讓使用者能夠掌控其桌面，您的程式無法自動執行這項升級。** Windows 仍會暫時加以升級，以顯示隱藏圖示的通知。

![通知區域和溢位的螢幕擷取畫面 ](images/winenv-notification-image7.png)

在 Windows 7 中，預設會隱藏大部分的通知區域圖示。

此外，Windows 7 也直接在工作列按鈕中支援許多功能。 具體而言，您可以使用：

-   跳躍清單和縮圖工具列可快速存取常用的命令。
-   覆迭圖示，以顯示執行中程式的狀態。
-   用來顯示長時間執行之工作進度的工作列按鈕進度列。

簡單地說，如果您的程式有桌上型電腦存在，請充分利用 Windows 7 工作列按鈕功能來進行這些工作。 **讓通知區域圖示專注于顯示通知和狀態。**

### <a name="keeping-users-in-control"></a>讓使用者保持掌控

讓控制項保持在外，不會正確地使用通知區域。 視圖標的本質而定，您可能會想要讓使用者執行下列動作：

-   **移除圖示。** 您的圖示可能會提供相關的有用狀態，但使用者可能不會想要看到它。 Windows 允許使用者隱藏圖示，但這項功能不容易找到。 若要讓使用者保持控制，請在圖示內容功能表上的 [ **通知區域] 選項中提供顯示圖示** 。 請注意，移除圖示不需要影響基礎程式、功能或流程。
-   **選取要顯示的通知類型。** 您的通知必須有用且相關，但可能會有使用者不想看到的通知。 這特別適用于僅供參考 [通知](mess-notif.md)。 讓使用者選擇啟用較不重要的專案。
-   **暫止選用功能。** 圖示可用來顯示功能的狀態，而不會顯示桌面。 這類功能通常會是長時間執行的選擇性背景工作，例如列印、編制索引、掃描或同步處理。 使用者可能會想要暫停這類功能，以提高系統效能、降低耗電量，或是因為它們已離線。
-   **結束程式。** 提供更適合的選項：
    -   **暫時退出程式。** 重新開機 Windows 時，會停止並重新啟動程式。 這種方法適用于重要的系統公用程式，例如安全性程式。
    -   **永久終止程式。** 重新開機 Windows 時，程式會停止且不會重新開機 (除非使用者選擇稍後重新開機) 。 使用者不想再執行程式，或想要視需要執行程式，也許可以改善系統效能。

雖然最好在圖示的內容功能表上提供大部分的設定，但程式的預設體驗應該適用于大部分的使用者。 預設不會開啟任何專案，而且會預期使用者關閉功能。 相反地，根據預設，將重要功能開啟，讓使用者能夠視需要啟用其他功能。

**如果您只進行四件事 .。。**

1.  請勿濫用通知區域。 您只能將它用於通知和狀態的來源，以及沒有桌上型電腦狀態的功能。
2.  讓使用者保持掌控。 提供適當的選項來控制圖示、其通知和基礎功能。
3.  提供適用于大部分使用者的預設體驗。 讓使用者能夠啟用所需的功能，而不是預期的功能。
4.  充分利用 Windows 7 工作列按鈕功能來顯示狀態，並讓您的程式最常執行的工作更有效率。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **每個元件只提供一個通知區域圖示。**
-   **使用具有16x16、20x20 和24x24 圖元版本的圖示。** 較大的版本會在高 DPI 的顯示模式中使用。

### <a name="when-to-show"></a>顯示時機

-   針對暫時通知來源模式：
    -   Windows 會在顯示通知時顯示圖示。
    -   根據其 [通知設計](mess-notif.md) 模式來移除圖示：



    | 模式                                     | 何時移除                                         |
    |--------------------------------------|------------------------------------------|
    | 動作成功<br/>            | 當通知被移除時。<br/> |
    | 動作失敗<br/>            | 解決問題時。<br/>     |
    | 非重大系統事件<br/> | 解決問題時。<br/>     |
    | 選用的使用者工作<br/>        | 當工作完成時。<br/>            |
    | 僅供參考<br/>                       | 當通知被移除時。<br/> |



 

-   **針對暫存事件狀態模式，在事件發生時顯示圖示。**
-   若為所有其他模式，請在 **程式、功能或進程正在執行時顯示圖示** ，除非使用者已 **在 [通知區域** ] 選項中清除其顯示圖示，否則會顯示圖示 (如需詳細資訊，請參閱 [內容功能表](#context-menus)) 。 在 Windows 7 中，大部分的圖示都是隱藏的，但可以由使用者升級至通知區域。
-   **不要顯示系統管理員對標準使用者的圖示。** 記錄 Windows 事件記錄檔中的資訊。

### <a name="where-to-show"></a>顯示位置

-   **避免在基底圖示中大量純紅色、黃色和綠色。** 為了避免混淆，請保留這些色彩以傳達狀態。 如果您的 [商標](exper-branding.md) 使用這些色彩，請考慮為您的基底通知區域圖示使用靜音。
-   針對 [漸進式擴大](mess-notif.md)， **當情況變得更緊急時，請使用具有逐漸 emphatic 外觀的圖示。**

    ![通知區域和五個圖示的螢幕擷取畫面 ](images/winenv-notification-image22.png)

    在這些範例中，當緊急增加時，電池圖示的外觀會變得更 emphatic。

-   **請勿變更太頻繁的狀態。** 通知區域圖示不應該出現雜訊、不穩定或需要注意。 眼睛對周邊周邊領域的變化很敏感，因此狀態變更必須很微妙。
    -   **請勿快速變更圖示。** 如果基礎狀態快速變更，請讓圖示反映高階狀態。

        **不正確：**

        ![通知區域和數據機圖示的螢幕擷取畫面 ](images/winenv-notification-image23.png)

        在此範例中，數據機圖示會顯示 (硬體數據機) 的閃爍燈，但這些狀態變更對使用者而言並不重要。

    -   **請勿使用長時間執行的動畫來顯示連續的活動。** 這類動畫是個干擾。 通知區域中的圖示存在，足以指出連續活動。
    -   **簡單、微妙的動畫可在重要的暫存、可轉移狀態變更期間顯示進度。**

        ![通知區域和無線圖示的螢幕擷取畫面 ](images/winenv-notification-image24.png)

        在此範例中，無線圖示會顯示活動指示器，以顯示工作正在進行中。

    -   **請勿將圖示閃爍。** 這麼做會造成太雜亂的事情。 如果事件需要立即注意，請改用對話方塊。 如果事件另需要注意，請使用通知。

-   **不要停用通知區域圖示。** 如果目前未套用圖示，請將它移除。 不過，如果使用者可以從圖示啟用，您可以顯示已停用狀態重迭的已啟用圖示。

    ![通知區域和音量滑杆的螢幕擷取畫面 ](images/winenv-notification-image25.png)

    在此範例中，使用者可以啟用圖示的音效輸出。

如需一般圖示指導方針和範例，請參閱 [圖示](vis-icons.md)。

### <a name="interaction"></a>互動

**注意：** 下列 click 事件應該會出現在滑鼠上，而不是滑鼠向下。

**暫留**

-   **顯示指出圖示代表內容的工具提示或提示。**

    ![通知區域和工具提示的螢幕擷取畫面 ](images/winenv-notification-image26.png)

    在此範例中，會使用工具提示來描述停留時的圖示。

如需提示文字的指導方針，請參閱本文的「 [文字](#context-menus) 」一節。

**按一下滑鼠左鍵**

-   **顯示最可能想要看到的任何使用者**，可能是：

    -   具有最實用設定和一般執行工作的飛出視窗、對話方塊或程式視窗。 如需展示指導方針，請參閱 [通知區域 flyouts](#notification-area-flyouts)。

        ![通知區域和 flyouts 的螢幕擷取畫面 ](images/winenv-notification-image27.png)

        在這些範例中，按一下滑鼠左鍵會顯示具有最有用設定的快顯視窗。

    -   狀態飛出視窗。

        ![通知區域和狀態飛出視窗的螢幕擷取畫面 ](images/winenv-notification-image28.png)

        在此範例中，按一下滑鼠左鍵會顯示狀態飛出視窗。

        -   相關的控制台專案。
        -   內容功能表。

    使用者只需要按一下滑鼠，就能顯示某個東西，因此不會顯示任何內容，讓通知區域的圖示沒有回應。

-   **只有當其他選項不適用時，才會顯示內容功能表**，並以粗體顯示預設命令。 在此情況下，會顯示在按一下滑鼠右鍵時所顯示的相同內容功能表，以避免混淆。
-   **偏好在對話方塊中使用快顯視窗** ，以提供更輕量的感覺。 只顯示最常見的設定，並讓它們立即生效，以提供更簡單的互動。 如果使用者按一下視窗外的任何位置，請關閉快顯視窗。
-   **在相關聯的圖示附近顯示小型視窗。** 不過，大型視窗（例如控制台專案）可以顯示在預設監視器的中央。

**按兩下滑鼠左鍵**

-   **在內容功能表上執行預設命令。** 一般而言，這會顯示與圖示相關聯的主要 UI，例如相關聯的 [控制台] 專案、[屬性工作表] 或 [程式] 視窗。
-   **如果沒有預設的命令，請執行與左側單鍵的相同動作。**

**以滑鼠右鍵按一下**

-   **顯示內容功能表**，並以粗體顯示預設命令。

### <a name="notification-area-flyouts"></a>通知區域 flyouts

-   適當時，會顯示具有三個區段的通知區域 flyouts：
    -   **總結。** 顯示出現在圖示工具提示或資訊提示中的相同資訊，可能會有更多詳細資料。 為求一致，請使用相同的文字和圖示，且通常相同的版面配置 (如果使用豐富的工具提示) 。 不同于提示，這項資訊可在使用觸控時存取。
    -   **一般工作。** 直接在飛出視窗中顯示最常執行的工作。
    -   **相關連結。** 請提供下列其中一種選擇性連結：
        -   主控台中最常執行之工作的連結。 提供 [一般工作] 區段中是否有經常執行的工作無法顯示。
        -   相關主控台專案的連結。 此主控台專案應允許使用者執行無法在 [一般工作] 區段中執行的任何工作。
        -   特定相關說明主題的連結。 遵循標準說明 [連結的指導方針](winenv-help.md)。

![通知區域和飛出視窗的螢幕擷取畫面 ](images/winenv-notification-image36.png)

此範例顯示使用建議簡報的通知區域飛出視窗。

### <a name="options-dialog-box"></a>選項對話方塊

-   無法直接從內容功能表存取的選項，必須位於 [選項] 對話方塊中。 此對話方塊可能是功能的 [控制台]。
-   **[選項] 對話方塊可能會在適當的情況下包含下列專案** (確切的文字是以引號括住) ：
    -   啟用 \[ 功能名稱 \] (核取方塊) 
        -   清除此選項會永久結束程式。 程式可以從其控制台專案重新開機。 內容功能表中的 [結束] 命令只會針對目前的 Windows 會話結束程式。
    -   「通知區域中顯示圖示」 (核取方塊) 
        -   從通知區域移除圖示，並不會影響基礎功能。
        -   選取此選項可讓使用者還原圖示，當然無法從圖示本身完成。
-   停用很少使用或可能令人討厭或混亂的功能。 讓使用者 [加入宣告](glossary.md) 這類功能。

如需一般選項對話方塊的指導方針和範例，請參閱 [屬性視窗](win-property-win.md)。

### <a name="minimizing-programs-to-the-notification-area"></a>將程式降至最低通知區域

**注意： Windows 7 不再建議將程式視窗最小化至通知區域。** 請改用一般 [工作列](winenv-taskbar.md) 按鈕。 您的程式可能會支援這兩種機制來提供回溯相容性。

-   若要減少工作列雜亂，請考慮提供將程式最小化到通知區域的功能，只適用于下列所有專案：
    -   程式只能有一個單一實例。
    -   此程式會執行一段很長的時間。
    -   圖示會顯示狀態。
    -   圖示可以是通知來源。
    -   這麼做是選擇性的，使用者必須 [加入宣告](glossary.md)。
-   使用應用程式標題列上的 [最小化] 按鈕，而不是 [關閉] 按鈕。
