# <a name="list-views"></a>列出視圖

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用清單視圖時，使用者可以使用單一選取或多重選取專案來查看資料物件的集合，並與之互動。

![清單視圖與資料行標頭的螢幕擷取畫面 ](images/ctrl-list-views-image1.png)

一般清單視圖。

清單視圖比清單方塊具有更大的彈性和功能。 不同于清單方塊，它們支援變更視圖、群組、具有標題的多個資料行、依資料行排序、變更資料行寬度和順序、拖曳來源或放置目標，以及將資料複製到剪貼簿，以及將資料複製到剪貼簿。

> [!Note]  
> 與 [版面](vis-layout.md) 配置和 [清單方塊](ctrl-list-boxes.md) 相關的指導方針會顯示在不同的文章中。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

清單視圖不只是更有彈性且功能更高的清單方塊：它的額外功能會導致不同的使用方式。 下表顯示比較。



|   使用方式                          | 清單方塊                 | 清單檢視               |
|-----------------------------|-------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| **Data type**<br/>    | 資料和程式選項。<br/> | 僅限資料。<br/>                                                                                                                         |
| **Contents**<br/>     | 僅限標籤。<br/>                   | 標籤和輔助資料，可能在多個資料行中。<br/>                                                                           |
| **互動**<br/>  | 用於進行選取。<br/>    | 可以用來進行選擇，但通常用於顯示資料和與資料互動。 可以是拖曳來源或放置目標。<br/> |
| **展示**<br/> | 固定。<br/>                         | 使用者可以變更視圖、群組、依資料行排序，以及變更資料行的寬度和順序。<br/>                                                |



 

若要判斷這是否為正確的控制項，請考慮下列問題：

-   **清單是否會顯示資料，而不是程式選項？** 如果沒有，請考慮改為使用清單方塊。
-   **使用者是否需要變更視圖、群組、依資料行排序，或變更資料行的寬度和順序？** 如果沒有，請改為使用清單方塊。
-   **控制項必須是拖曳來源或放置目標？** 如果是的話，請使用清單視圖。
-   **是否需要將清單專案複製到剪貼簿或從剪貼簿貼上？** 如果是的話，請使用清單視圖。

### <a name="check-box-list-views"></a>核取方塊清單視圖

-   **控制項是否可用來從資料清單中選擇零個以上的專案？** 若要選擇一個專案，請改為使用單一選項。
-   **這項工作或通常使用的是多重選取專案嗎？** 如果是的話，請使用核取方塊清單視圖讓多個選取明顯，尤其是當您的目標使用者不是 advanced。 如果沒有，請使用標準的多重選取清單查看，如果核取方塊會對多個選取專案造成太多的注意，或造成太多螢幕雜亂。
-   **多重選取專案的穩定性是否重要？** 如果是的話，請使用 [核取方塊清單](ctrl-list-boxes.md)、清單產生器或新增/移除清單，因為按一下一次只變更一個專案。 使用標準的多重選取清單時，很容易就能清除所有選取專案，即使是意外的。

> [!Note]  
> 有時候看起來像清單視圖的控制項是使用清單方塊來執行，反之亦然。 在這種情況下，請根據使用方式套用指導方針，而不是在執行時套用。

 

## <a name="usage-patterns"></a>使用模式

所有視圖都支援單一選取，其中使用者一次只能選取一個專案，而多個選項可讓使用者選取任意數目的專案，包括 none。 清單視圖支援 [延伸選取模式](glossary.md)，也就是透過拖曳或使用 Shift + Click 或 Ctrl + 按一下來選取連續或非相鄰值的群組，來擴充選取範圍。 與清單方塊不同的是，它們不支援 [多重選取模式](glossary.md)，因為不論 Shift 和 Ctrl 鍵為何，按一下任何專案都會切換其選取狀態。

## <a name="guidelines"></a>指導方針

### <a name="presentation"></a>簡報

-   **依邏輯順序排序清單專案。** 依字母順序排序名稱、依數位順序排序數位和日期（以時間順序排列）。
-   **如果有的話，可讓使用者變更排序次序。** 如果清單中有許多專案，或如果有使用預設值以外的排序次序更有效地找到專案，則使用者排序會很重要。
-   **使用 [永遠顯示選取範圍] 屬性** ，讓使用者可以輕鬆地判斷選取的專案，即使控制項沒有焦點也一樣。
-   **避免呈現空白的清單視圖。** 如果使用者建立清單，請使用使用者可能需要的指示或範例專案來初始化清單。

    ![[搜尋] 對話方塊的螢幕擷取畫面，包含指示 ](images/ctrl-list-views-image11.png)

    在此範例中，搜尋清單視圖一開始會顯示指示。

-   **如果使用者可以變更視圖、群組、依資料行排序，或變更資料行及其寬度和順序，請讓這些設定持續存在，使其在下次顯示清單視圖時生效。** 將它們保存在個別清單視圖上（每個使用者為基礎）。

### <a name="interaction"></a>互動

-   **使用按一下滑鼠來選取使用者指向的清單專案。例外狀況：** 在 [命令連結清單] 模式中，按一下 [選取專案]，然後關閉視窗或流覽至下一個頁面。
-   **請考慮提供按兩下行為。** 按兩下應該與選取專案並執行其預設命令的效果相同。
-   **讓按兩下行為重複。** 應該一律會有相同效果的命令按鈕或內容功能表命令。
-   如果清單專案需要進一步說明，請 **在提示中提供說明** [。](ctrl-tooltips-and-infotips.md) 使用完整的句子和結束標點符號。

    ![使用鍵盤提示顯示清單視圖的螢幕擷取畫面 ](images/ctrl-list-views-image12.png)

    在此範例中，會使用資訊提示來提供進一步的資訊。

-   **提供相關命令的內容功能表。** 這類命令包括剪下、複製、貼上、移除或刪除、重新命名和屬性。
-   **如果使用者可以變更排序次序和群組，請依內容功能表提供 [排序依據] 和 [群組]。** 第一次按一下資料行名稱，會依該資料行的遞增順序來排序或群組清單，第二次按一下會依遞減順序排序或群組。 使用先前的 order (從另一個資料行) 作為次要索引鍵。

    ![具有 [排序依據] 功能表的清單視圖螢幕擷取畫面 ](images/ctrl-list-views-image13.png)

    在此範例中，[排序依據] 內容功能表會變更排序次序。 按一下 [名稱一次]，依名稱以遞增順序排序。 再按一次 [名稱] 會以遞減順序排序。

-   **使用鍵盤讓清單檢視列標題可供存取。**
    -   **開發人員：** 您可以藉由設定資料行標頭控制項的焦點來完成這項作業。 這項功能是 Windows Vista 的新功能。
-   **停用清單視圖時，也會停用任何相關聯的標籤和命令按鈕。**
-   **避免水準滾動。** 清單模式使用水準滾動。 此模式通常是最精簡的，但水準滾動通常比垂直捲動更難使用。 如果壓縮度並不重要，請考慮使用小圖示視圖。 但是，如果有許多字母順序的專案，且有足夠的螢幕空間可供廣泛的控制項使用，則清單模式是不錯的選擇。

    **可以接受：**

    ![寬清單模式控制項的螢幕擷取畫面 ](images/ctrl-list-views-image14.png)

    在此範例中，會使用清單模式，因為廣泛控制項有許多專案和大量可用的螢幕空間。

### <a name="multiple-selection-lists"></a>多重選取清單

-   **請考慮在清單下方顯示選取的專案數目**，特別是當使用者可能選取數個專案時。 這項資訊不僅能提供有用的意見反應，也清楚指出清單視圖支援多重選取。

    ![五個所選縮圖清單的螢幕擷取畫面 ](images/ctrl-list-views-image15.png)

    在此範例中，選取的專案數目會顯示在清單下方。

-   或者，您可以提供可能更有意義的其他選取範圍計量，例如選取專案所需的資源，而不是選取的專案數目。

    ![顯示磁碟空間之對話方塊的螢幕擷取畫面 ](images/ctrl-list-views-image16.png)

    在此範例中，安裝元件所需的磁碟空間比選取的元件數目更有意義。

-   針對核取方塊清單視圖，如果有可能有許多專案，以及選取或清除所有的專案，請加入 [全選] 和 [全部清除] 命令按鈕。
-   **使用混合狀態核取方塊來指出部分在容器中選取專案。** 混合狀態不會當做個別專案的第三個狀態使用。

### <a name="changing-views"></a>變更視圖

如果使用者可以變更 views：

-   **依預設，選擇最方便的視圖。** 使用者所做的任何變更，都應該以每個清單的每個使用者為基礎來持續保存。
-   **使用分割按鈕、功能表按鈕或下拉式清單來變更視圖。** 只要可行，請使用工具列上的 [ [分割] 按鈕](ctrl-command-buttons.md) ，然後變更按鈕標籤以反映目前的視圖。

    ![具有分割 [views] 按鈕的清單螢幕擷取畫面 ](images/ctrl-list-views-image17.png)

    在此範例中，工具列上的 [分割] 按鈕可用來變更視圖。

-   **提供 View 內容功能表。**

    ![具有 [view] 內容功能表的 [清單] 螢幕擷取畫面 ](images/ctrl-list-views-image18.png)

在此範例中，會使用視圖內容功能表來變更視圖。

### <a name="details-views"></a>詳細資料檢視

-   **請考慮使用圖格視圖來改善可讀性。**

    **可以接受：**

    ![具有窄欄的詳細資料清單螢幕擷取畫面 ](images/ctrl-list-views-image19.png)

    在此範例中，有太多資料，且視窗、清單和資料行太小，使得清單專案難以讀取。

    **較佳：**

    ![群組中資料的詳細資料清單螢幕擷取畫面 ](images/ctrl-list-views-image20.png)

    在此範例中，磚視圖會顯示資料，而不進行截斷。

-   **選擇最長資料適用的預設資料行寬度。** 清單視圖會自動以省略號截斷長資料，因此，如果預設顯示一些省略號，則資料行寬度是適當的。 雖然使用者可以調整資料行的大小，但偏好使用其他方案：

    -   調整每個資料行寬度以容納其資料。
    -   調整控制項寬度以容納其資料行，再加上任何可能的捲軸。
    -   如有必要，請使用水準滾動。
    -   只有奇數個專案或最後手段有截斷的資料。

    如果預設的一般大小資料必須截斷，請讓視窗和清單視圖變成可調整大小。 包含額外的 30% (高達200% 的文字，以提供任何文字 (較短的文字) ，而不是) 將當地語系化的數位。

    **不正確：**

    ![清單資料行的螢幕擷取畫面，其中包含截斷的資料 ](images/ctrl-list-views-image21.png)

    在此範例中，大部分的資料會被截斷。 許多橢圓形顯然指出控制項和資料行寬度對資料而言太小。

    **不正確：**

    ![具有截斷資料的單一資料行清單螢幕擷取畫面 ](images/ctrl-list-views-image22.png)

    在此範例中，資料會被截斷而沒有原因。

-   **選擇適當的預設資料行順序。** 一般而言，依照下列方式排序資料行：

    -   首先，專案名稱或識別資料。
    -   接下來，其他資料在區分清單專案時很有説明。
    -   接下來，最有用的 (最好的簡短或固定長度) 資料。
    -   接下來， (慣用的簡短或固定長度) 資料較不實用。
    -   Last、long、可變長度的資料。

    Long、variable 長度-資料會放在最後一個資料行中，以減少水準滾動的需求。 在這些類別中，將相關的資訊放在邏輯順序中。

-   **適當時，允許使用者新增和移除資料行，以及變更順序。** 預設會顯示最有用的資料行。 這可透過標頭拖放屬性來達成。
-   選擇適合資料的對齊方式。 使用下列規則：
    -   靠右對齊數位、貨幣和時間。
    -   將文字、識別碼靠左對齊，即使數值) 和日期也 (。
-   針對可排序的資料行標題， **第一次按一下標題會以遞增順序排序資料行的清單，第二次按一下則會以遞減順序排序。** 使用先前的排序次序 (從另一個資料行) 做為次要排序索引鍵。

    ![已排序資料的詳細資料清單螢幕擷取畫面 ](images/ctrl-list-views-image23.png)

    在此範例中，先按一下 [名稱] 資料行，然後按一下 [類型] 資料行。 因此，以遞增順序輸入的是主要排序索引鍵，而名稱的遞增順序則是次要。

-   **使用 Full Row Select 屬性** ，讓使用者可以輕鬆地判斷所有資料行中選取的專案。
-   **除非資料可以排序，否則請勿使用可排序的資料行標題。**
-   **如果只有一個資料行，而且不需要反向排序，請不要使用資料行標頭。** 請改用標籤來識別資料。

    **不正確：**

    **正確：**

    在正確的範例中，會使用標籤，而不是資料行標頭。

## <a name="recommended-sizing-and-spacing"></a>建議的大小和間距

![清單大小和間距的螢幕擷取畫面 ](images/ctrl-list-views-image26.png)

清單視圖的建議大小和間距。

-   **選擇顯示整數專案數目的清單視圖高度。** 避免垂直截斷專案。
-   **選擇可在所有支援的視圖中消除不必要垂直和水準滾動的清單視圖大小。** 清單視圖應顯示3到20個專案。 如果這樣做會消除捲軸，請考慮讓清單視圖稍微放大。 可能有多個專案的清單應該會顯示至少五個專案，藉由一次顯示更多專案，並讓捲軸更容易放置，來加速滾動。
-   **如果使用者受益于使清單視圖更大，請讓清單視圖和其父視窗可調整大小。** 這麼做可讓使用者視需要調整清單視圖大小。 不過，可調整大小的清單視圖應該只顯示三個以上的專案。

## <a name="labels"></a>標籤

### <a name="control-labels"></a>控制項標籤

-   所有清單視圖都需要標籤。 以單字或片語撰寫標籤，而非句子，以使用靜態文字的冒號結尾。
-   為每個標籤指派唯一的 [存取金鑰](glossary.md) 。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
-   使用 [句子樣式的大小寫](glossary.md)。
-   將標籤置於控制項上方，並將標籤與控制項的左邊緣對齊。
-   若為多重選取清單視圖，請撰寫明確指出可能有多重選取的標籤。 核取方塊清單視圖標籤可能較不明確。

    **正確：**

    ![標籤的螢幕擷取畫面：選取一或多個附加元件 ](images/ctrl-list-views-image27.png)

    在此範例中，標籤會清楚指出可能會有多個選取專案。

    **不正確：**

    ![標籤：附加元件的螢幕擷取畫面 ](images/ctrl-list-views-image28.png)

    在此範例中，標籤不會提供多個選取專案的相關資訊。

    **可以接受：**

    ![核取方塊清單標籤的螢幕擷取畫面：附加元件 ](images/ctrl-list-views-image29.png)

    在此範例中，核取方塊會清楚指出可能有多重選取，因此標籤不需要明確。

-   您可以在標籤後面的括弧中，指定單位 (秒、連接等) 。

### <a name="heading-labels"></a>標題標籤

-   讓標題標籤簡短 (三個單字或較少的) 。
-   使用不含結束標點符號的單一名詞或名詞片語。
-   使用 [句子樣式的大小寫](glossary.md)。
-   讓標題的對齊方式與資料相同。

### <a name="group-labels"></a>群組標籤

-   針對高層級的集合，請使用下列群組標籤：
    -   名稱：使用名字或字母範圍的第一個字母。
    -   大小：未指定、0 KB、0-10 KB、10-100 KB、100 KB-1 MB、1-16 MB、16-128 MB
    -   日期：今天、昨天、過去一周、今年之前和很長一段時間。
-   否則，群組標籤會使用要分組之資料的確切文字，包括大小寫和標點符號。

### <a name="data-text"></a>資料文字

-   使用 [句子樣式的大小寫](glossary.md)。

### <a name="instructional-text"></a>教學文字

-   如果您需要新增有關清單視圖的解說文字，請將它加入標籤上方。 使用結尾標點符號的完整句子。
-   使用 [句子樣式的大小寫](glossary.md)。
-   有説明但不需要的其他資訊應該保持簡短。 請將這項資訊放在標籤和冒號之間的括弧內，或在控制項下方的括弧之外。
