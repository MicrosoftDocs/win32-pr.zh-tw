# <a name="tabs"></a>定位點

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

索引標籤可讓您在個別標記的頁面上顯示相關的資訊。

![五個索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image1.png)

一組一般的索引標籤。

索引標籤通常會與屬性視窗 (相關聯，反之亦然) ，不過，您可以在任何類型的視窗中使用 tab 鍵。

索引標籤控制項代表索引標籤式馬尼拉資料夾，用來組織通常位於美國的檔櫃中的資訊。  (不會在全球使用的馬尼拉資料夾。 ) 

> [!Note]  
> 與 [版面](vis-layout.md)配置、索引標籤 [功能表](cmd-menus.md)、 [對話方塊](win-dialog-box.md)和 [屬性視窗](win-property-win.md) 相關的指導方針會顯示在不同的文章中。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **控制項可以在單一、合理大小的頁面上輕鬆容納嗎？** 若是如此，請使用單一頁面。
-   **是否只有一個索引標籤？** 若是如此，請使用單一頁面。
-   **這些索引標籤是否以某種方式彼此相關？** 如果沒有，請考慮將資訊分割成個別的相關資訊視窗。
-   **如果用於設定，不同頁面上的設定是否完全獨立？** 變更一個頁面上的設定會影響其他頁面上的設定嗎？ 如果它們不是獨立的，請改用工作頁面或 [wizard](win-wizards.md) 。
-   **索引標籤大多對等，還是有階層式關聯性？** 如果有階層，請考慮使用漸進式洩漏或子 [對話方塊](win-dialog-box.md) 來顯示相關資訊。
-   **索引標籤是用來顯示工作內的步驟嗎？** 您可以使用「索引標籤」來顯示工作中的步驟（如果這些步驟的外觀類似步驟），而且有明顯的替代方式可進入文字步驟，例如 [下一步] 按鈕。 否則，如果需要這些步驟，請使用頁面流程或 [wizard](win-wizards.md)中的頁面。 如果步驟是選擇性的，請改為使用模式 [對話方塊](win-dialog-box.md) 來顯示選擇性的步驟。
-   **索引標籤是否會有相同資料的不同觀點？** 如果是的話，請考慮使用 [分割按鈕](ctrl-command-buttons.md) 或 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) 來變更 views。 雖然您可以有效地使用索引標籤來變更視圖，但替代方案更輕量。

## <a name="usage-patterns"></a>使用模式

索引標籤有數種使用模式：



|                                                                                                                                                                                                         |                                                                                                                                                                                                                                                                                                                                                                     |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **動態視窗介面**<br/> 和捲軸一樣，索引標籤可以用來增加視窗介面區，以顯示相關的資訊。<br/>                                                    | 使用這個模式時，使用索引標籤的概念類似于在單一可滾動表面上以線性方式將所有資訊放在索引標籤上，並以索引標籤標籤作為標題。 <br/> ![五個索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image1.png)<br/> 在此範例中，索引標籤會有效地增加視窗介面區域。<br/>                          |
| **多重檢視**<br/> 就像分割按鈕或下拉式清單一樣，索引標籤可用來顯示相同或相關資訊的不同視圖。 <br/>                                           | ![[設計]、[分割] 和 [預覽] 索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image2.png)<br/> 在此範例中，索引標籤會變更檔內的視圖。<br/>                                                                                                                                                                                                         |
| **多份檔**<br/> 索引標籤和多個視窗一樣，可以用來在單一視窗中顯示不同的檔。 <br/>                                                                   | ![不同檔的三個索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image3.png)<br/> ![不同月份的索引標籤圖 ](images/ctrl-tabs-image4.png)<br/> 在這些範例中，索引標籤會在單一應用程式視窗中顯示不同的檔。<br/>                                                                                       |
| **專屬選項**<br/> 索引標籤和選項按鈕一樣，可以用來呈現多個專屬選項。 在此模式中，只會套用選取的索引標籤，並忽略所有其他索引標籤。 <br/> | ![[位置]、[資料] 和 [訊息] 索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image5.png)<br/> 在此範例中，會使用索引標籤 (錯誤地) 做為換用的選項按鈕。<br/> **不建議使用此模式，** 因為它使用非標準行為。 索引標籤的行為就像是一個設定，而不是單純地在視窗內流覽的方法。 <br/> |



 

**如果您只做一件事 .。。**

請確定索引標籤上的資訊是相關的，但不同頁面上的設定都是獨立的。 選取的最後一個索引標籤應該沒有特殊意義。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **使用水準索引標籤的情況：**
    -   此視窗有七個或更少的索引標籤。
    -   **所有索引標籤都能容納在一個資料列上，即使使用者介面 (UI) 已當地語系化也是一樣。**
-   **使用垂直索引標籤的情況：**
    -   屬性視窗有八個以上的索引標籤。
    -   **使用水準索引標籤需要一個以上的資料列。**

        ![具有11個選項之屬性視窗的螢幕擷取畫面 ](images/ctrl-tabs-image6.png)

        在此範例中，垂直索引標籤能容納八個以上的索引標籤。

-   **請勿使用垂直索引標籤來嵌套索引標籤或合併水準索引標籤。** 相反地，請減少索引標籤數目、僅使用垂直索引標籤，或使用另一個控制項（例如下拉式清單）。
-   **不要滾動水準索引標籤。** 無法立即探索水準滾動。 不過，您可以滾動垂直索引標籤。

    **不正確：**

    ![截斷的水準索引標籤標籤的螢幕擷取畫面 ](images/ctrl-tabs-image7.png)

    在此範例中，會滾動水準索引標籤。

-   針對可調整大小的視窗或窗格上的索引標籤，視需要在頁面上（而非視窗或窗格）放置捲軸。 索引標籤應該一律是可見的，而不是從視野中取出。

    ![具有捲軸之垂直索引標籤頁面的螢幕擷取畫面 ](images/ctrl-tabs-image8.png)

    在此範例中，索引標籤頁面具有捲軸，而非窗格。

-   **確認索引標籤看起來像是索引標籤，而不是另一種類型的控制項。**

    **不正確：**

    ![具有索引標籤按鈕之視窗的螢幕擷取畫面 ](images/ctrl-tabs-image9.png)

    在此範例中，這些索引標籤看起來像命令按鈕。

### <a name="interaction"></a>互動

-   **當控制項只套用至頁面時，請將它們放在索引標籤式頁面的框線內。**
-   **當控制項套用至整個視窗時，請將它們放在索引標籤式頁面之外。**
-   **請勿指派變更索引標籤的效果。** 您必須以任何順序存取索引標籤。 變更目前的索引標籤絕不會有副作用、套用設定或產生錯誤訊息。
-   **請勿在選取的最後一個索引標籤上指派特殊意義。** 索引標籤選取用於流覽使用者的最後一個索引標籤選取專案不是設定。
-   **請勿讓頁面上的設定相依于其他頁面上的設定。** 請改為將任何相依的設定放在相同的頁面上。
-   **如果使用者可能會從顯示的最後一個索引標籤開始，請讓索引標籤保持不變，並依預設選取它。** 讓設定以每個視窗、每個使用者為基礎來保存。 否則，預設會選取第一個頁面。

### <a name="icons"></a>圖示

-   **請勿將圖示放在索引標籤上。** 圖示通常會加入不必要的視覺效果雜亂、取用螢幕空間，而且通常不會改善使用者理解。 只新增可協助理解的圖示，例如標準符號。

    **不正確：**

    ![索引標籤上有圖示之視窗的螢幕擷取畫面 ](images/ctrl-tabs-image10.png)

    在此範例中，圖示會新增視覺效果，而不會大幅改善使用者理解。

    **例外狀況：** 如果沒有足夠的空間可顯示有意義的標籤，您可以使用明顯可辨識的圖示：

    **正確：**

    ![具有圖示和縮減標籤之索引標籤的螢幕擷取畫面 ](images/ctrl-tabs-image11.png)

    在此範例中，視窗很窄，因此圖示會比標籤更妥善地傳達索引標籤。

-   **請勿使用標籤圖形的產品標誌。** 索引標籤不適用於 [商標](exper-branding.md)。

### <a name="dynamic-window-surface-pattern"></a>動態視窗介面模式

-   **請勿在索引標籤頁上使用捲軸。** 索引標籤的功能類似于捲軸，以增加視窗的有效區域。 其中一個機制應該就已足夠。
-   **使用簡潔的索引標籤標籤。** 使用可清楚描述頁面內容的一或兩個字組。 較長的標籤會耗用螢幕空間，特別是當標籤已當地語系化時。
-   **使用有意義的特定索引標籤標籤。** 避免可套用至任何索引標籤的一般索引標籤標籤，例如 [一般]、[Advanced] 或 [設定]。
-   **如果索引標籤不會套用至目前的內容，而且使用者不會預期它，請將它移除。** 這麼做可簡化 UI，且使用者不會錯過。

    **不正確：**

    ![選項視窗的螢幕擷取畫面，索引標籤名稱為暗灰色 ](images/ctrl-tabs-image12.png)

    在此範例中，使用 Microsoft Word 作為電子郵件編輯器時，會不正確地停用 [檔案位置] 索引標籤。 由於使用者不會預期會在此內容中查看或變更檔案位置，而不是停用此索引標籤。

-   **如果索引標籤不適用於目前的內容，使用者可能會預期：**

    -   顯示索引標籤。
    -   停用頁面上的控制項。
    -   包含說明控制項停用原因的文字。

    請勿停用此索引標籤，因為這樣做並不容易理解，且禁止探索。 尋找特定值的使用者會被迫查看所有其他索引標籤。

    ![視窗的螢幕擷取畫面，其中的 [視圖] 索引標籤選項為暗灰色 ](images/ctrl-tabs-image13.png)

    在此範例中，沒有任何 View 選項適用于讀取配置。 不過，使用者可能會預期他們會根據索引標籤標籤來套用，因此會顯示頁面，但選項會停用。

### <a name="multiple-views-and-documents-patterns"></a>多個視圖和檔案模式

-   **使用索引標籤標籤上的視圖或檔案名稱。**
-   **避免索引標籤名稱過長。** 如有必要，請使用省略號來取得最大名稱大小或截斷顯示的索引標籤標籤。 較長的標籤會耗用螢幕空間，特別是當標籤已當地語系化時。
-   **如果索引標籤不適用於目前的內容，請移除索引標籤。**

### <a name="exclusive-options-pattern"></a>專屬選項模式

-   **請勿使用此模式！** 改為使用選項按鈕或下拉式清單。

    **不正確：**

    ![具有兩個 [建立] 索引標籤之視窗的螢幕擷取畫面 ](images/ctrl-tabs-image14.png)

    在此範例中，索引標籤不正確地用來取代選項按鈕。

    **正確：**

    ![有兩個選項按鈕之視窗的螢幕擷取畫面 ](images/ctrl-tabs-image15.png)

    在此範例中，會正確地使用選項按鈕。

## <a name="labels"></a>標籤

-   根據其模式來標籤索引標籤。 使用名詞而不是動詞，而不需要結束標點符號。 如需詳細資訊，請參閱上述模式指導方針。
-   使用句型大寫。
-   請勿指派存取金鑰。 索引標籤可透過其快速鍵存取 (Ctrl + Tab、Ctrl + Shift + Tab、Ctrl + PgUp、Ctrl + >pgdn) 。 有個很好的存取金鑰選擇，因此不會將存取金鑰指派給索引標籤，可讓您更輕鬆地將它們指派給其他控制項。

## <a name="documentation"></a>文件

參考索引標籤時：

-   使用確切的標籤文字（包含其大小寫），並包含 [文字] 索引標籤。
-   若要描述使用者互動，請使用按一下。
-   可能的話，請使用粗體文字來格式化標籤。 否則，請只在必要時才將標籤放在引號中，以避免混淆。
-   由於多個用途可能不明確，特別是針對全球的物件，請單獨使用名詞索引標籤來參考索引標籤控制項。 若為其他用途，請使用描述項來明確說明意義： Tab 鍵、索引標籤停止或尺規上的定位字元標記。

範例：在 [ **工具** ] 功能表上，按一下 [ **選項**]，然後按一下 [ **流覽** ] 索引標籤。
