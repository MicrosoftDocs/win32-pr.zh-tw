# <a name="mouse-and-pointers"></a>滑鼠和指標

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

滑鼠是用來與 Windows 中的物件互動的主要輸入裝置。 滑鼠功能也可以包含其他指標裝置，例如 trackballs、touchpads 和內建于筆記本電腦的手指、搭配 Windows Tablet 及觸控技術使用的畫筆，以及在具有 touchscreens 的電腦上，甚至是使用者的手指。

> [!NOTE]
> [協助工具](inter-accessibility.md)、[畫筆](inter-pen.md)和[觸控](inter-touch.md)的相關指導方針會顯示在不同的文章中。

實際移動滑鼠會移動圖形指標 (也稱為畫面上的游標) 。 指標有各種不同的形狀，以表示其目前的行為。

![五個一般滑鼠指標的螢幕擷取畫面 ](images/inter-mouse-image1.png)

*一般滑鼠指標*

滑鼠裝置通常會有一個主要按鈕 (通常是左按鈕) 、次要按鈕 (通常是右) ，以及兩者之間的滑鼠滾輪。 藉由定位指標並按一下滑鼠上的主要和次要按鈕，使用者可以選取物件並對其執行動作。 針對大部分的互動，當游標位於目標上時按下滑鼠按鍵表示選取的目標，而放開按鈕則會執行與目標相關聯的任何動作。

除了忙碌指標以外的所有指標，都有一個定義滑鼠確切螢幕位置的單一圖元作用點。 作用點決定受滑鼠動作影響的物件。 物件會定義熱區域，也就是將作用點視為物件上方的區域。 通常，熱區域會與物件的框線一致，但可能會更大，讓使用者的意圖更容易執行。

插入號是當使用者在文字方塊或其他文字編輯器中輸入時，所顯示的閃爍垂直列。 根據預設，插入號與指標無關 (，Windows 在使用者輸入) 時隱藏指標。

![具有游標的文字方塊螢幕擷取畫面 ](images/inter-mouse-image2.png)

*插入號*

## <a name="design-concepts"></a>設計概念

### <a name="the-mouse-is-intuitive"></a>滑鼠是直覺的

因為滑鼠很容易用於一般的人，所以已成功輸入裝置。 以指標為基礎的互動已經成功，因為它是直覺的，可提供豐富的各種體驗。

**設計完善的使用者介面 (UI) 物件稱為 affordance，也就是物件的視覺效果和行為屬性，建議其使用方式。** 指標可作為手形的 proxy，讓使用者與螢幕物件互動，就像是使用實體物件一樣。 我們的人類對人類手的運作方式有固有的瞭解，因此，如果看起來像是可以推送的，我們會嘗試推送它;如果看起來像是可抓取的，我們會嘗試抓取它。 因此，使用者可以藉由查看物件並加以試用，來找出如何使用具有強式 affordance 的物件。

![按鈕和滑杆的螢幕擷取畫面](images/inter-mouse-image3.png)

*按鈕和滑杆有強式 affordance*

相較之下，affordance 不佳的物件很難找出。這類物件通常需要標籤或指示來說明它們。

![連結文字和網際網路地球圖示的螢幕擷取畫面](images/inter-mouse-image4.png)

*連結文字和圖示的 affordance 不良*

### <a name="some-aspects-of-mouse-use-are-not-intuitive"></a>滑鼠使用的某些層面並非直覺

**以滑鼠右鍵按一下、按兩下，然後按一下 Shift 或 Ctrl 鍵修飾詞，就會有三個非直覺的滑鼠互動**，因為它們沒有真正的對應專案。 與鍵盤快速鍵和存取金鑰不同的是，這些滑鼠互動通常不會記錄在 UI 的任何位置。 這表示以滑鼠右鍵按一下、按兩下，以及不需要鍵盤修飾詞來執行基本工作，特別是當使用者使用。 此外，也建議這些先進的互動必須有一致且可預測的行為，才能有效地使用。

### <a name="single-click-or-double-click"></a>按一下或按兩下？

按兩下會在 Windows 桌面上很廣泛地使用，這似乎不像是先進的互動。 例如，只要按兩下，就可以在 Windows 檔案總管的 [檔案] 窗格中開啟 [資料夾]、[程式] 或 [檔]。 開啟 Windows 桌面上的快捷方式時，也會使用按兩下。 相反地，在 [開始] 功能表中開啟資料夾或程式只需要按一下即可。

可選取的物件會使用單鍵來執行選取專案，因此需要按兩下以開啟，而不可選取的物件只需要按一下即可開啟。 許多使用者都無法理解這項區別 (按一下程式圖示，然後按一下程式圖示，就可以 ) ，如此一來，某些使用者就可以一直按一下圖示，直到他們取得所要的內容為止。

### <a name="direct-manipulation"></a>直接操作

直接與物件互動稱為直接操作。 指向、按一下、選取、移動、調整大小、分割、滾動、移動和縮放都是常見的直接操作。 相反地，透過物件的 [屬性] 視窗或其他對話方塊來與物件互動，可能會被描述為間接操作。

**不過，在有直接操作的情況下，可能會有意外的操作，因此需要容許。** 容許可讓您輕鬆地反向或更正不想要的動作。 您可以藉由提供復原、提供良好的視覺反應，並讓使用者輕鬆地更正錯誤，來進行直接操作容許。 與容許相關聯，會防止在一開始發生不想要的動作，您可以使用限制的控制項，並確認具有非預期結果的具風險動作或命令。

### <a name="standard-mouse-button-interactions"></a>標準滑鼠按鍵互動

標準的滑鼠互動取決於各種不同的因素，包括按下滑鼠按鍵、按一下滑鼠按鍵的次數、在點擊時的位置，以及是否已按下任何鍵盤修飾詞。 以下摘要說明這些因素通常會如何影響互動：

- 針對大部分的物件，以滑鼠左鍵按一下滑鼠左鍵，然後執行預設命令。 在內容功能表中會識別預設命令。
- 針對某些類型的可選取物件，每次按一下會展開按一下的效果。 例如，在文字方塊中按一下，即可設定輸入位置、按兩下以選取一個單字，然後按三下滑鼠選取一個句子或段落。
- 以滑鼠右鍵按一下會顯示物件的內容功能表。
- 將滑鼠停留在將結果指向滑鼠停留時，仍保持老鼠。
- 在按下滑鼠按鍵時仍保持滑鼠，表示按一下和選取單一物件。 移動滑鼠表示移動、調整大小、分割、拖曳，以及選取多個物件。
- Shift 鍵會連續擴充選取範圍。
- Ctrl 鍵會藉由切換已按一下專案的選取狀態，而不會影響其他物件的選取範圍，來擴充選取範圍。

### <a name="hand-pointers"></a>手指標

文字和圖形連結使用手邊或「連結選取」指標 (具有食指指標的手 ![具有食指指標的手螢幕擷取畫面 ](images/inter-mouse-image6.png)) ，因為它們的弱 affordance。 雖然連結可能會有其他的視覺線索，以表示它們是連結 (例如底線和特殊位置) ，但在滑鼠停留時顯示手形指標是指連結的明確指示。

**為了避免混淆，請務必不要使用手形指標進行其他用途。** 例如，命令按鈕已經有強式 affordance，因此不需要手形指標。 手形指標必須表示「此目標是連結」，而不是其他任何專案。

### <a name="fitts-law"></a>Fitts 的法律

Fitts 的定律是圖形化使用者介面設計體認中的知名準則，基本上是指：

- 目標愈遠，以滑鼠取得它的時間越長。
- 目標越小，以滑鼠取得它的時間越長。

因此，大型目標很好用。 請務必讓整個目的地區域都可供點擊。

| 不正確 | 正確 (可以按一下整個目標)  |
|:---|:---|
| ![只有可按標籤的圖示螢幕擷取畫面 ](images/inter-mouse-image24.png) | ![可按式圖示和可按式標籤的螢幕擷取畫面 ](images/inter-mouse-image25.png) |

您可以在指向以使其更容易取得時，動態變更目標的大小。

![字元對應的螢幕擷取畫面，其中包含放大的數位 ](images/inter-mouse-image26.png)

*當使用者指向使其更容易取得時，目標會變得更大*

而關閉目標也很好。 找出可點擊的專案，靠近最可能使用的位置。 在下圖中，色彩選擇區離工具選取器太遠。

![色彩選擇區的螢幕擷取畫面（以工具分隔） ](images/inter-mouse-image27.png)

*色彩選擇區太遠無法從可能使用的位置*

請考慮使用者的目前指標位置與目標相同，因此很容易取得。 因此，操作功能表可充分利用 Fitts 的定律，就像 Microsoft Office 使用的迷你工具列一樣。

![靠近下拉式清單的指標螢幕擷取畫面 ](images/inter-mouse-image28.png)

*目前的指標位置一律是最容易取得的位置*

此外，在決定物件大小時，請考慮替代的輸入裝置。 例如，針對觸控建議的最小目標大小為23x23 圖元， (13x13 Dlu) 。

### <a name="environments-without-a-mouse"></a>沒有滑鼠的環境

並非所有 Windows 環境都有滑鼠。 例如，kiosk 很少會有滑鼠，且通常會有觸控式螢幕。 這表示使用者可以執行簡單的互動，例如按一下滑鼠左鍵，或許也可以拖放。 不過，它們不能停留、按一下滑鼠右鍵，或按兩下。 這種情況很容易設計，因為通常會事先知道這些限制。

使用滑鼠需要良好的馬達技巧，因此並非所有使用者都可以使用滑鼠。 若要讓您的軟體可供最廣泛的物件存取，請務必使用鍵盤來執行良好的馬達技能不重要的所有互動。

如需詳細資訊和指導方針，請參閱 [協助工具](inter-accessibility.md)。

**如果您只進行四件事 .。。**

1.  在適當的情況下使用標準指標，讓滑鼠互動行為與標準效果一致。
2.  限制先進的滑鼠互動 (需要按下滑鼠右鍵、多次點擊或輔助按鍵的) ，以 advanced users 為目標的 advanced task。
3.  指派 advanced 老鼠互動一致且可預測的行為，讓它們可以有效地使用。
4.  請確定您的程式能夠反轉或更正任何不想要的動作，特別是針對破壞性命令。 使用直接操作時，可能會有意外的動作。

## <a name="guidelines"></a>指導方針

### <a name="click-affordance"></a>按一下 [affordance]

- **永遠不需要使用者按一下物件，以判斷是否可按一下物件。** 使用者必須能夠單獨判斷 clickability 的視覺化檢查。
  - 主要 UI (（例如認可按鈕）) 必須有靜態按一下 affordance。 使用者不應該停留以探索主要 UI。
  - 次要 UI (（例如次要命令或漸進式洩漏控制項）) 可以在滑鼠停留時顯示其 click affordance。
  - [文字連結](ctrl-links.md) 應靜態地建議連結文字，然後在滑鼠停留時顯示其 click affordance (底線或其他 [簡報) 變更](#hand-pointers) 。
  - [圖形連結](ctrl-links.md) 只會在滑鼠停留時顯示手形指標。
- **使用手形 (或 [連結選取] ) 指標僅適用于文字和圖形連結。** 否則，使用者必須按一下物件，以判斷它們是否為連結。

### <a name="mouse-interaction"></a>滑鼠互動

- **請按一下 [目標] 至少為16x16 圖元，讓任何輸入裝置都能輕鬆地按一下。** 針對 [觸控](inter-touch.md)，建議的最小控制項大小為23x23 圖元 (13x13 dlu) 。 當使用者指向使其更容易取得時，請考慮動態變更小型目標的大小。

    在此範例中，微調控制項按鈕太小，無法有效使用觸控或畫筆。

    ![具有小箭號之微調控制項的螢幕擷取畫面 ](images/inter-mouse-image29.png) 

- **將分隔器設為至少五個圖元，讓任何輸入裝置都能輕鬆地按一下。** 當使用者指向使其更容易取得時，請考慮動態變更小型目標的大小。

    在此範例中，Windows 檔案總管流覽窗格中的分隔器太窄，無法有效使用滑鼠或畫筆。

    ![窄、近乎隱藏的分隔器螢幕擷取畫面 ](images/inter-mouse-image30.png)

- **為使用者提供錯誤空間的邊界。** 允許進行某些滑鼠移動 (例如，當使用者放開滑鼠按鍵時，會) 三個圖元。 當使用者放開滑鼠按鍵時，有時會稍微移動滑鼠，如此一來，在按鈕發行之前的滑鼠位置越好，就會反映使用者的意圖，而不是緊接在之後的位置。
- **提供使用者錯誤暫時的邊界。** 使用系統的按兩下速度來區別單一和按兩下。
- **按下滑鼠按鍵會影響滑鼠按鍵。** 在放開滑鼠按鍵之前，允許使用者從有效的目標中移除滑鼠，以放棄滑鼠動作。 針對大部分的滑鼠互動，按下滑鼠按鍵只會指出選取的目標，而放開按鈕則會啟動該動作。 自動重複的函式 (例如按下滾動箭號，以持續滾動) 是例外狀況。
- [捕捉滑鼠](/windows/win32/api/winuser/nf-winuser-setcapture) 來選取、移動、調整大小、分割和拖曳。
- 使用 Esc 鍵可讓使用者放棄複合滑鼠互動，例如移動、調整大小、分割和拖曳。
- **如果物件不支援按兩下，但使用者可能會假設它的確如此，只要按一下就可解讀「按兩下」。** 假設使用者想要使用單一動作，而不是兩個。

    由於使用者可能會假設工作列按鈕支援按兩下，因此只要按一下，就應該處理一次「按兩下」。

    ![工作列按鈕和標準指標的螢幕擷取畫面 ](images/inter-mouse-image31.png)

- **當您的程式處於非使用中狀態時，忽略重複的滑鼠點擊。** 例如，如果使用者在程式處於非使用中狀態時，按一下按鈕10次，請將它視為按一下。
- **請勿使用雙拖曳或 chords。** 雙拖曳是使用按兩下生效的拖曳動作，而弦則是同步選取多個滑鼠按鍵時。 這些互動並非標準、無法探索、很難執行，而且很有可能會不小心執行。
- **請勿使用 Alt 作為滑鼠互動的修飾詞。** ALT 鍵會保留給工具列存取和存取金鑰。
- **請勿使用 Shift + Ctrl 作為滑鼠互動的修飾詞。** 這麼做會太難使用。
- **將滑鼠暫留為多餘。** 為了讓您的程式可觸式，請充分利用 [暫止]，但只以執行動作所需的方式來執行。 這通常表示動作也可以藉由按一下來執行，但不一定是以完全相同的方式執行。 大部分的觸控技術都不支援暫止，所以具有這類 touchscreens 的使用者無法執行任何需要停留的工作。

### <a name="mouse-wheel"></a>滑鼠滾輪

- **讓滑鼠滾輪影響目前指標的控制項、窗格或視窗。** 這樣做可避免非預期的結果。
- **讓滑鼠滾輪在沒有按一下或有輸入焦點的情況下生效。** 滑鼠暫留已足夠。
- **讓滑鼠滾輪影響到具有最特定範圍的物件。** 例如，如果指標位於可滾動視窗內可滾動窗格內的可滾動清單方塊控制項上方，則滑鼠滾輪會影響清單方塊控制項。
- **使用滑鼠滾輪時，請勿變更輸入焦點。**
- 讓滑鼠滾輪具有下列效果：
  - 針對可滾動的視窗、窗格和控制項：
    - **旋轉滑鼠滾輪會垂直捲動物件，也就是向上旋轉。** 為了讓滾輪具有自然對應，旋轉滑鼠滾輪應該永遠不會水準滾動，因為這樣做會 disorienting 且非預期的。
      - **如果按下 Ctrl 鍵，旋轉滑鼠滾輪會縮放物件，** 其中旋轉放大和旋轉縮小。
      - **傾斜滑鼠滾輪會水準滾動物件。**
  - 若要可不含捲軸的視窗和窗格 () ：
    - **旋轉滑鼠滾輪可縮放物件，** 其中旋轉放大和旋轉縮小。
    - 傾斜滑鼠滾輪沒有任何作用。
  - 針對索引標籤：
    - 無論索引標籤的方向為何 **，旋轉滑鼠滾輪都可以變更目前的** 索引標籤。
    - 傾斜滑鼠滾輪沒有任何作用。
  - 如果按下 Shift 和 ALT 鍵，則滑鼠滾輪沒有任何作用。
- **使用垂直捲動條大小的 Windows 系統設定 (來旋轉) ，並 (傾斜) 的水準捲軸大小。** 這些設定可透過滑鼠控制台專案來設定。
- **將滑鼠滾輪旋轉得更快速，讓滾動更快。** 這麼做可讓使用者更有效率地滾動大型檔。
- **針對可滾動的視窗，請考慮按一下滑鼠滾輪按鈕，將視窗置於「讀取器模式」。** 讀取器模式會植物特殊的滾動來源圖示，並以相對於捲軸原點的方向和速度來滾動視窗。

![頁面的螢幕擷取畫面，具有捲軸圖示 ](images/inter-mouse-image32.png)

*Internet Explorer 支援讀取器模式，其功能為滾動來源圖示*

### <a name="hiding-the-pointer"></a>隱藏指標

- **請勿隱藏指標。** 例外狀況：
  - 以全螢幕呈現模式執行的展示應用程式可能會隱藏指標。 不過，當使用者移動滑鼠時，指標必須立即還原，而且可以在閒置兩秒鐘後 rehidden。
  - 沒有滑鼠 (的環境（例如 kiosk) ）可以永久隱藏指標。
- 根據預設，當使用者在文字方塊中輸入時，Windows 隱藏指標。 此 Windows 系統設定可透過滑鼠控制台專案來設定。

### <a name="activity-pointers"></a>活動指標

Windows 中的活動指標是忙碌指標 (![環圈圖指標的螢幕擷取畫面 ](images/inter-mouse-image33.png)) 以及在背景指標中工作 (![環圈圖指標和箭號的螢幕擷取畫面 ](images/inter-mouse-image34.png)).

- 當使用者必須等待一秒以上才能完成動作時，顯示忙碌指標。 請注意，忙碌指標沒有作用點，所以使用者在顯示時無法按一下任何位置。
- 當使用者必須等待一秒以上才能完成動作，但是程式有回應，而且沒有任何其他視覺效果回饋表示動作未完成時，就會顯示在背景指標中工作。
- 請勿合併活動指標與進度列或進度動畫。

### <a name="caret"></a>插入點

- **在文字輸入視窗或控制項具有輸入焦點之前，請勿顯示插入號。** 插入號可將輸入焦點建議給使用者，但視窗或控制項可以顯示沒有輸入焦點的插入號。 當然，請勿遭竊輸入焦點，如此一來，超出內容的對話方塊就可以顯示插入號。

    Windows 的認證管理員會顯示在插入號的內容之外，但不含輸入焦點。 如此一來，使用者最後會在非預期的地方輸入密碼。

    ![認證管理員沒有焦點的螢幕擷取畫面 ](images/inter-mouse-image35.png)

- **將插入號放在使用者最可能先輸入的位置。** 這通常是使用者輸入或文字結尾的最後一個位置。

### <a name="accessibility"></a>協助工具選項

- 對於根本無法使用滑鼠的使用者，請使用鍵盤來使滑鼠變成多餘的。
  - 使用者應該可以使用滑鼠來執行所有動作，除了微調技術的動作（例如繪圖和遊戲播放）之外。
  - 使用者應該能夠使用鍵盤來進行所有動作，但有效率的文字輸入除外。
- 對於使用滑鼠的功能有限的使用者：
  - 請不要按兩下並拖曳唯一的方法來執行動作。

如需詳細資訊和指導方針，請參閱 [協助工具](inter-accessibility.md)。

## <a name="documentation"></a>文件

參考滑鼠時：

- 避免使用複數滑鼠;如果您需要參考一個以上的滑鼠，請使用滑鼠裝置。
- 使用滑鼠按鍵來指出滑鼠左鍵。 不要使用主要滑鼠按鍵。 同樣地，使用滑鼠右鍵，而不是次要滑鼠按鍵。 無論正確程度為何，使用者都能瞭解這些詞彙以及 reprogram 其按鈕的使用者，以產生精神的變化。
- 使用滾輪作為滑鼠滾輪的旋轉部分，並使用滾輪按鈕來參考可點按的部分。
- 使用如 click、point 和拖曳等動詞命令來參考滑鼠動作。 使用者會垂直旋轉滾輪、水準傾斜，然後按一下滾輪按鈕。
- 針對移動檔或資料夾的動作，使用 [拖曳]，而非 [拖放]。 您可以使用拖放功能做為形容詞，如同「移動資料夾為拖放作業」。
- 一律將按兩下和以滑鼠右鍵按一下作為動詞。
- 使用 [按一下]，不要按一下 [開啟]。 在 (中按一下 [在視窗中按一下] ) 是可接受的。

參考滑鼠指標時：

- 請將滑鼠指標稱為指標。 請只在技術檔中使用資料指標。
- 若為具有活動指標的指標，請針對僅由活動指標所組成的指標使用忙碌指標，然後針對組合指標和活動指標使用背景指標。
- 針對其他類型的指標，請不要使用描述性標籤來參考指標。 如有必要，請使用圖形來描述滑鼠指標如何出現在螢幕上。

**範例：**

- 指向 [視窗框線]。
- 使用滑鼠按一下 [ **最小化** ] 按鈕。
- 按住 Shift 鍵並按一下滑鼠右鍵。
- 當指標變成 ![有兩個 crossbars 的箭號螢幕擷取畫面](images/inter-mouse-image18.png)，拖曳指標以移動分割線。
