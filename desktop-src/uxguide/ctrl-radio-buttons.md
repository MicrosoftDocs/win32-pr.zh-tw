# <a name="radio-buttons"></a>選項按鈕

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用選項按鈕時，使用者可以在一組互斥、相關的選項之間做選擇。 使用者只能選擇一個選項。 因為選項按鈕的運作方式就像是無線電上的通道預設，所以會呼叫它們。

![三個選項按鈕的螢幕擷取畫面 ](images/radio-buttons-image1.png)

選項按鈕的一般群組。

一組選項按鈕的行為就像單一控制項一樣。 使用 Tab 鍵只能存取選取的選項，但是使用者可以使用方向鍵來迴圈流覽群組。

> [!Note]  
> 與 [版面](vis-layout.md) 配置和 [鍵盤導覽](inter-keyboard.md) 相關的指導方針會在個別的文章中顯示。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **控制項是否可用來從一組互斥的選項中選擇一個選項？** 如果不是，請使用其他控制項。 若要選擇多個選項，請改用 [核取方塊](ctrl-check-boxes.md)、 [多重選取清單](ctrl-list-boxes.md) 或核取方塊清單。
-   **選項的數目是介於2到7之間？** 由於所使用的螢幕空間與選項數目成正比，因此請將群組中的選項數目維持在二到七的群組中。 若有八個以上的選項，請使用 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) 或 [單一選取清單](ctrl-list-boxes.md)。
-   **核取方塊是較佳的選擇嗎？** 如果只有兩個選項，您可以改為使用單一 [核取方塊](ctrl-check-boxes.md) 。 不過，核取方塊僅適用于開啟或關閉單一選項，而選項按鈕可用於完全不同的替代方案。 如果這兩種解決方案都可行：
    -   如果已清除核取方塊的意義不完全明顯，請使用選項按鈕。

        **不正確：**

        ![[橫向] 核取方塊的螢幕擷取畫面 ](images/radio-buttons-image2.png)

        **正確：**

        ![橫向/縱向選項按鈕的螢幕擷取畫面 ](images/radio-buttons-image3.png)

        在正確的範例中，不會相反選項，因此選項按鈕是較佳的選擇。

    -   在 [wizard] 頁面上使用選項按鈕，讓替代方案更清楚，即使核取方塊是可接受的。
    -   如果您有足夠的螢幕空間，而且選項很重要，足以充分利用該螢幕空間，請使用選項按鈕。 否則，請使用核取方塊或下拉式清單。

        **不正確：**

        ![顯示/不顯示選項按鈕的螢幕擷取畫面 ](images/radio-buttons-image4.png)

        在此範例中，選項的重要性不足以使用選項按鈕。

        **正確：**

        ![[不要顯示此訊息] 核取方塊的螢幕擷取畫面 ](images/radio-buttons-image5.png)

        在此範例中，核取方塊是此周邊選項的螢幕空間有效率的使用方式。

    -   如果頁面上有其他核取方塊，請使用核取方塊。

-   **下拉式清單是較佳的選擇嗎？** 如果大部分的使用者建議使用預設選項，在大部分情況下，選項按鈕可能會比所需更多的選項。
    -   如果您不想要特別注意選項，或不想要鼓勵使用者進行變更，請考慮使用下拉式清單。 下拉式清單會將焦點放在目前的選取範圍，而選項按鈕則會平均強調所有選項。

        ![最高品質作為預設按鈕的螢幕擷取畫面 ](images/radio-buttons-image6.png)

        在此範例中，下拉式清單將焦點放在目前的選取範圍，並防止使用者進行變更。

    -   如果頁面上有其他下拉式清單，請考慮使用下拉式清單。

-   **一組命令按鈕、命令連結或分割按鈕是否是較佳的選擇？** 如果只使用選項按鈕來影響命令的執行方式，則改為顯示命令變化通常會比較好。 這麼做可讓使用者選擇具有單一互動的正確命令。
-   **選項是否存在方案選項，而不是資料？** 選項的值不應以內容或其他資料為基礎。 針對資料，請使用下拉式清單或單一選取清單。
-   如果您在嚮導頁面或控制台上使用控制項，則 **控制項是否會回應主要指令，而使用者是否可以在稍後變更選擇？** 如果是的話，請考慮使用命令連結，而不是使用選項按鈕，讓互動更有效率。
-   **這些值是否為非數值？** 若為數值資料，請使用 [文字方塊](ctrl-text-boxes.md)、 [下拉式清單](/windows/desktop/uxguide/ctrl-drop)或 [滑杆](ctrl-sliders.md)。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **以邏輯順序列出選項，** 例如最可能選取到最少、最簡單的作業（最複雜）或最小的風險。 不建議依字母順序排序，因為它是語言相依，因此不可當地語系化。
-   **如果沒有任何選項是有效的選項，請新增另一個選項來反映此選項，** 例如 [無] 或 [不適用]。
-   **偏好垂直對齊選項按鈕，而非水準對齊。** 水準對齊較難讀取和當地語系化。

    **正確：**

    ![垂直選項按鈕對齊方式的螢幕擷取畫面 ](images/radio-buttons-image7.png)

    在此範例中，選項按鈕會垂直對齊。

    **不正確：**

    ![水準選項按鈕對齊方式的螢幕擷取畫面 ](images/radio-buttons-image8.png)

    在此範例中，水準對齊較難閱讀。

-   重新 **考慮使用群組方塊來組織選項按鈕群組**，這通常會造成不必要的螢幕混亂。
-   **請勿使用選項按鈕標籤作為群組箱標籤。**
-   **請勿使用選項按鈕的選項來：**
    -   執行命令。
    -   顯示其他視窗，例如用來收集更多輸入的對話方塊。
    -   以動態方式顯示或隱藏與選取之控制項相關的其他控制項 (畫面讀取器無法偵測到這類事件) 。 不過，您可以根據選取範圍動態變更文字。

### <a name="subordinate-controls"></a>次級控制項

-   將從屬控制項放在或下方 (縮排，並使用選項按鈕標籤) 選項按鈕和其標籤。 以冒號結束選項按鈕標籤。

    ![控制項在其標籤右邊的螢幕擷取畫面 ](images/radio-buttons-image9.png)

    在此範例中，選項按鈕與其從屬控制項共用選項按鈕標籤及其存取金鑰。 在此情況下，方向鍵會將焦點從選項按鈕移至其附屬文字方塊。

-   **如果相依的可編輯文字方塊和下拉式清單共用選項按鈕的標籤，則保持啟用狀態。** 當使用者在方塊中輸入或貼上任何文字時，請自動選取對應的選項。 這麼做可簡化互動。

    ![包含文字方塊之頁面範圍對話方塊的螢幕擷取畫面 ](images/radio-buttons-image10.png)

    在此範例中，輸入頁碼會自動選取頁面。

-   **避免使用其他選項按鈕或核取方塊來嵌套選項按鈕。** 可能的話，請將所有選項保留在相同的層級。

    **正確：**

    ![靠左對齊之選項按鈕的螢幕擷取畫面 ](images/radio-buttons-image11.png)

    在此範例中，選項的等級相同。

    **不正確：**

    ![嵌套選項按鈕的螢幕擷取畫面 ](images/radio-buttons-image12.png)

    在此範例中，使用嵌套選項會增加不必要的複雜性。

-   如果您使用其他選項按鈕或核取方塊來嵌入選項按鈕，請 **停用這些附屬控制項，直到選取高階選項為止。** 這樣做可避免與從屬控制項的意義混淆。

### <a name="default-values"></a>預設值

-   因為一組選項按鈕代表一組互斥的選項，所以 **預設一律會選取一個選項按鈕。選取最安全的 (，以避免資料遺失或系統存取) ，以及最安全且私用的選項。** 如果安全性和安全性不是因素，請選取最有可能或方便的選項。
-   **例外狀況：** 如果有下列情況，則沒有預設選項：
    -   **基於安全、安全性或法律考慮，沒有可接受的預設選項，因此使用者必須做出明確的選擇。** 如果使用者未進行選取，則會顯示錯誤訊息來強制執行。
    -   **使用者介面 (UI) 必須反映目前的狀態，而且尚未設定該選項。** 預設值會不正確地表示使用者不需要進行選取。
    -   **目標是要收集非偏誤的資料。** 預設值會偏差資料收集。
    -   **選項按鈕群組代表混合狀態中的屬性**，當顯示多個沒有相同設定之物件的屬性時，就會發生此情況。 在此情況下不會顯示錯誤訊息，因為每個物件都有有效的狀態。
-   **請將第一個選項設為預設選項**，因為使用者通常會預期這一點，除非該順序不是邏輯的。 若要這樣做，您可能需要變更選項標籤。

    **不正確：**

    ![最後一個選項按鈕的螢幕擷取畫面作為預設選項 ](images/radio-buttons-image13.png)

    在此範例中，預設選項不是第一個選項。

    **正確：**

    ![第一個選項按鈕的螢幕擷取畫面（預設值） ](images/radio-buttons-image14.png)

    在此範例中，會改寫選項標籤，讓第一個選項成為預設選項。

## <a name="labels"></a>標籤

### <a name="radio-button-labels"></a>選項按鈕標籤

-   為每個選項按鈕加上標籤。

<!-- -->

-   將唯一的 [存取金鑰](glossary.md) 指派給每個標籤。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
-   使用 [句子樣式的大小寫](glossary.md)。
-   以片語的形式寫入標籤，而不是以句子撰寫，且不使用結束標點符號。
    -   **例外狀況：** 如果選項按鈕標籤也會標記其後的從屬控制項，請以冒號結束標籤。
-   使用平行片語，並嘗試保留所有標籤的相同長度。
-   將標籤文字的焦點放在選項之間的差異。 如果所有選項都有相同的簡介文字，請將該文字移至群組標籤。
-   使用正片語。 例如，使用 do 而非 [否]，並使用 [列印] 而非 [列印]。
-   只描述標籤的選項。 將標籤保持簡短，讓您可以輕鬆地在訊息和檔中參考它們。 如果選項需要進一步說明，請使用完整的句子和結束標點符號在 [靜態文字](glossary.md) 控制項中提供說明。

    ![具有解說文字之選項按鈕的螢幕擷取畫面](images/radio-buttons-image16.png)

    在此範例中，會使用個別的靜態文字控制項來說明這些選項。

    > [!Note]  
    > 將說明新增至一個選項按鈕，並不表示您必須提供所有選項按鈕的說明。 如果您可以的話，請在標籤中提供相關資訊，並且只在必要時使用說明。 請勿只重新聲明標籤以保持一致性。

     

-   **如果強烈建議選項，請將「 (建議的) 」新增至標籤。** 請務必新增至控制項標籤，而不是增補附注。
-   **如果選項僅供 advanced 使用者使用，請將「 (advanced) 」新增至標籤。** 請務必新增至控制項標籤，而不是增補附注。
-   如果您必須使用多行標籤，請將標籤的頂端與選項按鈕對齊。
-   請勿使用從屬控制項、它所包含的值，或其單位標籤來建立句子或片語。 這類設計無法當地語系化，因為句子結構會隨著語言而有所不同。

### <a name="radio-button-group-labels"></a>選項按鈕群組標籤

-   使用群組標籤來說明群組的用途，而不是如何進行選取。 假設使用者知道如何使用選項按鈕。 例如，不要說「選取下列其中一個選項」。
-   所有的選項按鈕群組都需要標籤。 以單字或片語（而非句子）的形式寫入標籤，並使用靜態文字或群組方塊以冒號結尾。

    **例外狀況：** 如果標籤只是對話方塊 [主要指示](glossary.md)的重述，請省略標籤。 在此情況下，除非有一個) ，否則 main 指令會接受冒號 (，除非它是問題) 和存取金鑰 (。

    **可以接受：**

    ![重複選項按鈕群組標籤的螢幕擷取畫面 ](images/radio-buttons-image17.png)

    在此範例中，選項按鈕群組標籤只是主要指令的重述。

    **較佳：**

    ![選項按鈕主要指示的螢幕擷取畫面 ](images/radio-buttons-image18.png)

    在此範例中，會移除多餘的標籤，所以主要指令會接受冒號。

-   請勿指派標籤的存取金鑰。 這並不是必要的，而是讓其他存取金鑰更難指派。
    -   **例外狀況：** 如果並非所有控制項都可以有唯一的存取金鑰，您可以將存取金鑰指派給標籤，而不是個別控制項。 如需詳細資訊，請參閱 [鍵盤](inter-keyboard.md)。

## <a name="documentation"></a>文件

參考選項按鈕時：

-   使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線或冒號。
-   在程式設計和其他技術檔中，請將選項按鈕稱為選項按鈕。 其他地方使用選項按鈕，尤其是在使用者檔中。
-   若要描述使用者互動，請使用按一下。
-   可能的話，請使用粗體文字來格式化標籤。 否則，請只在必要時才將標籤放在引號中，以避免混淆。

範例：按一下 [ **目前的頁面**]，然後按一下 **[確定]**。
