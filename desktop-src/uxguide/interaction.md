# <a name="interaction"></a>互動

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

互動是使用者與應用程式互動的各種方式，包括觸控、鍵盤、滑鼠等等。 使用這些指導方針來建立可針對觸控進行優化，但在輸入裝置間一致運作的直覺和獨特體驗。

## <a name="design-for-a-touch-first-experience"></a>觸控優先體驗的設計

首先，在設計應用程式時，請先預期觸控會是使用者的主要輸入方法。 如果您使用平臺控制項，則支援觸控板、滑鼠和畫筆/手寫筆不需要額外的程式設計，因為 Windows 免費提供此服務。

不過，請記住，針對觸控最佳化的 UI 未必優於傳統 UI。 兩者都提供技術和應用程式獨特的優點和缺點。 在移至觸控優先的 UI 中，請務必瞭解觸控 (之間的核心差異，包括觸控板) 、畫筆/手寫筆、滑鼠和鍵盤輸入。 請勿採用熟悉的輸入裝置屬性和行為來授與，因為 Windows 8 不只是模擬該功能。

您很快就會發現，觸控輸入需要不同的 UI 設計方法。

## <a name="visuals-for-feedback"></a>意見反應的視覺效果

與您的應用程式互動時的適當視覺效果，可協助使用者辨識、學習及調整其互動方式，讓應用程式和 Windows 視覺效果的意見反應能夠指出成功的互動、轉送系統狀態、改善控制的意義、減少錯誤、協助使用者瞭解系統和輸入裝置，以及鼓勵互動。

當使用者依賴觸控輸入進行要求正確與精確位置的活動時，視覺化回饋就顯得相當重要。 隨時隨地偵測到觸控輸入時便顯示回饋，可以協助使用者了解應用程式及其控制項所定義的任何自訂目標規則。

## <a name="optimize-for-accuracy"></a>針對精確度優化

觸控輸入牽涉到手指的整個連絡人區域。 此連絡人幾何用來判斷最可能的目標物件。 調整控制項的大小，以確保具有可輕鬆且安全地鎖定物件和控制項的熟悉 UI。

調整控制項的大小、空間以及定位，以協助消除手指和手遮蔽，其中 UI 會由使用者互動本身遮蔽。

可能的話，請在連絡人區域上方放置功能表、快顯視窗和工具提示。

## <a name="constrain-for-confidence"></a>信賴度限制

使用 UI 條件約束，以避免或最小化草率互動。

-   貼齊點可讓您更輕鬆地在所需的位置停止。 貼齊點會指定應用程式內容的邏輯停止點。 Cognitively，貼齊點可作為使用者的分頁機制，並將疲勞從過度滑動、輕量或旋轉的程度降至最低。 您可以使用它們來處理不精確的使用者輸入，並確定顯示特定的內容或金鑰資訊子集。
-   強調動作軸的方向「滑軌」 (垂直或水準) 。

## <a name="avoid-timed-interactions"></a>避免計時的互動

不應該以時間來辨別互動。 不論執行時間長短，相同的互動應該產生相同的結果。 以時間為基礎的啟動會對使用者造成強制性的延遲，同時對直接操作的沈浸式性質和系統回應感知功能造成減損。

計時互動一般是根據一些隱藏的閾值 (例如時間、距離或速度) 來決定要執行的命令。 直到系統執行動作，而使用者必須到達任意和不可見的閾值才能達成結果時，時間間隔的互動不會有任何視覺效果的意見反應。 互動時的立即視覺化回饋，可以讓使用者感覺更投入、更自信以及操控自如。

直接影響物件的互動和虛擬實境互動比較直覺式、易搜尋、易記憶。 它們不倚賴模糊或抽象的互動。

**注意：** 例外狀況是您使用特定的計時互動來協助學習和探索 (例如，按住) 。 使用適當的描述和視覺提示，對使用 advanced 互動有很大的影響。
