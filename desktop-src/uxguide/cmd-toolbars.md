## <a name="design-concepts"></a><span data-ttu-id="34d93-135">設計概念</span><span class="sxs-lookup"><span data-stu-id="34d93-135">Design concepts</span></span>

<span data-ttu-id="34d93-182">令人驚訝的是，此模型會使用簡單的程式來細分。</span><span class="sxs-lookup"><span data-stu-id="34d93-182">Surprisingly, this model breaks down with simple programs.</span></span> <span data-ttu-id="34d93-183">對於只有幾個命令的程式，同時擁有功能表列和工具列並不合理，因為功能表列最後是一個多餘、沒有效率的工具列版本。</span><span class="sxs-lookup"><span data-stu-id="34d93-183">For programs with only a few commands, having both a menu bar and a toolbar doesn't make sense because the menu bar ends up being a redundant, inefficient version of the toolbar.</span></span>

### <a name="keyboard-accessibility"></a><span data-ttu-id="34d93-205">鍵盤協助工具</span><span class="sxs-lookup"><span data-stu-id="34d93-205">Keyboard accessibility</span></span>

<span data-ttu-id="34d93-206">從鍵盤存取工具列與存取功能表列相當不同。</span><span class="sxs-lookup"><span data-stu-id="34d93-206">From the keyboard, accessing toolbars is quite different from accessing menu bars.</span></span> <span data-ttu-id="34d93-207">當使用者按下 ALT 鍵時，功能表列會收到輸入焦點，而且會失去使用 Esc 鍵的輸入焦點。</span><span class="sxs-lookup"><span data-stu-id="34d93-207">Menu bars receive input focus when users press the Alt key and they lose input focus with the Esc key.</span></span> <span data-ttu-id="34d93-208">如果功能表列具有輸入焦點，則會獨立流覽視窗的其餘部分，並處理所有箭號、首頁、結尾和 Tab 鍵。</span><span class="sxs-lookup"><span data-stu-id="34d93-208">Once a menu bar has input focus, it is navigated independently of the remainder of the window, handling all arrow keys, Home, End, and Tab keys.</span></span> <span data-ttu-id="34d93-209">相反地，當使用者按下 Tab 鍵時，工具列會透過視窗的整個內容來接收輸入焦點。</span><span class="sxs-lookup"><span data-stu-id="34d93-209">By contrast, toolbars receive input focus when users press the Tab key through the entire contents of the window.</span></span> <span data-ttu-id="34d93-210">因為工具列是最後一個定位順序，所以在忙碌頁面上啟動時可能需要投入大量的時間 (，除非使用者知道要使用 Shift + Tab 來向後移動) 。</span><span class="sxs-lookup"><span data-stu-id="34d93-210">Because toolbars are last in tab order, they might take some significant effort to activate on a busy page (unless users know to use Shift+Tab to move backwards).</span></span>

<span data-ttu-id="34d93-211">協助工具在此提供一個難題：雖然滑鼠使用者更容易使用工具列，但鍵盤使用者的存取權較小。</span><span class="sxs-lookup"><span data-stu-id="34d93-211">Accessibility presents a dilemma here: while toolbars are easier for mouse users, they are less accessible for keyboard users.</span></span> <span data-ttu-id="34d93-212">如果功能表列和工具列都有，但如果功能表列已移除或隱藏，就不會有問題。</span><span class="sxs-lookup"><span data-stu-id="34d93-212">This isn't a problem if there is both a menu bar and a toolbar, but it is if the menu bar is removed or hidden.</span></span>

<span data-ttu-id="34d93-213">基於協助工具的考慮，請偏好保留功能表列，而不是完全以工具列的方式移除。</span><span class="sxs-lookup"><span data-stu-id="34d93-213">For accessibility reasons, then, prefer to retain the menu bar rather than remove it completely in favor of a toolbar.</span></span> <span data-ttu-id="34d93-214">如果您必須選擇移除功能表列，並直接隱藏它，則偏好隱藏它。</span><span class="sxs-lookup"><span data-stu-id="34d93-214">If you must choose between removing the menu bar and simply hiding it, prefer to hide it.</span></span>
