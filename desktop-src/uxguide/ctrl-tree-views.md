## <a name="design-concepts"></a><span data-ttu-id="7e290-152">設計概念</span><span class="sxs-lookup"><span data-stu-id="7e290-152">Design concepts</span></span>

<span data-ttu-id="7e290-153">樹狀結構是用來組織資料並讓它更容易找到，但很難讓樹狀結構中的資料易於探索。</span><span class="sxs-lookup"><span data-stu-id="7e290-153">Trees are intended to organize data and make it easy to find, yet it's difficult to make data within a tree easily discoverable.</span></span> <span data-ttu-id="7e290-154">決定樹狀檢視及其組織時，請記住下列原則。</span><span class="sxs-lookup"><span data-stu-id="7e290-154">Keep the following principles in mind when deciding about tree views and their organization.</span></span>

### <a name="predictability-and-discoverability"></a><span data-ttu-id="7e290-155">可預測性和可探索性</span><span class="sxs-lookup"><span data-stu-id="7e290-155">Predictability and discoverability</span></span>

<span data-ttu-id="7e290-156">**樹狀檢視是以物件之間的關聯性為基礎。**</span><span class="sxs-lookup"><span data-stu-id="7e290-156">**A tree view is based on the relationships between objects.**</span></span> <span data-ttu-id="7e290-157">當物件形成明確、已知、互斥的關聯性時，樹狀結構的效果最佳，其中每個物件都對應至單一且易於判斷的容器。</span><span class="sxs-lookup"><span data-stu-id="7e290-157">Trees work best when the objects form a clear, well known, mutually exclusive relationship in which every object maps to a single, easy-to-determine container.</span></span>

<span data-ttu-id="7e290-158">**有一個重要的問題是，物件可以出現在不同的節點中。**</span><span class="sxs-lookup"><span data-stu-id="7e290-158">**A significant problem is that an object can appear in different nodes.**</span></span> <span data-ttu-id="7e290-159">例如，使用者應該在哪裡尋找播放音樂的硬體裝置、有大型硬碟，並使用 USB 埠？</span><span class="sxs-lookup"><span data-stu-id="7e290-159">For example, where would users expect to find a hardware device that plays music, has a large hard disk, and uses a USB port?</span></span> <span data-ttu-id="7e290-160">可能在數個不同的容器節點中，例如多媒體、儲存體、USB，以及可能的硬體資源。</span><span class="sxs-lookup"><span data-stu-id="7e290-160">Perhaps in any of several different container nodes, such as Multimedia, Storage, USB, and possibly in Hardware Resources.</span></span> <span data-ttu-id="7e290-161">其中一個解決方法是將每個物件放在單一最適當的容器下，而不考慮任何情況;另一種方法是將每個物件放在所有適用的容器之下。</span><span class="sxs-lookup"><span data-stu-id="7e290-161">One solution is to place each object under the single most appropriate container regardless of the circumstances; another approach is to place each object under all containers that apply.</span></span> <span data-ttu-id="7e290-162">前者會升級簡單的全新階層，後者會提升可探索性，每個都有優點和潛在的問題。</span><span class="sxs-lookup"><span data-stu-id="7e290-162">The former promotes a simple, clean hierarchy and the latter promotes discoverability each has advantages and potential problems.</span></span>

<span data-ttu-id="7e290-163">**使用者可能不會完全瞭解樹狀結構的版面配置，但會在與樹狀結構互動一段時間後，形成關聯性的精神模型。**</span><span class="sxs-lookup"><span data-stu-id="7e290-163">**Users may not completely understand the layout of the tree, but they will form a mental model of the relationships after interacting with the tree for a while.**</span></span> <span data-ttu-id="7e290-164">如果該精神模型不正確，就會導致混淆。</span><span class="sxs-lookup"><span data-stu-id="7e290-164">If that mental model is incorrect, it leads to confusion.</span></span> <span data-ttu-id="7e290-165">例如，假設音樂播放機可在多媒體、存放裝置和 USB 容器中找到。</span><span class="sxs-lookup"><span data-stu-id="7e290-165">For example, suppose a music player can be found in the Multimedia, Storage, and USB containers.</span></span> <span data-ttu-id="7e290-166">這種安排可改善探索能力。</span><span class="sxs-lookup"><span data-stu-id="7e290-166">This arrangement improves discoverability.</span></span> <span data-ttu-id="7e290-167">如果使用者先在多媒體中尋找裝置，則使用者可能會發現所有裝置（像是音樂播放機）都會出現在多媒體容器中。</span><span class="sxs-lookup"><span data-stu-id="7e290-167">If a user first finds the device in Multimedia, the user may conclude that all devices like music players appear in the Multimedia container.</span></span> <span data-ttu-id="7e290-168">然後，使用者會預期類似的裝置（例如數位相機）會出現在多媒體容器中，如果不是這種情況，將會混淆。</span><span class="sxs-lookup"><span data-stu-id="7e290-168">The user will then expect similar devices, such as digital cameras, to appear in the Multimedia container and will become confused if that isn't the case.</span></span>

<span data-ttu-id="7e290-169">**設計樹狀結構時的挑戰是將探索能力與可預測的使用者模型進行平衡，以將混淆降至最低。**</span><span class="sxs-lookup"><span data-stu-id="7e290-169">**The challenge when designing a tree is to balance discoverability with a predictable user model that minimizes confusion.**</span></span>

### <a name="breadth-vs-depth"></a><span data-ttu-id="7e290-170">廣度和深度</span><span class="sxs-lookup"><span data-stu-id="7e290-170">Breadth vs. depth</span></span>

<span data-ttu-id="7e290-171">可用性研究顯示， **使用者在尋找樹狀結構中的物件時，會比深度更廣**，因此在設計樹狀結構時，您應該優先使用廣度。</span><span class="sxs-lookup"><span data-stu-id="7e290-171">Usability studies have shown that **users are more successful at finding objects in a tree that is broad than in a tree that is deep**, so when designing a tree you should prefer breadth over depth.</span></span> <span data-ttu-id="7e290-172">在理想的情況下，樹狀結構不應超過四個層級 (不會計算根節點) ，而且最常存取的物件應該會出現在前兩個層級中。</span><span class="sxs-lookup"><span data-stu-id="7e290-172">Ideally, a tree should have no more than four levels (not counting the root node) and the most commonly accessed objects should appear in the first two levels.</span></span>
