# <a name="spin-controls"></a>微調控制項

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用微調控制項時，使用者可以按一下箭號按鈕，以累加方式變更其相關 [數值文字方塊](ctrl-text-boxes.md)中的值。 「微調」一詞是指文字方塊及其關聯之微調控制項的組合。

![微調控制項和文字方塊的螢幕擷取畫面 ](images/ctrl-spin-controls-image1.png)

典型的微調方塊。

使用者通常會偏好微調控制項，因為他們可以進行變更，而不需要從滑鼠移動手。 當微調控制項與文字方塊配對時，使用者可以直接在文字方塊中輸入或貼上輸入，因此微調控制項的使用是選擇性的。

當微調控制項用於數值輸入時，輸入不一定是純整數。 輸入可以是十進位數，而且可以有負號、分隔符號 (例如冒號或連字號) 和單位修飾詞。

> [!Note]  
> 與 [文字方塊](ctrl-text-boxes.md) 和 [版面](vis-layout.md) 配置相關的指導方針會顯示在不同的文章中。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **用於數值輸入的控制項？** 如果沒有，請使用另一個控制項（例如 [下拉式清單](/windows/desktop/uxguide/ctrl-drop) 或 [滑杆](ctrl-sliders.md)），從一組固定的值進行選取。 使用捲軸來進行滾動。
-   **使用者是否將值視為相對數量，而不是數值？** 如果是的話，請改用滑杆。 只針對確切的已知數值使用微調方塊。 例如，使用者想要將音訊音量設定為低或中，而不是設定值為 2 或 5。
-   **控制項是否與文字方塊配對？** 如果沒有，請不要使用。 微調控制項不可以單獨使用，也不應該與文字方塊以外的其他控制項類型一起使用。

    **不正確：**

    ![微調控制項、圖形、無文字方塊的螢幕擷取畫面 ](images/ctrl-spin-controls-image2.png)

    在此範例中，會使用微調控制項來控制動態圖形。

-   **連續值範圍是否有效？** 如果沒有，請改用有效的值下拉式清單。

    ![下拉式清單的螢幕擷取畫面 ](images/ctrl-spin-controls-image3.png)

    在此範例中，並非所有磁片磁碟機號碼都有效，所以下拉式清單是較佳的選擇。

-   **使用微調控制項實用嗎？** 針對下列情況，使用微調控制項非常實用：

    -   輸入較小的數位，通常低於100。
    -   對現有或預設值進行較小的變更。

    雖然微調控制項可用於任何數值輸入，但在這些情況下，它們不會有效率。

-   **微調控制項有説明嗎？** 控制項是在使用者可能使用其滑鼠的內容中使用嗎？ 如果沒有，請考慮使用微調控制項。
-   **是否為 [同級控制項] 下拉式清單？** 如果有其他的下拉式清單，請考慮使用下拉式清單來保持一致性。

    ![具有下拉式清單之對話方塊的螢幕擷取畫面 ](images/ctrl-spin-controls-image4.png)

    在此範例中，可以使用微調方塊，但會使用下拉式清單來提供一致性。

-   **觸控或畫筆使用者是否為主要目標？** 如果是的話，請考慮改為使用下拉式清單。 微調控制項中的箭號按鈕太小，無法使用觸控或畫筆有效率地使用。

如果有滑杆或微調方塊，請在下列情況下使用微調方塊：

-   螢幕空間很小。
-   使用者很可能會偏好使用鍵盤。

如果是下列情況，請使用滑桿：

-   使用者可受益於立即回應。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **使用微調控制項，只要它們是實用且有用的。** [這是正確的控制項嗎？](#is-this-the-right-control)

    -   **例外狀況：** 若要與相同使用者介面上的其他文字方塊保持一致 (UI) ，請使用微調控制項，即使它們不一定可行也一樣。

    **正確：**

    ![月、日、年微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image5.png)

    在此範例中，微調控制項會與 year 控制項搭配使用，以保持一致性，雖然它不一定是可行的。

    **不正確：**

    ![ip 位址微調控制的螢幕擷取畫面 ](images/ctrl-spin-controls-image6.png)

    在此範例中，微調控制項是無法使用的。

-   **一律讓微調控制項成為文字方塊的「合作者」。** 這麼做會將微調控制項放在文字方塊內。

    **正確：**

    ![在文字方塊內放置微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image7.png)

    **不正確：**

    ![在文字方塊外放置微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image8.png)

    在正確的範例中，微調控制項放在其相關聯的文字方塊內。

-   **停用微調控制項的相關聯文字方塊時，請將其停用。** 微調控制項是一種補充的輸入方法，絕對不是唯一的輸入方法。

### <a name="values"></a>值

-   **定義上方按鈕，將值增加一個單位，並將底部按鈕減少一個單位。** 一般而言，此單位是一，但它應該是最小的價值變更。 在理想的情況下，微調控制項應該涵蓋所有有效值，而且比輸入文字更方便。

    ![[邊界] 微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image9.png)

    在此範例中，按一下微調控制項會將值變更為 .1，這是值中最小的一般變更。 使用較小的單位會涵蓋有效值的範圍，但讓微調控制項無法使用。

-   **使用微調控制項可將輸入限制為有效的值。** 使用微調控制項絕不會導致不正確的值。
-   **在有效值範圍的結尾，重新開機範圍。** 微調控制項比喻是使用者正在旋轉值滾輪，因此是這類滾輪的行為。
    -   **例外狀況：** 如果產生的值是不正確的，請勿重新開機範圍。

        ![[份數] 微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image10.png)

        在此範例中，按一下向下箭號按鈕，並不會重新開機範圍 (，方法是前往最大值) ，因為該值一定不正確。

-   **使用文字，而非特殊數值。** 允許使用者微調這些特殊值，而不需要知道這些特殊值並將其輸入。

    ![[在 (永遠不) 時睡眠] 微調控制項的螢幕擷取畫面 ](images/ctrl-spin-controls-image11.png)

    在此範例中，絕對不是特殊值，但使用者可以微調。

-   **如果值具有分隔符號，則相關聯的文字方塊應該有多個輸入焦點點。** 這麼做可讓您個別運算元值區段。

    ![時間微調控制項的螢幕擷取畫面，選取的分鐘數 ](images/ctrl-spin-controls-image12.png)

    在此範例中，微調控制項會影響小時、分、秒和 am/P.M. 的值（以其焦點為准）。

-   **如果值具有單位，請使用微調控制項來變更這些單位。**

    ![時間微調控制項的螢幕擷取畫面，[a.m.] 已選取 ](images/ctrl-spin-controls-image13.png)

    在此範例中，微調控制項可以用來變更單位。

## <a name="labels"></a>標籤

-   套用 [文字方塊標籤](ctrl-text-boxes.md) 指導方針，以標示相關聯的文字方塊。 微調控制項絕不會直接標記。

## <a name="documentation"></a>文件

參考微調控制項時：

-   請勿在使用者檔中參考微調控制項。 相反地，請參閱相關聯文字方塊的標籤。
-   請只在程式設計和其他技術檔中參考微調控制項和微調方塊。

範例：在 [ **日期** ] 方塊中，輸入或選取您要變更的日期部分。
