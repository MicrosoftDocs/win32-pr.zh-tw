# <a name="progress-bars"></a>進度列

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

使用進度列時，使用者可以追蹤長時間操作的進度。 進度列可能會顯示完成 (確定) 的大約百分比，或表示作業正在進行中 (不定) 。

可用性研究顯示，使用者知道回應時間超過一秒。 因此，您應該考慮需要兩秒或更長時間才能完成的作業，這需要一些時間才能完成，而且需要一些類型的進度意見反應。

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **作業會在五秒內完成嗎？** 若是如此，請改為使用 [活動指標](inter-mouse.md) ，因為顯示短暫期間的進度列會造成干擾。 如果作業通常需要五秒或更少，但有時需要更多時間，請從忙碌指標開始，然後在五秒後轉換成進度列。
-   **是否有不定的進度列可用來等候使用者完成工作？** 如果是，請不要使用進度列。 進度列適用于電腦進度，而不是使用者進度。
-   **是否與動畫結合不定的進度列？** 若是如此，請改用動畫。 不定的進度列實際上是泛型動畫，不會將任何值新增至動畫。
-   **這項作業的 (長達兩分鐘的時間，) 背景工作是否比進度更有興趣完成？** 若是如此，請改為使用 [通知](mess-notif.md) 。 在此情況下，使用者會在此同時執行其他工作，且不會監視進度。 使用通知可讓使用者在不中斷的情況下執行其他工作。 這類冗長作業的範例包括列印、備份、系統掃描，以及大量資料傳輸或轉換。
-   **當作業完成時，使用者是否能夠重新播放結果？** 如果是的話，請改用滑杆。 這類作業的範例包括影片和音訊錄製和播放。

    ![media player 和滑杆的螢幕擷取畫面 ](images/progress-bars-image2.png)

    在此範例中，滑杆可用來指出播放音效的進度。 這麼做可讓使用者在稍後重新執行結果。

## <a name="design-concepts"></a>設計概念

在冗長的作業期間，使用者需要對作業進行作業的一般概念。 他們也需要知道：

-   這是一段冗長的作業已開始。
-   正在進行該進度，而且作業最後會完成 (，因此未鎖定) 。
-   已 (完成的作業大約百分比，因此是剩餘的百分比) 。
-   如果不值得繼續等候，則應取消作業。
-   當作業完成時，應該繼續等候或執行其他作業。

**針對需要限制時間的作業使用確定進度列，** 即使該時間量無法準確預測也一樣。 不定的進度列會顯示正在進行的進度，但不提供任何其他資訊。 請勿只根據可能缺少的精確度來選擇不定的進度列。

例如，假設某項作業需要五個步驟，且每個步驟都需要限定的時間量，但每個步驟的時間量可能會有很大的差異。 在此情況下，請使用確定進度列，並在每個步驟完成時顯示進度，並與每個步驟通常花費的時間量成正比。 只有在確定進度列會導致使用者不正確地結束作業已被鎖定時，才使用不定的進度列。

**如果您只做一件事 .。。**

請確定您提供冗長作業的進度意見反應，並清楚傳達上述資訊。 盡可能使用確定進度列。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **在執行冗長的作業時提供進度回饋。** 使用者永遠不需要猜測進度是否正在進行。
-   **明確指出真正的進度。** 進度列必須在進行進度時繼續進行。 如果預期的完成時間範圍很大，請考慮使用非線性尺規來指出較長時間的進度。 您不想讓使用者在程式未完成時，結束程式的鎖定。
-   **明確指出沒有進度。** 如果未進行任何進度，就不能事先前進進度列。 您不想讓使用者無限期地等候永遠不會完成的作業。
-   **提供有用的進度詳細資料。** 提供其他的進度資訊，但僅限使用者可以使用它來執行某些動作。 請確定文字顯示的時間夠長，讓使用者能夠讀取該文字。

    ![顯示傳輸速率的進度列螢幕擷取畫面 ](images/progress-bars-image11.png)

    在此範例中，使用者可以查看傳送速率。 這裡的低傳輸速率建議您需要使用高頻寬的網路連接。

-   **請勿提供不必要的詳細資料。** 通常使用者並不在意正在執行之作業的詳細資料。 例如，安裝程式的使用者不在意要複製的特定檔案，或系統元件因為沒有這些詳細資料的預期而註冊。 通常，標示正確的進度列會提供足夠的資訊，因此，只有在使用者可以利用它來執行某些動作時，才提供其他的進度資訊。 提供使用者不在意的詳細資料，讓使用者體驗變得過於複雜和技術性。 如果您需要更詳細的偵錯工具資訊，請勿將它顯示在發行組建中。

    **正確：**

    ![安裝進度的螢幕擷取畫面 ](images/progress-bars-image12.png)

    在此範例中，只需要標記的進度列。

    **正確：**

    ![顯示傳輸速率的進度列螢幕擷取畫面 ](images/progress-bars-image11.png)

    在此範例中，Windows 檔案總管正在複製使用者選取的檔案，因此顯示覆制的檔案名有意義。

    **不正確：**

    ![註冊進度的螢幕擷取畫面 ](images/progress-bars-image13.png)

    在此範例中，安裝程式會提供對使用者沒有意義的詳細資料。

-   **提供有用的動畫。** 如果順利完成，動畫可以協助使用者將作業視覺化，以改善使用者體驗。 良好的動畫會對單獨的文字產生更大的影響。 例如，如果可以復原檔案，則 [Outlook 刪除] 命令的進度列會顯示目的地的資源回收筒，但如果無法復原檔案，則不資源回收筒。

    ![刪除進度的螢幕擷取畫面 ](images/progress-bars-image14.png)

    在此範例中，缺少資源回收筒會強化檔案的永久刪除。 這種額外資訊不會以單獨使用的文字來傳達。

-   **請勿使用不必要的動畫。** 動畫可能會產生誤導，因為它們通常會在與實際工作不同的執行緒中執行，因此即使作業已鎖定，也可以建議進度。 此外，如果作業的速度低於預期，使用者有時會假設動畫是原因的一部分。 因此，只有在有清楚的理由時，才使用動畫;請勿使用它們來嘗試榮幸使用者。
-   **將動畫置於進度列的中央。** 將動畫放在進度列標籤上方（如果有的話）。 如果進度列右邊有 [取消] 或 [停止] 按鈕，請在決定中心時包含按鈕。
-   **只有當作業的長度很長時，才會在作業完成時播放音效效果 (超過兩分鐘) 、不頻繁且重要。** 如果使用者可能會在處理過程中離開重要的作業，則音效效果會還原使用者的注意力。 在其他情況下，在完成時使用音效效果可能會造成干擾。
-   **請勿竊取輸入焦點來顯示進度更新或完成。** 使用者通常會在等候但不想中斷時切換至其他程式。 背景工作必須停留在背景中。
-   **別擔心技術支援。** 由於進度列所提供的意見不一定是正確的，而且是 fleeting 的，所以進度列不是提供技術支援資訊的絕佳機制。 因此，如果作業可能會因為安裝程式) 而失敗 (，請不要提供僅適用于技術支援的額外進度資訊。 相反地，請提供替代機制（例如記錄檔）來記錄技術支援資訊。

    **不正確：**

    ![進度列的螢幕擷取畫面，顯示伺服器名稱 ](images/progress-bars-image15.png)

    在此範例中，進度列會顯示適用于技術支援的詳細資料。

-   **請勿將完成百分比或任何其他文字放在進度列上。** 這類文字無法存取，而且與使用主題不相容。

    **不正確：**

    ![橫條圖上含有文字的進度列螢幕擷取畫面 ](images/progress-bars-image16.png)

    在此範例中，無法存取進度列上的百分比文字。

-   **請勿將進度列與忙碌指標合併。** 請使用其中一種，但不能同時使用兩者。
-   **請勿使用垂直進度列。** 水準進度列具有更自然的對應和更好的流程。

### <a name="determinate-progress-bars"></a>確定進度列

-   **針對需要限制時間的作業使用確定進度列，** 即使該時間量無法準確預測也一樣。 不定的進度列會顯示正在進行的進度，但不提供任何其他資訊。 請勿只根據可能缺少的精確度來選擇不定的進度列。
-   **明確指出進度階段。** 進度列必須能夠指出作業是在作業的開頭、中間或結尾。 例如，進度列會立即獲得99% 的完成，然後長時間保持在最大的 uninformative 和討厭。 在這些情況下，進度列應一開始設定為最高33%，表示作業仍在開始階段。
-   **明確指出完成。** 除非作業已完成，否則不要讓進度列移至100%。
-   **如果您可以正確地完成，請提供剩餘的預估時間。** 精確度較高的估計值會很有用，但從標示或跳動開始的估計值卻很有用。 您可能需要先執行一些處理，才能提供精確的估計值。 如果是這種情況，請不要在此初始期間顯示可能不准確的估計值。
-   **請勿重新開機進度。** 如果進度列在重新開機時遺失其值 (可能是因為作業中的步驟完成) ，因為使用者無法得知作業何時會完成。 相反地，請讓作業中的所有步驟共用進度的一部分，並讓進度列移至完成一次。

    **不正確：**

    ![已重新開機之進度列的螢幕擷取畫面 ](images/progress-bars-image17.png)

    在此範例中，作業會移至複製檔案的步驟，並重設該步驟的進度列。 現在使用者已不知道有多少進度，或剩下多少時間。

-   **不要備份進度。** 在重新開機時，如果進度列備份，進度列就會失去其值。 一律以單純的進度增加進度。 不過，您可以有剩餘的估計值來增加 (以及減少) ，因為進度的比率可能不同。

### <a name="indeterminate-progress-bars"></a>不定的進度列

-   **請僅針對無法判斷其整體進度的作業使用不定進度列。** 針對需要無限制時間或存取未知物件數目的作業，請使用不定的進度列。 使用超時來提供以時間為基礎之作業的界限。
-   **一旦可以決定整體進度，請轉換成確定進度列。** 例如，如果花了很長的時間來判斷物件的數目，您可以在計算物件時使用不定的進度列，然後轉換成確定進度列。
-   **請勿將不定的進度列合併為完成或剩餘時間的估計值。** 如果您可以提供這種資訊，請改用確定進度列。
-   **請勿將不定的進度列與動畫結合在一起。** 不定的進度列實際上是一般動畫，因此您應該使用其中一個，但絕不會使用兩者。

    **正確：**

    ![偵測伺服器中進度的螢幕擷取畫面 ](images/progress-bars-image18.png)

    在此範例中，只會使用動畫來顯示作業正在進行中。

### <a name="modeless-progress-bars"></a>非模式進度列

-   **如果使用者可以在作業進行時提高生產力，請提供無狀態的意見反應。** 您可能需要停用需要完成此作業的功能子集。
-   **如果視窗有網址列，請在網址列中顯示非模式進度。**

    ![作為網址列一部分的進度列螢幕擷取畫面 ](images/progress-bars-image19.png)

    在此範例中，非模式進度會顯示在網址列中。

-   否則， **如果視窗有狀態列，請在狀態列中顯示非模式進度。** 將任何對應的文字放在狀態列的左邊。

    ![進度列的螢幕擷取畫面，作為狀態列的一部分 ](images/progress-bars-image7.png)

    在此範例中，無狀態的進度會顯示在狀態列中。

### <a name="modal-progress-bars"></a>模式進度列

### <a name="time-remaining"></a>剩餘時間

針對確定進度列：

-   **使用下列時間格式。** 從最大時間單位不是零的下列其中一種格式開始，然後在最大時間單位變成零之後變更為下一個格式。

    **針對進度列：**

    **如果相關資訊顯示為冒號格式：**

    剩餘時間： h 小時、m 分鐘

    剩餘時間： m 分鐘、秒

    剩餘時間：秒

    **如果螢幕空間位於 premium：**

    h 小時，剩餘 m 分鐘

    分鐘，剩餘的秒數

    秒數

    **否則：**

    小時，剩餘 m 分鐘

    分鐘，剩餘秒數

    秒數

    **針對標題列：**

    hh： mm 剩餘

    mm：剩餘的 ss

    0： ss 剩餘

    此壓縮格式會先顯示最重要的資訊，使其不會在工作列上被截斷。

-   **讓預估結果正確無誤，但不提供 false 精確度。** 如果最大單位為小時，請在有意義的) （而不是秒）時，提供分鐘 (。

    **不正確：**

    hh 小時、mm 分鐘、ss 秒

-   **將預估保持在最新狀態。** 更新剩餘時間至少每5秒估計一次。
-   將 **焦點放在剩餘的時間**，因為這是使用者最在意的資訊。 只有在某些情況下，已耗用時間很有説明 (例如工作可能重複) 時，才提供總經過時間。 如果剩餘的預估時間與進度列相關聯，則沒有百分比的完成文字，因為該資訊是由進度列本身所傳達。
-   **語法正確。** 當數位為1時，請使用單一單位。

    **不正確：**

    1分鐘，1秒

-   **使用句型大寫。**

### <a name="progress-bar-colors"></a>進度列色彩

-   **只使用紅色或黃色的進度列來指出進度狀態，而不是工作的最終結果。** 紅色或黃色的進度清單示使用者需要採取一些動作才能完成工作。 如果條件無法復原，請將進度列保持為綠色，並顯示錯誤訊息。
-   **當有使用者可復原的狀況，以防止進一步的進度時，請將進度列設為紅色。** 顯示訊息來說明問題並建議解決方案。
-   將 **進度列設為黃色，表示使用者已暫停工作，或有正在阻礙進度** 但進度 (仍在進行中的狀況，例如，具有網路連線能力不佳) 。 如果使用者已暫停，請將 [暫停] 按鈕標籤變更為 [繼續]。 如果進度是阻礙，則顯示訊息來說明問題並建議解決方案。
