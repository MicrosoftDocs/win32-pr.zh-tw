# <a name="style-and-tone"></a>樣式和語氣

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

撰寫語氣是寫入器傳達給讀者的態度。 這不一定是您所說的，而是您的意義。 語氣的目的是要在讀者中建立特定的回應或表情;它會建立可與使用者互動或足以擊退的個人化。

**注意：** 與 [使用者介面文字](text-ui.md) 相關的指導方針會在個別的文章中顯示。

## <a name="design-concepts"></a>設計概念

### <a name="the-windows-tone"></a>Windows 音訊

您可以使用 Windows 的語氣，藉由準確、鼓勵、深入見解、目標和使用者的專注，與個人級的使用者溝通，以激發信心。 Windows 語氣也致力於輕量、激勵人心、直接且值得信賴。

請勿使用分散、condescending 或是傲的語氣。 避免「電腦」語音 (的極端，也就是從語言中移除說話者) 和「銷售代表」語音 (在這種情況下，書面會嘗試銷售一些東西，偷天換日我們來大家獻唱節慶我們) 

Microsoft 的研究顯示在軟體中，通常會有：

-   **過度利用電腦術語和術語，** 許多使用者並不瞭解或誤譯。
-   **術語的使用不一致。**
-   **Impenetrable，通常會不慎 patronizing 訊息，** 讓使用者難以破解。

這些問題讓使用者感到困惑、unintelligent、disheartened，最後不會與軟體的使用經驗。

請注意訊息品質（包括文字的語氣），表示語言本身在整體客戶滿意度方面是重要的要素。 支援具有不同技術知識層級的使用者，特別是使用者，以充分利用您的程式可執行檔所有作業。

**如果您只做一件事 .。。**

確定您的文字準確、鼓勵、深入見解、目標和使用者導向。 既簡單又值得信任，在適當的情況下，應該使用輕量和激勵人心的語氣。

## <a name="guidelines"></a>指導方針

### <a name="use-the-windows-tone"></a>使用 Windows 音訊

程式中的語氣應該是：

-   **準確。** 使用者應該覺得讓安心資訊在技術上是正確的。 如果資訊不正確，則會期盼使用者對該特定工作的體驗，並對他從該來源讀取的任何其他協助感到信心。
-   **鼓勵。** 使用可傳達軟體讓使用者執行專案的語言，而不是讓他們執行動作。 例如，使用「您可以」而不是「Windows 讓您」或「這項功能可讓您」。  (例外狀況：當您參考允許或拒絕某個動作的功能時，可以使用「允許」（例如安全性功能）。 ) 
-   **深刻。** 使用者應認為您 (和延伸您的應用程式) 知道特定工作何時複雜，並引導您完成。 同時，也會將使用者視為需要特定問題協助的智慧型人員。
-   **目的。** 有時候，使用者需要更豐富的說明;雖然它們只是想要知道需要進行哪些工作， 這需要客觀性，才能辨識出目標 (生產力、好奇心、享受) 是使用者的目標，而不是作者的。 您也需要縮減任何有關使用者的 predisposed 概念。
-   **以使用者為焦點。** 從使用者的觀點來撰寫，最好是從使用者的觀點來看。 使用者應該覺得他們會找到與其相關且可存取的資訊。

您可以透過下列方式，進一步開發與使用者的通訊：

-   **光。** 輕量的聲音很容易處理並快速理解。 這並不困難或不繁瑣，而且可以避免出現太深遠或嚴重的情況。
-   **鼓舞人心。** 激勵人心 voice 應採用使用者，生動他們來採取行動。 有些熱忱會標示這種聲音，讓它比電腦上經常預期的電腦更有吸引力。
-   **簡單。** 您可以從 pretense 或 deceit q&a 講習會並開啟直接的語音。
-   **值得 信賴。** 值得信賴的聲音值得信賴、可靠、精確且誠實。 它只需要一些不精確的部分就會遺失使用者的信任，但可能需要很長的時間才能回頭獲得該信任。

相反地，請務必避免下列色調，而這可能更有可能收到負面反應：

-   **電腦語氣。** 與電腦或機器人的不近人情和非彈性交換一樣。
-   **公司語氣。** 很像是有一獨角戲的全功能、大家都知道不近人情 corporation。
-   **定律強制語氣。** 似乎查核了發問一連串的侵入性問題。
-   **Lawyerly 語氣。** 似乎被要求執行某些合法的動作。
-   **銷售代表語氣。** 感覺像是被要求購買或嘗試某件事，並說明 mda 任何疑慮。
-   **優質、condescending 或生氣的語氣。** 像是 belittling 的使用者、向下討論或對他們感到不滿意的軟體。 一般來說，這種語氣非常技術性，會將不必要的注意力吸引到使用者的錯誤，並感覺得相當吸引人。
-   **Boastful 語氣。** 好像是軟體個人榮譽榜的成就，或是將太多注意力放在一起。
-   **Flippant 語氣。** 感覺像是使用者的目標和表情不是嚴重的，或其使用程式是被授與的。

### <a name="use-real-world-language"></a>使用真實世界的語言

-   **您可以使用每日單字，並避免其他人不會說的單字。** 如果您要說明複雜的技術概念或動作，這項功能就特別有效。 想像一下，您想要查看使用者的肩，並說明如何完成工作。

    **可以接受：**

    使用此程式來變更您的密碼。

    **較佳：**

    請遵循下列步驟來變更您的密碼。

-   **請盡可能使用短的純字。** 較短的單字會更有對話、節省螢幕上的空間，而且更容易掃描。

    **可以接受：**

    此外，本節會說明 .。。

    數位攝影機採用微小的 microchips .。。

    數位相機利用微小的 microchips .。。

    **較佳：**

    本節也會說明 .。。

    數位相機使用微小的 microchips .。。

-   **請勿將單字組成單字，或對標準單字套用新意義。** 假設使用者比較熟悉單字所建立的意義，而不是技術產業提供的特殊意義。 當需要產業詞彙時，請提供內容中的定義。 避免使用術語，但請記住，電腦使用方式（駭客、燒錄 CD 等）特定的某些運算式已經是日常語音的一部分。

    **不正確：**

    您可以使用資料夾來分類我的最愛。

    **正確：**

    您可以使用資料夾來分類我的最愛。

-   **請勿使用符號做為簡單單字的替代符號。**

    **不正確：**

    使用者 & 電腦

    使用者 + 電腦

    網域/工作組

    \# 的使用者

    **正確：**

    使用者和電腦

    網域或工作組

    使用者數目

### <a name="be-precise"></a>精確

-   選擇具有明確意義的單字。

    **可以接受：**

    由於您已建立資料表，因此您可以對它進行變更。

    將您的防火牆關閉，因為關閉它可能會造成安全性風險。

    **較佳：**

    由於您已建立資料表，因此您可以對它進行變更。

    保持您的防火牆開啟，因為關閉它可能會造成安全性風險。

-   省略不必要的字組，如果有的話，請不要使用兩個或三個字組。

    **詳細：**

    請依照下列步驟來變更您的密碼：

    **較佳：**

    若要變更您的密碼：

-   避免不必要的副詞。

    **不正確：**

    變更您的密碼並不難。

    **正確：**

    變更您的密碼並不難。

-   選擇多重單字動詞命令的單一單字動詞。

    **可以接受：**

    當您鎖定電腦時 .。。

    **較佳：**

    當您鎖定電腦時 .。。

-   請勿將動詞和名詞轉換成動詞。

    **不正確：**

    以密碼保護您的電腦 .。。

    若要建立連接 .。。

    **正確：**

    使用密碼保護您的電腦 .。。

    連接 .。。

### <a name="be-consistent"></a>保持一致

-   一致的術語能促進學習，並進一步瞭解技術概念。 不一致會強制使用者找出不同的單字和動作是否代表相同的東西。

    範例：

    切換、切換

    啟動、執行、啟動、開機、執行

    啟用、啟動、開啟

    燒錄、複製

-   一致的語法可協助設定使用者的期望。 一旦設定這些期望之後，使用者可以更快速地剖析使用一致語法的文字。 例如，如果指示一律以命令式形式寫入，使用者將會學到更留意命令式句子。

### <a name="contractions"></a>簡寫

-   縮寫出借了較短、更快速得到、更具對話的節奏來撰寫。 在適當的內容中使用它們。 請勿使用縮寫搭配產品名稱或其他適當的名詞。

### <a name="colloquialisms-idioms-and-slang"></a>Colloquialisms、慣用語和俚語

-   請考慮只在特殊情況下使用 colloquialisms 或俚語，例如產品導覽、設定畫面或未當地語系化的內容。 最近的研究顯示，使用者喜歡非預期的單字或熟悉的片語。 請記住，使用 colloquialisms 和俚語可能會很難且昂貴地轉譯，因此這類語言最好謹慎使用。

    **範例：**

    另一方面，登錄中的某些值不應變更。

    使用 SDSL 時，這條線專用於您的網際網路使用，因此您不會在網際網路上遇到尖峰時間的流量，因為您可能會使用纜線服務。

### <a name="person"></a>個人

-   **您可以直接或間接處理使用者。**
-   **使用第二個人 (您的) 告訴使用者該怎麼做。** 通常會隱含第二個人。

    **範例：**

    選擇您要列印的圖片。

    選擇帳戶。  (隱含) 

-   **使用第一個人 (我，我的) 讓使用者告訴程式該怎麼做。**

    **範例︰**

    列印相機上的相片。

-   **請謹慎使用。** 第一個人複數可以建議公司的公司存在。 不過，最好是使用您的應用程式名稱。 建議您使用我們建議的建議，而不是建議的作法。
-   **避免 (使用者) 的協力廠商參考** ，因為他們建立了更正式、較不的個人音調。

### <a name="voice"></a>語音

-   **使用作用中的語音，** 強調執行動作的人員或動作。 它比被動聲音更直接且個人，可能會令人困惑或音效。

    **可以接受：**

    圖示可以依名稱以字母順序排列。

    當個人數位助理 (PDA) 或膝上型電腦插入時 .。。

    **較佳：**

    您可以依圖示名稱依字母順序排列圖示。

    當您插入任何個人數位助理 (PDA) 或膝上型電腦 .。。

-   您只能使用被動聲音來避免冗長或不容易的結構;當動作（而非惡人之手）是句子的焦點時，當主體未知時，或在錯誤訊息中，當使用者為主旨時，如果使用的是使用中的聲音，可能會覺得應該過度錯誤。

    **正確：**

    新圖示應該會出現在左上角。

    您必須先下載並安裝更新，才能運作。

-   **正面形式的片語陳述**，並強調使用者可以完成的動作，而不是他們無法完成的工作。

### <a name="attitude-toward-the-user"></a>針對使用者的態度

-   **有禮貌的、可獲得的和鼓勵。** 使用者絕對不應該覺得 condescended、應該過度或 intimidated。

    **可以接受：**

    無法刪除新的文字檔：拒絕存取。

    **較佳：**

    此檔案受到保護，無法在沒有特定許可權的情況下刪除。

-   **達到適當的平衡：當使用者不太熟悉或太過類似業務的情況下，請向使用者暖。** 假設您是在第一次協助朋友使用產品。 這位人員不是您最好的朋友或配偶/同居人，而是鄰近或家人的朋友。 使用您的程式時，使用者應該覺得很方便且在家中，但該語言不應該覺得 presumptuous 或太陌生。
-   **請限制在某些情況下會造成使用者不便的情況，** 例如：

    -   系統會要求使用者進行一些不方便的動作，例如等待、重複工作或更新程式。

        **正確：**

        請稍候，Windows 正在將檔案複製到您的電腦。

    -   因為缺少功能、設計瑕疵或程式 bug，所以使用者無法完成工作。

        **正確：**

        無法使用指定的格式儲存。 請選取另一個格式。

    -   使用者已經離開他或她很有説明，方法是參與客戶意見反應計畫或提出錯誤報表。

        **正確：**

        若要改善 Fabrikam 備份體驗，請參與 Fabrikam 客戶意見反應計畫。

    您也應該在每次不會被視為 curt 時使用。

    ![從 [開始] 的螢幕擷取畫面 ](images/text-style-tone-image1.png)

    在此範例中，內嵌錯誤訊息會 curt，而不需要。

-   請 **只在錯誤訊息中使用抱歉，這會導致使用者發生嚴重的問題** (例如資料遺失、使用者無法繼續使用電腦，或是使用者必須從技術代表) 取得協助。 如果程式正常運作期間發生此問題，請不要深感抱歉 (例如，如果使用者需要等候) 的網路連接。

    **正確：**

    很抱歉，Fabrikam 備份偵測到無法復原的問題，並已關閉。

### <a name="sentence-structure-and-length"></a>句子結構和長度

-   因為使用者通常會掃描文字，所以 **請讓每個字數都算。** 簡單、簡潔的句子 (和段落) 不僅能節省螢幕上的空間，也是傳達概念或動作很重要的最有效方法。 使用您的最佳判斷，讓句子更緊密，但不太嚴格，也就是色調看似突然惡意。
-   **避免重複。** 檢查每個視窗，並消除重複的單字和語句。 請勿避免重要的文字，請在必要時進行明確的，但不是多餘的，也不會解釋未說出的東西。
-   **如有需要，請使用句子片段。** 句子片段很簡短和 punchy，而且通常會採用詢問式形式，而是直接吸引使用者的好方法。

    **正確：**

    將變更儲存至「我的照片」？

    儲存檔案，然後不記得儲存檔案的位置嗎？

-   **使用結合來啟動句子** (以及) （如果需要的話）。
-   **以複雜句子替代清單和資料表。** 列出 (編號或點符) 和資料表是否更清楚且更容易掃描。
-   **使用平行語法結構。** 平行處理原則需要具有相同函式的單字和片語具有相同的格式。 當您表達相等權數的想法，以及在函式 (（例如標題、標籤、清單或頁面標題) ）中平行的 UI 元素時，請使用 [平行語言]。

    **正確：**

    接聽

    觀看

    共用

    Collect

    這些專案是平行的，因為它們都是單一單字的命令式動詞。

    **不正確：**

    音樂

    影片

    共用

    接聽

    這些專案不是平行的，因為音樂和影片是名詞，但共用和聆聽是動詞。
