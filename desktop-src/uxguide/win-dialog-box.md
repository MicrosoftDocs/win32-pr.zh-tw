# <a name="dialog-boxes-design-basics"></a> (設計基本概念的對話方塊) 

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

對話方塊是一種次要視窗，可讓使用者執行命令、詢問使用者問題，或是為使用者提供資訊或進度的意見反應。

![螢幕擷取畫面的識別對話方塊元素 ](images/win-dialog-box-image1.png)

一般的對話方塊。

對話方塊包含標題列 (用來識別對話方塊來自) 的命令、功能或程式、選擇性的主要指示 (以對話方塊來說明使用者的目標) 、內容區域中的各種控制項 (顯示選項) 和認可按鈕 (表示使用者想要如何認可至工作) 。

對話方塊有兩種基本類型：

-   **強制回應對話方塊** 需要使用者完成和關閉，才能繼續使用擁有者視窗。 這些對話方塊最適合用於需要完成的重要或不頻繁的一次性工作，再繼續進行。
-   非 **強制回應對話方塊** 可讓使用者視需要在對話方塊和擁有者視窗之間切換。 這些對話方塊最適合用來進行頻繁、重複的持續工作。

**工作對話方塊是使用工作對話方塊應用程式開發介面 (API) 來執行的對話方塊。** 它們是由下列元件所組成，這些元件可以組合成各種不同的組合：

-   **標題列**，用來識別對話方塊來源的應用程式或系統功能。
-   具有選擇性圖示的 **主要指示**，用來識別使用者在對話方塊中的目標。
-   描述資訊和控制項的 **內容區域** 。
-   認可按鈕的 **命令區域** ，包括 [取消] 按鈕，以及其他選擇性的選項但不顯示 <item> 同樣的控制項。
-   選擇性的其他說明和說明的 **註腳區域** ，通常以較不具經驗的使用者為目標。

![一般工作對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image2.png)

一般工作對話方塊。

**建議您盡可能使用工作對話方塊，因為這些對話方塊很容易建立，且能達到一致的外觀。** 工作對話方塊的確需要 Windows Vista 或更新版本，因此不適用於舊版的 Microsoft Windows。

工作窗格就像對話方塊一樣，不同之處在于它會在視窗窗格中顯示，而不是在另一個視窗中顯示。 如此一來，工作窗格就會有比對話方塊更直接、內容相關的感覺。 雖然技術上來說並不相同，但是工作 **窗格也類似于在本文中提供其指導方針的對話方塊**。

![一般工作窗格的螢幕擷取畫面 ](images/win-dialog-box-image3.png)

典型的工作窗格。

[[屬性視窗](win-property-win.md)] 是一種特製化的對話方塊類型，可用來查看和變更物件、物件集合或程式的屬性。 此外，屬性視窗通常支援數個工作，而對話方塊通常支援工作中的單一工作或步驟。 因為其使用方式是特製化的，所以 **屬性視窗會以一組不同的指導方針來涵蓋**。

對話方塊 [可以有索引](ctrl-tabs.md)標籤，如果是，則稱為索引標籤式對話方塊。 屬性視窗取決於屬性的內容呈現，而不是使用索引標籤。

**注意：** 與 [版面](vis-layout.md)配置、 [視窗管理](win-window-mgt.md)、通用對話方塊、 [屬性視窗](win-property-win.md)、 [嚮導](win-wizards.md)、 [確認](mess-confirm.md)、 [錯誤訊息](mess-error.md)和 [警告訊息](mess-warn.md) 相關的指導方針會在個別的文章中顯示。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

若要決定使用時機，請考量下列問題：

-   **這是為使用者提供資訊、詢問使用者問題，還是允許使用者選取選項來執行命令或工作的目的？** 如果沒有，請使用 (UI) 的其他使用者介面。
-   **這是用來查看和變更物件、物件集合或程式之屬性的用途嗎？** 如果是的話，請改用 [屬性視窗](win-property-win.md) 或 [工具列](cmd-toolbars.md) 。
-   **用來呈現命令或工具集合的目的是什麼？** 如果是的話，請使用工具列或 [ [調色板] 視窗](glossary.md)。
-   **驗證使用者是否想要繼續進行動作的目的是要驗證嗎？** 是否有清楚的理由不會繼續進行，而且有時候使用者可能不會有合理的可能性？ 若是如此，請使用 [確認](mess-confirm.md)。
-   **提供錯誤或警告訊息的目的是什麼？** 如果是的話，請使用 [錯誤訊息](mess-error.md) 或 [警告訊息](mess-warn.md)。
-   的目的是要：
    -   開啟檔案
    -   儲存檔案
    -   開啟資料夾
    -   尋找或取代文字
    -   列印檔案
    -   選取列印頁面的屬性
    -   選取字型
    -   選擇色彩
    -   流覽檔案、資料夾、電腦或印表機
    -   搜尋 Microsoft Active Directory 中的使用者、電腦或群組
    -   提示輸入使用者名稱和密碼嗎？

若是如此，請改用適當的 [通用對話方塊](win-common-dlg.md) 。 其中許多常見的對話方塊都是可擴充的。

-   **執行需要多個單一視窗的多步驟工作的目的是什麼？** 若是如此，請改用工作 [流程](glossary.md) 或 [wizard](win-wizards.md) 。
-   **這是用來通知使用者與目前使用者活動不相關的系統或程式事件的目的，而不需要立即的使用者動作，而且使用者可以自由地略過？** 若是如此，請改為使用 [通知](mess-notif.md) 。
-   **顯示程式狀態的目的是什麼？** 若是如此，請改用 [狀態列](ctrl-status-bars.md) 。
-   **最好使用就地 UI 嗎？** 對話方塊可以藉由要求注意來中斷使用者的流程。 有時候，流程中的中斷會對齊，例如，當使用者必須執行目前內容以外的動作時。 在其他情況下，更好的方法是將 UI 呈現在內容中，不論是直接使用就地 UI (（例如工作窗格) ），或是使用 [漸進式洩漏](ctrl-progressive-disclosure-controls.md)的隨選。
-   **顯示非關鍵性使用者輸入問題或特殊條件的目的為何？** 如果是的話，請改用 [球形球形](ctrl-balloons.md) 。
-   **針對工作流程，最好使用其他頁面嗎？** 一般來說，您會想要讓工作在單一視窗內從頁面流向頁面。 您可以使用對話方塊來確認就地命令、取得就地命令的輸入，以及執行最適合在主要工作流程之外獨立完成的次要獨立工作。
-   **針對選取選項，使用者是否有可能變更選項？** 如果沒有，請考慮替代方案，例如：
    -   使用預設選項而不要求，但允許使用者稍後進行變更。
    -   提供具有選項的版本 (例如，在功能表中的 [ **列印 ...** ]) 以及沒有選項的版本 (例如，在工具列) 中 **列印** 。 一般情況下，工具列命令應該是立即的，避免顯示對話方塊。
-   **針對選取選項，是否有更簡單、更直接的方式來呈現選項？** 若是如此，請考慮替代方案，例如：
    -   使用 [分割按鈕](ctrl-command-buttons.md) 選取命令的變化。
    -   針對命令、核取方塊、選項按鈕和簡單列表使用子功能表。

![顯示功能表和子功能表的螢幕擷取畫面。](images/win-dialog-box-image4.png)

![功能表和子功能表的螢幕擷取畫面 ](images/win-dialog-box-image5.png)

在這些範例中，會使用子功能表，而不是對話方塊來進行簡單的選擇。

## <a name="design-concepts"></a>設計概念

適當地使用時，對話方塊是為您的程式提供強大功能和彈性的絕佳方法。 當誤用時，對話方塊可讓使用者更輕鬆地干擾使用者、中斷其流程，並讓程式感覺更容易使用。 **強制回應對話方塊會要求使用者注意。** 對話方塊通常比替代 Ui 更容易執行，因此通常會過度採用。

**當對話方塊的設計特性符合其使用方式時，對話方塊會最有效。** 對話方塊的設計主要取決於其用途 (可提供選項、提出問題、提供資訊或意見反應) 、輸入 (強制回應) 或非強制回應，以及 (必要的使用者互動、選擇性的回應或通知) ，而其使用方式主要取決於使用者或程式起始 (的內容、使用者動作的機率，以及顯示的頻率。

若要設計有效的對話方塊，請有效地使用下列元素：

-   對話方塊文字
-   主要指示
-   不要顯示此 <item> 同樣的選項

**如果您只做一件事 .。。**

確定您的對話方塊設計 (取決於其用途、類型和使用者互動) 符合其使用方式 (取決於其內容、使用者動作的機率，以及顯示) 的頻率。

## <a name="usage-patterns"></a>使用模式

對話方塊有數種使用模式：

-   問題對話方塊 (使用按鈕) 詢問使用者單一問題或確認命令，並在水準排列的命令按鈕中使用簡單的回應。
-   問題對話方塊 (使用命令連結) 詢問使用者單一問題或選取要執行的工作，並在垂直排列的命令連結中使用詳細的回應。
-   選擇對話方塊會為使用者提供一組選項，通常可以更完整地指定命令。 不同于問題對話方塊，選擇對話方塊可以提出多個問題。
-   進度對話方塊讓使用者在冗長的作業期間提供進度回饋 (超過五秒的) ，以及取消或停止作業的命令。
-   資訊對話方塊顯示使用者要求的資訊。

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **請勿使用可滾動的對話方塊。** 請不要使用需要在正常使用期間完全查看捲軸的對話方塊。 請改為重新設計對話方塊。 請考慮使用[漸進式洩漏](ctrl-progressive-disclosure-controls.md)[或索引](ctrl-tabs.md)標籤。
-   **沒有功能表列或狀態列。** 相反地，您可以直接在對話方塊本身或使用相關控制項上的內容功能表，提供命令和狀態的存取權。

    -   **例外狀況：** 當使用對話方塊來執行主視窗 (（例如公用程式) ）時，可以接受功能表列。

    **不正確：**

    ![具有功能表列之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image6.png)

    在此範例中，[尋找憑證] 是具有功能表列的非強制回應對話方塊。

-   如果對話方塊需要立即注意，而且程式未在使用中，請將 **其工作列按鈕閃爍三次，以醒目提示，並將其反白顯示。** 請勿執行任何動作：請勿還原或啟用視窗，也不會播放任何音效效果。 相反地，請考慮使用者的視窗狀態選擇，並讓使用者在準備就緒時啟動視窗。
-   如需更多指導方針與範例，請參閱 [工作列](winenv-taskbar.md)。

### <a name="modal-dialog-boxes"></a>強制回應對話方塊

-   **用於需要完成的重要或不頻繁的一次性工作，再繼續進行。**
-   使用 [延遲的認可模型](glossary.md) ，讓變更在明確認可之前不會生效。
-   **在適當時使用工作對話方塊來達成一致的外觀。** 工作對話方塊的確需要 Windows Vista 或更新版本，因此不適用於舊版的 Windows。

### <a name="modeless-dialog-boxes"></a>非強制回應對話方塊

-   **使用於頻繁、重複的持續工作。**
-   使用 [立即認可模型](glossary.md) ，讓變更立即生效。
-   若為非強制回應對話方塊，請使用對話方塊中的明確關閉命令按鈕來關閉視窗。 針對這兩個專案，請使用標題列上的 [關閉] 按鈕來關閉視窗。
-   **請考慮使非強制回應對話方塊可停駐。** 可停駐的非強制回應對話方塊允許更具彈性的放置。

![可停駐、非強制回應對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image7.png)

Microsoft Office 中使用的某些非強制回應對話方塊是可停駐的。

### <a name="multiple-dialog-boxes"></a>多個對話方塊

-   **不要一次從擁有者選擇對話方塊顯示一個以上擁有的選擇對話方塊。** 顯示多個可讓使用者難以理解認可按鈕的意義。 您可以視需要顯示 (這類問題對話方塊) 的其他類型對話方塊。
-   **如需相關的對話順序，請考慮使用多頁對話方塊（如果可能的話）。** 如果沒有清楚相關，請使用個別的對話。

### <a name="multi-page-dialog-boxes"></a>多頁對話方塊

-   當您有下列相關頁面順序時，請使用多頁對話方塊，而不是個別的對話方塊：
    -    (選擇性的單一輸入頁面) 
    -   進度頁面
    -   單一結果頁面

輸入頁面是選擇性的，因為工作可能已在其他地方起始。 **這樣做會讓產生的體驗變得穩定、簡單、輕巧的感覺。**

![進度列的螢幕擷取畫面 ](images/win-dialog-box-image8.png)

![[找不到問題] 訊息的螢幕擷取畫面 ](images/win-dialog-box-image9.png)

在此範例中，Windows 網路診斷是由進度和結果頁面所組成。

-   **如果輸入頁面是標準對話方塊，請勿使用多頁對話方塊。** 在此情況下，使用標準對話方塊的一致性比較重要。
-   **請勿使用 [下一步] 或 [上一頁] 按鈕，而且沒有三個頁面。** 多頁對話方塊適用于具有意見反應的單一步驟工作。 它們不是用來進行多步驟工作的「 [嚮導](win-wizards.md)」。 與多頁對話方塊相較之下，嚮導會有相當大的間接風格。
-   **在 [輸入] 頁面上，使用特定的命令按鈕或命令連結來起始工作。**
-   **使用 [輸入] 和 [進度] 頁面上的 [取消] 按鈕，以及 [結果] 頁面上的 [關閉] 按鈕。**

**開發人員：** 您可以使用 [TDM \_ 流覽 \_ 頁面](../controls/tdm-navigate-page.md) 訊息來建立多頁工作對話方塊。

### <a name="presentation"></a>簡報

若要讓對話方塊易於尋找和存取，可清楚地將對話與其來源建立關聯，並與多個監視器搭配運作：

-   **一開始會在主控視窗頂端顯示「置中」對話方塊。** 針對後續顯示，請考慮將它顯示在其最後一個位置 (相對於擁有者視窗) 如果這麼做，可能會比較方便。

![以視窗背後的視窗為中心的對話方塊圖表 ](images/win-dialog-box-image10.png)

最初將對話置中在主控視窗上方。

-   **如果對話方塊為內容，則會將它顯示在其啟動所在的物件附近。** 不過，請將它放在 (最好的位移和右邊的) ，讓對話方塊不會涵蓋物件。

![對話方塊位移向下和向右的圖表 ](images/win-dialog-box-image11.png)

物件的屬性會顯示在物件附近。

-   **若為非強制回應對話方塊，請先在主控視窗頂端顯示，以方便尋找。** 如果使用者啟動擁有者視窗，可能會混淆非強制回應對話方塊。
-   **如有必要，請調整初始位置，讓整個對話方塊在目標監視器內都能看見。** 如果可調整大小的視窗大於目標監視器，請將它縮小為適當大小。
-   **重新顯示對話方塊時，請考慮將它顯示為與上次存取相同的狀態。** 關閉時，儲存所使用的監視器、視窗大小、位置和狀態 (最大化與還原) 。 重新顯示時，使用適當的監視來還原已儲存的對話方塊大小、位置和狀態。 此外，請考慮以每個使用者為基礎，將這些屬性保存在程式實例之間。
-   **若為可調整大小的視窗，請設定最小的視窗大小（如果大小低於此值，內容就無法再使用）。** 請考慮改變簡報，讓內容可以使用較小的大小。

![中央媒體播放機按鈕的螢幕擷取畫面 ](images/win-dialog-box-image12.png)

在此範例中，Windows Media Player 會在視窗對標準格式變得太小時變更其格式。

-   **請勿使用 Always on Top 屬性。**
    -   **例外狀況：** 只有在對話方塊執行基本上是強制回應的作業時才使用，但必須短暫暫停才能存取主控視窗。 例如，在檢查檔時，使用者可能偶爾會離開 [拼寫檢查] 對話方塊，並存取檔以更正錯誤。

如需詳細資訊和範例，請參閱 [視窗管理](win-window-mgt.md)。

### <a name="title-bars"></a>標題列

-   **對話方塊沒有標題列圖示。** 標題列圖示會用來做為 [主視窗](glossary.md) 與 [次要視窗](glossary.md)之間的視覺差異。
    -   **例外狀況：** 如果使用對話方塊來執行主視窗 (例如公用程式) ，因此會出現在工作列上，它的確有標題列圖示。 在這種情況下，請先精確地放置區分的資訊，以優化工作列上顯示的標題。
-   **對話方塊一律會有 [關閉] 按鈕。** 非強制回應對話方塊也可以有最小化按鈕。 可調整大小的對話方塊可以有最大化按鈕。
-   **請勿停用 [關閉] 按鈕。** 有了 [關閉] 按鈕，可讓使用者關閉他們不想要的視窗，以協助使用者保持控制。
    -   **例外狀況：** 在 [進度] 對話方塊中，如果工作必須執行到完成才能達成有效的狀態或防止資料遺失，您可以停用 [關閉] 按鈕。
-   **標題列上的 [關閉] 按鈕應該與對話方塊內的 [取消] 或 [關閉] 按鈕具有相同的效果** 。 請勿為它提供與「確定」相同的效果。
-   如果標題列標題和圖示已經以顯著的方式顯示在靠近視窗頂端的上方，您可以隱藏標題列標題和圖示來避免重複。 不過，您仍然必須在內部設定適當的標題，以供 Windows 使用。

### <a name="interaction"></a>互動

-   **顯示時，使用者起始的對話方塊應該一律採用輸入焦點。** 程式起始對話方塊不應取得輸入焦點，因為使用者可能會與另一個視窗互動。 在對話方塊中不當導向的互動可能會產生非預期的結果。
-   **將初始輸入焦點指派給使用者最有可能與第一個** 互動的控制項（通常是 (，但不一定會) 第一個互動式控制項）。 避免將初始輸入焦點指派給說明連結。
-   **針對鍵盤流覽，定位順序應該以邏輯順序流動，通常是由左至右，從上到下。** 索引標籤順序通常會遵循讀取順序，但請考慮進行下列例外狀況：

    -   將最常使用的控制項放在定位順序的前面。
    -   在定位順序中的 [認可] 按鈕之後，將 [說明] 連結放在對話方塊底部。

    指派訂單時，假設使用者針對其預定用途顯示對話方塊;例如，使用者會顯示選擇對話方塊來進行選擇，而不是檢查並按一下 [取消]。

-   **按下 Esc 鍵時，一律會關閉作用中的對話方塊。** 這適用于具有 [取消] 或 [關閉] 的對話方塊，即使取消已重新命名為 [關閉]，因為結果無法再復原。

**存取金鑰**

-   **請盡可能將唯一的存取金鑰指派給所有互動式控制項或其標籤。**[唯讀文字方塊](ctrl-text-boxes.md) 是互動式控制項 (因為使用者可以將文字滾動並複製文字，) 讓它們受益于存取金鑰。 **請勿將存取金鑰指派給：**
    -   **[確定]、[取消] 和 [關閉] 按鈕。** 輸入和 Esc 可用於其存取金鑰。 不過，請一律將便捷鍵指派給表示 [確定] 或 [取消]，但具有不同標籤的控制項。

        ![[刪除檔案] 對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image13.png)

        在此範例中，正認可按鈕具有指派的存取金鑰。

    -   **群組標籤。** 一般情況下，群組內的個別控制項都會被指派存取金鑰，因此群組標籤不需要一個。 但是，如果存取金鑰不足，請將存取金鑰指派給群組標籤，而不是個別控制項。
    -   以 F1 存取的一般 [說明]**按鈕**。
    -   **連結標籤。** 通常會有太多連結可指派唯一的存取金鑰，而用來表示連結的底線通常會隱藏存取金鑰底線。 改為使用 Tab 鍵存取連結。
    -   **Tab 鍵名稱。** 索引標籤會使用 Ctrl + Tab 和 Ctrl + Shift + Tab 鍵來迴圈。
    -   **流覽標示為「...」的按鈕。** 這些瀏覽按鈕無法唯一指派存取金鑰。
    -   未 **標記的控制項，** 例如微調控制項、圖形命令按鈕，以及未標記的漸進式洩漏控制項。
    -   **非互動式控制項（例如進度列）的非標籤靜態文字或標籤** 。

-   **如果可能，請根據標準存取金鑰指派，指派常用命令的存取金鑰**。 雖然不一定能夠維持一致的存取金鑰指派，但特別是常用的對話方塊。
-   **先指派認可按鈕存取金鑰，以確保它們具有標準金鑰指派。** 如果沒有標準金鑰指派，請使用第一個單字的第一個字母。 例如，不論對話方塊中的其他控制項為何，[是] 和 [無認可] 按鈕的存取金鑰都應該一律為 "Y" 和 "N"。
-   **若要讓存取金鑰易於尋找，請將存取金鑰指派給在標籤中提早出現的字元，** 最好是第一個字元，即使標籤稍後出現關鍵字也是如此。
-   **偏好寬寬度的字元，** 例如 w、m 和大寫字母。
-   **偏好使用特殊的輔音或母音，** 例如 "x" 結束。
-   **避免使用使底線難以查看的字元，例如，** 從最有問題的 (到最不有問題的) ：
    -   只有一個圖元寬度的字母，例如 i 和 l。
    -   具有下行的字母，例如 g、j、p、q 和 y。
    -   包含下行的字母旁的字母。

如需更多指導方針與範例，請參閱 [鍵盤](inter-keyboard.md)。

### <a name="progress-dialogs"></a>進度對話方塊

若為長時間執行的工作，請 **假設使用者會在工作完成時執行其他** 作業。 設計要自動執行的工作。

-   **如果作業花費的時間超過五秒，則顯示具有進度意見反應對話方塊的使用者**，以及取消或停止作業的命令。
    -   **例外狀況：** 針對 [嚮導] 和 [工作流程]，只有在工作維持在相同的頁面上 (時，才會使用強制回應對話方塊進行處理，而不是前進到另一個頁面) 而且使用者在等待時不會執行任何動作。 否則，請使用進度頁面或就地進度。
-   如果作業是長時間執行的工作 (超過30秒) 並可在背景中執行，請使用非模式進度對話方塊，讓使用者可以在等候時繼續使用您的程式。
-   無模式進度對話方塊：
    -   在標題列上具有最小化按鈕。
    -   會顯示在工作列上。
-   執行非強制回應的進度對話方塊，讓它們繼續執行，即使擁有者視窗已關閉也一樣。

![具有進度列之 [複製] 對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image14.png)

在此範例中，即使擁有者視窗已關閉，仍會繼續進行檔案複製。

-   **如果需要超過幾秒鐘的時間才能完成作業，或有可能永遠無法完成，請提供命令按鈕來停止作業。** 如果取消將環境傳回至先前的狀態，請將按鈕標示為 [取消]， (不會造成任何副作用) ;否則，請將按鈕標示為停止，表示它會讓部分完成的作業保持不變。 您可以將按鈕標籤從 [取消] 變更為在作業過程中停止，如果某個時間點，則無法將環境回復為先前的狀態。

![具有 [取消] 按鈕之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image15.png)

在此範例中，暫停問題診斷沒有副作用。

-   **如果需要超過數分鐘才能完成作業，請提供命令按鈕來暫停作業，並讓使用者能夠完成工作。** 這樣做並不會強制使用者在完成工作和完成工作之間進行選擇。
-   **在開始工作之前，盡可能收集最多的資訊。**
-   **如果偵測到可復原的問題，請讓使用者處理在工作結束時所發現的所有問題。** 如果這不可行，請讓使用者在發生問題時處理問題。
-   **請勿放棄復原錯誤的結果。**

![對話方塊的螢幕擷取畫面，其中包含 [再試一次] 按鈕 ](images/win-dialog-box-image16.png)

在此範例中，Windows 檔案總管可讓使用者在可復原的錯誤之後繼續執行工作。

-   **藉由將進度列變成紅色來指出問題。**

![進度列的螢幕擷取畫面，然後再試一次按鈕 ](images/win-dialog-box-image17.png)

在此範例中，會在檔案複製期間移除卸載式磁片。

-   **如果使用者清楚看出結果，請在成功完成時自動關閉進度對話方塊。** 否則，請只使用意見反應來回報問題：
    -   若要顯示簡單的意見反應，請在 [進度] 對話方塊中顯示意見反應，然後變更 [取消] 按鈕以關閉。
    -   若要顯示詳細的意見反應，請關閉 [進度] 對話方塊並顯示資訊對話方塊。

**請勿使用通知完成意見反應。** 您可以使用進度對話方塊或 [動作成功通知](mess-notif.md)，但不能同時使用兩者。

### <a name="command-links"></a>命令連結

-   **使用命令連結，而非命令按鈕或選項按鈕和 [確定] 按鈕的組合，來呈現一組冗長的命令。** 這樣做可讓使用者只需按一下就能回應。 不過，此方法只適用于單一問題。
-   **先顯示最常使用的命令連結。** 產生的順序大致上應該遵循使用的可能性，但也有邏輯流程。
    -   **例外狀況：** 導致執行所有工作的命令連結應先放置。
-   如果命令連結需要進一步的說明，請 **提供補充說明。** 補充說明說明使用者可能會想要選擇命令的原因，或如果選擇命令會發生什麼事。
-   **請勿使用冗長重述命令連結的補充說明。** 只有當您無法讓命令連結一目了然時，才使用補充說明。 提供一個命令連結的補充說明，並不表示您必須為所有命令提供它們。

![具有文字注意選項之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image40.png)

-   **使用以動詞開頭的片語，不含結束標點符號。**
-   **如果是強烈建議的命令，請考慮將「 (建議的) 」新增至標籤。** 請務必新增至連結標籤，而不是補充說明。
-   **如果命令僅供 advanced 使用者使用，請考慮將「 (advanced) 」新增至標籤。** 請務必新增至連結標籤，而不是補充說明。
-   **一律提供明確的 [取消] 按鈕**。 請勿將命令連結用於此用途。

-   **使用這個特定的措辭取代 <item> 為特定的專案。** 例如，不要再顯示此提醒。 在一般情況下參考對話方塊時，請使用 [不要再顯示此訊息]。
-   藉由在選項下新增下列句子，**清楚地指出使用者輸入將用於未來預設值**：未來將會使用您的選取專案。

在此範例中，訊息只會顯示一次。 不需要詢問。

### <a name="ask-me-later"></a>稍後詢問我

-   只有在下列情況下，才提供此選項來關閉對話方塊：
    -   **對話方塊是間接的**，因此使用者可能會專注于另一項工作。
    -   **使用者必須回應但不能立即** 進行，以便他們繼續工作。
    -   **問題需要足夠的想法或努力** ，讓使用者可以在有更多時間時做出更好的決策。
    -   **對話方塊或選項稍後會自動顯示** (，讓使用者在稍後) 。
-   **不正確：**
-   ![具有 [稍後詢問我] 選項之 [訊息的螢幕擷取畫面] 選項 ](images/win-dialog-box-image43.png)
-   在此範例中，問題很簡單，只要新增 [稍後詢問我] 選項就會讓它變得更複雜。

-   **請勿使用更多/較少的控制項來顯示說明。** 請改用說明連結或註腳。

### <a name="disabling-or-removing-controls-vs-giving-error-messages"></a>停用或移除控制項與提供錯誤訊息

-   當控制項未套用至目前的內容時，請考慮下列選項：
    -   **當使用者無法啟用控制項時，請移除該控制項，否則使用者不會預期它會套用，而且其狀態不會經常變更。** 這麼做可簡化對話方塊，使用者也不會錯過。 具有控制項的出現和經常消失很討厭。
    -   **當使用者預期應用程式會經常套用或其狀態變更時，請停用該控制項，而且使用者可以輕鬆地推算控制項停用的原因。** 如果有單一的空白文字方塊需要任何輸入，則很容易推算的範例會停用認可按鈕。 您可以使用 [氣球](ctrl-balloons.md) 來顯示非關鍵性的使用者輸入問題，以及文字方塊和可編輯的下拉式清單。 但是，如果無法使用氣球來說明問題，或是牽涉到多個控制項，則推算將不再很簡單。
    -   **否則，請讓控制項保持啟用狀態，但使用錯誤訊息時，請提供錯誤訊息。** 在此情況下停用會讓使用者難以瞭解控制項的停用原因。 使用者會被迫透過實驗和 deductive 邏輯來判斷問題。 最好是提供實用的錯誤訊息，以明確地說明問題。
-   **秘訣：** 如果您不確定是否應該停用控制項或提供錯誤訊息，請從撰寫您可能提供的錯誤訊息開始。 如果錯誤訊息包含目標使用者無法快速推算的實用資訊，請讓控制項保持啟用並提供錯誤。 否則，請停用控制項。
-   **如果您停用控制項，也會停用所有相關聯的控制項**，例如其標籤、補充說明或命令按鈕。 不過，請勿停用其 [群組方塊](ctrl-group-boxes.md)、群組標籤或群組說明（如果有的話）。

![具有暗灰色控制項之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image45.png)

在此範例中，已停用的文字方塊標籤也會停用，但其群組標籤和群組說明則否。

### <a name="required-input"></a>必要輸入

-   若要指出使用者必須在控制項中提供資訊，請考慮下列選項：
    -   **請勿指出任何問題，但會處理遺失的必要輸入，並顯示錯誤訊息。** 如果大部分的輸入都是選擇性的，或是使用者不太可能略過控制項，因而讓錯誤訊息的數目保持不變，這種方法會減少雜亂的情況。
    -   **使用標籤開頭的星號表示必要輸入。** 使用下列其中一項解說星號：

        -   內容區域底部的註腳，指出 \* 必要的輸入。
        -   星號上指出必要輸入的工具提示。

        如果沒有許多必要的控制項，這種方法就很好用，但如果大部分的控制項都需要，則會很不良。

        ![包含星號之文字方塊標籤的螢幕擷取畫面 ](images/win-dialog-box-image46.png)

        在此範例中，星號會用來指出所需的輸入。

    -   **如果所有控制項都需要輸入，請在內容區域頂端的適當位置，「需要所有輸入」狀態。** 這種方法可減少此特定案例的雜亂情形。
    -   **使用標籤之後的 " (選擇性) " 表示選擇性輸入。** 如果需要大部分的輸入，這種方法就很好用，但不太可能。

-   **為求一致，請嘗試使用相同的方法來表示整個程式中所需的輸入。** 具體來說，請視需要指定必要或選擇性的輸入，但請避免在同一個程式中使用這兩種方法。

### <a name="error-handling"></a>錯誤處理

-   使用受限制為有效使用者輸入的控制項來防止錯誤。 您也可以藉由提供合理的預設值來協助減少錯誤數目。
-   請儘快驗證使用者輸入，並盡可能將錯誤顯示為最接近輸入點。
-   **針對使用者輸入問題，使用非強制回應錯誤處理 (就地錯誤或氣球) 。**
    -   **在文字方塊中或在文字方塊失去焦點之後，于文字方塊中偵測到非關鍵性、單一點使用者輸入問題時，請使用球標。** 氣球不需要可用的螢幕空間或顯示就地訊息所需的動態配置。 一次只顯示一個氣球。 因為此問題並不重要，所以不需要任何錯誤圖示。 當您按下、解決問題或在超時後，就會離開氣球。

        ![「不正確的字元」訊息的螢幕擷取畫面 ](images/win-dialog-box-image47.png)

        在此範例中，氣球表示輸入問題仍然存在於控制項中。

-   **使用就地錯誤偵測延遲的錯誤偵測**，通常是按一下 [認可] 按鈕找到的錯誤。  (不會在立即認可的設定中使用就地錯誤。 ) 一次可能會有多個就地錯誤。 使用一般文字和16x16 圖元錯誤圖示，盡可能將它們直接放在問題旁。 除非使用者認可，且找不到任何其他錯誤，否則就地錯誤不會消失。

    ![有兩個錯誤訊息之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image48.png)

    在此範例中，按一下 [認可] 按鈕時，會使用就地錯誤來進行找到。

-   針對其他所有問題（包括牽涉到多個控制項的錯誤），以及按一下 [認可] 按鈕時所發現的非內容或非輸入錯誤 **，使用 (工作對話方塊或消息) 框的模式錯誤處理**。
-   **當發現並報告輸入問題時，請將輸入焦點設定為具有不正確資料的第一個控制項。** 必要時，將控制項滾動至 view。

如需詳細資訊和範例，請參閱 [錯誤訊息](mess-error.md) 和 [批註](ctrl-balloons.md)提示。

### <a name="help"></a>說明

-   提供使用者協助時，請考慮下列選項 (依照其喜好設定) 順序列出：
    -   提供互動式控制項自理解標籤。 使用者比較有可能讀取互動式控制項上的標籤，而不是任何其他文字。
    -   使用 [靜態文字標籤](text-ui.md)提供內容中的說明。
    -   提供特定說明主題的特定說明連結。
-   **在對話方塊的內容區域底部找出說明連結。** 如果對話方塊有註腳，而且說明連結與其相關，請將 [說明] 連結放在註腳內。

    ![具有說明連結之對話方塊的螢幕擷取畫面 ](images/win-dialog-box-image40.png)

    在此範例中，[說明] 連結會套用至整個對話方塊。

    -   **例外狀況：** 如果對話方塊有數個不同的設定群組，其中有個別的說明主題 (可能在群組方塊) 內，請找出群組底部的 [說明] 連結。

-   **請勿使用一般或模糊的說明主題連結或一般説明按鈕。** 使用者通常會忽略一般說明。

如需詳細資訊和範例， [請參閱說明](winenv-help.md)。

### <a name="default-values"></a>預設值

-   在每個對話方塊中包含預設的 [認可] 按鈕。
-   針對問題對話方塊：
    -   **選取最安全的 (，以防止遺失資料或系統存取) ，最安全的回應是預設值。** 如果安全性和安全性不是因素，請選取最有可能或方便的回應。
        -   **例外狀況：** 除非有簡單、明顯的方法可復原命令，否則請勿將破壞性回應設為預設值。
-   針對選擇對話方塊：
    -   針對初始預設值，請 **選取最安全的 (，以防止遺失資料或系統存取) 以及每個控制項最安全的值。** 如果安全性和安全性不是因素，請選取最有可能或方便的選項。
    -   針對後續的預設值，請重新 **選取先前選取的選項（如果這些值可能會重複），這樣做就是安全且安全的。** 否則，請選取初始預設值。

## <a name="recommended-sizing-and-spacing"></a>建議的大小和間距

-   **支援 800 x 600 圖元的最小 Windows Vista 螢幕解析度。** 您可以使用 1024 x 768 圖元的螢幕解析度，針對可調整大小的視窗優化版面配置。
-   **請盡可能使用可調整大小的視窗，以避免捲軸和截斷的資料。** 具有動態內容和清單的 Windows，從可調整大小的視窗獲益最大。
-   **固定大小的 windows 必須是完全可見且大小調整，才能納入工作區域中。**
-   **可調整大小的視窗可能已針對較高的解析度進行優化，但在顯示時視需要調整為實際螢幕解析度的大小。**
-   **選擇適合其內容的預設視窗大小。** 如果您可以有效地使用空間，請不要害怕使用較大的初始視窗大小。

## <a name="text"></a>Text

### <a name="general"></a>一般

-   **移除多餘的文字。** 在標題、主要指示、補充指示、內容區域、命令連結和認可按鈕中尋找重複的文字。 一般而言，請在指示和互動式控制項中保留完整文字，然後從其他位置移除任何多餘的。
-   **使用正片語。** 對使用者來說，可以更輕鬆地理解正面片語。

**正確：**

要啟用檔案和印表機共用嗎？

**不正確：**

您要停用檔案和印表機共用嗎？

不過，片語必須符合相關聯的命令，即使命令有負面片語;例如，使用 disable 來確認 Disable 命令。

-   **如有必要，請使用 "window" 這個字來表示對話方塊本身。**
-   **使用第二個人 ( 「您/您」的 ) ，告訴使用者要** 在主要指示和內容區域中執行的動作。 通常會隱含第二個人。

**範例：**

選擇您要列印的圖片

選擇帳戶

-   **使用第一個 person ( "I/me/my" ) 讓使用者告訴程式要** 在內容區域中回應主要指令的控制項。

**範例：** 列印相機上的相片。

### <a name="dialog-box-titles"></a>對話方塊標題

-   **使用標題來識別對話方塊來源的命令、功能或程式。**
    -   如果對話方塊是使用者起始的，請使用命令或功能名稱來識別。 **異常：**
        -   如果有許多不同的命令顯示對話方塊，請考慮改為使用程式名稱。
        -   如果該標題在主要指示中是多餘的，請改為使用程式名稱。
    -   如果是程式或系統起始的 (，因此不在內容) 中，請使用程式或功能名稱提供內容來識別。
    -   請勿使用標題來說明在主要指令用途的對話方塊中要執行的動作。
-   使用命令名稱的確切命令名稱，但如果有的話，請不要包含省略號。 如有需要，您可以包含命令的功能表標題，以撰寫良好的標題。 範例：物件 .。。命令在 [插入] 功能表中，使用標題插入物件。
-   **如果工作列上出現非強制回應對話方塊，請先將區分的資訊先放在工作列上，以優化工作列上顯示的標題** 。 範例：「66% 完成」和「3個提醒」。
-   **請勿在標題中包含 "dialog" 或 "進度" 單字。** 這是隱含的，而不是讓使用者能夠更輕鬆地進行掃描。
-   使用 [標題樣式的大小寫](glossary.md)，但不含結束標點符號。

### <a name="main-instructions"></a>主要指示

-   **使用主要指令來清楚解說對話方塊中的動作。** 指令應該是特定的語句、命令式方向或問題。 良好的指示會使用對話方塊來傳達使用者的目標，而不是單純專注于操作它的機制。
-   **當您唯一可以說的是顯而易見的時候，請省略 main 指令。** 在這種情況下，對話方塊的內容很容易理解。 例如，[檔案開啟] 和 [檔案儲存] 通用對話方塊不需要主要指示，因為它們的內容和設計會讓它們的目的變得很明顯。
-   **省略重新聲明主要指令的控制項標籤。** 在此情況下，主要指令會採用存取金鑰。

**可以接受：**

![具有重複標籤的文字方塊螢幕擷取畫面 ](images/win-dialog-box-image50.png)

在此範例中，文字方塊標籤只是主要指令的重述。

**較佳：**

![具有一個標籤之相同文字方塊的螢幕擷取畫面 ](images/win-dialog-box-image51.png)

在此範例中，會移除多餘的標籤，所以主要指令會取得存取金鑰。

-   **請務必只使用單一的完整句子。** 削減主要指示以瞭解基本資訊。 如果您必須進一步說明，請使用補充指示。
-   **盡可能使用特定動詞。** 特定動詞 (範例： [連接]、[儲存]、[安裝) 比一般使用者更有意義， (範例：設定、管理、設定) 。
-   使用 [句子樣式的大小寫](glossary.md)。
-   **如果指令是語句，則不包含最後句號。** 如果指令是問題，請包含最後一個問號。
-   在 **[進度] 對話方塊中，使用現在片語來簡短說明進行中的作業，** 並以省略號結尾。 範例：列印您的圖片 .。。
-   **秘訣：** 您可以藉由想像您對朋友的看法來評估主要指示。 如果回應主要指示的非自然、無用或麻煩，請重新修改指令。

### <a name="supplemental-instructions"></a>補充指示

-   **必要時，請使用選擇性的補充指示，以提供有助於瞭解或使用頁面的其他資訊。** 您可以提供更詳細的資訊並定義術語。
-   **如果對話方塊的外觀是 [程式] 或 [系統起始 (]，因此不在內容) 中，請使用補充指示來說明對話方塊出現的原因。** 針對這類對話方塊，內容通常並不明顯。
-   **請勿以稍微不同的用語重複主要指令。** 相反地，如果沒有其他要新增的指令，請省略補充指令。
-   使用完整句子、句子樣式的大小寫和結束標點符號。

### <a name="command-links"></a>命令連結

-   **選擇清楚的連結文字，以明確地傳達和區分命令連結的用途。** 它應該是一目了然的，且對應至主要指令。 使用者不應該需要找出連結真正的意義，或與其他連結有何不同。
-   **一律使用動詞命令來啟動命令連結。**
-   使用句型大寫。
-   請勿使用結束標點符號。
-   **如有必要，請使用完整的句子和結束標點符號來提供任何進一步的說明。** 不過，只有在必要時才新增這類說明，因為一個命令連結需要一個命令連結的說明。

如需詳細資訊和範例，請參閱 [命令連結](ctrl-command-links.md) 指導方針。

### <a name="commit-buttons"></a>認可按鈕

-   **使用適當的特定認可按鈕標籤，並回應主要指令。** 在理想的情況下，使用者不需要讀取任何其他人來瞭解標籤。 使用者比靜態文字更可能讀取命令按鈕標籤。
-   **使用動詞啟動認可按鈕標籤。例外狀況是 [確定]、[是] 和 [否]。**
-   使用句型大寫。
-   請勿使用結束標點符號。
-   指派唯一的 [存取金鑰](glossary.md)。
    -   **例外狀況：** 請勿將存取金鑰指派給 [確定] 和 [取消] 按鈕，因為 Enter 和 Esc 是其存取金鑰。 這麼做可讓您更輕鬆地指派其他存取金鑰。
