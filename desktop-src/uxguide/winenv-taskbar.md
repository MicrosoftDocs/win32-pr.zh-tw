# <a name="taskbar"></a>工作列

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新版本的 Windows 更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

工作列是顯示在桌面上的程式存取點。 透過新的 Windows 7 工作列功能，使用者可以直接從工作列提供命令、存取資源，以及查看程式狀態。

工作列是顯示在桌面上的程式存取點，即使程式最小化也是如此。 這類程式稱為有桌上型電腦存在。 使用工作列時，使用者可以在桌面上查看開啟的主要視窗和特定的次要視窗，而且可以在兩者之間快速切換。

![工作列的螢幕擷取畫面，其中包含已叫用的功能 ](images/winenv-taskbar-image1.png)

Microsoft Windows 工作列。

工作列上的控制項稱為工作列按鈕。 當程式建立主視窗 (或具有特定特性的次要視窗) 時，Windows 會為該視窗新增工作列按鈕，並在該視窗關閉時將其移除。

針對 Windows 7 設計的程式可以利用這些新的工作列按鈕功能：

-   快速入門清單可讓您快速存取常用的目的地 (例如檔案、資料夾和連結) 和命令，可透過從程式的工作列按鈕存取的內容功能表，以及 [開始] 功能表專案，即使程式目前不在執行中也一樣。
-   縮圖工具列可讓您快速存取特定視窗常用的命令。 縮圖工具列會顯示在工作列按鈕的縮圖中。
-   覆迭圖示會顯示程式的工作列按鈕圖示上的狀態變更。
-   進度列會顯示程式的工作列按鈕上長時間執行之工作的進度。
-   子視窗的工作列按鈕可讓使用者使用工作列按鈕縮圖，直接切換到視窗索引標籤、專案視窗、多重文件介面 (MDI) 子視窗和次要視窗。
-   釘選的工作列按鈕可讓使用者將程式按鈕釘選到工作列，以提供程式的快速存取權，即使它們沒有執行也一樣。

技術上來說，工作列會將 [開始] 按鈕的整個橫條橫跨通知區域，不過，工作列較常見的只是指包含工作列按鈕的區域。 針對多個監視設定，只有一個監視器具有工作列，而該監視是預設監視。

**注意：** 與 [桌面](winenv-desktop.md)、 [通知區域](winenv-notification.md)和 [視窗管理](win-window-mgt.md) 相關的指導方針會顯示在不同的文章中。

## <a name="is-this-the-right-user-interface"></a>這是正確的使用者介面嗎？

針對 Windows 7 設計的程式可以利用這些工作列按鈕功能。 請詢問自己下列重要問題，以判斷是否要使用它們：

**跳躍清單**

-   **使用者通常需要使用您的程式啟動新工作嗎？** 如果是的話，請考慮提供捷徑清單。 雖然跳躍清單可以用於其他用途，但大部分案例都牽涉到開始新的工作。
-   **使用者通常需要存取最近或經常使用的檔案、資料夾、連結或其他資源？** 如果是的話，請考慮提供捷徑清單來存取這些有用的資源。

    ![使用 internet explorer 跳躍清單的工作列螢幕擷取畫面 ](images/winenv-taskbar-image2.png)

    在此範例中，Windows Internet Explorer 會使用捷徑清單來呈現經常造訪的頁面。

-   **當您使用其他程式時，即使您的程式未執行，使用者是否經常需要快速存取少量的程式命令？** 如果是的話，請考慮使用這些常用的命令來提供捷徑清單。 即使您的程式未執行，這些命令也必須正常運作，且必須套用至整個程式，而不是特定的視窗。 或者，請考慮提供適用于特定視窗之命令的縮圖工具列。

    ![具有粘滯便箋跳躍清單的工作列螢幕擷取畫面 ](images/winenv-taskbar-image3.png)

    在此範例中，自黏便箋配件可讓使用者在使用其他程式時快速建立新的附注。

-   **您是否正在升級新的、單一用途或難以尋找的功能？** 如果是，請勿使用跳躍清單，因為它們不是用於此用途。 請改為直接在程式中改善這類命令的可探索性。

**縮圖工具列**

是否適用下列所有條件？

-   **命令是否適用于特定的視窗？** 縮圖工具列適用于適用于現有工作的命令，而捷徑清單命令則用於啟動新的工作。
-   **使用其他程式時，使用者是否需要快速與執行中的工作互動？** 如果是，縮圖工具列是不錯的選擇。 縮圖工具列最多可顯示7個命令，但通常偏好最多五個命令。
-   **命令是否立即進行？** 也就是說，他們不需要額外的輸入嗎？ 縮圖工具列必須要有立即的命令才能有效率，而跳躍清單更適合需要額外輸入的命令。

    **不正確：**

    ![具有重迭視窗的工作列螢幕擷取畫面 ](images/winenv-taskbar-image4.png)

    需要額外輸入的命令不適用於縮圖工具列。

-   **命令是否 direct？** 也就是說，使用者只要按一下，就可以與他們互動嗎？ 工具列必須有直接命令才能有效率。
-   **命令是否適當地以圖示表示？** 縮圖工具列命令會使用不是文字標籤的圖示來呈現，而捷徑清單的命令則會以文字標籤表示。

    **不正確：**

    ![具有圖示的縮圖命令螢幕擷取畫面 ](images/winenv-taskbar-image5.png)

    在此範例中，命令未妥善地以圖示表示。

**重迭圖示**

-   **程式是否有「桌面存在」？** 如果沒有，請改用通知區域圖示。 若是如此，請考慮使用覆迭圖示，而不是在針對 Windows 7 設計之程式的通知區域圖標上放置狀態。 這麼做可確保在) 使用大型圖示時，圖示一律會顯示 (，並將該程式的狀態合併到一個位置。
-   **覆迭圖示是否會暫時顯示，以顯示狀態的變更？** 如果是的話，視下列因素而定，重迭圖示可能會適當：
    -   **使用其他程式時，狀態是否有用且相關？** 如果沒有，請在程式的 [ [狀態列](ctrl-status-bars.md) ] 或 [其他程式狀態] 區域中顯示資訊。

        ![internet explorer 視窗狀態列的螢幕擷取畫面 ](images/winenv-taskbar-image7.png)

        在此範例中，會使用狀態列，因為在使用其他程式時，狀態不會很有用。

    -   **狀態是否顯示進度？** 若是如此，請改用工作列按鈕的進度列。
    -   **狀態是否危急？是否需要立即採取動作？** 如果是的話，請以需要注意的方式來顯示資訊，而且無法輕易地忽略，例如 [對話方塊](win-dialog-box.md)。

**進度列**

-   **使用其他程式時，進度回饋是否有用且相關？** 也就是說，使用者可能會在使用其他程式時監視進度，並變更其行為？ 這類有用且相關的狀態通常會使用無狀態的進度對話方塊或專用的進度頁面來顯示，但不會顯示在狀態列上的忙碌指標、活動指標或進度列。 如果在使用其他程式時狀態不實用，只要直接在程式中顯示進度意見反應。

    **正確：**

    ![具有進度列之 [複製] 對話方塊的螢幕擷取畫面 ](images/winenv-taskbar-image8.png)

    **不正確：**

    ![工作列按鈕上進度列的螢幕擷取畫面 ](images/winenv-taskbar-image9.png)

    在不正確的範例中，工作列按鈕的進度列不太實用。

-   **工作是否連續？** 如果工作從未完成，就不需要顯示其進度。 連續工作的範例包括使用者不會起始的防毒軟體掃描，以及檔案編制索引。

    **不正確：**

    ![連續工作的進度圖示螢幕擷取畫面 ](images/winenv-taskbar-image10.png)

    在此範例中，連續工作不需要顯示進度。

**子視窗工作列**

-   **您的程式是否包含使用者通常想要直接切換的索引標籤、專案視窗、MDI 子視窗或次要視窗？** 如果是這樣，則可以適當地提供這些 windows 的工作列按鈕縮圖。

## <a name="design-concepts"></a>設計概念

### <a name="using-jump-lists-and-thumbnail-toolbars-effectively"></a>有效使用跳躍清單和縮圖工具列

跳躍清單和縮圖工具列可協助使用者更有效率地存取資源和執行命令。 不過，在設計您的程式如何支援這些功能時，並不會提高授與的效率。 如果使用者無法準確預測哪些功能具有所需的命令，或他們必須檢查多個位置，則最終使用者將會感到挫折並停止使用這些功能。

當快捷方式清單和縮圖工具列是：

-   **清楚區別。** 使用者知道何時要尋找捷徑清單中的目的地或命令，以及何時查看縮圖工具列。 每個都有明確的用途，因此使用者很少會混淆兩者的內容。 一般而言，會使用跳躍清單來啟動新的工作，而縮圖工具列會在使用其他程式時用來與執行中的工作進行互動。
-   **有用。** 提供的目的地和命令是使用者所需的。 如果使用者不太可能需要任何東西，就不會包含它。 如果不需要，請不要使用專案數目上限。
-   **預測。** 提供的目的地和命令是使用者預期要尋找的專案。 使用者很少需要查看一個以上的位置。
-   **妥善整理。** 使用者可以找到他們想要快速尋找的內容。 它們使用描述性但簡潔的標籤，以及適合辨識的適當圖示。

請務必進行使用者研究，以確定您有正確的許可權。 如果您最後發現您無法設計可達成這些目標的跳躍清單和縮圖工具列，請考慮只提供其中一項。 最好要有一個可預測的方式來提供命令，而不是兩個混淆的命令。

## <a name="guidelines"></a>指導方針

### <a name="taskbar-buttons"></a>工作列按鈕

-   **使用工作列按鈕縮圖) ，讓下列視窗類型出現在 Windows 7 的工作列 (：**
    -   包含沒有擁有者之對話方塊的主要 windows () 
    -   屬性工作表
    -   非模式進度對話方塊
    -   精靈
-   **若為 Windows 7，請使用工作列按鈕縮圖，將下列視窗類型與啟動的主視窗工作列按鈕分組。** 每個程式都 (具體來說，每個程式都可以被視為個別的程式) 應該有一個工作列按鈕。

    -   次要視窗
    -   工作區索引標籤
    -   Project 視窗
    -   MDI 子視窗

    **正確：**

    ![windows explorer 和進度列的螢幕擷取畫面 ](images/winenv-taskbar-image11.png)

    在此範例中，次要視窗會以其主視窗的工作列按鈕分組。

    **不正確：**

    ![windows explorer 和控制台的螢幕擷取畫面 ](images/winenv-taskbar-image12.png)

    在此範例中，主控台的群組 Windows 檔案總管不正確。 使用者將這些視為個別的程式。

    **不正確：**

    ![程式、進度列和工作列的螢幕擷取畫面 ](images/winenv-taskbar-image13.png)

    在此範例中，Windows 備份錯誤地針對單一程式使用兩個工作列按鈕。

-   **還原主視窗也應還原其所有的次要視窗，** 即使這些次要視窗有自己的工作列按鈕。 還原時，請將次要視窗放在主視窗的最上層。
-   **若是 Windows 7，通常有桌上型電腦的程式可能會暫時顯示工作列按鈕來顯示狀態。** 只有當您的程式通常會顯示在桌面上，而且使用者經常與其互動時，才需要這麼做。 通常在沒有桌上型電腦的情況下執行的程式應該改為使用其通知區域圖示，即使它可能不會一直顯示也一樣。

    **不正確：**

    ![windows 同步中心工作列按鈕的螢幕擷取畫面 ](images/winenv-taskbar-image14.png)

    在此範例中，Windows 同步中心不正確地使用暫存工作列按鈕來顯示狀態。 它應該改用其通知區域圖示。

### <a name="icons"></a>圖示

-   **將您的程式圖示設計成在工作列上看起來很棒。** 確定它有意義，並反映其功能和您的品牌。 使其成為相異的，使其成為特殊的，並確保其在所有圖示大小中都能妥善呈現。 花時間取得正確的時間。 遵循 [Aero 樣式的圖示指導方針](vis-icons.md)。
-   **如果您的程式使用重迭圖示，請設計您的程式基底圖示來妥善處理重迭。** 重迭圖示會顯示在右下角，因此請設計圖示，讓該區域可以遮蔽。

    ![圖示和右下角重迭的螢幕擷取畫面 ](images/winenv-taskbar-image15.png)

    在此範例中，程式的工作列按鈕圖示在右下方區域中沒有重要的資訊。

-   **請勿在程式的基底圖示中使用** 覆迭，不論您的程式是否使用重迭圖示。 在基底圖示中使用覆迭將會造成混淆，因為使用者必須找出它沒有通訊的狀態。

    **不正確：**

    ![具有重迭基底圖示的螢幕擷取畫面 ](images/winenv-taskbar-image16.png)

    在此範例中，程式的基底圖示看起來像是顯示狀態。

如需一般圖示指導方針和範例，請參閱 [圖示](vis-icons.md)。

### <a name="overlay-icons"></a>重迭圖示

-   **使用重迭圖示來指出僅有用且相關的狀態。** 請考慮將重迭圖示顯示為使用者的工作可能會中斷，因此狀態變更必須足以應付可能的中斷。

    **不正確：**

    ![三個重迭圖示的螢幕擷取畫面 ](images/winenv-taskbar-image17.png)

    在這些範例中，覆迭圖示的重要性不足以應付可能的中斷。

-   **使用重迭圖示來暫存狀態。** 如果不斷顯示，重迭圖示會遺失其值，因此一般程式狀態不應顯示圖示。 當圖示為時，請移除覆迭圖示：

    -   **是否有問題：** 解決問題後，請移除圖示。
    -   **警示：有一些新功能：** 當使用者啟用程式之後，請移除圖示。

    **例外狀況：** 如果使用者一律需要知道其狀態，您的程式可能會經常顯示重迭圖示。

    ![具有重迭圖示的 live messenger 螢幕擷取畫面 ](images/winenv-taskbar-image18.png)

    在此範例中，Windows Live Messenger 一律會顯示重迭圖示，讓使用者一律可以檢查其回報的狀態。

-   **請勿顯示圖示來指出問題已獲得解決。** 相反地，只要移除任何先前表示有問題的圖示即可。 假設使用者通常會預期您的程式會在沒有問題的情況下執行。
-   **顯示重迭圖示或通知區域圖示，但不能同時顯示兩者。** 您的程式可能會支援這兩種機制來提供回溯相容性，但如果您的程式使用重迭圖示來顯示狀態，則它也不應該使用通知區域圖示來取得狀態。

    **不正確：**

    ![顯示圖示兩次的工作列螢幕擷取畫面 ](images/winenv-taskbar-image19.png)

    在此範例中，新的郵件圖示會重複顯示。

-   **請勿快閃工作列按鈕，以吸引狀態變更。** 這麼做會造成太雜亂的情況。 讓使用者自行探索重迭圖示。
-   **偏好使用標準重迭圖示來指出狀態或狀態變更。** 使用這些標準重迭圖示： 

    | 重疊 | 狀態 |
    |---------------------------------------------------------------------------------------------------|----------------------------------|
    | ![小型警告圖示的螢幕擷取畫面 ](images/winenv-taskbar-image20.png)<br/>               | 警告<br/>               |
    | ![小錯誤圖示的螢幕擷取畫面 ](images/winenv-taskbar-image21.png)<br/>                 | 錯誤<br/>                 |
    | ![小型停用/中斷連線圖示的螢幕擷取畫面 ](images/winenv-taskbar-image22.png)<br/> | 已停用/已中斷連線<br/> |
    | ![小型封鎖/離線圖示的螢幕擷取畫面 ](images/winenv-taskbar-image23.png)<br/>       | 封鎖/離線<br/>       |

    

     

-   **針對自訂覆迭圖示，選擇容易辨識的設計。** 使用高品質的16x16 圖元、完整色彩圖示。 偏好在正方形或矩形形狀圖示上具有特殊輪廓的圖示。 也套用其他 [Aero 樣式的圖示指導方針](vis-icons.md) 。
-   **讓自訂覆迭圖示的設計更簡單。** 請勿嘗試傳達複雜、不熟悉或抽象的想法。 如果您不能想像出合適的自訂圖示，請在適當的情況下改用標準圖示錯誤或警告圖示。 這些圖示可以有效地用來傳達許多類型的狀態。
-   **請勿變更太頻繁的狀態。** 重迭圖示不應該出現雜訊、不穩定或需要注意。 眼睛對周邊周邊領域的變化很敏感，因此狀態變更必須很微妙。
    -   **請勿快速變更圖示。** 如果基礎狀態快速變更，請讓圖示反映高階狀態。

        ![兩種狀態中重迭圖示的螢幕擷取畫面 ](images/winenv-taskbar-image24.png)

        在此範例中，快速變更的重迭圖示會要求注意。

    -   **請勿使用動畫。** 這麼做會造成太雜亂的事情。
    -   **請勿將圖示閃爍。** 這麼做會造成太雜亂的事情。 如果事件需要立即注意，請改用對話方塊。 如果事件另需要注意，請使用通知。

### <a name="taskbar-button-flashing"></a>工作列按鈕閃爍

-   **請謹慎地使用工作列按鈕來要求使用者立即注意，讓執行中的工作持續進行。** 當工作列按鈕閃爍時，使用者很難專注，因此假設他們會中斷其執行的動作，使其停止。 雖然閃爍工作列按鈕的效果比竊取輸入焦點還好，但閃爍的工作列按鈕仍會有很大的干擾。 請確定中斷已對齊，例如，表示使用者需要在關閉視窗之前儲存資料。 非使用中的程式很少需要立即採取行動。 如果使用者唯一要做的事是啟動程式、讀取訊息，或查看狀態變更，請不要閃爍工作列按鈕。
-   **如果不需要立即採取行動，請考慮下列替代方案：**
    -   使用 [動作成功通知](mess-notif.md) ，表示工作已完成。
    -   不執行任何動作。 等到使用者下一次啟動程式時，才等待使用者參與問題。 這通常是最佳選擇。
-   **如果非使用中的程式需要立即注意，請快閃其工作列按鈕以強調顯示並將其醒目提示。** 請勿執行任何動作：請勿還原或啟用視窗，也不會播放任何音效效果。 相反地，請考慮使用者的視窗狀態選擇，並讓使用者在準備就緒時啟動視窗。
-   **針對具有工作列按鈕的次要視窗，請將其按鈕閃爍，而不是主視窗的工作列按鈕。** 這麼做可讓使用者直接參與視窗。
-   **針對沒有工作列按鈕的次要視窗，請將主視窗的工作列按鈕閃爍，然後將次要視窗放在該程式的所有其他視窗之上。** 需要注意的次要視窗必須是最上層的，以確保使用者看見它們。
-   **一次只針對一個視窗快閃一個工作列按鈕。** 不需要閃爍一個以上的按鈕，也不會造成太大的干擾。
-   **當程式變成使用中狀態時，移除工作列按鈕醒目提示。**
-   **當程式變成使用中狀態時，請確定有明顯的東西。** 一般來說，這個目標是藉由顯示詢問問題或起始動作的對話方塊來完成。

### <a name="quick-launch-shortcuts"></a>快速啟動快速鍵

-   **只有當使用者加入宣告時，才將程式快捷方式放在快速啟動區域中。** 因為快速啟動已從 Windows 7 移除，所以針對 Windows 7 設計的程式不應將程式快捷方式新增至快速啟動區域，或提供選項來執行此動作。

### <a name="jump-lists"></a>跳躍清單

**設計**

-   **設計跳躍清單來滿足使用者日常工作的目標。** 考量：
    -   **您的程式用途。** 請思考下一步最可能的使用者。 針對檔建立程式，使用者可能會回到最近使用的檔。 針對顯示現有內容的程式，使用者可能會想要存取經常使用的資源。 針對其他程式，使用者可能會執行他們之前未完成的工作，例如讀取新郵件、觀看新的影片，或查看其下一個會議。
    -   **使用者最在意的資訊。** 考慮使用者為何會使用捷徑清單而不是其他方法。 例如，使用者很可能會在意明確識別為重要 (的目的地，例如，使用者放在其連結列或我的最愛中的網址，或輸入) 的網路位址。 他們比較不可能在意間接取得或只需要進行一些工作 (例如透過重新導向流覽的 Web 位址，或按一下 [連結]) 。

        **正確：**

        ![具有一個目標連結的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image25.png)

        **不正確：**

        ![具有目標五個連結的跳躍清單螢幕擷取畫面 ](images/winenv-taskbar-image26.png)

        在不正確的範例中，捷徑清單包含使用者不太可能在意的許多目的地。

-   **不要讓目的地太細微。** 將目的地設為窄且特定可能會導致重複，有數種方式可移至相同的位置。 例如，不要列出個別網頁，而是改為列出最上層首頁;而不是列出歌曲、列出專輯。

    **正確：**

    ![依群組組織的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image27.png)

    **不正確：**

    ![按歌曲組織之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image28.png)

    在不正確的範例中，列出捷徑清單中的歌曲將會填入單一專輯。

-   **如果您不需要，請不要填滿所有可用的捷徑清單位置。** 如果您的程式只有三個有用的專案，請將焦點放在最有用的專案捷徑清單內容，僅提供三個專案。 捷徑清單中的專案越多，尋找任何特定專案所需的工作就越多。

    ![具有一個命令的跳躍清單螢幕擷取畫面 ](images/winenv-taskbar-image29.png)

    在此範例中，自黏便箋的配件會提供單一捷徑清單命令，因為這是必要的。

-   **只有在需要時才提供工具提示，以協助使用者瞭解捷徑清單專案。** 避免重複的工具提示，因為它們是不必要的干擾。 如需其他工具提示方針，請參閱 [工具提示和提示](ctrl-tooltips-and-infotips.md)。

    **不正確：**

    ![具有多餘工具提示之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image30.png)

    在此範例中，捷徑清單工具提示是多餘的。

**捷徑清單功能與程式功能的比較**

-   **不要讓目的地和命令只能透過跳躍清單使用。** 相同的目的地和命令應該可直接從程式本身取得。
-   **針對命令的目的地和標籤使用一致的名稱。** 捷徑清單專案應標示為與直接從程式存取的對等專案相同。
-   **即使程式未執行，也可讓程式處理目的地和命令。** 這麼做是為了提供一致、可靠且方便的體驗。

**分組**

-   **至少提供一個和最多三個群組。** 捷徑清單的專案一律會分組以標示其用途。 擁有三個以上的群組，可讓專案更難找到。
-   **適當時，請使用標準組名。** 標準組名對於使用者來說很熟悉且更容易瞭解。

    命令會被指定工作組名（由 Windows 指派），因此無法變更。

    **正確：**

    ![最近的組名之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image31.png)

    **不正確：**

    ![具有歷程記錄組名之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image32.png)

    最新的組名是比較好的組名，因為它很熟悉，而且歷程記錄和最近的差異不值得進行。

**命令**

-   **提供一組固定的命令，無論程式執行狀態、目前檔或目前的使用者。** 命令應該套用至整個程式，而不是特定的視窗或檔。 這麼做是為了提供一致、可靠且方便的體驗。 命令不應移除或停用。

    **例外狀況：** 當下列情況時，您可以替代或移除命令：

    -   只要一個命令永遠適用，一組互斥的命令就會共用單一命令位置。
    -   在使用特定功能之前，命令不會套用，只要在其他情況下一律套用命令即可。

    **不正確：**

    ![具有列印工作之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image33.png)

    在此範例中，Print 不是理想的捷徑清單命令，因為它相依于目前的檔。

    **正確：**

    ![登入和登出的跳躍清單螢幕擷取畫面 ](images/winenv-taskbar-image34.png)

    在此範例中，登入和登出都是互斥的命令。 此外，也會使用分隔符號來分組相關的命令。

-   **適當時，請使用下列標準命令標籤。** 標準命令標籤可讓使用者更容易瞭解。
-   **以邏輯順序顯示命令。** 一般訂單包括依使用頻率或使用順序。 將高度相關的命令放在彼此旁邊。 在 [工作] 群組中，視需要在相關命令群組之間放置分隔符號。
-   **請勿提供用來開啟或關閉程式的命令。** 這些命令都內建在所有跳躍清單中。

**命令圖示**

-   在 **[工作] 群組內，只有在可協助使用者瞭解、辨識或區分命令** 時，特別是在程式中使用的命令已建立圖示時，才提供命令圖示。

    -   **例外狀況：** 如果您的程式使用兩個目的地 (一律會有圖示) 和命令，請考慮為所有命令提供圖示（如果沒有這麼做）。

    **不正確：**

    ![跳躍清單不一致使用圖示的螢幕擷取畫面 ](images/winenv-taskbar-image35.png)

    在此範例中，Internet Explorer 應該提供所有命令的圖示，以避免出現麻煩的外觀。

**目的地**

-   **提供目前使用者專屬的一組動態目的地，但不受程式執行狀態或目前檔的影響。** 如先前所述，請確定其符合您的程式目的、使用者最在意的專案，以及具有適當的明確程度。
-   **適用時，請使用「自動」目的地清單。** 自動目的地是由 Windows 管理，但您的程式會控制傳遞的特定目的地。
    -   請考慮使用最新的檔建立程式，使用者可能會回到最近使用的目的地。

        ![具有「最近」組名之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image36.png)

        在此範例中，Windows 記事本使用最近的目的地。

    -   請考慮經常在顯示現有內容的程式中使用，使用者可能會在其中返回經常使用的專案。 頻繁的目的地會依頻率順序排序，最常優先。

        ![常用組名的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image37.png)

        在此範例中，Windows 檔案總管會使用頻繁的目的地。

    -   如果最近會導致許多無用的目的地，請經常使用。 當使用者移至許多不同的目的地，但不太可能會傳回很少使用的目的地時，經常會有更穩定的清單，以及更好的選擇。

        **不正確：**

        ![含有數個最近專案的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image38.png)

        在 Windows Internet Explorer 中使用 [最近] 會導致許多無用的目的地。

    -   如果 [最近] 或 [頻繁] 是相當適合的選項，請使用 [最近]，因為這種方法可讓使用者更容易瞭解並可預測。
    -   如果使用 [最近]，而程式在 [檔案] 功能表中有對等專案，請以相同的順序讓清單具有相同的內容。 對使用者而言，這些應該會顯示為相同的清單。

-   **必要時，請使用自訂目的地清單。** 您的程式可以完全掌控自訂目的地清單的內容和排序次序，因此可以根據任何因素來建立清單。
    -   如果適合，請建立最新或經常的自訂版本，但自動管理不適合您的程式。 例如，您的程式可能需要追蹤開啟檔案命令以外的各種因素。 在此情況下，請使用相同名稱 (最近的) 和排序次序，因為使用者不會察覺到差異。
    -   否則，請使用不同類型的目的地，以進一步滿足您的使用者目標。 這些清單通常可協助使用者執行他們之前未完成的工作，例如讀取新郵件、觀看新影片，或檢查其下一個會議。

        ![具有「新增」組名之跳躍清單的螢幕擷取畫面 ](images/winenv-taskbar-image39.png)

        在此範例中，Windows Media Center 會列出使用者尚未看到的最近錄製的節目。

    -   選擇對應至使用者的清單之精神模型的排序次序。 例如，待辦事項樣式清單會先列出下一件事。 如果沒有明確的精神模型，請依字母順序排序目的地清單。

-   **請勿使用多個目的地清單來提供相同資料的不同觀點。** 相反地，多個目的地清單應該有最不同的資料，以支援不同的案例。 例如，您可以提供最新清單或常用清單，但不能同時提供兩者。 如果有重迭的專案，這麼做會很浪費，但如果移除重迭的專案，就會造成混淆。

    **不正確：**

    ![重複群組專案的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image40.png)

    在此範例中，為相同的目的地提供不同的觀點，會造成浪費。

    **正確：**

    ![具有妥善組織之工作的快捷方式清單螢幕擷取畫面 ](images/winenv-taskbar-image41.png)

    在此範例中，目的地清單針對不同的工作具有不同的資料。

-   **如果您的程式有可清除隱私權資料的命令，也請清除目的地清單。** 目的地清單可能包含機密資料。

### <a name="thumbnail-toolbars"></a>縮圖工具列

**互動**

-   **提供最多七個最重要、最常使用的命令，以套用至縮圖中顯示的視窗。** 如果您的程式只有三個重要且常用的命令，請務必提供最多的命令，只提供三個。

    **不正確：**

    ![具有太多命令的工具列螢幕擷取畫面 ](images/winenv-taskbar-image42.png)

    在此範例中，縮圖工具列有不重要的命令。

-   **使用直接和立即的命令。** 這些命令應該會立即生效，按一下命令不應該顯示下拉式功能表或對話方塊以進行更多輸入。

    **不正確：**

    ![具有下拉式功能表的縮圖螢幕擷取畫面 ](images/winenv-taskbar-image43.png)

    縮圖工具列命令必須立即生效。

-   **停用未套用至目前內容的命令，或直接導致錯誤的命令。** 請勿隱藏這類命令，因為這樣做會使工具列呈現不穩定。
-   **如果使用者按一下命令可能會檢查結果，或立即按一下另一個命令，請勿關閉縮圖。** 移除指出使用者現在已完成的命令縮圖，例如使用顯示其他視窗的命令。

    ![使用命令的 media player 縮圖螢幕擷取畫面 ](images/winenv-taskbar-image44.png)

    在此範例中，按一下 Windows Media Player 中的 [下一步] 會繼續顯示縮圖，因為使用者可能會想要提供其他命令。

    ![具有聊天圖示的縮圖螢幕擷取畫面 ](images/winenv-taskbar-image45.png)

    在此範例中，按一下 Windows Live Messenger 的聊天會關閉縮圖，因為使用者最有可能傳送訊息。

**展示**

-   **請確定縮圖工具列圖示符合 Aero 樣式圖示的指導方針。** 針對每個命令，提供高品質的16x16、20x20 和24x24 圖元、完整色彩圖示。 較大的版本會在高 DPI 的顯示模式中使用。
-   **請確定在一般和暫留狀態中，工具列背景色彩清楚看到圖示。** 一律在內容和高對比模式中評估圖示。
-   **選擇清楚傳達其效果的命令圖示設計。** 設計完善的命令圖示一目了然，可協助使用者有效率地尋找及瞭解命令。
-   **選擇可辨識且可辨別的圖示。** 請確定圖示有獨特的圖形和色彩。 這麼做可協助使用者快速尋找命令，即使它們不記得圖示符號也一樣。 初次使用之後，使用者應該不需要依賴工具提示來區分命令。
-   **提供工具提示以標記每個命令。** 很好的工具提示會標示所指向的未標記控制項。 如需指導方針與範例，請參閱 [工具提示和提示](ctrl-tooltips-and-infotips.md)。

### <a name="progress-bars"></a>進度列

-   **遵循一般進度列指導方針，** 包括不重新開機或備份進度，以及使用紅色進度列來指出問題。
-   **避免使用不定的進度列。** 不定的進度列會顯示活動，而不是進度。 針對不接受授與活動的罕見情況，保留不定的進度列。

如需詳細的指導方針，請參閱 [進度](progress-bars.md)列。
