# <a name="drop-down-lists--combo-boxes"></a>下拉式方塊 & 下拉式清單

> [!NOTE]
> 此設計指南是針對 Windows 7 所建立，而且尚未針對較新的 Windows 版本進行更新。 大部分的指引仍然適用于準則，但展示和範例不會反映我們目前的 [設計指引](/windows/uwp/design/)。

在下拉式清單或下拉式方塊中，使用者可以在互斥值清單中進行選擇。 使用者只能選擇一個選項。 使用標準下拉式清單時，使用者只能使用清單中的選項，但是您可以使用下拉式方塊來輸入不在清單中的選項。

![[提醒時間] 下拉式方塊的螢幕擷取畫面 ](images/ctrl-drop-image1.png)

典型的下拉式方塊。

當您閱讀這篇文章時，請務必瞭解下列詞彙：

-   標準清單方塊是一個方塊，其中包含多個專案的清單，並顯示多個專案。
-   下拉式清單是一份清單，其中一律會顯示選取的專案，而其他專案則會按下下拉式按鈕，視需要顯示。
-   下拉式方塊是標準清單方塊或下拉式清單和可編輯 [文字方塊](ctrl-text-boxes.md)的組合，因此可讓使用者輸入不在清單中的值。
    -   可編輯的下拉式清單是下拉式清單和可編輯文字方塊的組合。
    -   可編輯的清單方塊是標準清單方塊和可編輯文字方塊的組合。

> [!Note]  
> 與 [版面](vis-layout.md) 配置相關的指導方針會在個別的文章中顯示。

 

## <a name="is-this-the-right-control"></a>這是正確的控制項嗎？

若要決定使用時機，請考量下列問題：

-   **控制項是否可用來從互斥值清單中選擇一個選項？** 如果不是，請使用其他控制項。 若要選擇多個選項，請改為使用 [標準的多重選取清單](ctrl-list-boxes.md)、核取方塊清單、清單產生器或新增/移除清單。
-   **選項是命令嗎？** 若是如此，請改用 [功能表按鈕](ctrl-command-buttons.md) 或分割按鈕。 針對 (名詞) 或屬性 (形容詞) ，而不是 (動詞命令) 命令的物件，請使用下拉式清單與下拉式方塊。
-   **清單是否會顯示資料，而不是程式選項？** 無論何種方式，下拉式清單或下拉式方塊都是適合的選擇。 相較之下， [選項按鈕](ctrl-radio-buttons.md) 只適用于少數的程式選項。

**下拉式清單**

-   **大部分的使用者在大多數情況下，是否有建議使用的預設選項？** 看到選取的選項遠比看到替代方案更重要嗎？ 如果您不想要鼓勵使用者藉由隱藏替代方案來進行變更，請考慮使用下拉式清單。 如果沒有，請考慮使用選項按鈕、單選清單，或可編輯的清單方塊，以提供更多強調替代選項。

    ![最高品質作為預設按鈕的螢幕擷取畫面 ](images/ctrl-drop-image2.png)

    在此範例中，最高的色彩品質是大部分使用者的最佳選擇，因此下拉式清單是把替代方案的理想選擇。

-   **您是否想要強調選項？** 如果是，請考慮使用選項按鈕、單選清單，或可編輯的清單方塊，這通常會以更多的螢幕空間來繪製更多的注意力。 因為下拉式清單是 compact，所以它們是您想要 underemphasize 之選項的理想選擇。
-   **螢幕空間是否在 premium？** 如果是的話，請使用下拉式清單，因為所使用的螢幕空間是固定的，而且與選擇的數目無關。
-   **視窗上是否有其他下拉式清單？** 如果是的話，請考慮使用下拉式清單來保持一致性。

**可編輯的下拉式清單**

除了針對下拉式清單所提供的準則之外，下列也適用：

-   **是否有可能的選項受限制？** 如果是的話，請改用一般下拉式清單。 下拉式方塊適用于未受限制的輸入，其中使用者可能需要輸入目前不在清單中的值。 因為輸入不受限制，所以如果使用者輸入不正確文字，您必須處理錯誤並顯示錯誤訊息。
-   **您可以事先列舉最有可能的選擇** 嗎？ 如果沒有，請改用文字方塊。
-   **下拉式清單是用來列出先前的使用者輸入嗎？** 除非使用者需要檢查先前輸入的完整清單，否則請改用具有自動完成選項的文字方塊。

    ![具有下拉式清單的 [執行] 對話方塊的螢幕擷取畫面 ](images/ctrl-drop-image3.png)

    在此範例中，使用者可能需要檢查先前的輸入，因此可編輯的下拉式清單是不錯的選擇。

    ![outlook 至折線圖和自動完成的螢幕擷取畫面 ](images/ctrl-drop-image4.png)

    在此範例中，具有自動完成的文字方塊是不錯的選擇。

-   **使用者是否需要協助來選取有效的值？** 如果是的話，請改用具有 [瀏覽按鈕](ctrl-command-buttons.md) 的文字方塊。

    ![outlook 至折線圖和瀏覽按鈕的螢幕擷取畫面 ](images/ctrl-drop-image5.png)

    在此範例中，使用者可以按一下 [到] 以協助他們選取有效的值。

-   **建議使用者查看替代選項或邀請變更是否很重要？** 如果是的話，請考慮改用可編輯的清單方塊。 使用可編輯的下拉式清單時，使用者將不會察覺到此清單中的替代方案。
-   **使用者是否需要在大型清單中快速找出專案？**  (Win32 僅) 如果有，請使用下拉式方塊，因為使用者可以輸入其完整名稱來選取專案。 相較之下，Win32 下拉式清單只會根據輸入的最後一個字元來選取專案 (因此，在月份清單中輸入「六月」將符合11月，而不是6月) 。 在此情況下，即使可能的選項受到限制，也請使用下拉式方塊。

**可編輯的清單方塊**

-   **是否有可能的選項受限制？** 如果是的話，請改為使用單一選取清單或一般下拉式清單。 下拉式方塊適用于未受限制的輸入，其中使用者可能需要輸入目前不在清單中的值。 因為輸入不受限制，所以如果使用者輸入不正確文字，您必須處理錯誤並顯示錯誤訊息。
-   **您可以事先列舉最有可能的選擇嗎？** 如果沒有，請改用文字方塊。
-   **建議使用者查看替代選項或邀請變更是否很重要？** 如果沒有，請考慮改用可編輯的下拉式清單。
-   **您是否想要強調選項？** 如果沒有，請考慮改用可編輯的下拉式清單。 因為下拉式清單是 compact，所以它們是您想要 underemphasize 之選項的理想選擇。
-   **螢幕空間是否在 premium？** 如果是的話，請使用可編輯的下拉式清單，因為所使用的螢幕空間是固定的，而且與選擇的數目無關。

如果是下拉式清單， **清單中的專案數目不是選擇控制項的因素** ，因為它們的範圍從上千個專案到一。 可編輯的下拉式清單會從上千個專案縮小為無，因為使用者可以輸入不在清單中的值。 因為下拉式清單可用於資料，所以可能事先不知道專案的數量，而且可能無法保證。 **在可編輯的清單方塊中一律包含至少三個專案，以證明額外的螢幕空間。**

## <a name="guidelines"></a>指導方針

### <a name="general"></a>一般

-   **請勿使用下拉式清單或下拉式方塊的變更來**：
    -   執行命令。
    -   顯示其他視窗，例如用來收集更多輸入的對話方塊。
    -   以動態方式顯示與所選取控制項相關的其他控制項 ([畫面讀取器](inter-accessibility.md) 無法偵測到這類事件) 。

### <a name="presentation"></a>簡報

-   **以邏輯順序排序清單專案**，例如將高度相關的選項群組在一起，並優先放置最常見的選項，或使用依字母順序排列的選項。 依字母順序排序名稱、依數位順序排序數位和日期（以時間順序排列）。 具有12個以上專案的清單應依字母順序排序，讓專案更容易尋找。

    **正確：** ![邏輯下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image12.png)

    在此範例中，清單專案依其空間關聯性排序。

    **不正確：** ![[雜亂無章] 下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image13.png)

    在此範例中，有很多清單專案需要依字母順序排序。

    **正確：** ![依字母順序排列下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image14.png)

    在此範例中，清單專案會依字母順序排序，但代表所有專案的選項除外。

-   **無論其餘專案的排序次序為何，都可在清單的開頭放置代表 All 或 None 的選項。**
-   **以括弧括住中繼選項。**

    ![顯示已選取 [無 () ] 之下拉式清單的螢幕擷取畫面。](images/ctrl-drop-image15.png)

    在此範例中，「 (無) 」是中繼選項，因為它不是有效的選項值，而是描述該選項本身未使用。

-   **停用下拉式清單或下拉式方塊時，也會停用任何相關聯的標籤和命令按鈕。**

### <a name="drop-down-lists"></a>下拉式清單

-   當您使用單一下拉式清單來變更相關聯控制項的視圖時，請 **在選取時立即變更視圖，而不需要個別的命令按鈕。** 只有在清單需要很長的時間才能轉譯時，才使用個別的命令按鈕。 不過，清單標頭和 [功能表按鈕](ctrl-command-buttons.md) 是此用途的慣用控制項。
-   **不要有空白清單專案** 改用 **中繼選項**。 使用者不知道如何解讀空白專案，而中繼選項的意義是明確的。

    **正確：** ![下拉式清單的螢幕擷取畫面，其中已選取 [無] ](images/ctrl-drop-image16.png)

    **不正確：** ![已選取空白的下拉式清單螢幕擷取畫面 ](images/ctrl-drop-image17.png)

    在不正確的範例中，空白選項的意義不清楚。

### <a name="preview-drop-down-lists"></a>預覽下拉式清單

-   **在清單專案中使用 [預覽]，比使用單獨描述的影像更清楚地顯示。**

    ![預覽字體下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image18.png)

    在此範例中，預覽會解釋比單獨文字更好的選項。

-   **請勿在預覽中使用不必要的無用圖示**。

    **不正確：** ![具有圖示之標籤下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image19.png)

    在此範例中，預覽圖示是不必要的，因為它們不會傳達任何資訊。

### <a name="combo-boxes"></a>下拉式方塊

-   **當您可以時，限制輸入文字的長度。** 例如，如果有效的輸入是介於0和999之間的數位，請使用限制為三個字元的下拉式方塊。
-   **如果有許多可能的選項，請將清單內容放在最有可能的選項上**。 由於使用者可以輸入不在清單中的值，因此下拉式方塊不需要列出所有選項，只要是可能的選擇或代表性範例。

    ![字型大小下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image10.png)

    在此範例中，不會列出許多有效的選項，例如15或一半大小的字型，例如9.5。

### <a name="default-values"></a>預設值

-   **依預設，請選取最安全的 (，以防止遺失資料或系統存取) 和最安全的選項。** 如果安全性和安全性不是因素，請選取最有可能或方便的選項。
    -   **例外狀況：** 如果控制項代表 [混合狀態](glossary.md)中的屬性，則顯示空白的預設值，這會在顯示多個沒有相同設定之物件的屬性時發生。

## <a name="prompts"></a>提示

提示是在可編輯的下拉式清單內放置的標籤或簡短指令，作為其預設值。 與靜態文字不同的是，當使用者在下拉式方塊中輸入某個內容或取得輸入焦點時，會從畫面中消失。

![搜尋方塊的螢幕擷取畫面 ](images/ctrl-drop-image20.png)

典型的提示。

在下列情況下使用提示：

-   螢幕空間是在不需要使用標籤或指示的高階頁面上，例如工具列上。
-   提示的主要目的是要以精簡的方式來識別清單的用途。 當使用者使用下拉式方塊時，它不一定是使用者需要看到的重要資訊。

請不要使用提示來指示使用者從清單中選取某個內容，或按一下按鈕。 例如，像是選取選項或輸入檔案名，然後按一下 [傳送] 是不必要的提示。

使用提示時：

-   以斜體灰色和純文字的一般黑色繪製提示文字。 提示文字不能與真正的文字混淆。
-   將提示文字保持簡潔。 您可以使用片段，而不是完整的句子。
-   使用 [句子樣式的大小寫](glossary.md)。
-   請勿使用結尾標點符號或省略號。
-   提示文字不能是可編輯的，且應該在使用者按一下或 tab 進入文字方塊後消失。
    -   **例外狀況：** 如果文字方塊具有預設輸入焦點，而且只有在使用者開始鍵入後才會消失，則會顯示提示。
-   如果文字方塊在遺失輸入焦點時仍為空白，則會還原提示文字。

**不正確：** ![六個可編輯下拉式清單的螢幕擷取畫面](images/ctrl-drop-image21.png)

在此範例中，螢幕空間不是 premium;填妥可編輯的下拉式清單之後，使用者很難記住其用途;提示文字是可編輯的，並以與實際文字相同的方式繪製。

## <a name="recommended-sizing-and-spacing"></a>建議的大小和間距

![下拉式清單和規格的螢幕擷取畫面 ](images/ctrl-drop-image22.png)

下拉式清單和下拉式方塊的建議調整大小和間距。

-   **選擇適合最長有效資料的寬度。** 下拉式清單無法水準滾動，因此使用者只能看到控制項中可見的內容。 但 (請注意，該下拉式方塊可以啟用 AutoScroll 功能。 ) 
-   **包含額外的 30%** (高達200% 的文字，以提供任何文字 (較短的文字) ，而不是) 將當地語系化的數位。
-   **選擇可消除不必要垂直捲動的清單長度。** 因為下拉式清單會隨選顯示，所以其清單最多可顯示30個專案。 可編輯的清單方塊 (沒有下拉式按鈕的) 應該會顯示3到12個專案。

## <a name="labels"></a>標籤

**控制項標籤**

-   將標籤以單字或片語的形式寫入，而不是句子，並以冒號結尾。 **異常：**
    -   可編輯的下拉式清單，其中包含空間位於 premium 的提示。
    -   如果下拉式清單或下拉式方塊附屬於選項按鈕或核取方塊，且其標籤以冒號結尾，則請勿在控制項上放置額外的標籤。
-   為每個標籤指派唯一的 [存取金鑰](glossary.md) 。 如需指導方針，請參閱 [鍵盤](inter-keyboard.md)。
-   使用 [句子樣式的大小寫](glossary.md)。
-   將標籤置於控制項的左邊或上方，然後將標籤與控制項的左邊緣對齊。 如果標籤位於左邊，請垂直對齊標籤文字與控制項文字。

    **正確：** ![下拉式清單標籤對齊方式的螢幕擷取畫面 ](images/ctrl-drop-image23.png)

    在此範例中，標籤會與控制項文字正確對齊。

    **不正確：** ![下拉式清單的螢幕擷取畫面未正確對齊 ](images/ctrl-drop-image24.png)

    在此範例中，標籤與控制項文字的對齊方式不正確。

-   您可以在標籤後面的括弧中，指定單位 (秒、連接等) 。
-   請勿讓下拉式清單或下拉式方塊的內容 (或其單位標籤) 句子的一部分，因為這不是可當地語系化的。

**選項文字**

-   將唯一名稱指派給每個選項。
-   除非專案是適當的名詞，否則請使用 [句子樣式的大小寫](glossary.md)。
-   將標籤以單字或片語的形式寫入，而不是以句子形式寫入，而且不使用結尾標點符號。
-   使用平行片語，並嘗試保留所有選項的相同長度。

**教學文字**

-   如果您需要加入下拉式清單或下拉式方塊的解說文字，請將它加入至標籤上方。 使用結尾標點符號的完整句子。
-   使用 [句子樣式的大小寫](glossary.md)。
-   有説明但不需要的其他資訊應該保持簡短。 請將這項資訊放在標籤和冒號之間的括弧中，或是在控制項下方的括弧內。

    ![已新增資料之下拉式清單的螢幕擷取畫面 ](images/ctrl-drop-image25.png)

    此範例會顯示位於控制項下方的其他資訊。

## <a name="documentation"></a>文件

參考下拉式清單時：

-   使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線或冒號;包含其中一個清單或方塊，以較清楚的方式進行。
-   針對清單選項，請使用確切的選項文字，包括其大小寫。
-   在程式設計和其他技術檔中，請將下拉式清單視為下拉式清單。 在其他地方，使用 [清單] 或 [box]，以較清楚的方式。
-   若要描述使用者互動，請使用按一下。
-   可能的話，請使用粗體文字來格式化標籤和清單選項。 否則，請只在必要時才將標籤和選項放在引號中，以避免混淆。

範例：在 [ **大小** ] 清單中，按一下 [ **大型** 字型]。

參考下拉式方塊時：

-   使用確切的標籤文字，包括其大小寫，但不包含存取金鑰底線或冒號;包含 [單字] 方塊。
-   針對清單選項，請使用確切的選項文字，包括其大小寫。
-   在程式設計和其他技術檔中，請將下拉式方塊稱為下拉式方塊。 在其他地方使用 box。
-   若要描述使用者互動，請使用 enter。
-   可能的話，請使用粗體文字來格式化標籤和清單選項。 否則，請只在必要時才將標籤和選項放在引號中，以避免混淆。

範例：在 [ **字型** ] 方塊中，輸入您要使用的字型。
