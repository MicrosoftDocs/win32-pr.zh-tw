# <a name="color-approximations-and-dithering"></a>色彩近似值和遞色

雖然應用程式可以使用色彩，而不考慮裝置的色彩功能，但產生的輸出可能不會是資訊，而且可以像是謹慎選擇色彩的輸出。 如果有的話，裝置會保證每個可能的色彩值完全相符;因此，如果應用程式要求裝置無法產生的色彩，系統就會使用裝置可以產生的色彩來接近該色彩。 例如，如果應用程式嘗試建立黑色和白色印表機的紅色畫筆，則會收到黑色畫筆，而系統會使用黑色作為紅色的近似值。

應用程式可以使用 [**GetNearestColor**](/windows/desktop/api/Wingdi/nf-wingdi-getnearestcolor) 函式，來探索系統是否近似于指定的色彩。 此函式會接受色彩值，並傳回裝置可以產生的最接近相符色彩的色彩值。 系統用來判斷此近似值的方法取決於設備磁碟機及其色彩功能。 在大多數情況下，近似色彩的整體濃度最接近所要求色彩的亮度。

當應用程式建立畫筆或設定文字的色彩時，如果沒有完全相符的結果，系統一律會接近色彩。 當應用程式建立實心筆刷時，系統可能會嘗試透過遞色來模擬要求的色彩。 *遞* 色會在模式中替換兩個或多個色彩來模擬色彩。 例如，您可以藉由交替不同的紅色和白色組合來模擬不同的粉紅色陰影。 根據色彩和模式而定，遞色可能會產生合理的模擬。 它最適合用於單色裝置，因為它會將可用的「色彩」數目擴充至簡單的黑色和白色以外。

用來建立遞色色彩的方法取決於設備磁碟機。 大部分的設備磁碟機都使用標準的遞色演算法，此演算法會根據要求的紅色、綠色和藍色色彩的濃度值產生模式。 一般而言，任何無法由裝置產生的要求色彩都會受到模擬的要求，但在系統模擬色彩時，不會通知應用程式。 此外，應用程式無法修改或變更設備磁碟機的遞色演算法。 不過，應用程式可以藉由建立和使用模式筆刷來略過演算法。 如此一來，應用程式就會藉由合併點陣圖中用來建立筆刷的純色，來建立自己的遞色色彩。
