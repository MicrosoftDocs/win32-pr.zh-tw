# <a name="windows-format-metafiles"></a>Windows-Format 中繼檔

Microsoft Windows-格式的中繼檔在其功能上有限，因此很少使用。 支援 Windows 格式函式，以便與撰寫為以16位 Windows 應用程式執行的應用程式維持回溯相容性。 相反地，您應該使用增強格式函數。

*Windows 格式的中繼檔* 是由16位 Windows 應用程式使用。 格式是由標頭和中繼檔記錄陣列所組成。

以下是此格式的限制：

-   Windows 格式中繼檔是與應用程式和裝置相依。 應用程式對應模式或裝置解析的變更，會影響以此格式建立之中繼檔的外觀。
-   Windows 格式中繼檔未包含完整的標頭，其描述原始圖片尺寸、圖片建立所在的裝置解析度、選擇性的文字描述或選擇性的調色板。
-   Windows 格式的中繼檔不支援新的曲線、路徑和轉換函數。 請參閱下清單格中支援的函數清單。
-   某些 Windows 格式的中繼檔記錄無法調整。
-   無法查詢與 Windows 格式中繼檔相關聯的中繼檔裝置內容 (亦即，應用程式無法抓取裝置解析資料、字型計量等) 。

若要將 Windows 格式中繼檔轉換成增強格式的中繼檔，請呼叫 [**GetMetaFileBitsEx**](/windows/desktop/api/Wingdi/nf-wingdi-getmetafilebitsex)函式以從 Windows 格式中繼檔取出資料，然後呼叫 [**SetWinMetaFileBits**](/windows/desktop/api/Wingdi/nf-wingdi-setwinmetafilebits)函式，將此資料轉換成增強格式的中繼檔。 若要將增強格式記錄轉換成 Windows 格式記錄，請呼叫 [**GetWinMetaFileBits**](/windows/desktop/api/Wingdi/nf-wingdi-getwinmetafilebits)函數。
