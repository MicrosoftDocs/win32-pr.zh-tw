# <a name="d-isolated-applications-and-side-by-side-assemblies"></a>D (隔離的應用程式和並存元件) 

[B C](a-sbscs-gly.md) D E F [G](g-sbscs-gly.md) H [I](i-sbscs-gly.md) J K L [M](m-sbscs-gly.md) N O [P](p-sbscs-gly.md) Q [R](r-sbscs-gly.md) [S](s-sbscs-gly.md) T W X Y Z

<dl> <dt>

<span id="_win32_def_context_assemblyidentity_gly"></span><span id="_WIN32_DEF_CONTEXT_ASSEMBLYIDENTITY_GLY"></span>**DEF-coNtext assemblyIdentity**
</dt> <dd>

**AssemblyIdentity** 子項目，定義資訊清單所描述的並存元件。 DEF 內容 **assemblyIdentity** 子項目一律是 **元件** 專案的第一個子項目。

</dd> <dt>

<span id="_win32_dll_versioning_conflict_gly"></span><span id="_WIN32_DLL_VERSIONING_CONFLICT_GLY"></span>**DLL 版本控制衝突**
</dt> <dd>

相容性問題。 當應用程式安裝的共用元件版本與先前安裝的版本不相容時，就會發生元件共用問題。 DLL 版本控制衝突的解決方案是使用並存元件共用和隔離的應用程式。 使用並存元件共用時，可以同時執行相同 Windows 元件的多個版本。 開發人員可以選擇要使用哪個並存元件。

</dd> </dl>
