# <a name="isolated-applications"></a><span data-ttu-id="68307-104">隔離的應用程式</span><span class="sxs-lookup"><span data-stu-id="68307-104">Isolated Applications</span></span>

<span data-ttu-id="68307-105">隔離的應用程式是與 [資訊清單](manifests.md)一起安裝的自我描述應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-105">Isolated applications are self-describing applications installed with [manifests](manifests.md).</span></span> <span data-ttu-id="68307-106">隔離的應用程式可以同時使用 [私用元件](/windows/desktop/Msi/private-assemblies) 和 [共用元件](/windows/desktop/Msi/shared-assemblies)。</span><span class="sxs-lookup"><span data-stu-id="68307-106">Isolated applications can use both [private assemblies](/windows/desktop/Msi/private-assemblies) and [shared assemblies](/windows/desktop/Msi/shared-assemblies).</span></span>

<span data-ttu-id="68307-107">如果應用程式的所有元件都是共用 [並存元件](about-side-by-side-assemblies-.md) 或私用元件，則會被視為完全隔離的應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-107">An application is considered fully isolated if all of its components are either shared [side-by-side assemblies](about-side-by-side-assemblies-.md) or private assemblies.</span></span> <span data-ttu-id="68307-108">如果它使用不是並存元件的部分元件，它就稱為部分隔離。</span><span class="sxs-lookup"><span data-stu-id="68307-108">It is called partially isolated if it uses some components that are not side-by-side assemblies.</span></span> <span data-ttu-id="68307-109">請注意，如果應用程式使用的部分元件並非並存元件，或使用私用元件，則應用程式可能會受到系統上其他應用程式的安裝或移除所影響。</span><span class="sxs-lookup"><span data-stu-id="68307-109">Note that if an application uses some components that are not side-by-side assemblies, or uses private assemblies, the application may be affected by the installation or removal of other applications on the system.</span></span> <span data-ttu-id="68307-110">如需詳細資訊，請參閱 [並存元件共用](side-by-side-assembly-sharing.md)。</span><span class="sxs-lookup"><span data-stu-id="68307-110">For more information, see [Side-by-side Assembly Sharing](side-by-side-assembly-sharing.md).</span></span>

<span data-ttu-id="68307-111">開發人員建議您設計隔離的應用程式，並將現有的應用程式更新為隔離的應用程式，原因如下：</span><span class="sxs-lookup"><span data-stu-id="68307-111">Developers are encouraged to design isolated applications and to update existing applications into isolated applications for the following reasons:</span></span>

-   <span data-ttu-id="68307-112">隔離的應用程式更穩定且可靠地更新，因為它們不會受到系統上其他應用程式的安裝、移除或升級所影響。</span><span class="sxs-lookup"><span data-stu-id="68307-112">Isolated applications are more stable and reliably updated because they are unaffected by the installation, removal, or upgrading of other applications on the system.</span></span>
-   <span data-ttu-id="68307-113">您可以設計隔離的應用程式，使其一律使用建立和測試時所用的相同元件版本來執行。</span><span class="sxs-lookup"><span data-stu-id="68307-113">Isolated applications can be designed so that they always run using the same assembly versions with which they were built and tested.</span></span>
-   <span data-ttu-id="68307-114">隔離的應用程式可以使用由 Microsoft 提供的並存元件所提供的功能。</span><span class="sxs-lookup"><span data-stu-id="68307-114">Isolated applications can use functionality provided by the side-by-side assemblies made available by Microsoft.</span></span> <span data-ttu-id="68307-115">如需詳細資訊，請參閱 [支援的 Microsoft 並存元件](supported-microsoft-side-by-side-assemblies.md)。</span><span class="sxs-lookup"><span data-stu-id="68307-115">For more information, see [Supported Microsoft Side-by-side Assemblies](supported-microsoft-side-by-side-assemblies.md).</span></span>
-   <span data-ttu-id="68307-116">隔離的應用程式不會系結至其並存元件的出貨排程，因為應用程式和系統管理員可以在部署後更新設定，而不需要重新安裝應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-116">Isolated applications are not tied to the shipping schedule of their side-by-side assemblies because applications and administrators can update the configuration after deployment without having to reinstall the application.</span></span> <span data-ttu-id="68307-117">在只有一個版本的元件可供使用的情況下，這並不適用。</span><span class="sxs-lookup"><span data-stu-id="68307-117">This would not apply in the case where only one version of the assembly is being made available.</span></span>
-   <span data-ttu-id="68307-118">您可以使用 **xcopy** 命令來安裝完全隔離的應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-118">A fully isolated application may be installed by using the **xcopy** command.</span></span> <span data-ttu-id="68307-119">[Windows Installer](../msi/windows-installer-portal.md) 也可以用來安裝隔離的應用程式，而不會影響登錄。</span><span class="sxs-lookup"><span data-stu-id="68307-119">[Windows Installer](../msi/windows-installer-portal.md) can also be used to install an isolated application without impact to the registry.</span></span> <span data-ttu-id="68307-120">如需詳細資訊，請參閱 [Win32 元件的安裝](../msi/installation-of-win32-assemblies.md)。</span><span class="sxs-lookup"><span data-stu-id="68307-120">For more information, see [Installation of Win32 Assemblies](../msi/installation-of-win32-assemblies.md).</span></span>

<span data-ttu-id="68307-121">在某些情況下，現有的應用程式可以更新為隔離的應用程式，而不需要重寫應用程式程式碼。</span><span class="sxs-lookup"><span data-stu-id="68307-121">In some cases, existing applications can be updated into an isolated application without having to rewrite the application code.</span></span> <span data-ttu-id="68307-122">您可以建立 [應用程式資訊清單](application-manifests.md) ，以描述應用程式對 [並存元件](about-side-by-side-assemblies-.md)的相依性。</span><span class="sxs-lookup"><span data-stu-id="68307-122">An [application manifest](application-manifests.md) can be created that describes the application's dependencies on [side-by-side assemblies](about-side-by-side-assemblies-.md).</span></span> <span data-ttu-id="68307-123">如果應用程式使用非並存元件的元件，這些元件可能會部署為 [私](/windows/desktop/Msi/private-assemblies)用元件。</span><span class="sxs-lookup"><span data-stu-id="68307-123">If the application uses components that are not side-by-side assemblies, these may be deployed as [private assemblies](/windows/desktop/Msi/private-assemblies).</span></span> <span data-ttu-id="68307-124">請注意，使用協力廠商元件來執行此操作的可能性可能取決於授權，因為元件必須撰寫為元件。</span><span class="sxs-lookup"><span data-stu-id="68307-124">Note that the possibility of doing this with third-party components may depend on licensing because the component will need to be authored as an assembly.</span></span> <span data-ttu-id="68307-125">例如，藉由建立應用程式資訊清單，並指定對並存通用控制項的相依性 (COMCTL32.DLL) ，在 Windows XP 上執行的應用程式可以利用 Windows *主題*。</span><span class="sxs-lookup"><span data-stu-id="68307-125">For example, by creating an application manifest and specifying a dependence on the side-by-side common controls (COMCTL32), an application running on Windows XP can take advantage of Windows *theming*.</span></span> <span data-ttu-id="68307-126">您應一律測試您的應用程式，以確保它與新版本的 COMCTL32.DLL 元件相容。</span><span class="sxs-lookup"><span data-stu-id="68307-126">You should always test your application to ensure it is compatible with the new version of the COMCTL32 assembly.</span></span>

<span data-ttu-id="68307-127">可能無法將每個現有的應用程式更新為完全隔離的應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-127">It may not be possible to update every existing application into a fully isolated application.</span></span> <span data-ttu-id="68307-128">例如，某些 [Windows 檔案保護 (WFP) ](/windows/desktop/Wfp/windows-resource-protection-portal) 系統元件無法以並存元件的方式提供，而且無法與應用程式一起安裝為私用元件。</span><span class="sxs-lookup"><span data-stu-id="68307-128">For example, some [Windows File Protection (WFP)](/windows/desktop/Wfp/windows-resource-protection-portal) system assemblies are not available as side-by-side assemblies and cannot be installed with the application as a private assembly.</span></span> <span data-ttu-id="68307-129">您可以針對應用程式資訊清單中的部分應用程式元件指定並存元件相依性，以部分隔離這類應用程式。</span><span class="sxs-lookup"><span data-stu-id="68307-129">It may be possible to partially isolate such applications by specifying side-by-side assembly dependencies for some of the application's assemblies in an application manifest.</span></span>
