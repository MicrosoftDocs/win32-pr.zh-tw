# <a name="architecture-windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>架構 (Windows 7 和 Windows Server 2008 R2 應用程式品質操作手冊) 

*鬆散結合的 Internet Explorer* (LCIE) 藉由分隔元件的元件並放寬其類型，來改善瀏覽器的可靠性。

尤其是，LCIE 會嘗試將 Windows Internet Explorer 框架和其索引標籤隔離到個別的進程。 在 Windows Internet Explorer 8 中，此隔離可改善效能和擴充性。 此架構變更可能會影響延伸模組和附加元件的相容性，包括 ActiveX 控制項、瀏覽器協助程式物件 (bho) ，以及使用舊版程式設計技術建立的 UI 工具列元件。
