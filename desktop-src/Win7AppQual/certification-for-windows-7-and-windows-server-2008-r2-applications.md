# <a name="windows-7-client-software-logo-program"></a>Windows 7 用戶端軟體標誌計畫

## <a name="affected-platforms"></a>受影響的平臺

 **客戶** 端-Windows 7  


## <a name="description-client-application-certification"></a>描述：用戶端應用程式認證

在 Windows 7 的時間範圍內，我們想要繼續建立 Windows 的成功，並展開標誌程式，使其與客戶更相關。 根據合作夥伴和客戶的意見反應，Windows 7 標誌計畫將著重于確保裝置、系統和應用程式相容、可靠且可執行以符合 Windows 標準。 Windows 7 的建議程式只會有一個適用于系統、裝置和應用程式的標誌。 本主題特別著重于用戶端軟體相容性。

我們已改善將軟體應用程式與「相容 Windows 7」標誌相容的流程。 Microsoft 不會再要求開發人員以經授權的測試授權單位，測試其應用程式的認證。 相反地，Microsoft 正在開發標誌工具組，可讓您用來確保您的應用程式與 Windows 7 相容，並符合 Windows 7 用戶端軟體標誌計畫需求。 我們大幅減少了需求的數目。 與硬體程式類似，我們建立了一些原則;如果未連結到特定的測試案例，我們會強制執行這些原則。 合作夥伴必須符合所有需求，並觀察其軟體應用程式的所有原則，以符合「與 Windows 7 相容」標誌。

## <a name="process"></a>處理序

當軟體應用程式準備好進行測試時，請移至 Microsoft 連線 Windows 7 軟體標誌計畫頁面，並遵循指示進行：

1.  開始及早審核需求，並找出符合這些需求的任何程式碼變更。
2.  使用 Alpha 自我測試控管來進行驗證下載，並安裝最新版本的工具組。 請注意 readme.txt 檔案中的64位架構和作業系統需求。 依照總覽中所述的工具組執行您的應用程式，以根據需求確認應用程式是否符合規範。
3.  視需要變更任何程式碼 (如有必要) 和重新測試是否需要對應用程式進行任何變更以符合需求、進行必要的變更，並使用工具組重新測試以確認相容性。
4.  若加入宣告以接收電子郵件通知，Microsoft 將會在5月底結束時，開始接受發行 Beta 版工具組的標誌提交。 我們強烈建議您選擇在發行工具組時接收電子郵件通知，以及開啟程式以進行提交。

    只有在發行 Beta 版工具組之後，才可以採取下列步驟：

5.  一旦有 Beta 版工具組可用、下載、安裝及測試您的應用程式，請下載 Beta 版工具組、測試及提交標誌。 將傳遞結果提交給 Microsoft。
6.  一旦提交結果後，簽署軟體標誌授權合約，簽署 [授權合約](/windows-hardware/drivers/dashboard/winqual-submission-tool--winqualexe-)。 然後您就可以下載並開始使用標誌作品，而且有資格接收所有可用的標誌權益。
