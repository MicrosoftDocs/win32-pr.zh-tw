# <a name="scenario-3-performance-counters"></a>案例3：效能計數器

效能計數器會根據所要求或接收的資料量、大小、持續時間和速率來測量資訊或資料的數量。 您不應該預期會取得計數器的詳細資料清單，例如錯誤訊息的清單。 相反地，請使用效能計數器來取得數量，例如啟動後的錯誤訊息數目，或產生錯誤訊息的速率。

## <a name="performance-counters-for-httpsys"></a>HTTP.sys 的效能計數器

從 Windows Vista 和 Windows Server 2008 開始，HTTP.sys 具有下列效能計量計數器，可協助您監視、診斷和規劃網頁伺服器的容量： HTTP 伺服器 API 元件具有下列效能計數器，可協助您監視、診斷和規劃 Web 服務器的容量：

- HTTP 服務計數器：
  - 快取中的 Uri 數目，自啟動後新增、自啟動後刪除，以及快取排清數目
  - 每秒快取點擊次數和快取遺漏數/秒
- HTTP 服務 URL 群組：
  - 資料傳送速率、資料接收速率、傳送和接收的位元組 (傳送) 
  - 連線數目上限、連接嘗試率、GET 和 HEAD 要求的速率，以及要求總數
- HTTP 服務要求佇列：
  - 佇列中的要求數、佇列中最舊要求的存留期 (佇列中最後一個要求的存留期) 
  - 要求抵達佇列的速率、拒絕的速率、拒絕的要求總數、快取點擊率

**存取效能計數器**

1.  在命令提示字元中輸入 **perfmon** ，以啟動效能診斷主控台。
2.  選取樹狀目錄控制項中的 **效能監視器** ，然後按一下以開啟 [ **新增計數器** ] **+** 。
3.  從「 **新增計數器** 」中，選取三個效能計數器集合： **HTTP 服務**、 **Http 服務要求佇列** 或 **HTTP 服務 Url 群組**。
4.  若要從 **Http 服務要求佇列** 和 **Http 服務 Url 群組** 計數器集合中查看計數器，請選取 [ **實例 (s])** 然後按一下 [ **新增**]，再按一下 **[確定]**。 若要查看 HTTP 服務計數器，請選取左窗格中的計數器集合，然後按一下 [ **新增**]。

> [!Note]  
> 每一部電腦只能有一個 HTTP 伺服器 API 計數器實例，因為這些計數器代表全元件的狀態。 使用 Url 群組效能計數器的實例時，效能計數器的實例識別碼 () 將會符合 Url 群組識別碼。 您可以藉由執行 **netsh HTTP show servicestate** 來查看 URL 群組識別碼。 使用要求佇列效能計數器的實例，實例名稱會對應至要求佇列名稱。 要求佇列名稱 (（如果有的話）) 可由相同的 **netsh HTTP show servicestate** 來顯示。 不過，某些伺服器應用程式可能會有無法與效能計數器實例識別碼相符的未命名要求佇列。
