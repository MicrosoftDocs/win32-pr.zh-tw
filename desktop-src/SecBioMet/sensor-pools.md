# <a name="sensor-pools"></a>感應器集區

為了支援 Windows 驗證案例和廠商定義的驗證，Windows 生物特徵辨識服務會將生物識別單位組織成三個可能的感應器集區：

-   **私** 用集區：配置給用戶端應用程式專用的生物識別單位集合。 私人集區可支援不是以 Windows 為基礎的驗證案例，而且可讓應用程式以廠商定義的方式存取生物特徵辨識裝置的硬體。 系統上的私用感應器集區數量可能會和生物識別單位一樣多。
-   **系統感應器** 集區可共用的生物特徵辨識單位集合，以提供 Windows 驗證服務的存取權。 此集區是由 Winlogon、UAC 和任何其他用戶端所使用，這些用戶端會將 SID 與特定生物特徵辨識範本產生關聯。 每個生物特徵辨識服務提供者都有一個系統感應器集區。
-   **未** 指派的集區包含 (可能是未指派給系統集區或私人集區的生物特徵辨識單位) 集合。

應用程式可以使用共用的系統集區，也可以建立從系統或未指派的集區中移除的生物識別單位所組成的私人集區。 當應用程式釋放其私人集區時，會重新設定生物特徵辨識單位，並將其傳回至其原創組區。 為了防止阻絕服務攻擊，只允許具有特殊許可權的使用者從系統集區移除最後一個感應器。 如需詳細資訊，請參閱下列主題。
