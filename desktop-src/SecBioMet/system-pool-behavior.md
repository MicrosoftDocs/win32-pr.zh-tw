# <a name="system-pool-behavior"></a>系統集區行為

下列討論內容會強調傳送事件通知時的系統集區所採取的動作，以及未擱置生物特徵辨識作業的時間。

## <a name="event-dispatching"></a>事件分派

當生物識別單位產生事件通知時，系統集區會使用串聯篩選器來分派通知，並為其指派下列其中一個優先權層級：

-   高優先順序會指派給用戶端所產生的明確相符和註冊要求。
-   將中優先順序指派給非預期或取消認領的相符或註冊事件。
-   低優先順序會指派給導覽事件。

捕捉事件會以下列順序傳遞：

-   如果目前的焦點視窗正在等候比對或註冊作業，則會處理此範例，並將其傳送至擁有目前焦點視窗的用戶端。
-   如果目前焦點視窗取消認領了 capture 事件，而且已向 Windows 生物特徵辨識服務註冊取消認領事件處理常式，則會將 capture 事件傳送至此處理程式。
-   如果事件仍維持取消認領，則會將它捨棄。

如果事件是導覽事件，而且已向 Windows 生物特徵辨識服務註冊導覽事件處理常式，則會將捕獲事件傳送至此處理程式。 如果沒有事件處理常式，則會捨棄該事件。

## <a name="idle-mode"></a>閒置模式

當沒有任何用戶端等待明確比對或註冊要求完成時，系統集區會決定是否要自動產生重複的捕獲要求，並將產生的事件通知傳送至取消認領事件處理常式，或等候導覽事件，並將它們傳送至流覽事件處理常式。

如果已向 Windows 生物識別服務註冊取消認領事件處理常式，系統集區會執行下列動作：

-   感應器的瀏覽模式已停用。
-   無論視窗焦點為何，取消認領作業都會傳送至事件處理常式。
-   如果生物識別作業沒有未處理的要求，則會執行自動捕獲。

如果已向 Windows 生物識別服務註冊導覽處理常式，系統集區會執行下列動作：

-   如果沒有任何生物特徵辨識作業處於擱置狀態，系統集區中的生物特徵辨識單位會置於流覽狀態。
-   如果用戶端傳送符合或註冊事件通知，則會停用導覽事件。
-   如果已註冊取消認領事件處理常式，則會停用導覽事件。
