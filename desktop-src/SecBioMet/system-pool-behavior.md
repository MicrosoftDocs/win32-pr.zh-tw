# <a name="system-pool-behavior"></a><span data-ttu-id="cd376-103">系統集區行為</span><span class="sxs-lookup"><span data-stu-id="cd376-103">System Pool Behavior</span></span>

<span data-ttu-id="cd376-104">下列討論內容會強調傳送事件通知時的系統集區所採取的動作，以及未擱置生物特徵辨識作業的時間。</span><span class="sxs-lookup"><span data-stu-id="cd376-104">The following discussion highlights the actions taken by the system pool when event notices are sent and when no biometric operations are pending.</span></span>

## <a name="event-dispatching"></a><span data-ttu-id="cd376-105">事件分派</span><span class="sxs-lookup"><span data-stu-id="cd376-105">Event Dispatching</span></span>

<span data-ttu-id="cd376-106">當生物識別單位產生事件通知時，系統集區會使用串聯篩選器來分派通知，並為其指派下列其中一個優先權層級：</span><span class="sxs-lookup"><span data-stu-id="cd376-106">When a biometric unit generates an event notice, the system pool uses a cascading filter to dispatch the notice and assign it one of the following priority levels:</span></span>

-   <span data-ttu-id="cd376-107">高優先順序會指派給用戶端所產生的明確相符和註冊要求。</span><span class="sxs-lookup"><span data-stu-id="cd376-107">High priority is assigned to explicit matching and enrollment requests generated by clients.</span></span>
-   <span data-ttu-id="cd376-108">將中優先順序指派給非預期或取消認領的相符或註冊事件。</span><span class="sxs-lookup"><span data-stu-id="cd376-108">Medium priority is assigned to unexpected or unclaimed matching or enrollment events.</span></span>
-   <span data-ttu-id="cd376-109">低優先順序會指派給導覽事件。</span><span class="sxs-lookup"><span data-stu-id="cd376-109">Low priority is assigned to navigation events.</span></span>

<span data-ttu-id="cd376-110">捕捉事件會以下列順序傳遞：</span><span class="sxs-lookup"><span data-stu-id="cd376-110">Capture events are delivered in the following sequence:</span></span>

-   <span data-ttu-id="cd376-111">如果目前的焦點視窗正在等候比對或註冊作業，則會處理此範例，並將其傳送至擁有目前焦點視窗的用戶端。</span><span class="sxs-lookup"><span data-stu-id="cd376-111">If the current focus window is waiting for a matching or an enrollment operation, the sample is processed and sent to the client that owns the current focus window.</span></span>
-   <span data-ttu-id="cd376-112">如果目前的焦點視窗取消認領了 capture 事件，而取消認領事件處理常式已向 Windows 生物特徵辨識服務註冊，則會將 capture 事件傳送至此處理程式。</span><span class="sxs-lookup"><span data-stu-id="cd376-112">If the capture event is unclaimed by the current focus window and an unclaimed event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span>
-   <span data-ttu-id="cd376-113">如果事件仍維持取消認領，則會將它捨棄。</span><span class="sxs-lookup"><span data-stu-id="cd376-113">If the event remains unclaimed, it is discarded.</span></span>

<span data-ttu-id="cd376-114">如果事件是導覽事件，而且已向 Windows 生物特徵辨識服務註冊流覽事件處理常式，則會將 capture 事件傳送至此處理程式。</span><span class="sxs-lookup"><span data-stu-id="cd376-114">If the event is a navigation event and a navigation event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span> <span data-ttu-id="cd376-115">如果沒有事件處理常式，則會捨棄該事件。</span><span class="sxs-lookup"><span data-stu-id="cd376-115">If there is no event handler, the event is discarded.</span></span>

## <a name="idle-mode"></a><span data-ttu-id="cd376-116">閒置模式</span><span class="sxs-lookup"><span data-stu-id="cd376-116">Idle Mode</span></span>

<span data-ttu-id="cd376-117">當沒有任何用戶端等待明確比對或註冊要求完成時，系統集區會決定是否要自動產生重複的捕獲要求，並將產生的事件通知傳送至取消認領事件處理常式，或等候導覽事件，並將它們傳送至流覽事件處理常式。</span><span class="sxs-lookup"><span data-stu-id="cd376-117">When there are no clients waiting for explicit matching or enrollment requests to complete, the system pool determines whether to automatically generate repeated capture requests and send the resulting event notice to the unclaimed event handler or wait for navigation events and send them to the navigation event handler.</span></span>

<span data-ttu-id="cd376-118">如果已向 Windows 生物特徵辨識服務註冊取消認領事件處理常式，系統集區會執行下列動作：</span><span class="sxs-lookup"><span data-stu-id="cd376-118">If an unclaimed event handler has been registered with the Windows Biometric service, the system pool performs the following actions:</span></span>

-   <span data-ttu-id="cd376-119">感應器的瀏覽模式已停用。</span><span class="sxs-lookup"><span data-stu-id="cd376-119">The navigation mode for the sensor is disabled.</span></span>
-   <span data-ttu-id="cd376-120">無論視窗焦點為何，取消認領作業都會傳送至事件處理常式。</span><span class="sxs-lookup"><span data-stu-id="cd376-120">Unclaimed operations are sent to the event handler regardless of window focus.</span></span>
-   <span data-ttu-id="cd376-121">如果生物識別作業沒有未處理的要求，則會執行自動捕獲。</span><span class="sxs-lookup"><span data-stu-id="cd376-121">If there are no outstanding requests for a biometric operation, an automatic capture is performed.</span></span>

<span data-ttu-id="cd376-122">如果已向 Windows 生物特徵辨識服務註冊流覽處理常式，系統集區會執行下列動作：</span><span class="sxs-lookup"><span data-stu-id="cd376-122">If a navigation handler has been registered with the Windows Biometric service, the system pool does the following:</span></span>

-   <span data-ttu-id="cd376-123">如果沒有任何生物特徵辨識作業處於擱置狀態，系統集區中的生物特徵辨識單位會置於流覽狀態。</span><span class="sxs-lookup"><span data-stu-id="cd376-123">The biometric units in the system pool are placed into a navigation state if no biometric operations are pending.</span></span>
-   <span data-ttu-id="cd376-124">如果用戶端傳送符合或註冊事件通知，則會停用導覽事件。</span><span class="sxs-lookup"><span data-stu-id="cd376-124">Navigation events are disabled if a match or enrollment event notice is sent by a client.</span></span>
-   <span data-ttu-id="cd376-125">如果已註冊取消認領事件處理常式，則會停用導覽事件。</span><span class="sxs-lookup"><span data-stu-id="cd376-125">If an unclaimed events handler has been registered, navigation events are disabled.</span></span>
