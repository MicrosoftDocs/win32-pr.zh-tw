# <a name="desktop-app-functionality-is-impacted-if-not-run-in-windowed-mode"></a>如果未在視窗模式中執行，則會影響桌面應用程式功能

在 Windows 10 中，Windows 的應用程式預設不再是全螢幕，而是以視窗化的方式運作，以及最小化、還原、最大化、調整大小 (以及您一直可以對任何其他傳統 Windows 應用程式) 視窗進行調整大小的任何其他作業，現在可以正常運作。

## <a name="manifestations"></a>表現

現在最顯著的變更，就是您可以將大小調整為任意大小，而不只是螢幕的螢幕/高度。 使用者可以持續將應用程式視窗的大小調整為其喜好 (應用程式的最小視窗大小) 。 這與 Windows 8 .0 或 Windows 8.1 不同，因為調整視窗大小的動作是一種獨立的動作 (使用者調整視窗縮圖的大小，然後在使用者認可動作) 之後，重設視窗大小。 現在，當使用者將視窗拖曳 (或其他框線位置時，) 它是連續的調整大小，而應用程式會收到調整資料列大小的訊息，而不是大小變更。

## <a name="mitigations"></a>風險降低

為 Windows 8 的0和8.1 應用程式緩解此問題：

-   如果應用程式功能中的預期功能已中斷，則使用者的緩和措施是使用標題列右上角的 [移全螢幕] 按鈕 (將應用程式放入全螢幕模式中) 。
-   如果應用程式啟動受到影響，但未如預期般開啟，則使用者仍應能切換至 Tablet 模式，以強制應用程式在全螢幕模式下啟動，而不需要使用者介入。

處理此情況的最佳方式是變更應用程式，以瞭解它可以調整大小為非監視器大小的位置/高度 (亦即，沒有高度/寬度的硬式編碼資料表，且只預期這些，或預期的硬式編碼比例) 。 應用程式開發人員應該預期在調整應用程式時，可能會在目前的調整大小訊息出現後立即進行其他調整大小的訊息，因此，如果應用程式在調整大小期間開始動畫，可能會在 (之後，應用程式可能會收到另一個調整大小的訊息，因此請務必確保這種情況不會導致應用程式損毀)
