# <a name="inputtype-complex-type"></a>InputType 複雜型別

定義輸入資料類型。

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>子元素



| 元素                                                                  | 類型                                                             | Description                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**描述**](eventmanifestschema-description-inputtype-element.md) | 字串                                                           | 輸入類型的描述。<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**outType**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | 決定如何呈現資料的輸出資料類型清單。 您必須至少指定一個輸出類型。 如果輸入類型可以轉譯為不同的輸出類型，請指定清單。 例如，輸入類型 win： UInt16 可以轉譯為不帶正負號的短、埠號碼或十六進位數位。 您指定的其中一個輸出類型必須將 [預設屬性] 設定為 [true]。<br/> |



## <a name="attributes"></a>屬性



| 名稱   | 類型                                                              | 描述                                                                                                                                                                                                                                              |
|--------|-------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| NAME   | **QName**                                                         | 輸入類型的名稱。<br/>                                                                                                                                                                                                                   |
| 符號 | [**CSymbolType**](eventmanifestschema-csymboltype-simpletype.md) | 用來參考應用程式中輸入類型的符號。 [**訊息編譯器 (MC.exe)**](message-compiler--mc-exe-.md)會使用符號，在編譯器所產生的標頭檔中建立輸入類型的常數。<br/> |
| value  | 字串                                                            | 在您定義的輸入類型清單中，可唯一識別輸入類型的數值識別碼。<br/>                                                                                                                                        |
