# <a name="ipv6-automatic-tunneling-and-6to4"></a>IPv6 自動通道和6to4

與 IPv4 相容的位址和6to4 的自動通道都是透過連結至介面2的首碼來進行 \# 。 前置詞後面的32位會解壓縮，並用作封裝封包的 IPv4 目的地位址。

自動通道使用：：/96 首碼，預設為啟用。 您可以藉由移除下列各項的路由來停用：：/96。

6to4 會使用2002::/16 首碼（預設不會啟用）。
