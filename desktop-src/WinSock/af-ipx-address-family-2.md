# <a name="af_ipx-address-family"></a>AF \_ IPX 位址系列

IPX 位址系列會定義使用標準 IPX 通訊端定址之通訊協定的定址結構。 針對這些傳輸，端點位址是由網路編號、節點位址和通訊端編號所組成。

網路編號是系統管理網域，通常會命名單一乙太網路或權杖環形區段。 節點編號是工作站的實體位址。 Net 和 node 的組合形成唯一的站位址，在世界中假設是唯一的。 淨和節點編號會以 ASCII 文字表示，區塊或虛線標記法為： ' 0101a040，00001b498765 ' 或 ' 01-01-a0-40，00-00-1b-49-87-65 '。 前置零不需要存在。

IPX 通訊端編號是網路/傳輸服務編號，與 TCP 通訊埠編號很類似，且不會與 Winsock 通訊端描述元混淆。 IPX 通訊端編號對終端機而言是全域的，且無法系結至特定的網路/節點位址。 比方說，如果終端工作站有兩張網路介面卡，系結的通訊端可以在這兩張卡片上傳送和接收。 尤其是，資料包通訊端會在這兩張卡片上接收廣播資料包。

> [!Caution]  
> SOCKADDR \_ IPX 長度為14個位元組，且比16位元組的 [**SOCKADDR**](sockaddr-2.md) 參考結構短。 IPX/SPX 的執行可能會接受16位元組長度以及真正的長度。 如果您使用 SOCKADDR \_ 的 IPX 和硬式編碼長度的16個位元組，則實作為可能會假設它可以存取您的結構之後的2個位元組。

 



| 欄位           | 值                                    |
|-----------------|------------------------------------------|
| **sa \_ 系列**  | 依 \_ 主機順序的位址家族 AF IPX。    |
| **Sa \_ netnum**  | 網路順序中的 IPX 網路識別碼。 |
| **Sa \_ nodenum** | 工作站節點位址，排清右邊。     |
| **Sa \_ 通訊端**  | 網路順序的 IPX 通訊端編號。      |
