# <a name="graceful-shutdown-linger-options-and-socket-closure"></a><span data-ttu-id="79fdb-104">正常關機、延遲選項和通訊端關閉</span><span class="sxs-lookup"><span data-stu-id="79fdb-104">Graceful Shutdown, Linger Options, and Socket Closure</span></span>

<span data-ttu-id="79fdb-105">以下是針對關閉通訊端的關閉通訊端連線主體所提供的詳細資料。</span><span class="sxs-lookup"><span data-stu-id="79fdb-105">The following material is provided as clarification for the subject of shutting down socket connections closing the sockets.</span></span> <span data-ttu-id="79fdb-106">請務必區分關機通訊端連線和關閉通訊端之間的差異。</span><span class="sxs-lookup"><span data-stu-id="79fdb-106">It is important to distinguish the difference between shutting down a socket connection and closing a socket.</span></span>

<span data-ttu-id="79fdb-107">關閉通訊端連線時，會在這兩個端點之間交換通訊協定訊息，而這兩個端點又稱為關機順序。</span><span class="sxs-lookup"><span data-stu-id="79fdb-107">Shutting down a socket connection involves an exchange of protocol messages between the two endpoints, hereafter referred to as a shutdown sequence.</span></span> <span data-ttu-id="79fdb-108">系統會定義關機順序的兩個一般類別：「正常」和「abortive (也稱為「硬) 」。</span><span class="sxs-lookup"><span data-stu-id="79fdb-108">Two general classes of shutdown sequences are defined: graceful and abortive (also called hard).</span></span> <span data-ttu-id="79fdb-109">在正常關機順序中，任何已排入佇列但尚未傳送的資料，都可以在關閉連接之前傳送。</span><span class="sxs-lookup"><span data-stu-id="79fdb-109">In a graceful shutdown sequence, any data that has been queued, but not yet transmitted can be sent prior to the connection being closed.</span></span> <span data-ttu-id="79fdb-110">在 abortive 關機中，任何未傳送的資料都會遺失。</span><span class="sxs-lookup"><span data-stu-id="79fdb-110">In an abortive shutdown, any unsent data is lost.</span></span> <span data-ttu-id="79fdb-111"> (正常或 abortive) 出現的關機順序，也可以用來提供 FD \_ 關閉指示給關聯的應用程式，表示正在進行關機。</span><span class="sxs-lookup"><span data-stu-id="79fdb-111">The occurrence of a shutdown sequence (graceful or abortive) can also be used to provide an FD\_CLOSE indication to the associated applications signifying that a shutdown is in progress.</span></span>

<span data-ttu-id="79fdb-112">另一方面，關閉通訊端會導致通訊端控制碼被解除配置，讓應用程式無法再以任何方式參考或使用通訊端。</span><span class="sxs-lookup"><span data-stu-id="79fdb-112">Closing a socket, on the other hand, causes the socket handle to become deallocated so that the application can no longer reference or use the socket in any manner.</span></span>

<span data-ttu-id="79fdb-113">在 Windows 通訊端中， [**shutdown**](/windows/desktop/api/winsock/nf-winsock-shutdown) 函式和 [**WSASendDisconnect**](/windows/desktop/api/Winsock2/nf-winsock2-wsasenddisconnect) 函式都可以用來起始關機順序，而 [**導致 closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) 函式則可用來解除配置通訊端控制碼，並釋出任何相關聯的資源。</span><span class="sxs-lookup"><span data-stu-id="79fdb-113">In Windows Sockets, both the [**shutdown**](/windows/desktop/api/winsock/nf-winsock-shutdown) function, and the [**WSASendDisconnect**](/windows/desktop/api/Winsock2/nf-winsock2-wsasenddisconnect) function can be used to initiate a shutdown sequence, while the [**closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) function is used to deallocate socket handles and free up any associated resources.</span></span> <span data-ttu-id="79fdb-114">不過， **導致 closesocket** 函式會隱含地導致關閉順序發生（如果尚未發生），因此會產生一些混淆。</span><span class="sxs-lookup"><span data-stu-id="79fdb-114">Some amount of confusion arises, however, from the fact that the **closesocket** function implicitly causes a shutdown sequence to occur if it has not already happened.</span></span> <span data-ttu-id="79fdb-115">事實上，它已經成為依賴這項功能的一般程式設計實務，並且使用 **導致 closesocket** 來起始關機順序並解除配置通訊端控制碼。</span><span class="sxs-lookup"><span data-stu-id="79fdb-115">In fact, it has become a rather common programming practice to rely on this feature and to use **closesocket** to both initiate the shutdown sequence and deallocate the socket handle.</span></span>

<span data-ttu-id="79fdb-116">為了促進這種使用方式，通訊端介面會透過通訊端選項機制來提供控制項，讓程式設計人員能夠指出隱含關機順序是否應該是正常的或 abortive，以及 [**導致 closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) 函式是否應該停止 (不會立即完成) 以允許時間正常關機順序完成。</span><span class="sxs-lookup"><span data-stu-id="79fdb-116">To facilitate this usage, the sockets interface provides for controls by way of the socket option mechanism that allow the programmer to indicate whether the implicit shutdown sequence should be graceful or abortive, and also whether the [**closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) function should linger (that is not complete immediately) to allow time for a graceful shutdown sequence to complete.</span></span> <span data-ttu-id="79fdb-117">這些重要的差異和以這種方式使用 **導致 closesocket** 的後果仍不會被廣泛瞭解。</span><span class="sxs-lookup"><span data-stu-id="79fdb-117">These important distinctions and the ramifications of using **closesocket** in this manner are still not widely understood.</span></span>

<span data-ttu-id="79fdb-118">藉由為通訊端選項建立適當的值以進行 \_ 延遲和 \_ DONTLINGER，您可以使用 [**導致 closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) 函數來取得下列類型的行為：</span><span class="sxs-lookup"><span data-stu-id="79fdb-118">By establishing appropriate values for the socket options SO\_LINGER and SO\_DONTLINGER, the following types of behavior can be obtained with the [**closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket) function:</span></span>

-   <span data-ttu-id="79fdb-119">Abortive 關機順序，從 [**導致 closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket)立即返回。</span><span class="sxs-lookup"><span data-stu-id="79fdb-119">Abortive shutdown sequence, immediate return from [**closesocket**](/windows/desktop/api/winsock/nf-winsock-closesocket).</span></span>
