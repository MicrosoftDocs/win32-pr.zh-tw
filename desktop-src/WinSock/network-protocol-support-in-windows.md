# <a name="winsock-network-protocol-support-in-windows"></a><span data-ttu-id="2d7e6-103">Windows 中的 Winsock 網路通訊協定支援</span><span class="sxs-lookup"><span data-stu-id="2d7e6-103">Winsock Network Protocol Support in Windows</span></span>

<span data-ttu-id="2d7e6-104">網際網路通訊協定套件是商業網路和網際網路上所使用的主要網路通訊協定。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-104">The Internet Protocol Suite is the dominant network protocol used in enterprise networks and across the Internet.</span></span> <span data-ttu-id="2d7e6-105">網際網路通訊協定套件代表大型的分層網路通訊協定集合。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-105">The Internet Protocol Suite represents a large collection of layered network protocols.</span></span> <span data-ttu-id="2d7e6-106">網際網路通訊協定套件通常是以套件中包含的兩個最重要的通訊協定為基礎，稱為 TCP/IP：網際網路通訊協定 (IP) 和傳輸控制通訊協定 (TCP) 。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-106">The Internet Protocol Suite is often referred to as TCP/IP based on two of the most important protocols included in the suite: the Internet Protocol (IP) and the Transmission Control Protocol (TCP).</span></span>

<span data-ttu-id="2d7e6-107">IPv6 和 IPv4 代表網際網路通訊協定的兩個可用版本。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-107">IPv6 and IPv4 represent the two available versions of the Internet Protocol.</span></span> <span data-ttu-id="2d7e6-108">TCP 是數個重要網路服務的其中一種，通常稱為 IP 通訊協定，可透過 IPv6 和 IPv4 網路運作。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-108">TCP is one of several important network services often referred to as IP protocols that operate over IPv6 and IPv4 networks.</span></span> <span data-ttu-id="2d7e6-109">使用者資料包協定 (UDP) 和網際網路控制訊息通訊協定 (ICMP) 是其他透過 IPv6 和 IPv4 網路使用的重要 IP 通訊協定。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-109">The User Datagram Protocol (UDP) and Internet Control Message Protocol (ICMP) are other important IP protocols used over IPv6 and IPv4 networks.</span></span> <span data-ttu-id="2d7e6-110">有一些其他 IP 通訊協定可用於 IPv6 和 IPv4 網路。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-110">There are a number of other IP protocols that can be used over IPv6 and IPv4 networks.</span></span>

<span data-ttu-id="2d7e6-111">Windows 通訊端會將每個網路通訊協定套件視為唯一的位址系列。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-111">Windows Sockets considers each network protocol suite as a unique address family.</span></span> <span data-ttu-id="2d7e6-112">因此，IPv6 通訊協定被視為 **af \_ INET6** 位址家族，而 IPv4 通訊協定則視為 **af \_ INET** 位址系列。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-112">So the IPv6 protocol is considered the **AF\_INET6** address family and the IPv4 protocol is considered the **AF\_INET** address family.</span></span> <span data-ttu-id="2d7e6-113">IPv6 和 IPv4 通訊協定支援使用各種分層式 IP 通訊協定，例如 TCP、UDP 和 ICMP。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-113">The IPv6 and IPv4 protocols support the use of various layered IP protocols such as TCP, UDP, and ICMP.</span></span>

<span data-ttu-id="2d7e6-114">Windows 通訊端一開始是設計來將 IPv4 的支援新增至 Windows。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-114">Windows Sockets were initially designed to add support for IPv4 to Windows.</span></span> <span data-ttu-id="2d7e6-115">不過，Windows 通訊端程式設計介面是設計自驗證，而且能夠支援其他網路通訊協定套件。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-115">However, the Windows Sockets programming interface was designed from the onset with the ability to support other network protocol suites.</span></span> <span data-ttu-id="2d7e6-116">經過一段時間之後，Windows 的版本和相關聯的 Windows 通訊端已包含其他網路通訊協定套件的原生支援 (IPX/SPX 和 AppleTalk，例如) 。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-116">Over time, versions of Windows and the associated Windows Sockets have included native support for other network protocol suites (IPX/SPX and AppleTalk, for example).</span></span> <span data-ttu-id="2d7e6-117">其他網路通訊協定的支援也適用于 Windows 版本，作為廠商提供的協力廠商軟體。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-117">Support for other network protocols was also available for versions of Windows as third-party software from vendors.</span></span>

<span data-ttu-id="2d7e6-118">在網際網路的成長和熱門程度之前，會在網路環境中使用各種其他網路通訊協定套件，特別是針對本機內部網路。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-118">Before the growth and popularity of the Internet, various other network protocol suites were used in networked environments, particularly for local Intranets.</span></span> <span data-ttu-id="2d7e6-119">網路通訊協定套件的選擇通常是根據網路大小或 IT 網路人員的專業知識。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-119">The choice of a network protocol suite was often based on the size of the network or the expertise of the IT networking staff.</span></span> <span data-ttu-id="2d7e6-120">由於現今的全球網際網路連線能力，即使是最小的網路，也能連結到全球各地的網路專業人員。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-120">With today's global Internet connectivity linking even the smallest networks to the rest of the world, networking expertise in IPv6 and IPv4 is essential for networking professionals.</span></span> <span data-ttu-id="2d7e6-121">如此一來，其他先前重要的網路通訊協定套件現在會以非常有限的方式使用，並已成為排除。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-121">As a result, other previously important network protocol suites are now in very limited use and have become obviated.</span></span> <span data-ttu-id="2d7e6-122">這些排除網路通訊協定套件的原生支援（通常稱為傳統網路通訊協定）已從最新版本的 Microsoft Windows 中卸載。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-122">Native support for these obviated network protocol suites, often referred to as legacy network protocols, has been dropped from recent versions of Microsoft Windows.</span></span> <span data-ttu-id="2d7e6-123">其中某些舊版通訊協定的支援，可能會以廠商的協力廠商軟體的形式提供， (具有 ATM 網路硬體的 ATM，例如) 。</span><span class="sxs-lookup"><span data-stu-id="2d7e6-123">Support for some of these legacy protocols may be available as third-party software from vendors (ATM with ATM network hardware, for example).</span></span>
