# <a name="dual-stack-sockets-for-ipv6-winsock-applications"></a><span data-ttu-id="95944-103">IPv6 Winsock 應用程式 Dual-Stack 通訊端</span><span class="sxs-lookup"><span data-stu-id="95944-103">Dual-Stack Sockets for IPv6 Winsock Applications</span></span>

<span data-ttu-id="95944-104">為了在 Windows XP Service Pack 1 上支援 IPv4 和 IPv6 (SP1) 和 Windows Server 2003 上的應用程式，應用程式必須建立兩個通訊端，一個通訊端用於 IPv4，另一個通訊端用於 IPv6。</span><span class="sxs-lookup"><span data-stu-id="95944-104">In order to support both IPv4 and IPv6 on Windows XP with Service Pack 1 (SP1) and on Windows Server 2003, an application has to create two sockets, one socket for use with IPv4 and one socket for use with IPv6.</span></span> <span data-ttu-id="95944-105">這兩個通訊端必須由應用程式分開處理。</span><span class="sxs-lookup"><span data-stu-id="95944-105">These two sockets must be handled separately by the application.</span></span>

<span data-ttu-id="95944-106">Windows Vista 和更新版本可讓您建立單一 IPv6 通訊端，以同時處理 IPv6 和 IPv4 流量。</span><span class="sxs-lookup"><span data-stu-id="95944-106">Windows Vista and later offer the ability to create a single IPv6 socket which can handle both IPv6 and IPv4 traffic.</span></span> <span data-ttu-id="95944-107">例如，會建立 IPv6 的 TCP 接聽通訊端，並進入雙堆疊模式，並系結至埠5001。</span><span class="sxs-lookup"><span data-stu-id="95944-107">For example, a TCP listening socket for IPv6 is created, put into dual stack mode, and bound to port 5001.</span></span> <span data-ttu-id="95944-108">此雙重堆疊通訊端可以接受從連線到埠5001的 IPv6 TCP 用戶端，以及從連線到埠5001的 IPv4 TCP 用戶端的連線。</span><span class="sxs-lookup"><span data-stu-id="95944-108">This dual-stack socket can accept connections from IPv6 TCP clients connecting to port 5001 and from IPv4 TCP clients connecting to port 5001.</span></span> <span data-ttu-id="95944-109">這項功能可大幅簡化應用程式設計，並減少在兩個不同的通訊端上張貼作業所需的資源負擔。</span><span class="sxs-lookup"><span data-stu-id="95944-109">This feature allows for greatly simplified application design and reduces the resource overhead required of posting operations on two separate sockets.</span></span>

## <a name="creating-a-dual-stack-socket"></a><span data-ttu-id="95944-110">建立 Dual-Stack 通訊端</span><span class="sxs-lookup"><span data-stu-id="95944-110">Creating a Dual-Stack Socket</span></span>

<span data-ttu-id="95944-111">在 Windows Vista 和更新版本上建立的 IPv6 通訊端預設只會透過 IPv6 通訊協定運作。</span><span class="sxs-lookup"><span data-stu-id="95944-111">By default, an IPv6 socket created on Windows Vista and later only operates over the IPv6 protocol.</span></span> <span data-ttu-id="95944-112">若要讓 IPv6 通訊端成為雙重堆疊通訊端，必須使用 **IPv6 \_ V6ONLY** 通訊端選項呼叫 [**setsockopt**](/windows/desktop/api/winsock/nf-winsock-setsockopt)函式，將此值設定為零，再將通訊端系結至 IP 位址。</span><span class="sxs-lookup"><span data-stu-id="95944-112">In order to make an IPv6 socket into a dual-stack socket, the [**setsockopt**](/windows/desktop/api/winsock/nf-winsock-setsockopt) function must be called with the **IPV6\_V6ONLY** socket option to set this value to zero before the socket is bound to an IP address.</span></span> <span data-ttu-id="95944-113">當 **ipv6 \_ V6ONLY** 通訊端選項設定為零時，為 **AF \_ INET6** 位址系列建立的通訊端可以用來在 IPV6 位址或 IPv4 對應位址之間傳送和接收封包。</span><span class="sxs-lookup"><span data-stu-id="95944-113">When the **IPV6\_V6ONLY** socket option is set to zero, a socket created for the **AF\_INET6** address family can be used to send and receive packets to and from an IPv6 address or an IPv4 mapped address.</span></span>

## <a name="ip-addresses-with-a-dual-stack-socket"></a><span data-ttu-id="95944-114">具有 Dual-Stack 通訊端的 IP 位址</span><span class="sxs-lookup"><span data-stu-id="95944-114">IP Addresses with a Dual-Stack Socket</span></span>

<span data-ttu-id="95944-115">雙堆疊通訊端一律需要 IPv6 位址。</span><span class="sxs-lookup"><span data-stu-id="95944-115">Dual-stack sockets always require IPv6 addresses.</span></span> <span IPv4-mapped IPv6 address format.</span></span> <span data-ttu-id="95944-117">任何 IPv4 位址都必須以 IPv4 對應的 IPv6 位址格式表示，讓 IPv6 的應用程式能夠與 IPv4 節點進行通訊。</span><span class="sxs-lookup"><span IPv4 node to be represented as an IPv6 address.</span></span> <span data-ttu-id="95944-119">IPv4 位址會編碼為 IPv6 位址的低序位32位，而高序位96位會保存固定的前置詞0：0：0：0：0： FFFF。</span><span class="sxs-lookup"><span data-stu-id="95944-119">The IPv4 address is encoded into the low-order 32 bits of the IPv6 address, and the high-order 96 bits hold the fixed prefix 0:0:0:0:0:FFFF.</span></span> <span data-ttu-id="95944-120">IPv4 對應的 IPv6 位址格式是在 RFC 4291 中指定的。</span><span class="sxs-lookup"><span data-stu-id="95944-120">The IPv4-mapped IPv6 address format is specified in RFC 4291.</span></span> <span data-ttu-id="95944-121">如需詳細資訊，請參閱 [www.ietf.org/rfc/rfc4291.txt](https://tools.ietf.org/html/rfc4291)。</span><span class="sxs-lookup"><span data-stu-id="95944-121">For more information, see [www.ietf.org/rfc/rfc4291.txt](https://tools.ietf.org/html/rfc4291).</span></span> <span data-ttu-id="95944-122">\_ *Mstcpip* 中的 IN6ADDR SETV4MAPPED 宏可以用來將 ipv4 位址轉換為所需的 ipv4 對應 IPv6 位址格式。</span><span class="sxs-lookup"><span data-stu-id="95944-122">The IN6ADDR\_SETV4MAPPED macro in *Mstcpip.h* can be used to convert an IPv4 address to the required IPv4-mapped IPv6 address format.</span></span>

<span data-ttu-id="95944-123">如果基礎通訊協定實際上是 IPv4，則 IPv4 位址會對應至 IPv4 對應的 IPv6 位址格式。</span><span class="sxs-lookup"><span data-stu-id="95944-123">If the underlying protocol is actually IPv4, then the IPv4 address is mapped into an IPv4-mapped IPv6 address format.</span></span> <span data-ttu-id="95944-124">亦即， [SOCKADDR](sockaddr-2.md) 結構中的 [系列] 欄位會指出 AF \_ INET6，但 IPv4 對應的 ipv6 位址會編碼于 IPv6 位址結構中。</span><span class="sxs-lookup"><span data-stu-id="95944-124">That is the, family field in the [SOCKADDR](sockaddr-2.md) structure indicates AF\_INET6, but an IPv4-mapped IPv6 address is encoded in the IPv6 address structure.</span></span> <span data-ttu-id="95944-125">在接聽模式的雙重堆疊通訊端中，這表示任何已接受的 IPv4 連線都會傳回 IPv4 對應的 IPv6 位址。</span><span class="sxs-lookup"><span data-stu-id="95944-125">For a dual-stack socket in listening mode, this means that any accepted IPv4 connections will return an IPv4-mapped IPv6 address.</span></span> <span data-ttu-id="95944-126">若為連接至 IPv4 目的地的雙重堆疊通訊端，傳遞給 connect 的 SOCKADDR 結構必須是 IPv4 對應的 IPv6 位址。</span><span class="sxs-lookup"><span data-stu-id="95944-126">For a dual-stack socket that is connecting to an IPv4 destination, the SOCKADDR structure passed to connect must be an IPv4-mapped IPv6 address.</span></span> <span data-ttu-id="95944-127">應用程式必須小心地處理這些 IPv4 對應的 IPv6 位址，並只搭配雙重堆疊通訊端使用它們。</span><span class="sxs-lookup"><span data-stu-id="95944-127">Applications must take care to handle these IPv4-mapped IPv6 addresses appropriately and only use them with dual stack sockets.</span></span> <span data-ttu-id="95944-128">如果要將 IP 位址傳遞給一般 IPv4 通訊端，位址必須是一般的 IPv4 位址，而不是 IPv4 對應的 IPv6 位址。</span><span class="sxs-lookup"><span data-stu-id="95944-128">If an IP address is to be passed to a regular IPv4 socket, the address must be a regular IPv4 address not a IPv4-mapped IPv6 address.</span></span>
