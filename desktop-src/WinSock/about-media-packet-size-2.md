# <a name="about-media-packet-size"></a>關於媒體封包大小

媒體封包大小會影響 IPX 通訊協定在網路之間傳輸資料的能力，並可證明以與傳輸無關的方式處理的挑戰。 IPX 不會分割封包，也不會在因為大小違規而捨棄封包時進行報告。 這表示，終端機上的某些實體必須知道要在任何指定的網路路徑上使用的封包大小上限。 傳統上，IPX 資料包和 SPX 連接導向的服務已將這種負擔留給應用程式，而 SPXII 則使用大型的網際網路封包協商以透明的方式處理它。

Winsock 會嘗試為其各種 IPX 通訊協定設定合理的封包大小限制，如下一節所述。 這些限制可透過 get/set 通訊端選項，由應用程式來查看及修改。 判斷封包大小上限時，有三個重點：

-   \# 媒體封包大小
-   \# 可路由傳送的封包大小
-   \# 終端站封包大小

*媒體封包大小* 會反映封包流經目的地的任何媒體上可接受的封包大小上限。 封包大小會隨著不同的媒體（例如乙太網路和權杖環）而有所不同。 封包內的資料空間量也可能在指定的媒體內有所差異，視封包標頭的相片順序而定。 例如，乙太網路封包的有效資料大小會根據其類型是否為 Ethernet II、Ethernet 802.2 或 Ethernet 嵌入式管理單元而有所不同。

可路由傳送的封 *包大小* 反映中繼路由器願意轉寄的封包大小上限。 大部分的 IPX 路由器都是為了路由傳送任何大小的資料包而建立的，只要它仍在傳送和接收網路的媒體大小內。 不過，較舊的路由器可能會將封包大小上限限制為576個位元組，包括通訊協定標頭。

*終端站封包大小* 會反映終端電臺張貼以接收連入封包的接聽緩衝區大小。 即使媒體和路由器的限制允許封包通過，如果接收的應用程式已公佈較小的緩衝區，終端站可能會捨棄該封包。 許多傳統的 IPX/SPX 應用程式會限制接收緩衝區大小，讓資料部分不能大於512或1024位元組。
