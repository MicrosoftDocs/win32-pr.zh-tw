# <a name="windows-media-metafiles-overview"></a>Windows媒體中繼檔總覽

成功使用 Windows 媒體中繼檔最重要的部分，就是針對中繼檔元素使用正確的語法。 Windows 媒體中繼檔中的語法錯誤，可能會導致任何來自被忽略的單一屬性，或中繼檔無法辨識為有效且無法運作。

專案出現在 Windows 媒體中繼檔中的順序幾乎很重要。 某些專案的屬性會暫時覆寫中繼檔不同區段中類似專案的屬性。 有一個已定義 [的優先順序](order-of-precedence.md)。

Windows媒體中繼檔播放清單是 Windows 媒體中繼檔，提供 Windows Media Player 用來接收單點傳送串流、多播串流，以及來自內部網路或網際網路的其他受支援媒體的資訊。 中繼檔播放清單基本上是媒體內容的快捷方式。 中繼檔播放清單可以當做電子郵件傳送，作為網頁上的連結參考、使用 Active Server Pages (ASP) 動態建立，或以獨立檔案的形式存在於本機磁片磁碟機上。 中繼檔播放清單可以參考另一個中繼檔播放清單、ASP 頁面或 Windows 的 Media 站檔案， (副檔名為 .nsc 的) 。 .nsc 檔案是用來定義要 Windows Media Player Windows 媒體站。 每個案例的基本處理常式都相同。
