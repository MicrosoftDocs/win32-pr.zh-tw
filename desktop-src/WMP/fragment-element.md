# <a name="fragment-element"></a>片段元素

**片段** 元素會指定從程式庫中選取專案之查詢的其中一個條件。 條件是由條件字串所指定。 條件字串通常有名稱部分、條件部分和值部分。

``` syntax
<fragment
    name = "string"
>
</fragment>
```

</dt> <dd>

## <a name="remarks"></a>備註

某些條件字串具有中繼資料屬性部分、條件部分和值部分。 例如，在條件字串「專輯演出者是 Joe」中，中繼資料屬性部分是「專輯演出者」，條件部分是「是」，而值部分是「Joe」。

範例：


```XML
<fragment name = "Album Artist">
    <argument name = "condition">Is</argument>
    <argument name = "value">Joe</argument>
</fragment>
```

某些條件字串具有限制器部分、數位部分和格式部分。 例如，在條件字串「限制總大小為 3 Mb」中，限制器部分是「限制總大小」，數位部分為 "3"，而格式部分為 "Mb"。

範例：


```XML
<fragment name = "Limit Total Size To">
  <argument name = "number">3</argument>
  <argument name = "format">Megabytes</argument>
</fragment>
```



針對此類型的條件字串，下表顯示可能的 limiters 和格式。



| 限制                 | 可能的數位 | 可能的格式                |
|-------------------------|------------------|---------------------------------|
| 將大小總計限制為     | 任何數字       | Kb、Mb、Gb |
| 將總持續時間限制為 | 任何數字       | 秒、分鐘、小時、天   |



 

某些條件字串具有限制器部分和數位部分。 例如，在條件字串「將專案數限制為25」中，限制器部分是「限制專案數目」，而數位部分是「25」。

範例：


```XML
<fragment name = "Limit Number of Items">
    <argument name = "number">25</argument>
</fragment>
```



針對此類型的條件字串，下表顯示唯一可能的限制器。



| 限制               | 可能的數位 |
|-----------------------|------------------|
| 限制專案數目 | 任何數字       |



 

某些條件字串具有保護部分和條件部分。 例如，在條件字串「保護存在」中，保護部分是「保護」，而條件部分是「是」。

範例：


```XML
<fragment name = "Protection">
    <argument name = "condition">Is</argument>
</fragment>
```



針對此類型的條件字串，下表顯示可能的條件。



| 保護部分 | 可能的條件             |
|--------------------|---------------------------------|
| 保護         | 為<br/> 不是<br/> |



 

有一種類型的 **片段** 元素未包含條件字串。 如果 **片段** 專案的 name 屬性為「隨機化播放順序」，則 **片段** 元素不會包含任何 **引數** 元素。 這個 **片段** 元素會指示播放清單以隨機順序播放清單。

範例：


```XML
<fragment name = "Randomize Playback Order">
</fragment>
```



某些條件字串具有排序部分、值部分和條件部分。 例如，在條件字串「依標題遞增順序排序」中，排序部分是「排序依據」、值部分是「標題」，而條件部分是「遞增」。 請注意，在此情況下，值部分是中繼資料屬性。

範例：


```XML
<fragment name = "Sort By">
    <argument name = "value">Title</argument>
    <argument name = "condition">Ascending</argument>
</fragment>
```
