## <a name="remoting-in-the-background"></a><span data-ttu-id="83c1e-173">背景中的遠端處理</span><span class="sxs-lookup"><span data-stu-id="83c1e-173">Remoting in the Background</span></span>

<span data-ttu-id="83c1e-174">您應該避免在控制項未使用時，將播放程式的遠端實例保留在背景中執行。</span><span class="sxs-lookup"><span data-stu-id="83c1e-174">You should avoid keeping a remoted instance of the Player running in the background during times when the control is not in use.</span></span> <span data-ttu-id="83c1e-175">由於「遠端播放機控制項」實例會以完整模式播放機共用其播放引擎，讓背景實例保持執行可能會導致非預期的行為。</span><span class="sxs-lookup"><span data-stu-id="83c1e-175">Because the remoted Player control instance shares its playback engine with the full mode Player, keeping a background instance running can cause unexpected behavior.</span></span> <span data-ttu-id="83c1e-176">例如，使用者可能會在檔案播放時關閉完整模式播放機。</span><span class="sxs-lookup"><span data-stu-id="83c1e-176">For example, the user might close the full mode Player while a file is playing.</span></span> <span data-ttu-id="83c1e-177">使用者預期播放程式關閉時，檔案播放會完全停止，但因為播放引擎仍在使用中，所以音訊可能會繼續播放。</span><span class="sxs-lookup"><span data-stu-id="83c1e-177">The user would expect that file playback would completely stop when the Player closes, but audio might continue to play because the playback engine is still active.</span></span>
