# <a name="wpd_command_device_hints_get_content_location-command"></a>WPD \_ 命令 \_ 裝置 \_ 提示 \_ 取得 \_ 內容 \_ 位置命令

**WPD \_ 命令 \_ 裝置 \_ 提示 \_ 取得 \_ 內容 \_ 位置** 命令會抓取可保存指定類型物件之資料夾的物件識別碼。 此命令會以更快的方式提供，讓用戶端探索裝置儲存特定物件的位置，而不是暴力物件列舉。

## <a name="command-category"></a>命令類別目錄

**WPD \_ 類別 \_ 裝置 \_ 提示**

## <a name="parameters"></a>參數

驅動程式需要下列參數。



| 參數                                   | VarType   | 描述                                                                                                                                                                                                                                                                                                                                     |
|---------------------------------------------|-----------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| WPD \_ 屬性 \_ 裝置 \_ 提示 \_ 內容 \_ 類型 | VT \_ CLSID | 必要。 呼叫端希望尋找容器的物件類型。 例如，若要尋找用來在數位相機上保存影像的最上層資料夾，呼叫者會提交 WPD \_ 內容 \_ 類型 \_ 影像。 針對 Windows 可攜式裝置所定義的物件類型清單，請參閱[物件的需求](requirements-for-objects.md)。 |



 

## <a name="return-value"></a>傳回值

驅動程式應該會傳回下列結果。



| 結果                                               | VarType     | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|------------------------------------------------------|-------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **WPD \_ 屬性 \_ 裝置 \_ 提示 \_ 內容 \_ 位置** | VT \_ 不明 | 必要。 VT LPWSTR 數值型別的 [**IPortableDevicePropVariantCollection**](iportabledevicepropvariantcollection.md) \_ ，指定包含由呼叫參數表示之類型物件的資料夾物件識別碼。 如果找不到資料夾，這應該是空的清單。結果所指出的資料夾可能包含或可能不包含其他內容類型的物件。 如需資料夾限制的詳細資訊，請參閱 [WPD \_ 資料夾 \_ 內容 \_ 類型的 \_ 允許](miscellaneous-properties.md) 屬性。<br/> |
| **WPD \_ 屬性 \_ 通用 \_ HRESULT**                   | VT \_ 錯誤   | 必要。 **HRESULT** ，指出處理命令的成功或失敗。 如果呼叫端提出不正確要求，驅動程式應該會傳回 **\_ \_ \_ 不 \_ 支援的 WIN32 (錯誤)** ，而且不需要傳回任何其他的結果值。 錯誤碼包含[Windows 可攜式裝置的錯誤碼](error-constants.md)或任何其他適當的錯誤碼。                                                                                                                                                                            |
| **WPD \_ 屬性 \_ 常見的 \_ 驅動程式 \_ 錯誤碼 \_**       | VT \_ UI4     | 選擇性。 驅動程式特定的錯誤碼。 這通常只會用於驅動程式測試，或者驅動程式、裝置和用戶端都是一起設計的。                                                                                                                                                                                                                                                                                                                                                                                                          |



 

## <a name="calling-methods"></a>呼叫方法

只能使用 [**IPortableDevice：： SendCommand**](/windows/desktop/api/PortableDeviceApi/nf-portabledeviceapi-iportabledevice-sendcommand)直接呼叫。
