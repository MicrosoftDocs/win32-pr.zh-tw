# <a name="cmm-transform-creation-flags"></a>CMM 轉換建立旗標

Cmm 使用轉換建立旗標作為如何建立色彩轉換的提示。 它是由 CMM 決定最適合使用這些旗標的方式。

所有使用這些旗標的函式會透過名為 *dwFlags* 的參數來傳遞或接收旗標值。 *DwFlags* 的高序位 **字** 應該設定為下表中的值。



| 常數                    | 描述                                                                                                                                  |
|-----------------------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| 啟用 \_ GAMUT \_ 檢查     | 使用此轉換進行範圍檢查。                                                                                                       |
| 使用 \_ 相對 \_ 色度 | 請勿保留白色點。 如果輸出 gamut 不支援指定的色彩，請使用最接近的支援色彩。 請參閱轉譯意圖。 |
| 快速 \_ 翻譯             | 只查詢色彩。 請勿插入色彩。                                                                                            |
| PRESERVEBLACK               | 將適當的黑色產生 GMMP 插入轉換序列中的最後一個 GMMP                                                     |
| WCS \_ 永遠                 | 即使是針對 ICC 轉換，也請使用 WCS 程式碼路徑。                                                                                               |
| 順序 \_ 轉換       | 用來建立順序 (非優化) 色彩轉換的轉換建立旗標。                                                           |



 

低序位 **單字** 可以具有下列其中一個常數值。



| 常數     | 描述                                                                                        |
|--------------|----------------------------------------------------------------------------------------------------|
| 證明 \_ 模式  | 轉換將用來預覽映射。 影像品質低。                                    |
| 標準 \_ 模式 | 轉換將用於一般影像顯示。 平均影像品質。                            |
| 最佳 \_ 模式   | 轉換將會用來顯示目標裝置上可能的最高品質影像。 |



 

從證明 \_ 模式移至最佳 \_ 模式，輸出品質通常會改善，並轉換速度下降。
