# <a name="rendering-intents"></a>轉譯意圖

國際色彩協會 (ICC) 已定義四個不同的值，稱為轉譯 *意圖*。 這些代表四種不同的方法來建立色彩轉譯。 這四個意圖和在程式碼中用來參考這些意圖的常數如下所示。



| Intent                            | ICC 名稱              | Description                    |
|-----------------------------------|-----------------------|--------------------------------|
| Picture | 感性            | 意圖 \_ 感知             |
| Graphic | 飽和度            | 意圖 \_ 飽和度             |
| 證明     | 相對色度 | 意圖 \_ 相對 \_ 色度 |
| 比對     | 絕對色度 | 意圖 \_ 絕對 \_ 色度 |




 

您可以從 color.org 下載 ICC 配置檔案格式規格版本3.4 （描述這些意圖）。

## <a name="picture-intent"></a>圖片意圖

在 ICC 規格子句4.9 中稱為 [感知意圖]，圖片意圖會將影像的整個範圍壓縮或展開以 [填滿目的地](./g.md) 裝置的範圍，因此會保留灰色平衡，但不會保留色階的精確度。

換句話說，如果影像中的某些色彩落在輸出裝置可轉譯的色彩範圍之外，圖片意圖將會調整影像中的所有色彩，以便影像中的每個色彩都落在可轉譯的範圍內，如此一來，色彩之間的關聯性就會盡可能地保留。

此意圖最適合顯示相片和影像，而且通常是預設意圖。

## <a name="graphic-intent"></a>圖形意圖

ICC 規格子句4.12 會以 [飽和度](s.md) 意圖呼叫圖形意圖。 它會將色彩的色度保留在影像中，可能會有 [色調](h.md) 和 [亮度](l.md)的費用。

這種意圖的實現仍有一些問題，而且 ICC 仍在使用方法來達成所需的效果。

這種意圖最適合用來做為圖表等商務圖形，在這種情況下更重要的是，色彩更重要，且彼此對比，而不是特定的色彩。

## <a name="proof-intent"></a>證明意圖

證明意圖（即 ICC 規格中的色度意圖）的定義，是在輸出裝置可轉譯的範圍之外的任何色彩都調整為可轉譯的最接近色彩，而其他所有色彩則會保持不變。

證明意圖不會保留 [白色點](w.md)。

例如，紙張的 whitest 白色會比電腦監視器的 whitest 白色更黃。 使用相對色度意圖將影像轉換成印表機的範圍，會導致所有色彩變得更黃色。 影像的白色點會移動以符合印表機的白色點。 影像中的所有其他色彩都會保持其相對於白色點的位置。 這會產生一個影像，以更精確地反映出印刷影像的樣子。 不過，使用者可能會發現它以視覺化的方式令人不安。

## <a name="match-intent"></a>符合意圖

在比對意圖中，任何落在輸出裝置所能轉譯範圍之外的色彩，都會調整為最接近可以轉譯的色彩，而其他所有色彩則會保持不變。 ICC 規格會呼叫 match 意圖的絕對色度意圖。

比對意圖會保留白色點。

例如，紙張的 whitest 白色會比電腦監視器的 whitest 白色更黃。 使用 match 意圖將影像轉換成 [印表機的範圍，會](./g.md) 導致所有色彩轉換並符合印表機的範圍。 影像的白色點不會移動以符合印表機的白色點。 因此，色彩與白色點之間的距離可能會變更。 這會產生較不會以視覺方式令人不安給使用者的影像，但也是較不精確的印表機輸出呈現方式。
