# <a name="color-conversion-and-color-matching"></a>色彩轉換和色彩比對

將色彩從某個 [色彩空間](c.md) 轉換成另一個色彩的程式稱為 *色彩轉換*。 色彩空間中的所有色彩都是相對於色彩空間的 [白色點](w.md)固定的。 由於色彩空間的白色點會因裝置而異，因此轉換後的色彩必須符合目的色彩空間中其視覺效果最接近的色彩。 此進程稱為 *色彩對應*。

例如，如果您有在顯示器上建立的數位影像，它可能會在與裝置相關的 RGB 色彩空間中。 如果您想要在印表機上列印，則必須將它轉換成印表機的色彩空間。 因為印表機可能會使用裝置相關的 CMYK 色彩空間，所以所有的 RGB 值都必須轉換成 CYMK。 這是 [色彩轉換](c.md)。 一旦根據 CYMK 空間指定值，就必須與印表機所能產生的最接近色彩相符。 這稱為色彩對應或 [色彩](c.md)比對。

色彩轉換和色彩對應都必須考慮許多裝置特定的因素。 例如，螢幕影像的建立區塊是圖元。 每個圖元都有一組要儲存其色彩或色彩索引值的位數。 每圖元的位數取決於所使用的顯示和顯示介面卡類型，以及該介面卡的設定模式。 大部分的印表機類型都會使用不同的 [colorants](c.md) ，並在特定的解析度列印。

此外，裝置的實體色調特性通常會在不同的裝置上有所不同。 比方說，當色彩是由電腦監視器產生時，它們的顯示方式可能會與按下列印時所產生的色彩不同。 更正因數（稱為 *gamma 更正*）經常用來彌補這類差異。

這些裝置相關因素的結果是每個色彩裝置都有一組特定的色彩可產生。 此色彩集 *稱為其範圍*。 若要執行色彩轉換和色彩對應，影像中的色彩必須從來源裝置的色彩空間和範圍轉換成目的地裝置的色彩空間。 然後，它們會符合目的地裝置的範圍。
