# <a name="directional-attributes-applied-to-the-parameter"></a>套用至參數的方向屬性

和輸出中的方向屬性會 \[ [](/windows/desktop/Midl/in) \] \[ [](/windows/desktop/Midl/out-idl) \] 決定用戶端和伺服器如何配置和釋放記憶體。 下表摘要說明雙向屬性對記憶體配置的影響。



| 方向屬性    | 用戶端上的記憶體                                                                                            | 伺服器上的記憶體                                                                                                                                        |
|--------------------------|-------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| \[[in](/windows/desktop/Midl/in)\]       | 用戶端應用程式必須在呼叫之前配置。                                                           | 伺服器 stub 會配置。                                                                                                                                  |
| \[[out](/windows/desktop/Midl/out-idl)\] | 用戶端 stub 會在傳回時配置。                                                                            | 伺服器 stub 只會配置最上層指標;伺服器應用程式必須配置所有內嵌指標。 伺服器也會視需要配置新的資料。 |
| \[**in**、 **out**\]      | 用戶端應用程式必須配置傳送至伺服器的初始資料;用戶端 stub 會配置額外的資料。 | 伺服器 stub 會配置從用戶端傳輸的初始資料;伺服器應用程式會視需要配置新的資料。                                        |



 

在上述所有情況下，用戶端 stub 都不會釋放記憶體。 用戶端應用程式必須在記憶體終止之前釋放記憶體。 當遠端程序呼叫傳回 (受制于 \[ [配置](/windows/desktop/Midl/allocate) \] ACF 屬性) 時，伺服器 stub 會釋出記憶體。
