# <a name="use-context-handles-for-keeping-state-on-the-server"></a>使用內容控制碼在伺服器上保持狀態

**最佳做法：** 每當呼叫之間的狀態保留在伺服器上時，請使用內容控制碼。

內容控制碼通常是針對新的 RPC 程式設計人員所很嚇人，而學習內容控制碼的額外負荷可能不會一開始就需要投入時間。 這是因為內容控制碼有內建的解決方案，可用於網路程式設計中遇到的許多常見陷阱，否則就必須從頭開始執行。 瞭解內容控點會在稍後支付紅利。
