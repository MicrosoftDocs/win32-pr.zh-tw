# <a name="who-manages-memory"></a>神秘管理記憶體？

一般來說，存根負責封裝和解除封裝資料、配置和釋放記憶體，以及將資料傳入和傳出記憶體。 不過，在某些情況下，應用程式會負責配置和釋放記憶體。 下列主題將討論判斷哪個元件負責記憶體管理的因素：

-   [最上層和內嵌指標](top-level-and-embedded-pointers.md)
-   [套用至參數的方向屬性](directional-attributes-applied-to-the-parameter.md)
-   [長度、大小和方向屬性](length-size-and-directional-attributes.md)
-   [套用至參數的指標屬性](pointer-attributes-applied-to-the-parameter.md)
-   [合併指標和方向屬性](combining-pointer-and-directional-attributes.md)
-   [MCCP 緩衝區保護](mccp-buffer-protection.md)
-   [函式的傳回值](function-return-values.md)
