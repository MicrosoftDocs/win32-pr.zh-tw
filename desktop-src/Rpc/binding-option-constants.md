## <a name="remarks"></a><span data-ttu-id="bf8d5-139">備註</span><span class="sxs-lookup"><span data-stu-id="bf8d5-139">Remarks</span></span>

<span data-ttu-id="bf8d5-140">根據預設，RPC 執行時間程式庫會以嚴格的提交順序，從應用程式的每個執行緒在給定的系結控制碼上執行呼叫。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-140">By default, the RPC run-time library executes the calls on a given binding handle from each thread of an application in strict order of submission.</span></span> <span data-ttu-id="bf8d5-141">這並不保證會序列化相同系結控制碼上不同執行緒的呼叫。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-141">This does not guarantee that calls from different threads on the same binding handle are serialized.</span></span> <span data-ttu-id="bf8d5-142">多執行緒應用程式必須序列化其 RPC 呼叫。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-142">Multithreaded applications must serialize their RPC calls.</span></span> <span data-ttu-id="bf8d5-143">如果此行為過於嚴格，您可以啟用 noncausal 順序。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-143">If this behavior is too restrictive, you can enable noncausal ordering.</span></span> <span data-ttu-id="bf8d5-144">當您這樣做時，RPC 執行時間程式庫會獨立執行呼叫。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-144">When you do, the RPC run-time library executes calls independently.</span></span> <span data-ttu-id="bf8d5-145">它不會在提交時進行排序。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-145">It imposes no ordering on their submission.</span></span>

<span data-ttu-id="bf8d5-146">可能會發現 noncausal 順序的應用程式範例之一，就是執行緒在相同系結控制碼上進行呼叫的多執行緒程式。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-146">One example of an application that might find noncausal ordering useful is a multithreaded program whose threads make calls on the same binding handle.</span></span> <span data-ttu-id="bf8d5-147">同樣地，在系結控制碼上使用多個非同步呼叫的程式，會發現 noncausal 順序是一個方便的選項。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-147">Similarly, a program that uses multiple asynchronous calls on a binding handle will find noncausal ordering a convenient option.</span></span> <span data-ttu-id="bf8d5-148">另一個範例可能是使用單一線程處理數個用戶端要求的網際網路 proxy 程式。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-148">Another example might be an Internet proxy program that uses a single thread to handle requests for several clients.</span></span> <span data-ttu-id="bf8d5-149">在上述每一種情況下，嘗試序列化遠端程序呼叫是非常嚴格的。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-149">In each of these cases, it would be extremely restrictive to try to serialize the remote procedure calls.</span></span>

<span data-ttu-id="bf8d5-150">您只能在使用 **ncalrpc** 或 \**\_ \* ncacn* _ 通訊協定序列的系結控制碼上設定「 **RPC \_ C \_ OPT 未 \_ \_ 逗留**」選項。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-150">The **RPC\_C\_OPT\_DONT\_LINGER** option can be set only on binding handles that use the **ncalrpc** or \**ncacn\_\** _ protocol sequences.</span></span> <span data-ttu-id="bf8d5-151">它不能用在 _*ncadg \_ \**_ 通訊協定序列上。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-151">It cannot be used on _*ncadg\_\**_ protocol sequences.</span></span> <span data-ttu-id="bf8d5-152">具有此選項的 [_ *RpcBindingSetOption* \*](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingsetoption)函式必須在至少進行一個 RPC 呼叫的系結控制碼上呼叫。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-152">The [_ *RpcBindingSetOption*\*](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingsetoption) function with this option must be called on a binding handle on which at least one RPC call has been made.</span></span> <span data-ttu-id="bf8d5-153">如果未在系結控制碼上進行 RPC 呼叫，則會從 **RpcBindingSetOption** 函式呼叫傳回 **rpc \_ S \_ 錯誤 \_ \_ 的 \_** 系結類型。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-153">If no RPC call have been made on the binding handle, **RPC\_S\_WRONG\_KIND\_OF\_BINDING** is returned from the **RpcBindingSetOption** function call.</span></span> <span data-ttu-id="bf8d5-154">不論附加至關聯的系結控制碼有多少，此選項都會對整個關聯生效。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-154">The option takes effect for the entire association, regardless of how many binding handles are attached to the association.</span></span> <span data-ttu-id="bf8d5-155">由於在關聯終結之前會先進行檢查，因此可以在系結控制碼關閉之前隨時進行設定。</span><span class="sxs-lookup"><span data-stu-id="bf8d5-155">Since it is checked before the association is destroyed, it can be set at any time before the binding handle is closed.</span></span>
