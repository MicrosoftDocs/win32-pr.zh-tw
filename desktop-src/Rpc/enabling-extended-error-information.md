# <a name="enabling-extended-error-information"></a>啟用延伸的錯誤資訊

若要啟用延伸錯誤資訊，請執行下列步驟：

使用 gpedit.msc 介面開啟本機電腦原則。

流覽至位於 [電腦設定]/[系統管理範本/系統/遠端程序呼叫/Rpc 疑難排解支援/傳播延伸錯誤資訊的原則

啟用原則，並將 **擴充錯誤資訊** 的欄位傳播設定為 [開啟]。

此程式會針對電腦上的所有處理常式，開啟的延伸錯誤資訊的傳播。

若只要在電腦上啟用特定處理常式的延伸錯誤資訊，或只針對電腦上的特定處理常式停用它，請使用 **Off with 例外** 狀況，或使用 **with 例外** 狀況選項。 這兩個選項都會使用「欄位 **延伸的錯誤資訊」例外** 狀況，來指定哪些處理常式具有預設設定，而哪些處理常式的預設值相反。 **擴充的錯誤資訊例外** 狀況包含進程名稱的清單。

如果處理常式的命令列是以 **延伸的錯誤資訊例外** 狀況中的字串開始，進程將會收到預設設定的相反設定。 例如，如果您想要讓電腦上的所有處理常式傳播延伸的錯誤資訊（lsass 和稱為 **Secure Server** 的程式除外），請將 **擴充錯誤資訊的傳播** 設為 **on，但有例外** 狀況，並在 [ **擴充錯誤資訊例外** ] 欄位中指定下列字串： lsass "Secure Server"。

字串可以用雙引號括住，但如果字串中沒有空格，則可以選擇性地執行此動作。 此外，也可以指定進程命令列的開頭。 例如，如果在 [**擴充錯誤資訊**] 欄位的 [傳播] 中指定 **Off** ，而且在 [**擴充錯誤資訊例外** 狀況] 欄位中指定了下列內容： win。

開頭為 "win" 的每個進程都會傳播擴充的錯誤資訊。 例如，在許多電腦上，這些處理常式會 winlogon.exe 並 WINWORD.EXE。
