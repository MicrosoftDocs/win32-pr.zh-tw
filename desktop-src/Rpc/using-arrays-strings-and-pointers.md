# <a name="using-arrays-strings-and-pointers"></a>使用陣列、字串和指標

醫生計畫 (查看 \\ \\ \\ 平臺軟體發展工具組中的 RPC 醫生 (SDK) ) 是示範陣列和字串屬性相關設計取捨的應用程式範例。 醫生計畫是由1960年代中 MIT 的人工智慧研究人員 Joseph Weizenbaum 所開發之較知名程式（Eliza）的小型範例版本。 程式會使用簡單的模式比對來模擬 therapist，以回應自然語言輸入。

Eliza 程式已記錄下列會話。



| User                                                | 伊麗莎                                                     |
|-----------------------------------------------------|-----------------------------------------------------------|
| 男性都很相似。                                  | 以何種方式？                                              |
| 他們一律會困擾我們的內容。 | 您可以考慮特定範例嗎？                      |
| 好了，我的男友變成了。               | 您的男友會在這裡提出。                        |
| 他說我的時間很長。             | 很抱歉聽到您已按下。                |
| 沒錯，我不太高興。                            | 您認為這裡有助您不太滿意嗎？ |
| 我需要一些協助;這看起來很有特定的。          | 如果您獲得一些協助，這對您有何意義？           |
| 或許我可以瞭解如何與我的母親一起進行。  | 告訴我更多關於您家人的資訊。                           |
| 我的母親負責處理。                         | 您家人中的其他人會負責您？                |
| 我父親。                                          | 你父親。                                              |
| 你喜歡我的父親的部分方式。                | 您看到什麼非常相似？                              |



 

醫生計畫可以分割成用戶端和伺服器端應用程式。 用戶端會提示患者輸入，並顯示醫生的回應。 伺服器端會處理患者輸入，並產生醫生的回應。 這是典型的用戶端-伺服器應用程式範例：用戶端會負責使用者互動，而伺服器會處理大量的計算負載。 由於資料可能需要大量的處理，因此它不會將資料傳遞給函式，也不會傳回資料，而是由伺服器處理。

醫生計畫使用字元陣列進行輸入，並傳回另一個字元陣列作為輸出。 下表列出在用戶端和伺服器之間傳遞字元陣列的四種方式，以及執行每個方法所需的屬性和函式。



| 方法                       | 屬性或函數                                                                                                        |
|--------------------------------|--------------------------------------------------------------------------------------------------------------------------------|
| 計算的字元陣列       | \[[大小 \_ 為](/windows/desktop/Midl/size-is) \] ， \[ [長度 \_ 為](/windows/desktop/Midl/length-is) \] ， \[ [參考](/windows/desktop/Midl/ref)\]                                         |
| Stub 管理的字串           | \[[](/windows/desktop/Midl/string) \] \[ [](/windows/desktop/Midl/ref) \] 伺服器上的字串、ref、 [midl \_ 使用者 \_ 配置](/windows/desktop/Midl/midl-user-allocate-1)                  |
| Stub 管理的字串           | \[[](/windows/desktop/Midl/string) \] \[ [](/windows/desktop/Midl/unique) \] 用戶端和伺服器上的字串、唯一、 [midl \_ 使用者 \_ 配置](/windows/desktop/Midl/midl-user-allocate-1) |
| 傳回字串的函式 | \[[唯一](/windows/desktop/Midl/unique)\]                                                                                                     |



 

在與這些屬性組合相關聯的條件約束中，有其他方式可將一個字元陣列從用戶端傳送到伺服器，並將另一個字元陣列從伺服器傳回至用戶端。
