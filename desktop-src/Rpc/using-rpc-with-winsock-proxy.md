# <a name="using-rpc-with-winsock-proxy"></a><span data-ttu-id="bf6d9-103">使用 RPC 搭配 Winsock Proxy</span><span class="sxs-lookup"><span data-stu-id="bf6d9-103">Using RPC with Winsock Proxy</span></span>

<span data-ttu-id="bf6d9-104">Microsoft Internet Access Server 的發行版本包含 Winsock Proxy，這是 Windows 通訊端 API 1.1 版的增強版。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-104">The release of Microsoft Internet Access Server included Winsock Proxy, an enhanced version of Windows Sockets API version 1.1.</span></span> <span data-ttu-id="bf6d9-105">Winsock Proxy 可讓在私人網路用戶端上執行的 Windows 通訊端應用程式，其行為就像是直接連接到遠端網際網路伺服器應用程式一樣。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-105">Winsock Proxy lets a Windows Sockets application, running on a private network client, behave as if it were directly connected to a remote Internet server application.</span></span> <span data-ttu-id="bf6d9-106">Microsoft Proxy 伺服器作為此連線的主機。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-106">The Microsoft Proxy Server acts as the host for this connection.</span></span> <span data-ttu-id="bf6d9-107">這表示所有應用層級的通訊都是透過單一受保護的電腦（執行 Microsoft Proxy 伺服器的閘道電腦）來傳遞。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-107">This means that all application-level communications are channeled through a single secured computer—the gateway computer running Microsoft Proxy Server.</span></span>

<span data-ttu-id="bf6d9-108">一般來說，對於資料包封包傳輸，RPC 傳輸 DLL 會略過 Wsock32.dll 中提供的 [**sendto**](/windows/desktop/api/winsock/nf-winsock-sendto) 和 [**recvfrom**](/windows/desktop/api/winsock/nf-winsock-recvfrom) 函式，並直接與基礎設備磁碟機通訊。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-108">Ordinarily, for datagram-packet transfers, the RPC transport DLL bypasses the [**sendto**](/windows/desktop/api/winsock/nf-winsock-sendto) and [**recvfrom**](/windows/desktop/api/winsock/nf-winsock-recvfrom) functions provided in Wsock32.dll, and communicates directly with the underlying device driver.</span></span> <span data-ttu-id="bf6d9-109">這可改善封包傳送的速度，但無法讓應用程式使用 Winsock Proxy 功能。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-109">This improves the speed of packet transfers but makes Winsock Proxy features unavailable to the application.</span></span>

<span data-ttu-id="bf6d9-110">每個網路通訊協定提供者都有相關聯的 GUID。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-110">Each network protocol provider to have an associated GUID.</span></span> <span data-ttu-id="bf6d9-111">RPC 執行時間程式庫會將 UDP 和 IPX Guid 與知名的 Microsoft 識別碼進行比較。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-111">The RPC run-time library compares the UDP and IPX GUIDs to the well known Microsoft identifiers.</span></span> <span data-ttu-id="bf6d9-112">如果不相符，RPC 會自動使用 Winsock。</span><span class="sxs-lookup"><span data-stu-id="bf6d9-112">If they don't match, RPC automatically uses Winsock.</span></span>
