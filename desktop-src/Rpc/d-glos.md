# <a name="d-rpc"></a>D (RPC) 

[](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [E](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [](u-glos.md) [](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**傳送**
</dt> <dd>

通訊協定或傳輸，其中的資料封包會獨立路由傳送。 它們可能會遵循不同的路由，並以與其傳送的不同順序抵達。 [*UDP*](u-glos.md) 和 [*IPX*](i-glos.md) 是傳輸層-資料包通訊協定的範例。 另請參閱 [*連線導向*](c-glos.md)。

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**判別**
</dt> <dd>

變數，指定可以儲存在聯集內的資料類型。

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**區分聯集**
</dt> <dd>

 (或 *變異記錄*) 聯集，其中包含做為資料結構一部分的鑒別子，如此一來，目前有效的資料類型會與聯集一起傳送。 另請參閱 [*封裝聯集*](e-glos.md)、nonencapsulated 聯 [*集*](n-glos.md)。

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**(DCE) 的分散式運算環境**
</dt> <dd>

為一組整合式服務（包括遠端程序呼叫、分散式檔案系統和安全性服務）提供開放式 Software Foundation 的規格。 憑證-DCE RPC standard 是 Microsoft RPC 的基礎。

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**動態端點**
</dt> <dd>

網路專用–在執行時間要求並指派的伺服器位址。 另請參閱 [*知名的端點*](w-glos.md)。

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**動態身分識別追蹤**
</dt> <dd>

規格：每當用戶端呼叫遠端程式時，RPC 執行時間程式庫會使用呼叫執行緒的認證，而不是系結控制碼，以便進行驗證。 另請參閱 [*靜態身分識別追蹤*](s-glos.md)。

</dd> </dl>
