# <a name="client-impersonation"></a><span data-ttu-id="b3914-103">用戶端模擬</span><span class="sxs-lookup"><span data-stu-id="b3914-103">Client Impersonation</span></span>

<span data-ttu-id="b3914-104">當伺服器必須將用戶端要求傳遞給其他伺服器進程或作業系統時，模擬在分散式運算環境中很有用。</span><span class="sxs-lookup"><span data-stu-id="b3914-104">Impersonation is useful in a distributed computing environment when servers must pass client requests to other server processes or to the operating system.</span></span> <span data-ttu-id="b3914-105">在此情況下，伺服器會模擬用戶端的安全性內容。</span><span class="sxs-lookup"><span data-stu-id="b3914-105">In this case, a server impersonates the client's security context.</span></span> <span data-ttu-id="b3914-106">然後，其他伺服器進程可以處理要求，就像原始用戶端所做的一樣。</span><span class="sxs-lookup"><span data-stu-id="b3914-106">Other server processes can then handle the request as if the original client made it.</span></span>

![當代表用戶端進行後續呼叫時，伺服器會模擬呼叫用戶端](images/impr.png)

<span data-ttu-id="b3914-108">例如，用戶端會向伺服器 A 提出要求。如果伺服器 A 必須查詢伺服器 B 才能完成要求，伺服器 A 會模擬用戶端安全性內容，並代表用戶端提出對伺服器 B 的要求。</span><span class="sxs-lookup"><span data-stu-id="b3914-108">For example, a client makes a request to Server A. If Server A must query Server B to complete the request, Server A impersonates the client security context and makes the request to Server B on behalf of the client.</span></span> <span data-ttu-id="b3914-109">伺服器 B 會使用原始用戶端的安全性內容，而不是伺服器 A 的安全性識別，以判斷是否要完成工作。</span><span class="sxs-lookup"><span data-stu-id="b3914-109">Server B uses the original client's security context, instead of the security identity for Server A, to determine whether to complete the task.</span></span>

<span data-ttu-id="b3914-110">伺服器會呼叫 [**RpcImpersonateClient**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcimpersonateclient) ，以使用用戶端安全性內容來覆寫伺服器執行緒的安全性。</span><span class="sxs-lookup"><span data-stu-id="b3914-110">The server calls [**RpcImpersonateClient**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcimpersonateclient) to overwrite the security for the server thread with the client security context.</span></span> <span data-ttu-id="b3914-111">工作完成之後，伺服器會呼叫 [**RpcRevertToSelf**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcreverttoself) 或 [**RpcRevertToSelfEx**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcreverttoselfex) 來還原為伺服器執行緒定義的安全性內容。</span><span class="sxs-lookup"><span data-stu-id="b3914-111">After the task is completed, the server calls [**RpcRevertToSelf**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcreverttoself) or [**RpcRevertToSelfEx**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcreverttoselfex) to restore the security context defined for the server thread.</span></span>

<span data-ttu-id="b3914-112">當系結時，用戶端可以指定安全性的服務品質資訊，以指定伺服器如何模擬用戶端。</span><span class="sxs-lookup"><span data-stu-id="b3914-112">When binding, the client can specify quality-of-service information about security which specifies how the server can impersonate the client.</span></span> <span data-ttu-id="b3914-113">例如，其中一個設定可讓用戶端指定不允許伺服器模擬該伺服器。</span><span class="sxs-lookup"><span data-stu-id="b3914-113">For example, one of the settings lets the client specify that the server is not allowed to impersonate it.</span></span> <span data-ttu-id="b3914-114">如需詳細資訊，請參閱 [服務品質](quality-of-service.md)。</span><span class="sxs-lookup"><span data-stu-id="b3914-114">For more information, see [Quality of Service](quality-of-service.md).</span></span>

<span data-ttu-id="b3914-115">在模擬原始用戶端時呼叫其他伺服器的功能稱為「 *委派*」。</span><span class="sxs-lookup"><span data-stu-id="b3914-115">The capability to call other servers while impersonating the original client is called *delegation*.</span></span> <span data-ttu-id="b3914-116">使用委派時，模擬用戶端的伺服器可以呼叫另一部伺服器，而且可以使用用戶端的認證進行網路呼叫。</span><span class="sxs-lookup"><span data-stu-id="b3914-116">When delegation is used, a server impersonating a client can call another server, and can make network calls with the credentials of the client.</span></span> <span data-ttu-id="b3914-117">也就是說，從第二部伺服器的角度來看，來自第一部伺服器的要求，與來自用戶端的要求並不區分。</span><span class="sxs-lookup"><span data-stu-id="b3914-117">That is, from the perspective of the second server, requests coming from the first server are indistinguishable from requests coming from the client.</span></span> <span data-ttu-id="b3914-118">並非所有安全性提供者都支援委派。</span><span class="sxs-lookup"><span data-stu-id="b3914-118">Not all security providers support delegation.</span></span> <span data-ttu-id="b3914-119">Microsoft 只提供一個支援委派的安全性提供者： Kerberos。</span><span class="sxs-lookup"><span data-stu-id="b3914-119">Microsoft ships only one security provider that supports delegation: Kerberos.</span></span>

<span data-ttu-id="b3914-120">委派可能是危險的功能，因為用戶端會在遠端程序呼叫期間提供伺服器的許可權。</span><span class="sxs-lookup"><span data-stu-id="b3914-120">Delegation can be a dangerous capability due to the privileges the client gives the server during a remote procedure call.</span></span> <span data-ttu-id="b3914-121">這就是為什麼 Kerberos 只在要求相互驗證時才允許使用模擬層級的委派呼叫。</span><span class="sxs-lookup"><span data-stu-id="b3914-121">This is why Kerberos allows calls with the impersonation level of delegation only if mutual authentication is requested.</span></span> <span data-ttu-id="b3914-122">網域系統管理員可以限制對委派模擬等級發出呼叫的電腦，以防止不滿意的用戶端呼叫可能會濫用認證的伺服器。</span><span class="sxs-lookup"><span data-stu-id="b3914-122">Domain administrators can limit the computers to which calls with delegation impersonation level are made to prevent unsuspecting clients from making calls to servers that could abuse their credentials.</span></span>

<span data-ttu-id="b3914-123">委派規則有一個例外狀況：使用 [**ncalrpc**](/windows/desktop/Midl/ncalrpc)呼叫。</span><span class="sxs-lookup"><span data-stu-id="b3914-123">One exception to the delegation rule exists: calls using [**ncalrpc**](/windows/desktop/Midl/ncalrpc).</span></span> <span data-ttu-id="b3914-124">當這些呼叫進行時，伺服器會取得委派許可權，即使已指定模擬的模擬等級也是如此。</span><span class="sxs-lookup"><span data-stu-id="b3914-124">When these calls are made the server gets delegation rights, even if an impersonation level of impersonate is specified.</span></span> <span data-ttu-id="b3914-125">也就是說，伺服器可以代表用戶端呼叫其他伺服器。</span><span class="sxs-lookup"><span data-stu-id="b3914-125">That is, a server can call other servers on behalf of the client.</span></span> <span data-ttu-id="b3914-126">這僅適用于一個遠端呼叫。</span><span class="sxs-lookup"><span data-stu-id="b3914-126">This works for one remote call only.</span></span> <span data-ttu-id="b3914-127">例如，如果用戶端使用 **ncalrpc** 本機伺服器 lb 來呼叫本機伺服器 lb，就可以模擬和呼叫 REMOTE server RB。</span><span class="sxs-lookup"><span data-stu-id="b3914-127">For example, if client A calls local server LB using **ncalrpc** local server LB can impersonate and call remote server RB.</span></span> <span data-ttu-id="b3914-128">RB 將能夠代表用戶端 A 執行動作，但只能在執行 RB 的遠端電腦上執行。</span><span class="sxs-lookup"><span data-stu-id="b3914-128">RB will be able to act on behalf of client A, but only on the remote computer that RB is running on.</span></span> <span data-ttu-id="b3914-129">它無法對遠端電腦 C 進行其他網路呼叫，除非 LB 在呼叫 RB 時指定了委派的模擬層級。</span><span class="sxs-lookup"><span data-stu-id="b3914-129">It cannot make another network call to remote computer C unless LB specified an impersonation level of delegate when it called RB.</span></span>
