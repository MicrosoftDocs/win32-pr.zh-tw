# <a name="implementing-output-pipes-on-the-client"></a><span data-ttu-id="0b9b6-103">在用戶端上執行輸出管道</span><span class="sxs-lookup"><span data-stu-id="0b9b6-103">Implementing Output Pipes on the Client</span></span>

<span data-ttu-id="0b9b6-104">使用輸出管道將資料從伺服器傳送至用戶端時，您必須在用戶端中執行推播程式。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-104">When using an output pipe to transfer data from the server to the client, you must implement a push procedure in your client.</span></span> <span data-ttu-id="0b9b6-105">推送程式會從用戶端 stub 取得緩衝區和元素計數的指標，而如果專案計數大於0，則會處理資料。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-105">The push procedure takes a pointer to a buffer and an element count from the client stub and, if the element count is greater than 0, processes the data.</span></span> <span data-ttu-id="0b9b6-106">例如，它可以將資料從存根的緩衝區複製到它自己的記憶體。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-106">For example, it could copy the data from the stub's buffer to its own memory.</span></span> <span data-ttu-id="0b9b6-107">或者，它可以處理存根緩衝區中的資料，並將它儲存到檔案中。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-107">Alternately, it could process the data in the stub's buffer and save it to a file.</span></span> <span data-ttu-id="0b9b6-108">當元素計數等於零時，推送程式會先完成任何所需的清除工作，然後再傳回。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-108">When the element count equals zero, the push procedure completes any needed cleanup tasks before returning.</span></span>

<span data-ttu-id="0b9b6-109">在下列範例中，用戶端函數 ReceiveLongs 會配置管道結構和全域記憶體緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-109">In the following example, the client function ReceiveLongs allocates a pipe structure and a global memory buffer.</span></span> <span data-ttu-id="0b9b6-110">它會初始化結構、進行遠端程序呼叫，然後釋出記憶體。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-110">It initializes the structure, makes the remote procedure call, and then frees the memory.</span></span>

## <a name="example"></a><span data-ttu-id="0b9b6-111">範例</span><span class="sxs-lookup"><span data-stu-id="0b9b6-111">Example</span></span>


```C++
//file: client.c (fragment)
#include <windows.h>
#include "pipedemo.h"
long *  globalPipeData;
long    globalBuffer[BUF_SIZE];
 
ulong   pipeDataIndex; /* state variable */
 
void ReceiveLongs()
{
    LONG_PIPE *outputPipe;
    idl_long_int i;
 
    globalPipeData =
        (long *)malloc( sizeof(long) * PIPE_SIZE );
    
    pipeDataIndex = 0;
    outputPipe.state =  (rpc_ss_pipe_state_t )&pipeDataIndex;
    outputPipe.push  = PipePush;
    outputPipe.alloc = PipeAlloc;
 
    OutPipe( &outputPipe ); /* Make the rpc */
 
    free( (void *)globalPipeData );
 
}//end ReceiveLongs()

void PipeAlloc( rpc_ss_pipe_state_t stateInfo,
                ulong requestedSize,
                long **allocatedBuffer,
                ulong *allocatedSize )
{ 
    ulong *state = (ulong *)stateInfo;
    if ( requestedSize > (BUF_SIZE*sizeof(long)) )
    {
       *allocatedSize = BUF_SIZE * sizeof(long);
    }
    else
    {
       *allocatedSize = requestedSize;
    }
    *allocatedBuffer = globalBuffer; 
} //end PipeAlloc
 
void PipePush( rpc_ss_pipe_state_t stateInfo,
               long *buffer,
               ulong numberOfElements )
{
    ulong elementsToCopy, i;
    ulong *state = (ulong *)stateInfo;

    if (numberOfElements == 0)/* end of data */
    {
        *state = 0; /* Reset the state = global index */
    }
    else
    {
        if (*state + numberOfElements > PIPE_SIZE)
            elementsToCopy = PIPE_SIZE - *state;
        else
            elementsToCopy = numberOfElements;
 
        for (i=0; i <elementsToCopy; i++)
        { 
            /*client receives data */
            globalPipeData[*state] = buffer[i];
            (*state)++;
        }
    }
}//end PipePush
```



<span data-ttu-id="0b9b6-112">此範例包含 MIDL 編譯器所產生的標頭檔。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-112">This example includes the header file generated by the MIDL compiler.</span></span> <span data-ttu-id="0b9b6-113">如需詳細資訊，請參閱 [在 IDL 檔案中定義管道](defining-pipes-in-idl-files.md)。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-113">For details see [Defining Pipes in IDL File](defining-pipes-in-idl-files.md).</span></span> <span data-ttu-id="0b9b6-114">它也會宣告變數 globalPipeData，它會用來做為資料接收。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-114">It also declares a variable, globalPipeData, that it uses as the data sink.</span></span> <span data-ttu-id="0b9b6-115">變數 globalBuffer 是推播程式用來接收其儲存在 globalPipeData 中之資料區塊的緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-115">The variable globalBuffer is a buffer that the push procedure uses to receive blocks of data it stores in globalPipeData.</span></span>

<span data-ttu-id="0b9b6-116">ReceiveLongs 函式會宣告管道並配置全域資料接收變數的記憶體空間。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-116">The ReceiveLongs function declares a pipe and allocates memory space for the global data sink variable.</span></span> <span data-ttu-id="0b9b6-117">在您的用戶端/伺服器程式中，資料接收器可以是用戶端所建立的檔案或資料結構。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-117">In your client/server program, the data sink can be a file or data structure the client creates.</span></span> <span data-ttu-id="0b9b6-118">在這個簡單的範例中，資料來源是一種動態配置的長整數緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-118">In this simple example, the data source is a dynamically allocated buffer of long integers.</span></span>

<span data-ttu-id="0b9b6-119">在開始進行資料傳輸之前，您的用戶端程式必須初始化輸出管道結構。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-119">Before the data transfer can begin, your client program must initialize the output pipe structure.</span></span> <span data-ttu-id="0b9b6-120">它必須設定狀態變數、推送程式和配置程式的指標。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-120">It must set pointers to the state variable, the push procedure, and the alloc procedure.</span></span> <span data-ttu-id="0b9b6-121">在此範例中，輸出管道變數稱為 outputPipe。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-121">In this example, the output pipe variable is called outputPipe.</span></span>

<span data-ttu-id="0b9b6-122">用戶端會透過叫用伺服器上的遠端程式，來通知伺服器他們已準備好接收資料。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-122">Clients signal servers that they are ready to receive data by invoking a remote procedure on the server.</span></span> <span data-ttu-id="0b9b6-123">在此範例中，遠端程式稱為 OutPipe。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-123">In this example, the remote procedure is called OutPipe.</span></span> <span data-ttu-id="0b9b6-124">當用戶端呼叫遠端程式時，伺服器就會開始進行資料傳輸。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-124">When the client calls the remote procedure, the server begins the data transfer.</span></span> <span data-ttu-id="0b9b6-125">每次資料抵達時，用戶端 stub 都會視需要呼叫用戶端的配置和推送程式。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-125">Each time data arrives, the client stub calls the client's alloc and push procedures as needed.</span></span>

<span data-ttu-id="0b9b6-126">此範例中的配置程式只會將指標設定為變數 globalBuffer，而不是在每次需要緩衝區時配置記憶體。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-126">Rather than allocate memory each time a buffer is needed, the alloc procedure in this example simply sets a pointer to the variable globalBuffer.</span></span> <span data-ttu-id="0b9b6-127">提取程式接著會在每次傳送資料時重複使用此緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-127">The pull procedure then reuses this buffer each time it transfers data.</span></span> <span data-ttu-id="0b9b6-128">每次伺服器從用戶端提取資料時，更複雜的用戶端程式可能需要配置新的緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-128">More complex client programs may need to allocate a new buffer each time the server pulls data from the client.</span></span>

<span data-ttu-id="0b9b6-129">此範例中的推送程式會使用狀態變數來追蹤將資料儲存在全域資料接收緩衝區中的下一個位置。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-129">The push procedure in this example uses the state variable to track the next position where it will store data in the global data sink buffer.</span></span> <span data-ttu-id="0b9b6-130">它會將管線緩衝區的資料寫入接收緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-130">It writes data from the pipe buffer into sink buffer.</span></span> <span data-ttu-id="0b9b6-131">然後，用戶端 stub 會從伺服器接收下一個資料區塊，並將它儲存在管道緩衝區中。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-131">The client stub then receives the next block of data from the server and stores it in the pipe buffer.</span></span> <span data-ttu-id="0b9b6-132">傳送所有資料之後，伺服器會傳輸大小為零的緩衝區。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-132">When all the data has been sent, the server transmits a zero-sized buffer.</span></span> <span data-ttu-id="0b9b6-133">這會提示推送程式以停止接收資料。</span><span class="sxs-lookup"><span data-stu-id="0b9b6-133">This cues the push procedure to stop receiving data.</span></span>
