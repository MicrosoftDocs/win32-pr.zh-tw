# <a name="s-rpc"></a>S (RPC) 

[](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [E](e-glos.md) [F](f-glos.md) G [](u-glos.md) [](v-glos.md) H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) S [T](t-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**可 伸縮 性**
</dt> <dd>

能夠在多處理器系統上充分利用額外的處理能力， (2、4、8、32或更多處理器) 。 擴充性也是服務大量用戶端的能力。

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**排序的封包通訊協定 (SPP)**
</dt> <dd>

Banyan Vines 以 [*連接導向*](c-glos.md) 的通訊協定，用於透過區域網路的路由資訊封包。

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**已排序的封包交換 (SPX)**
</dt> <dd>

Novell NetWare [*連接導向*](c-glos.md) 通訊協定，可透過區域網路和廣域網路來路由傳送資訊封包。

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**序列 化**
</dt> <dd>

將資料封送處理至 (編碼的程式) 和將資料從 (解碼) 緩衝區進行封送處理。 這與傳統的 RPC 使用方式相反，因為存根和 RPC 執行時間會控制封送處理緩衝區。 也稱為 pickling。 另請參閱程式 [*序列化*](p-glos.md)， [*類型序列化*](t-glos.md)。

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**伺服器 stub**
</dt> <dd>

MIDL 產生的 C 語言原始程式碼，其中包含伺服器應用程式使用本機程序呼叫處理遠端要求所需的所有功能。 另請參閱 [*用戶端存根*](c-glos.md)。

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**會話**
</dt> <dd>

建立用戶端應用程式與伺服器應用程式之間的關聯性。 另請參閱 [*bind*](b-glos.md)、系結 [*控制碼*](b-glos.md)。

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**靜態回呼函數**
</dt> <dd>

屬於分散式應用程式用戶端一部分的遠端程式，伺服器可呼叫以取得用戶端的資訊。 \[[回呼](/windows/desktop/Midl/callback) \] 屬性會指定靜態回呼函數。

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**靜態身分識別追蹤**
</dt> <dd>

靜態身分識別追蹤指定 RPC 執行時間在用戶端的系結控制碼中，針對所有 RPC 呼叫使用安全性認證。 另請參閱 [*動態身分識別追蹤*](d-glos.md)。

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**字串系結**
</dt> <dd>

包含物件 [*UUID*](u-glos.md)、 [*通訊協定順序*](p-glos.md)、 [*網路位址*](n-glos.md)、 [*端點*](e-glos.md)和端點選項的字元字串，這些選項全都可用來建立指定之伺服器的系結 [*控制碼*](b-glos.md) 。

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**強型別**
</dt> <dd>

嚴格控制資料類型的編譯器強制執行。 在 MIDL 和 RPC 中，強型別是用來確保分散式環境中的不同電腦都能一致地解讀資料。

</dd> </dl>
