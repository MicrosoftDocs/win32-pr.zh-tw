# <a name="c-and-ole-programming-concepts"></a>C + + 和 OLE 程式設計概念

Windows 隨附的檔案和資料流程處理常式使用物件導向設計來提升標準介面並共用功能。 這些處理常式是以 c + + 撰寫，並使用 OLE 元件物件模型。

您可以使用 C 或 c + + 開發系統開發自訂處理常式;不過，強烈建議使用 c + +，因為它提供更簡單且更簡單的方法來執行處理常式。 使用 c + +，您可以將資料明確定義為物件，也可以將運算元據的函式與物件的成員函式產生關聯。

本節將識別並簡短摘要說明 c + + 的重要概念，以及適用于設計和執行檔案和串流處理常式的 OLE 元件物件模型。 有許多關於 c + + 程式設計的書籍，您可以參考這些書籍以取得詳細資訊。 如需 OLE 的詳細資訊，請參閱 *ole 程式設計人員參考*。
