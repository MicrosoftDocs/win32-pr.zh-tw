# <a name="multimedia"></a>多媒體

本詞彙包含 Windows 多媒體檔案中所使用之詞彙的定義。

<dl> <dt>

音訊壓縮技術。

</dd> <dt>

<span id="_win32_adpcm_gloss"></span><span id="_WIN32_ADPCM_GLOSS"></span>**ADPCM**
</dt> <dd>

請參閱彈性 [差異脈衝程式碼 (ADPCM) ](/windows)。

</dd> <dt>

<span id="_win32_break_key_gloss"></span><span id="_WIN32_BREAK_KEY_GLOSS"></span>**break 鍵**
</dt> <dd>

在 Media Control 介面 (MCI) ，這是中斷等待作業的按鍵。 根據預設，MCI 會將此機碼定義為 CTRL + BREAK。 應用程式可以使用 MCI BREAK 命令訊息來重新定義此金鑰 \_ 。

</dd> <dt>

<span id="_win32_cd_rom_extended_architecture_cd_xa_gloss"></span><span id="_WIN32_CD_ROM_EXTENDED_ARCHITECTURE_CD_XA_GLOSS"></span>**Cd-rom 擴充架構 (cd-XA)**
</dt> <dd>

CD-ROM 標準的延伸模組，提供壓縮音訊資料的儲存以及光碟上的其他資料。此標準也定義從光碟讀取資料的方式。音訊信號會在單一軌上結合文字和圖形資料，因此幾乎可以同時讀取。

</dd> <dt>

<span id="_win32_channel_gloss"></span><span id="_WIN32_CHANNEL_GLOSS"></span>**頻道**
</dt> <dd>

由樂器數位介面所提供的方法 (MIDI) ，可將訊息傳送至 MIDI 設定內的個別裝置。 有16個 MIDI 頻道號碼。 您可以將 MIDI 設定中的裝置導向至僅回應以裝置特定通道號碼標記的訊息。

</dd> <dt>

<span id="_win32_channel_map_gloss"></span><span id="_WIN32_CHANNEL_MAP_GLOSS"></span>**通道對應**
</dt> <dd>

由 MIDI 對應程式提供的通道對應，可將音樂檢測數位介面 (MIDI) 訊息從一個通道重新導向至另一個通道。 另請參閱 [midi](/windows)對應工具、 [音樂檢測數位介面 (midi) ](/windows)。

</dd> <dt>

<span id="_win32_chunk_gloss"></span><span id="_WIN32_CHUNK_GLOSS"></span>**區塊**
</dt> <dd>

資源交換檔案格式的基本組建區塊 (RIFF) 檔案，其中包含識別碼 (稱為區塊識別碼) 、區塊大小變數，以及變數大小的區塊資料區域。

</dd> <dt>

<span id="_win32_command_message_gloss"></span><span id="_WIN32_COMMAND_MESSAGE_GLOSS"></span>**命令訊息**
</dt> <dd>

在 Media Control 介面 (MCI) 中，代表 MCI 裝置唯一命令的符號常數。 命令訊息具有相關聯的資料結構，可提供裝置執行要求所需的資訊。

</dd> <dt>

<span id="_win32_command_string_gloss"></span><span id="_WIN32_COMMAND_STRING_GLOSS"></span>**命令字串**
</dt> <dd>

在 Media Control 介面 (MCI) ，這是以 null 結束的字元字串，代表 MCI 裝置的命令。 文字字串包含 MCI 裝置執行要求所需的所有資訊。 MCI 會剖析文字字串，並將它轉譯為對等的命令訊息和資料結構，然後將它傳送至 MCI 設備磁碟機。

</dd> <dt>

<span id="_win32_compact_disc_digital_audio_cd_da_gloss"></span><span id="_WIN32_COMPACT_DISC_DIGITAL_AUDIO_CD_DA_GLOSS"></span>光碟 **–數位音訊 (CD-DA)**
</dt> <dd>

</dd> <dt>

</dd> <dt>

</dd> <dt>
