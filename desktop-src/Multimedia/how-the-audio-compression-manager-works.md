# <a name="how-the-audio-compression-manager-works"></a>音訊壓縮管理員的運作方式

使用現有的驅動程式介面勾點來覆寫波形音訊裝置的預設對應演算法。 這可讓進行中的要求攔截裝置開啟的呼叫。 在攔截呼叫之後，執行程式可執行各種不同的工作來處理音訊資料，例如將外部壓縮程式或解壓縮程式插入序列中。

這類的驅動程式會管理下列類型的驅動程式：

-    (編解碼器) 驅動程式的壓縮程式和解壓縮程式
-   格式化轉換器驅動程式
-   篩選器驅動程式

壓縮機和 decompressors 會將一種格式類型變更為另一種格式。 例如，「壓縮程式」或「解壓縮程式」可以將 PCM (脈衝程式碼調製) 檔變更為 ADPCM (彈性差異脈衝程式碼調製) 檔。 格式轉換器會變更格式，但不會變更資料類型。 例如，轉換器可以將 44-kHz、16位資料變更為 44-kHz、8位的資料。 篩選器完全不會變更資料格式，但會以某種方式變更波形音訊資料。 例如，篩選可以結合資料流程和本身的 echo。 在驅動程式中，單一的 [的] 或 [篩選標記] 或 [格式] 標記，也可能支援上述類型的組合。

針對波形音訊輸出，當器抵達時，會將資料的每個緩衝區傳遞給轉換器。 轉換器會將資料解壓縮，並將解壓縮的資料傳回至「影子」緩衝區中的 [資料]。 然後，會將解壓縮的陰影緩衝區傳遞給波形音訊驅動程式。 當配置每次收到準備訊息時，都會配置陰影緩衝區。

若為波形音訊輸入，則會將空的陰影緩衝區傳遞給驅動程式。 它會使用背景工作在驅動程式填滿陰影緩衝區之後收到通知。 然後，將緩衝區傳遞至驅動程式進行壓縮。 壓縮完成之後，驅動程式會將資料傳遞至應用程式。
