# <a name="text-recognizers"></a><span data-ttu-id="f50a5-103">文字辨識器</span><span class="sxs-lookup"><span data-stu-id="f50a5-103">Text Recognizers</span></span>

<span data-ttu-id="f50a5-104">文字辨識器會將筆墨樣本分割成區段，並將筆墨區段轉譯成文字。</span><span class="sxs-lookup"><span data-stu-id="f50a5-104">Text recognizers divide an ink sample into segments and translate the ink segments into text.</span></span> <span data-ttu-id="f50a5-105">這在辨識手寫方面很有用。</span><span class="sxs-lookup"><span data-stu-id="f50a5-105">This is useful for recognizing handwriting.</span></span> <span data-ttu-id="f50a5-106">它也適用于複雜的撰寫案例，例如辨識個別的日文中文字元，並讓使用者在寫入時自動完成字元。</span><span class="sxs-lookup"><span data-stu-id="f50a5-106">It is also useful in complex writing scenarios, such as recognizing individual Kanji glyphs and offering users automatic completion of a character while they are writing it.</span></span>

<span data-ttu-id="f50a5-107">文字辨識器會針對每個筆墨辨識區段傳回 Unicode 字串。</span><span class="sxs-lookup"><span data-stu-id="f50a5-107">A text recognizer returns a Unicode string for each ink recognition segment.</span></span> <span data-ttu-id="f50a5-108">伴隨字串是辨識器指派給結果的信賴等級，以及將結果對應回原始筆跡的信賴等級。</span><span class="sxs-lookup"><span data-stu-id="f50a5-108">Accompanying the string is the confidence level that the recognizer assigns to the result and a mapping of the result back to the original ink.</span></span> <span data-ttu-id="f50a5-109">此對應會顯示原始筆跡中的哪一個區段對應至 Unicode 程式碼字串。</span><span class="sxs-lookup"><span data-stu-id="f50a5-109">This mapping shows which segment in the original ink corresponds to the Unicode code string.</span></span> <span data-ttu-id="f50a5-110">代表文字辨識器輸出的字串一律會以線性順序表示，而不是以空間順序或依時間順序輸入區段。</span><span class="sxs-lookup"><span data-stu-id="f50a5-110">The string that represents the output of the text recognizer is always represented in the linear order, rather than in the spatial order or chronological order in which the segments were input.</span></span>
