# <a name="hrecocontext-handle"></a><span data-ttu-id="13520-103">HRECOCONTEXT 控制碼</span><span class="sxs-lookup"><span data-stu-id="13520-103">HRECOCONTEXT Handle</span></span>

<span data-ttu-id="13520-104">**HRECOCONTEXT** 控制碼是用來將筆墨新增至內容、以同步或非同步方式執行筆墨辨識 () 、取出辨識結果，以及抓取替代項。</span><span class="sxs-lookup"><span data-stu-id="13520-104">An **HRECOCONTEXT** handle is used to add ink to the context, perform ink recognition (synchronously or asynchronously), retrieve the recognition result, and retrieve alternates.</span></span>

<span data-ttu-id="13520-105">具有辨識器內容控制碼的主要原因是要區分筆跡輸入。</span><span class="sxs-lookup"><span data-stu-id="13520-105">The primary reason for having recognizer context handles is to differentiate between ink inputs.</span></span> <span data-ttu-id="13520-106">例如，您可以建立具有兩個視窗的應用程式，使用者可以在任一視窗中進行筆跡。</span><span class="sxs-lookup"><span data-stu-id="13520-106">For example, you can create an application with two windows, the user being able to ink in either window.</span></span> <span data-ttu-id="13520-107">當您要求辨識器辨識其中一個視窗的筆墨時，不希望第一個視窗的筆墨與第二個視窗的筆墨混合。</span><span class="sxs-lookup"><span data-stu-id="13520-107">You do not want to have the ink of the first window mixed up with the ink of the second window when you ask the recognizer to recognize the ink of one of the windows.</span></span> <span data-ttu-id="13520-108">在這種類型的應用程式中，您會建立兩個辨識器內容 (每個視窗) 一個，並將進入視窗1的筆觸加入至辨識器內容1，並將視窗2中的筆觸新增至辨識器內容2。</span><span class="sxs-lookup"><span data-stu-id="13520-108">In this kind of application, you create two recognizer contexts (one for each window) and you add strokes coming into window 1 into recognizer context 1 and strokes from window 2 into recognizer context 2.</span></span> <span data-ttu-id="13520-109">若要傳回辨識結果，請根據您想要的是視窗1或2的結果，呼叫辨識器內容1或辨識器內容2上的進程。</span><span class="sxs-lookup"><span data-stu-id="13520-109">To return recognition results, call process on recognizer context 1 or recognizer context 2 depending on whether you want the results for window 1 or 2.</span></span>

<span data-ttu-id="13520-110">辨識器內容控制碼可以是您想要的任何內容。</span><span class="sxs-lookup"><span data-stu-id="13520-110">The recognizer context handle can be anything you want.</span></span> <span data-ttu-id="13520-111">但一般而言，它是結構全域陣列中的索引。</span><span class="sxs-lookup"><span data-stu-id="13520-111">But typically it is an index in a global array of structures.</span></span> <span data-ttu-id="13520-112">這些結構可能包含已輸入的所有筆劃，以及辨識器針對該特定筆跡所使用的所有變數 (例如，您的內部 >lattice 結構或辨識) 的目前狀態。</span><span class="sxs-lookup"><span data-stu-id="13520-112">The structures may contain all the strokes that have been entered and all the variables that the recognizer uses for that particular piece of ink (for example, your internal lattice structure, or the current state of recognition).</span></span> <span data-ttu-id="13520-113">其中一個結構可能包含辨識器所需的所有資訊，並用於一段特定的筆跡。</span><span class="sxs-lookup"><span data-stu-id="13520-113">One structure may contain all the information the recognizer needs and uses for one particular piece of ink.</span></span>

<span data-ttu-id="13520-114">若要取得 **HRECOCONTEXT** 控制碼，請呼叫 [**CreateCoNtext**](/windows/desktop/api/recapis/nf-recapis-createcontext) 函數。</span><span class="sxs-lookup"><span data-stu-id="13520-114">To get an **HRECOCONTEXT** handle, call the [**CreateContext**](/windows/desktop/api/recapis/nf-recapis-createcontext) function.</span></span>
