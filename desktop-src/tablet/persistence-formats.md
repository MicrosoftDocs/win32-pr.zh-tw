# <a name="persistence-formats"></a>持續性格式

應用程式應該能夠產生和取用多個格式的資料。 這些通常包含專屬的二進位格式，而且也應該包含一些標準格式，例如 rtf) 或 HTML (rtf 格式。

下表列出可包含筆墨的一些格式。



| 格式                                      | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|---------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Binary<br/>                           | 應用程式應該使用筆墨序列化格式 (ISF) 將筆墨編碼成二進位格式。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| HTML<br/>                             | 強烈建議您使用 HTML 格式來表示異類內容。 應用程式應該使用嚴加防禦圖形交換格式 (GIF) 將筆墨編碼成其 HTML 檔案。 如需嚴加防禦 Gif 的詳細資訊，請參閱 [建立區塊](building-blocks.md)。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Image<br/>                            | 針對沒有其他相容性交集的應用程式，啟用筆墨的應用程式應該將點陣圖和元檔案格式的影像移至剪貼簿。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 筆跡序列化格式 (ISF)<br/>      | ISF 是最簡單且易於保留格式的筆跡表示法。 雖然它通常只包含筆墨資料，但 ISF 是可擴充的。 應用程式可以將自訂屬性 (由全域唯一識別碼識別 (GUID) ) [**筆跡**](inkdisp-class.md) 物件、筆墨筆劃或筆跡點。 這可讓您將任何種類的資料或中繼資料儲存為 ISF 資料流程中的屬性。 針對剪貼簿互通性，可以將筆墨放入在軟體發展工具組 (SDK) 標頭檔中定義的 ISF 標準剪貼簿位置。<br/> ISF 是一種適用于 Microsoft Tablet PC 技術的格式，而且只有在 [**筆跡**](inkdisp-class.md) 物件的 [**Load**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkdisp-load) 和 [**Save**](/windows/desktop/api/msinkaut/nf-msinkaut-iinkdisp-save) 方法中才支援。<br/> |
| RTF<br/>                              | 您可以產生 RTF 剪貼簿格式，並且將 RTF 格式的筆墨編碼為 OLE 物件。 這可讓筆墨貼到 OLE 容器中，例如 Microsoft Word 或 RichEdit 架構的應用程式。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Extensible Markup Language (XML) - 可延伸標記語言 (XML)<br/> | 應用程式可以使用其中一種以64編碼的筆墨格式，以 XML 檔案格式儲存筆跡。 XML 格式適合用來將筆墨內容輸入至資料庫，例如簽章欄位，或甚至是應用程式的主要檔案格式。 這樣可減輕撰寫剖析器的需求。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                        |
