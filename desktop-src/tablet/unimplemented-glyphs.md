# <a name="unimplemented-glyphs"></a>未實現的字元

以下是 Microsoft 打算在未來作為 *microsoft 手勢辨識器* 一部分支援的手勢圖像清單。 正在進行工作，以確保這些字元的辨識精確度很高 (，以避免意外啟用) ，而且它們對應至適當的作業。

若要確保應用程式之間的一般 *動作* 使用的 *手勢* 一致性，您應該遵循下列建議：

-   *動作* 是與手勢相關聯的建議語義行為。
-   針對下表中標示為 *固定* 的手勢，建議您不要變更建議的語義行為。 如果應用程式不需要指定的語義行為，建議您不要將手勢重複用於另一個動作或語義。
-   您應該放心地將相關的語義行為與所有其他手勢產生關聯。 這些手勢標示為 *特定應用程式*。 如果您的應用程式中有對應的功能，建議您針對具有建議語義行為的手勢，使用筆勢來取得建議的語義。
-   筆勢的作用點是筆勢幾何中的區別點。 作用點可以用來判斷手勢的進行位置。 手勢 API 可讓您判斷指定筆勢的作用點。 不過，並非所有手勢都有特定的作用點。 如果沒有特定的作用點，就會將起始點回報為作用點。

> 有些手勢有很容易的作用點，也就是起點。 這些都可在資料表中辨識。
