# <a name="internet-explorer-8"></a>Internet Explorer 8

*網際網路* 的演進引進了豐富資訊的新來源，以及更多方式來存取它。 這種成長已為 web 創造了新的商機、體驗、線上服務和標準。 有了這項強度和依賴，網頁程式開發人員就會面臨一組不斷演進的需求，Windows 以三種主要方式 Internet Explorer 8 個位址。

首先，Internet Explorer 8 為現有的網站提供與其他瀏覽器和相容性的真實世界互通性。 它包含核心平臺和架構的增強功能，可提供更佳的效能、安全性、可靠性和相容性。 使用舊版的 Windows Internet Explorer，開發人員和設計人員有時會注意到 Internet Explorer 已經有自己的 web 標準解釋，以及瀏覽器處理 HTML 的方式，階層式樣式表 (CSS) 、腳本等等。 在某些情況下，因為 Internet Explorer 在完成對應的標準之前支援某些功能，所以決定瞭解釋。 如果這些標準在完成時變更，Internet Explorer 的執行方式可能會與標準指定的不同。 在 Internet Explorer 8 中，支援舊版行為與嚴格標準的決策將會藉由讓您以逐頁逐頁選取轉譯模式，來推動開發人員的手。

其次，Internet Explorer 8 可讓您使用內建的開發人員工具，更快且更輕鬆地進行 網頁程式開發。 透過對非同步 JavaScript 和 XML (AJAX) 應用程式的支援，以及簡化跨瀏覽器應用程式建立程式的改良功能，Internet Explorer 8 可讓開發人員在建立最強大的 web 應用程式時更具生產力。

最後，Internet Explorer 8 可讓您透過新的瀏覽器功能，輕鬆地將使用者連接到創新的 web 服務，以達成超越網頁的體驗。 Internet Explorer 8 的功能是一組整合式開發人員工具，也就是瀏覽器的元件。 如此一來，就能改善效能，而且當工具未執行時，就不會使用任何記憶體。 整合式開發人員工具可讓開發人員更快速、更輕鬆地開發和疑難排解豐富的內容網站。 Internet Explorer 8 的標準模式配置引擎也是以階層式樣式表、層級2修訂 1 (css 2.1) 規格和完全合規性為基礎，協助開發人員輕鬆地支援最新的標準，並在必要時，可選擇切換回 Windows Internet Explorer 7 相容 (或較早的) 行為。 為了協助開發人員充分利用 HTML 4 所提供的元素，Internet Explorer 8 提供數個表示性元素的升級支援。 例如，Q 元素代表內嵌引號的字串，而物件專案現在可代表任何「物件」（包括影像）。 透過改善這些和其他 HTML 專案的支援，網頁程式開發人員可以提供更具表達性和可存取的 HTML 標籤。

除了對基礎平臺和開發人員體驗所做的改進之外，Internet Explorer 8 還引進了新功能，可為 網頁程式開發人員提供新的功能，讓 網頁程式開發人員將其體驗延伸至網頁以外的範圍。 這些新功能是透過社區標準來啟用，而不是建立無法跨瀏覽器互通的自訂用戶端程式代碼。
