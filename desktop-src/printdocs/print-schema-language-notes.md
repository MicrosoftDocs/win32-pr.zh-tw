# <a name="print-schema-language-notes"></a>列印架構語言注意事項

本主題並非最新的。 如需最新資訊，請參閱 [列印架構規格](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip)。

在此規格中，用來定義每個需求之重要性的單字會以大寫寫入。 這些字組會根據 RFC 2119 中的定義來使用，而其各自的意義則會在下面重現：



| 詞彙                                                                | 描述                                                                                                                                                                                                                                                                                         |
|---------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="MUST"></span><span id="must"></span>必須<br/>       | 這個單字或形容詞的必要專案，表示該專案是規格的絕對需求。<br/>                                                                                                                                                                               |
| <span id="SHOULD"></span><span id="should"></span>應該<br/> | 這個單字或形容詞建議的意思是，在特定情況下可能會忽略此專案的有效原因，但您應該瞭解完整的含意，並在選擇不同的課程之前仔細權衡。<br/>                                     |
| <span id="MAY"></span><span id="may"></span>可能<br/>          | 此單字或形容詞選擇性，表示此專案是真正的選擇性專案。 例如，某個執行可能會選擇包含專案，因為特定的 marketplace 或案例需要它，或因為它會增強產品。 另一個執行可能會省略相同的專案。<br/> |
