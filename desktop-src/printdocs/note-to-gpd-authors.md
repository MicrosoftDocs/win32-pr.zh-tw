# <a name="note-to-gpd-authors"></a>GPD 作者注意事項

本主題並非最新的。 如需最新資訊，請參閱 [列印架構規格](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip)。

針對熟悉 GPD 檔案的 PrintCapabilities 檔作者，某些 GPD 關鍵字在 PrintCapabilities 檔中沒有任何對等專案。 下表包含 GPD 關鍵字，不含 PrintCapabilities 檔對等專案，而且沒有相等的原因。



| GPD 關鍵字                                                                            | 說明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|----------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| \*約束<br/> \*InvalidCombination<br/> \*ConflictPriority<br/> | PrintCapabilities 檔中未定義條件約束，因為 PrintCapabilities 用戶端不應該處理、強制執行或解決它們。 這些工作會留給 PrintTicket 提供者在 PrintTicket 驗證期間執行。 Unidrv 外掛程式可以提供自己的 PrintTicket 驗證程式代碼，也可以依賴 Unidrv 來執行這項驗證。 在後者的情況下，Unidrv 會強制執行 GPD 檔中定義的任何條件約束。<br/> 整合型驅動程式必須提供自己的 PrintTicket 驗證程式代碼，並且必須提供自己的方法來表達和強制執行條件約束。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                       |
| \*DefaultOption<br/>                                                             | 有一個可傳回預設 PrintTicket 的 PrintTicket 方法，其依定義具有每項功能的所有預設設定。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| \*FeatureType<br/>                                                               | 功能可以分為三種不同的類別：<br/> 其設定定義于 PrintTicket 中的功能。 這種類型的功能稱為檔粘滯，因為這些設定會直接判斷檔的處理方式。<br/> 一項功能，其設定會反映未由使用者控制的實體裝置屬性，例如裝置中的記憶體數量，或指出是否有選用附加元件（例如紙紙送紙或 duplexers）。 這種類型的功能稱為裝置-粘滯或印表機。 這類功能的狀態很重要，因為它可以限制屬於檔粘滯功能的選項。<br/> 裝置-粘滯或印表機粘滯功能可進一步分類為使用者介面 (UI) 印表機的功能，或自動印表機的功能。 UI 印表機-粘滯功能必須顯示在系統管理員可以設定的使用者介面中。 裝置會自動偵測自動印表機的功能。<br/> |
| \*切換 ... \*情況 下<br/>                                                         | PrintCapabilities 提供者必須執行方法，以根據呼叫者指定的類型來建立 PrintCapabilities 檔，以列舉印表機的附注或檔粘滯功能。 基於這個理由，不需要透過 PrintCapabilities 檔本身呈現相同的資訊。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
