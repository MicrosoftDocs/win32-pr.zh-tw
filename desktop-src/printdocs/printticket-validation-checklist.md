<span data-ttu-id="69639-111">針對在 PrintCapabilities 檔中定義的一組功能、選項和 ParameterDef 實例的特定裝置，一律會執行驗證。</span><span class="sxs-lookup"><span data-stu-id="69639-111">Validation is always performed against a specific device that has a set of Feature, Option and ParameterDef instances defined in a PrintCapabilities document.</span></span> <span data-ttu-id="69639-112">驗證程式代碼應該可存取一組功能實例 (以及特定裝置的包含選項實例) 和 ParameterDef 實例，而且不應該存取 PrintCapabilities。</span><span class="sxs-lookup"><span data-stu-id="69639-112">The validation code should have access to the set of Feature instances (and the contained Option instances) and ParameterDef instances for the specific device, and should not need to access the PrintCapabilities.</span></span> <span data-ttu-id="69639-113">在驗證程式的某些部分中，需要功能、選項和 ParameterDef 實例的資訊。</span><span class="sxs-lookup"><span data-stu-id="69639-113">The information from the Feature, Option, and ParameterDef instances is needed in certain parts of the validation process.</span></span>

1.  <span data-ttu-id="69639-114">在嘗試尋找對應或比對專案的期間，請注意，元素的命名空間必須符合，才能將任何限定名稱視為相符。</span><span class="sxs-lookup"><span data-stu-id="69639-114">During attempts to locate corresponding or matching elements, note that the namespaces of the elements must match before any Qualified Name can be considered a match.</span></span> <span data-ttu-id="69639-115">所有專案名稱、屬性名稱和實例名稱都是限定命名空間。</span><span class="sxs-lookup"><span data-stu-id="69639-115">All element names, Attribute names, and instance names are namespace-qualified.</span></span> <span data-ttu-id="69639-116">針對已嵌套的專案，其位置必須符合，才能將專案視為相符。</span><span class="sxs-lookup"><span data-stu-id="69639-116">For elements that are nested, their locations must match before the elements are considered a match.</span></span>

2.  <span data-ttu-id="69639-117">確認所有元素標記都在公用命名空間中、由 PrintTicket 架構定義、包含適當的 XML 屬性和屬性值，而且每個專案類型的位置符合 PrintTicket 架構定義的使用方式。</span><span class="sxs-lookup"><span data-stu-id="69639-117">Verify that all element tags are in the public namespace, are defined by the PrintTicket Schema, contain the proper XML Attributes and Attribute Values, and that the location of each element type conforms to the PrintTicket Schema-defined usage.</span></span>

3.  <span data-ttu-id="69639-118">判斷 PrintCapabilities 檔所報告的所有命名空間。</span><span class="sxs-lookup"><span data-stu-id="69639-118">Determine all namespaces reported by PrintCapabilities document.</span></span> <span data-ttu-id="69639-119">從 PrintCapabilities 檔所報告的命名空間所屬的 PrintTicket 中，移除所有 (和其子系) 的元素。</span><span class="sxs-lookup"><span data-stu-id="69639-119">Remove all elements (and their descendants) from the PrintTicket whose instance names belong to namespaces not reported by PrintCapabilities document.</span></span> <span data-ttu-id="69639-120">請注意此案例與下列案例之間的差異，這牽涉到已知命名空間內無法辨識的實例名稱。</span><span class="sxs-lookup"><span data-stu-id="69639-120">Note the difference between this case and the following case, which involves unrecognized instance names within a known namespace.</span></span>

4.  <span data-ttu-id="69639-121">由於架構會持續擴充，並新增新的元素實例定義，因此不應該在假設指定命名空間中的每個實例名稱都知道時，撰寫驗證碼。</span><span class="sxs-lookup"><span data-stu-id="69639-121">Due to the fact that the schemas are continually being extended with the addition of new element instance definitions, the validation code should not be written under the assumption that every instance name in a given namespace is known.</span></span> <span data-ttu-id="69639-122">驗證程式代碼無法在已知的命名空間中將無法辨識的實例名稱視為錯誤，也無法從 PrintTicket 刪除它們。</span><span class="sxs-lookup"><span data-stu-id="69639-122">The validation code cannot treat unrecognized instance names within a known namespace as errors, nor can it delete them from the PrintTicket.</span></span>

5.  <span data-ttu-id="69639-123">如果任何專案實例具有 PrintTicket 架構不允許的重複同級，請只保留第一個出現的專案，然後刪除重複專案，包括重複專案的內容。</span><span class="sxs-lookup"><span data-stu-id="69639-123">If any element instance has a duplicate sibling that is not permitted by the PrintTicket Schema, keep only the first occurrence and delete the duplicates, including the contents of the duplicated element.</span></span>

6.  <span data-ttu-id="69639-124">從 PrintTicket 的任何功能或 subfeature (，以及 PrintCapabilities 檔中沒有對應功能的所有子系) 移除。</span><span class="sxs-lookup"><span data-stu-id="69639-124">Remove from the PrintTicket any Feature or subfeature (and all its children) that has no corresponding Feature in the PrintCapabilities document.</span></span>

7.  <span data-ttu-id="69639-125">針對 PrintTicket 中的每個剩餘功能實例，檢查 PrintCapabilities 檔中定義的 SelectionType 屬性。</span><span class="sxs-lookup"><span data-stu-id="69639-125">Check the SelectionType Property defined in the PrintCapabilities document for each of the remaining Feature instances in the PrintTicket.</span></span> <span data-ttu-id="69639-126">SelectionType 屬性設定為 PickOne 的任何功能，都必須只有一個選項實例存在於 PrintTicket 中，而 SelectionType 屬性為 PickMany 的功能可能會有一個以上的選項實例。</span><span class="sxs-lookup"><span data-stu-id="69639-126">Any Feature whose SelectionType Property is set to PickOne must have exactly one Option instance present in the PrintTicket, while a Feature whose SelectionType Property is PickMany may have more than one.</span></span> <span data-ttu-id="69639-127">如果 PrintTicket 功能沒有選項實例，請提供預設選項實例。</span><span class="sxs-lookup"><span data-stu-id="69639-127">If a PrintTicket Feature has no Option instance, supply the default Option instance.</span></span> <span data-ttu-id="69639-128">由於您是提供者，因此您知道哪一個選項是每項功能的預設選項。</span><span class="sxs-lookup"><span data-stu-id="69639-128">Since you are the provider, you know which Option is the default Option for every Feature.</span></span>

    <span data-ttu-id="69639-129">針對 SelectionType 屬性為 PickMany 的功能，在 PrintTicket 中選取一個以上的選項時，請確認未將任何選項指定為 IdentityOption。</span><span class="sxs-lookup"><span data-stu-id="69639-129">For a Feature whose SelectionType Property is PickMany, with more than one Option selected in the PrintTicket, verify that no Option is designated as IdentityOption.</span></span> <span data-ttu-id="69639-130">如果有，請刪除每個其他選項，只留下一個指定的 IdentityOption。</span><span class="sxs-lookup"><span data-stu-id="69639-130">If there is, delete every other Option, leaving only the one designated IdentityOption.</span></span>

8.  <span data-ttu-id="69639-131">移除在 PrintCapabilities 檔中沒有對應 ParameterDef 實例之 PrintTicket 中的任何 ParameterInit 實例。</span><span class="sxs-lookup"><span data-stu-id="69639-131">Remove any ParameterInit instance in the PrintTicket that has no corresponding ParameterDef instance in the PrintCapabilities document.</span></span>

    <span data-ttu-id="69639-132">針對 PrintTicket 中的任何其他 ParameterInit 實例，請確認每個實例的值都符合 PrintCapabilities 檔的 ParameterDef 實例。</span><span class="sxs-lookup"><span data-stu-id="69639-132">For any other ParameterInit instances in the PrintTicket, verify that the Value for each conforms to the PrintCapabilities document's ParameterDef instance.</span></span> <span data-ttu-id="69639-133">如果缺少值，請提供 ParameterDef 中提供的預設值。</span><span class="sxs-lookup"><span data-stu-id="69639-133">If a Value is missing, supply the default that is provided in the ParameterDef.</span></span>

9.  <span data-ttu-id="69639-134">根據評分流程的結果，將 PrintTicket 中的每個選項實例與 PrintCapabilities 檔中對應功能所列的選項配對。</span><span class="sxs-lookup"><span data-stu-id="69639-134">Pair each Option instance in the PrintTicket with an Option listed in the corresponding Feature in the PrintCapabilities document, based on the results of the scoring process.</span></span> <span data-ttu-id="69639-135">評分是在 PrintCapabilities 檔中尋找最符合 PrintTicket 中所指定選項之選項的程式。</span><span class="sxs-lookup"><span data-stu-id="69639-135">Scoring is the process of finding the Option in the PrintCapabilities document that best matches the Option named in the PrintTicket.</span></span> <span data-ttu-id="69639-136">如需評分程式期間所考慮之事項的說明，請參閱 [選項定義](option-definitions.md)。</span><span class="sxs-lookup"><span data-stu-id="69639-136">For a description of what is taken into account during the scoring process, see [Option Definitions](option-definitions.md).</span></span> <span data-ttu-id="69639-137">以對應的最符合候選 PrintCapabilities 檔選項取代 PrintTicket 中的每個參考選項。</span><span class="sxs-lookup"><span data-stu-id="69639-137">Replace each reference Option in the PrintTicket by the corresponding best matching candidate PrintCapabilities document Option.</span></span> <span data-ttu-id="69639-138">您也可以藉由分數來將這項資訊排名，並將這項資訊傳遞給解決階段，以防條件約束衝突導致無法使用最符合的候選項目。</span><span class="sxs-lookup"><span data-stu-id="69639-138">You may also rank all of the candidates by score and pass this information on to the resolving stage in case a constraint conflict prevents the best matching candidate from being used.</span></span> <span data-ttu-id="69639-139">在這種情況下，解析程式可以使用第二個最佳候選項，而不是隨機挑選另一個候選項。</span><span class="sxs-lookup"><span data-stu-id="69639-139">In such cases, the resolving process can use the second-best candidate rather than pick another candidate at random.</span></span>

10. <span data-ttu-id="69639-140">若為 SelectionType 屬性設定為 PickMany 的功能，且在 PrintTicket 中選取了一個以上的選項，請確認沒有任何選項指定為 IdentityOption。</span><span class="sxs-lookup"><span data-stu-id="69639-140">For a Feature whose SelectionType Property is set to PickMany, and that has more than one Option selected in the PrintTicket, verify that no Option is designated as IdentityOption.</span></span> <span data-ttu-id="69639-141">如果有這樣的選項，請刪除每個其他選項，只留下一個指定的 IdentityOption。</span><span class="sxs-lookup"><span data-stu-id="69639-141">If such an Option exists, delete every other Option, leaving only the one designated IdentityOption.</span></span> <span data-ttu-id="69639-142">您必須在套用評分程式之前和之後執行此步驟。</span><span class="sxs-lookup"><span data-stu-id="69639-142">This step must be performed before and after the scoring process is applied.</span></span>

    <span data-ttu-id="69639-143">此步驟必須執行兩次的原因是，評分程式可能會將多個參考選項實例對應至相同的候選選項。</span><span class="sxs-lookup"><span data-stu-id="69639-143">The reason that this step must be performed twice is that it is possible for the scoring process to map multiple reference Option instances to the same candidate Option.</span></span> <span data-ttu-id="69639-144">如果發生這種情況，請刪除任何重複的選項實例，讓針對特定 PickMany 功能列出的選項是唯一的。</span><span class="sxs-lookup"><span data-stu-id="69639-144">If that happens, delete any duplicate Option instances so that the Options listed for a particular PickMany Feature are unique.</span></span>

11. <span data-ttu-id="69639-145">將任何存在於 PrintCapabilities 檔中且未出現在 PrintTicket 中的功能加入至 PrintTicket。</span><span class="sxs-lookup"><span data-stu-id="69639-145">Add to the PrintTicket any Feature present in the PrintCapabilities document that does not appear in the PrintTicket.</span></span> <span data-ttu-id="69639-146">針對這類功能，請指定預設選項做為選取的選項。</span><span class="sxs-lookup"><span data-stu-id="69639-146">For such a Feature, designate the default Option as the selected Option.</span></span>

12. <span data-ttu-id="69639-147">判斷是否有任何符合下列條件的 ParameterDef 實例：</span><span class="sxs-lookup"><span data-stu-id="69639-147">Determine whether there are any ParameterDef instances that meet all of the following criteria:</span></span>

    -   <span data-ttu-id="69639-148">ParameterDef 實例會出現在 PrintCapabilities 檔中，但不會出現在 PrintTicket 中。</span><span class="sxs-lookup"><span data-stu-id="69639-148">The ParameterDef instance appears in the PrintCapabilities document, but not in the PrintTicket.</span></span>

    -   <span data-ttu-id="69639-149">ParameterDef 實例的必要屬性會設定為無條件或條件式。</span><span class="sxs-lookup"><span data-stu-id="69639-149">The Mandatory Property of the ParameterDef instance is set to either Unconditional or Conditional.</span></span>

    -   <span data-ttu-id="69639-150">ParameterDef 實例是由選項實例內的 PrintTicket 中的 ParameterRef 實例所參考。</span><span class="sxs-lookup"><span data-stu-id="69639-150">The ParameterDef instance is referenced by a ParameterRef instance in the PrintTicket within an Option instance.</span></span>

    <span data-ttu-id="69639-151">針對 PrintCapabilities 檔中的每個這類 ParameterDef 實例，將對應的 ParameterInit 實例加入至 PrintTicket。</span><span class="sxs-lookup"><span data-stu-id="69639-151">For each such ParameterDef instance in the PrintCapabilities document, add to the PrintTicket a corresponding ParameterInit instance.</span></span> <span data-ttu-id="69639-152">將新加入的 ParameterInit 實例值設定為對應 ParameterDef 實例所指定的預設值。</span><span class="sxs-lookup"><span data-stu-id="69639-152">Set the Value of the newly added ParameterInit instances to the default Value specified by the corresponding ParameterDef instances.</span></span>

13. <span data-ttu-id="69639-153">執行條件約束衝突偵測並修改設定，以排除任何這類衝突。</span><span class="sxs-lookup"><span data-stu-id="69639-153">Perform constraint conflict detection and modify the configuration to eliminate any such conflicts.</span></span> <span data-ttu-id="69639-154">本主題不會定義用來解決條件約束衝突的特定處理常式。</span><span class="sxs-lookup"><span data-stu-id="69639-154">This topic does not define a particular process to use to resolve constraint conflicts.</span></span> <span data-ttu-id="69639-155">您必須決定可以變更哪些功能或 ParameterInit 實例，以及分別選取適當的選項或值，以針對 PrintTicket 中指定之設定的整體意圖產生最小的影響。</span><span class="sxs-lookup"><span data-stu-id="69639-155">You need to decide which Feature or ParameterInit instance can be changed, and an appropriate Option or Value, respectively, to select that has the least impact on the overall intent of the configuration specified in the PrintTicket.</span></span> <span data-ttu-id="69639-156">如先前所述，您可能會想要使用每個選項的對應分數，並使用具有第二個最高分分數的選項。</span><span class="sxs-lookup"><span data-stu-id="69639-156">As mentioned earlier, you might wish to make use of the mapping score of each Option, and use the Option with the second highest score.</span></span> <span data-ttu-id="69639-157">若要判斷要變更的功能或 ParameterInit，您可能會想要定義用戶端可以加入至 PrintTicket 的私用屬性。</span><span class="sxs-lookup"><span data-stu-id="69639-157">To determine which Feature or ParameterInit to change, you might want to define a private Property that the client can add to the PrintTicket.</span></span> <span data-ttu-id="69639-158">這個屬性可以定義 Feature 和 ParameterInit 實例的優先順序，讓解析程式知道哪些功能或 ParameterInit 實例對用戶端而言是很重要的 (而且需要保留在 PrintTicket) 中，這一點比較不重要。</span><span class="sxs-lookup"><span data-stu-id="69639-158">This Property can define a priority for Feature and ParameterInit instances so that the resolver process is informed of which Feature or ParameterInit instances are important to the client (and need to be preserved in the PrintTicket), and which are less important.</span></span>

16. <span data-ttu-id="69639-164">如果您的 PrintTicket 提供者可辨識並支援任何已在此點存活的私用或公用屬性實例，請在這些實例上執行驗證。</span><span class="sxs-lookup"><span data-stu-id="69639-164">If your PrintTicket provider recognizes and supports any private or public Property instances that have survived to this point, perform validation on them.</span></span> <span data-ttu-id="69639-165">請勿在此時刪除屬性，因為您無法辨識該屬性，它可能適用于另一個檔處理階段。</span><span class="sxs-lookup"><span data-stu-id="69639-165">Do not delete a Property at this point just because you do not recognize it, it might be intended for another phase of document processing.</span></span>
