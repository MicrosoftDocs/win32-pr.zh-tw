# <a name="terms-used-in-the-print-schema"></a>列印架構中使用的詞彙

本主題並非最新的。 如需最新資訊，請參閱 [列印架構規格](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip)。

下列清單包含列印架構和相關技術特有的詞彙。



| 詞彙                                                                                                                                                                 | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>屬性名稱<br/>                                                                          | 屬性名稱是標準 XML 屬性中等號左邊的標記。 XML 屬性的標準標記法為 *attribute name*= "*attribute value*"。 雖然類似，但 "attribute name" 一詞不能與 Print Schema Framework 中定義的 name 屬性混淆。 在 Print Schema Framework 中定義的名稱屬性是屬性名稱的特定實例。 <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>候選人<br/>                                                                                         | 選項。 屬於計分進程期間所評估之清單的任何選項。 請參閱詞彙、計分或對應選項，以及參考選項。<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>配置<br/>                                             | 描述裝置可採用或放置的其中一種狀態。 裝置會在每個狀態中以不同的方式執行其工作。 在這些狀態中，裝置的行為之間有相異且可辨識的差異。 例如，在某個狀態下，媒體會從一個紙匣中取出，並在另一個紙匣中結束。 其中一個或兩個紙匣的變更都決定不同的狀態。<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>約束<br/>                                                         | 判斷設定有效性所需的資訊。 條件約束的格式通常為： FeatureA 設定為 Option1 與 FeatureB 設定為 Option2 不相容。<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>對應的元素<br/>                                                  | 與指定的一組來源元素有相同位置的兩個元素。 原始元素通常是兩個比較的選項元素。<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>預設<br/>                                                                                               | 選項。 當 PrintTicket 未包含該功能的明確設定時，由列印架構或 PrintCapabilities 提供者指定的選項，做為指派給功能的選項。 每項功能都有指定的預設選項。<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>裝置設定屬性<br/>                          | 提供裝置設定定義的裝置屬性。 裝置設定屬性是在 Print Schema Framework 中，透過功能/選項或參數結構來表示。 請注意，在此內容中使用的屬性與 XML 屬性無關，或是列印架構架構定義的屬性元素。<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>元素類型<br/>                                                                                | 在 Print Schema Framework 中定義之每個 XML 專案的型別，都是由其 XML 元素標記來識別。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>共通元素<br/>                                                              | 請參閱對應的元素。<br/> 特徵。 表示特定裝置設定屬性的列印架構架構架構定義的結構。 每個功能實例都代表一個裝置屬性。 功能實例的定義方式是明確定義其每個狀態。<br/> Identity 選項。 選取時沒有可觀察效果的選項;實際上是「無作業」。 無論選取這個選項的次數為何，結果一律相同。<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>實例名稱<br/>                                                                             | XML 屬性，識別元素類型的特定實例。 請參閱 name 屬性。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>介面用戶端<br/>                                                                    | 取用 PrintCapabilities 檔，並會建立和取用 PrintTickets 的實體。 介面用戶端的常見範例包括使用者介面 (UI) 模組和轉譯模組。<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>介面提供者<br/>                                                              | 藉由介面執行並公開功能的實體。 在列印架構中，這是用來建立 PrintCapabilities 檔及驗證 PrintTickets 的數量。 <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>元素的位置<br/>                                                  | 元素階層內元素的特定實例位置。 位置是由特定元素和該專案的所有上階所定義。 換句話說，location 相當於檔案系統中的完整路徑。 <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>name 屬性<br/>                                                                          | 針對屬性名稱為 "name" 的 XML 屬性指定的指定。 這個屬性會出現在 Print Schema Framework 中定義的大部分元素類型中。 它會識別並定義每個元素類型的實例。 下列兩個詞彙是指 name 屬性：同輩唯一性規則，以及命名空間慣例。<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>命名空間慣例<br/>                 | 指出架構定義的名稱屬性以及任何架構定義的物件，都存在於專用命名空間、公用命名空間，而且私用定義的名稱屬性必須存在於不同的命名空間中，其中一個可以識別為屬於定義的合作物件。<br/> 選項。 用來表示每項功能可以假設或放置之狀態的列印架構架構架構。 例如，如果此功能代表媒體大小，則一般的選項值可能是字母、合法、A4 及其他值。<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>參數<br/>                                                             | 列印架構支援對使用者輸入的目的描述參數化資料的能力。 有三種列印架構架構元素類型，可在參數化輸入中扮演角色： ParameterDef、ParameterInit 和 ParameterRef。 ParameterDef 元素會將參數的特性 (其值範圍、其資料類型，以及其他特性) 。 ParameterInit 元素是用來初始化參數。 ParameterRef 元素是用來參考 ParameterInit 元素的特定實例。<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>參數<br/>                                                                             | 選項。 包含至少一個 ScoredProperty 專案的選項實例，其中包含一個 ParameterRef 元素，而不是值元素。<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>完全相符<br/>                                                              | 選項。 如果有一個選項實例中的每個 ScoredProperty 元素、另一個選項實例中有對應的 ScoredProperty 實例，而且對應的 ScoredProperty 實例的兩個值元素相同，則會將兩個選項實例視為完全相符。<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>PrintCapabilities 架構<br/> | 定義 PrintCapabilities 檔所使用之語法和語義的架構。 PrintCapabilities 架構和 PrintTicket 架構是列印架構的子集。 另請參閱 Print Schema Framework。<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>PrintTicket 架構<br/>                         | 定義 PrintTicket 所使用之語法和語義的架構。 PrintTicket 架構和 PrintCapabilities 架構是列印架構的子集。 另請參閱 Print Schema Framework。<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Print Schema Framework<br/>         | 定義列印架構技術所使用之語法和語義的架構。 例如，Print Schema Framework 會定義元素類型及其用法的集合，但不會列舉所有元素實例。<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>列印架構關鍵字<br/>             | 定義屬性、功能和選項之個別實例的架構，這些專案通常用來描述裝置和作業的格式。 此架構會定義讓 PrintCapabilities 檔和 PrintTickets 可攜的通用語言。 也稱為公用架構。<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>私下定義<br/>                                                                 | 出現在 PrintCapabilities 檔或 PrintTicket 中且未在 Print Schema 關鍵字中明確定義的任何元素實例。 您必須使用每個廠商的私用命名空間來明確限定這些元素實例。<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>參考<br/>                                                                                         | 選項。 比較每個候選選項的選項。 參考選項通常源自于 PrintTicket。 請參閱詞彙、計分或對應選項，以及候選選項。<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>根層級<br/>                                                                                      | 根層級是 Print Schema Framework 階層式結構的基底。 根層級元素會定義為 PrintCapabilities 專案或 PrintTicket 元素的子系。<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>範圍前置詞<br/>                                                                          | 將文字標籤預先附加至 schema 關鍵字，以提供內容約制。 允許的列印架構範圍前置詞為工作、檔和頁面。 例如，JobInputBin 的範圍前置詞為「作業」。<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | 用來表示品質的列印架構架構結構，可區別或區分一個選項實例與另一個選項實例。 例如，字母和合法大小的紙張之間的差異。<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>評分或對應<br/>                                                              | 選項。 從清單中選取最符合指定之 (參考) 選項實例的 (候選) 選項實例的進程。 密切相關的詞彙是候選選項和參考選項。 <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>同輩唯一性規則<br/>     | 指出相同元素類型的所有同級都必須有唯一的名稱屬性。 某些元素類型不會套用此規則。 這些例外狀況是在 Print Schema Framework 中定義。<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>單一值和多重值<br/>                    | 屬性。 值為非變異 (的屬性不會相依于設定) 視為單一值。 在任何可能的設定下，這類屬性只有一個值。 相依于設定的屬性會視為多重值。<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>尾碼/尾碼<br/>                                                                       | 架構關鍵字，表示功能或參數名稱。 例如，JobInputBin 的尾碼為 ' InputBin'。<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>驗證<br/>                                                         | 驗證 PrintTicket 符合架構的程式，並且代表可由特定裝置執行的設定。<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_定義\_<br/>                                                   | 此處所示的特定資料是特定驅動程式、裝置或應用程式所要產生的預留位置。 它不會傳達資料應該是字串。<br/>                                                                                                                                                                                                                                                                                                                                                            |
