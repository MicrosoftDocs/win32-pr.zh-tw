## <a name="remarks"></a><span data-ttu-id="031e5-141">備註</span><span class="sxs-lookup"><span data-stu-id="031e5-141">Remarks</span></span>

<span data-ttu-id="031e5-142">應用程式通常會使用 **WM \_ DEADCHAR** 訊息，為使用者提供每個按鍵的相關意見反應。</span><span class="sxs-lookup"><span data-stu-id="031e5-142">The **WM\_DEADCHAR** message typically is used by applications to give the user feedback about each key pressed.</span></span> <span data-ttu-id="031e5-143">例如，應用程式可以在目前的字元位置顯示輔色，而不移動插入號。</span><span class="sxs-lookup"><span data-stu-id="031e5-143">For example, an application can display the accent in the current character position without moving the caret.</span></span>

<span data-ttu-id="031e5-144">因為所按下的按鍵和產生的字元訊息之間不一定要有一對一的對應，所以 *lParam* 參數的高序位字中的資訊通常對應用程式而言並不有用。</span><span class="sxs-lookup"><span data-stu-id="031e5-144">Because there is not necessarily a one-to-one correspondence between keys pressed and character messages generated, the information in the high-order word of the *lParam* parameter is generally not useful to applications.</span></span> <span data-ttu-id="031e5-145">高序位字組中的資訊只適用于在 **\_ DEADCHAR** 訊息張貼之前的最新 wm 的一則的 [**wm \_**](wm-keydown.md)訊息。</span><span class="sxs-lookup"><span data-stu-id="031e5-145">The information in the high-order word applies only to the most recent [**WM\_KEYDOWN**](wm-keydown.md) message that precedes the posting of the **WM\_DEADCHAR** message.</span></span>
