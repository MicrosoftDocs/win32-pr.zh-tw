# <a name="addblockedcommand-method-of-the-win32_tpm-class"></a>Win32 Tpm 類別的 AddBlockedCommand 方法 \_

[**Win32 \_ tpm**](win32-tpm.md)類別的 **ADDBLOCKEDCOMMAND** 方法會將 Tpm 命令新增至封鎖在作業系統上執行之命令的本機清單。

> [!Caution]  
> 作業系統預設會防止一組預先定義的命令執行。 對此預設值所做的變更可能會影響您電腦的安全性、隱私權或功能。

 

## <a name="syntax"></a>語法


```mof
uint32 AddBlockedCommand(
  [in] uint32 CommandOrdinal
);
```

## <a name="security-considerations"></a>安全性考量

變更封鎖命令的預設清單，可能會讓您的電腦暴露在安全性和隱私權風險之下。

## <a name="remarks"></a>備註

群組原則可以覆寫 **AddBlockedCommand** 方法的效果。 系統管理員可以設定群組原則忽略封鎖命令的本機清單。

如果 *CommandOrdinal* 所指出的值已出現在封鎖命令的本機清單上，則會傳回零。

受控物件格式 (MOF) 檔包含 Windows Management Instrumentation (WMI) 類別的定義。 MOF 檔案不會安裝為 Windows SDK 的一部分。 當您使用伺服器管理員新增相關聯的角色時，它們會安裝在伺服器上。 如需 MOF 檔案的詳細資訊，請參閱 [受控物件格式 (mof) ](../wmisdk/managed-object-format--mof-.md)。
