# <a name="dynamic-data-exchange-management-library"></a>動態資料交換管理程式庫

動態資料交換 (的 DDE) 是一種處理序間通訊形式，可使用共用記憶體來交換應用程式之間的資料。 應用程式可以使用 DDE 進行單次資料傳輸，以及進行資料的進行中交換和更新。

### <a name="ddeml-transactions"></a>DDEML 交易



| Name                                                   | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|--------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**XTYP \_ ADVDATA**](xtyp-advdata.md)                  | 通知用戶端資料項目目的值已變更。 DDE 用戶端回呼函式 [**DdeCallback**](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會在建立與伺服器的建議迴圈之後，接收此交易。 <br/>                                                                                                                                                                                                                                                                                              |
| [**XTYP \_ ADVREQ**](xtyp-advreq.md)                    | 通知伺服器，指定的主題名稱和專案名稱組上的建議交易未完成，而且對應到主題名稱和專案名稱組的資料已經變更。 當伺服器呼叫 [**DdePostAdvise**](/windows/desktop/api/Ddeml/nf-ddeml-ddepostadvise)函數之後，系統會將此交易傳送至 DDE 回呼函式 [**DdeCallback**](/windows/win32/api/ddeml/nc-ddeml-pfncallback)。 <br/>                                                                                                                                     |
| [**XTYP \_ ADVSTART**](xtyp-advstart.md)                | 用戶端會使用 [**XTYP \_ ADVSTART**](xtyp-advstart.md) 交易來建立與伺服器之間的建議迴圈。 當用戶端將 **XTYP \_ ADVSTART** 指定為 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函數的 *wType* 參數時，DDE 伺服器回呼函式 [**DdeCallback**](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                                      |
| [**XTYP \_ ADVSTOP**](xtyp-advstop.md)                  | 用戶端會使用 [**XTYP \_ ADVSTOP**](xtyp-advstop.md) 交易，以伺服器結束「建議」迴圈。 當用戶端在 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函數中指定 **XTYP \_ ADVSTOP** 時，DDE 伺服器回呼函式 [**DdeCallback**](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                                                                        |
| [**XTYP \_ 連接**](xtyp-connect.md)                  | 用戶端會使用 [**XTYP \_ CONNECT**](xtyp-connect.md) 交易來建立交談。 當用戶端指定伺服器支援 (的服務名稱，以及 [**DdeConnect**](/windows/desktop/api/Ddeml/nf-ddeml-ddeconnect)函式呼叫中不是 **Null**) 的主題名稱時，DDE 伺服器回呼函式 [**DdeCallback**](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                            |
| [**XTYP \_ CONNECT \_ 確認**](xtyp-connect-confirm.md) | 「DDE 伺服器回呼函式」（ [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)）會接收 [**XTYP \_ CONNECT \_ CONFIRM**](xtyp-connect-confirm.md) 交易，以確認已與用戶端建立交談，並為伺服器提供交談控制碼。 由於先前的 [**XTYP \_ CONNECT**](xtyp-connect.md) 或 [**XTYP \_ WILDCONNECT**](xtyp-wildconnect.md) 交易的結果，系統會傳送此交易。 <br/>                                                                   |
| [**XTYP \_ 中斷連線**](xtyp-disconnect.md)            | 當交談中的應用程式夥伴使用 [**DdeDisconnect**](/windows/desktop/api/Ddeml/nf-ddeml-ddedisconnect)函式來終止交談時，應用程式的 DDE 回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收 [**XTYP \_ 中斷連接**](xtyp-disconnect.md)交易。 <br/>                                                                                                                                                                                                                       |
| [**XTYP \_ 錯誤**](xtyp-error.md)                      | 當發生重大錯誤時，DDE 回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會收到 [**XTYP \_ 錯誤**](xtyp-error.md) 交易。 <br/>                                                                                                                                                                                                                                                                                                                                                           |
| [**XTYP \_ EXECUTE**](xtyp-execute.md)                  | 用戶端會使用 [**XTYP \_ EXECUTE**](xtyp-execute.md) transaction 將命令字串傳送至伺服器。 當用戶端在 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函數中指定 **XTYP \_ EXECUTE** 時，DDE 伺服器回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                                                                       |
| [**XTYP \_ 監視**](xtyp-monitor.md)                  | 當系統中發生 DDE 事件時，DDE 偵錯工具的 DDE 回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收 [**XTYP \_ 監視器**](xtyp-monitor.md) 交易。 若要接收此交易，應用程式必須在呼叫 [**DdeInitialize**](/windows/desktop/api/Ddeml/nf-ddeml-ddeinitializea)函式時指定 **APPCLASS \_ 監視器** 值。 <br/>                                                                                                                                                               |
| [**XTYP \_**](xtyp-poke.md)                        | 用戶端會使用 [**XTYP \_**](xtyp-poke.md) 請求交易，將未經要求的資料傳送到伺服器。 當用戶端在 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函數中指定 **XTYP \_** 時，DDE 伺服器回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                                                                                |
| [**XTYP \_ 註冊**](xtyp-register.md)                | 每當 DDEML 伺服器應用程式使用 [**DdeNameService**](/windows/desktop/api/Ddeml/nf-ddeml-ddenameservice)函式來註冊服務名稱，或每次支援系統主題的非 DDEML 應用程式啟動時，DDE 回呼函式（ [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)）就會收到 **XTYP \_ REGISTER** 交易類型。 <br/>                                                                                                                                                                                         |
| [**XTYP \_ 要求**](xtyp-request.md)                  | 用戶端會使用 [**XTYP \_ 要求**](xtyp-request.md) 交易來向伺服器要求資料。 當用戶端在 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函數中指定 **XTYP \_ 要求** 時，DDE 伺服器回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/>                                                                                                                                                                                                |
| [**XTYP \_ 取消註冊**](xtyp-unregister.md)            | 每當 DDEML 伺服器應用程式使用 [**DdeNameService**](/windows/desktop/api/Ddeml/nf-ddeml-ddenameservice)函式取消註冊服務名稱，或每次支援系統主題的非 DDEML 應用程式終止時，DDE 回呼函式（ [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)）就會收到 [**XTYP \_ 取消註冊**](xtyp-unregister.md)的交易。 <br/>                                                                                                                                                                |
| [**XTYP \_ WILDCONNECT**](xtyp-wildconnect.md)          | [**XTYP \_ WILDCONNECT**](xtyp-wildconnect.md)交易可讓用戶端在每個伺服器的服務名稱和主題名稱組上，建立符合指定服務名稱和主題名稱的交談。 當用戶端在呼叫 [**DdeConnect**](/windows/desktop/api/Ddeml/nf-ddeml-ddeconnect)或 [**DdeConnectList**](/windows/desktop/api/Ddeml/nf-ddeml-ddeconnectlist)函式時，當用戶端指定 **null** 服務名稱、 **Null** 主題名稱或兩者時，DDE 伺服器回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收此交易。 <br/> |
| [**XTYP \_ 交易 \_ 完成**](xtyp-xact-complete.md)     | 當非同步交易（由 [**DdeClientTransaction**](/windows/desktop/api/Ddeml/nf-ddeml-ddeclienttransaction)函式的呼叫所起始）完成時，DDE 用戶端回呼函式 [*DdeCallback*](/windows/win32/api/ddeml/nc-ddeml-pfncallback)會接收 XTYP 的交易 [**\_ \_ 完成**](xtyp-xact-complete.md)交易。 <br/>                                                                                                                                                                                                                      |
