# <a name="directory-services-today"></a>今天的目錄服務

通常會在單一組織內找到多個系統管理目錄。 這些目錄包含網路資源目錄，例如以 LDAP 為基礎的目錄，例如 Microsoft Active Directory directory 服務、Windows 作業系統目錄服務，以及應用程式特定目錄，例如 Microsoft Exchange。

![部署多個目錄](images/ds2chal.png)

## <a name="the-directory-challenge"></a>目錄挑戰

組織中的多個目錄對使用者、系統管理員和開發人員而言，會造成複雜的挑戰。 這些問題具有有限的全目錄部署。 使用者會在多個目錄之間，面對多個登入和各種介面的資訊。 系統管理員面臨管理多個目錄的複雜性。 終端使用者和系統管理員想要讓應用程式開發人員使用現有的系統管理目錄，但開發人員會面臨有關要使用哪一個的困境。 每個目錄都提供獨特的應用程式介面。 開發人員必須選擇特定的目錄執行，或支援其應用程式的多個版本。 因此，開發人員很少會使用現有的目錄服務。
