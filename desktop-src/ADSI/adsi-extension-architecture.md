# <a name="adsi-extension-architecture"></a><span data-ttu-id="05fd5-106">ADSI 擴充功能架構</span><span class="sxs-lookup"><span data-stu-id="05fd5-106">ADSI Extension Architecture</span></span>

<span data-ttu-id="05fd5-107">ADSI 擴充功能是以 COM 匯總模型為基礎，具有數個增強功能。</span><span class="sxs-lookup"><span data-stu-id="05fd5-107">ADSI extensions are based on the COM aggregation model with several enhancements.</span></span> <span data-ttu-id="05fd5-108">擴充功能必須遵守所有 COM 規則。</span><span class="sxs-lookup"><span data-stu-id="05fd5-108">Extensions must adhere to all COM rules.</span></span> <span data-ttu-id="05fd5-109">如需詳細資訊，請參閱 COM 規格。</span><span class="sxs-lookup"><span data-stu-id="05fd5-109">For more information, see the COM specification.</span></span>

<span data-ttu-id="05fd5-110">以下是 COM 匯總模型的評論。</span><span class="sxs-lookup"><span data-stu-id="05fd5-110">Here is a review of the COM aggregation model.</span></span>

![com 匯總模型](images/comagmod.png)

<span data-ttu-id="05fd5-112">匯總（也稱為內建物件）是匯總工具所建立的物件。</span><span class="sxs-lookup"><span data-stu-id="05fd5-112">An aggregate, also known as an inner object, is an object that an aggregator creates.</span></span> <span data-ttu-id="05fd5-113">您的擴充物件是匯總。</span><span class="sxs-lookup"><span data-stu-id="05fd5-113">Your extension object is an aggregate.</span></span>

<span data-ttu-id="05fd5-114">匯總工具（也稱為外部物件）是建立匯總的物件。</span><span class="sxs-lookup"><span data-stu-id="05fd5-114">An aggregator, also known as an outer object, is an object that creates an aggregate.</span></span> <span data-ttu-id="05fd5-115">ADSI 是一個匯總工具。</span><span class="sxs-lookup"><span data-stu-id="05fd5-115">ADSI is an aggregator.</span></span>

<span data-ttu-id="05fd5-116">內建物件會將其 [**iunknown**](/windows/win32/api/unknwn/nn-unknwn-iunknown) 委派給匯總工具的 **iunknown**。</span><span class="sxs-lookup"><span data-stu-id="05fd5-116">The inner object delegates its [**IUnknown**](/windows/win32/api/unknwn/nn-unknwn-iunknown) to the aggregator's **IUnknown**.</span></span>

<span data-ttu-id="05fd5-117">ADSI 擴充功能會將下列增強功能新增至 COM 匯總，以滿足其需求：</span><span class="sxs-lookup"><span data-stu-id="05fd5-117">ADSI extensions add the following enhancements to COM aggregation to satisfy its requirements:</span></span>

-   <span data-ttu-id="05fd5-118">可讓每個延伸模組寫入器擴充 ADSI 物件。</span><span class="sxs-lookup"><span data-stu-id="05fd5-118">Enables each extension writer to extend ADSI objects.</span></span> <span data-ttu-id="05fd5-119">延伸模組寫入器可以使用 ADSI 註冊它的擴充功能，而不會受到其他副檔名的存在影響。</span><span class="sxs-lookup"><span data-stu-id="05fd5-119">An extension writer can register its extension with ADSI and not be affected by the existence of other extensions.</span></span> <span data-ttu-id="05fd5-120">在 COM 匯總模型中，匯總工具必須有匯總的 CLSID。</span><span class="sxs-lookup"><span data-stu-id="05fd5-120">In the COM aggregation model, the aggregator must have the aggregate's CLSID.</span></span> <span data-ttu-id="05fd5-121">ADSI 會放寬這項需求，使其成為所有延伸模組的匯總工具。</span><span class="sxs-lookup"><span data-stu-id="05fd5-121">ADSI relaxes this requirement by making itself act as the aggregator for all extensions.</span></span> <span data-ttu-id="05fd5-122">因此，延伸模組會位於相同層級，而不是形成一層嵌套的元件。</span><span class="sxs-lookup"><span data-stu-id="05fd5-122">Therefore, instead of forming a layer of nested components, extensions are at the same level.</span></span>
-   <span data-ttu-id="05fd5-123">允許一個物件，一個 IDispatch。</span><span class="sxs-lookup"><span data-stu-id="05fd5-123">Allows one object, one IDispatch.</span></span> <span data-ttu-id="05fd5-124">自動化支援是 ADSI 最重要的功能之一。</span><span class="sxs-lookup"><span data-stu-id="05fd5-124">Automation support is one of the most important features of ADSI.</span></span> <span data-ttu-id="05fd5-125">因為 ADSI 支援 [**IDispatch**](/windows/win32/api/oaidl/nn-oaidl-idispatch) 介面，所以可達成自動化支援。</span><span class="sxs-lookup"><span data-stu-id="05fd5-125">Automation support is achieved because ADSI supports the [**IDispatch**](/windows/win32/api/oaidl/nn-oaidl-idispatch) interface.</span></span> <span data-ttu-id="05fd5-126">建議延伸模組寫入器支援 **IDispatch** 介面。</span><span class="sxs-lookup"><span data-stu-id="05fd5-126">Extension writers are encouraged to support the **IDispatch** interface.</span></span> <span data-ttu-id="05fd5-127">不過，指定的物件上應該只有一個 **IDispatch** 介面。</span><span class="sxs-lookup"><span data-stu-id="05fd5-127">However, there should be only one **IDispatch** interface on a given object.</span></span> <span data-ttu-id="05fd5-128">ADSI 整合並收集不同擴充功能的許多 **IDispatch** 介面，並將其顯示為自動化控制器的一個一致 **idispatch** 。</span><span class="sxs-lookup"><span data-stu-id="05fd5-128">ADSI integrates and collects the many **IDispatch** interfaces from different extensions and presents them as one consistent **IDispatch** to the Automation controller.</span></span> <span data-ttu-id="05fd5-129">每個擴充功能在匯總時，都必須將其 **idispatch** 呼叫重新路由傳送至 ADSI 所提供的 **idispatch** 。</span><span class="sxs-lookup"><span data-stu-id="05fd5-129">Each extension, when aggregated, must re-route its **IDispatch** calls to the **IDispatch** provided by ADSI.</span></span>

<span data-ttu-id="05fd5-130">由於 ADSI 物件管理員提供的服務（位於每個 ADSI 提供者上），因此這些解決方案都是可行的。</span><span class="sxs-lookup"><span data-stu-id="05fd5-130">All these solutions are possible because of services that the ADSI Object Manager provides, which reside on each ADSI provider.</span></span>
