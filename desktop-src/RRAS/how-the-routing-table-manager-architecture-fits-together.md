# <a name="how-the-routing-table-manager-architecture-fits-together"></a>路由表管理員架構如何搭配在一起

下圖顯示路由器的不同元件之間的關聯性。

![路由器元件之間的關聯性](images/rtsrvarch.png)

啟動路由器時，會啟動路由器管理員服務，以及一或多個路由通訊協定。 路由通訊協定與路由器上的各種介面相關聯。 路由器管理員也會啟動路由表管理員。

下圖顯示用戶端與路由表管理員的不同元件之間的關聯性。

![用戶端與路由表管理員元件之間的關聯性](images/rtmentrel.png)

路由器管理員會啟動路由表管理員的一或多個實例。 當路由表管理員的多個實例啟動時，路由器已設定為可作為一或多個虛擬路由器。 路由表管理員的每個實例都擁有一或多個介面;每一個介面只能由一個路由表管理員的實例所擁有。

路由表管理員的每個實例都與其他實例無關;實例之間不會交換任何資訊。

路由表管理員的每個實例都包含一或多個路由表。 每個路由表都與位址系列相關聯。

每個路由表都包含一或多個視圖。 在此範例中，路由表會顯示為單播和多播的觀點。 每個視圖都是路由表的子集。

下圖顯示用戶端與路由表管理員的多個實例之間的關聯性、路由表和 views。

![關聯性用戶端，路由表管理員，路由表，視圖](images/multrtabrel.png)

用戶端的實例可以使用路由表管理員的實例多次註冊（每個位址系列一次）。 用戶端可以註冊每個路由表管理員的實例。

下圖顯示路由表專案的關聯方式。 如需路由表專案的詳細資訊，請參閱 [路由表專案](routing-table-entries.md)。

![路由表專案之間的關聯性](images/nexthop.png)

路由表包含目的地。 每個目的地都與一或多個路由相關。 每個路由都有零個、一個或多個指標指向與路由相關聯的下一個躍點。 每個指標都會參考下一個躍點清單中的實際下一個躍點。 使用路由表管理員註冊的每個用戶端都會建立用戶端所擁有的下一個躍點清單。

路由只能包含與擁有路由之用戶端相關聯的下一個躍點清單的指標。
