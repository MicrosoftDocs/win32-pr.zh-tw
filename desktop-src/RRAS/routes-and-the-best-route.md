# <a name="routes-and-the-best-route"></a>路由和最佳路由

路由是目的地的「網路路徑」，具有與它相關聯的特定成本。 成本是以其系統管理喜好設定和其通訊協定特定度量來表示。 成本較低的路由優先于所有其他路由。

路由表中的路由專案包括：

-   目的地的控制碼
-   此路由的擁有者
-   提供路由資訊的鄰近 (對等) 
-   與路由狀態相關聯的旗標
-   與路由相關聯的旗標
-   路由的喜好設定和度量
-   路由所屬的視圖清單
-   路由擁有者私用的資訊
-   用來到達目的地的下一個躍點清單

下列值會結合在一起，以唯一的方式識別路由表中的路由：

-   目的地網路
-   路由的擁有者
-   提供路由的鄰居

## <a name="metrics-and-preference"></a>計量和喜好設定

每個路由都有一個管理喜好設定 (由路由原則) 指定，以及與用戶端相依的度量。 路由表管理員會使用這項資訊來判斷哪個路由是目的地的較佳路由。 較低喜好設定的路由是較佳的路由， (一個較低的路由，因此最適合) 。 如果兩個路由具有相同的喜好設定，則具有較低計量的路由會是較佳的路由。

一般來說，路由的喜好設定取決於新增路由之用戶端的喜好設定。 不過，對於使用 Netsh.exe 管理工具新增的任何路由，可以根據每個路由來指定喜好設定值。

喜好設定通常用來指出用戶端之間的優先順序。 例如，系統管理員可以將較低的 () 喜好設定指派為較 RIP 更低的版本。 在此情況下，適用于 RIP 路由的 OSPF 路由比較理想。
