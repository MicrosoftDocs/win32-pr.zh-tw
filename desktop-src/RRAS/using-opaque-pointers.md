# <a name="using-opaque-pointers"></a>使用不透明的指標

用戶端通常必須儲存其他用戶端特定的目的地相關資訊。 路由表管理員可讓用戶端將此資訊儲存在路由表的目的地結構中。 您可以使用不 *透明的指標* 來儲存和取出資訊。 儲存的資訊是私用的，而且只有擁有不透明指標的用戶端才能存取。

例如，多播群組管理員會保留相依于特定目的地的多播轉送專案清單。 多播群組管理員會在該目的地中使用不透明的指標。 在另一個範例中，通告特定目的地的路由通訊協定可以使用不透明的指標，來保存其本身的路由公告相關資訊，即使它沒有最佳路由也一樣。

不透明指標的數目有限;這些指標會先以先服務的基礎來配置給用戶端。 路由器系統管理員必須在路由器設定期間配置正確的指標數目;因此，路由通訊協定和其他用戶端必須記錄其使用不透明指標。
