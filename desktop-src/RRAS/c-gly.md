<dl> <dt>

<span id="_rras_cyclic_redundancy_check_crc__gly"></span><span id="_RRAS_CYCLIC_REDUNDANCY_CHECK_CRC__GLY"></span>**迴圈冗余檢查 (CRC)**
</dt> <dd>

CRC 是與資料區塊相關的數值。 此值所提供的資訊可讓應用程式判斷資料是否已修改。 CRC 類似于總和檢查碼，但視為更強的錯誤檢查機制。

</dd> <dt>

<span id="_rras_compression_control_protocol_ccp__gly"></span><span id="_RRAS_COMPRESSION_CONTROL_PROTOCOL_CCP__GLY"></span>**(CCP) 的壓縮控制通訊協定**
</dt> <dd>

 (CCP 的壓縮控制通訊協定) 可讓兩部透過 PPP 進行通訊的電腦，以協調相容的演算法來傳送和接收已壓縮的 PPP 框架。 在 PPP 連線的網路控制通訊協定階段之前，這兩部電腦可能不會使用 CCP。

如需詳細資訊，請參閱 [RFC 1962](remote-access-request-for-comments.md)。 另請參閱 [**PPP \_ CCP \_ 資訊**](/windows/desktop/api/Mprapi/ns-mprapi-ppp_ccp_info) 和 [**RASPPPCCP**](/previous-versions/windows/desktop/legacy/aa377620(v=vs.85))。

</dd> </dl>
