# <a name="components-of-the-router-architecture"></a>路由器架構的元件

路由器系統管理檔會經常參考路由器的下列元件。

動態介面管理員 (維度) 

所有的路由器管理功能都會通過 DIM。 根據函數的本質而定，DIM 可能會將呼叫傳遞給其中一個路由器管理員。 只處理介面的函式是由 DIM 處理。 如果此函式會影響路由通訊協定，則 DIM 會針對對應至該通訊協定的傳輸呼叫路由器管理員。 例如，如果函式會影響 Open 最短路徑 First (OSPF) protocol，DIM 將會呼叫 IP 路由器管理員，因為 OSPF 是 IP 路由通訊協定。

路由器管理員

每個符合路由器架構的可路由傳輸都有自己的路由器管理員。 目前的路由器管理員存在於 IP 和 IPX 傳輸。 路由器管理員會管理在本機電腦上的介面上執行的路由器通訊協定和其他類型的路由用戶端。

路由通訊協定和其他用戶端

用戶端是在路由器架構架構內運作的服務提供者。 路由通訊協定是路由器支援的一種用戶端。

用戶端是特定可路由傳送的特定： IP 或 IPX。 IP 傳輸的路由通訊協定包括 Open 最短路徑 First (OSPF) 和路由資訊通訊協定 (RIP) 。 IPX 路由通訊協定的範例是 (SAP) 和適用于 IPX 的 RIP 的服務公告通訊協定。 非路由通訊協定的用戶端範例為 IP (NAT) 的網路位址轉譯。

路由器管理員和用戶端之間的介面，會在 [路由通訊協定介面](about-routing-protocol-interface.md)一節中說明。 所有用戶端都符合這個介面。 使用這個介面時，廠商可以執行自己與路由器相容的用戶端。

[介面](interfaces.md)

介面是與外部網路的連接。 每個介面都是由唯一的介面 *索引* 來識別。 介面是邏輯實體;路由器用戶端（例如 NAT 或 OSPF）會以同樣的方式處理所有類型的介面。 不過，在執行方面，介面可代表專用連線 (例如區域網路 (LAN) ) 或非專用的撥號連線 (例如，與廣域網路 (的廣域網路連線的 PPP 連線。

如果是 LAN 介面，介面會對應到電腦中的實際實體裝置，也就是 LAN 介面卡。 在 WAN 介面的案例中，介面會在建立連接時對應至埠。 埠可以是 COM 埠、平行埠或虛擬埠 (用於通道，例如 PPTP 和 L2TP) 。

WAN 介面具有額外的品質，通常只會在建立連線時接收網路位址。 例如，使用 PPP 的 WAN 介面會在連線過程中從遠端對等端接收其網路層位址。 在連接過程中接收網路位址有時稱為 *晚期繫結*。
