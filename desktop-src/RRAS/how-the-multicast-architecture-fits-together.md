# <a name="how-the-multicast-architecture-fits-together"></a>多播架構如何搭配

本節說明範例設定，以及主要元件如何彼此配合。

下圖顯示路由器各元件之間的關聯性。

![windows 2000 路由器元件之間的關聯性](images/mrarch1.png)

多播群組管理員是在以路由器運作的伺服器上執行之 RRAS 服務的一部分。

顯示的路由器有三個多播路由通訊協定， (Protocol 1、Protocol 2、Protocol 3) 在其上執行。 每個通訊協定都可以擁有一或多個介面 (在此圖中，通訊協定1擁有介面1、Protocol 2 擁有介面2，而通訊協定3擁有介面 3) 。 除了 IGMP 之外，每個介面都只能擁有一個路由通訊協定 (，這是) 的特殊案例。

多播群組管理員會在路由器上執行，並在路由通訊協定之間協調群組資訊的散發。

下圖顯示多播架構中兩個路由器之間的關聯性。

![多播架構中的兩個路由器之間的關聯性](images/mrarch2.png)

在上圖中，路由器2會將多播訊息傳送至介面3上的網路2。 路由器1會在介面3接收來自網路2的多播訊息。 在這兩個路由器上，通訊協定3擁有各自的介面3。

下圖顯示從多播來源 (到多播群組之訊息的路徑，) 會到達已加入多播群組的主機。 圖中的路由器會使用與上圖相同的設定;但是，不會顯示介面和通訊協定詳細資料，以便讓圖表保持簡單。

![從多播來源到目的地主機的訊息路徑](images/mrarch3.png)

下列案例描述當主機加入多播群組時所發生的事件。 請參閱上圖，以瞭解各種實體之間的關聯性。

1.  主機1：在網路3上加入多播群組 G。
2.  路由器3透過 IGMP 學習 G。
3.  路由器3上的多播群組管理員會通知路由器3上有 G 的新接收器的通訊協定3。
4.  路由器3上的通訊協定3接著會在路由器1上通知 G 的通訊協定3。
5.  接著，路由器1上的通訊協定3會通知路由器1上的多播群組管理員約 G。
6.  路由器1上的多播群組管理員接著會向通訊協定1和通訊協定2通知 G。
7.  如果通訊協定是設計來這麼做，通訊協定2可能會通知路由器2。

下列案例描述將訊息傳送至多播群組時所發生的事件。 請參閱上圖，以瞭解各種實體之間的關聯性。

1.  網路1上的來源會將訊息傳送至群組 G。
2.  從來源 S 傳送的訊息會先傳送到路由器2，然後使用介面2將它轉送到路由器 1 (因為路由器2已經由通訊協定2告知，接收者會出現下游) 。
3.  路由器1會將訊息轉送到路由器 3 (，因為通訊協定2已通知路由器1，表示接收者存在於下游) 。
4.  路由器3會將訊息轉送到網路3，因此抵達主機1。

如需多播路由通訊協定互動的詳細資訊，請參閱 [RFC 2715](routing-protocols-request-for-comments.md)、多播路由通訊協定的互通性規則。
