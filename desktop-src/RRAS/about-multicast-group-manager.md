# <a name="about-multicast-group-manager"></a>關於多播群組管理員

本檔說明多播群組管理員 (MGM) 技術。

多播可讓主機只將資料傳送到特別要求接收資料的目的地。 如此一來，多播與傳送廣播資料不同，因為廣播資料會傳送至所有主機。

多播會節省網路頻寬，因為多播資料只會由要求資料的主機接收，而資料只會透過任何連結傳送一次。 多播會儲存伺服器頻寬，因為伺服器必須每個網路只傳送一個多播訊息，而不是每個接收者一個單播訊息。 常用的多播應用程式範例包括線上會議和網際網路廣播。

MGM API 可讓開發人員撰寫多播路由通訊協定，以便與執行多播群組管理員的路由器相交互操作。

在路由器上啟用一個以上的多播路由通訊協定時，多播群組管理員會協調所有路由通訊協定之間的作業。 多播群組管理員會在群組成員資格變更時，以及當收到來自新來源或目的地為新群組的多播資料時，通知每個路由通訊協定。

MGM API 提供下列功能：

-   通訊協定註冊
-   群組管理
-   多播轉送專案 (MFE) 列舉
-   多播路由通訊協定的回呼定義

本總覽說明多播架構的元件、用來與多播群組管理員交互操作的用戶端案例，以及使用 MGM API 的程式設計考慮。
