<dl> <dt>

<span id="_rras_best_route_gly"></span><span id="_RRAS_BEST_ROUTE_GLY"></span>**最佳路由**
</dt> <dd>

最佳路由是網路目的地的最佳路由，以指定的準則為基礎。 此概念是以在網路間建立路由所需的特定「成本」為基礎。

要採取的最佳路由是成本最低的路由，以指定的準則為基礎。 此準則可包含跨越的網路數目、跨越 (的網路類型，例如公用或私用) ，或貨幣或頻寬限制。

</dd> <dt>

<span id="_rras_border_gateway_protocol_bgp__gly"></span><span id="_RRAS_BORDER_GATEWAY_PROTOCOL_BGP__GLY"></span>**邊界閘道協定 (BGP)**
</dt> <dd>

BGP 是 [*外部閘道通訊協定*](e-gly.md)。 BGP 會在兩個 [*自主系統*](a-gly.md)之間轉送資訊。 它是在 [rfc 1267](routing-protocols-request-for-comments.md) 和 [rfc 1268](routing-protocols-request-for-comments.md)中定義。

</dd> <dt>

<span id="_mpr_brute_force_attack"></span><span id="_MPR_BRUTE_FORCE_ATTACK"></span>**暴力密碼破解攻擊**
</dt> <dd>

嘗試每次可能都會入侵密碼。

</dd> </dl>
