# <a name="about-monitor-configuration"></a>關於監視設定

監視設定功能的用途包括：

-   色彩校正。 正確校準的監視器會正確地重現色彩。 色彩校正應用程式通常會顯示測試模式，並且具有可變更監視設定的控制項。 使用者會變更設定，直到符合條件為止，然後針對每個監視設定重複此程式，直到校準監視器為止。
-   調整監視器的顯示區域。 您可以使用監視設定功能來變更監視器顯示區域的大小和位置。 例如，當 LCD 監視器使用類比連接器連接到電腦時，會在圖形配接器架構緩衝區中的圖元之間有一對一的對應，以及在 LCD 監視器上的圖元之間取得最佳影像品質。
-   正在儲存和還原監視設定。 有些使用者需要多組監視設定，因為不同的案例需要不同的監視設定。 例如，針對桌面應用程式最理想的監視設定，不是觀看電視的最佳設定。
-   使用廠商特定的監視器功能。 製造商可以使用監視設定功能來建立應用程式，以使用監視的廠商特定功能。

監視設定函數分為高階層級和低層級的函式。 高階函式比低層級函數更容易使用。 若要使用高階函式，您不需要知道顯示資料通道命令介面 (DDC/CI) 的任何相關資訊。 不過，高階函式不會讓您存取監視的廠商特定功能。

低層級函式是 DDC/CI 命令周圍的精簡型包裝函式。 若要使用低層級的函式，您必須熟悉 DDC/CI 標準，也必須使用 [VESA 監視器控制] 命令集 (MCCS) 標準。 一般而言，除非您只需要使用低層級函式中可用的功能，否則您應該使用高階函式。

高階監視設定功能的設計，是為了讓應用程式無法將監視器置於無法使用的狀態。 您可以使用低層級的函式，將 VCP 碼傳送給監視器。 不過，請注意，有些 VCP 碼可以停用重要的功能，或讓監視進入使用者看不到顯示影像的狀態。 如需詳細資訊，請參閱) standard (MCCS 的 [VESA 監視器控制] 命令集。
