# <a name="event-data"></a>事件資料

每個事件都可以有相關聯的事件特定資料。 事件檢視器不會解讀此資料;它只會以結合的十六進位和文字格式顯示額外的資料。 偶數記錄檔中事件大小的最大限制是0x3FFFF 位元組。 因此，請謹慎使用事件特定的資料，只有在您確定對問題進行某些工作時，才會有説明。 例如，許多網路相關的事件包括網路控制區塊 (的 NCB) 作為事件特定資料。

當您使用事件特定的資料時，其描述字串的最後一個部分應該會提供以事件特定資料提供的資訊相關附注。 例如，網路軟體可以提供如下的附注：「 (NCB 是事件資料。 ) 」作為慣例，請在這類備註周圍使用括弧，如下列範例所示。

您也可以使用事件特定資料來儲存應用程式可獨立處理事件檢視器的資訊。 例如，您可以特別為您的事件撰寫檢視器，或撰寫程式來掃描記錄檔並建立包含事件特定資料資訊的報告。
