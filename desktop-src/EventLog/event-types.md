# <a name="event-types"></a>事件類型

有五種類型的事件可以記錄。 這些都有定義完善的一般資料，並可選擇性地包含事件特定的資料。

應用程式會在報告事件時指出事件種類。 每個事件都必須是單一類型。 事件檢視器會在事件記錄檔的清單查看中，針對每個類型顯示不同的圖示。

下表描述事件記錄中使用的五種事件種類。



| 事件類型        | 描述                                                                                                                                                                                                                                                                                              |
|-------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **錯誤**         | 表示重大問題的事件，例如資料遺失或功能遺失。 例如，如果服務無法在啟動時載入，就會記錄錯誤事件。                                                                                                                           |
| **警告**       | 不一定很重要，但可能表示未來可能發生問題的事件。 例如，當磁碟空間不足時，就會記錄警告事件。 如果應用程式可以從沒有遺失功能或資料的事件中復原，通常可以將事件分類為警告事件。     |
| **資訊**   | 描述應用程式、驅動程式或服務之成功操作的事件。 例如，當網路驅動程式成功載入時，可能會需要記錄資訊事件。 請注意，在每次啟動時，桌面應用程式都不適合用來記錄事件。 |
| **稽核成功** | 此事件會記錄成功的已審核安全性存取嘗試。 例如，使用者成功登入系統的嘗試會記錄為成功審核事件。                                                                                                                        |
| **稽核失敗** | 此事件會記錄失敗的已審核安全性存取嘗試。 例如，如果使用者嘗試存取網路磁碟機機而失敗，則會將嘗試記錄為「失敗審核」事件。                                                                                                                   |



 

您可以透過審核安全性事件，然後將專案放在電腦的安全性記錄檔中，來追蹤使用者的選取活動。
