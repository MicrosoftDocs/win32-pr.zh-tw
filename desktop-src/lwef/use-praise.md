# <a name="use-praise"></a>使用讚揚

\[Microsoft Agent 已于 Windows 7 淘汰，在後續的 Windows 版本中可能無法使用。\]

即使讚揚可能是預期，人類仍能回應讚揚，而不是批評。 雖然大部分的軟體介面都致力於避免評估意見反應，但讚揚的好比完全不提供任何評估更有效。 此外，許多設計為中性的介面通常會視為使用者的關鍵，因為它們很少會在正常運作時取得正面的意見反應，並在發生錯誤時收到錯誤訊息。 同樣地，告訴使用者有更好的方法可以執行工作，以隱含方式 criticizes 使用者。

因為字元會建立社交內容，所以在傳統的使用者介面中，謹慎使用讚揚和批評的重要性更高。 雖然有很多人都享受 sycophantic 的行為，但仍可充分利用讚揚的限制。 讚揚在使用者對工作效能較不可靠的情況下特別有效。 另一方面，批評應該謹慎使用。 即使您認為批評是適當的，但請注意，人類會將它關閉，或重新導向回其來源。

不過，請避免自行讚揚，除非它是字元專案中特性的幽默部分。 我們傾向于使用 skepticism 來判斷自我 conceit。 如果您想要將 approbation 導向字元，請從另一個來源（例如其他字元或描述性說明）將它導向。
