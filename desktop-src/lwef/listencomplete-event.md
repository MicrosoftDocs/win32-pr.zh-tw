# <a name="listencomplete-event"></a>ListenComplete 事件

\[Microsoft Agent 已于 Windows 7 淘汰，在後續版本的 Windows 中可能無法使用。\]

<dl> <dt>

<span id="Description"></span><span id="description"></span><span id="DESCRIPTION"></span>**描述**
</dt> <dd>

當接聽模式 (語音辨識) 結束時發生。

</dd> <dt>

<span id="Syntax"></span><span id="syntax"></span><span id="SYNTAX"></span>**語法**
</dt> <dd>

**子***代理程式。 * * * ListenComplete (byval* *  *CharacterID*， **byval** *原因 * * * )**



| 部分          | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|---------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| *CharacterID* | 以字串形式傳回接聽字元的識別碼。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| *原因*       | 以整數形式傳回完整事件的原因，可能是下列其中一項：已由程式碼關閉1個接聽模式。<br/> 程式碼) 超時， (開啟了2個接聽模式。<br/>  (由接聽鍵開啟3個接聽模式) 超時。<br/> 因為使用者已釋放接聽金鑰，所以已關閉4個接聽模式。<br/> 5個接聽模式已結束，因為使用者已完成說話。<br/> 6個接聽模式已結束，因為輸入-主動用戶端已停用。<br/> 7接聽模式已結束，因為預設字元已變更。<br/> 8接聽模式已結束，因為使用者已停用語音輸入。<br/> |



 

</dd> </dl>

### <a name="remarks"></a>備註

此事件會在接聽模式超時結束、使用者放開接聽金鑰之後、輸入作用中用戶端以 **False** 呼叫 [**接聽**](listen-method.md)方法，或使用者已完成說話時，傳送至所有用戶端。 您可以使用此事件來判斷何時繼續 (音訊) 輸出的字元。

如果您使用 [**接聽**](listen-method.md) 方法開啟接聽模式，然後使用者按下接聽鍵，則接聽模式會重設並繼續進行，直到接聽的金鑰超時時間完成、接聽的金鑰已釋出，或使用者已完成說話（以較晚者為准）。 在此情況下，您將不會收到 **ListenComplete** 事件，直到接聽鍵的模式完成為止。

此事件會將字元傳回給目前已載入此字元的用戶端。 所有其他用戶端都會收到 null 字元， (空的字串) 。
