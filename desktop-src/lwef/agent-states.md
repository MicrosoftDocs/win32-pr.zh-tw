# <a name="agent-states"></a>代理程式狀態

\[Microsoft Agent 已于 Windows 7 淘汰，在後續的 Windows 版本中可能無法使用。\]

Microsoft Agent 動畫服務會為您自動播放特定動畫。 例如，當您使用 [**MoveTo**](moveto-method.md) 或 [**GestureAt**](gestureat-method.md) 命令時，動畫服務會播放適當的動畫。 同樣地，在閒置時間過期之後，服務會自動播放動畫。 若要支援這些狀態，您可以定義適當的動畫，然後將它們指派給這些狀態。 即使您將它指派給狀態，還是可以使用 [**play**](play-method.md) 方法直接播放您定義的任何動畫。

您可以將多個動畫指派給相同的狀態，而且動畫服務會隨機播放其中一個動畫。 這可讓您的字元在其行為上表現出更自然的不同。

雖然您指派給狀態的動畫可包含分支框架，但請避免將動畫迴圈 (不會) 分支的動畫。 否則，您將必須使用 [**Stop**](play-method.md) 方法，才能播放另一個動畫。

針對字元所發生的每個狀態，定義並指派至少一個動畫是很重要的。 如果您未提供這些動畫和狀態指派，則您的字元可能不會與使用者有適當的行為。 但是，如果特定字元沒有發生狀態，您就不需要指派動畫到該狀態。 例如，如果您的主應用程式永遠不會呼叫 [**MoveTo**](moveto-method.md) 方法，您可以略過建立和指派 **移動** 狀態動畫。



| 狀態              | 使用範例                                                                         |
|--------------------|----------------------------------------------------------------------------------------|
| **GesturingDown**  | 處理 [**GestureAt**](gestureat-method.md) 動畫方法時。          |
| **GesturingLeft**  | 處理 [**GestureAt**](gestureat-method.md) 動畫方法時。          |
| **GesturingRight** | 處理 [**GestureAt**](gestureat-method.md) 動畫方法時。          |
| **GesturingUp**    | 處理 [**GestureAt**](gestureat-method.md) 動畫方法時。          |
| **聽覺**        | 偵測到說話輸入的開頭時。                                        |
| **隱藏**         | 當使用者或應用程式隱藏字元時。                                  |
| **IdlingLevel1**   | 當字元開始 **閒置** 狀態時。                                        |
| **IdlingLevel2**   | 當字元開始第二個 **閒置** 層級狀態時。                           |
| **IdlingLevel3**   | 當字元開始最終的 **閒置** 層級狀態時。                            |
| **聽**      | 當字元開始接聽 (使用者第一次按下語音輸入快速鍵) 。 |
| **MovingDown**     | 處理 [**MoveTo**](moveto-method.md) 動畫方法時。                |
| **MovingLeft**     | 處理 [**MoveTo**](moveto-method.md) 動畫方法時。                |
| **MovingRight**    | 處理 [**MoveTo**](moveto-method.md) 動畫方法時。                |
| **MovingUp**       | 處理 [**MoveTo**](moveto-method.md) 動畫方法時。                |
| **正在顯示**        | 當使用者或應用程式顯示字元時。                                  |
| **說**       | 處理 [**說話**](speak-method.md) 動畫方法時。                  |



 

### <a name="the-hearing-and-listening-states"></a>聽力和聆聽州

當使用者按下語音輸入的推播對話快速鍵時，您指派給 **接聽** 狀態的動畫會播放。 建立並指派會讓字元看起來用心的簡短動畫。 同樣地，將 **其傳回** 動畫定義為有短暫的持續時間，讓字元在使用者說話時播放其 **聽覺** 狀態動畫。 **聽力** 狀態動畫也應該是短暫的，而且是設計來讓使用者知道字元正在主動接聽使用者所說的內容。 標頭傾斜或其他輕微手勢都適用。 若要提供自然的變化，請提供數個 **聽力** 狀態動畫。

### <a name="the-gesturing-states"></a>Gesturing 狀態

只有當您打算使用 [**GestureAt**](gestureat-method.md)方法時，才需要建立和指派 **Gesturing** 狀態動畫。 當 Microsoft Agent 處理 **GestureAt** 方法的呼叫時，會播放 **Gesturing** 狀態動畫。 如果您定義 **Gesturing** 狀態動畫的嘴重迭，則字元可以說為手勢。

動畫服務會判斷字元的位置，以及該方法中所指定座標位置的關聯性，並播放適當的動畫。 Gesturing 方向一律與字元相關;例如， **GestureRight** 應該是字元右邊的手勢。

### <a name="the-showing-and-hiding-states"></a>顯示和隱藏狀態

當使用者或主應用程式要求顯示或隱藏字元時， **顯示** 和 **隱藏** 狀態會播放指派的動畫。 這些狀態也會適當地設定字元框架的 **可見** 狀態。 定義這些狀態的動畫時，請記住，字元可能會在任何螢幕位置出現或偏離。 由於使用者可以顯示或隱藏任何字元，因此一律針對這些狀態至少支援一個動畫。

您指派給 **顯示** 狀態的動畫通常會以包含字元中性位置影像的框架作為結尾。 相反地， **隱藏** 狀態動畫通常是從中性位置開始。 **顯示** 和 **隱藏** 狀態動畫可分別在開頭或結尾包含空的框架，以提供來自字元目前狀態的轉換。

### <a name="the-idling-states"></a>閒置狀態

**閒置** 狀態為漸進式。 動畫服務會開始使用第一次閒置期間的層級1指派，並針對第二個動畫使用層級2動畫。 在此之後，閒置週期會前進至層級3指派的動畫，並維持在此狀態中，直到取消為止，例如當新的動畫要求開始時。

設計 **閒置** 狀態的動畫以傳達字元的狀態，但不會干擾使用者。 動畫應以微妙但清楚的方式適當地反映字元的回應性。 例如，glancing 四處或閃爍是指派給 **IdlingLevel1** 狀態的好動畫。 讀取動畫適用于 **IdlingLevel2** 狀態。 使用耳機睡眠或聆聽音樂，是將動畫指派給 **IdlingLevel3** 狀態的絕佳範例。 包含許多或大型移動的動畫並不適合閒置動畫，因為它們會繪製使用者的注意力。 因為 **閒置** 狀態動畫經常播放，所以請提供數個 **閒置** 狀態動畫，特別是針對 **IdlingLevel1** 和 **IdlingLevel2** 狀態。

請注意，應用程式可以關閉字元的自動閒置處理，並管理字元的 **閒置** 狀態本身。 代理程式 **閒置** 狀態是設計用來協助您避免字元沒有播放動畫的任何情況。 在一小段時間之後不會變更的字元映射，就像是一段長時間顯示等候指標的應用程式，會使從 believability 和互動的意義。 維持假像不需要太多：有時候只有動畫閃爍、可見吸一口氣或內文轉換。

### <a name="the-speaking-state"></a>說話狀態

當找不到目前動畫的說話動畫時，動畫服務會使用 **說話** 狀態。 將簡單的說話動畫指派給此狀態。 例如，您可以使用單一框架，其中包含字元的中性位置和嘴重迭。

### <a name="the-moving-states"></a>移動狀態

**移動** 狀態會在應用程式呼叫 [**MoveTo**](moveto-method.md)方法時播放。 動畫服務會根據字元目前的位置和指定的座標來決定要播放的動畫。 移動方向是以字元的位置為基礎。 因此，您指派給 **MovingLeft** 動畫的動畫應以字元左邊為基礎。 如果您未使用 **MoveTo** 方法，可以略過建立和指派動畫。

**移動** 狀態動畫應以動畫顯示字元到其移動位置。 這個動畫的最後一個畫面格會隨著字元的畫面格移動而顯示。 不支援在框架移動時製作字元動畫。

### <a name="standard-animation-set"></a>標準動畫組

雖然您可以設計自訂字元來取得您想要使用的動畫，但 Microsoft Agent 會定義標準的動畫集。 您可以選取符合此定義的字元做為預設字元。
