# <a name="the-listening-tip"></a>接聽秘訣

\[Microsoft Agent 已于 Windows 7 淘汰，在後續的 Windows 版本中可能無法使用。\]

接聽秘訣是 Microsoft 代理程式所提供的另一個語音輸入服務。 當您安裝語音輸入時，Agent 會包含特殊的工具提示視窗，當使用者按下接聽的熱鍵或呼叫接聽方法時，就會出現此視窗。 只有當語音服務可供使用時，才會顯示接聽提示。 如果沒有任何用戶端撰寫語音命令或成功載入語音引擎，則不會出現接聽提示。 此外，您必須啟用 [Advanced Character] 選項中的 [語音輸入] 和 [顯示接聽秘訣] 選項，提示才會出現。

接聽提示會在呈現之後自動發生。 當使用者仍然按住快速鍵時，如果「聽到」文字超時時間完成，則提示會還原為「正在接聽」文字，除非伺服器收到另一個相符的語句。 在此情況下，提示會顯示新的「赫德」文字，並開始該提示文字的超時時間。 如果使用者釋出熱鍵，而伺服器顯示「聽到」的文字，則會繼續進行，並在逾時間隔到期時隱藏接聽提示視窗。

如果伺服器尚未嘗試載入語音辨識引擎，將不會顯示接聽提示。 同樣地，如果使用者已停用 [先進字元選項] 中的接聽提示或停用語音輸入，則不會顯示接聽提示。

當指標停留在字元的工作列圖示上時，不會出現接聽提示。 相反地，標準通知提示視窗會隨即出現，並顯示字元的名稱。

用戶端應用程式無法直接寫入接聽提示，但您可以指定伺服器在符合語音命令的辨識時所顯示的替代文字。 若要這樣做，請設定此命令的 [**信賴**](confidence-property.md) 屬性和新的 [**ConfidenceText**](confidencetext-property.md)屬性。 如果讀出輸入符合命令，但是最符合項不超過信賴設定，則伺服器會使用提示視窗中 **ConfidenceText** 屬性的文字集。 如果用戶端未提供此值，伺服器會顯示符合的文字 (文法) 。

接聽提示文字會根據輸入-使用中用戶端的字元語言識別項設定出現在語言中，不論是否有可用的語言相容語音辨識引擎。
