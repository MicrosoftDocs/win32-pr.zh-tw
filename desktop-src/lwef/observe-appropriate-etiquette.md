# <a name="observe-appropriate-etiquette"></a>觀察適當的禮儀

\[Microsoft Agent 已于 Windows 7 淘汰，在後續版本的 Windows 中可能無法使用。\]

所有人都瞭解社交禮儀的規則。 此外，我們會根據這些規則符合這些規則的程度來判斷其他人。 您介面中的字元存在，可讓禮儀規則直接適用于您的設計。

我們預期互動中的 reciprocity，而不會發生這種情況時，我們通常會將行為視為 incompetent 或冒犯性。 在新的或正式的情況下，通常會需要禮貌程度。 只有在建立關聯性之後，才有熟悉的互動。 因此，當您不確定時，請按禮貌。

例如，請考慮使用適當的通訊協定來開始和結束社交互動。 當您參與新的社交內容時，我們通常會彼此歡迎，並指出我們要在我們偏離之前保留的意圖。 您可以直接將此模型套用至字元互動;避免出現或消失的字元，而不會有適當的指示。

禮貌程度的一部分是我們的回應方式。 我們預期會有適當的回應。 例如，如果您問：「這是哪一天？」 您不希望回應「10蘋果」。 不過，精確度和相關性不足;您可能也需要判斷內容。 精確的回應可能是「星期一」或「8月<sup>25 日</sup>」。 同樣地，回應的用語可以是正確的，但仍會有冒犯性，取決於聲調、臉部運算式或其他非口頭行為。

此外，我們會在討論時互相解決：我們面對的是，只是要將注意力導向到東西。 重新開啟一回，而不需要某些用途，意指 disinterest，通常視為 impolite。

禮貌程度也會影響我們回應或察覺他人的方式。 例如，「社交研究」指出當其他人詢問他人的要求時，我們比較有可能會對其他人提出評價。 同樣地，我們會比正面的自我評估更容易接受其他人的精確評估。 Immodest 行為通常視為 impolite。 因此，除非刻意嘗試升級 impolite 的特質樣式，否則請避免字元補充其本身的效能。

也請考慮文化對社交禮儀的影響。 這可能適用于字元的服裝、手勢或單字選擇。 雖然有禮貌的行為可能因文化特性而異，但禮貌程度的概念是跨文化。
