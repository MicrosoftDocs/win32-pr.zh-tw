# <a name="speech-recognition"></a>語音辨識

\[Microsoft Agent 已于 Windows 7 淘汰，在後續的 Windows 版本中可能無法使用。\]

語音辨識提供非常自然且熟悉的介面來與字元互動。 不過，語音輸入也會帶來許多挑戰。 語音引擎目前沒有大量的人類語音通訊所有產品，例如手勢、聲調和臉部表情。 此外，自然語音通常是未系結的。 說話者很容易就能超過引擎目前的詞彙或 *文法*。 同樣地，任何給定的要求或回應都可能會有不同的用語或單字順序。 此外，語音辨識引擎通常必須處理喇叭環境中的大型變化。 例如，背景雜訊、麥克風品質和位置可能會影響輸入品質。 同樣地，不同的說話者發音或甚至是相同的喇叭變化，例如當喇叭有冷時，將聲場資料轉換成具像的理解是一項挑戰。 最後，語音引擎也必須處理類似發音的單字或片語，例如「新增」、「知道」和「gnu」、「wreck 優秀的海灘」和「辨識語音」。

語音不一定是工作輸入的最佳形式。 由於語音的自然本質，通常會比其他形式的輸入更慢。 就像鍵盤一樣，除非提供某種類型的助憶鍵標記法，否則語音輸入是不佳的指標介面。 因此，請一律考慮語音是否為工作的最合適輸入。 最好避免使用語音作為任何工作的專屬介面。 提供使用滑鼠或鍵盤等方法存取任何基本功能的其他方式。 此外，藉由將語音輸入與可協助指定內容和選項的視覺資訊結合，來利用在視覺化介面中使用語音的多重模式本質。

最後，成功使用語音輸入只是因為技術的品質所致。 即使是人類辨識（超過任何目前的辨識技術），有時也會失敗。 不過，在「人為通訊」中，我們會使用可改善成功機率的策略，並在發生錯誤時提供錯誤修復。 因此，語音輸入的效果也取決於呈現它的使用者介面品質。

設計更自然的語音介面時，研究語音互動的人類模型可能會很有用。 錄製實際的人類語音對話以進行特定案例，可協助您進一步瞭解所使用的結構和模式，以及有效形式的意見反應和錯誤復原。 它可協助您判斷適當的詞彙，以針對輸入和輸出) 使用 (。 最好是根據人們實際說話的方式來設計語音介面，而不只是從其運作所在的圖形介面來衍生。

請注意，Microsoft Agent 會使用 Microsoft Speech API (SAPI) 來支援語音辨識。 這可讓 Microsoft Agent 搭配各種相容引擎使用。 雖然 Microsoft Agent 指定了特定的基本介面，但引擎的效能需求和品質可能會有所不同。

語音不是支援對話式介面的唯一方法。 您也可以使用鍵盤輸入的自然語言處理來取代語音。 在這些情況下，您仍然可以套用語音輸入的指導方針。
