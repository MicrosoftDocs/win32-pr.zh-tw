# <a name="be-non-exclusive"></a>非獨佔

\[Microsoft Agent 已于 Windows 7 淘汰，在後續的 Windows 版本中可能無法使用。\]

您可以在使用者介面中採用互動式字元作為助理、指南、entertainers、storytellers、銷售專員或各種其他角色。 自動執行或協助的字元不應設計成與保持使用者控制的設計原則相反。 將字元新增至網站或傳統應用程式的介面時，請使用字元作為增強功能，而不是主要介面的取代。 避免執行只需要一個字元的任何功能或作業。

同樣地，讓使用者選擇何時要與您的字元互動。 使用者應該要能夠關閉該字元，而且只會傳回使用者的許可權。 在使用者上強制字元互動可能會有嚴重的負面影響。 為了支援使用者控制字元互動，Microsoft Agent 會自動加入隱藏和顯示命令。 Microsoft Agent API 也支援這些方法，因此您可以在自己的介面中包含這些函式的支援。 此外，Microsoft 代理程式的使用者介面包含全域屬性，可讓使用者覆寫特定字元輸出選項。 為了確保維護使用者的喜好設定，無法透過 API 覆寫這些屬性。
