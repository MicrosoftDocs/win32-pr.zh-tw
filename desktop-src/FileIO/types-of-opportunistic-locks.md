# <a name="types-of-opportunistic-locks"></a><span data-ttu-id="5fb94-103">隨機鎖定的類型</span><span class="sxs-lookup"><span data-stu-id="5fb94-103">Types of Opportunistic Locks</span></span>

<span data-ttu-id="5fb94-104">隨機鎖定作業適用于四種類型的隨機鎖定：層級1、層級2、批次和篩選器。</span><span class="sxs-lookup"><span data-stu-id="5fb94-104">The opportunistic lock operations work with four types of opportunistic locks: level 1, level 2, batch, and filter.</span></span> <span data-ttu-id="5fb94-105">*專屬* 的隨機鎖定是層級1、批次和篩選鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-105">*Exclusive opportunistic locks* are level 1, batch, and filter locks.</span></span> <span data-ttu-id="5fb94-106">如果執行緒在檔案上具有任何獨佔鎖定類型，則在相同檔案上也不能有層級2鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-106">If a thread has any type of exclusive lock on a file, it cannot also have a level 2 lock on the same file.</span></span>

## <a name="level-1-opportunistic-locks"></a><span data-ttu-id="5fb94-107">層級1隨機鎖定</span><span class="sxs-lookup"><span data-stu-id="5fb94-107">Level 1 Opportunistic Locks</span></span>

<span data-ttu-id="5fb94-108">檔案層級1的隨機鎖定可讓用戶端預先讀取檔案，並快取讀取和寫入本機檔案中的資料。</span><span class="sxs-lookup"><span data-stu-id="5fb94-108">A level 1 opportunistic lock on a file allows a client to read ahead in the file and cache both read-ahead and write data from the file locally.</span></span> <span data-ttu-id="5fb94-109">只要用戶端具有檔案的唯一存取權，就不會有提供層級1隨機鎖定的資料一致性。</span><span class="sxs-lookup"><span data-stu-id="5fb94-109">As long as the client has sole access to a file, there is no danger to data coherency in providing a level 1 opportunistic lock.</span></span>

<span data-ttu-id="5fb94-110">用戶端可以在開啟檔案之後，要求層級1的隨機鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-110">The client can request a level 1 opportunistic lock after opening a file.</span></span> <span data-ttu-id="5fb94-111">如果沒有其他用戶端 (或相同用戶端上沒有其他執行緒) 開啟該檔案，則伺服器可能會授與隨機鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-111">If no other client (or no other thread on the same client) has the file open, the server may grant the opportunistic lock.</span></span> <span data-ttu-id="5fb94-112">然後，用戶端就可以在本機快取檔案的讀取和寫入資料。</span><span class="sxs-lookup"><span data-stu-id="5fb94-112">The client can then cache read and write data from the file locally.</span></span> <span data-ttu-id="5fb94-113">如果有其他用戶端開啟檔案，則伺服器會拒絕隨機鎖定，且用戶端不會進行本機快取。</span><span class="sxs-lookup"><span data-stu-id="5fb94-113">If another client has opened the file, then the server refuses the opportunistic lock and the client does no local caching.</span></span> <span data-ttu-id="5fb94-114"> (伺服器也可能因為其他原因（例如不支援隨機鎖定）而拒絕無機會鎖定。 ) </span><span class="sxs-lookup"><span data-stu-id="5fb94-114">(The server may refuse the opportunistic lock for other reasons as well, such as not supporting opportunistic locks.)</span></span>

<span data-ttu-id="5fb94-115">當伺服器開啟已經有層級1隨機鎖定的檔案時，伺服器會先檢查檔案的共用狀態，再中斷層級1的隨機鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-115">When the server opens a file that already has a level 1 opportunistic lock on it, the server examines the sharing state of the file before it breaks the level 1 opportunistic lock.</span></span> <span data-ttu-id="5fb94-116">如果伺服器在這項檢查之後中斷鎖定，則擁有先前鎖定的用戶端花在清除其寫入快取的時間，就是要求檔案的用戶端必須等待的時間。</span><span class="sxs-lookup"><span data-stu-id="5fb94-116">If the server breaks the lock after this examination, the time the client with the former lock spends flushing its write cache is time the client requesting the file must wait.</span></span> <span data-ttu-id="5fb94-117">這段時間的支出表示，在許多用戶端開啟的檔案上，應避免層級1的隨機鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-117">This time expenditure means that level 1 opportunistic locks should be avoided on files that many clients open.</span></span>

<span data-ttu-id="5fb94-118">不過，因為伺服器會在中斷鎖定之前檢查共用狀態，所以伺服器會因為共用衝突而拒絕開啟的要求，所以伺服器不會中斷鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-118">However, because the server checks the sharing state before it breaks the lock, in the case where the server would deny an open request due to a sharing conflict the server does not break the lock.</span></span> <span data-ttu-id="5fb94-119">例如，如果您已開啟檔案、拒絕寫入作業的共用，並取得層級1鎖定，則伺服器會拒絕其他用戶端開啟檔案以進行寫入的要求，然後再檢查檔案的鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-119">For example, if you have opened a file, denied sharing for write operations, and obtained a level 1 lock, the server denies another client's request to open the file for writing before it even examines your lock on the file.</span></span> <span data-ttu-id="5fb94-120">在此情況下，您的隨機鎖定不會中斷。</span><span class="sxs-lookup"><span data-stu-id="5fb94-120">In this instance, your opportunistic lock is not broken.</span></span>

<span data-ttu-id="5fb94-121">如需層級1隨機鎖定運作方式的範例，請參閱層級1隨機鎖定範例。</span><span class="sxs-lookup"><span data-stu-id="5fb94-121">For an example of how a level 1 opportunistic lock works, see Level 1 Opportunistic Lock Example.</span></span> <span data-ttu-id="5fb94-122">如需有關中斷隨機鎖定的詳細資訊，請參閱 [中斷隨機鎖定](breaking-opportunistic-locks.md)。</span><span class="sxs-lookup"><span data-stu-id="5fb94-122">For more information on breaking opportunistic locks, see [Breaking Opportunistic Locks](breaking-opportunistic-locks.md).</span></span>

## <a name="level-2-opportunistic-locks"></a><span data-ttu-id="5fb94-123">層級2隨機鎖定</span><span class="sxs-lookup"><span data-stu-id="5fb94-123">Level 2 Opportunistic Locks</span></span>

<span data-ttu-id="5fb94-124">層級2隨機鎖定會通知用戶端有多個並行的檔案用戶端，而且沒有修改。</span><span class="sxs-lookup"><span data-stu-id="5fb94-124">A level 2 opportunistic lock informs a client that there are multiple concurrent clients of a file and that none has yet modified it.</span></span> <span data-ttu-id="5fb94-125">此鎖定可讓用戶端使用快取或預先讀取的本機資訊來執行讀取作業並取得檔案屬性，但用戶端必須將所有其他要求傳送 (例如) 至伺服器的寫入作業。</span><span class="sxs-lookup"><span data-stu-id="5fb94-125">This lock allows the client to perform read operations and obtain file attributes using cached or read-ahead local information, but the client must send all other requests (such as for write operations) to the server.</span></span> <span data-ttu-id="5fb94-126">當您預期其他應用程式會隨機寫入檔案，或以隨機或順序讀取檔案時，您的應用程式應該使用層級2的隨機鎖定。</span><span class="sxs-lookup"><span data-stu-id="5fb94-126">Your application should use the level 2 opportunistic lock when you expect other applications to write to the file at random or read the file at random or sequentially.</span></span>

<span data-ttu-id="5fb94-131">如需有關中斷隨機鎖定的詳細資訊，請參閱 [中斷隨機鎖定](breaking-opportunistic-locks.md)。</span><span class="sxs-lookup"><span data-stu-id="5fb94-131">For more information on breaking opportunistic locks, see [Breaking Opportunistic Locks](breaking-opportunistic-locks.md).</span></span>

## <a name="filter-opportunistic-locks"></a><span data-ttu-id="5fb94-144">篩選隨機鎖定</span><span class="sxs-lookup"><span data-stu-id="5fb94-144">Filter Opportunistic Locks</span></span>
