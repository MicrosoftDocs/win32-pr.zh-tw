# <a name="master-file-table-local-file-systems"></a><span data-ttu-id="17f66-103">Master File Table (本機檔案系統) </span><span class="sxs-lookup"><span data-stu-id="17f66-103">Master File Table (Local File Systems)</span></span>

<span data-ttu-id="17f66-104">NTFS 檔案系統包含稱為 *主要檔案資料表* 或 MFT 的檔案。</span><span class="sxs-lookup"><span data-stu-id="17f66-104">The NTFS file system contains a file called the *master file table*, or MFT.</span></span> <span data-ttu-id="17f66-105">針對 NTFS 檔案系統磁片區上的每個檔案，在 MFT 中至少有一個專案（包括 MFT 本身）。</span><span class="sxs-lookup"><span data-stu-id="17f66-105">There is at least one entry in the MFT for every file on an NTFS file system volume, including the MFT itself.</span></span> <span data-ttu-id="17f66-106">檔案的所有相關資訊，包括其大小、時間和日期戳記、許可權和資料內容，都會儲存在 MFT 專案中，或儲存在 mft 專案所描述之 MFT 以外的空間中。</span><span class="sxs-lookup"><span data-stu-id="17f66-106">All information about a file, including its size, time and date stamps, permissions, and data content, is stored either in MFT entries, or in space outside the MFT that is described by MFT entries.</span></span>

<span data-ttu-id="17f66-107">當檔案新增至 NTFS 檔案系統磁片區時，會將多個專案新增至 MFT，而 MFT 的大小會增加。</span><span class="sxs-lookup"><span data-stu-id="17f66-107">As files are added to an NTFS file system volume, more entries are added to the MFT and the MFT increases in size.</span></span> <span data-ttu-id="17f66-108">從 NTFS 檔案系統磁片區刪除檔案時，它們的 MFT 專案會標示為可用，而且可能會重複使用。</span><span class="sxs-lookup"><span data-stu-id="17f66-108">When files are deleted from an NTFS file system volume, their MFT entries are marked as free and may be reused.</span></span> <span data-ttu-id="17f66-109">不過，已配置給這些專案的磁碟空間不會重新配置，且不會減少 MFT 的大小。</span><span class="sxs-lookup"><span data-stu-id="17f66-109">However, disk space that has been allocated for these entries is not reallocated, and the size of the MFT does not decrease.</span></span>

<span data-ttu-id="17f66-110">NTFS 檔案系統會保留此 MFT 的空間，以便在該 MFT 成長時盡可能保持連續。</span><span class="sxs-lookup"><span data-stu-id="17f66-110">The NTFS file system reserves space for the MFT to keep the MFT as contiguous as possible as it grows.</span></span> <span data-ttu-id="17f66-111">每個磁片區中的 MFT NTFS 檔案系統所保留的空間稱為「MFT 區域」。</span><span class="sxs-lookup"><span data-stu-id="17f66-111">The space reserved by the NTFS file system for the MFT in each volume is called the MFT zone.</span></span> <span data-ttu-id="17f66-112">檔案和目錄的空間也會從此空間配置，但只有在已配置了 MFT 區域以外的所有磁片區空間之後。</span><span class="sxs-lookup"><span data-stu-id="17f66-112">Space for file and directories are also allocated from this space, but only after all of the volume space outside of the MFT zone has been allocated.</span></span>

<span data-ttu-id="17f66-113">根據平均檔案大小和其他變數，磁片上的保留的 MFT 區域或未保留的空間可能會先配置，因為磁片會填滿容量。</span><span class="sxs-lookup"><span data-stu-id="17f66-113">Depending on the average file size and other variables, either the reserved MFT zone or the unreserved space on the disk may be allocated first as the disk fills to capacity.</span></span> <span data-ttu-id="17f66-114">具有少量相對較大檔案的磁片區會先配置未保留的空間，而具有大量較小檔案的磁片區則會先配置 MFT 區域。</span><span class="sxs-lookup"><span data-stu-id="17f66-114">Volumes with a small number of relatively large files will allocate the unreserved space first, while volumes with a large number of relatively small files allocate the MFT zone first.</span></span> <span data-ttu-id="17f66-115">無論是哪一種情況，當一個區域或另一個區域都已完全配置時，MFT 的片段都會開始發生。</span><span class="sxs-lookup"><span data-stu-id="17f66-115">In either case, fragmentation of the MFT starts to take place when one region or the other becomes fully allocated.</span></span> <span data-ttu-id="17f66-116">如果未配置的空間已完全配置，則會從 MFT 區域配置使用者檔案和目錄的空間。</span><span class="sxs-lookup"><span data-stu-id="17f66-116">If the unreserved space is completely allocated, space for user files and directories will be allocated from the MFT zone.</span></span> <span data-ttu-id="17f66-117">如果已完全配置 MFT 區域，則會從未保留的空間配置新的 MFT 專案的空間。</span><span class="sxs-lookup"><span data-stu-id="17f66-117">If the MFT zone is completely allocated, space for new MFT entries will be allocated from the unreserved space.</span></span>

<span data-ttu-id="17f66-118">MFT 本身可以進行磁碟重組。</span><span class="sxs-lookup"><span data-stu-id="17f66-118">The MFT itself can be defragmented.</span></span> <span data-ttu-id="17f66-119">若要減少在重組程式完成之前，將 MFT 區域完全配置的機率，請在將磁片區重組之前，盡可能在該 MFT 區域開頭保留更多空間。</span><span class="sxs-lookup"><span data-stu-id="17f66-119">To reduce the chance of the MFT zone becoming fully allocated before the defragmentation process is complete, leave as much space at the beginning of the MFT zone as possible before defragmenting the volume.</span></span> <span data-ttu-id="17f66-120">如果在磁碟重組完成之前，已完全配置了 MFT 區域，則在 MFT 區域以外必須有未配置的空間。</span><span class="sxs-lookup"><span data-stu-id="17f66-120">If the MFT zone becomes fully allocated before defragmentation has completed, there must be unallocated space outside of the MFT zone.</span></span>

<span data-ttu-id="17f66-121">預設的 MFT 區域會在掛接磁片區時計算並保留，且是以磁片區大小為基礎。</span><span class="sxs-lookup"><span data-stu-id="17f66-121">The default MFT zone is calculated and reserved by the system when it mounts the volume, and is based on volume size.</span></span> <span data-ttu-id="17f66-122">您可以透過 [Microsoft 知識庫文章 174619](https://support.microsoft.com/kb/174619)中詳細說明的登錄專案來增加 MFT 區域，但您無法讓預設的 MFT 區域小於計算的專案。</span><span class="sxs-lookup"><span data-stu-id="17f66-122">You can increase the MFT zone by means of the registry entry detailed in [Microsoft Knowledge Base Article 174619](https://support.microsoft.com/kb/174619), but you cannot make the default MFT zone smaller than what is calculated.</span></span> <span data-ttu-id="17f66-123">增加 MFT 區域不會減少使用者可用於資料檔案的磁碟空間。</span><span class="sxs-lookup"><span data-stu-id="17f66-123">Increasing the MFT zone does not decrease the disk space that users can use for data files.</span></span>

<span data-ttu-id="17f66-124">若要判斷目前的 MFT 大小，請使用「磁碟重組工具」來分析 NTFS 檔案系統磁片磁碟機，然後按一下 [ **視圖報告** ] 按鈕。</span><span class="sxs-lookup"><span data-stu-id="17f66-124">To determine the current size of the MFT, analyze the NTFS file system drive with Disk Defragmenter, then click the **View Report** button.</span></span> <span data-ttu-id="17f66-125">將會顯示磁片磁碟機統計資料，包括目前的 MFT 大小和片段數目。</span><span class="sxs-lookup"><span data-stu-id="17f66-125">The drive statistics will be displayed, including the current MFT size, and number of fragments.</span></span> <span data-ttu-id="17f66-126">您也可以使用 [**FSCTL \_ 取得 \_ NTFS \_ 磁片區 \_ 資料**](/windows/win32/api/winioctl/ni-winioctl-fsctl_get_ntfs_volume_data) 控制程式代碼來取得 MFT 的大小。</span><span class="sxs-lookup"><span data-stu-id="17f66-126">You can also obtain the size of the MFT by using the [**FSCTL\_GET\_NTFS\_VOLUME\_DATA**](/windows/win32/api/winioctl/ni-winioctl-fsctl_get_ntfs_volume_data) control code.</span></span>
