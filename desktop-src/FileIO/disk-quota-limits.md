# <a name="disk-quota-limits"></a>磁片配額限制

每個檔案使用的磁碟空間會直接向擁有檔案的使用者收取費用。 檔案的擁有者是以檔案安全性資訊中的安全識別碼 (SID) 來識別。 對使用者收費的總磁碟空間是所有資料流程長度的總和。 換句話說，屬性集資料流程和常駐使用者資料流程會影響使用者的配額。

重新剖析點、安全描述項或其他與檔案相關聯的中繼資料，不需支付配額。 壓縮或解壓縮檔案不會影響針對檔案所報告的磁碟空間。 因此，一個磁片區上的配額設定可以與另一個磁片區上的設定進行比較。

磁片配額限制有兩種類型，如下表所示。



| 限制                        | 描述                                                                                                                                                                                                                                                                    |
|------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 警告臨界值<br/> | 您可以設定系統在向使用者收取的磁碟空間超過此值時產生系統記錄檔專案。<br/>                                                                                                                                         |
| 固定配額<br/>        | 您可以設定系統在向使用者收取的磁碟空間超過此值時產生系統記錄檔專案。 您也可以設定系統在使用者向使用者收取的磁碟空間超過此值時，拒絕額外的磁碟空間給使用者。<br/> |



 

當使用者第一次寫入磁片區時，NTFS 檔案系統會自動建立使用者配額專案。 系統會將自動建立的專案指派給磁片區的預設警告臨界值和固定配額限制值。
