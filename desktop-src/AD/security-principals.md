# <a name="security-principals"></a>安全性主體

在 Windows 2000 中，安全性主體是一個使用者、群組或電腦，也就是安全性系統可辨識的實體。 這包括人類使用者和自發流程。 嚴格來說，安全性系統無法分辨登入的使用者與電腦上執行的處理常式之間的差異。 它會同時以安全性主體名稱的形式來查看安全性主體。

系統會建立使用者、群組和電腦，並將其儲存為 Active Directory Domain Services 中的物件。 此外，也有已知的安全性主體，代表由 Windows 2000 安全性系統所定義的特殊身分識別，例如 Everyone、Local System、Principal Self、已驗證的使用者、Creator Owner 等等。 代表知名安全性主體（例如匿名登入）的物件會儲存在設定容器底下的 W w n 安全性主體容器中。
