# <a name="characteristics-of-attributes"></a>屬性的特性

Active Directory Domain Services 物件中的每個屬性都是由架構容器中的 [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) 物件所定義。 **AttributeSchema** 物件的屬性會指定屬性的特性，例如：

-   屬性識別碼。 屬性有數個識別碼，從程式設計的觀點來看，最有趣的是 [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname)，它是由 LDAP 用戶端用來讀取和寫入屬性，而 [**schemaIDGUID**](/windows/desktop/ADSchema/a-schemaidguid)則是用於安全描述項來控制屬性的存取。
-   屬性實例所包含的資料類型。 屬性的語法屬性會決定資料類型，例如整數、字串或二進位。 其他屬性可以指定屬性所允許的值範圍，以及屬性的實例是否可以有多個值。
-   將屬性包含在群組中。 其他屬性（property）會將屬性（attribute）標記為要包含在屬性集內，也就是一組相關屬性，或包含在通用類別目錄或索引中複寫的屬性集合中，以將搜尋效能優化。

下表列出 [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema)物件之索引鍵屬性的 [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname)和描述。 如需 **attributeSchema** 物件的必要和選擇性屬性的詳細資訊和完整清單，請參閱 [**屬性架構類別**](/windows/desktop/ADSchema/c-attributeschema)。

| [**rangeLower**](/windows/desktop/ADSchema/a-rangelower)、 [ **rangeUpper**](/windows/desktop/ADSchema/a-rangeupper)         | 一對整數，指定這個屬性的值範圍的下限和上限。 為屬性設定的所有值都必須在或等於指定的界限。 若為具有數值語法的屬性，範圍會指定最小值和最大值。 若為具有字串語法的屬性，範圍會指定最小和最大大小（以字元為單位）。 若為具有二進位語法的屬性，範圍會指定位元組數目。<br/> 如果同時設定了 [**rangeLower**](/windows/desktop/ADSchema/a-rangelower) 和 [**RangeUpper**](/windows/desktop/ADSchema/a-rangeupper) ，則 **rangeLower** 必須小於 **rangeUpper**。 如果有一個條件約束沒有另一個條件約束，則遺漏的條件約束為未系結。 例如，如果整數的 **rangeLower** 為3，而 **rangeUpper** 不存在，就表示屬性上沒有上層條件約束。 同樣地，如果字串的 rangeUpper 是2000，而且 rangeLower 不存在，就表示長度沒有較低的條件約束。<br/> |
