# <a name="access-control-inheritance"></a>存取控制繼承

物件存取控制清單中 (Ace) 的存取控制專案 (ACL) 可以屬於兩個類別的其中一種：

-   有效的 ACL：此類別目錄中的 Ace 適用于物件。
-   繼承 ACL：在容器中建立的物件會繼承此類別中的 Ace。

DACL 中的每個 ACE 可以屬於一個或多個類別。 ACE 所屬的類別是由 ACE 中設定的繼承控制旗標所決定。

您可以在 ACE 的 [**AceFlags**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) 屬性中設定三個繼承控制旗標。



| 旗標                                          | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|-----------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ADS \_ ACEFLAG \_ 繼承 \_ ACE**                | 此旗標指出 ACE 是繼承 ACL 的一部分，而子物件會繼承此 ACE 的繼承控制旗標。                                                                                                                                                                                                                                                                                                                         |
| **ADS \_ ACEFLAG \_ 沒有 \_ 傳播 \_ 繼承 \_ ACE** | 此旗標指出 ACE 是繼承 ACL 的一部分，但不會將繼承控制旗標傳播至直接子物件 (直接下階) ，而且 ACE 在直接子物件上有效。                                                                                                                                                                                                                                          |
| **ADS \_ ACEFLAG \_ 只會繼承 \_ \_ ACE**          | 此旗標指出 ACE 不是有效 ACL 的一部分。 如果未設定此旗標，則 ACE 是有效 ACL 的一部分。 此旗標適用于設定子工作可繼承的許可權，但不會影響容器的協助工具。 例如，如果 ACE 是要由組織單位中的使用者物件所繼承，則可能不會強制執行該 ACE 來存取組織單位本身。<br/> |



 

**Ads \_ ACEFLAG \_ NO \_ 傳播 \_ 繼承 \_ ace** 和 ADS ACEFLAG 只有在有 **ADS \_ ACEFLAG \_ 繼承 \_ ace** 時，才會 **\_ \_ 繼承 \_ \_ ace** 旗標。 這是因為 **ADS \_ ACEFLAG \_ INHERIT \_ ace** 旗標會將繼承行為新增至可繼承的 ace，但不會定義繼承的類型。 **Ads \_ ACEFLAG \_ NO \_ \_ inherit inherit \_ ace** 和 **ADS \_ ACEFLAG 只會 \_ 繼承 \_ \_ ace** 旗標來定義特定類型的繼承行為。

請務必記住，系統也會根據 ACE 的型別和狀態來設定下列旗標。



| 旗標                                    | 描述                                           |
|-----------------------------------------|-------------------------------------------------------|
| **ADS \_ ACEFLAG \_ 繼承的 \_ ACE**        | 此旗標表示已繼承 ACE。       |
| **ADS \_ ACEFLAG \_ 有效的 \_ 繼承 \_ 旗標** | 此旗標表示繼承旗標有效。 |



 

下表列出 ACE 的 [**AceFlags**](/windows/desktop/ADSI/iadsaccesscontrolentry-property-methods) 屬性不同旗標組合的效果。



| 旗標                                                                                                                    | 對包含 ACE 的物件效果                     | 對直接子物件的影響                                                      | 對下列直接子系的物件所造成的影響               |
|-------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------|-------------------------------------------------------------------------------------|-------------------------------------------------------|
| 未設定旗標。                                                                                                           | 有效 ACE： ACE 適用于物件。               | ACE 不是繼承的。                                                               | ACE 不是繼承的。                                 |
| **ADS \_ ACEFLAG \_ 繼承 \_ ACE**                                                                                          | 有效 ACE                                           | ACE 是繼承的。 ACE 是有效的 ACE。<br/>                               | ACE 是繼承的。 ACE 是有效的 ACE。<br/> |
| **廣告 \_ACEFLAG \_ 繼承 \_ ACE** \| **ADS \_ ACEFLAG \_ \_ 只繼承 \_ ACE**                                                  | 不是有效的 ACE： ACE 不適用於物件。 | ACE 是繼承的。 ACE 是有效的 ACE。<br/>                               | ACE 是繼承的。 ACE 是有效的 ACE。<br/> |
| **廣告 \_ACEFLAG \_ INHERIT \_ ACE** \| **ADS \_ ACEFLAG \_ NO \_ 傳播 \_ INHERIT \_ ACE**                                         | 有效 ACE                                           | ACE 是繼承的，但沒有繼承旗標。 ACE 是有效的 ACE<br/>  | ACE 不是繼承的。                                 |
| **廣告 \_ACEFLAG \_ 繼承 \_ ACE** \| **ADS \_ ACEFLAG \_ \_ 只繼承 \_ ACE** \| **ADS \_ ACEFLAG \_ 沒有 \_ 傳播 \_ 繼承 \_ ACE** | 不是有效的 ACE。                                   | ACE 是繼承的，但沒有繼承旗標。 ACE 是有效的 ACE。<br/> | ACE 不是繼承的。                                 |
