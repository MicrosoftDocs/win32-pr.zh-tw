# <a name="what-can-you-know-and-when-can-you-know-it"></a>您可以知道什麼，以及何時可以知道？

區分延遲引發狀態的應用程式必須辨識問題狀態，並採取適當的動作。 有兩個問題狀態：版本扭曲和部分更新。 藉由檢查目錄服務來偵測版本扭曲， (記得 [Active Directory Domain Services 使用此複寫模型](why-active-directory-domain-services-uses-this-replication-model.md)) 的原因所述的分散式運算條真理。 您可以藉由將中繼資料加入組成相關集合的物件，來偵測部分更新。 這類中繼資料的建議會出現在本檔的後續章節中。 有一些規避策略可讓應用程式減少版本扭曲和部分更新的可能性。 本檔的後續章節也會討論這些策略。
