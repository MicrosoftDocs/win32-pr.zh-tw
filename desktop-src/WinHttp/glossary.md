# <a name="glossary-winhttp"></a> (WinHTTP) 的詞彙

<dl> <dt>

<span id="term_authentication_data"></span><span id="TERM_AUTHENTICATION_DATA"></span>**驗證資料**
</dt> <dd>

在驗證期間，在伺服器與用戶端之間交換的配置特定資料區塊。 為了證明其身分識別，用戶端會使用使用者名稱和密碼來加密部分或全部資料。 用戶端會將加密的資料傳送至伺服器，該伺服器會將資料解密，並將資料與原始資料進行比較。 如果解密的資料與原始資料相符，用戶端就會經過驗證。

</dd> <dt>

<span id="term_base64_encoding"></span><span id="TERM_BASE64_ENCODING"></span>**base64 編碼**
</dt> <dd>

用來傳輸二進位資料的配置。 Base64 以24位群組的形式處理資料，將此資料對應到四個編碼的字元。 有時也稱為「3對4編碼」。 24位群組的每6位都是用來做為對應表的索引， (base64 字母) 來取得編碼資料的字元。 編碼的資料行長度限制為76個字元。

</dd> <dt>

<span id="term_certificate_store"></span><span id="TERM_CERTIFICATE_STORE"></span>**憑證存放區**
</dt> <dd>

通常會儲存憑證、憑證撤銷清單 (CRL) 的永久儲存體，以及儲存 (CTL) 的憑證信任清單。 使用會話型憑證時，憑證存放區也可以是暫時性的。

</dd> <dt>

<span id="term_cobranding"></span><span id="TERM_COBRANDING"></span>**cobranding**
</dt> <dd>

Microsoft Passport 的參與者網站在 Passport network service 頁面上提供自己商標元素和說明的能力。 Cobranding 包括自訂 Passport network 頁面上的外觀與操作、特定文字和特定行為。

</dd> <dt>

<span id="term_code_page"></span><span id="TERM_CODE_PAGE"></span>**字碼頁**
</dt> <dd>

將程式所使用的二進位字元碼與鍵盤上的按鍵或監視器上的字元外觀相關聯的資料表。 使用字碼頁可提供不同國家/地區所使用之字元集和鍵盤配置的支援。 您可以設定裝置（例如監視器和鍵盤）來使用特定的字碼頁，並從一個字碼頁 (（例如美國) ）切換至另一個 (，例如使用者要求的葡萄牙) 。

</dd> <dt>

<span id="term_http_verb"></span><span id="TERM_HTTP_VERB"></span>**HTTP 指令動詞**
</dt> <dd>

HTTP 動詞命令 (或 HTTP 方法) 是在要求訊息中傳送的指令，會通知 HTTP 伺服器所指定資源上執行的動作。 例如，"GET" 會指定從伺服器抓取資源。 常見的動詞包括 "GET"、"POST" 和 "HEAD"。 如需詳細資訊和標準 HTTP 動詞命令的完整清單，請參閱 HTTP/1.1 規格。

</dd> <dt>

<span id="term_ticket"></span><span id="TERM_TICKET"></span>**票**
</dt> <dd>

包含 Passport 驗證使用者設定檔的 Cookie。 每個票證都對應至一個 URL。 Passport 網域授權單位的登入伺服器會提供用戶端傳送給 Passport 成員進行驗證的票證。

</dd> <dt>

<span id="term_user_agent"></span><span id="TERM_USER_AGENT"></span>**使用者代理程式**
</dt> <dd>

使用者代理程式是從 HTTP 伺服器要求檔的用戶端應用程式。 當用戶端將要求傳送至 HTTP 伺服器時，通常會傳送具有要求標頭的使用者代理程式名稱，讓伺服器能夠判斷用戶端軟體的功能。

</dd> </dl>
