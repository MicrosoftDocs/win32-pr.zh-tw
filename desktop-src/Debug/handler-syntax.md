# <a name="handler-syntax"></a>處理常式語法

本節說明在 Microsoft C/c + + 優化編譯器中執行時，結構化例外狀況處理的語法和用法。 編譯器會將下列關鍵字解釋為結構化例外狀況處理機制的一部分。



| 關鍵字         | 描述                                                                                                                                                                                                                                      |
|-----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **\_\_嘗試**     | 開始受保護的程式碼主體。 使用 with **\_ \_ except** 關鍵字來建立例外狀況 [處理常式](exception-handler-syntax.md)，或使用 **\_ \_ finally** 關鍵字來建立 [終止處理常式](termination-handler-syntax.md)。 |
| **\_\_除了**  | 開始程式碼區塊，此程式碼只會在其相關聯的 **\_ \_ try** 區塊中發生例外狀況時才執行。                                                                                                                                   |
| **\_\_最後** | 開始一段程式碼，每當控制項的流程離開其相關聯的 **\_ \_ try** 區塊時，就會執行此程式碼區塊。                                                                                                                                    |
| **\_\_離開**   | 允許立即終止 **\_ \_ try** 區塊，而不會造成異常終止和效能損失。                                                                                                                      |



 

編譯器也會將 [**GetExceptionCode**](getexceptioncode.md)、 [**GetExceptionInformation**](getexceptioninformation.md)和 [**AbnormalTermination**](abnormaltermination.md) 函式解釋為關鍵字，而在適當的例外狀況處理語法外部使用，會產生編譯器錯誤。 以下是這些函數的簡短說明。



| 函式                                                   | 描述                                                                                                                                                                                                                                             |
|------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**GetExceptionCode**](getexceptioncode.md)               | 傳回識別例外狀況類型的代碼。 您只能從篩選運算式或例外狀況處理常式區塊內呼叫這個函數。                                                                                                |
| [**GetExceptionInformation**](getexceptioninformation.md) | 傳回 [**例外狀況 \_ 指標**](/windows/desktop/api/WinNT/ns-winnt-exception_pointers) 結構的指標，其中包含內容記錄的指標和例外狀況記錄。 您只能從例外狀況處理常式的篩選運算式內呼叫這個函數。 |
| [**AbnormalTermination**](abnormaltermination.md)         | 指出在執行區塊中的最後一個語句之後，控制項的流程是否會依序離開相關聯的 **\_ \_ try** 區塊。 您只能從終止處理常式的 **\_ \_ finally** 區塊內呼叫這個函數。              |
