<span data-ttu-id="f4847-106">在正在進行調試的進程中發生例外狀況時，系統會將例外狀況傳遞給偵錯工具，以通知偵錯工具。</span><span class="sxs-lookup"><span data-stu-id="f4847-106">When an exception occurs in a process that is being debugged, the system notifies the debugger by passing the exception to it.</span></span> <span data-ttu-id="f4847-107">這就是所謂的 *第一次通知*。</span><span class="sxs-lookup"><span data-stu-id="f4847-107">This is known as *first-chance notification*.</span></span> <span data-ttu-id="f4847-108">系統接著會暫止正在進行調試之進程中的所有線程。</span><span class="sxs-lookup"><span data-stu-id="f4847-108">The system then suspends all threads in the process being debugged.</span></span>

<span data-ttu-id="f4847-109">如果偵錯工具未處理例外狀況，系統會嘗試找出適當的例外狀況處理常式。</span><span class="sxs-lookup"><span data-stu-id="f4847-109">If the debugger does not handle the exception, the system attempts to locate an appropriate exception handler.</span></span> <span data-ttu-id="f4847-110">如果系統找不到適當的系統，系統會再次通知偵錯工具發生例外狀況。</span><span class="sxs-lookup"><span data-stu-id="f4847-110">If the system cannot locate an appropriate one, the system again notifies the debugger that an exception has occurred.</span></span> <span data-ttu-id="f4847-111">這就是所謂的 *最後機會通知*。</span><span class="sxs-lookup"><span data-stu-id="f4847-111">This is known as *last-chance notification*.</span></span> <span data-ttu-id="f4847-112">如果偵錯工具未在最後一次發生通知之後處理例外狀況，則系統會終止正在進行調試的進程。</span><span class="sxs-lookup"><span data-stu-id="f4847-112">If the debugger does not handle the exception after the last-chance notification, the system terminates the process being debugged.</span></span>
