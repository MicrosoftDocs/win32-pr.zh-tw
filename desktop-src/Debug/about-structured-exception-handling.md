# <a name="about-structured-exception-handling"></a>關於結構化例外狀況處理

結構化例外狀況處理和終止處理機制是系統不可或缺的一部分;它們可讓系統穩定。 您可以使用這些機制來建立一致穩定且可靠的應用程式。

結構化例外狀況處理主要是透過編譯器支援來提供。 例如，Microsoft C/c + + 優化編譯器支援用來識別程式碼受防護主體的 **\_ \_ try** 關鍵字、識別例外狀況處理常式的 **\_ \_ except** 關鍵字，以及用來識別終止處理常式的 **\_ \_ finally** 關鍵字。 雖然本總覽使用 Microsoft C/c + + 編譯器特定的範例，但其他編譯器也會提供此支援。
