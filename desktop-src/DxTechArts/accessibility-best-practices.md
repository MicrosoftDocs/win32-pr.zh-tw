## <a name="what-is-accessibility"></a>什麼是協助工具？

通常，當人們想要存取協助工具時，他們會考慮像是在電視上輪椅斜坡和隱藏式字幕等東西。 這是因為這類的協助工具功能很容易使用，而且會由具有明顯障礙的人使用。 不過，協助工具功能不是專為具有最嚴重障礙的功能而設計的。 美國的電腦使用者（範圍從18年到64歲），57% (74200000) 可能會因為殘障和障礙而無法使用可影響電腦使用的輔助技術。  ( 「[可供存取的技術市場：廣泛的能力及其對電腦使用的影響](https://www.microsoft.com/enable/research/phase1.aspx)」 Microsoft Corporation) 能夠開啟 payphone 的磁片區，讓具有輕微聽力的人可以使用它們。 飛行航班上的手軌可讓行動性受損的人員更輕鬆地進行順維。

有時候，產品的一般功能最終會成為可協助殘障人士的功能。 比方說，有視覺障礙的人可以使用電視上的對比設定，讓螢幕更容易看見。 具有帕金森氏疾病的人可以使用一次觸控撥號來輕鬆撥打電話。

協助工具功能通常會提供五種殘障類型的其中一種：

-   願景、無法區分色彩、模糊視覺等。
-   聽力障礙、聽障。
-   語音-語音受損、語言差異。
-   行動性，手腕、arm、腿和手上的障礙。
-   認知 Learning 的障礙和推理挑戰，包括障礙。

在影片遊戲的內容中，新增協助工具表示讓有一位殘障人士的人可以使用標題。

## <a name="why-is-accessibility-important"></a>為什麼協助工具很重要？

遊戲開發人員應該考慮將其產品變成可供存取的社交和財務原因。

如果兒童和年輕人的障礙是從輕度到嚴重的，則影片遊戲可提供許多好處。 Wheeling Jesuit 大學的研究人員最近發現，玩運動遊戲或打擊遊戲有助於讓兒童和年輕人的困擾，在2006年2月13日)  (埃德蒙頓期刊的長期難題。 此外，我們也已證明影片遊戲，以更有效率的方式（而不是 tranquilizers (相關聯的按壓，) 12 月 2004 19 日）來協助兒童更有效率。 遊戲甚至用於癌症處理;在 chemotherapy 之後進行復原的關鍵是，我們鼓勵您在兒童拒絕參與其他形式的實體活動時，透過使用 Dance Dance 革命 (c) 的遊戲。

此外，讓殘障人士的 (特別是子女) 參與大部分人享有和參與的活動，有助於降低情緒難題和無疑的感覺。

社交理由不是遊戲開發人員應該將協助工具功能引入其標題的唯一原因。 協助工具功能可以藉由鼓勵殘障人士購買可存取的標題，來增加銷售量。 增加的銷售也可以來自想要支援支援協助工具之公司的玩家。 最後，正面的 P.R。 從媒體和協助工具的支援小組提供免費的廣告。

協助工具的需求會隨著遊戲人口的年齡持續成長。 當人們越來越久時，有輕微的障礙可能會越來越嚴重。 此外，人們可能會開發新的問題，並因其年齡而受損。 將基本的協助工具功能新增至標題，可協助發行者和開發人員繼續向這些客戶描繪收入。

## <a name="the-state-of-accessibility-in-the-games-industry"></a>遊戲產業的協助工具狀態

針對大部分的遊戲產業，影片遊戲中的協助工具為低優先順序。 其中一個原因是因為開發人員對於協助工具問題缺乏認知，所以未停用的開發人員可能不知道他們能夠讓殘障人士更容易存取的標題。

另一個原因是開發人員的時間和資源數量有限。 成本效益分析通常會讓協助工具問題不值得考慮和投資遊戲產業，因為其假設如下：

-   執行協助工具功能的成本不值得傳回。
-   沒有足夠的物件可讓協助工具開發更有價值。

這些假設有問題。 讓遊戲可供存取，很值得投資。

## <a name="visual-impairments"></a>視覺障礙

*「我的簡報之後是生動問與答的課程，當其中一個員工問到我們的遊戲中的協助工具有問題時，就會發生一個值得注意的時候 ... \[ \] 這28歲的 staffer 是一位 avid 的玩家，用來播放 \[ 全形 \] 朋友的遊戲。但因為他是色盲，所以很難說出糟糕的人，而遊戲最後也變得太挫折。當新版本 .。。但我們還 \[ 沒 \] 解決問題，他和他的朋友都決定改為購買競爭者的遊戲。」是匿名的產業主管*

「視覺障礙」這一詞通常會為您提供完全失明的人。 不過，令人震驚知道男性人口的8.7% 受到某種程度的色盲影響 ( 「[人們如何繼承 colorblindness？」](https://www.webexhibits.org/causesofcolor/2C.mdl)WebExhibits.org) 的頻率。 另一個1.2% 的個人會受到更嚴重的視覺障礙形式所影響 ( 「[殘障資訊：視覺障礙的事實表](https://nichcy.org/disability/specific/visualimpairment)」，適用于殘障人士的國家散佈中心) 。 這表示，每十個可能的玩家都有一個問題影響到其遊戲體驗的視力。

為協助您瞭解視覺障礙問題，請想像：



| 您是一間玩家      | 您在此案例中                                                                                                                                                                         |
|----------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 使用一般視覺   | 它是很鮮的 sunny，所以您無法在螢幕上看到深色的物件。 <br/> 您有舊的電視組，因此因為圖片品質不良，所以您看不到小型物件和文字。 <br/> |
| 視力受損 | 有些遊戲文字太小，所以無法讀取。 <br/> 您是色盲，因此當遊戲告訴您按下紅色按鈕時，您不知道要按哪一個按鈕。 <br/>                   |



 

只要幾個簡單的步驟和功能，您就可以解決這些問題，並透過視力障礙的玩家以及具有視覺障礙的玩家來改善遊戲體驗。

1.  黑色和白色電視上的測試標題。 請注意，專案、播放機、目標和命令無法辨別的任何實例，並據以調整您的色板。
2.  讓玩家可以選擇增加螢幕上的文字大小。 也提供變更文字滾動速率的功能。 請務必記住，主控台體驗是10英尺，而不是許多電腦開發人員使用的2英尺遊戲體驗。 即使是針對沒有視覺問題的玩家，小型 UI 和文字可能很難以長距離閱讀。
3.  提供可語音所有遊戲文字的文字轉換語音功能，包括追蹤按鈕焦點的遊戲功能表。 允許使用者控制聲音的速度、音調和性別。 為了防止其他遊戲雜訊 drowned 文字轉換語音，讓使用者能夠調整語音、環境雜訊、主動式遊戲音效和音樂的音量。 此外，也可以在透過功能表項目和透過按鈕進行轉換時，包含用來播放不同音效的選項。
4.  最後，為玩家提供在遊戲中變更亮度和對比設定的選項。 讓使用者能夠選擇自己的自訂色彩配置，讓文字、背景和抬頭顯示器色彩可以自訂，以符合個人的需求。

## <a name="auditory-impairments"></a>聽覺障礙

*「Half-Life 的回到輪你」我們，還有另一種技術 \[ 卻說 \] 的光暈對失聰玩家沒什麼説明 .。。我們希望沒有任何祈求！如果 Halo 2 曾看到一天的燈將完全 subtitled。」 www.DeafGamers.com*

下一種可能影響遊戲播放的最普遍形式，是聽力上的障礙。 在美國境內，超過28000000人會受到某種聽力障礙的影響。 雖然聽力障礙通常與年齡相關聯，但在18歲的每個1000子女中，有17個都受到聽力障礙 ( 「美國[失常、Ear 感染和聽障的相關統計資料](https://www.nidcd.nih.gov/health/statistics/pages/hearing.aspx)」、「國家/地區協會聽障和其他通訊失常) 。 當有人考慮到今天的玩家有更舊的玩家，並以不斷增加的速率遺失其聽覺時，就表示音訊存取範圍的需求只會成長。

您可以使用一些相對較少的工作，讓玩家有正常聽到的玩家，以及具有聽覺障礙的玩家，讓遊戲更有樂趣。

1.  關閉所有對話方塊的標題。 這包括遊戲內內容和 cinematics。 讓玩家能夠開啟和關閉這些標題。
2.  當音效效果提供重要資訊時，請提供文字或 tactile (振動) 機制來提供意見反應。 比方說，如果您遊戲中的炸彈通常會讓嗶聲的雜訊更快，則請提供視覺指標 (例如時間列) ，也可讓玩家知道在爆炸之前剩下多少時間。
3.  如果您的遊戲支援線上播放，請讓玩家選擇傳送文字訊息以及使用語音，在小組成員與其他線上玩家之間傳遞資訊。 耳機對於聽不到、更多的玩家想要與其他人（他們可以在線上進行通訊和制定工作）的人來說，並不有用。

## <a name="mobility-impairments"></a>行動障礙

*「Videogames 為殘障人士提供殘障人士重新連接其對等的機會，以及遺失或從未發生過的能力。我的個人經驗是使癱瘓在過去14歲的年齡，並且造訪醫院的「娛樂中心」，而我只需要中斷我的 depression，就可以玩 videogame 系統。當我學到無法播放時，我很快就會失去興趣 ...」Robert Florio*

行動性受損可能是最難獲得公司統計資料的各種障礙。 這主要是由於疾病、neurological 失常、樹幹/數位、paralysis 等等所造成的原因，而這兩者都可能對影片玩家的體驗產生不同程度的影響。 這些障礙可能 congenital，或可能在稍後的存留期間發生。

為協助您瞭解聽覺障礙問題，請想像：

|-------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 沒有行動性障礙                     | 遊戲控制器有很多按鈕，您 () intimidated，而不想要學習如何使用。 <br/>                                                                                                                                                                          |
| 具有暫時性的行動性障礙            | 您有一個已中斷的經驗，所以您無法在控制器上使用操縱杆。 <br/> 您有中斷的階段，所以您無法使用 dance pad 作為舞蹈標題。<br/>                                                                                                                                     |
| 具有永久且嚴重的行動性障礙 | 您已遺失 arm，所以您無法使用雙站控制器。 <br/> 您有帕金森氏的疾病、您的手搖動，並讓您不小心在控制器上觸發按鈕。 <br/> 您正在向下使癱瘓，所以根本無法使用標準遊戲控制器。 <br/> |



 

想要讓所有的玩家都能發揮最大的挑戰，但是在開發遊戲時，有一些簡單的事可以牢記在心。

1.  最小化按鈕使用，並考慮更多命令的功能表介面。 這特別適用于可能遺失數位或手的個人。 它也適用于使用自訂控制器的使癱瘓人員。
2.  允許玩家自訂其控制器設定和按鈕/操縱杆的敏感度。 這將可讓具有良好汽車技能問題的個人自訂控制器，以將殘障人士對遊戲的影響降到最低。 它也可讓殘障人士更有效地支援自訂控制器。
3.  如果您的遊戲利用特定類型的周邊 (dance pad、強光等 ) ，可讓其他控制器執行相同的功能。 比方說，像是 Dance Dance 革命 (c) 可讓輪椅限制的個人透過使用一般的固定控制站來與朋友一起播放。

| 您是一間玩家             | 您在此案例中                                                                                                                                                                                                                                           |
|-----------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 沒有關切這個領域障礙    | 您正在玩需要說出命令的遊戲來控制您的字元，而且因為您沒有麥克風而無法播放。 <br/> 您即將在夜間播放，而不想要打擾任何人，因此無法使用您的 communicator。 <br/> |
| 神秘有語音障礙 | 您正在玩需要說出命令的遊戲來控制您的字元，因為遊戲無法辨識您所說的內容，所以無法玩。                                                                                                               |
| 神秘無法說話      | 您現正播放的遊戲需要語音輸入，因此無法播放。 <br/> 您正在玩的線上遊戲要求您透過 communicator 協調策略，讓您無法有效率地播放。 <br/>                                               |



 

幸運的是，有一些簡單的修正程式，可讓您的遊戲更容易使用，並讓這些玩家愉快。

1.  如果遊戲使用語音辨識，請提供玩家選擇功能表或按鈕組合中命令的選項。
2.  如果您的標題也支援線上多人遊戲，請讓玩家選擇可自訂宏，其中包含音訊訊息或 (甚至更適合具有聽力障礙的人) 的文字訊息。 提供聊天的鍵盤支援也是一個選項。

## <a name="conclusion"></a>結論

到目前為止，您可能會想要在所有這些情況下都無法容納所有的玩家。 而且，即使您要執行這份檔中的每個建議，也無法確保所有人都能完全存取標題。 但藉由遵循這些協助工具的指導方針，您可以讓您的標題更吸引協助工具群體。 而且只能增加銷售額。

為了讓標題更容易存取，開發人員和發行者需要找出具有各種殘障人士的人，以測試其遊戲。 這種方法會提供特定物件是否可存取遊戲的第一手資訊。 另外還有一項優點，就是有各式各樣的開發和測試資源可以提供額外的見解，以改善所有玩家的遊戲播放。 最重要的是，參與協助工具群體，並瞭解這些潛在客戶。 在當地的失聰服務中心、兒童醫院或經驗豐富的中心舉辦遊戲 bash。 鼓勵開發人員和測試人員自願與殘障人士合作、採用正負號語言類別，或是註冊可存取協助工具的電子報，以與社區保持聯繫。 從已停用的玩家要求意見反應-當地學校和大學的玩家。

沒人覺得無疑。 藉由在遊戲測試與設計中包含協助工具，您將能夠以更廣大的物件為您的標題市場，並對該社區和您的下一行進行正確的動作。
