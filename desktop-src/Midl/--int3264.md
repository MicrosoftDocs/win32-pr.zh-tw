# <a name="__int3264-attribute"></a>\_\_int3264 屬性

關鍵字 **\_ \_ int3264** 指定具有下列屬性的整數型別：

-   在32位平臺上是32位
-   在64位平臺上是64位
-   這是網路上的32位，以提供回溯相容性。 在傳送端會截斷，並且在接收端上適當地 (已簽署或未簽署的) 進行擴充。

``` syntax
[ signed | unsigned ] __int3264 [ int ] declarator-list;
```

## <a name="parameters"></a>參數

<dl> <dt>

*宣告子清單* 
</dt> <dd>

指定一或多個標準 C 宣告子，例如識別碼、指標宣告子和陣列宣告子。 在遠端程序呼叫中傳輸的結構中，不允許 (函式宣告子和位欄位宣告。 未傳輸的結構中允許這些宣告子。 ) 以逗號分隔多個宣告子。

</dd> </dl>

## <a name="remarks"></a>備註

這種類型是專門為了促進32位介面移植至64位所引進。 它通常僅適用于現有介面以整數形式傳送指標或使用類似的因應措施技術的情況。 Microsoft 不建議使用此類型 (或其他衍生自它的公用類型，例如 **INT \_ ptr**、 **DWORD \_ ptr**) 可避免的情況。 尤其是，這些類型不能用在 OLE Automation 介面和類型程式庫中。 不建議在新的介面中使用此類型或衍生自它的類型。 如需有關這些問題的詳細資訊，以及其他64位問題的批註和建議，請參閱 Microsoft 介面定義語言 (MIDL) ：64位移植指南。

\_ \_ **Int3264** 關鍵字的前面可以是關鍵字帶正負號或不帶正負號的關鍵字。 Int 關鍵字是選擇性的，可以省略。 對 MIDL 編譯器而言， \_ \_ 預設會簽署 **int3264** 整數，而且與帶正負號的 \_ \_ int3264 int 同義。

**\_ \_ Int3264** 整數類型是 IDL 語言的其中一種基底類型。 **\_ \_ Int3264** 整數型別可以在 const 宣告、typedef 宣告、一般宣告和函式宣告子中顯示為類型指定名稱， (為函式傳回型別規範，以及) 的參數類型規範。 如需類型規範出現的內容，請參閱 IDL。
