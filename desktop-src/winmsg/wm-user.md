<span data-ttu-id="c1d07-104">用來定義私用訊息以供私用視窗類別使用，通常是以 **WM \_ 使用者** x 形式來使用， + 其中 *x* 是整數值。</span><span class="sxs-lookup"><span data-stu-id="c1d07-104">Used to define private messages for use by private window classes, usually of the form **WM\_USER**+*x*, where *x* is an integer value.</span></span>

<span data-ttu-id="c1d07-119">第一個範圍中的訊息編號 (0 到 **WM \_ 使用者** -1) 是由系統所定義。</span><span class="sxs-lookup"><span data-stu-id="c1d07-119">Message numbers in the first range (0 through **WM\_USER** –1) are defined by the system.</span></span> <span data-ttu-id="c1d07-120">此範圍中未明確定義的值是由系統所保留。</span><span class="sxs-lookup"><span data-stu-id="c1d07-120">Values in this range that are not explicitly defined are reserved by the system.</span></span>

<span data-ttu-id="c1d07-121">第二個範圍中的訊息編號 (**WM \_ 使用者** 透過 0x7FFF) 可以由應用程式定義和使用，以在私用視窗類別內傳送訊息。</span><span class="sxs-lookup"><span data-stu-id="c1d07-121">Message numbers in the second range (**WM\_USER** through 0x7FFF) can be defined and used by an application to send messages within a private window class.</span></span> <span data-ttu-id="c1d07-122">這些值不能用來定義整個應用程式中有意義的訊息，因為某些預先定義的視窗類別已經在這個範圍中定義值。</span><span class="sxs-lookup"><span data-stu-id="c1d07-122">These values cannot be used to define messages that are meaningful throughout an application because some predefined window classes already define values in this range.</span></span> <span data-ttu-id="c1d07-123">例如，[ **按鈕**]、[ **編輯**]、[ **LISTBOX**] 和 [ **COMBOBOX** ] 等預先定義的控制項類別，都可能會使用這些值。</span><span class="sxs-lookup"><span data-stu-id="c1d07-123">For example, predefined control classes such as **BUTTON**, **EDIT**, **LISTBOX**, and **COMBOBOX** may use these values.</span></span> <span data-ttu-id="c1d07-124">除非應用程式已設計為交換訊息，並將相同意義附加至訊息編號，否則不應將此範圍中的訊息傳送至其他應用程式。</span><span class="sxs-lookup"><span data-stu-id="c1d07-124">Messages in this range should not be sent to other applications unless the applications have been designed to exchange messages and to attach the same meaning to the message numbers.</span></span>

<span data-ttu-id="c1d07-125">第三個範圍中的訊息編號 (0x8000 到 0xBFFF) 可供應用程式用來作為私用訊息。</span><span class="sxs-lookup"><span data-stu-id="c1d07-125">Message numbers in the third range (0x8000 through 0xBFFF) are available for applications to use as private messages.</span></span> <span data-ttu-id="c1d07-126">此範圍內的訊息不會與系統訊息發生衝突。</span><span class="sxs-lookup"><span data-stu-id="c1d07-126">Messages in this range do not conflict with system messages.</span></span>

<span data-ttu-id="c1d07-127">第四個範圍內的訊息編號 (0xC000 至 0xFFFF) 是在執行時間定義，而應用程式會呼叫 [**RegisterWindowMessage**](/windows/win32/api/winuser/nf-winuser-registerwindowmessagea) 函式來取得字串的訊息編號。</span><span class="sxs-lookup"><span data-stu-id="c1d07-127">Message numbers in the fourth range (0xC000 through 0xFFFF) are defined at run time when an application calls the [**RegisterWindowMessage**](/windows/win32/api/winuser/nf-winuser-registerwindowmessagea) function to retrieve a message number for a string.</span></span> <span data-ttu-id="c1d07-128">所有註冊相同字串的應用程式都可以使用相關聯的訊息編號來交換訊息。</span><span class="sxs-lookup"><span data-stu-id="c1d07-128">All applications that register the same string can use the associated message number for exchanging messages.</span></span> <span data-ttu-id="c1d07-129">不過，實際的訊息編號不是常數，而且在不同的會話之間不能假設是相同的。</span><span class="sxs-lookup"><span data-stu-id="c1d07-129">The actual message number, however, is not a constant and cannot be assumed to be the same between different sessions.</span></span>
