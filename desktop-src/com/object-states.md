# <a name="object-states"></a>物件狀態

綜合物件存在於下列三種狀態之一：被動、已載入或正在執行。 複合檔案物件的狀態會描述其容器中的物件與負責其建立的應用程式之間的關聯性。 下表摘要說明這些狀態。



| 物件狀態       | Description                                                                                                                                                                                                                                                                                                                                                                                         |
|--------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 被動<br/> | 複合檔案物件只存在於磁片上或資料庫中的儲存體。 在此狀態下，物件無法用於進行查看或編輯。<br/>                                                                                                                                                                                                                                   |
| 已載入<br/>  | 物件處理常式所建立的物件資料結構位於容器的記憶體中。 容器已與物件處理常式建立通訊，而且有快取的呈現資料可用於轉譯物件。 呼叫是由物件處理常式處理。 由於此狀態的負擔很低，因此當使用者只是要查看或列印物件時，就會使用此狀態。<br/> |
| 執行中<br/> | 已建立控制遠端處理的物件，而且正在執行 OLE 伺服器應用程式。 物件的介面可供存取，而容器可以接收變更的通知。 在此狀態下，終端使用者可以編輯或以其他方式操作物件。<br/>                                                                                                                    |
