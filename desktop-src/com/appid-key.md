## <a name="remarks"></a><span data-ttu-id="bf330-148">備註</span><span class="sxs-lookup"><span data-stu-id="bf330-148">Remarks</span></span>

<span data-ttu-id="bf330-149">Appid 會使用兩種不同的機制對應到可執行檔和類別：</span><span class="sxs-lookup"><span data-stu-id="bf330-149">AppIDs are mapped to executables and classes using two different mechanisms:</span></span>

-   <span data-ttu-id="bf330-150">使用128位的全域唯一識別碼 (GUID) 來識別 **AppID** 金鑰。</span><span class="sxs-lookup"><span data-stu-id="bf330-150">Using a 128-bit Globally Unique Identifier (GUID) that identifies the **AppID** key.</span></span> <span data-ttu-id="bf330-151">類別會在名為 "AppID" 的 [**CLSID**](clsid-key-hklm.md) 機碼下，指出其對應的 AppID。</span><span class="sxs-lookup"><span data-stu-id="bf330-151">A class indicates its corresponding AppID under the [**CLSID**](clsid-key-hklm.md) key in a named value "AppID".</span></span> <span data-ttu-id="bf330-152">此對應會在啟用期間使用。</span><span class="sxs-lookup"><span data-stu-id="bf330-152">This mapping is used during activation.</span></span>
-   <span data-ttu-id="bf330-153">使用表示可執行檔名稱的指名值 (例如 "MYOLDAPP.EXE" ) 。</span><span class="sxs-lookup"><span data-stu-id="bf330-153">Using a named value that indicates an executable name (such as "MYOLDAPP.EXE").</span></span> <span data-ttu-id="bf330-154">此命名值的型別為 **REG \_ SZ** ，並且包含與可執行檔相關聯之 AppID 的字串表示。</span><span class="sxs-lookup"><span data-stu-id="bf330-154">This named value is of type **REG\_SZ** and contains the string representation of the AppID associated with the executable.</span></span> <span data-ttu-id="bf330-155">此對應是用來取得預設存取權限和驗證層級。</span><span class="sxs-lookup"><span data-stu-id="bf330-155">This mapping is used to obtain the default access permissions and authentication level.</span></span>

<span data-ttu-id="bf330-156">**HKEY \_ 本機 \_ 電腦 \\ 軟體 \\ 類別** 金鑰組應至 **HKEY \_ 類別 \_ 根金鑰**，此機碼是為了與舊版 COM 相容而保留的。</span><span class="sxs-lookup"><span data-stu-id="bf330-156">The **HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Classes** key corresponds to the **HKEY\_CLASSES\_ROOT** key, which was retained for compatibility with earlier versions of COM.</span></span>

<span data-ttu-id="bf330-157">若是 COM 伺服器，在註冊程式期間或執行 dcomcnfg.exe 時，通常會產生對應並寫入登錄中。</span><span class="sxs-lookup"><span data-stu-id="bf330-157">For COM servers, the mapping is usually generated and written to the registry during the registration process or when running dcomcnfg.exe.</span></span> <span data-ttu-id="bf330-158">不過，想要使用 **AppID** 金鑰設定安全性的 COM 用戶端必須建立適當的登錄機碼，並藉由呼叫登錄函 [式或使用](/windows/desktop/SysInfo/registry-functions) Regedit.exe 來指定所需的對應。</span><span class="sxs-lookup"><span data-stu-id="bf330-158">However, COM clients that want to set security using the **AppID** key must create appropriate registry keys and specify the required mapping by calling the [registry functions](/windows/desktop/SysInfo/registry-functions) or using Regedit.exe.</span></span> <span data-ttu-id="bf330-159">然後，您可以為用戶端設定 [**AccessPermission**](accesspermission.md) 或 [**AuthenticationLevel**](authenticationlevel.md) 之類的值。</span><span class="sxs-lookup"><span data-stu-id="bf330-159">Then values such as [**AccessPermission**](accesspermission.md) or [**AuthenticationLevel**](authenticationlevel.md) can be set for the client.</span></span> <span data-ttu-id="bf330-160">例如，假設您的用戶端進程的可執行檔名稱是 "YourClient.exe"，而您想要將驗證層級設定為「無」。</span><span class="sxs-lookup"><span data-stu-id="bf330-160">For example, suppose the name of your executable for your client process is "YourClient.exe" and you want to set the authentication level to "None".</span></span> <span data-ttu-id="bf330-161">您可以使用 Guidgen.exe 或 Uuidgen.exe 來建立可執行檔之 AppID 的 GUID。</span><span class="sxs-lookup"><span data-stu-id="bf330-161">You would use Guidgen.exe or Uuidgen.exe to create the GUID that is the AppID for your executable.</span></span> <span data-ttu-id="bf330-162">然後您會在登錄中設定值，如下列範例所示，其中00000001代表「無」的驗證層級：</span><span class="sxs-lookup"><span data-stu-id="bf330-162">Then you would set values in the registry as shown in the following example, where 00000001 represents an authentication level of "None":</span></span>

```
   {MyGuid}
      AuthenticationLevel = 00000001
   MyClient.exe
      AppID = {MyGUID}
```
