# <a name="delegation-and-impersonation"></a>委派和模擬

在用戶端/伺服器案例中，一部伺服器通常會呼叫另一部伺服器來代表用戶端完成一些工作。 如果伺服器被授與代表用戶端的許可權，就稱為「 *委派*」。

從安全性的觀點來看，有兩個關於委派的問題：

-   代表用戶端時，應該允許伺服器執行哪些動作？
-   當伺服器代表用戶端呼叫其他伺服器時，會顯示哪個身分識別？

為了處理這些問題，COM 提供下列功能。 用戶端可以設定模擬 *等級* ，判斷伺服器將能夠做為用戶端的程度。 如果用戶端將足夠的許可權授與伺服器，伺服器就 *可以模擬* (假裝) 用戶端。 模擬用戶端時，伺服器只會獲得用戶端有權使用之物件或資源的存取權。 作為用戶端的伺服器也可以啟用遮罩來遮罩自己的身分 *識別，並* 在呼叫其他 COM 元件時投射用戶端的身分識別。

![此圖顯示作為用戶端的伺服器如何啟用遮蓋。](images/172e04f7-568d-450b-9785-2c1a2b40e549.png)

請考慮上圖所示的案例，其中 A 和 B 是在不同電腦上的 C 處理常式。處理 A 呼叫 B，而 B 呼叫 C. 用戶端 A 設定模擬層級。 B 設定「掩蓋」功能。 如果設定了允許模擬的模擬層級，B 可以在代表的時候模擬。 呈現給處理 C 的身分識別將會是身分識別或 B 的身分識別，這取決於是否由 B 啟用遮蓋。如果啟用了掩蓋，則呈現給處理 C 的身分識別將會是的。如果未啟用掩蓋功能，B 的身分識別將會顯示為 C。
