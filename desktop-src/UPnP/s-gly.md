# <a name="s-upnp-apis"></a> (UPnP Api) 

包含以字母 S 開頭的 UPnP 相關詞彙。

[B](a-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) F G [H](h-gly.md) I J K L M [N](n-gly.md) O [P](p-gly.md) Q [R](r-gly.md) S T W [](u-gly.md) X Y Z

<dl> <dt>

<span id="upnp.s_1_gly"></span><span id="UPNP.S_1_GLY"></span>**服務**
</dt> <dd>

服務是邏輯功能單位。 服務會使用狀態變數來公開動作，並為實體裝置的部分或所有狀態建立模型。

</dd> <dt>

<span id="upnp.s_2_gly"></span><span id="UPNP.S_2_GLY"></span>**服務描述**
</dt> <dd>

服務描述是服務所提供的動作清單，以及與動作相關聯的狀態變數。

</dd> <dt>

<span id="upnp.s_3_gly"></span><span id="UPNP.S_3_GLY"></span>**服務物件**
</dt> <dd>

服務物件是服務的控制點 API 標記法。 服務物件會執行 [**IUPnPService**](/windows/desktop/api/Upnp/nn-upnp-iupnpservice) 介面。

</dd> <dt>

<span id="upnp.s_4_gly"></span><span id="UPNP.S_4_GLY"></span>**服務類型**
</dt> <dd>

服務類型是用來識別服務的 URN。 UPnP 服務範本與服務類型之間有一對一的關聯性。 UPnP 論壇工作委員會定義了數種標準服務類型。 服務類型的格式為 urn：架構-upnp-org： service： serviceType： version。 例如，掃描器服務類型可能是 urn：架構-upnp-org： service：掃描器：1。

UPnP 廠商可指定額外的服務。 這類服務是以 urn： domain name： service： serviceType：版本表示，其中的功能變數名稱是向廠商註冊的功能變數名稱。

</dd> <dt>

<span id="upnp.s_5_gly"></span><span id="UPNP.S_5_GLY"></span>**狀態變數**
</dt> <dd>

狀態變數是一段資料，會描述服務狀態的某些部分。

</dd> </dl>
