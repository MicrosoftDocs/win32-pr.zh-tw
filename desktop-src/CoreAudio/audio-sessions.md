# <a name="audio-sessions"></a><span data-ttu-id="6f066-103">音訊會話</span><span class="sxs-lookup"><span data-stu-id="6f066-103">Audio Sessions</span></span>

<span data-ttu-id="6f066-107">當用戶端初始化音訊串流時，它會將音訊串流指派給音訊會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-107">When a client initializes an audio stream, it assigns the audio stream to an audio session.</span></span> <span data-ttu-id="6f066-108">如需詳細資訊，請參閱 [**IAudioClient：： Initialize**](/windows/desktop/api/Audioclient/nf-audioclient-iaudioclient-initialize)。</span><span class="sxs-lookup"><span data-stu-id="6f066-108">For more information, see [**IAudioClient::Initialize**](/windows/desktop/api/Audioclient/nf-audioclient-iaudioclient-initialize).</span></span>

<span data-ttu-id="6f066-109">音訊會話包含轉譯資料流程或抓取資料流程，但不能同時包含兩者。</span><span class="sxs-lookup"><span data-stu-id="6f066-109">An audio session contains either rendering streams or capture streams, but not both.</span></span> <span data-ttu-id="6f066-110">根據預設，轉譯會話的 [磁片區] 和 [靜音] 設定會在系統重新開機時持續進行。</span><span class="sxs-lookup"><span data-stu-id="6f066-110">By default, the volume and mute settings for a rendering session are persistent across system restarts.</span></span> <span data-ttu-id="6f066-111">捕捉會話的 [磁片區] 和 [靜音] 設定不是持續性。</span><span class="sxs-lookup"><span data-stu-id="6f066-111">The volume and mute settings for a capture session are not persistent.</span></span> <span data-ttu-id="6f066-112"> (包含在回送模式下運作之資料流程的會話，會被視為與捕捉會話相同。</span><span class="sxs-lookup"><span data-stu-id="6f066-112">(A session that contains streams that operate in loopback mode is treated the same as a capture session.</span></span> <span data-ttu-id="6f066-113">也就是說，會話設定不是持續性。</span><span class="sxs-lookup"><span data-stu-id="6f066-113">That is, the session settings are not persistent.</span></span> <span data-ttu-id="6f066-114">如需回送模式的詳細資訊，請參閱 [回送記錄](loopback-recording.md)。 ) </span><span class="sxs-lookup"><span data-stu-id="6f066-114">For more information about loopback mode, see [Loopback Recording](loopback-recording.md).)</span></span>

<span data-ttu-id="6f066-115">每個音訊串流都只屬於一個會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-115">Every audio stream belongs to exactly one session.</span></span> <span data-ttu-id="6f066-116">用戶端會在初始化資料流程物件時，將音訊串流指派給特定的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-116">A client assigns an audio stream to a particular session at the time that it initializes the stream object.</span></span> <span data-ttu-id="6f066-117">資料流程會在資料流程的存留期內保留其成員資格。</span><span class="sxs-lookup"><span data-stu-id="6f066-117">The stream retains its membership in the session for the lifetime of the stream.</span></span> <span data-ttu-id="6f066-118">建立資料流程物件之後，物件會一直存在，直到用戶端釋出物件的最後一個計數參考，然後刪除該物件為止。</span><span class="sxs-lookup"><span data-stu-id="6f066-118">After a stream object is created, the object exists until a client releases the last counted reference to the object, and then the object is deleted.</span></span>

<span data-ttu-id="6f066-119">雖然用戶端無法變更已指派現有資料流程的會話，但它可以藉由釋出資料流程的所有參考來 (來達到類似的效果) 、建立新的資料流程來取代已刪除的資料流程，以及將新的資料流程指派給另一個會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-119">Although a client cannot change the session to which an existing stream is assigned, it can achieve a similar effect by deleting the stream (by releasing all references to it), creating a new stream to replace the deleted stream, and assigning the new stream to another session.</span></span>

<span data-ttu-id="6f066-120">每個轉譯會話都代表資料流程的子集，這些資料流程會形成透過特定 [音訊端點裝置](audio-endpoint-devices.md)播放的全域混合。</span><span class="sxs-lookup"><span data-stu-id="6f066-120">Each rendering session represents a subset of the streams that form the global mix that plays through a particular [audio endpoint device](audio-endpoint-devices.md).</span></span> <span data-ttu-id="6f066-121">全域混合將所有的會話與共享裝置的所有應用程式結合在一起。</span><span class="sxs-lookup"><span data-stu-id="6f066-121">The global mix combines all of the sessions from all of the applications that share the device.</span></span>

<span data-ttu-id="6f066-122">通常，具有數個數據流的應用程式會將其所有資料流程指派給相同的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-122">Frequently, an application with several streams assigns all of its streams to the same session.</span></span> <span data-ttu-id="6f066-123">不過，應用程式可以選擇將不同的串流指派給不同的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-123">However, the application can, as an option, assign different streams to different sessions.</span></span> <span data-ttu-id="6f066-124">應用程式未明確指派給會話的任何資料流程都屬於預設的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-124">Any stream that the application does not explicitly assign to a session belongs to the default session.</span></span>

<span data-ttu-id="6f066-125">典型的音訊應用程式應該避免修改會話的音量和靜音設定。</span><span class="sxs-lookup"><span data-stu-id="6f066-125">Typical audio applications should avoid modifying the volume and mute settings for sessions.</span></span> <span data-ttu-id="6f066-126">相反地，使用者會透過控制項程式的使用者介面來控制這些設定。</span><span class="sxs-lookup"><span data-stu-id="6f066-126">Instead, users control these settings through the user interfaces of control programs.</span></span> <span data-ttu-id="6f066-127">例如，在 Windows Vista 中，系統提供的程式 Sndvol.exe，會針對系統中的每個作用中或最近使用中的轉譯會話，顯示音量控制項和靜音控制。</span><span class="sxs-lookup"><span data-stu-id="6f066-127">For example, in Windows Vista, the system-supplied program, Sndvol.exe, displays a volume control and mute control for each active or recently active rendering session in the system.</span></span> <span data-ttu-id="6f066-128">透過這些控制項，使用者可以調整系統中所有會話的音量和靜音設定。</span><span class="sxs-lookup"><span data-stu-id="6f066-128">Through these controls, users can adjust the volume and mute settings for all of the sessions in the system.</span></span>

<span data-ttu-id="6f066-129">Sndvol 程式目前只會顯示音訊轉譯端點裝置的音量控制項。</span><span class="sxs-lookup"><span data-stu-id="6f066-129">The Sndvol program currently displays volume controls for audio-rendering endpoint devices only.</span></span> <span data-ttu-id="6f066-130">它不會顯示音訊捕獲裝置的音量控制項。</span><span class="sxs-lookup"><span data-stu-id="6f066-130">It does not display volume controls for audio-capture devices.</span></span>

<span data-ttu-id="6f066-131">如果會話包含一或多個作用中的資料流程，則會話會處於作用中狀態。</span><span class="sxs-lookup"><span data-stu-id="6f066-131">A session is active if it contains one or more active streams.</span></span> <span data-ttu-id="6f066-132">作用中的資料流程處於執行中 *狀態*。</span><span class="sxs-lookup"><span data-stu-id="6f066-132">An active stream is in the *running state*.</span></span> <span data-ttu-id="6f066-133">非作用中資料流程處於 *已停止狀態*。</span><span class="sxs-lookup"><span data-stu-id="6f066-133">An inactive stream is in the *stopped state*.</span></span> <span data-ttu-id="6f066-134">當會話的第一個資料流程變成作用中時，會話就會變成作用中狀態。</span><span class="sxs-lookup"><span data-stu-id="6f066-134">A session becomes active when its first stream becomes active.</span></span> <span data-ttu-id="6f066-135">當上一個作用中的資料流程變成非使用中時，會話會變成非使用中。</span><span class="sxs-lookup"><span data-stu-id="6f066-135">A session becomes inactive when its last active stream becomes inactive.</span></span> <span data-ttu-id="6f066-136">當會話在一段時間內處於非使用中狀態時，系統會將會話的狀態從非作用中變更為已過期。</span><span class="sxs-lookup"><span data-stu-id="6f066-136">After a session has been inactive for a period of time, the system changes the state of the session from inactive to expired.</span></span>

<span data-ttu-id="6f066-137">Sndvol 會顯示所有作用中和非使用中轉譯會話的音量和靜音控制項。</span><span class="sxs-lookup"><span data-stu-id="6f066-137">Sndvol displays volume and mute controls for all active and inactive rendering sessions.</span></span> <span data-ttu-id="6f066-138">當會話的狀態從非作用中變更為已過期，或當會話終止時，Sndvol 會移除會話的磁片區和靜音控制項。</span><span class="sxs-lookup"><span data-stu-id="6f066-138">Sndvol removes the volume and mute controls for a session when the state of the session changes from inactive to expired, or when the session terminates.</span></span> <span data-ttu-id="6f066-139">當會話的最後一個資料流程被刪除時，會話就會終止 (;亦即，當用戶端釋出會話中最後一個剩餘資料流程物件的最後一個參考計數時，) 此規則的例外狀況。</span><span class="sxs-lookup"><span data-stu-id="6f066-139">(A session terminates when the last of its streams is deleted; that is, when a client releases the final reference count on the last remaining stream object in the session.) The one exception to this rule is for system-notification sounds.</span></span> <span data-ttu-id="6f066-140">無論這些音效的會話狀態為何，Sndvol 一律會顯示系統通知音效的音量和靜音控制項。</span><span class="sxs-lookup"><span data-stu-id="6f066-140">Sndvol always displays the volume and mute controls for system-notification sounds regardless of the state of the session for these sounds.</span></span>

<span data-ttu-id="6f066-141">通常，資料流程屬於只跨越包含建立資料流程的應用程式之進程的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-141">Typically, a stream belongs to a session that spans only the process that contains the application that created the stream.</span></span> <span data-ttu-id="6f066-142">不過，應用程式可以選擇定義跨進程會話，以結合來自兩個或多個進程的串流。</span><span class="sxs-lookup"><span data-stu-id="6f066-142">However, applications have the option of defining cross-process sessions that combine streams from two or more processes.</span></span>

<span data-ttu-id="6f066-143">WASAPI 支援主要的跨進程會話，如此一來：</span><span class="sxs-lookup"><span data-stu-id="6f066-143">WASAPI supports cross-process sessions primarily so that:</span></span>

-   <span data-ttu-id="6f066-144">Sndvol 程式可以向使用者呈現單一的音量控制項，以管理所有應用程式的系統通知音效。</span><span class="sxs-lookup"><span data-stu-id="6f066-144">The Sndvol program can present the user with a single volume control to manage system-notification sounds across all applications.</span></span>
-   <span data-ttu-id="6f066-145">在一個進程中執行的 media player 可將受保護的內容串流至在另一個進程中執行的解密程式。</span><span class="sxs-lookup"><span data-stu-id="6f066-145">A media player that runs in one process can stream protected content to a decryption program that runs in another process.</span></span>

<span data-ttu-id="6f066-146">類似于進程特定轉譯會話的控制項設定，跨進程轉譯會話的控制項設定依預設會在系統重新開機時持續存在。</span><span class="sxs-lookup"><span data-stu-id="6f066-146">Similar to the control settings for process-specific rendering sessions, the control settings for cross-process rendering sessions are, by default, persistent across system restarts.</span></span> <span data-ttu-id="6f066-147">WASAPI 主要是為了讓系統通知音效的優點，而這種行為必須保留使用者的音量和靜音設定，因為應用程式混合會隨著時間而改變。</span><span class="sxs-lookup"><span data-stu-id="6f066-147">WASAPI provides this behavior primarily for the benefit of system-notification sounds, which must retain the user's volume and mute settings as the application mix varies over time.</span></span>

<span data-ttu-id="6f066-148">Sndvol 程式會為每個會話的音量控制項加上顯示名稱和圖示的標籤。</span><span class="sxs-lookup"><span data-stu-id="6f066-148">The Sndvol program labels the volume control for each session with a display name and an icon.</span></span> <span data-ttu-id="6f066-149">用戶端可以選擇將顯示名稱和圖示明確指派給會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-149">A client has the option of explicitly assigning a display name and icon to a session.</span></span> <span data-ttu-id="6f066-150">如果用戶端未提供這些專案，則 Sndvol 會改為顯示預設名稱和預設圖示。</span><span class="sxs-lookup"><span data-stu-id="6f066-150">If the client does not provide these items, then Sndvol displays a default name and default icon instead.</span></span> <span data-ttu-id="6f066-151">預設名稱會併入資訊，例如應用程式視窗的標題。</span><span class="sxs-lookup"><span data-stu-id="6f066-151">The default name incorporates information such as the title of the application window.</span></span> <span data-ttu-id="6f066-152">預設圖示是應用程式視窗的圖示。</span><span class="sxs-lookup"><span data-stu-id="6f066-152">The default icon is the icon for the application window.</span></span> <span data-ttu-id="6f066-153">只有在處理常式特定會話的情況下，這些預設值才會提供有意義的資訊給使用者。</span><span class="sxs-lookup"><span data-stu-id="6f066-153">Only in the case of process-specific sessions do these defaults provide meaningful information to users.</span></span> <span data-ttu-id="6f066-154">請注意，跨進程會話可以與一個以上的應用程式相關聯。</span><span class="sxs-lookup"><span data-stu-id="6f066-154">Note that a cross-process session can be associated with more than one application.</span></span> <span data-ttu-id="6f066-155">在此情況下，只有用戶端提供的顯示名稱和圖示才有意義。</span><span class="sxs-lookup"><span data-stu-id="6f066-155">In this case, only a client-provided display name and icon are meaningful.</span></span>

<span data-ttu-id="6f066-156">雖然轉譯會話的音量和靜音設定預設會在系統重新開機時持續，但用戶端提供的顯示名稱和圖示並不是。</span><span class="sxs-lookup"><span data-stu-id="6f066-156">Although the volume and mute settings for a rendering session are, by default, persistent across system restarts, the client-provided display name and icon are not.</span></span> <span data-ttu-id="6f066-157">為了確保 Sndvol 顯示用戶端提供的名稱和圖示，用戶端必須在用戶端將第一個串流指派給會話時，明確地將名稱和圖示指派給會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-157">To ensure that Sndvol displays the client-provided name and icon, the client must explicitly assign the name and icon to the session at the time that the client assigns the first stream to the session.</span></span> <span data-ttu-id="6f066-158">系統只會保留會話的顯示名稱和圖示，直到會話終止為止。</span><span class="sxs-lookup"><span data-stu-id="6f066-158">The system retains the display name and icon for a session only until the session terminates.</span></span>

<span data-ttu-id="6f066-159">每個會話都是由會話 GUID 所識別。</span><span class="sxs-lookup"><span data-stu-id="6f066-159">Each session is identified by a session GUID.</span></span> <span data-ttu-id="6f066-160">當用戶端開啟串流時，用戶端會將該資料流程指派給特定的會話。</span><span class="sxs-lookup"><span data-stu-id="6f066-160">At the time that a client opens a stream, the client assigns that stream to a particular session.</span></span> <span data-ttu-id="6f066-161">用戶端會提供下列兩項資訊來識別該會話：</span><span class="sxs-lookup"><span data-stu-id="6f066-161">The client supplies the following two pieces of information to identify that session:</span></span>

-   <span data-ttu-id="6f066-162">會話 GUID。</span><span class="sxs-lookup"><span data-stu-id="6f066-162">A session GUID.</span></span>
-   <span data-ttu-id="6f066-163">無論會話是跨進程或進程特定的會話，進程特定的會話都只會包含來自用戶端進程的資料流程。</span><span class="sxs-lookup"><span data-stu-id="6f066-163">Whether the session is a cross-process or process-specific session—a process-specific session contains only streams from the client's process.</span></span>
