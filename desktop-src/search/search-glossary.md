## <a name="a"></a>A

**Advanced Query 語法 (AQS)**

Windows Search 用來查詢索引，以及精簡和縮小搜尋參數的預設查詢語法。 AQS 主要是使用者面向的，可供使用者用來建立 AQS 查詢，但也可以透過程式設計方式使用。 另請參閱： NQS) 的自然查詢語法 (。

**AQS**

請參閱： Advanced Query 語法 (AQS) 的定義。

**協會**

副檔名的對應 (例如 .mp3) 或通訊協定 (例如，HTTP) 至程式設計識別碼 (ProgID) 。 這項對應會儲存在登錄中，作為每一使用者的每一電腦的回復設定。 參與預設程式系統的應用程式，會設定副檔名或通訊協定的關聯對應，以指向它們所擁有的 ProgID 金鑰。

**關聯陣列**

登錄位置的排序清單，用來儲存專案類型的相關資訊，包括處理常式、動詞和其他屬性，例如類型的圖示和顯示名稱。 例如，.jpg 檔案在預設 Windows 系統上有下列關聯陣列： "HKCR \\ jpgfile"、"HKCR \\ SystemFileAssociations \\.jpg"、"HKCR \\ SystemFileAssociations \\ image"、"HKCR \\ \* "、"HKCR \\ AllFileSystemObjects"。

**Atom**

用於 web 摘要和內容發佈的 XML 架構，作為真正簡單新聞訂閱 (RSS) 的替代方案進行開發。 Atom 新聞訂閱格式已在 RFC 4287 中發佈為 IETF 建議的標準。

## <a name="b"></a>B

**bind**

載入或關聯程式碼與資料。 例如，處理常式可能會與 Shell 資料來源產生關聯。

**綁定**

在搜尋查詢中，針對傳回的資料列集內的資料行提出要求。 系結會指定要包含在搜尋結果中的屬性。

**bookmark**

在一組資料列中唯一識別資料列的指標。

## <a name="c"></a>C

**正式名稱**

資源的唯一名稱。 標準表示「根據規則」。 請參閱：標準動詞名稱。

**標準動詞名稱**

非語言相關名稱，不論使用者介面中的當地語系化字串為何，都可以用程式設計方式來參考動詞。 請參閱：標準名稱、動詞。

**catalog**

Windows Search 中最高層級的組織單位。 目錄代表一組可以查詢的索引檔。 目錄是由具有文字或值的屬性資料表，以及儲存在資料表資料行中的對應位置所組成。 資料表的每個資料列都會對應至目錄範圍中的個別檔，而且資料表的每個資料行都會對應至屬性。 請參閱： index、Windows Search 服務。

**類別**

階層式資料列群組。 例如，包含 author 和 title 資料行的查詢結果可以根據作者分類。 在此範例中，包含相同作者值的每個資料列群組都會構成一個類別目錄。

**章**

一組資料列內的資料列集合。 另請參閱：類別目錄、類別目錄。

**column**

資料列中單一資訊類型的容器。 資料行會對應至屬性名稱，並指定搜尋查詢的命令樹元素所使用的屬性。 另請參閱：類別目錄。

**Command Tree - 命令樹**

針對搜尋查詢所指定之限制、類別和排序次序的組合。 另請參閱：類別目錄。

**container**

可包含其他專案的 Shell 專案類型。 容器中的專案會使用 Shell 資料來源公開給 Shell 命名空間。 範例包括資料夾、磁片磁碟機、網路伺服器，以及副檔名為 .zip 的壓縮檔案。 另請參閱： Shell 資料來源、資料夾、Shell 專案。

**content**

與可編制索引之 Shell 專案或內容來源相關聯的文字和屬性。

**內容來源**

可由索引子存取的專案。 內容來源可透過 URL 定址，並由通訊協定處理常式提供給索引子。 範例包括：檔案系統檔案和資料夾、microsoft Outlook 專案和資料夾、資料庫記錄和 microsoft SharePoint 儲存的專案。 您可以藉由執行 Shell 資料來源，將內容來源公開為 Shell 專案。 另請參閱： content、Shell 專案。

**content view (內容檢視)**

Windows 7 和更新) 版本中提供的 Windows 檔案總管 (視圖，會根據檔案的副檔名或種類關聯，顯示清單中每個專案的最相關內容。 當視窗大小減少時，內容視圖會使用調整大小的邏輯，以確保最重要的屬性仍然具有可清楚讀取的空間。 另請參閱：版面配置模式、種類、種類關聯。

**內容視圖模式**

**內容功能表處理常式**

這個詞彙有時候是用來表示快速鍵功能表處理常式。 請參閱：快捷方式功能表處理常式的定義。

**搜耙**

**編目範圍**

資料存放區的集合 (可透過 URL) 識別，以代表索引子編目和索引的內容。

**cursor**

在本機索引的內容中，資料指標是一種指標，可在結果集中傳回的一組資料中，一次處理一個資料列或一個小型資料列區塊。 將資料指標置於資料列之後，就可以在該資料列或從該位置開始的資料列區塊上執行作業。

**副檔名**

檔案系統專案之檔案類型的主要指標，是最後一個點後面的檔案名部分。 副檔名不能包含空格或非 ASCII 字元，而且只會套用至)  (不是資料夾的檔案。 使用不區分大小寫或地區設定的比較函式來比較副檔名。 另請參閱：檔案格式、檔案類型。

**檔案類型處理常式**

為檔案類型註冊的處理常式。 另請參閱：處理常式。

**filter**

## <a name="p"></a>P

**PerceivedType**

廣泛分類的檔案格式類型。 PerceivedType 是在 Windows XP 中引進，並支援一組有限的已知檔案類型 (範例包括影像、文字、音訊和壓縮檔案類型) 。 檔案類型（通常是公用檔案類型）也可以有察覺的型別。 例如，影像檔案類型 .bmp、.png、.jpg 和 .gif 也是認知類型（影像）。 在程式設計層，PerceivedType 是以整數表示。 由於有使用種類和 PerceivedType 的程式碼，因此檔案格式擁有者必須註冊這兩者。 例如，「全部播放」取決於 PerceivedType。 另請參閱：檔案類型。

**預覽處理常式**

此處理程式可快速產生唯讀、簡單的 Shell 專案查看，以顯示在 Windows 檔案總管預覽窗格中。

**器**

這個詞彙有時候是用來表示預覽處理常式。 請參閱的定義：預覽處理常式。

**屬性處理常式**

將儲存在檔案中的資料轉譯成可辨識之結構化架構的處理常式，可由 Windows 檔案總管、Windows Search 和其他應用程式存取。 這些系統接著可以與屬性處理常式互動，以在檔案中寫入和讀取屬性。 轉譯的資料包含詳細資料檢視、提示、詳細資料窗格、屬性頁等等。 每個屬性處理常式都與特定檔案類型相關聯，該檔案類型是由副檔名所識別。 另請參閱：屬性系統。

**屬性工作表處理常式**

用來建立自訂屬性工作表的處理常式，該工作表包含允許與檔案類型進行自訂互動的 UI 圖片和控制項。

**屬性系統**

資料定義的可延伸讀取/寫入系統，使用實作為成對名稱/值的屬性。 另請參閱：屬性處理常式、Shell 專案。

**屬性值**

與 Shell 專案的屬性名稱相關聯的值。 例如，「作者」、「大小」和「拍攝日期」都是屬性。 屬性值會以 PROPVARIANT 結構表示。

**通訊協定處理常式**

存取內容來源並為指定的通訊協定和 URL 提供 IUrlAccessor 物件的處理常式。 通訊協定處理常式可延伸 Windows Search 的功能，並可提供變更通知給索引子。 需要不同的通訊協定處理常式來編制特定類型的資料存放區索引。 若要提供合理的使用者體驗，您也必須提供資料存放區的 Shell 資料來源，以及執行您的通訊協定處理常式。 通訊協定處理常式會將資料存放區中的專案公開給索引子，而 Shell 資料來源則會將資料存放區中的專案公開給 Shell。

## <a name="r"></a>R

**限制**

檔案必須符合才能包含在 Windows Search 所傳回之搜尋結果中的條件。

**row**

包含屬性值的資料行，這些值會描述項合搜尋查詢中所指定之限制的物件集合中的單一結果。

**rowset**

在搜尋結果中傳回的一組資料列。

## <a name="s"></a>S

**搜尋連接器**

XML 檔案，其中包含資料存放區的相關資訊。 已針對同盟搜尋部署搜尋連接器。

**搜尋取用者**

查詢索引的元件或應用程式。

**搜尋同盟**

請參閱：同盟搜尋提供者的定義。

**搜尋提供者**

提供資料給 Windows Search 的元件或應用程式。

**搜尋範圍**

此詞彙有時候是用來表示編目範圍。 請參閱：編目範圍的定義。

**Shell 資料來源**

用來擴充 Shell 命名空間和公開資料存放區中專案的元件。 在過去，Shell 資料來源稱為 Shell 命名空間延伸模組。 另請參閱：容器、處理常式、Shell 專案。

**Shell 延伸模組**

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

**Shell 擴充處理常式**

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

**Shell 處理常式**

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

**Shell 專案**

單一內容片段。 某些 Shell 專案為內容來源，有些則不是。 例如，一個資料夾是一個內容來源，但是 .jpg 檔案則不是。 檔案類型處理常式會公開 Shell 專案。 在某些內容中，會使用「專案」來區別容器與 noncontainers。 另請參閱：容器、內容來源、檔案類型處理常式。

**Shell 命名空間延伸模組**

這個詞彙有時候是用來表示 Shell 資料來源。 請參閱： Shell 資料來源的定義。

**快速鍵功能表**

使用者介面，用來顯示與使用者介面專案（例如檔案或資料夾）相關聯的動詞集合。

**快速鍵功能表處理常式**

為專案或專案加入動詞的處理常式。 這些動詞通常會顯示在快捷方式功能表中。 另請參閱：快捷方式功能表。

**靜態動詞**

適用于 Shell 專案的動詞，不需要檢查項目或系統的目前狀態。 靜態動詞命令是以專案之相關專案的靜態註冊為基礎，而且不會變更。

## <a name="t"></a>T

**縮圖處理常式**

提供靜態影像以表示 Shell 專案的處理常式。

**縮圖提供者**

這個詞彙有時候是用來表示縮圖處理常式。 請參閱：縮圖處理常式的定義。

## <a name="u"></a>U

**URL template**

以 URL 為基礎的連接字串，用來查詢 web 伺服器的搜尋結果。 範本看起來像 URL，但包含數個預留位置值 (例如 {searchTerms} ) ，用戶端必須將其取代為要抓取之結果的相關資料。 URL 範本的定義是執行同盟搜尋和 OpenSearch 標準的關鍵。

**使用者易記種類名稱**

請參閱：種類的定義。

## <a name="v"></a>V

**動詞**

可由 Shell 專案呼叫的個別動作。 範例包括 [開啟] 和 [列印]。 動詞有時也稱為命令或 task。 另請參閱：動態動詞、快捷方式功能表處理常式、靜態動詞。

**動詞處理常式**

這個詞彙有時候是用來表示快速鍵功能表處理常式。 請參閱：快捷方式功能表處理常式的定義。

## <a name="w"></a>W

**走**

請參閱：命名空間逐步解說的定義。

**Windows 搜尋**

請參閱： Windows Search 服務的定義。

**Windows搜尋屬性存放區**

在 Windows Search 服務的執行中所使用之屬性值的快取。 您可以使用 Windows Search OLE DB 提供者，以程式設計方式查詢這些屬性值。 當編制檔（例如 Word 檔）的索引時，Windows Search 屬性存放區會收集並儲存由篩選處理常式或屬性處理常式所發出的屬性。 重建索引時，會捨棄並重建此存放區。

**Windows搜尋服務**

是指 Windows Search 3.0 和更新版本。 這項服務會分析一組檔、解壓縮有用的資訊，然後組織解壓縮的資訊，以便有效率地傳回這些檔的屬性來回應查詢。 另請參閱：目錄。
