# <a name="languages-supported-by-windows-search"></a><span data-ttu-id="87e38-103">Windows Search 支援的語言</span><span class="sxs-lookup"><span data-stu-id="87e38-103">Languages Supported by Windows Search</span></span>

<span data-ttu-id="87e38-104">本主題說明 Windows Search 如何支援多種語言。</span><span class="sxs-lookup"><span data-stu-id="87e38-104">This topic describes how Windows Search supports multiple languages.</span></span>

## <a name="tokenization-wordbreakers-and-language-resources"></a><span data-ttu-id="87e38-105">Token 化、文字分隔和語言資源</span><span class="sxs-lookup"><span data-stu-id="87e38-105">Tokenization, Wordbreakers, and Language Resources</span></span>

<span data-ttu-id="87e38-106">Windows Search 與語言無關，但是跨語言搜尋的精確度可能因文字分隔 token 化文字的方式而有所不同。</span><span class="sxs-lookup"><span data-stu-id="87e38-106">Windows Search is language-independent, but the accuracy of search across languages may vary because of the way wordbreakers tokenize text.</span></span> <span data-ttu-id="87e38-107">文字分隔會針對語言執行各種 token 化規則，並將文字分成個別的權杖或單字，以進行編制索引或搜尋。</span><span class="sxs-lookup"><span data-stu-id="87e38-107">Wordbreakers implement various tokenization rules for languages and break text into individual tokens, or words, to be indexed or searched.</span></span>

<span data-ttu-id="87e38-108">索引文字和查詢字串的語言都會分成標記。</span><span class="sxs-lookup"><span data-stu-id="87e38-108">Both the language of the indexed text and the query string are broken into tokens.</span></span> <span data-ttu-id="87e38-109">由於 token 化規則會因語言而異，因此每個語言或語言系列都有個別的文字分隔。</span><span class="sxs-lookup"><span data-stu-id="87e38-109">Because tokenization rules vary by language, there are separate wordbreakers for each language or family of languages.</span></span> <span data-ttu-id="87e38-110">如果查詢語言與索引語言不相符，則結果可能無法預測。</span><span class="sxs-lookup"><span data-stu-id="87e38-110">If there is a mismatch between the query language and the indexed language, the results can be unpredictable.</span></span>

<span data-ttu-id="87e38-111">Windows Search 隨附一組妥善定義的文字分隔。</span><span class="sxs-lookup"><span data-stu-id="87e38-111">Windows Search ships with a well defined set of wordbreakers.</span></span> <span data-ttu-id="87e38-112">Windows Vista 和更新版本支援傳統的斷詞工具和字幹分析器元件。</span><span class="sxs-lookup"><span data-stu-id="87e38-112">Classic wordbreaker and stemmer components are supported in Windows Vista and later.</span></span> <span data-ttu-id="87e38-113">如果無法判斷檔的語言，Windows Search 會嘗試偵測語言以找出最適當的斷詞工具。</span><span class="sxs-lookup"><span data-stu-id="87e38-113">If the language of a document cannot be determined, Windows Search attempts to detect the language to identify the most appropriate wordbreaker.</span></span> <span data-ttu-id="87e38-114">Windows Search 嘗試偵測語言，方法是呼叫 [**GetSystemPreferredUILanguages**](/windows/win32/api/winnls/nf-winnls-getsystempreferreduilanguages) 函式來判斷前多個消費者介面 (的 mui) 語言 (通常是系統 UI 語言，除非) 安裝 MUI 語言套件。</span><span class="sxs-lookup"><span data-stu-id="87e38-114">Windows Search attempts to detect the language by calling the [**GetSystemPreferredUILanguages**](/windows/win32/api/winnls/nf-winnls-getsystempreferreduilanguages) function to determine the first Multiple User Interface (MUI) language (which is typically the system UI language unless MUI language packs are installed).</span></span> <span data-ttu-id="87e38-115">如果該呼叫成功，則會使用第一個 MUI 語言的斷詞工具。</span><span class="sxs-lookup"><span data-stu-id="87e38-115">If that call succeeds, the wordbreaker for the first MUI language is used.</span></span> <span data-ttu-id="87e38-116">如果呼叫 **GetSystemPreferredUILanguages** 失敗，Windows Search 會呼叫 [**GetSystemDefaultLCID**](/windows/win32/api/winnls/nf-winnls-getsystemdefaultlcid) 函式，並使用與該地區設定相關聯的斷詞工具，藉以抓取系統地區設定。</span><span class="sxs-lookup"><span data-stu-id="87e38-116">If the call to **GetSystemPreferredUILanguages** fails, Windows Search retrieves the system locale by calling the [**GetSystemDefaultLCID**](/windows/win32/api/winnls/nf-winnls-getsystemdefaultlcid) function and uses the wordbreaker associated with that locale.</span></span>

<span data-ttu-id="87e38-117">如果未安裝語言的斷詞工具，Windows Search 使用 **中性** 的斷詞工具來中斷空白字元。</span><span class="sxs-lookup"><span data-stu-id="87e38-117">If no wordbreaker is installed for a language, Windows Search breaks on white space by using the **Neutral** wordbreaker.</span></span>

<span data-ttu-id="87e38-118">您可以透過登錄移除語言，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="87e38-118">You can remove a language through the registry, as illustrated in the following example.</span></span>

```
            ContentIndex
               Language
                  Dutch_Dutch
                     (Default)
                     Locale
                     NoiseFile
                     StemmerClass = CLSID
                     WBreakerClass = CLSID
```

> <span data-ttu-id="87e38-119">如果您對登錄進行變更，請重新開機 Windows Search。</span><span class="sxs-lookup"><span data-stu-id="87e38-119">If you make changes to the registry, restart Windows Search.</span></span>

 

<span data-ttu-id="87e38-120">當 Windows Search 需要新的斷詞工具時，會讀取 (CLSID) 的類別識別碼，並快取具現化的斷詞工具。</span><span class="sxs-lookup"><span data-stu-id="87e38-120">When Windows Search requires a new wordbreaker, the class identifier (CLSID) is read, and the instantiated wordbreaker is cached.</span></span>

<span data-ttu-id="87e38-121">您可以藉由執行 [**IWordBreaker**](/windows/desktop/api/Indexsrv/nn-indexsrv-iwordbreaker) 介面，建立語言的自訂斷詞工具。</span><span class="sxs-lookup"><span data-stu-id="87e38-121">You can create a custom wordbreaker for a language by implementing the [**IWordBreaker**](/windows/desktop/api/Indexsrv/nn-indexsrv-iwordbreaker) interface.</span></span> <span data-ttu-id="87e38-122">Windows Search 接著會在建立內容索引和執行查詢時，呼叫 **IWordBreaker** 方法。</span><span class="sxs-lookup"><span data-stu-id="87e38-122">Windows Search then calls the **IWordBreaker** methods when it builds content indexes and runs queries.</span></span>

<span data-ttu-id="87e38-123">索引內容的地區設定資訊是從內容來源抓取。</span><span class="sxs-lookup"><span data-stu-id="87e38-123">Locale information for indexed content is retrieved from the source of the content.</span></span> <span data-ttu-id="87e38-124">如果來源實施者不知道已編制索引之內容的地區設定，則應該將地區設定設定為 [**\_ 中性地區**](../intl/locale-neutral.md)設定。</span><span class="sxs-lookup"><span data-stu-id="87e38-124">If the source implementer does not know the locale of the indexed content, it should set the locale to [**LOCALE\_NEUTRAL**](../intl/locale-neutral.md).</span></span>

<span data-ttu-id="87e38-125">例如，如果您 () 、屬性處理常式或通訊協定處理常式的 [**IFilter**](https://www.bing.com/search?q=**IFilter**) 介面執行的篩選處理常式，則應該將已編制索引的內容地區設定設定為 [**\_ 中性地區**](../intl/locale-neutral.md) 設定，除非您有特定的地區設定資訊，並確信其精確度。</span><span class="sxs-lookup"><span data-stu-id="87e38-125">For example, if you implement a filter handler (an implementation of the [**IFilter**](https://www.bing.com/search?q=**IFilter**) interface), property handler, or protocol handler, you should set the locale for indexed content to [**LOCALE\_NEUTRAL**](../intl/locale-neutral.md) unless you have specific locale information and are confident of its accuracy.</span></span>

> [!TIP]
> <span data-ttu-id="87e38-126">如果索引查詢是以使用者輸入為基礎，則地區設定應該符合使用者輸入的語言。</span><span class="sxs-lookup"><span data-stu-id="87e38-126">If an index query is based on user input, the locale should match the language in which the user is typing.</span></span> <span data-ttu-id="87e38-127">您可以藉由呼叫 [**GetKeyboardLayout**](/windows/win32/api/winuser/nf-winuser-getkeyboardlayout) 函數來判斷此地區設定。</span><span class="sxs-lookup"><span data-stu-id="87e38-127">You can determine this locale by calling the [**GetKeyboardLayout**](/windows/win32/api/winuser/nf-winuser-getkeyboardlayout) function.</span></span>
