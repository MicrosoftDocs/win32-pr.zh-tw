## <a name="slash-direction-trailing-star-and-trailing-slash-sensitivity"></a><span data-ttu-id="1c6dc-139">斜線方向、尾端星號和結尾的斜線敏感度</span><span class="sxs-lookup"><span data-stu-id="1c6dc-139">Slash Direction, Trailing Star, and Trailing Slash Sensitivity</span></span>

<span data-ttu-id="1c6dc-140">在 Windows Search 的斜線方向幾乎沒有任何機密性。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-140">In Windows Search there is largely no sensitivity to slash direction.</span></span> <span data-ttu-id="1c6dc-141">如果接受格式 `c:\test\example` ，則也會接受 c：/test/example。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-141">If the format `c:\test\example` is accepted, then c:/test/example is accepted as well.</span></span> <span data-ttu-id="1c6dc-142">不過，雖然 [範圍](-search-sql-folderdepth.md) 通常不區分斜線方向，但在遠端 URL 格式 F 的情況下，它對斜線方向是敏感的。因此， `Scope = '//server/share'` 無法運作。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-142">However, although [SCOPE](-search-sql-folderdepth.md) is generally insensitive to slash direction, it is sensitive to the slash direction in the case of remote URL format F. Hence, `Scope = '//server/share'` does not work.</span></span>

<span data-ttu-id="1c6dc-143">唯一可區分尾端星星並區分 `c:\test\ ` 和 `c:\test\*` 的 API 是 [**ISearchCrawlScopeManager**](/windows/desktop/api/Searchapi/nn-searchapi-isearchcrawlscopemanager)。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-143">The only API that is sensitive to trailing stars and distinguishes between `c:\test\ ` and `c:\test\*` is [**ISearchCrawlScopeManager**](/windows/desktop/api/Searchapi/nn-searchapi-isearchcrawlscopemanager).</span></span> <span data-ttu-id="1c6dc-144">如果有排除規則 `c:\test\*` ，則 URL 目錄 `c:\test` 本身仍會編制索引。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-144">If there is an exclusion rule for `c:\test\*`, the URL directory `c:\test` itself will still be indexed.</span></span> <span data-ttu-id="1c6dc-145">但是，如果排除 URL 是 `c:\test\` ，則不會 `c:\test` 編制 url 目錄本身的索引。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-145">But if the exclusion URL is `c:\test\`, the URL directory `c:\test` itself will not be indexed.</span></span>

<span data-ttu-id="1c6dc-146">有兩個地方 Windows Search 可區分尾端斜線： ItemUrl 和 Path 查詢。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-146">There are two places where Windows Search is sensitive to trailing slashes: ItemUrl and Path queries.</span></span> <span data-ttu-id="1c6dc-147">如果有目錄 `c:\test` ，Windows Search 會將與 `c:\test\` `c:\test` 和類似的述詞視為不同 `path = 'c:\test'` `System.ItemUrl = 'c:\test'` 。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-147">If there is a directory `c:\test`, Windows Search treats `c:\test\` differently from `c:\test` for predicates like `path = 'c:\test'` and `System.ItemUrl = 'c:\test'`.</span></span> <span data-ttu-id="1c6dc-148">例如，此述詞 `path='file:c:/test'` 會比對目錄 `c:\test` ，但 `path='file:c:/test/'` 不會因為尾端斜線而不相符。</span><span class="sxs-lookup"><span data-stu-id="1c6dc-148">For example, the predicate `path='file:c:/test'` would match the directory `c:\test`, but `path='file:c:/test/'` would not, due to the trailing slash.</span></span>
