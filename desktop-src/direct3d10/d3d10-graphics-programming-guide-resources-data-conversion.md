## <a name="floating-point-conversion"></a><span data-ttu-id="c3a20-140">浮點轉換</span><span class="sxs-lookup"><span data-stu-id="c3a20-140">Floating Point Conversion</span></span>

## <a name="fixed-point-integer-conversion"></a><span data-ttu-id="c3a20-249">固定點整數轉換</span><span class="sxs-lookup"><span data-stu-id="c3a20-249">Fixed Point Integer Conversion</span></span>

<span data-ttu-id="c3a20-250">固定點整數單純是某個位元大小的整數，其中某個固定位置有隱含的小數點。</span><span class="sxs-lookup"><span data-stu-id="c3a20-250">Fixed point integers are simply integers of some bit size that have an implicit decimal point at a fixed location.</span></span>

<span data-ttu-id="c3a20-251">普遍的「整數」資料類型是固定點整數的特殊案例，小數點位於數字結尾。</span><span class="sxs-lookup"><span data-stu-id="c3a20-251">The ubiquitous "integer" data type is a special case of a fixed point integer with the decimal at the end of the number.</span></span>

<span data-ttu-id="c3a20-252">固定點數字表示法描述如下：i.f，其中 i 是整數位元數，f 是分數位元數。</span><span class="sxs-lookup"><span data-stu-id="c3a20-252">Fixed point number representations are characterized as: i.f, where i is the number of integer bits and f is the number of fractional bits.</span></span> <span data-ttu-id="c3a20-253">例如 16.8 表示 16 位元整數，後面接著 8 位元分數。</span><span class="sxs-lookup"><span data-stu-id="c3a20-253">e.g. 16.8 means 16 bits integer followed by 8 bits of fraction.</span></span> <span data-ttu-id="c3a20-254">整數部分是以 2 的補數儲存，至少如此處鎖定一 (雖然它同樣可以對不帶正負號的整數定義)。</span><span class="sxs-lookup"><span data-stu-id="c3a20-254">The integer part is stored in 2's complement, at least as defined here (though it can be defined equally for unsigned integers as well).</span></span> <span data-ttu-id="c3a20-255">分數部分是以不帶正負號的形式儲存。</span><span class="sxs-lookup"><span data-stu-id="c3a20-255">The fractional part is stored in unsigned form.</span></span> <span data-ttu-id="c3a20-256">分數部分一律代表正分數，介於最接近的兩個整數值之間，從最大負數開始。</span><span class="sxs-lookup"><span data-stu-id="c3a20-256">The fractional part always represents the positive fraction between the two nearest integral values, starting from the most negative.</span></span>

<span data-ttu-id="c3a20-257">固定點數字的加減運算是單純使用標準整數算術執行，不考慮隱含的小數點位置。</span><span class="sxs-lookup"><span data-stu-id="c3a20-257">Addition and subtraction operations on fixed point numbers are performed simply using standard integer arithmetic, without any consideration for where the implied decimal lies.</span></span> <span data-ttu-id="c3a20-258">將 16.8 固定點數字加 1，僅表示加上 256，因為小數點在數字最低有效結尾起算的第 8 位。</span><span class="sxs-lookup"><span data-stu-id="c3a20-258">Adding 1 to a 16.8 fixed point number just means adding 256, since the decimal is 8 places in from the least significant end of the number.</span></span> <span data-ttu-id="c3a20-259">其他運算像是乘法，同樣可以單純使用整數算術執行，假設考慮對固定小數點的效果。</span><span class="sxs-lookup"><span data-stu-id="c3a20-259">Other operations such as multiplication, can be performed as well simply using integer arithmetic, provided the effect on the fixed decimal is accounted for.</span></span> <span data-ttu-id="c3a20-260">例如，使用整數乘法將兩個 16.8 整數相乘，會得到 32.16 的結果。</span><span class="sxs-lookup"><span data-stu-id="c3a20-260">For example, multiplying two 16.8 integers using an integer multiply produces a 32.16 result.</span></span>

<span data-ttu-id="c3a20-261">固定點整數表示法在 Direct3D 中使用的方式有兩種。</span><span class="sxs-lookup"><span data-stu-id="c3a20-261">Fixed point integer representations are used in two ways in Direct3D.</span></span>

-   <span data-ttu-id="c3a20-264">取樣作業的紋理座標會貼齊固定點 (經過紋理大小縮放後)，以跨紋理空間統一分配精確度，以便選擇篩選點選位置/權重。</span><span class="sxs-lookup"><span data-stu-id="c3a20-264">Texture coordinates for sampling operations are snapped to fixed point (after being scaled by texture size), to uniformly distribute precision across texture space, in choosing filter tap locations/weights.</span></span> <span data-ttu-id="c3a20-265">權重值會在進行實際篩選算術之前轉換回浮點數。</span><span class="sxs-lookup"><span data-stu-id="c3a20-265">Weight values are converted back to floating point before actual filtering arithmetic is performed.</span></span>
