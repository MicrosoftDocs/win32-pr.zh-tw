# <a name="s-network-monitor"></a>S (網路監視器) 

<dl> <dt>

<span id="_netmon_sap_gly"></span><span id="_NETMON_SAP_GLY"></span>**Sap**
</dt> <dd>

請參閱服務存取點。

</dd> <dt>

<span id="_netmon_server_message_block_gly"></span><span id="_NETMON_SERVER_MESSAGE_BLOCK_GLY"></span>**伺服器訊息區 (SMB)**
</dt> <dd>

包含基本專案的檔案層級通訊協定，用於 **開啟**、 **關閉**、 **讀取** 和 **寫入** 等作業。

</dd> <dt>

<span id="_netmon_service_access_point_gly"></span><span id="_NETMON_SERVICE_ACCESS_POINT_GLY"></span>**(SAP) 的服務存取點**
</dt> <dd>

框架識別碼。

</dd> <dt>

<span id="_netmon_session_gly"></span><span id="_NETMON_SESSION_GLY"></span>**會話**
</dt> <dd>

一組資料封包，一組會在捕捉期間傳送至另一個工作站。 會話是封包的單向交換，其中每個會話都有一個擁有者 (傳送) 的資料封包的站，以及) 接收資料封包的站 (的夥伴。 另請參閱會話 \_ 資訊。

</dd> <dt>

<span id="_netmon_session_information_gly"></span><span id="_NETMON_SESSION_INFORMATION_GLY"></span>**會話資訊**
</dt> <dd>

描述會話的資訊。 網路監視器在資料捕獲期間儲存會話資訊。 會話資訊稱為對話統計資料，可透過呼叫 **GetConversationStatistics** 方法來抓取。 另請參閱會話、 [*捕獲*](c.md)和 [*對話 \_ 統計資料*](c.md)。

</dd> <dt>

<span id="_netmon_smb_gly"></span><span id="_NETMON_SMB_GLY"></span>**Smb**
</dt> <dd>

請參閱伺服器訊息區。

</dd> <dt>

<span id="_netmon_source_address_gly"></span><span id="_NETMON_SOURCE_ADDRESS_GLY"></span>**來源位址**
</dt> <dd>

唯一的十六進位數位，可識別從中傳送資料的來源電腦。

</dd> <dt>

<span id="_netmon_special_blobs_gly"></span><span id="_NETMON_SPECIAL_BLOBS_GLY"></span>**特殊 BLOB**
</dt> <dd>

網路封包提供者 (NPP) 或篩選二進位大型物件 (BLOB) ，其中包含其他 Blob 未提供的補充資訊。 另請參閱 [*網路封包提供者*](n.md) 和 [*BLOB*](b.md)。

</dd> <dt>

<span id="_netmon_station_gly"></span><span id="_NETMON_STATION_GLY"></span>**站**
</dt> <dd>

在正在捕獲資料的電腦上安裝特定的 NIC。 另請參閱站資訊。

</dd> <dt>

<span id="_netmon_station_information_gly"></span><span id="_NETMON_STATION_INFORMATION_GLY"></span>**站資訊**
</dt> <dd>

描述工作站的資訊。 網路監視器在資料捕獲期間儲存站資訊。 站資訊也稱為對話統計資料，您可以藉由呼叫 **GetConversationStatistics** 方法來取得這些統計資料。 另請參閱工作站和 [*對話統計資料*](c.md)。

</dd> <dt>

<span id="_netmon_summary_property_gly"></span><span id="_NETMON_SUMMARY_PROPERTY_GLY"></span>**摘要屬性**
</dt> <dd>

剖析器定義的屬性，用來將通訊協定的屬性分組，以顯示在網路監視器 UI 中。 **PROPERTYINFO** 結構會以定義其他剖析器定義屬性的相同方式定義摘要屬性。 當網路監視器呼叫 **AttachProperties** 函式時，[摘要] 屬性通常是附加至框架的第一個屬性定義。

</dd> <dt>

<span id="_netmon_summary_pane_gly"></span><span id="_NETMON_SUMMARY_PANE_GLY"></span>**摘要窗格**
</dt> <dd>

網路監視器 [框架檢視器] 視窗中的窗格，其中列出捕捉檔案中包含的畫面格。 針對每個畫面格，[摘要] 窗格會列出框架號碼、框架的拍攝時間、框架的來源和目的地位址、用於傳送框架的通訊協定，以及框架的簡短描述。 如果原始位址有相關聯的 IP 或 IPX/XNS 位址，這個位址會顯示在 **Src 其他** 位址和 **其他 Dst** 的位址中。網址類別型 (IP 或 IPX/XNS) 會出現在 **類型 [其他位址**] 中。

</dd> </dl>
