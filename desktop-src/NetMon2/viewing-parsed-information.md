# <a name="viewing-parsed-information"></a>查看剖析的資訊

網路監視器的框架檢視器會以三個窗格顯示剖析的資料。 上方窗格會顯示包含已識別之通訊協定的框架摘要。 中間的窗格會顯示格式化的資料屬性，這些屬性可以在剖析器顯示中以階層方式組織。 下方的窗格會顯示從中間窗格選取的反白顯示原始資料。

您可以指定當您開發自己的剖析器時，如何顯示檢視器中的資訊。 下圖顯示如何在網路監視器框架檢視器中顯示資訊。

![網路監視器框架檢視器](images/parseui.png)

> [!Note]  
> 剖析器不會顯示畫面格。 剖析器會辨識所提供資訊中的欄位，然後通知網路監視器顯示畫面格。 篩選是較高層級的作業，可讓篩選查詢跨越剖析器。

 

在您的剖析器中，請只使用可在 Microsoft Win32 應用程式上執行的函式。 將屬性附加至原始資料之前，剖析器必須先向網路監視器核心註冊所有可能的屬性。 剖析器會將訊息傳送至核心以建立屬性資料庫，然後使用其通訊協定的每個可能屬性來填滿屬性資料庫。 屬性資料庫中的每個屬性都包含資訊，例如文字描述、用來格式化原始資料的資料類型和辨識符號，以及用來顯示資料的格式化常式。
