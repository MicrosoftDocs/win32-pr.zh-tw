# <a name="parser"></a>剖析器

剖析器是在 [*延遲的捕獲*](d.md)中檢查資料的網路監視器元件，並將特定的通訊協定資訊傳遞給呼叫剖析器的應用程式。 剖析器是被動的，因為它只會在網路監視器或 [*專家*](e.md) 呼叫它時運作。

每個剖析器都會識別一個通訊協定，而且通常會在它自己的剖析器 DLL 內實作為剖析器。 不過，剖析器 DLL 可以包含多個剖析器，這表示一個 DLL 可用來偵測一個以上的通訊協定。

傳遞給剖析器的資料是取自延遲的 [*捕獲*](d.md)，並以每個畫面格為基礎傳遞給剖析器。 您無法剖析即時捕捉。

若要剖析框架中的資料，剖析器必須辨識通訊協定實例、識別存在於通訊協定實例中的屬性，並將屬性定義附加至每個屬性。 請注意，框架只會包含資料的資料流程。 此框架不包含指出資料所代表之通訊協定或通訊協定屬性的資料。

下圖顯示包含通訊協定實例的框架。

![包含通訊協定實例的框架](images/parser1.png)

如果網路監視器要在 UI 中顯示剖析的資料，剖析器必須將資料格式化。 不過，某些專家會以程式設計方式使用剖析器輸出，而不會在網路監視器 UI 中顯示輸出。 顯示的資料包含剖析器定義的資料，以及捕捉中的資料。 例如，剖析器通常會提供所顯示內容的名稱，以及捕捉中與屬性相關聯的資料。
