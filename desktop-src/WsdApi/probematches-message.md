# <a name="probematches-message"></a><span data-ttu-id="66a3a-103">ProbeMatches 訊息</span><span class="sxs-lookup"><span data-stu-id="66a3a-103">ProbeMatches Message</span></span>

<span data-ttu-id="66a3a-104">ProbeMatches 訊息是服務為了回應用戶端的 [探查](probe-message.md) 訊息而傳送的 WS-Discovery 訊息。</span><span class="sxs-lookup"><span data-stu-id="66a3a-104">A ProbeMatches message is a WS-Discovery message sent by a service in response to a client's [Probe](probe-message.md) message.</span></span> <span data-ttu-id="66a3a-105">如需 ProbeMatches 訊息的詳細資訊，請參閱 [WS 探索規格](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf)的5.3 節。</span><span class="sxs-lookup"><span data-stu-id="66a3a-105">For more information about ProbeMatches messages, see section 5.3 of the [WS-Discovery Specification](https://specs.xmlsoap.org/ws/2005/04/discovery/ws-discovery.pdf).</span></span>

<span data-ttu-id="66a3a-106">ProbeMatches 訊息會由 UDP 單播傳送到傳送用戶端 [探查](probe-message.md) 訊息的埠。</span><span class="sxs-lookup"><span data-stu-id="66a3a-106">A ProbeMatches message is sent by UDP unicast to the port from which the client's [Probe](probe-message.md) message was sent.</span></span> <span data-ttu-id="66a3a-107">ProbeMatches 必須在探查訊息的4秒內傳送;否則，Windows 防火牆可能會捨棄封包。</span><span class="sxs-lookup"><span data-stu-id="66a3a-107">ProbeMatches must be sent within 4 seconds of the Probe message; otherwise, Windows Firewall may drop the packet.</span></span>

<span data-ttu-id="66a3a-108">如果 ProbeMatches 訊息中未包含任何 XAddrs，則用戶端可能會透過 UDP 多播將 [解析](resolve-message.md) 訊息傳送到埠3702。</span><span class="sxs-lookup"><span data-stu-id="66a3a-108">If no XAddrs are included in the ProbeMatches message, then the client may send a [Resolve](resolve-message.md) message by UDP multicast to port 3702.</span></span> <span data-ttu-id="66a3a-109">用戶端只會在 HTTP 訊息 (（例如 [Get](get--metadata-exchange--http-request-and-message.md) metadata exchange 要求或) 傳送的服務訊息）時傳送解析訊息。</span><span class="sxs-lookup"><span data-stu-id="66a3a-109">The client will only send a Resolve message when an HTTP message (such as a [Get](get--metadata-exchange--http-request-and-message.md) metadata exchange request or a service message) will be sent.</span></span>

<span data-ttu-id="66a3a-110">任何傳送 [探查](probe-message.md) 訊息的 DPWS 應用程式都會收到 ProbeMatches 訊息。</span><span class="sxs-lookup"><span data-stu-id="66a3a-110">Any DPWS application that sends [Probe](probe-message.md) messages will receive ProbeMatches messages.</span></span>

> [!Note]  
> <span data-ttu-id="66a3a-111">本主題說明 WSDAPI 用戶端和主機所產生的 DPWS 訊息範例。</span><span class="sxs-lookup"><span data-stu-id="66a3a-111">This topic shows a sample DPWS message generated by WSDAPI clients and hosts.</span></span> <span data-ttu-id="66a3a-112">WSDAPI 會剖析並接受其他不符合此範例的 DPWS 相容訊息。</span><span class="sxs-lookup"><span data-stu-id="66a3a-112">WSDAPI will parse and accept other DPWS-compliant messages that do not conform to this sample.</span></span> <span data-ttu-id="66a3a-113">請勿使用此範例來確認 DPWS 互通性;請改用 [ [WSDAPI 基本互通性] 工具 (WSDBIT) ](https://msdn.microsoft.com/library/cc264250.aspx) 。</span><span class="sxs-lookup"><span data-stu-id="66a3a-113">Do not use this sample to verify DPWS interoperability; use the [WSDAPI Basic Interoperability Tool (WSDBIT)](https://msdn.microsoft.com/library/cc264250.aspx) instead.</span></span>

 

<span data-ttu-id="66a3a-114">下列 SOAP 訊息會顯示範例 ProbeMatches 訊息。</span><span class="sxs-lookup"><span data-stu-id="66a3a-114">The following SOAP message shows a sample ProbeMatches message.</span></span>

``` syntax
<?xml version="1.0" encoding="utf-8" ?>
<soap:Envelope
    xmlns:soap="https://www.w3.org/2003/05/soap-envelope"
    xmlns:wsa="https://schemas.xmlsoap.org/ws/2004/08/addressing"
    xmlns:wsd="https://schemas.xmlsoap.org/ws/2005/04/discovery"
    xmlns:wsdp="https://schemas.xmlsoap.org/ws/2006/02/devprof">
<soap:Header>
    <wsa:To>
        https://schemas.xmlsoap.org/ws/2004/08/addressing/role/anonymous
    </wsa:To>
    <wsa:Action>
        https://schemas.xmlsoap.org/ws/2005/04/discovery/ProbeMatches
    </wsa:Action>
    <wsa:MessageID>
        urn:uuid:967d0036-fe69-40ad-8191-dd1fc8ef64ab
    </wsa:MessageID>
    <wsa:RelatesTo>
        urn:uuid:29cf10da-5c41-4d55-b184-5ee15e38ce23
    </wsa:RelatesTo>
    <wsd:AppSequence InstanceId="1"
        SequenceId="urn:uuid:369a7d7b-5f87-48a4-aa9a-189edf2a8772"
        MessageNumber="9">
    </wsd:AppSequence>
</soap:Header>
<soap:Body>
    <wsd:ProbeMatches>
        <wsd:ProbeMatch>
            <wsa:EndpointReference>
                <wsa:Address>
                    urn:uuid:37f86d35-e6ac-4241-964f-1d9ae46fb366
                </wsa:Address>
            </wsa:EndpointReference>
            <wsd:Types>wsdp:Device</wsd:Types>
            <wsd:XAddrs>
                https://192.168.0.2:5357/37f86d35-e6ac-4241-964f-1d9ae46fb366
            </wsd:XAddrs>
            <wsd:MetadataVersion>2</wsd:MetadataVersion>
        </wsd:ProbeMatch>
    </wsd:ProbeMatches>
</soap:Body>
</soap:Envelope>
```
