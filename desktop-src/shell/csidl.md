# <a name="csidl"></a>CSIDL

> [!Note]  
> 在 Windows Vista 中，這些值已由 [**KNOWNFOLDERID**](knownfolderid.md) 值取代。 如需新的常數及其對應的 CSIDL 值清單，請參閱該主題。 為了方便起見，此處也會針對每個 CSIDL 值注明對應的 **KNOWNFOLDERID** 值。
>
> 基於相容性因素，Windows Vista 支援 CSIDL 系統。 不過，新的開發應該使用 [**KNOWNFOLDERID**](knownfolderid.md) 值，而不是 CSIDL 值。

 

CSIDL (固定的特殊專案識別碼清單) 值提供獨特的系統獨立方式來識別應用程式經常使用的特殊資料夾，但在任何指定的系統上可能不會有相同的名稱或位置。 例如，系統資料夾在一個系統上可能是 "C： \\ Windows"，另一個則為 "c： \\ Winnt"。 這些常數定義于 Shlobj.h 中。

## <a name="remarks"></a>備註

這些值會取代環境變數的用途。 它們接著會在 Windows Vista 和更新版本中由 [**KNOWNFOLDERID**](knownfolderid.md) 值取代。

CSIDL 可搭配四個 Shell 函式（ [**SHGetFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderlocation)、 [**SHGetFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetfolderpatha)、 [**SHGetSpecialFolderLocation**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderlocation)和 [**SHGetSpecialFolderPath**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shgetspecialfolderpatha)）中的其中一個使用，以抓取特殊資料夾的路徑或 (PIDL) 的專案識別碼清單指標。

結合 * * * * CSIDL \_ 旗標 \_ CREATE * * * 與其他任何 CSIDLs，但 * * * * CSIDL 旗標沒有 \_ \_ 驗證 * * * *，則會 \_ 強制建立相關聯的資料夾。 其餘的 CSIDLs 會對應至檔系統資料夾或虛擬資料夾。 CSIDL 識別檔系統資料夾時，會提供常用的路徑作為範例。 您可以使用其他路徑。 某些 CSIDLs 可以對應至對等的 *% VariableName%* 環境變數。 不過，CSIDLs 更可靠，而且應該盡可能使用。
