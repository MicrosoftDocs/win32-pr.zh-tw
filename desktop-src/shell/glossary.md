# <a name="shell-glossary"></a>Shell 詞彙

## <a name="a"></a>A

<dl> <dt>

**協會**
</dt> <dd>

副檔名的對應 (例如 .mp3) 或通訊協定 (例如，HTTP) 至程式設計識別碼 (ProgID) 。 這項對應會儲存在登錄中，作為每一使用者的每一電腦的回復設定。 參與預設程式系統的應用程式，會設定副檔名或通訊協定的關聯對應，以指向它們所擁有的 ProgID 金鑰。

</dd> <dt>

**關聯陣列**
</dt> <dd>

登錄位置的排序清單，用來儲存專案類型的相關資訊，包括處理常式、動詞和其他屬性，例如類型的圖示和顯示名稱。 例如，.jpg 檔案在預設 Windows 系統上有下列關聯陣列： "HKCR \\ jpgfile"、"HKCR \\ SystemFileAssociations \\.jpg"、"HKCR \\ SystemFileAssociations \\ image"、"HKCR \\ \* "、"HKCR \\ AllFileSystemObjects"。

</dd> </dl>

## <a name="b"></a>B

<dl> <dt>

**bind**
</dt> <dd>

載入或關聯程式碼與資料。 例如，處理常式可能會與 Shell 資料來源產生關聯。

</dd> </dl>

## <a name="c"></a>C

<dl> <dt>

**正式名稱**
</dt> <dd>

資源的唯一名稱。 標準表示「根據規則」。 請參閱：標準動詞名稱。

</dd> <dt>

**標準動詞名稱**
</dt> <dd>

非語言相關名稱，不論使用者介面中的當地語系化字串為何，都可以用程式設計方式來參考動詞。 請參閱：標準名稱、動詞。

</dd> <dt>

**container**
</dt> <dd>

可包含其他專案的 Shell 專案類型。 容器中的專案會使用 Shell 資料來源公開給 Shell 命名空間。 範例包括資料夾、磁片磁碟機、網路伺服器，以及副檔名為 .zip 的壓縮檔案。 另請參閱： Shell 資料來源、資料夾、Shell 專案。

</dd> <dt>

**content**
</dt> <dd>

與可編制索引之 Shell 專案或內容來源相關聯的文字和屬性。

</dd> <dt>

**內容來源**
</dt> <dd>

可由索引子存取的專案。 內容來源可透過 URL 定址，並由通訊協定處理常式提供給索引子。 範例包括：檔案系統檔案和資料夾、microsoft Outlook 專案和資料夾、資料庫記錄和 microsoft SharePoint 儲存的專案。 您可以藉由執行 Shell 資料來源，將內容來源公開為 Shell 專案。 另請參閱： content、Shell 專案。

</dd> <dt>

**content view (內容檢視)**
</dt> <dd>

Windows 7 和更新) 版本中提供的 Windows 檔案總管 (視圖，會根據檔案的副檔名或種類關聯，顯示清單中每個專案的最相關內容。 當視窗大小減少時，內容視圖會使用調整大小的邏輯，以確保最重要的屬性仍然具有可清楚讀取的空間。 另請參閱：版面配置模式、種類、種類關聯。

</dd> <dt>

**內容視圖模式**
</dt> <dd>

請參閱： content view 的定義。

</dd> <dt>

**內容功能表**
</dt> <dd>

這個詞彙有時候是用來表示快捷方式功能表。 請參閱：快捷方式功能表的定義。

</dd> <dt>

**內容功能表處理常式**
</dt> <dd>

這個詞彙有時候是用來表示快速鍵功能表處理常式。 請參閱：快捷方式功能表處理常式的定義。

</dd> </dl>

## <a name="d"></a>D

<dl> <dt>

**資料物件處理常式**
</dt> <dd>

為數據物件提供額外剪貼簿格式的處理常式， (IDataObject 的專案) 。 資料物件用於拖放和複製/貼上案例中。

</dd> <dt>

**資料來源**
</dt> <dd>

這個詞彙有時候是用來表示資料存放區或 Shell 資料來源。 請參閱：資料存放區、Shell 資料來源的定義。

</dd> <dt>

**資料存放區**
</dt> <dd>

資料存放庫。 資料存放區可以使用 Shell 資料來源，以容器的形式公開給 Shell 程式設計模型。 您可以使用通訊協定處理常式，利用 Windows Search 系統為數據存放區中的專案編制索引。

</dd> <dt>

**桌面電腦群組合**
</dt> <dd>

一種 Windows Vista 功能，可讓個別視窗繪製至視訊記憶體中的螢幕表面，而不是直接繪製到主要顯示裝置。

</dd> <dt>

**document**
</dt> <dd>

包含文字以及可以實作為 IFilter 介面的 Shell 專案。

</dd> <dt>

**捨棄處理常式**
</dt> <dd>

可讓特定專案類型支援拖放和複製/貼上案例的處理常式。

</dd> <dt>

**放置目標**
</dt> <dd>

拖放到檔案上的資料物件。 另請參閱：資料處理程式、drop handler。

</dd> <dt>

**動態動詞**
</dt> <dd>

相依于 Shell 專案或系統狀態的動詞;專案的外觀是以狀態為基礎，而且需要執行中的程式碼判斷專案是否應該出現。 另請參閱：快捷方式功能表處理常式、靜態動詞命令、動詞。

</dd> </dl>

## <a name="e"></a>E

<dl> <dt>

**Explorer 命令**
</dt> <dd>

可以在 Windows 檔案總管視窗頂端附近顯示為按鈕的物件，以提供該視窗中專案和容器的功能。 Shell 資料來源為特定的容器專案提供 Windows 檔案總管的命令物件。 命令有時候會當做動詞使用。

</dd> </dl>

## <a name="f"></a>F

<dl> <dt>

**檔案關聯**
</dt> <dd>

請參閱：檔案類型關聯的定義。

</dd> <dt>

**檔案格式**
</dt> <dd>

儲存在具有記載格式規格之檔案中的資料格式。 範例包括 OLE e、OPC、XML、ZIP 和其他知名的檔案格式規格。 檔案類型建立者通常會使用現有的檔案格式做為新檔案類型的基礎。 檔案格式可以只是未具現化為檔案類型的定義。

</dd> <dt>

**檔案格式處理常式**
</dt> <dd>

此詞彙是檔案類型處理常式的同義字。 請參閱的定義：檔案類型處理常式。

</dd> <dt>

**副檔名**
</dt> <dd>

檔案系統專案之檔案類型的主要指標，是最後一個點後面的檔案名部分。 副檔名不能包含空格或非 ASCII 字元，而且只會套用至)  (不是資料夾的檔案。 使用不區分大小寫或地區設定的比較函式來比較副檔名。 另請參閱：檔案格式、檔案類型。

</dd> <dt>

**檔案類型**
</dt> <dd>

特定的副檔名值（例如 ".htm" 或 ".jpg"），可定義屬於相同類型且擁有一組通用關聯的檔案類別。 另請參閱：種類、檔案類型關聯。

</dd> <dt>

**檔案類型關聯**
</dt> <dd>

針對特定的副檔名，定義可以註冊處理常式和其他屬性的關聯陣列元素。 請參閱：關聯陣列、檔案類型。

</dd> <dt>

**自訂檔案類型**
</dt> <dd>

此關聯可讓 Shell 自訂 Shell 如何處理檔案類型。 檔案類型自訂包括：指定在按兩下時用來開啟檔案的應用程式、將命令新增至檔案類型的快捷方式功能表、指定自訂圖示、指定要與檔案類型相關聯的 MIME 內容類型、指定可見的類型，以及使用 [開啟檔案] 對話方塊指定與檔案類型相關聯的一或多個應用程式。 另請參閱： PerceivedType。

</dd> <dt>

**檔案類型處理常式**
</dt> <dd>

為檔案類型註冊的處理常式。 另請參閱：處理常式。

</dd> <dt>

**資料夾**
</dt> <dd>

請參閱： container 的定義。

</dd> <dt>

**完整 PIDL**
</dt> <dd>

可唯一描述相對於桌面資料夾之物件的 PIDL。

</dd> </dl>

## <a name="h"></a>H

<dl> <dt>

**處理器**
</dt> <dd>

提供 Shell 專案功能的 COM 物件。 大部分的 Shell 資料來源都會提供可延伸的系統，以便將處理常式系結至專案。 例如，檔系統資料夾使用關聯系統來查閱特定檔案類型的處理常式。 另請參閱：檔案關聯、檔案類型、檔案類型自訂。

</dd> </dl>

## <a name="i"></a>I

<dl> <dt>

**圖示處理常式**
</dt> <dd>

提供產生和快取專案圖示所需資訊的處理常式。 檔案系統資料存放區支援根據檔案類型載入專案的圖示處理常式，讓該處理常式能夠提供用於該檔案類型之所有實例的圖示。

</dd> <dt>

**提示處理常式**
</dt> <dd>

當使用者將滑鼠指標停留在使用者介面物件上方時，提供快顯文字的處理常式。

</dd> <dt>

**item**
</dt> <dd>

請參閱： Shell 專案的定義。

</dd> <dt>

**item 類別**
</dt> <dd>

請參閱：檔案類型的定義。

</dd> <dt>

**專案識別碼清單**
</dt> <dd>

一或多個 SHITEMID 結構的序列，可唯一定義相對於某個根物件的物件。

</dd> </dl>

## <a name="k"></a>K

<dl> <dt>

**種類**
</dt> <dd>

提供使用者易記種類名稱的屬性，而且可以與屬性清單和配置模式產生關聯。 種類是在 Windows Vista 中引進，以表達更適合使用者的檔案類型概念，並將其定義為多重值字串屬性 (標準字串值) ，因此您可以有「音訊」、「影片」或「連結; 檔」種類的值。 某些使用者易記的種類名稱已與屬性和配置模式相關聯。 例如，與 [>ument] 相關聯的專案與 [圖片] 和 [Kind.Doc] 相關聯的專案會顯示不同的屬性，即使它們位於相同的視圖中也一樣。 每個專案種類都可以與四種獨特的版面配置模式之一建立關聯，以定義每個專案和其配置所顯示的屬性數目。 另請參閱：種類關聯、內容視圖、版面配置模式。

</dd> </dl>

## <a name="l"></a>L

<dl> <dt>

**版面配置模式**
</dt> <dd>

顯示內容之數種相片順序的其中一種。 在 Windows 7 和更新版本中，當您註冊新的檔案類型時，可以使用內容視圖，為您的檔案類型註冊自訂屬性清單和配置模式。 您可以從四種不同的配置模式中選擇： Alpha (適用于檔搜尋結果，其中包含程式碼片段)  (、適用于程式碼片段) 的電子郵件搜尋結果、Gamma (類似于 Alpha，但有兩行配置（而不是四) ），而 Delta (則顯示許多較短的屬性，例如使用音樂和圖片) 。 另請參閱：內容視圖、種類、種類關聯。

</dd> </dl>

## <a name="m"></a>M

<dl> <dt>

**元資料處理程式**
</dt> <dd>

這個詞彙有時候是用來表示屬性處理常式。 請參閱：屬性處理常式的定義。

</dd> </dl>

## <a name="n"></a>N

<dl> <dt>

**命名空間延伸模組**
</dt> <dd>

請參閱： Shell 資料來源的定義。

</dd> </dl>

## <a name="o"></a>O

<dl> <dt>

**物件連結與嵌入資料庫 (OLE DB)**
</dt> <dd>

一組標準的介面，可提供隨處存取不同來源的資訊來源，例如檔案系統、電子郵件資料夾和資料庫。

</dd> <dt>

**OLE DB**
</dt> <dd>

請參閱：物件連結與嵌入資料庫的定義。

</dd> </dl>

## <a name="p"></a>P

<dl> <dt>

**PerceivedType**
</dt> <dd>

廣泛分類的檔案格式類型。 PerceivedType 是在 Windows XP 中引進，並支援一組有限的已知檔案類型 (範例包括影像、文字、音訊和壓縮檔案類型) 。 檔案類型（通常是公用檔案類型）也可以有察覺的型別。 例如，影像檔案類型 .bmp、.png、.jpg 和 .gif 也是認知類型（影像）。 在程式設計層，PerceivedType 是以整數表示。 由於有使用種類和 PerceivedType 的程式碼，因此檔案格式擁有者必須註冊這兩者。 例如，「全部播放」取決於 PerceivedType。 另請參閱：檔案類型。

</dd> <dt>

**預覽處理常式**
</dt> <dd>

此處理程式可快速產生唯讀、簡單的 Shell 專案查看，以顯示在 Windows 檔案總管預覽窗格中。

</dd> <dt>

**屬性處理常式**
</dt> <dd>

將儲存在檔案中的資料轉譯成可辨識之結構化架構的處理常式，可由 Windows 檔案總管、Windows Search 和其他應用程式存取。 這些系統接著可以與屬性處理常式互動，以在檔案中寫入和讀取屬性。 轉譯的資料包含詳細資料檢視、提示、詳細資料窗格、屬性頁等等。 每個屬性處理常式都與特定檔案類型相關聯，該檔案類型是由副檔名所識別。 另請參閱：屬性系統。

</dd> <dt>

**屬性工作表處理常式**
</dt> <dd>

用來建立自訂屬性工作表的處理常式，該工作表包含允許與檔案類型進行自訂互動的 UI 圖片和控制項。

</dd> <dt>

**屬性系統**
</dt> <dd>

資料定義的可延伸讀取/寫入系統，使用實作為成對名稱/值的屬性。 另請參閱：屬性處理常式、Shell 專案。

</dd> <dt>

**屬性值**
</dt> <dd>

與 Shell 專案的屬性名稱相關聯的值。 例如，「作者」、「大小」和「拍攝日期」都是屬性。 屬性值會以 PROPVARIANT 結構表示。

</dd> <dt>

**通訊協定處理常式**
</dt> <dd>

存取內容來源並為指定的通訊協定和 URL 提供 IUrlAccessor 物件的處理常式。 通訊協定處理常式可延伸 Windows Search 的功能，並可提供變更通知給索引子。 需要不同的通訊協定處理常式來編制特定類型的資料存放區索引。 若要提供合理的使用者體驗，您也必須提供資料存放區的 Shell 資料來源，以及執行您的通訊協定處理常式。 通訊協定處理常式會將資料存放區中的專案公開給索引子，而 Shell 資料來源則會將資料存放區中的專案公開給 Shell。

</dd> </dl>

## <a name="r"></a>R

<dl> <dt>

**相對 PIDL**
</dt> <dd>

相對於 [桌面] 資料夾以外的 shell 命名空間中某個根物件的 PIDL。 這通常是專案的父資料夾。

</dd> </dl>

## <a name="s"></a>S

<dl> <dt>

**Shell 資料來源**
</dt> <dd>

用來擴充 Shell 命名空間和公開資料存放區中專案的元件。 在過去，Shell 資料來源稱為 Shell 命名空間延伸模組。 另請參閱：容器、處理常式、Shell 專案。

</dd> <dt>

**Shell 延伸模組**
</dt> <dd>

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

</dd> <dt>

**Shell 擴充處理常式**
</dt> <dd>

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

</dd> <dt>

**Shell 處理常式**
</dt> <dd>

這個詞彙有時候是用來表示檔案類型處理常式。 請參閱的定義：檔案類型處理常式。

</dd> <dt>

**Shell 專案**
</dt> <dd>

單一內容片段。 某些 Shell 專案為內容來源，有些則不是。 例如，一個資料夾是一個內容來源，但是 .jpg 檔案則不是。 檔案類型處理常式會公開 Shell 專案。 在某些內容中，會使用「專案」來區別容器與 noncontainers。 另請參閱：容器、內容來源、檔案類型處理常式。

</dd> <dt>

**Shell 命名空間延伸模組**
</dt> <dd>

這個詞彙有時候是用來表示 Shell 資料來源。 請參閱： Shell 資料來源的定義。

</dd> <dt>

**快速鍵功能表**
</dt> <dd>

使用者介面，用來顯示與使用者介面專案（例如檔案或資料夾）相關聯的動詞集合。

</dd> <dt>

**快速鍵功能表處理常式**
</dt> <dd>

為專案或專案加入動詞的處理常式。 這些動詞通常會顯示在快捷方式功能表中。 另請參閱：快捷方式功能表。

</dd> <dt>

**簡單 PIDL**
</dt> <dd>

經過剖析而沒有磁片驗證的 PIDL。

</dd> <dt>

**靜態動詞**
</dt> <dd>

適用于 Shell 專案的動詞，不需要檢查項目或系統的目前狀態。 靜態動詞命令是以專案之相關專案的靜態註冊為基礎，而且不會變更。

</dd> </dl>

## <a name="t"></a>T

<dl> <dt>

**縮圖處理常式**
</dt> <dd>

提供靜態影像以表示 Shell 專案的處理常式。

</dd> <dt>

**縮圖提供者**
</dt> <dd>

這個詞彙有時候是用來表示縮圖處理常式。 請參閱：縮圖處理常式的定義。

</dd> </dl>

## <a name="v"></a>V

<dl> <dt>

**動詞**
</dt> <dd>

可由 Shell 專案呼叫的個別動作。 範例包括 [開啟] 和 [列印]。 動詞有時也稱為命令或 task。 另請參閱：動態動詞、快捷方式功能表處理常式、靜態動詞。

</dd> <dt>

**動詞處理常式**
</dt> <dd>

這個詞彙有時候是用來表示快速鍵功能表處理常式。 請參閱：快捷方式功能表處理常式的定義。

</dd> </dl>
