# <a name="choosing-an-audio-codec-microsoft-media-foundation"></a>選擇音訊編解碼器 (Microsoft 媒體基礎) 

[**Windows Media 音訊編碼器**](windowsmediaaudioencoder.md)提供三種類型的音訊編碼，如下表所示。



| 類別                         | 目的                                                    |
|----------------------------------|------------------------------------------------------------|
| Windows Media 音訊標準     | 音訊的一般用途壓縮。                      |
| Windows Media 音訊專業人員 | 壓縮多重通道和高定義音訊。       |
| Windows Media 音訊無損     | 壓縮音訊，而不會遺失任何原始資料。 |



 

標準類別提供適用于各種播放案例的一般用途音訊編碼。 例如，它可以將音訊壓縮成低位速率，以在頻寬有限的網路上進行串流處理，或在可攜式裝置上轉譯。 在尺規的另一端，它可以產生壓縮的音訊內容以提供高品質的播放。 標準編碼演算法的重點在於音樂，但也適用于其他複雜的音訊內容。 標準類別應為音訊內容的預設值。 專業和不失真類別應用來滿足特定需求。

個別的編碼器，也就是 [**Windows Media Voice 編碼器**](windowsmediaaudiovoiceencoder.md)，提供語音壓縮。
