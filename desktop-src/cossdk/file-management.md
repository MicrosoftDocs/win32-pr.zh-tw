# <a name="file-management"></a>檔案管理

為了啟用應用程式檔的傳輸，COMREPL 會在來源和目標上自動管理檔系統資料夾的集合。 這些 COMREPL 資料夾全都以% systemdir% com 複寫為根目錄 \\ \\ 。

## <a name="source-folders"></a>來源資料夾



| 資料夾                   | 目的                                                                                                                                                                                                                                                                                                                                                                                                               |
|--------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ReplicaSource<br/> | 在準備階段期間匯出的應用程式會儲存在這裡。<br/> 每次針對指定的來源電腦執行準備階段時，就會覆寫此資料夾。 此資料夾永遠不會明確刪除，因此在來源準備好之後，可以隨時進行複寫。<br/> 每個應用程式都會儲存在它自己的子資料夾中，名為 <appName> + <appID> 。<br/> |



 

## <a name="target-folders"></a>目標資料夾



| 資料夾                    | 目的                                                                                                                                                                                                                                                                                                                                      |
|---------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ReplicaNew<br/>     | 複製階段會將來源上 ReplicaSource 中的所有檔案和子資料夾複製到目標上的 ReplicaNew。 每次針對指定的目標執行複製階段時，就會刪除任何先前的 ReplicaNew 內容。<br/> 只有當複寫程式正在執行時，這個資料夾才會存在。  (參閱 ReplicaCurrent。 ) <br/>           |
| ReplicaCurrent<br/> | 目前安裝在目標上的已複寫應用程式會儲存在這裡。<br/> 當安裝階段開始時，ReplicaNew 資料夾會重新命名為 ReplicaCurrent。 如果有現有的 ReplicaCurrent 資料夾，則會重新命名為 ReplicaOld。 如果有現有的 ReplicaOld 資料夾，則會刪除其內容。<br/> |
| ReplicaOld<br/>     | 儲存在上一次複寫期間安裝的應用程式檔。 這些檔案只會儲存供備份之用。<br/>                                                                                                                                                                                                        |
