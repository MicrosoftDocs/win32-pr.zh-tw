# <a name="general-design-tips-for-using-com"></a>使用 COM + 的一般設計秘訣

設計良好的關鍵是規劃。 如果您不熟悉如何設計三層式應用程式，請參閱 [使用 UML 設計 COM + 應用程式](designing-the-com--application-using-uml.md)一節。

您如何建立組成 COM + 應用程式中介層的 COM 元件，可能會大幅影響效能、擴充性和部署的便利性。 下列主題提供設計 COM 元件時，您應該瞭解的設計考慮和執行秘訣。



| 主題                                                                   | 描述                                                                                                                |
|-------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| [擴充性設計](designing-for-scalability.md)<br/>   | 建議增加 COM + 應用程式擴充性的方式。<br/>                                             |
| [可用性設計](designing-for-availability.md)<br/> | 討論您可以用來改善在 COM + 應用程式中中介層功能可用性的服務。<br/> |
| [安全性設計](designing-for-security.md)<br/>         | 討論最有效率地設定 COM + 應用程式安全性的方式。<br/>                                      |
| [設計部署](designing-for-deployment.md)<br/>     | 討論在設計分散式應用程式時，將部署的麻煩降至最低的方式。<br/>                       |



 

此外，也請務必瞭解如何使用 [物件共用改善效能](improving-performance-with-object-pooling.md) ，以取得最有效使用物件共用的方法。
