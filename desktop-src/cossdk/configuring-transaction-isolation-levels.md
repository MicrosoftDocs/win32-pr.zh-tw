# <a name="configuring-transaction-isolation-levels"></a>設定交易隔離等級

COM + 可讓開發人員藉由允許可設定的交易隔離等級，更充分掌控應用程式。 COM + 1.5 之前的 COM + 版本一律使用交易的最高層級隔離。 雖然這個層級保證一定會保留資料的完整性，但在大型資料庫上需要執行許多交易時，可能會導致效能問題（例如超時）。 有了可設定的隔離等級，有經驗的開發人員可以增加並行，以改善效能和擴充性。

COM + 提供下列交易隔離等級。



| 層級            | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 序列化       | 在目前的交易完成之前，其他交易無法變更目前交易讀取的資料。 無法插入任何可能會影響目前交易的新資料。 這是最安全的隔離等級，而且是預設值。                                                                                                                                                                                                                                                                                                                                                    |
| 可重複讀取  | 在目前的交易完成之前，其他交易無法變更目前交易讀取的資料。 任何類型的新資料都可以在交易期間插入。                                                                                                                                                                                                                                                                                                                                                                                                                       |
| 讀取認可   | 交易無法讀取其他尚未認可的交易正在修改的資料。 這是 Microsoft SQL Server 中的預設隔離等級。                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 讀取未認可 | 交易可以讀取任何資料，即使是由另一筆交易修改也是一樣。 這是最不安全的隔離等級，但允許最高的平行存取。                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 任意              | 支援任何隔離等級。 下游元件最常使用此設定，以避免發生衝突。 這項設定很有用，因為任何下游元件的設定都必須使用等於或小於其直屬上游元件隔離等級的隔離等級。 因此，其隔離等級設定為 [任何] 的下游元件，一律會使用其直屬上游元件所使用的相同隔離等級。 如果交易中的根物件將其隔離等級設定為 [任何]，則會將其隔離等級序列化。 |



 

> [!Note]  
> 如果下游元件的設定比上游元件更高的隔離等級，並嘗試在交易中登記，則會產生錯誤結果，並中止交易。
