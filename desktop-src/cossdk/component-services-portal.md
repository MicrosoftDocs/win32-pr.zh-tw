# <a name="com-component-services"></a>COM + (元件服務) 

## <a name="purpose"></a>目的

COM + 是 Microsoft 元件物件模型的演進， (COM) 和 Microsoft Transaction Server (MTS) 。 COM + 是以 COM、MTS 和其他以 COM 為基礎的技術為基礎，建立並擴充以撰寫的應用程式。 COM + 會處理您先前必須自行程式設計的許多資源管理工作，例如執行緒配置和安全性。 COM + 也藉由提供執行緒共用、物件共用和即時物件啟用，讓您的應用程式更具擴充性。 COM + 也會藉由提供交易支援來協助保護資料的完整性，即使交易橫跨網路的多個資料庫也是如此。

## <a name="where-applicable"></a>適用時

COM + 可以用來開發適用于 Windows 的全企業、任務關鍵性、分散式應用程式。

如果您是系統管理員，您將會安裝、部署及設定 COM + 應用程式及其元件。 如果您是應用程式的程式設計人員，您將會撰寫元件，並將其與應用程式整合。 如果您是工具廠商，您將會開發或修改可在 COM + 環境中工作的工具。

## <a name="developer-audience"></a>開發人員對象

com + 主要是針對 Microsoft Visual C++ 和 Microsoft Visual Basic 開發人員所設計。

## <a name="run-time-requirements"></a>執行階段需求求

從 Windows XP 和 Windows Server 2003 開始 Windows 中包含 com + 1.5 版。 com + 1.0 版包含在 Windows 2000 中。
