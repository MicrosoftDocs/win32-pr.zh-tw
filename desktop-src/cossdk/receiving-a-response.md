# <a name="receiving-a-response"></a>接收回應

由於已排入佇列的元件是設計來以非同步方式運作，因此用戶端應用程式應該不會在等候佇列要求的回應時封鎖。 不過，這對於用戶端應用程式或用戶端電腦上的相關應用程式來說，最終會很有用。 例如，用戶端可能會想要在要求的交易順利完成時收到通知。

有許多方式可讓佇列元件以非同步方式將回應傳回給呼叫端。 例如，它可能會傳送電子郵件。 或者，伺服器也可以發佈鬆散結合的事件，讓用戶端訂閱。

用戶端從伺服器上執行的佇列元件取得回應的另一種方式，是讓用戶端將呼叫的方法傳遞給通知物件。 通知物件是在用戶端上執行之已排入佇列的元件實例。 這類通知物件可能相當簡單，只包含用來表示錯誤值的整數，或者可能相當複雜，其中包含在用戶端上回複交易所需的所有資訊。 無論是哪一種情況，呼叫端用戶端都必須在伺服器上執行佇列元件的回應時，傳遞通知物件做為輸入參數。 因為通知物件已排入佇列，所以伺服器可以在其方法上呼叫來改變其狀態，而用戶端之後可以將其讀取。 在此案例中，會在用戶端和伺服器上使用 COM + 佇列的元件服務，以允許雙向進行非同步通訊。
