# <a name="debugging-com-applications"></a>偵錯工具 COM + 應用程式

COM + 應用程式的調試技術取決於您選擇用來撰寫元件的語言。

如果您在 Microsoft Visual C++ 中撰寫程式碼，您可以在 c + + 中啟動偵錯工具，或使用遠端用戶端，透過使用 OLE remote procedure control (RPC) 和即時 (JIT) 偵錯工具來進行偵錯工具。 您一律可以使用 [元件服務] 系統管理工具，透過 [COM + 應用程式 **屬性**] 對話方塊之 [ **Advanced** ] 索引標籤上的 [com +**啟動于偵錯工具**] 設定來對元件進行偵錯工具。 如需有關以 c + + 撰寫程式碼之元件的詳細資訊，請參閱 [以 Visual C++ 撰寫的偵錯工具元件](debugging-components-written-in-visual-c--.md)。

除非您目前正在進行多執行緒處理、元件追蹤、遠端呼叫或進程隔離，否則您可以使用 Microsoft Visual Basic 環境進行調試。 [以 Visual Basic 撰寫的調試](debugging-components-written-in-visual-basic.md)程式描述 Visual Basic 環境內的偵錯工具。

[Visual Basic ide 中](debugging-in-the-visual-basic-ide.md)的主題偵錯工具，提供有關在整合式開發環境 (IDE) 中進行偵錯工具的指導方針、秘訣和程式的一般總覽。

若要查看有關偵錯工具高階進程的詳細資訊，請參閱[Visual Basic 元件的調試](debugging-compiled-visual-basic-components.md)程式。

> [!Note]  
> 針對 com +，使用 Visual Basic 環境來對具有 MTS 的元件進行偵錯工具的幾項限制已解決。 如需詳細資訊，請參閱[com + Visual Basic 與 MTS 對比的支援](com--visual-basic-debugging-support-contrasted-with-mts.md)。
