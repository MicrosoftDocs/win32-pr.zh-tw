# <a name="replication-phases"></a>複寫階段

COMREPL 會在三個階段中執行其工作。 複寫程式分成不同的階段，原因有下列兩個：

-   將完成的工作與每個目標上完成的工作進行分隔，以準備來源
-   延遲修改目標，直到從來源取得所有資料為止

複寫階段如下所示。



| 階段         | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|---------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 準備階段 | 將所有已安裝的應用程式匯出到來源電腦上的本機。 這個階段不會接觸任何目標的設定。                                                                                                                                                                                                                                                                                                                                                                                                           |
| 複製階段    | 將資料複製到目的電腦。 來源上匯出的所有應用程式都會複製到目標。 這是檔案複製作業。  (請參閱檔案 [管理](file-management.md)。 ) 此步驟也會從來源電腦取得未封裝在匯出應用程式檔內的一些資料 (例如，應用程式識別) 。 這項資料會保留在 COMREPL 內的記憶體中。 這個階段不會接觸任何目的電腦的設定。                                                                               |
| 安裝階段 | 在目的電腦上安裝來原始目錄。 當這個步驟起始時，重新設定目標所需的所有資料都位於目的檔案系統的應用程式檔內，或在 COMREPL 中保留為實例資料。 這個階段將會刪除目標 (上已安裝的所有應用程式，但在安裝從來源複製的應用程式之前，會先 [複寫) 中](what-gets-replicated.md) 所述的應用程式。 COMREPL 會使用每個目標的安裝執行緒，同時在多個目標上安裝。 |
