## <a name="database-overview"></a><span data-ttu-id="85432-105">資料庫總覽</span><span class="sxs-lookup"><span data-stu-id="85432-105">Database Overview</span></span>

<span data-ttu-id="85432-106">ESE 資料庫是索引循序存取方法， (用於儲存和取出資料的 ISAM) 。</span><span class="sxs-lookup"><span data-stu-id="85432-106">The ESE database is an indexed sequential access method (ISAM) for storing and retrieving data.</span></span> <span data-ttu-id="85432-107">ESE 資料庫會儲存在單一檔案中，並且由一或多個使用者定義資料表所組成。</span><span class="sxs-lookup"><span data-stu-id="85432-107">An ESE database is stored in a single file and consists of one or more user-defined tables.</span></span> <span data-ttu-id="85432-108">資料是以資料表中的記錄來組織，其中包含一或多個使用者定義的資料行。</span><span class="sxs-lookup"><span data-stu-id="85432-108">Data is organized in records in the table with one or more user-defined columns.</span></span> <span data-ttu-id="85432-109">所建立的索引會針對整個集合或資料表中的一部分記錄提供不同的組織。</span><span class="sxs-lookup"><span data-stu-id="85432-109">Indexes that are created provide different organization for the entire set or a subset of records in the table.</span></span> <span data-ttu-id="85432-110">使用 ESE API，應用程式可以建立資料指標，以不同順序在資料庫中流覽記錄。</span><span class="sxs-lookup"><span data-stu-id="85432-110">Using the ESE API, applications can create cursors that navigate records in the database in different sequential orders.</span></span> <span data-ttu-id="85432-111">資料表的元素定義如下：</span><span class="sxs-lookup"><span data-stu-id="85432-111">The elements of the table are defined below:</span></span>

  - <span data-ttu-id="85432-112">資料 **行**：資料行是資料表中儲存特定類型資訊的欄位。</span><span class="sxs-lookup"><span data-stu-id="85432-112">**Column**: The column is a field in the table that stores a specific type of information.</span></span> <span data-ttu-id="85432-113">您可以根據儲存在資料行中的資料類型，固定或可變長度的資料行。</span><span class="sxs-lookup"><span data-stu-id="85432-113">Columns can be fixed, or variable length, depending on the data type stored in them.</span></span> <span data-ttu-id="85432-114">某些資料行（例如標記的資料行）在 **Null** 或設定為預設值時不會使用空格，而且可以包含多個值。</span><span class="sxs-lookup"><span data-stu-id="85432-114">Some columns, such as tagged columns take no space when **NULL** or set to the default value, and can contain multiple values.</span></span>

  - <span data-ttu-id="85432-115">**記錄**：記錄是資料行值的集合，這些值具有主鍵所定義的唯一身分識別。</span><span class="sxs-lookup"><span data-stu-id="85432-115">**Records**: A record is a collection of columns values that have a unique identity as defined by the primary key.</span></span>

  - <span data-ttu-id="85432-116">**Index**：索引是索引鍵資料行的集合，可定義資料表中記錄的儲存順序。</span><span class="sxs-lookup"><span data-stu-id="85432-116">**Index**: The index is a collection of key columns that define a stored ordering of records in the table.</span></span> <span data-ttu-id="85432-117">叢集或主要索引會定義記錄儲存在資料表中的順序。</span><span class="sxs-lookup"><span data-stu-id="85432-117">The clustered, or primary, index defines the order in which the records are stored within the table.</span></span> <span data-ttu-id="85432-118">您可以定義多個索引，以便在資料表中指定不同的排序來進行記錄。</span><span class="sxs-lookup"><span data-stu-id="85432-118">Multiple indices can be defined in order to specify different orderings of traversal through records in the table.</span></span> <span data-ttu-id="85432-119">索引也可以根據簡單的準則來限制可見的記錄集，例如記錄中是否存在特定索引鍵資料行值。</span><span class="sxs-lookup"><span data-stu-id="85432-119">An index may also limit the set of records visible based on simple criteria such as the presence or absence of a particular key column value in the record.</span></span>

  - <span data-ttu-id="85432-120">**Cursor**：資料指標指出資料表中的目前記錄，並使用目前的索引導覽至資料表中的記錄。</span><span class="sxs-lookup"><span data-stu-id="85432-120">**Cursor**: The cursor indicates the current record in the table and navigates to records in the table using the current index.</span></span> <span data-ttu-id="85432-121">資料指標也包含目前已備妥之更新狀態的資訊。</span><span class="sxs-lookup"><span data-stu-id="85432-121">The cursor also contains information on the state of the currently prepared update.</span></span>

<span data-ttu-id="85432-122">您可以隨時在資料表中新增或移除資料行和索引。</span><span class="sxs-lookup"><span data-stu-id="85432-122">Columns and indices may be added-to or removed-from the table at any time.</span></span> <span data-ttu-id="85432-123">雖然可以定義多個索引，但資料表中的資料會根據 B + 樹狀結構中的主要索引定義，實際儲存並以邏輯方式進行叢集化。</span><span class="sxs-lookup"><span data-stu-id="85432-123">Although multiple indices may be defined, the data in the table is physically stored and logically clustered according to the primary index definition in a B+ tree.</span></span> <span data-ttu-id="85432-124">每個次要索引都會儲存在個別的 B + 樹狀結構中，其中只包含儲存在主表中之實際資料的邏輯指標。</span><span class="sxs-lookup"><span data-stu-id="85432-124">Each secondary index is stored in a separate B+ tree that contains only logical pointers to the actual data that is stored in the primary table.</span></span> <span data-ttu-id="85432-125">如果未定義任何索引，則資料表中的記錄會以插入的順序儲存在 B + 樹狀結構中，並稱為「連續索引」。</span><span class="sxs-lookup"><span data-stu-id="85432-125">If no index is defined, the records in the table are stored in a B+ tree in the order of insertion and are referred to as the sequential index.</span></span>

<span data-ttu-id="85432-126">這裡的圖表是一個範例，說明如何根據主要索引，將資料表的資料儲存在 B + 樹狀結構中。</span><span class="sxs-lookup"><span data-stu-id="85432-126">The diagram here is an example of how the data for the table is stored in a B+ tree according to the primary index.</span></span> <span data-ttu-id="85432-127">主要索引適用于名稱和識別碼，並為員工的辦公室編號建立次要索引。</span><span class="sxs-lookup"><span data-stu-id="85432-127">The primary index is for Name and ID, and a secondary index is created for the employee's office number.</span></span> <span data-ttu-id="85432-128">次要索引的專案會儲存在個別的 B + 樹狀結構中，其中只包含儲存在主表中的記錄指標。</span><span class="sxs-lookup"><span data-stu-id="85432-128">The entries for the secondary index are stored in a separate B+ tree that contains only pointers to the records stored in the primary table.</span></span> <span data-ttu-id="85432-129">例如，第二個數據表中的 office 編號12348與主表中的記錄3有關。</span><span class="sxs-lookup"><span data-stu-id="85432-129">For example, the office number 12348 in the secondary table is related to record 3 in the primary table.</span></span> <span data-ttu-id="85432-130">記錄3包含 office 12348 中員工的資料行值。</span><span class="sxs-lookup"><span data-stu-id="85432-130">Record 3 contains the column values for the employee in office 12348.</span></span> <span data-ttu-id="85432-131">如需詳細資訊，請參閱 [資料表主題中的索引](./indexing-in-the-table.md) 。</span><span class="sxs-lookup"><span data-stu-id="85432-131">For more information, see the [Indexing in the Table](./indexing-in-the-table.md) topic.</span></span>
